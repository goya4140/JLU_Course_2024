# 大二下课程总览

大二下的课程更偏向计算机系统主干与数学工具并行推进：一边学习机器、体系结构和操作系统，一边补足概率统计和算法分析能力。复习时可以把它们看成三条线索。

## 系统主线

微机系统、计算机系统结构和操作系统共同构成系统方向的核心。

- 微机系统更贴近硬件和接口，适合按“处理器、汇编、存储、输入输出、接口”顺序复习。
- 计算机系统结构关注性能、流水线、存储层次和并行思想，需要把硬件结构和性能指标联系起来。
- 操作系统关注资源管理和抽象机制，包括进程线程、同步互斥、内存管理、文件系统和 I/O。

推荐顺序：先复习微机系统的基础结构，再看系统结构中的性能与层次化设计，最后回到操作系统理解软件如何管理硬件资源。

## 数学与算法主线

概率论和算法分析与设计是方法论课程。

- 概率论重点掌握随机变量、分布、数字特征、大数定律、参数估计和假设检验。公式较多，建议用“概念条件 - 公式结论 - 题型应用”的方式整理。
- 算法分析与设计重点关注复杂度、递归、分治、动态规划、贪心、图算法等内容。复习时要把算法思想和题目模板对应起来。

推荐顺序：概率论先抓定义和公式使用条件，算法课先抓典型问题的建模方法，再集中做题巩固。

## 复习节奏

第一轮适合通读章节资料，建立课程地图；第二轮按照题型和知识点查缺补漏；第三轮集中处理易错题、公式条件和概念辨析。

对系统类课程，不要只背结论，要能解释机制为什么这样设计。对数学和算法类课程，不要只记公式或模板，要知道使用条件、边界情况和常见变形。

## 快速入口

- 微机系统：优先按章节阅读“复习资料”，再看对应“习题精讲”和“自学理解手册”。
- 操作系统：先看课程 README 和章节笔记，再结合题目理解调度、同步、内存等核心机制。
- 概率论：先按章节建立公式体系，再重点检查大数定律、参数估计、区间估计和假设检验。
- 计算机系统结构：结合图片和章节资料理解结构关系，注意性能计算和概念辨析。
- 算法分析与设计：按算法范式复习，并把每类题的状态定义、转移或证明思路写清楚。
