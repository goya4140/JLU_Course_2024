## 📝 简答题题库

**Q：简述向量的三种计算方式。**

*   **横向计算**：按行从左到右横向进行，数据相关和功能切换频繁，不适合并行处理。
*   **纵向计算**：按列从上到下纵向进行，减少了数据相关和功能切换，适合向量计算。
*   **纵横（分组）计算**：将长向量分为若干组，组内按纵向方式处理，组间依次执行，可适配有限的向量寄存器容量。

**Q：简述向量长度寄存器（VL）和向量屏蔽寄存器（VM）的作用。**

*   **向量长度寄存器（VL）**：控制所有向量运算的长度，将软件程序中实际的向量长度 N 与硬件层向量寄存器中的固定元素数目相适配。
*   **向量屏蔽寄存器（VM）**：当向量长度小于硬件支持的最大长度，或者在条件语句控制下只需对向量某些元素进行单独运算时使用，用于屏蔽不参与运算的元素。

**Q：简述链接技术的思想。**

*   **应用场景**：当两条指令出现“写后读”数据相关时。
*   **原理**：若它们不存在功能部件冲突和向量寄存器（源或目的）冲突，就把它们所用的功能部件头尾相接，形成一个链接（长）流水线进行流水处理。
*   **效果**：后面的指令只需等待前一个指令产生**第一个结果**即可开始，实现了计算过程的延迟重叠。

**Q：简述指令编队的思想。**

*   **定义**：由一组不包含结构冒险的向量指令组成一个编队。
*   **执行方式**：同一个编队中的所有向量指令，在硬件条件允许时可以在同一个时钟周期内一起并行执行，从而提高吞吐率。


**Q：简述分段开采技术的思想。**

*   **问题背景**：实际操作的向量长度 N 大于硬件向量寄存器的长度 n。
*   **解决方案**：将长向量 N 分成长度固定为 n 的若干段，然后循环分段处理，每一次循环只处理一个向量段。
*   **实现方式**：该技术由系统硬件和编译软件合作完成控制，对程序员是透明的。

**Q：简述向量体系结构和 GPU 体系结构的差异。**

*   **向量体系结构**：“窄而深”。指令流水线深，ALU 宽度窄，单次指令流水后能处理更多数据，从而掩盖不必要的流水线时间。
*   **GPU 体系结构**：“宽而浅”。指令流水线浅，ALU 宽度宽，流水本身比较简单，直接对更多的数据进行并行计算，同一时刻处理更多数据。

**Q：简述 GPU 和 CPU 在设计理念上的差异性。**

*   **结构分配**：CPU 被 Cache 和复杂的控制逻辑占据了大量空间；GPU 采用数量众多的计算单元和超长流水线，省去了 Cache，控制逻辑非常简单。
*   **任务导向**：CPU 核心少缓存大，重控制、低延迟，适合处理复杂的串行计算任务（指令级并行 ILP）；GPU 核心多缓存小，重计算、高带宽，适合面向多媒体的海量数据同构计算（数据级并行 SIMD）。

**Q：简述 GPU 各个层次组件间的相似性（自相似性）。**

*   **执行层次**：任务划分为网格（Grid）、线程块（Block）和线程（Thread）。每个第 $i$ 级的任务都可划分为一组 $i-1$ 级的任务，同级任务间并行执行。
*   **硬件层次**：逐层呈现相似结构，每一层组件中都包含：任务单元、调度单元、存储单元、计算单元以及互联网络或总线。

**Q：简述 GPGPU 虚拟化的思想。**

*   **核心思想**：将一个物理 GPU 的计算能力在硬件上进行时间片切片。
*   **应用效果**：每个切片虚拟成一个逻辑 GPU 分配给不同的虚拟机或用户共享，使每个用户都能满负荷使用，大幅提高 GPU 的整体资源利用率。

## 🔢 计算题专项

### 题型说明
本章计算题极其固定，专门考查**向量链接技术（Chaining）下的流水线执行时间计算**。通常给出几条存在“写后读”关系的向量指令、各功能部件的节拍数（cycle）以及向量长度，要求计算流水线的通过时间（首个结果产出时间）和全部结果产出时间。

### 解题方法与步骤

**步骤 1：画出/分析链接流水的关键路径**
找出指令之间的数据依赖关系，判断哪些部件可以并行（通常从存储器取数和并行的算术运算可以形成多条支路，汇聚到一条主链上），这也是编队的思想。

**步骤 2：计算流水线通过时间（$T_{\text{通过}}$）**
流水线建立时间（通过时间）等于关键路径上所有功能部件的时间开销之和，**注意：**每次向功能部件送数据和从功能部件存结果，通常各需要 1 拍（部分题目若特殊规定了访存自身的传输拍数则按规定，否则默认首尾各加1拍）。
$$T_{\text{通过}} = \sum (\text{各阶段部件处理时间} + \text{数据传输额外节拍})$$

**步骤 3：计算全部结果产生时间（$T_{\text{总共}}$）**
在流水线充满（第一个结果产出）后，剩余的 $(N-1)$ 个向量元素将以每个时钟周期 1 个的速度流出。
$$T_{\text{总共}} = T_{\text{通过}} + (N - 1)$$

### 课件例题精解

**【题目】**
>假设功能单元的时间开销为：浮点加减 6 拍，浮点乘法 7 拍。把向量元素送往功能部件、以及把结果存入向量寄存器需要 1 拍，从存储器取数送入功能部件需要 1 拍。向量长度为 $N$。
分析以下程序链接执行的执行时间：\
$V_3 \leftarrow \text{存储器}$ （访存取向量A，访存 6 拍）\
$V_2 \leftarrow V_0 + V_1$ （向量B和C浮点加，6 拍） \
$V_4 \leftarrow V_2 \times V_3$ （浮点乘，7 拍）

**【解】**
第 1 条指令（访存）和第 2 条指令（浮点加）不存在数据相关，可**并行**执行。第 3 条指令（浮点乘）依赖于前两者的结果，形成链接。由于访存和浮点加时间相同（均为 6 拍），流水线到达乘法部件的时间是一致的。

*   **第一步：计算通过时间**
    从访存/加法开始，到第一个结果存入 $V_4$，需要经历并行段和乘法段：
    $$T_{\text{通过}} = (1 + 6 + 1) + (1 + 7 + 1) = 17 \text{（拍）}$$
*   **第二步：计算总时间**
    生成所有 $N$ 个结果的总执行时间为：
    $$T_{\text{总共}} = 17 + (N - 1) = N + 16 \text{（拍）}$$

### 作业例题精解（⭐ 必考重点）

**【题目】**
>在向量机器上，按照链接方式执行下述 4 条向量指令（括号中给出了相应功能部件的执行时间），如果向量寄存器和功能部件之间的数据传送需要 1 拍，试求此链接流水线的通过时间是多少拍？如果向量长度为 64，则需多少拍才能得到全部结果？\
$V_0 \leftarrow \text{存储器}$ （从存储器中取数：7 拍）\
$V_2 \leftarrow V_0 + V_1$ （向量加：3 拍）\
$V_3 \leftarrow V_2 < A_3$ （按 $A_3$ 左移：4 拍）\
$V_5 \leftarrow V_3 \wedge V_4$ （向量逻辑乘：2 拍）

**【解】**
四条指令严格按“写后读”顺序形成一条长长的单向链接流水线。通过时间即为各功能流水线由空到满的时间。

*   **第一步：计算通过时间**
    访存、加法、左移、逻辑乘依次链接：
    $$T_{\text{通过}} = (7 + 1) + (1 + 3 + 1) + (1 + 4 + 1) + (1 + 2 + 1) = 23 \text{（拍）}$$
    *(注：作业标准答案中，首个访存操作直接视作 $7$ 拍处理加上 $1$ 拍写回寄存器，后续运算每次均为 读取1拍+运算+写入1拍)*

*   **第二步：计算总时间**
    当 $N = 64$ 时，流水线充满之后，后续操作数继续以流水方式执行：
    $$T_{\text{总共}} = T_{\text{通过}} + (N - 1) = 23 + (64 - 1) = 23 + 63 = 86 \text{（拍）}$$

---

### ⚠️ 易错提醒
1. **忽略数据传送节拍**：千万不要只把功能部件的节拍加起来！必须仔细看题目中关于“传送数据需要X拍”的设定。通常每次运算的框架是 `(1 + 部件时间 + 1)`。
2. **并行支路计算错误**：如果两条指令没有依赖关系，它们是**并行启动**的，不应该把时间累加，而是取两条支路中耗时较长的那一条作为后续链接的基准时间。
