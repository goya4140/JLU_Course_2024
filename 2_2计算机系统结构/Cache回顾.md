# 计算机组成原理复习：Cache（高速缓冲存储器）

## 📌 考试定位
> 本章是计算机组成原理的核心必考点。考试重点不局限于高层性能指标，而是聚焦于底层的硬件实现逻辑。核心题型包括：主存地址位数的拆分与计算（Tag/Index/Offset 位数）、Cache 实际总容量（含有效位、脏位等控制位）的计算，以及替换算法与写策略的电路原理。

---

## 🗂️ 核心知识体系

### 1. 地址映射方式 ⭐简答 🔢计算
- **直接映射**：主存块只能映射到 Cache 的唯一固定行。 
  - 地址划分：`[ 主存字块标记 Tag ] | [ Cache行号 Index ] | [ 块内地址 Offset ]`
- **全相联映射**：主存块可映射到 Cache 的任意一行。
  - 地址划分：`[ 主存字块标记 Tag ] | [ 块内地址 Offset ]`
- **组相联映射**：主存块映射到固定组的任意一行（现代 CPU 最常用）。
  - 地址划分：`[ 主存字块标记 Tag ] | [ 组号 Set Index ] | [ 块内地址 Offset ]`

### 2. Cache 内部微观结构 ⭐简答 🔢计算
- **一个完整 Cache 行的物理组成**：
  - **有效位 (Valid Bit)**：1 位，标记数据是否有效（开机默认为 0）。
  - **脏位 (Dirty Bit)**：1 位，仅在**写回法**中使用，标记数据是否被 CPU 修改过。
  - **替换控制位 (LRU 位)**：若干位，用于记录近期访问状态。
  - **标记 (Tag)**：由地址映射方式决定位数，用于比对主存块号。
  - **数据块 (Data Block)**：实际存放的数据或指令。

### 3. 替换算法 ⭐简答
- **LRU (近期最少使用)**：硬件上通过计数器实现，替换掉很久没被访问的块，命中率高。
- **FIFO (先进先出)**：替换最早调入的块，实现简单但可能出现“贝拉迪异常”。
- **RAND (随机替换)**：硬件通过随机数发生器决定，效率不可控。

### 4. 写策略与主存一致性 ⭐简答
- **写命中策略**：
  - **写直达法 (Write-through)**：同时修改 Cache 和主存，安全但慢。
  - **写回法 (Write-back)**：只改 Cache 并将“脏位”置 1，换出时才写回主存，快但逻辑复杂。
- **写不命中策略**：
  - **写分配 (Write-allocate)**：把主存块调入 Cache 后再修改（常配合写回法）。
  - **非写分配 (Not-write-allocate)**：直接修改主存，不调块（常配合写直达法）。

## 📝 核心概念辨析与简答



**Q：直接映射与组相联映射在地址结构上非常相似，其本质区别是什么？** **A：**
- **索引的物理含义不同**：直接映射的 Index 是精确到具体的“行号”；而组相联的 Index 仅精确到“组号”，在组内可以任意停放。
- **标记（Tag）的长度不同**：在相同容量下，组相联由于组数少于总行数，Index 占用的位数变少，因此 Tag 的位数比直接映射更长。
- **硬件比较器数量不同**：直接映射在查找时只需要 1 个比较器单挑；组相联（假设为 $N$ 路）需要 $N$ 个比较器并行工作，电路更复杂。

**Q：如果一个拥有 2048 行的 Cache，采用了 2048 路组相联，它会退化成什么映射方式？** **A：**
- 会退化为**全相联映射**。
- 因为 2048 行划分成 2048 路，意味着整个 Cache 只有 1 个组，主存块可以存入该大组内的任意位置，这正是全相联映射的定义。

## 🔢 计算题专项

### 题型说明
计组中的计算题具有极强的套路性，通常分为两步：第一步考察**主存地址的划分**；第二步考察 **Cache 总容量（总位数）的计算**。

### 解题通用预处理（万能公式）

1. **确定主存地址总位数**：

$$\text{总位数} = \log_2(\text{主存容量})$$

2. **确定块内地址 (Offset) 位数**：

$$\text{Offset 位数} = \log_2(\text{字块大小})$$

---

### 课件与考研真题精解

> **【经典大题场景】**
> 假设一台计算机的**主存容量为 16MB**，**Cache 容量为 64KB**，**每个字块大小为 32B**。系统按字节编址。

#### 1. 地址映射位数划分计算

**（1）若采用直接映射：**
- 块内地址 Offset = $\log_2(32\text{B}) = 5\text{位}$。
- Cache 总行数 = $\frac{64\text{KB}}{32\text{B}} = 2048 = 2^{11}$ 行，则 Index = $11\text{位}$。
- 标记 Tag = $\text{总位数} 24 - 11 - 5 = 8\text{位}$。
- **地址结构**：`[ Tag: 8位 ] | [ Index: 11位 ] | [ Offset: 5位 ]`

**（2）若采用全相联映射：**
- 块内地址 Offset = $5\text{位}$。
- 标记 Tag = $\text{总位数} 24 - 5 = 19\text{位}$。
- **地址结构**：`[ Tag: 19位 ] | [ Offset: 5位 ]`

**（3）若采用 4 路组相联映射：**
- 块内地址 Offset = $5\text{位}$。
- Cache 总行数 = 2048 行。由于是 4 行为一组，则总组数 = $\frac{2048}{4} = 512 = 2^9$ 组。则组号 Set Index = $9\text{位}$。
- 标记 Tag = $\text{总位数} 24 - 9 - 5 = 10\text{位}$。
- **地址结构**：`[ Tag: 10位 ] | [ Set Index: 9位 ] | [ Offset: 5位 ]`

---

#### 2. Cache 实际总容量计算（进阶高分大题）

**【续上题】** 若该 Cache 采用**直接映射**和**写回法（Write-back）**，求该 Cache 的实际总容量（单位：bit）。

**解题步骤：**
1. **明确单行 Cache 包含的控制位**：
   - 因为是写回法，所以必须包含：有效位（1位） + 脏位（1位）。
   - 由上步计算可知，直接映射的 Tag = $8\text{位}$。
   - 每个数据块大小 = $32\text{B} = 32 \times 8 = 256\text{位}$。

2. **计算单行 Cache 的总位数**：

$$\text{单行位数} = \text{有效位}(1) + \text{脏位}(1) + \text{Tag}(8) + \text{Data}(256) = 266\text{位}$$

3. **乘以总行数求总容量**：
   - Cache 总行数为 2048 行。

$$\text{Cache 总容量} = 2048 \times 266 = 544,768\text{ bit}$$

---

### ⚠️ 易错提醒
- **写回法 vs 写直达法**：计算 Cache 总容量时，必须审清写策略。若是**写回法**，控制位一定要加 1 位**脏位**；若是**写直达法**，则**没有脏位**。
- **字内地址单位**：注意题目给出的块大小是“字（Word）”还是“字节（Byte）”。若按字节编址且块大小为字，需要先将字转换成字节数，再取 $\log_2$。
- **组数的计算**：组相联的组数 = $\frac{\text{Cache总行数}}{\text{路数}}$，绝对不要用主存总块数去触碰路数。
