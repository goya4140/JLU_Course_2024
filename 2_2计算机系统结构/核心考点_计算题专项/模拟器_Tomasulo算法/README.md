# Tomasulo 算法可视化模拟器

这是一个面向教学和自学的 Tomasulo 算法可视化演示。仓库里保留了原来的 `tomasulo_viz.jsx` 课本例题快照版，同时新增了 `index.html` 交互版，普通用户不需要安装依赖即可编辑指令、寄存器、内存、保留站数量和执行周期。

## 快速开始

直接用浏览器打开 `index.html` 即可。

如果浏览器限制本地文件，也可以在当前目录启动一个静态服务器：

```bash
python3 -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 页面怎么用

1. 在左侧“程序”区域输入指令，每行一条。
2. 在“机器配置”中调整保留站数量、功能部件数量、各类指令执行周期、流出宽度。
3. 在“初始数据”中编辑寄存器 JSON 和内存 JSON。
4. 点击“重新开始”应用配置。
5. 点击“单步”逐周期推进，或点击“自动运行”连续执行。
6. 在右侧切换“周期 / 状态”视图，观察指令时序、保留站、Qi 表和寄存器值变化。

## 支持的指令格式

当前交互版支持常见浮点 Load/Store 和四则运算：

```asm
LD.D F2, 0(R1)
SD.D F4, 5(R1)
ADD.D F0, F4, F2
SUB.D F0, F4, F2
MUL.D F6, F4, F0
DIV.D F8, F2, F0
```

数字支持十进制和 `0x` 十六进制。行尾可以用 `#` 写注释。

## 配置示例

寄存器 JSON：

```json
{
  "R1": "0xC0",
  "F0": 5,
  "F2": 11,
  "F4": 1,
  "F6": 0,
  "F8": 0,
  "F10": 0
}
```

内存 JSON：

```json
{
  "0xC0": 4,
  "0xC3": 6,
  "0xC5": 20
}
```

## 模拟规则说明

- 默认单流出，可在页面中调整流出宽度。
- 每类指令使用对应类型的保留站：`LD.D/SD.D` 使用 Load/Store，`ADD.D/SUB.D` 使用 Add/Sub，`MUL.D/DIV.D` 使用 Mul/Div。
- 保留站数量和功能部件数量分开配置。默认每类 1 个功能部件，因此多个同类保留站会竞争执行资源；如果想演示多功能部件机器，可以把对应功能部件数调大。
- 流出时读取 Qi 表：源操作数已就绪则进入 `Vj/Vk`，未就绪则记录生产者到 `Qj/Qk`。
- 写回时通过 CDB 广播结果，等待该生产者的保留站会更新自己的 `Vj/Vk`。
- 目标寄存器通过 Qi 表做寄存器重命名，能够演示 WAW 保护。
- 当前模型为了教学清晰，默认每周期最多广播一个结果。
