# Cache 与 CPU 访存性能 专项详解

## 📌 本节定位

> 本节是整门课程最核心的计算题考区，主要考察对 Cache 性能三大指标（命中时间、失效率、失效开销）的综合运用，以及多级 Cache 和伪相联机制下的状态追踪。本文的目标是让你不再死记硬背公式，而是真正理解数据在多级存储间的流转过程。

---

## 🧱 前置知识

**在学习本算法之前，你需要理解以下概念。**

### 平均访存时间（AMAT）

**是什么：**
它是 CPU 每次向存储器要数据时，平均需要等待的时间。它由“顺手拿到数据的时间”加上“没拿到数据时的惩罚时间”组成。

**为什么需要它：**
CPU 的速度极快，而主存很慢。我们通过 Cache 掩盖了主存的慢速，但 Cache 不能保证每次都命中。为了准确评估这套“快慢搭配”的存储系统到底有多快，我们需要一个综合指标。

**关键术语速查：**

* **命中时间（Hit Time）**：在 Cache 中找到数据并送给 CPU 的时间（通常是 1 个或几个时钟周期）。
* **不命中率（Miss Rate）**：去 Cache 找数据但没找到的概率。
* **不命中开销（Miss Penalty）**：没找到数据时，去下一级更慢的存储器搬数据所付出的额外时间代价。

> 💡 **直觉理解**：就像你在书桌（Cache）上找笔。**命中时间**就是扫一眼书桌拿到笔的时间（1秒）；**不命中率**是笔不在书桌上的概率（10%）；**不命中开销**是你不得不站起来去客厅抽屉（主存）拿笔花费的时间（50秒）。

---

### CPU 停顿周期（Stall Cycles）

**是什么：**
CPU 因为等待数据从慢速存储器搬运过来，而被迫停下来什么都不干的时钟周期数。

**为什么需要它：**
程序的总执行时间不仅取决于指令本身的执行速度（理想 CPI），还要加上因为 Cache 未命中而导致的等待时间。

---

### 局部失效率与全局失效率

**是什么：**
在多级 Cache 系统中，**局部失效率**是指单看这一级 Cache，它自己没找到数据的概率；**全局失效率**则是站在 CPU 的上帝视角，数据不仅没在 L1 找到，也没在 L2 找到的终极概率。

**为什么需要它：**
评估第一级 Cache 时局部失效率就够了；但评估第二级 Cache 对整个系统的贡献时，必须用全局失效率，因为它代表了真正要去“折磨”最慢主存的请求比例。

## 💡 算法思想

**本节回答：这些复杂的 Cache 优化算法究竟在做什么？它为什么这样设计？**

### 问题背景：它解决了什么困境？

在没有高级 Cache 优化之前，处理器面临着残酷的物理定律限制：**速度与容量不可兼得**。

* 如果用直接映像 Cache（一排座位对应一个人），命中速度极快，但容易发生冲突（两个人抢一个座位），导致**失效率高**。
* 如果用多路组相联 Cache（一个包间随便坐），冲突少了，失效率低了，但每次找人要遍历整个包间，导致**命中时间长**。
此外，单层 Cache 已经无法填补 CPU 与主存之间日益扩大的性能鸿沟。

### 核心思路：设计者如何思考？

设计者决定采用“既要又要”的策略，引入了两种极为精妙的设计：

1. **伪相联 Cache（Pseudo-associative Cache）**：
表面上是直接映像（保证了极快的命中速度），但如果第一下没找到，系统不会立刻去求助慢速主存，而是“换个特定的座位再找一次”（伪命中）。这样既保持了直接映像的速度，又获得了类似两路组相联的低失效率。
2. **多级 Cache（Multi-level Cache）**：
既然一层缓冲不够，就铺两层。L1 Cache 做得极小极快，完全配合 CPU 的时钟节奏；L2 Cache 做得很大，不在乎稍微慢一点，但专门用来拦截那些被 L1 漏掉的请求，死死守住不去访问主存的底线。

### 关键数据结构

**1. 伪相联 Cache 查找表**

| 查找阶段 | 匹配位置 | 结果 | 付出代价 |
| --- | --- | --- | --- |
| 第一次查找 | 原始哈希位置 | 命中 (正常命中) | 正常命中时间 (极短) |
| 第二次查找 | 翻转最高位对应的另一位置 | 命中 (伪命中) | 正常命中时间 + 额外周期 |
| 最终判定 | 两处均无 | 不命中 (真正失效) | 不命中开销 (去下一级找) |

### 算法与前序方案的对比

| 对比维度 | 直接映像 (Direct Mapped) | 两路组相联 (2-way Set Associative) | 伪相联 (Pseudo-associative) |
| --- | --- | --- | --- |
| 命中时间 | 极短 (直接定位) | 较长 (需多路比较与选择) | 大部分极短，少部分略长 |
| 失效率 | 最高 (冲突严重) | 较低 (缓解冲突) | 较低 (接近两路组相联) |
| 硬件复杂度 | 最简单 | 较高 | 中等 |

## ⚙️ 核心流程

### 阶段名称：伪相联 Cache 访存执行流程

**触发条件：** CPU 发出一个内存访问地址。

**执行动作：**

1. 提取地址的索引位（Index），按直接映像的规则定位到 Cache 的主区（Block A）。
2. 检查 Block A 的 Tag 是否匹配：
* 如果匹配：**快速命中**，立即将数据送回 CPU，流程结束。
* 如果不匹配：进入下一步。


3. 将索引的最高位取反（跳到另一半区的备用块 Block B），检查 Block B 的 Tag：
* 如果匹配：**伪命中**，花费少量的额外周期将数据送回 CPU，流程结束。
* 如果不匹配：**真正不命中**，触发缺失处理，去主存（或 L2 Cache）搬运数据，承受完整的不命中开销。



**注意事项：**

* ⚠️ （补充理解）伪命中的概率其实就等于“使用两路组相联能命中，但使用直接映像命不中的那部分概率”。

**阶段小结：** 伪相联通过引入一次“备胎查找”，成功将昂贵的主存访问替换为廉价的内部额外周期。

---

> 📋 **流程总结速查卡**
> 访存请求到达 → [查主位置] → 若命中 → 快速返回
> ↓ [主位置未命中]
> 查备用位置 → 若命中 → 延迟返回 (伪命中额外周期)
> ↓ [备用位置未命中]
> 访问下一级存储 → 承受不命中开销 (Miss Penalty)

## 📐 例题精解

### 课件例题

**【例题 1】基础 CPU 性能与 Cache 影响**

**题目描述：**
假设使用 Alpha AXP 类似的机器。Cache不命中开销为 50 个时钟周期，当不考虑存储器停顿时，所有指令执行时间均为 2.0 个时钟周期（理想 CPI）。Cache 访问不命中率为 2%，平均每条指令访存 1.33 次。试分析 Cache 对性能的影响。

**审题要点：**
拿到这道题，首先要明确：

* 初始条件：理想 CPI = 2.0，不命中开销 = 50 周期。
* 频率变量：每条指令访存 = 1.33 次，不命中率 = 2%。
* 要追踪的信息：加入 Cache 缺失惩罚后的实际 CPI 和 CPU 执行时间。

**解题过程：**

1. 写出带 Cache 的 CPU 时间公式：

>$$\text{CPU时间} = \text{IC} \times (\text{CPI}_{\text{理想}} + \text{每条指令的平均访存次数} \times \text{不命中率} \times \text{不命中开销(周期数)}) \times \text{时钟周期时间}$$


2. 代入具体数值：

$$\text{CPU时间} = \text{IC} \times (2.0 + 1.33 \times 2\% \times 50) \times \text{时钟周期时间}$$


3. 计算括号内的实际 CPI：

$$\text{实际CPI} = 2.0 + (1.33 \times 0.02 \times 50) = 2.0 + 1.33 = 3.33$$


4. 计算无 Cache 时的极端情况作为对比：

$$\text{CPI}_{\text{无Cache}} = 2.0 + 50 \times 1.33 = 68.5$$



**关键决策点解析：**

> **为什么每条指令要乘以 1.33？**
> 因为一次指令执行不仅包括“取指令”这 1 次访存，还可能包括“Load/Store”读写数据。1.33 代表平均每条指令会触发 1.33 次 Cache 访问，这 1.33 次每一次都有 2% 的概率失效。

**结论：**
引入 Cache 失效后，实际 CPI 从 2.0 恶化到了 3.33，CPU 时间增加为原来的 1.67 倍（$3.33 / 2.0$）。但对比没有 Cache 时 68.5 的灾难级 CPI，Cache 依然是绝对必须的。

---

**【例题 2】组织结构对比：直接映像 vs 两路组相联**

**题目描述：**
理想情况 CPI 为 2.0，时钟周期为 2ns，平均每条指令访存 1.3 次。
直接映像 Cache：命中时间 1 个周期，不命中率 1.4%。
两路组相联 Cache：命中时间 1 个周期（但使 CPU 时钟周期增加到 1.10 倍），不命中率 1.0%。
两者不命中开销均为 70ns。求平均访存时间与 CPU 性能对比。

**解题过程：**

*步骤 1：计算平均访存时间 (AMAT)*
注意这里单位是时间（ns），命中时间 1 个周期即 2.0ns。

| 结构 | 计算公式 (AMAT = 命中时间 + 不命中率 $\times$ 开销) | 结果 (ns) |
| --- | --- | --- |
| 直接映像 (1路) | $2.0 + (0.014 \times 70)$ | **2.98** |
| 两路组相联 (2路) | $(2.0 \times 1.10) + (0.010 \times 70)$ | **2.90** |

*步骤 2：计算 CPU 执行时间*
公式：

>$$\text{CPU时间} = \text{IC} \times (\text{CPI}_{\text{理想}} \times \text{时钟周期} + \text{单指令访存次数} \times \text{不命中率} \times \text{不命中开销(时间)})$$

| 结构 | 计算公式 | 结果 |
| --- | --- | --- |
| 直接映像 (1路) | $\text{IC} \times [ (2.0 \times 2.0) + (1.3 \times 0.014 \times 70) ]$ | **$5.27 \times \text{IC}$** |
| 两路组相联 (2路) | $\text{IC} \times [ (2.0 \times 2.0 \times 1.10) + (1.3 \times 0.010 \times 70) ]$ | **$5.31 \times \text{IC}$** |

**关键决策点解析：**

> **为什么两路组相联 AMAT 更短，但 CPU 整体性能反而更差？**
> 因为组相联结构增加了多路比较的硬件，导致整个 CPU 的心跳（时钟周期）被拖慢了 10%。这 10% 惩罚不仅作用在访存指令上，还作用在**所有不访存的算术指令**上（乘在了理想 CPI 上），得不偿失。

**结论：**
直接映像 Cache 的平均性能比两路组相联好（快 1.01 倍）。

**【例题 3】伪相联 Cache 复杂公式推导与计算**

**题目描述：**
假设在另一个位置找到数据（伪命中）需要 2 个额外周期。失效开销均为 50 周期。
2KB 场景：1路失效率 0.098，2路失效率 0.076。
128KB 场景：1路失效率 0.010，2路失效率 0.007。
问：在直接映像、2路组相联、伪相联中，哪种速度最快？

**解题过程：**

（补充理解）推导伪相联 AMAT 公式：
伪相联本质是“拿 1 路的命中速度”+“拿 2 路的整体失效率”+“兜底中间的伪命中代价”。

1. 真正失效率：伪相联的最终失效率就等于 2 路组相联的失效率（$\text{失效率}_{\text{2路}}$）。
2. 伪命中率：1 路没命中，但 2 路命中了。所以 $\text{伪命中率} = \text{失效率}_{\text{1路}} - \text{失效率}_{\text{2路}}$。

终极推导公式：


$$\text{AMAT}_{\text{伪相联}} = \text{命中时间}_{\text{1路}} + (\text{失效率}_{\text{1路}} - \text{失效率}_{\text{2路}}) \times \text{额外周期} + \text{失效率}_{\text{2路}} \times \text{失效开销}$$

*步骤 1：2KB Cache 性能计算*

| 结构 | 计算过程 | 结果 (周期) |
| --- | --- | --- |
| 1路直接映像 | $1 + 0.098 \times 50$ | 5.90 |
| 2路组相联 | $1 + 0.076 \times 50$ | 4.90 |
| 伪相联 | $1 + (0.098 - 0.076) \times 2 + (0.076 \times 50)$ | **4.844** |

*步骤 2：128KB Cache 性能计算*

| 结构 | 计算过程 | 结果 (周期) |
| --- | --- | --- |
| 1路直接映像 | $1 + 0.010 \times 50$ | 1.50 |
| 2路组相联 | （根据课件结论，略大或等于伪相联，假定约为1.45） | 1.45 |
| 伪相联 | $1 + (0.010 - 0.007) \times 2 + (0.007 \times 50)$ | **1.356** |

**结论：**
无论是小容量 (2KB) 还是大容量 (128KB)，伪相联 Cache 都能完美融合两者的优点，实现最快的访存速度。

**【例题 4】多级 Cache 综合分析（L1 + L2）**

**题目描述：**
在 1000 次访存中，L1 不命中 40 次，L2 不命中 20 次。L1 命中时间 1 个周期，L2 命中时间 10 个周期，L2 不命中开销 100 周期。每条指令平均访存 1.5 次。
求：各种失效率、平均访存时间、每条指令的平均停顿时间。

**解题过程：**

*步骤 1：求解各种失效率*

| 指标 | 含义 | 计算逻辑 | 结果 |
| --- | --- | --- | --- |
| L1 局部/全局失效率 | L1 拦不住的概率 | $40 / 1000$ | 4% |
| L2 局部失效率 | 到达 L2 后，L2 拦不住的概率 | $20 / 40$ | **50%** |
| L2 全局失效率 | 连穿两层防线的终极概率 | $20 / 1000$ | **2%** |

*步骤 2：计算两级体系的 AMAT*


$$\text{AMAT} = \text{命中时间}_{\text{L1}} + \text{失效率}_{\text{L1}} \times (\text{命中时间}_{\text{L2}} + \text{局部失效率}_{\text{L2}} \times \text{不命中开销}_{\text{L2}})$$

$$\text{AMAT} = 1 + 4\% \times (10 + 50\% \times 100) = 1 + 0.04 \times 60 = 3.4 \text{ 周期}$$

*步骤 3：计算每条指令平均停顿时间*
停顿时间就是“实际访存时间”比“理想命中时间(1)”多出来的部分。
每次访存停顿：$3.4 - 1.0 = 2.4 \text{ 周期}$
每条指令停顿：$2.4 \times 1.5 = 3.6 \text{ 周期}$

**结论：**
系统平均访存时间为 3.4 个周期，每条指令因为访存会平均被停顿 3.6 个周期。

**【例题 5】第二级 Cache (L2) 结构设计的影响**

**题目描述：**
L2 直接映像：命中时间 10 周期，局部不命中率 25%。
L2 两路组相联：命中时间 10.1 周期，局部不命中率 20%。
主存不命中开销为 50 个时钟周期。试问哪种 L2 更好？（衡量标准为：它们提供给 L1 的“不命中开销”谁更小）。

**解题过程：**

对于 L1 来说，它的“不命中开销”就是去 L2 走一遭的总时间。

| L2 组织结构 | 提供给 L1 的不命中开销计算公式 | 结果 (周期) |
| --- | --- | --- |
| 直接映像 | $10 + 25\% \times 50$ | 22.5 |
| 两路组相联 | $10.1 + 20\% \times 50$ | **20.1** |

*(补充理解：如果硬要把 L2 命中时间取整为 10 或 11，组相联结果分别为 20.0 和 21.0，依然小于 22.5)*

**关键决策点解析：**

> **为什么在这道题里，两路组相联又变好了？**
> 在例题2中，L1 使用组相联会拉低整个 CPU 的主频；但在多级体系中，**L2 Cache 不处于 CPU 的关键时序路径上**，它稍微慢个 0.1 周期完全不影响 CPU 主频。因此，在 L2 阶段，尽一切可能降低失效率（提升相联度）是划算的。

**结论：**
对于第二级 Cache，两路组相联优于直接映像。

## ⚠️ 易错点与变体提醒

### ❌ 混淆局部失效率与全局失效率 → ✅ 明确分母对象

**常见错误：** 在计算多级 Cache 时，套用 AMAT 公式：$\text{AMAT} = \text{L1时间} + \text{L1全局失效} \times (\text{L2时间} + \text{L2全局失效} \times \text{惩罚})$，导致结果严重偏小。

**正确规则：** 在嵌套公式中，L2 括号内的惩罚概率必须是 **局部失效率（L2 未命中数 / 到达 L2 的请求数）**。只有在算“每条指令惩罚周期”并使用简化加法公式时，才可以直接用全局失效率（例题 4 的补充解法）。

### ❌ 漏乘“每条指令平均访存次数” → ✅ 区分访存视角与指令视角

**常见错误：** 算出了 AMAT = 3.4，直接认为每条指令停顿 2.4 个周期。

**正确规则：** AMAT 只是“一次内存读写”的平均耗时。指令执行不仅包含取指令，还有 Load/Store 数据。算 CPU 停顿和 CPU 时间时，务必乘以 `1.x` 这个访存频率因子（见例题 1 和例题 4）。

**变体提示：** 若考题分别给出了“指令 Cache (I-Cache)”和“数据 Cache (D-Cache)”的失效率，你需要分别计算停顿，然后加总：
$\text{停顿} = 100\% \times \text{I-Cache停顿} + \text{Load/Store比例} \times \text{D-Cache停顿}$。

## 🔗 与其他算法的关联

本章的缓存优化思想贯穿整个计算机系统架构：

* **写缓冲器（Write Buffer）与非阻塞 Cache**：这里的硬件并发思想，是后续学习动态调度（Tomasulo 算法、ROB）和流水线乱序执行的基础。
* **多级包容性（Inclusion Property）**：L1 的数据是否必然在 L2 中？这直接决定了多核 CPU 章节中 **监听协议 (Snooping) 和 目录协议 (Directory)** 的状态广播策略。
