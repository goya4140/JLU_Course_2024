## 1. 本章考试定位

第 8 章的核心链条：

```text
路径名 -> 目录项 -> FCB/inode -> 文件数据块
open 后：fd -> 打开文件表 -> inode -> 文件数据块
```

常见题型：

1. 概念题：文件、目录、FCB、inode、目录项、目录文件、打开文件表。
2. 比较题：顺序结构、链接结构、索引结构；顺序访问、随机访问。
3. 计算题：两级索引最大文件长度。
4. 流程题：`open/read/write/close/link/unlink` 改哪些表。
5. 判断题：删除文件不可能删除所在目录；`unlink` 不一定立即删文件数据。
6. 空闲管理题：位示图、空闲块链、成组链接。

本章最重要的一句话：

```text
文件名只是目录里的名字，文件实体由 FCB/inode 和数据块描述。
```

## 3. 文件、访问方式与文件组织

### 3.1 知识点速记

【必背】文件：

```text
具有符号名的、逻辑上完整的一组信息项序列。
```

【必背】文件系统：

```text
文件以及管理文件所需的数据结构和程序的总体。
```

文件系统不只是“文件集合”，还包括：

```text
目录、FCB/inode、打开文件表、空闲空间管理、保护和共享机制。
```

UNIX 三类文件：

| 类型 | 含义 |
|---|---|
| 普通文件 | 用户数据或程序 |
| 目录文件 | 保存目录项的特殊文件 |
| 特殊文件 | 设备文件，用文件式接口访问设备 |

访问方式：

| 访问方式 | 含义 | 典型适用 |
|---|---|---|
| 顺序访问 | 按文件信息排列顺序依次访问 | 日志、文本流 |
| 随机访问 | 按记录号/关键字/偏移直接访问 | 数据库、索引文件 |

文件组织：

| 层次 | 内容 |
|---|---|
| 逻辑组织 | 记录文件、流式文件 |
| 物理组织 | 顺序结构、链接结构、索引结构、Hash 结构 |

### 3.2 对应例题：文件系统概念判断

题目：

```text
判断下列说法：
A. 文件只是磁盘块的集合。
B. 文件系统只包括用户创建的文件，不包括目录和管理表。
C. UNIX 中目录本身也可以看成一种文件。
D. UNIX 中设备可以通过特殊文件提供统一访问接口。
```

答案：

```text
A 错，B 错，C 对，D 对。
```

解答过程：

1. A 错：文件是有符号名、有逻辑完整性的信息对象，不只是磁盘块。
2. B 错：文件系统还包括目录、FCB/inode、打开文件表、空闲块管理等。
3. C 对：目录文件由目录项组成。
4. D 对：UNIX 把设备抽象成特殊文件，便于统一读写和保护。

【一句话记法】

```text
用户看见文件名；系统管理 inode、目录项和磁盘块。
```

## 4. 文件物理组织：顺序、链接、索引、Hash

### 4.1 知识点速记

| 物理结构 | 核心思想 | 优点 | 缺点 |
|---|---|---|---|
| 顺序结构 | 文件占连续磁盘块 | 顺序访问快，随机定位也容易 | 文件增长困难，需要连续空间 |
| 链接结构 | 数据块分散，用指针串起来 | 增长方便，不需连续空间 | 随机访问慢，指针有开销 |
| 索引结构 | 索引块记录数据块号 | 随机访问好，不需连续空间 | 索引块占空间 |
| Hash 结构 | 用关键字映射位置 | 查找快 | 冲突处理复杂 |

【易错】顺序结构不是“只能顺序访问”，它随机定位也容易；真正的问题是扩展困难和连续空间要求。

### 4.2 对应例题：物理结构判断

题目：

```text
判断下列描述对应哪种文件物理结构：
1. 文件占用连续磁盘块，顺序读写快，但扩展困难。
2. 文件块可分散存放，每块保存下一块指针，增长方便但随机访问慢。
3. 文件有索引块，索引块保存各数据块块号，随机访问较好。
4. 根据关键字计算存放位置，查找快但要处理冲突。
```

答案：

```text
1. 顺序结构
2. 链接结构
3. 索引结构
4. Hash 结构
```

解答过程：

看到关键词直接判：

| 关键词 | 结构 |
|---|---|
| 连续磁盘块、顺序读快、扩展困难 | 顺序结构 |
| 指针串起来、增长方便、随机访问慢 | 链接结构 |
| 索引块、数据块号、随机访问好 | 索引结构 |
| 关键字、散列、冲突 | Hash 结构 |

【一句话记法】

```text
顺序要连续，链接靠指针，索引用目录页，Hash 靠关键字。
```

## 5. 索引结构与最大文件长度计算

### 5.1 知识点速记

UNIX 混合索引思想：

```text
直接地址 + 一级间接 + 二级间接 + 三级间接
```

如果一个索引块可放 256 个块号，课件中的混合索引最大可寻址数据块数可写成：

```text
10 + 256 + 256^2 + 256^3
```

【易错】混合索引和纯两级索引不同。

纯 k 级索引通用公式：

```text
每个索引块可放指针数 p = 块大小 / 块号大小
k 级索引可指向数据块数 = p^k
最大文件长度 = p^k * 块大小
```

### 5.2 对应例题：真题两级索引最大长度

题目：

```text
某系统采用两级索引文件结构，磁盘块大小 1KB，
每个磁盘块号占 4B，
则该系统的最大单个文件长度是多少？
```

答案：

```text
64MB。
```

解答过程：

第一步，算一个索引块能放多少个块号：

```text
1KB = 1024B
p = 1024B / 4B = 256
```

第二步，两级索引能指向多少个数据块：

```text
p^2 = 256^2 = 65536 个数据块
```

第三步，每个数据块 1KB：

```text
最大文件长度 = 65536 * 1KB
             = 65536KB
             = 64MB
```

【易错排雷】

1. 两级索引不是 `2 * 256`，而是 `256^2`。
2. 最大文件长度通常只算数据块容量，不把索引块本身当成文件数据。
3. `1KB` 要换成 `1024B` 来算指针数。

## 6. 目录、FCB 与 inode

### 6.1 知识点速记

目录的作用：

```text
文件名 -> 文件号/FCB/inode -> 文件数据块
```

【必背】FCB：

```text
文件控制块，是文件存在的标志，保存文件管理信息。
```

FCB 常见字段：

```text
文件名、文件号、文件主、类型、属性、长度、地址、权限、时间、共享信息。
```

概念区分：

| 概念 | 含义 |
|---|---|
| 目录项 | 目录文件中的一条记录 |
| 目录文件 | 由目录项组成的特殊文件 |
| 文件目录 | 整个目录体系 |
| FCB/inode | 文件管理信息，不是文件内容 |

UNIX 目录改进：

```text
目录项 = 文件名 + inode 号
inode = FCB 主部，保存权限、长度、地址、链接计数等
```

优点：

1. 目录项小，目录查找快。
2. 完整管理信息放 inode 区。
3. 多个目录项可以指向同一 inode，便于硬链接。

### 6.2 对应例题：目录项、FCB、inode 判断

题目：

```text
判断下列说法：
A. FCB 是文件存在的标志。
B. 目录项就是整个目录文件。
C. UNIX 目录项通常保存完整 FCB。
D. UNIX 目录项保存文件名和 inode 号，inode 保存文件管理信息。
```

答案：

```text
A 对，B 错，C 错，D 对。
```

解答过程：

1. A 对：课内明确 FCB 是文件存在的标志。
2. B 错：目录项是一条记录，目录文件由多个目录项组成。
3. C 错：UNIX 目录项通常不保存完整 FCB，而是保存文件名和 inode 号。
4. D 对：inode 才保存权限、长度、数据块地址、链接计数等 FCB 主部信息。

【一句话记法】

```text
目录项管“名字到号码”，inode 管“文件实体信息”。
```

## 7. 目录结构与路径查找

### 7.1 知识点速记

目录结构：

| 目录结构 | 特点 | 缺点/优点 |
|---|---|---|
| 单级目录 | 所有文件在一张表 | 简单，但不能重名、查找慢 |
| 二级目录 | 每个用户一个目录 | 解决不同用户重名 |
| 多级目录 | 树形目录 | 支持分类、路径、当前目录 |

路径：

| 路径 | 起点 | 例子 |
|---|---|---|
| 绝对路径 | 根目录 `/` | `/usr/users/wang/d1/f1` |
| 相对路径 | 当前目录 | `d1/f1` |

目录查找算法：

| 算法 | 特点 |
|---|---|
| 顺序查找 | 简单，大目录慢 |
| Hash 查找 | 平均快，要处理冲突 |
| 对分查找 | 要求目录项有序 |

### 7.2 对应例题：目录结构与路径

题目：

```text
判断下列说法：
A. 单级目录中所有文件名必须唯一。
B. 二级目录允许不同用户拥有相同文件名。
C. 多级目录支持绝对路径和相对路径。
D. 对分查找目录项不要求目录有序。
```

答案：

```text
A 对，B 对，C 对，D 错。
```

解答过程：

1. A 对：单级目录所有文件在一张目录表中，不能重名。
2. B 对：二级目录按用户划分，不同用户目录下可重名。
3. C 对：树形目录支持从根目录或当前目录查找。
4. D 错：对分查找的前提是目录项按关键字有序。

【易错】Hash 查找快但要处理冲突；对分查找快但要求有序。

## 8. 打开文件表与 UNIX 三表关系

### 8.1 知识点速记

为什么要 `open`：

```text
避免每次 read/write 都重新按路径查目录。
```

打开后访问链：

```text
fd -> 用户打开文件表 -> 系统打开文件表 -> FCB/inode -> 数据块
```

两类打开文件表：

| 表 | 范围 | 内容 |
|---|---|---|
| 用户打开文件表 | 每进程一张 | fd、打开方式、读写指针、系统表入口 |
| 系统打开文件表 | 全系统共享 | FCB 主部、文件号、共享计数、修改标志 |

UNIX 更具体：

```text
fd -> u_ofile -> file -> inode -> 数据块
```

三个计数：

| 计数 | 含义 |
|---|---|
| `f_count` | 有多少引用指向 file 表项 |
| `i_count` | 内存 inode 被引用次数 |
| `i_nlink` | 磁盘目录项硬链接数 |

【易错】`i_nlink` 不是打开计数，而是有多少目录项指向该 inode。

### 8.2 对应例题：打开文件表关系

题目：

```text
用户打开文件表和系统打开文件表有什么区别？
UNIX 中 fd、u_ofile、file、inode 之间是什么关系？
```

答案：

```text
用户打开文件表属于每个进程，系统打开文件表属于全系统。
UNIX 中关系为 fd -> u_ofile -> file -> inode。
```

解答过程：

用户打开文件表：

```text
每个进程一张。
fd 是该表的下标或入口标识。
表项记录本进程对文件的打开状态，如打开方式、当前位置、系统表入口。
```

系统打开文件表：

```text
全系统共享。
记录打开文件实例的共享信息、引用计数、修改标志，并指向 FCB/inode。
```

UNIX：

```text
fd
 -> u_ofile[fd]
 -> file 表项
 -> inode 表项
 -> 磁盘数据块
```

【一句话记法】

```text
fd 是进程手里的句柄，inode 是文件实体的档案。
```

## 9. 文件系统调用：open/read/write/seek/close

### 9.1 知识点速记

`open`：

```text
路径名 -> 查目录 -> 找 inode -> 检查权限 -> 建打开表项 -> 返回 fd
```

`read`：

```text
fd -> 打开文件表 -> inode -> 根据偏移找数据块 -> 读数据 -> 更新偏移
```

`write`：

```text
fd -> 打开文件表 -> inode -> 检查写权限 -> 找或分配数据块 -> 写入 -> 更新大小/时间/修改标志
```

`seek`：

```text
只改变当前读写偏移，不直接读写磁盘数据。
```

`close`：

```text
fd -> 用户打开文件表 -> 系统打开文件表；
减少引用计数，必要时写回 inode/FCB，清除表项。
```

### 9.2 对应例题：系统调用判断

题目：

```text
判断下列说法：
A. open 后，后续 read/write 通常使用文件描述符 fd，而不是路径名。
B. seek 会直接把磁盘上的文件数据读入内存。
C. write 可能导致分配新的磁盘块。
D. close 可能在引用计数归零时写回修改过的 inode/FCB。
```

答案：

```text
A 对，B 错，C 对，D 对。
```

解答过程：

1. A 对：`open` 的意义就是把路径查找结果缓存成打开文件状态和 fd。
2. B 错：`seek` 只改变偏移，不直接读写磁盘。
3. C 对：写到文件末尾或扩展文件时，可能需要新数据块。
4. D 对：关闭最后引用时，修改过的管理信息可能写回外存。

【易错】读写指针是“打开实例”的状态，不是文件永久属性。

## 10. link/unlink 与文件删除

### 10.1 知识点速记

`link(old, new)`：

```text
查旧路径，找到 inode；
在新目录中增加目录项；
新目录项指向同一个 inode；
i_nlink++；
不复制文件数据。
```

`unlink(path)`：

```text
删除 path 对应目录项；
i_nlink--；
若 i_nlink == 0，才释放数据块和 inode。
```

【易错】`unlink` 删除的是文件名/目录项，不一定立即删除文件实体。

如果文件仍被打开，实际系统还可能等打开引用也释放后才真正回收；课内重点先抓链接计数。

### 10.2 对应例题：link/unlink 流程

题目：

```text
已知文件 A 的 inode 链接计数 i_nlink=1。
执行：
link("A", "B");
unlink("A");

问：
1. link 是否复制文件数据？
2. 执行 link 后 i_nlink 为多少？
3. 执行 unlink("A") 后，文件数据是否一定被释放？
```

答案：

```text
1. 不复制。
2. link 后 i_nlink=2。
3. 不一定；此时 B 仍指向该 inode，unlink("A") 后 i_nlink=1，文件数据不能释放。
```

解答过程：

执行 `link("A","B")`：

```text
A -> inode X
B -> inode X
i_nlink: 1 -> 2
```

执行 `unlink("A")`：

```text
删除目录项 A；
i_nlink: 2 -> 1；
B 仍然指向 inode X。
```

所以文件实体仍然存在。

【一句话记法】

```text
link 增加名字，unlink 删除名字；最后一个名字没了，实体才可能释放。
```

### 10.3 对应例题：删除文件不可能做什么

题目：

```text
用户执行删除文件的过程中，操作系统不可能执行的操作是：
A. 删除此文件所在的目录
B. 删除与此文件关联的目录项
C. 删除与此文件对应的文件控制块
D. 释放与此文件关联的内存缓冲区
```

答案：

```text
A。
```

解答过程：

逐项判断：

| 选项 | 判断 | 原因 |
|---|---|---|
| A | 不可能 | 删除文件不等于删除它所在的目录 |
| B | 可能 | 删除文件的核心动作就是删除目录项 |
| C | 可能 | 链接计数归零且无引用时可释放 FCB/inode |
| D | 可能 | 系统可清理相关缓冲/cache |

【易错】题干问“不可能”，不是“不一定”。释放 FCB/缓冲区可能有条件，但删除所在目录不属于正常删除文件流程。

## 11. 文件保护、保密与安全

### 11.1 知识点速记

| 概念 | 关注点 | 典型手段 |
|---|---|---|
| 文件保护 | 防止未授权访问 | 权限、访问控制矩阵 |
| 文件保密 | 防止信息泄露 | 口令、加密 |
| 文件安全 | 防止文件被破坏 | 备份、转储 |

访问类型：

```text
Read, Write, Execute, Append, Delete, List
```

UNIX 权限：

```text
owner:  r w x
group:  r w x
others: r w x
```

权限判断顺序：

1. 如果当前用户是文件主，按 owner 判断。
2. 否则如果属于文件组，按 group 判断。
3. 否则按 others 判断。

### 11.2 对应例题：保护/保密/安全区分

题目：

```text
判断下列措施主要对应文件保护、文件保密还是文件安全：
1. 给文件设置读/写/执行权限。
2. 对文件内容加密。
3. 定期备份文件系统。
```

答案：

```text
1. 文件保护
2. 文件保密
3. 文件安全
```

解答过程：

1. 权限控制“谁能访问”，属于保护。
2. 加密防止信息泄露，属于保密。
3. 备份防止文件因损坏、误删、故障而不可恢复，属于安全。

【一句话记法】

```text
保护管权限，保密管泄露，安全管别坏。
```

## 12. 外存空闲空间管理

### 12.1 知识点速记

文件系统要记录哪些磁盘块空闲。

常见方法：

| 方法 | 思想 | 优点 | 缺点 |
|---|---|---|---|
| 空闲块表 | 记录空闲区起始块号和长度 | 适合连续分配 | 分裂合并管理复杂 |
| 空闲块链 | 把空闲块用指针串起来 | 分配单块简单 | 难找连续空间 |
| 位示图 | 一位表示一个块空闲/占用 | 紧凑，适合找连续块 | 位图本身占空间 |
| 成组链接 | 分组缓存空闲块号，组间链接 | 减少空闲链 I/O | 边界情况要掌握 |

【易错】位示图中 0/1 含义看题目规定，不能固定死记。

## 13. UNIX 成组链接

### 13.1 知识点速记

成组链接思想：

```text
把空闲块分成若干组；
超级块中缓存当前一组空闲块号；
组与组之间通过登记块链接。
```

关键字段：

| 字段 | 含义 |
|---|---|
| `s_nfree` | 当前缓存空闲块号个数 |
| `s_free[100]` | 当前缓存的一组空闲块号 |

分配规则：

```text
若 s_nfree > 1：
    直接取出 s_free[--s_nfree]

若 s_nfree == 1：
    s_free[0] 指向下一组登记块；
    读入下一组空闲块号到超级块；
    再继续分配。
```

回收规则：

```text
若 s_nfree < 100：
    s_free[s_nfree++] = 回收块号

若 s_nfree == 100：
    把当前 s_nfree 和 s_free 写入刚释放块；
    让刚释放块成为新的组链块；
    s_free[0] = 刚释放块号；
    s_nfree = 1。
```

### 13.2 对应例题：成组链接边界

题目：

```text
UNIX 成组链接管理空闲块时：
1. 分配空闲块时，若 s_nfree > 1，如何处理？
2. 分配空闲块时，若 s_nfree == 1，为什么特殊？
3. 回收空闲块时，若 s_nfree == 100，如何处理？
```

答案：

```text
1. 直接从 s_free 中取出一个块号。
2. s_free[0] 通常是下一组登记块，需要读入下一组信息。
3. 把当前组信息写入刚释放块，让刚释放块成为新的组链块，然后超级块只记录该块。
```

解答过程：

`s_nfree > 1`：

```text
说明超级块缓存中还有普通空闲块号，直接取出即可。
```

`s_nfree == 1`：

```text
只剩 s_free[0]。
这个块号用于链接下一组，不是普通意义上直接拿完就结束。
要读入该块中的下一组空闲块号信息。
```

`s_nfree == 100` 回收：

```text
当前超级块缓存已满。
把这 100 个块号写入刚释放的块；
刚释放块成为登记块；
超级块重新只记录这个登记块号，s_nfree=1。
```

【一句话记法】

```text
s_nfree==1 是分配时换组；s_nfree==100 是回收时新建一组。
```

## 14. 管道、日志结构文件系统、内存映射文件

### 14.1 管道

管道虽然属于进程通信，但课件在 UNIX 文件接口中提到。

直觉：

```text
管道像内核维护的临时文件/缓冲区；
读端 fd 和写端 fd 通过文件接口通信。
```

考试定位：

```text
体现 UNIX “一切皆文件”式接口思想。
```

### 14.2 日志结构文件系统

日志结构文件系统思想：

```text
不在磁盘各处随机修改，而是把修改像日志一样顺序追加到磁盘。
```

关键词：

```text
顺序写、segment、清理器、inode map。
```

【易错】它不是简单的“写操作日志”，而是把文件系统更新组织成日志式追加存储。

### 14.3 内存映射文件

内存映射文件：

```text
把文件的一段映射到进程虚拟地址空间；
之后像访问内存一样访问文件。
```

作用：

1. 减少频繁 `read/write` 系统调用。
2. 适合大文件随机访问。
3. 和虚拟存储机制结合。

## 15. 题型模板汇总

### 15.1 【题型】索引文件最大长度

模板：

```text
1. p = 块大小 / 块号大小。
2. k 级索引可指向数据块数 = p^k。
3. 最大文件长度 = p^k * 块大小。
```

两级索引：

```text
最大长度 = p^2 * block_size。
```

### 15.2 【题型】目录与 inode

模板：

```text
目录项：文件名 + 文件号/inode 号。
目录文件：目录项组成的特殊文件。
FCB/inode：文件管理信息，是文件存在标志。
UNIX 把完整 FCB 主部放 inode 区，使目录项变小。
```

### 15.3 【题型】打开文件表关系

模板：

```text
fd -> 用户打开文件表/u_ofile -> 系统打开文件表/file -> inode -> 数据块
```

### 15.4 【题型】link/unlink

模板：

```text
link：增加目录项，指向同一 inode，i_nlink++，不复制数据。
unlink：删除目录项，i_nlink--，为 0 时才释放文件实体。
```

### 15.5 【题型】删除文件判断

模板：

```text
删除文件会删除目录项；
可能释放 FCB/inode 和数据块；
不会删除该文件所在目录。
```

### 15.6 【题型】成组链接

模板：

```text
s_nfree > 1：直接分配一个空闲块。
s_nfree == 1：读入下一组空闲块号。
s_nfree < 100：回收块加入当前组。
s_nfree == 100：把当前组写入释放块，释放块成为新组链块。
```

## 16. 易错判断总表

| 说法 | 判断 | 理由 |
|---|---|---|
| 文件只是磁盘块集合 | 错 | 文件是有名的逻辑信息对象 |
| 文件系统只包括文件数据 | 错 | 还包括目录、FCB、打开表、空闲管理等 |
| 顺序结构只能顺序访问 | 错 | 也能随机定位，但扩展困难 |
| 链接结构随机访问快 | 错 | 需沿链查找 |
| 索引结构随机访问好 | 对 | 索引块记录数据块号 |
| 两级索引最大块数是 `2p` | 错 | 是 `p^2` |
| 索引块本身算入文件数据长度 | 错 | 通常只算数据块容量 |
| FCB 是文件存在标志 | 对 | 保存文件管理信息 |
| 目录项就是目录文件 | 错 | 目录项是一条记录 |
| UNIX 目录项保存完整 FCB | 错 | 通常保存文件名 + inode 号 |
| inode 号就是文件名 | 错 | inode 标识文件实体，文件名在目录项中 |
| 用户打开文件表每进程一张 | 对 | fd 属于进程 |
| `i_nlink` 是打开计数 | 错 | 是目录项硬链接数 |
| `link` 会复制文件数据 | 错 | 只增加目录项 |
| `unlink` 一定立即删除数据块 | 错 | 链接计数归零才释放 |
| 删除文件会删除所在目录 | 错 | 只删除目录项 |
| `seek` 会读写磁盘数据 | 错 | 只改偏移 |
| 文件保护、保密、安全完全相同 | 错 | 分别偏权限、加密、防破坏 |
| 位示图 0 一定表示空闲 | 错 | 看题目规定 |
| 成组链接中 `s_nfree==1` 是普通情况 | 错 | 分配时要读下一组 |

## 18. 本章最后压缩版

只记八句话：

```text
1. 文件系统主线：路径名 -> 目录项 -> inode/FCB -> 数据块。
2. FCB 是文件存在标志；UNIX 目录项通常是文件名 + inode 号。
3. 顺序结构连续且快但扩展难；链接结构扩展方便但随机慢；索引结构随机好但有索引开销。
4. 两级索引最大长度 = (块大小/块号大小)^2 * 块大小。
5. open 后用 fd 访问：fd -> u_ofile -> file -> inode。
6. link 增加目录项和 i_nlink，不复制数据；unlink 删除目录项，计数归零才释放实体。
7. 删除文件不会删除它所在的目录。
8. 成组链接：s_nfree==1 分配时换组，s_nfree==100 回收时建新组。
```
