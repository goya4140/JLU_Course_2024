## 1. 本章考试定位

第 2 章主要考选择题和基础判断题。题目本身通常不复杂，但选项会故意混淆：

- 进程 vs 程序
- 进程 vs 线程
- 就绪 vs 等待
- 父进程 vs 子进程
- 总进程数 vs 新创建进程数
- 用户级线程 vs 核心级线程

一句话主线：

```text
多道程序需要 OS 管理多个动态运行实体，
于是引入进程；
进程用 PCB 描述，会在运行/就绪/等待间转换；
为了减少进程开销，引入线程；
用户提交的完整任务叫作业。
```

## 3. 多道程序：提高利用率，但不是越多越好

### 3.1 知识点速记

【必背】多道程序设计的目的：提高处理机、内存、设备等系统资源利用率，提高系统吞吐量。

判断表：

| 说法 | 判断 | 理由 |
|---|---|---|
| 多道程序支持进程并发执行 | 对 | 多个程序可交替推进 |
| 多道程序需要共享资源管理 | 对 | 多个进程竞争 CPU、内存、设备 |
| 多道程序一定要求虚拟存储 | 错 | 多道不必然依赖虚拟存储 |
| 进程数越多 CPU 效率越高 | 错 | 过多会带来调度、内存、资源竞争开销 |

一句话记法：

```text
多道提高系统利用率，不保证单个程序更快，也不是越多越好。
```

### 3.2 对应例题：作业一第 4 题

题目：

```text
下列关于多道程序系统的叙述中，不正确的是____。
A. 需要实现对共享资源的管理
B. 不必支持虚拟存储管理
C. 进程数越多 CPU 效率越高
D. 支持进程的并发执行
```

答案：

```text
C
```

解答过程：

1. 多道程序系统中，多个进程同时存在，必然要管理共享 CPU、内存、设备等资源，所以 A 对。
2. 多道程序不等于虚拟存储。没有虚拟存储也可以多道，只是内存管理能力较弱，所以 B 对。
3. 多道程序能提高资源利用率，但进程数过多会增加调度开销、内存压力和资源竞争，CPU 效率不一定继续提高，所以 C 错。
4. 多道程序的基本特点就是并发执行，所以 D 对。

易错排雷：

```text
“多道提高效率”不能推出“进程越多越高效”。
```

## 4. 进程 vs 程序：动态 vs 静态

### 4.1 知识点速记

【必背】程序是静态代码，进程是程序的一次动态运行活动。

| 对比 | 程序 | 进程 |
|---|---|---|
| 性质 | 静态 | 动态 |
| 位置 | 通常是文件/代码 | 在系统中运行、等待、恢复 |
| 生命周期 | 可长期保存 | 有创建、运行、等待、终止 |
| 状态 | 无运行状态 | 有运行/就绪/等待等状态 |
| 管理信息 | 无 PCB | 有 PCB |

进程定义可背：

```text
进程是具有一定独立功能的程序关于某个数据集合的一次运行活动。
```

【易错】进程不是“完整的程序”。说进程是程序，会漏掉数据、状态、PCB、上下文和资源信息。

### 4.2 对应例题：作业一第 1 题

题目：

```text
下列几种关于进程的叙述，最不符合对操作系统中的进程的理解的是____。
A. 进程可以由程序、数据和进程控制块描述
B. 进程是程序在一个数据集合上运行的过程，它是系统进行资源分配和调度的一个独立单位
C. 线程是一种特殊的进程
D. 进程是在多程序并行环境中的完整的程序
```

答案：

```text
D
```

解答过程：

1. A 对。进程实体通常可由程序、数据和 PCB 描述。
2. B 对。它符合“程序在数据集合上的一次运行活动”这个定义。
3. C 不严谨。线程不是脱离进程的特殊进程，而是进程内部执行流；但本题问“最不符合”，C 不是最明显错误。
4. D 错得最明显。它直接把进程说成“完整的程序”，忽略了进程的动态状态、数据集合、PCB 和运行上下文。

一句话记法：

```text
程序是菜谱，进程是正在做菜的过程；考试中见到“进程=程序”要警惕。
```

### 4.3 对应例题：作业一第 9 题

题目：

```text
进程和程序的一个本质区别是____。
A. 前者分时使用 CPU，后者独占 CPU
B. 前者在一个文件中，后者在多个文件中
C. 前者存储在内存，后者存储在外存
D. 前者是动态的，后者是静态的
```

答案：

```text
D
```

解答过程：

1. A 不是本质区别。进程如何使用 CPU 取决于调度方式。
2. B 与本质无关。文件个数不是区分进程和程序的标准。
3. C 不是本质区别。程序也可能被装入内存；进程也有外存相关信息。
4. D 正确。程序是静态代码，进程是一次动态执行过程。

一句话记法：

```text
进程和程序的本质区别：动态 vs 静态。
```

## 5. PCB：操作系统认识进程的方式

### 5.1 知识点速记

【必背】PCB 是进程控制块，是进程存在的标志。没有 PCB，操作系统就无法调度、恢复、阻塞或撤销该进程。

PCB 常见信息：

| 类别 | 内容 |
|---|---|
| 标识信息 | PID、用户 ID、父进程 PID |
| 状态信息 | 运行、就绪、等待 |
| CPU 现场 | PC、寄存器、PSW |
| 调度信息 | 优先级、队列指针 |
| 资源信息 | 内存、打开文件、设备 |
| 通信信息 | 消息队列/信号等相关指针 |

【会判】“进程可以由程序、数据和 PCB 描述”是对的。

### 5.2 对应例题：嵌入作业一第 1 题

题目中的选项：

```text
A. 进程可以由程序、数据和进程控制块描述
```

判断：

```text
正确
```

解答过程：

1. 程序给出执行代码。
2. 数据给出进程运行所依赖的数据集合。
3. PCB 给出 OS 管理进程所需的状态、现场、资源和调度信息。

易错排雷：

```text
PCB 不等于程序代码；
PCB 是 OS 管理进程的“档案”，不是用户程序内容本身。
```

## 6. 进程状态：运行、就绪、等待

### 6.1 知识点速记

三种基本状态：

| 状态 | 含义 | 缺什么 |
|---|---|---|
| 运行 | 正在 CPU 上执行 | 不缺 CPU |
| 就绪 | 已具备运行条件，等 CPU | 只缺 CPU |
| 等待/阻塞 | 等事件、资源或 I/O | 缺事件/资源/I/O |

转换图：

```text
就绪 --调度--> 运行
运行 --时间片到/被剥夺--> 就绪
运行 --等待事件/I/O/资源--> 等待
等待 --事件发生/I/O完成/资源可用--> 就绪
```

状态判断口诀：

```text
等 CPU = 就绪；
等 I/O/资源/事件 = 等待；
正在 CPU 上执行 = 运行。
```

### 6.2 对应例题：作业一第 5 题

题目：

```text
下面所列进程的 3 种基本状态之间的转换不正确的是____。
A. 运行状态 -> 就绪状态
B. 就绪状态 -> 运行状态
C. 就绪状态 -> 等待状态
D. 运行状态 -> 等待状态
```

答案：

```text
C
```

解答过程：

1. A 对。时间片用完或被高优先级进程抢占，运行态可变为就绪态。
2. B 对。调度程序选中就绪进程后，就绪态可变为运行态。
3. C 错。就绪进程没有 CPU，不能执行“申请 I/O、等待资源、等待事件”等动作，所以不能直接进入等待态。
4. D 对。运行进程执行 I/O 请求、P 操作失败或等待事件时，可进入等待态。

一句话记法：

```text
只有正在运行的进程，才能主动去等待。
```

### 6.3 对应例题：作业一第 6 题

题目：

```text
当一个进程处于____状态时，称为等待状态。
A. 正等待输入一批数据
B. 正唤醒一个协作进程
C. 正等待进入内存
D. 正等待分配给它一个时间片
```

答案：

```text
A
```

解答过程：

1. A 是等待 I/O 完成，属于等待态。
2. B 是正在执行“唤醒”动作，需要 CPU，不能叫等待态。
3. C 更接近后备/挂起/调入相关状态，不是三态模型中最典型的等待态选项。
4. D 是就绪态，因为它只差 CPU 时间片。

一句话记法：

```text
等输入是等待态；等时间片是就绪态。
```

## 7. fork：总进程数 vs 新创建进程数

### 7.1 知识点速记

【会算】连续 n 个无条件 `fork()`：

```text
总进程数 = 2^n
新创建进程数 = 2^n - 1
```

关键原因：

```text
每次 fork 后，父子进程都会继续执行后面的代码。
```

### 7.2 对应例题：作业一第 2 题

题目：

```c
void main() {
  fork();
  fork();
}
```

问题：

```text
程序执行完成后，总共创建了____个进程。
A. 5
B. 4
C. 3
D. 2
```

答案：

```text
B. 4
```

解答过程：

```text
初始：1 个进程
第 1 次 fork：1 -> 2
第 2 次 fork：当前 2 个进程都执行 fork，所以 2 -> 4
```

所以程序执行完成后总进程数是 4。

易错排雷：

```text
若问“新创建多少个进程”，答案是 3；
本题按课程作业答案，问执行完成后总共有几个进程，答案是 4。
```

## 8. 父子进程关系

### 8.1 知识点速记

【必背】`fork()` 后父子进程关系：

| 判断点 | 结论 |
|---|---|
| 可否并发执行 | 可以 |
| PCB 是否相同 | 不同 |
| PID 是否相同 | 不同 |
| 是否共享虚拟地址空间 | 不能简单说共享，作业中判错 |

父子进程不是同一进程内的线程。父子进程是两个独立进程；同一进程内线程才共享进程地址空间。

### 8.2 对应例题：作业一第 7 题

题目：

```text
关于父进程与子进程的叙述中，错误的是____。
A. 父进程与子进程可以并发执行
B. 父进程与子进程有不同的进程控制块
C. 父进程与子进程共享虚拟地址空间
D. 父进程与子进程有不同的 PID
```

答案：

```text
C
```

解答过程：

1. A 对。父进程和子进程都是可调度实体，可以并发推进。
2. B 对。每个进程都有自己的 PCB。
3. C 错。父子进程不是同一进程内的线程，不能按线程那样说共享虚拟地址空间。
4. D 对。PID 用来区分不同进程，父子 PID 不同。

一句话记法：

```text
父子可并发，PID/PCB 不同；地址空间不要按线程那样说共享。
```

## 9. 线程：进程内部的轻量执行流

### 9.1 知识点速记

【必背】线程是进程内部的执行流。引入线程是为了提高并发性、降低创建/切换/通信开销。

| 维度 | 进程 | 线程 |
|---|---|---|
| 资源 | 资源分配和保护基本单位 | 共享所属进程资源 |
| 调度 | 传统系统中进程可调度 | 支持线程后，线程可作为调度基本单位 |
| 地址空间 | 进程之间通常独立 | 同一进程内线程共享 |
| 开销 | 创建/切换/通信开销较大 | 更轻量 |

线程共享与私有：

| 共享 | 私有 |
|---|---|
| 代码、数据、堆、地址空间、打开文件等进程资源 | 栈、寄存器、程序计数器、TCB |

### 9.2 对应例题：作业一第 3 题

题目：

```text
下列关于进程和线程的叙述中，正确的是____。
A. 线程是资源分配的基本单位，进程是调度的基本单位
B. 不管系统是否支持线程，进程都是资源分配的基本单位
C. 系统级线程和用户级线程的切换都需要内核的支持
D. 同一进程中的各个线程拥有各自不同的地址空间
```

答案：

```text
B
```

解答过程：

1. A 错。资源分配基本单位是进程；支持线程后，线程可作为调度单位。
2. B 对。资源归进程所有，不管系统是否支持线程，进程仍是资源分配基本单位。
3. C 错。用户级线程由用户态线程库管理，切换通常不需要内核支持。
4. D 错。同一进程中的线程共享该进程地址空间。

一句话记法：

```text
资源看进程，执行看线程；同进程线程共享地址空间。
```

### 9.3 对应例题：真题多线程系统特长

题目方向：

```text
下列描述中，哪一个并不是多线程系统的特长？
A. 利用线程并行地执行矩阵乘法运算
B. Web 服务器利用线程响应 HTTP 请求
C. 键盘驱动程序为每个正在运行的应用配备一个线程，用以响应该应用的键盘输入
D. 基于 GUI 的调试程序用不同的线程分别处理用户输入、计算和跟踪等操作
```

答案：

```text
C
```

解答过程：

1. A 是多线程特长。矩阵乘法可拆分为多个并发计算任务。
2. B 是多线程特长。Web 服务器可用多个线程处理多个请求。
3. C 不是典型多线程应用特长。它混淆了设备驱动和应用程序内部线程设计。
4. D 是多线程特长。GUI 程序常把用户输入、后台计算、跟踪等分给不同线程。

一句话记法：

```text
多线程特长看“同一应用内部的多个并发执行流”。
```

### 9.4 对应例题：真题线程正确叙述

题目方向：

```text
下面的叙述中，正确的是____。
A. 线程是比进程更小的能独立运行的基本单位，可以脱离进程独立运行
B. 引入线程可提高程序并发执行的程度，可进一步提高系统效率
C. 线程的引入增加了程序执行时的时空开销
D. 一个进程一定包含多个线程
```

答案：

```text
B
```

解答过程：

1. A 错。线程不能脱离进程独立运行。
2. B 对。线程是轻量执行流，可提高并发程度，降低部分开销。
3. C 错。引入线程通常是为了降低创建、切换和通信开销。
4. D 错。一个进程至少包含一个线程，不一定包含多个线程。

一句话记法：

```text
线程轻量、共享、不能独立；一个进程至少一个线程。
```

## 10. 用户级线程、核心级线程、混合线程

### 10.1 知识点速记

| 实现 | 谁管理 | 优点 | 缺点 |
|---|---|---|---|
| 用户级线程 | 用户态线程库 | 切换快，内核无关 | 内核不可见，一个线程阻塞可能影响整个进程 |
| 核心级线程 | 操作系统内核 | 可真正并行，一个线程阻塞不影响全部 | 切换需要内核支持，开销较大 |
| 混合线程 | 用户线程 + LWP + 核心线程 | 兼顾灵活与并行 | 实现复杂 |

### 10.2 对应例题：嵌入作业一第 3 题

题目中的选项：

```text
C. 系统级线程和用户级线程的切换都需要内核的支持
```

判断：

```text
错误
```

解答过程：

1. 核心级线程由 OS 内核管理，切换需要内核支持。
2. 用户级线程由用户态线程库管理，内核不知道这些线程存在，因此同一进程内用户线程切换通常不需要内核支持。
3. 所以“都需要内核支持”错。

一句话记法：

```text
用户级线程：内核看不见，切换快；
核心级线程：内核看得见，可并行。
```

## 11. 作业、进程、线程关系

### 11.1 知识点速记

| 概念 | 含义 |
|---|---|
| 作业 | 用户提交给系统的一项完整任务 |
| 作业步 | 作业中的一个处理步骤，如编译、链接、运行 |
| 进程 | 系统中实际运行和管理的实体 |
| 线程 | 进程内部执行流 |

关系：

```text
一个作业可包含多个作业步；
一个作业进入系统后可对应一个或多个进程；
一个进程至少包含一个线程；
不支持多线程的系统可把进程看成单线程进程。
```

### 11.2 可能考法

题目常把“作业、进程、线程”混成一个层次。判断时按这条线：

```text
用户提交任务 = 作业；
OS 调度和分配资源的运行实体 = 进程；
进程内部更轻量的执行流 = 线程。
```

## 12. 题型模板汇总

### 12.1 【题型】状态转换题

判断步骤：

1. 看源状态是否有 CPU。
2. 主动等待 I/O/资源/事件，必须从运行态发生。
3. 等待事件完成后回就绪态，不直接运行。
4. 就绪态只差 CPU，所以不能直接等待。

模板答案：

```text
就绪 -> 等待 不合法。
因为就绪进程没有占用 CPU，不能执行申请 I/O 或等待事件的动作。
等待状态通常由运行态进程主动等待事件导致。
```

### 12.2 【题型】fork 计数题

判断步骤：

1. 从 1 个初始进程开始。
2. 每次无条件 `fork()`，所有当前进程都会复制一次。
3. 连续 n 次，总数 `2^n`。
4. 问新创建数，则减去初始进程。

模板：

```text
第 k 次 fork 后，进程数翻倍。
总进程数 = 2^n，新创建进程数 = 2^n - 1。
```

### 12.3 【题型】进程/线程判断题

先问三件事：

1. 题目说的是资源，还是执行流？
2. 题目说的是同一进程内部，还是不同进程之间？
3. 题目说的是用户级线程，还是核心级线程？

模板：

```text
进程是资源分配和保护的基本单位；
线程是进程内部执行流，支持线程后可作为 CPU 调度单位；
同一进程内线程共享地址空间和进程资源，但私有栈、寄存器和 TCB。
```

### 12.4 【题型】多道程序判断题

模板：

```text
多道程序通过让多个程序同时驻留并交替推进，提高 CPU、内存、设备利用率。
它支持并发执行，并要求资源共享管理。
但进程数不是越多越好，道数过多会增加系统开销。
```

### 12.5 【题型】父子进程判断题

模板：

```text
fork 后父子进程可以并发执行；
父子进程有不同 PID 和 PCB；
子进程由父进程复制而来，但作为独立进程运行，
不能按同一进程内线程那样说共享虚拟地址空间。
```

## 13. 易错判断总表

| 易错说法 | 正误 | 改正 |
|---|---|---|
| 进程是完整的程序 | 错 | 进程是程序的一次运行活动 |
| 进程和程序本质区别是是否在内存 | 错 | 本质区别是动态/静态 |
| 进程可以由程序、数据和 PCB 描述 | 对 | 这是进程实体常见描述 |
| PCB 是进程存在的标志 | 对 | 无 PCB，OS 无法管理进程 |
| 多道程序中进程数越多 CPU 效率越高 | 错 | 道数过多增加开销 |
| 多道程序必须支持虚拟存储 | 错 | 多道不必然依赖虚拟存储 |
| 就绪态可直接转等待态 | 错 | 就绪进程没有 CPU |
| 等待输入数据是等待态 | 对 | 等 I/O |
| 等待时间片是等待态 | 错 | 是就绪态 |
| 运行态可转等待态 | 对 | 申请 I/O/资源等 |
| 两次无条件 fork 后总进程数为 4 | 对 | 每次翻倍 |
| 父子进程有不同 PID | 对 | PID 区分进程 |
| 父子进程共享虚拟地址空间 | 错 | 不是线程式共享 |
| 线程是资源分配基本单位 | 错 | 进程是资源分配单位 |
| 进程是线程内部执行流 | 错 | 线程是进程内部执行流 |
| 同一进程线程共享地址空间 | 对 | 共享进程资源 |
| 同一进程线程私有栈和寄存器 | 对 | 各自执行流需要独立现场 |
| 用户级线程切换需要内核支持 | 错 | 通常用户态完成 |
| 核心级线程可被内核调度 | 对 | 内核可见 |
| 一个进程一定包含多个线程 | 错 | 至少一个线程 |
| 线程可脱离进程独立运行 | 错 | 线程属于进程 |
