## 3. 中断系统：中断装置 + 中断处理程序

### 3.1 知识点速记

【必背】中断系统由两部分组成：

| 部分 | 性质 | 作用 |
|---|---|---|
| 中断装置 | 硬件 | 发现并响应中断 |
| 中断处理程序 | 软件，OS 程序 | 分析中断原因并处理 |

中断装置三件事：

```text
识别中断源
保存现场
引出中断处理程序
```

【易错】中断装置是硬件；中断处理程序是操作系统程序。

### 3.2 对应例题：作业二第 1 题

题目：

```text
中断处理程序属于____。
A. 用户程序
B. 可能是应用程序，也可能是操作系统程序
C. 操作系统程序
D. 既不是应用程序，也不是操作系统程序
```

答案：

```text
C
```

解答过程：

1. 中断处理程序要在内核态处理中断原因、唤醒进程、触发调度或返回现场。
2. 这些操作涉及系统资源和硬件状态，不能交给普通用户程序。
3. 因此中断处理程序属于操作系统程序。

选项排雷：

| 选项 | 判断 | 原因 |
|---|---|---|
| A | 错 | 用户程序不能处理中断核心逻辑 |
| B | 错 | 不会“可能是应用程序” |
| C | 对 | 中断处理程序是 OS 程序 |
| D | 错 | 它明确属于 OS 软件 |

一句话记法：

```text
中断装置是硬件，中断处理程序是 OS 软件。
```

## 4. 中断、异常、系统调用与外部中断

### 4.1 知识点速记

| 类型 | 来源 | 例子 | 判断关键词 |
|---|---|---|---|
| 外部中断 | CPU 外部事件 | 时钟、I/O、控制台 | 外部设备/时钟 |
| 异常/程序性中断 | 当前指令执行导致 | 除零、缺页、越界、非法指令 | 当前指令出问题 |
| 系统调用/访管 | 用户主动请求 OS | `read`, `open`, `fork` | 主动进入内核 |

【易错】课内会把这些都放在“中断系统”里讨论，但选择题问“外部中断”时，只选 CPU 外部来的事件。

### 4.2 对应例题：真题外部中断判断

题目：

```text
对如下中断事件：
Ⅰ. 时钟中断
Ⅱ. 访管中断
Ⅲ. 缺页中断
Ⅳ. 控制台中断
能引起外部中断的事件是____。
A. Ⅰ和Ⅱ
B. Ⅱ和Ⅲ
C. Ⅲ和Ⅳ
D. Ⅰ和Ⅳ
```

答案：

```text
D. Ⅰ和Ⅳ
```

解答过程：

1. 时钟中断来自外部定时装置，是外部中断。
2. 访管中断是用户程序主动请求内核服务，属于系统调用/自愿性中断。
3. 缺页中断是当前指令访问不在内存的页面导致，属于异常/程序性中断。
4. 控制台中断来自外部设备，是外部中断。

一句话记法：

```text
外部中断看“CPU 外部来的”：时钟、I/O、控制台。
访管是系统调用，缺页是异常。
```

### 4.3 对应例题：真题中断/异常/系统调用错误项

题目方向：

```text
下面关于中断、异常和系统调用的叙述中，错误的是____。
A. 中断或异常发生时，CPU 一定处于内核态
B. 每个系统调用都有对应的内核服务例程
C. 中断处理程序开始执行时，CPU 处于内核态
D. 系统添加新类型设备时，需注册相应的中断服务例程
```

答案：

```text
A
```

解答过程：

1. A 错。中断或异常发生前，CPU 可能正在用户态执行用户程序；发生后才转入内核态处理中断/异常。
2. B 对。系统调用通过内核服务例程完成。
3. C 对。中断处理程序开始执行时已经进入内核态。
4. D 对。新设备需要对应中断服务处理逻辑。

一句话记法：

```text
发生前不一定在内核态；处理程序开始执行时在内核态。
```

## 5. 中断向量

### 5.1 知识点速记

【必背】中断向量用于引出中断处理程序。按课内口径，它保存中断处理程序入口相关信息，如 PSW、PC。

常见判断：

| 说法 | 判断 |
|---|---|
| 每个中断事件有一个中断向量 | 对 |
| 中断向量存放位置由硬件规定 | 对 |
| 中断向量内容由 OS 初始化设置 | 对 |
| 中断向量保存被中断程序的完整现场 | 错，现场通常保存到系统栈或 PCB |

### 5.2 对应例题：真题中断向量错误论述

题目：

```text
关于中断向量的错误论述是____。
A. 中断向量保存中断处理程序的运行环境与入口地址(PSW, PC)。
B. 每个中断事件有一个中断向量。
C. 中断向量的存放位置是由硬件规定的。
D. 中断向量的内容是操作系统在系统初始化时设置好的。
```

答案口径：

```text
按课内深度版整理，A 容易被设为错误项；但若严格按课件“中断向量 = 处理程序 PSW/PC”口径，A 可视为正确。
```

合理应试处理：

1. B、C、D 都是课件明确结论，通常不选。
2. A 的风险在“运行环境”四个字。如果它被理解成“被中断程序的现场”，则错；如果理解成“处理程序入口 PSW/PC”，则符合课件。
3. 真题若必须选错误项，优先选 A，并在心里区分：中断向量不是保存被中断程序现场的地方。

一句话记法：

```text
中断向量管“去哪儿处理中断”；
被中断程序现场存在系统栈/PCB。
```

## 6. 调度基本公式与 dispatcher

### 6.1 知识点速记

调度回答：

```text
CPU 接下来给哪个就绪进程？
```

dispatcher 做：

```text
保存当前进程现场
选择上升进程
恢复新进程现场
转入新进程运行
```

计算公式：

```text
周转时间 = 完成时间 - 到达时间
等待时间 = 周转时间 - 服务时间
带权周转时间 = 周转时间 / 服务时间
```

【易错】等待时间通常指在就绪队列中等待 CPU 的时间；调度大题中常用 `周转 - 服务` 算最稳。

## 7. SJF：短作业优先

### 7.1 知识点速记

SJF 规则：

```text
选择服务时间最短的作业/进程优先运行。
```

若三个作业同时到达，且：

```text
T1 < T2 < T3
```

SJF 顺序：

```text
J1 -> J2 -> J3
```

周转时间：

```text
J1: T1
J2: T1 + T2
J3: T1 + T2 + T3
```

平均周转时间：

```text
(T1 + (T1+T2) + (T1+T2+T3)) / 3
= (3T1 + 2T2 + T3) / 3
```

### 7.2 对应例题：作业二第 2 题

题目：

```text
现有 3 个同时到达的作业 J1、J2、J3，
它们的执行时间分别是 T1、T2 和 T3，且 T1<T2<T3。
若系统按单道方式运行且采用短作业优先调度算法，
则平均周转时间是____。
A. T1+T2+T3
B. (3T1+2T2+T3)/3
C. (T1+T2+T3)/3
D. (T1+2T2+3T3)/3
```

答案：

```text
B
```

解答过程：

1. 三个作业同时到达，SJF 先运行最短的 J1，再 J2，再 J3。
2. J1 完成时刻 `T1`，周转时间 `T1`。
3. J2 完成时刻 `T1+T2`，周转时间 `T1+T2`。
4. J3 完成时刻 `T1+T2+T3`，周转时间 `T1+T2+T3`。
5. 求平均：

```text
[T1 + (T1+T2) + (T1+T2+T3)] / 3
= (3T1 + 2T2 + T3) / 3
```

一句话记法：

```text
同时到达的 SJF：完成时间逐个累加，平均周转就是累加和再平均。
```

## 8. HRN 与 FCFS 选择题

### 8.1 HRN 知识点

HRN = 最高响应比优先。

公式：

```text
响应比 = (等待时间 + 服务时间) / 服务时间
      = 1 + 等待时间 / 服务时间
```

特点：

- 同时考虑等待时间和执行时间。
- 短作业服务时间小，响应比容易高。
- 长作业等待久后响应比也会升高，不容易一直饥饿。

### 8.2 对应例题：作业二第 3 题

题目：

```text
既考虑作业等待时间，又考虑作业执行时间的调度算法是____。
A. 最高响应比优先调度算法
B. 短作业优先调度算法
C. 优先级调度算法
D. 先来先服务调度算法
```

答案：

```text
A
```

解答过程：

1. SJF 主要看服务时间。
2. FCFS 主要看到达先后。
3. 优先级调度看优先级。
4. HRN 的响应比公式中同时含等待时间和服务时间，所以选 A。

一句话记法：

```text
看到“等待时间 + 执行时间”，立刻想到 HRN。
```

### 8.3 FCFS 知识点

FCFS = 先来先服务。

特点：

- 简单公平。
- 不抢占。
- 若长进程先到，短进程可能等很久。

### 8.4 对应例题：作业二第 4 题

题目：

```text
在进程调度算法中，对短进程不利的是____。
A. 短进程优先调度算法
B. 先来先服务调度算法
C. 高响应比优先调度算法
D. 多级反馈队列调度算法
```

答案：

```text
B
```

解答过程：

1. 短进程优先显然照顾短进程。
2. FCFS 如果长进程先到，短进程只能排队等待，短进程很吃亏。
3. HRN 会让短进程响应比较高，也会随着等待提升长进程响应比。
4. 多级反馈队列通常有利于短交互进程。

一句话记法：

```text
FCFS 怕“长作业堵门”，所以对短进程不利。
```

## 9. SRTF/SRTN：最短剩余时间优先

### 9.1 知识点速记

SRTF/SRTN 是抢占式 SJF：

```text
每当新进程到达或当前进程完成时，
从就绪进程中选择剩余运行时间最短者运行。
```

【易错】不要只在进程完成时比较；新进程到达时也要比较剩余时间。

### 9.2 对应例题：作业二简答题 1

题目：

```text
某系统按最短剩余时间（Shortest Remaining Time First）优先算法调度 CPU。
已知四个进程 P1、P2、P3、P4 的到达时间和 CPU 阵发时间如下：

| Process | Arrival time | Burst time |
|---------|--------------|------------|
| P1      | 0            | 10         |
| P2      | 3            | 6          |
| P3      | 4            | 3          |
| P4      | 7            | 4          |

问题：
⑴ 画出按 SRTF 算法得到的 Gantt 图；
⑵ 计算平均等待时间、平均周转时间、平均带权周转时间。
```

答案：

```text
Gantt 图：
0-3 P1 | 3-4 P2 | 4-7 P3 | 7-11 P4 | 11-16 P2 | 16-23 P1
```

解答过程：

1. `t=0`：只有 P1 到达，运行 P1。P1 运行到 `t=3`，剩余 `7`。
2. `t=3`：P2 到达，P1 剩余 `7`，P2 需要 `6`，P2 更短，抢占 P1。P2 运行到 `t=4`，剩余 `5`。
3. `t=4`：P3 到达，P1 剩余 `7`，P2 剩余 `5`，P3 需要 `3`，运行 P3。
4. `t=7`：P3 完成，P4 到达。此时 P1 剩 `7`，P2 剩 `5`，P4 需 `4`，运行 P4。
5. `t=11`：P4 完成。P1 剩 `7`，P2 剩 `5`，运行 P2。
6. `t=16`：P2 完成，最后运行 P1 到 `t=23`。

完成时间：

| 进程 | 到达 | 服务 | 完成 |
|---|---:|---:|---:|
| P1 | 0 | 10 | 23 |
| P2 | 3 | 6 | 16 |
| P3 | 4 | 3 | 7 |
| P4 | 7 | 4 | 11 |

周转时间：

```text
P1 = 23 - 0 = 23
P2 = 16 - 3 = 13
P3 = 7 - 4 = 3
P4 = 11 - 7 = 4
平均周转时间 = (23 + 13 + 3 + 4) / 4 = 10.75
```

等待时间：

```text
P1 = 23 - 10 = 13
P2 = 13 - 6 = 7
P3 = 3 - 3 = 0
P4 = 4 - 4 = 0
平均等待时间 = (13 + 7 + 0 + 0) / 4 = 5
```

带权周转时间：

```text
P1 = 23 / 10 = 2.30
P2 = 13 / 6 ≈ 2.17
P3 = 3 / 3 = 1.00
P4 = 4 / 4 = 1.00
平均带权周转时间 ≈ (2.30 + 2.17 + 1.00 + 1.00) / 4 ≈ 1.62
```

一句话记法：

```text
SRTF 每逢新进程到达，就比较“剩余时间”，不是比较总服务时间。
```

## 10. EDF：最早截止期优先

### 10.1 知识点速记

EDF = Earliest Deadline First。

规则：

```text
总是运行绝对截止期最早的就绪任务。
```

周期实时任务常用可调度性判断：

```text
利用率 U = sum(Ci / Ti)
若 U <= 1，则 EDF 可调度
```

其中：

- `Ci`：任务每次需要处理时间。
- `Ti`：任务周期，也常等于相对截止期。

### 10.2 对应例题：作业二简答题 2

题目：

```text
设有周期性实时任务集合如下表所示，用 EDF 算法是否可以调度？
画出前 100 个时间单位的 Gantt 图。

| 任务 | 发生周期 Ti | 处理时间 Ci |
|------|-------------|-------------|
| A    | 30          | 10          |
| B    | 40          | 15          |
| C    | 50          | 5           |
```

第一步：计算利用率。

```text
U = 10/30 + 15/40 + 5/50
  = 1/3 + 3/8 + 1/10
  = 40/120 + 45/120 + 12/120
  = 97/120
  ≈ 0.808 < 1
```

结论：

```text
EDF 可以调度。
```

第二步：列释放与截止期。

| 任务 | 释放时刻 | 绝对截止期 |
|---|---|---|
| A1 | 0 | 30 |
| B1 | 0 | 40 |
| C1 | 0 | 50 |
| A2 | 30 | 60 |
| B2 | 40 | 80 |
| C2 | 50 | 100 |
| A3 | 60 | 90 |
| B3 | 80 | 120 |
| A4 | 90 | 120 |

一种前 100 时间单位的 EDF Gantt 图：

```text
0-10   A1
10-25  B1
25-30  C1
30-40  A2
40-55  B2
55-60  C2
60-70  A3
70-80  idle
80-95  B3
95-100 A4
```

说明：

1. `0` 时 A1、B1、C1 同时释放，截止期分别 30、40、50，所以 A1 -> B1 -> C1。
2. `30` 时 A2 释放，截止期 60，优先于后续任务。
3. `40` 时 B2 释放，截止期 80；`50` 时 C2 释放，截止期 100，所以 B2 先于 C2。
4. `60` 时 A3 释放，截止期 90。
5. `70-80` 没有就绪任务，空闲。
6. `80` 时 B3 释放，截止期 120；`90` 时 A4 释放，截止期 120，截止期相同可按先来或题目约定处理，这里先继续 B3，再 A4。

一句话记法：

```text
EDF 两步：先算利用率，再按绝对截止期最早者运行。
```

## 11. 调度算法对比速记

| 算法 | 规则 | 考点 |
|---|---|---|
| FCFS | 先到先服务 | 对短进程不利 |
| SJF | 服务时间最短优先 | 同时到达时平均等待/周转较小 |
| SRTF/SRTN | 剩余时间最短优先 | 抢占式，新到达要比较 |
| HRN | 响应比最高优先 | 同时考虑等待和服务时间 |
| RR | 时间片轮转 | 时间片太短开销大，太长像 FCFS |
| EDF | 截止期最早优先 | 实时任务，`sum(Ci/Ti)<=1` |
| RMS | 周期越短优先级越高 | 比 EDF 更保守 |

## 12. 题型模板汇总

### 12.1 【题型】中断选择题

判断步骤：

1. 问“中断处理程序属于什么”：选操作系统程序。
2. 问“中断装置做什么”：识别、保存、引出。
3. 问“外部中断”：只选 CPU 外部事件，如时钟、I/O、控制台。
4. 问“发生时/处理时是否内核态”：发生前不一定，处理程序开始时是。
5. 问“中断向量”：管处理程序入口，不是保存被中断程序完整现场。

### 12.2 【题型】SRTF 调度题

模板：

```text
1. 按到达时刻排序。
2. 当前运行进程的剩余时间随运行减少。
3. 每逢新进程到达，比较所有就绪进程剩余时间。
4. 剩余时间更短者抢占。
5. 画 Gantt 图。
6. 完成时间 -> 周转时间 -> 等待时间 -> 带权周转时间。
```

### 12.3 【题型】EDF 实时调度题

模板：

```text
1. 计算 U = sum(Ci/Ti)。
2. 若 U <= 1，EDF 可调度。
3. 列任务释放时刻和绝对截止期。
4. 每个调度点选择绝对截止期最早的就绪任务。
5. 画指定时间范围 Gantt 图。
```

### 12.4 【题型】SJF 三作业平均周转

模板：

```text
T1<T2<T3 且同时到达：
顺序 J1,J2,J3
平均周转 = [T1 + (T1+T2) + (T1+T2+T3)] / 3
         = (3T1 + 2T2 + T3) / 3
```

## 13. 易错判断总表

| 易错说法 | 正误 | 改正 |
|---|---|---|
| 中断装置是软件 | 错 | 中断装置是硬件 |
| 中断处理程序属于用户程序 | 错 | 属于操作系统程序 |
| 中断装置负责识别、保存、引出 | 对 | 习题课重点 |
| 时钟中断是外部中断 | 对 | 来自外部时钟装置 |
| 访管中断是外部中断 | 错 | 是系统调用/自愿性中断 |
| 缺页中断是外部中断 | 错 | 是异常/程序性中断 |
| 中断发生前 CPU 一定处于内核态 | 错 | 可能正在用户态 |
| 中断处理程序开始执行时 CPU 处于内核态 | 对 | 已进入内核处理 |
| 中断向量保存被中断程序完整现场 | 错 | 现场通常在系统栈/PCB |
| FCFS 对短进程不利 | 对 | 长进程先来会堵住短进程 |
| HRN 只考虑服务时间 | 错 | 同时考虑等待和服务 |
| SRTF 是非抢占式 | 错 | SRTF 是抢占式 |
| SRTF 新进程到达不必比较 | 错 | 必须比较剩余时间 |
| EDF 只看周期短者优先 | 错 | EDF 看绝对截止期 |
| `sum(Ci/Ti)<=1` 可用于 EDF 可调度判断 | 对 | 课内重要条件 |
