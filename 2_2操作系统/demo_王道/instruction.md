# 操作系统复习资料生成提示词

## 使用方式
新会话开始时，上传王道对应小节的PDF（以及该小节的视频笔记脑图截图，如果有的话），然后将本提示词完整粘贴。两者同时上传时，以PDF为内容深度来源，以脑图为知识点覆盖完整性的核对标准。

---

## 提示词正文

我正在为操作系统408考研科目生成复习资料，来源**仅使用王道2026版《操作系统考研复习指导》**。请根据我上传的王道PDF内容，生成对应小节的Markdown复习笔记。

### 知识点重要程度分级（核心判断标准）

生成笔记前，先对该小节每个知识点按以下三级分类，再决定展开深度：

#### 🔴 A级：理解+计算（最高优先级）
需要真正理解原理并能做题计算，不能只靠背结论。

**判断标准——满足以下任一条件即为A级：**
- 王道原书标注了命题追踪（有历年统考题）
- 存在配套例题或计算题（如调度算法甘特图、页面置换计算、银行家算法、PV操作分析）
- 概念之间有"为什么"的因果链需要推导（如为什么阻塞→运行不可能发生）
- 涉及"对比多种方案的优缺点"（如ULT vs KLT，需要理解权衡而非死记）

**写法要求：**
- 表格对比必须覆盖所有维度，不能省略
- 有"不可能/不正确"的情况必须显式标出并解释原因
- 原理性推导要写出来（"为什么"比"是什么"更重要）
- 配 `[!IMPORTANT]` 或 `[!WARNING]` 提示框

#### 🟡 B级：背记+理解（中等优先级）
需要记住结论，同时要有足够理解才能在选择题中辨别干扰项。

**判断标准：**
- 概念定义、特征列举（如进程五大特征）
- 流程步骤（如原语的执行过程）
- 分类框架（如进程通信的四种方式）

**写法要求：**
- 定义用一句话准确表述，不展开背景推导
- 流程用①②③有序列表，步骤完整，不省略
- 分类用表格，"是什么/特点"列必须有，原理列可以简化
- 配 `[!NOTE]` 提示框标注记忆要点

#### 🟢 C级：了解即可（最低优先级）
王道原书明确标注"了解"/"理解即可"，或历年从未单独命题的背景性内容。

**判断标准：**
- 王道原书原文写了"通常不会直接考查"
- 纯粹的历史背景、发展沿革
- 实现细节过于底层（如具体的线程库API名称）

**写法要求：**
- 一句话带过，或直接在括号里补充
- 不开独立小节
- 不加提示框

### 单个文件的内部结构

#### 1. 文件头

```markdown
# X.X [王道小节标题]

> 本节定位：[说清楚这节讲什么，哪些内容放在其他节]
> 来源：王道2026版《操作系统考研复习指导》X.X节。
```

#### 2. 知识框架（必须有）

用纯文本ASCII树状大纲，**不使用Mermaid flowchart/graph**。

```
X.X [小节标题]
│
├── 第一层：[回答"XXX"的问题]   ← 层标题必须是"回答某个具体问题"的形式
│   ├── 子节点1
│   └── 子节点2
│
└── 第N层：...
```

框架后附一句说明，解释各层之间的递进逻辑。

#### 3. 正文各层展开

标题层级：`## 第一层` → `### 1.1 / 1.2 ...`

| 内容类型 | 写法 |
|---|---|
| 对比类（A/B级） | 表格，覆盖所有对比维度 |
| 流程步骤（B级） | 有序列表①②③，步骤完整 |
| 状态转换图 | 允许使用Mermaid stateDiagram-v2（唯一允许的可视化） |
| 分类列举（B级） | 表格，"方式/核心机制/特点"三列 |
| C级内容 | 正文一句话带过，不单独开小节 |

#### 4. Callout提示框

- `[!IMPORTANT]`：A级核心结论，选择题必考
- `[!WARNING]`：易错陷阱，"不可能发生的情况"、容易混淆的概念
- `[!NOTE]`：B级记忆要点，或王道标注"理解即可"的提示
- `[!TIP]`：跨节联系，帮助串联前后知识点

#### 5. 命题追踪

嵌入对应callout框内，格式：`命题追踪：[考查内容简述]（XXXX、XXXX年统考）`

不单独开"命题追踪"章节。

#### 6. 本节考点速记（必须有）

```markdown
---

## 本节考点速记

- [A级结论，直接可背诵的句子]
- [B级结论，关键概念的一句话定义]
- ...
```

6-8条，每条一句话，只写结论。A级考点优先列出。

### 内容完整性要求

**生成前必须执行：**

1. **先列清单**：将PDF该小节的每一个叶子节点列出，同时标注判断的重要程度（A/B/C）
2. **逐项核对**：确认每一项都在笔记中有对应覆盖
3. **再生成正文**

如果同时上传了视频笔记脑图，以脑图作为叶子节点的完整性标准（脑图通常比PDF更穷举）。

**特别容易漏掉的高优先级内容（A/B级）：**
- "不可能/不正确"的情况（如状态转换中的❌）——A级陷阱，必须显式写出并解释
- 概念的细分小类（如共享存储分低级/高级两种，特点不同）
- 补充性小结论（如"Block与Wakeup必须成对使用"）
- 私有/共享资源的具体列举（如线程私有PC/寄存器/栈，共享代码段/数据段/堆）

### 禁止事项

- ❌ 不引用任何校内课件内容，只用王道
- ❌ 不使用Mermaid flowchart/graph（只允许stateDiagram-v2）
- ❌ C级内容不开独立小节，不加粗，不加提示框
- ❌ 知识框架不能省略
- ❌ `X_0`串讲文件只占位，不生成内容
