## 从 0 理解

程序是静态文件，进程是程序的一次动态执行。你可以把程序理解为菜谱，把进程理解为正在做菜的过程：做菜时需要锅、火、食材、当前做到哪一步，这些运行状态都不是菜谱本身能表达的。

PCB 是操作系统认识进程的“身份证 + 档案袋”。没有 PCB，操作系统就无法调度、阻塞、唤醒或终止这个进程。

线程是进程内部更小的执行流。一个进程可以有多个线程，它们共享进程的资源，但每个线程必须有自己的栈和寄存器现场，否则无法独立执行函数调用和恢复运行。

## 核心机制/算法/公式

### 进程组成

进程实体通常包括：

- PCB：进程控制块，记录 PID、状态、优先级、资源清单、CPU 现场等。
- 程序代码：要执行的指令。
- 数据和堆栈：运行时处理的数据、函数调用栈。

### 三种基本状态

| 状态 | 含义 | 关键判断 |
|---|---|---|
| 运行 | 正在 CPU 上执行 | 单处理机某一时刻最多一个运行进程 |
| 就绪 | 只差 CPU | 已具备运行条件，等待调度 |
| 等待/阻塞 | 等待事件或资源 | 即使 CPU 空闲也不能运行 |

合法转换：

- 就绪 -> 运行：被调度选中。
- 运行 -> 就绪：时间片到，或被更高优先级进程剥夺。
- 运行 -> 等待：主动等待 I/O、资源或事件。
- 等待 -> 就绪：事件发生或资源可用。

不合法高频陷阱：

- 就绪 -> 等待：就绪进程没有运行，不能主动发出等待请求。
- 等待 -> 运行：等待结束后要先回到就绪队列，再等待 CPU 调度。

### fork 进程数

每次 `fork()` 会让现有每个进程各复制出一个子进程。因此连续两次 `fork()` 后共有 `2^2 = 4` 个进程。

### 进程与线程比较

| 维度 | 进程 | 线程 |
|---|---|---|
| 资源 | 资源分配基本单位 | 共享所属进程资源 |
| 调度 | 传统系统中是调度单位 | 支持线程后通常是调度单位 |
| 地址空间 | 进程间通常不同 | 同进程线程共享地址空间 |
| 开销 | 创建、切换开销较大 | 创建、切换开销较小 |
| 独立性 | 强 | 弱，不能脱离进程存在 |

## 易错点与区分表

| 易错说法 | 正确说法 |
|---|---|
| 线程是特殊进程 | 线程是进程内的执行流，不是脱离进程的特殊进程 |
| 同一进程线程有不同地址空间 | 同一进程线程共享地址空间 |
| 进程越多 CPU 效率越高 | 过多进程会导致调度和切换开销增加 |
| 父子进程共享虚拟地址空间 | fork 后父子进程有不同 PCB 和 PID，地址空间逻辑上分离 |
| 等待时间片是阻塞态 | 等待 CPU 时间片是就绪态 |

## 题型与解析

### 作业题：进程定义

题干：进程最不符合理解的是“进程是在多程序并行环境中的完整的程序”。

解析：进程不是程序本身，而是程序在某个数据集合上的运行过程。程序是静态的，进程是动态的。

### 作业题：状态转换

题干：三种基本状态之间转换不正确的是“就绪 -> 等待”。

解析：等待/阻塞是运行进程主动发出的行为，例如请求 I/O。就绪进程没有获得 CPU，不能执行导致阻塞的指令。

### 真题题型：多线程适用性

多线程适合一个任务内部存在多个可并发执行的活动，如 Web 服务器响应多个请求、GUI 同时处理输入和计算。若题目说线程脱离进程独立运行，就是错误。
