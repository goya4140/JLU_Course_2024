# 06 存储管理（精简版样板）

> 用途：考前快速复习第 6 章。结构采用“知识点 -> 对应例题 -> 解答过程 -> 易错排雷”。

## 1. 本章考试定位

第 6 章的核心不是背很多名词，而是回答三个问题：

```text
1. 内存怎么分？
2. 逻辑地址怎么变成物理地址？
3. 不同管理方式的碎片、共享、保护、访存次数有什么区别？
```

本章常见题型：

1. 选择题：存储管理功能、分页/分段特点、碎片类型、界地址代价、段页式访存次数。
2. 地址计算题：界地址 `b+a`，分页 `p,d -> f,d`，段式 `s,d -> b+d`。
3. 表项计算题：逻辑页数、页表长度、页表占用空间、页表首址。
4. 算法判断题：首次适应、最佳适应、最坏适应。

本章最重要的一句话：

```text
第 6 章所有地址题，本质都是“先拆地址，再查表，再拼物理地址”。
```

## 3. 存储管理功能与地址概念

### 3.1 知识点速记

【必背】存储管理五大功能：

| 功能 | 含义 | 常见考法 |
|---|---|---|
| 存储分配和去配 | 给进程分配内存，用完回收 | 问 OS 管什么 |
| 存储共享 | 多进程共用代码/数据 | 段式更方便 |
| 存储保护 | 防止越界、越权 | 基址/限长、权限位 |
| 存储扩充 | 内外存配合，让程序感觉空间更大 | 虚拟存储前置 |
| 地址映射 | 逻辑地址变物理地址 | 本章计算核心 |

【必背】地址：

| 地址 | 含义 |
|---|---|
| 逻辑地址 | 程序生成/看到的地址，也可称相对地址 |
| 物理地址 | 主存中的真实地址 |

地址映射：

```text
逻辑地址 -> 地址映射机构 -> 物理地址
```

【易错】地址映射运行时主要依赖硬件。操作系统负责建立表、设置寄存器；每次访存时由硬件快速转换。

### 3.2 对应例题：存储管理功能判断

题目：

```text
判断下列哪些属于操作系统存储管理的功能：
A. 存储分配和去配
B. 存储共享
C. 存储保护
D. 存储扩充
E. 地址映射
```

答案：

```text
A、B、C、D、E 都属于。
```

解答过程：

课内存储管理功能就是五项：

```text
分配去配、共享、保护、扩充、地址映射。
```

选择题遇到“存储管理功能”时，不要只想到“分配内存”。保护、共享和地址转换同样属于核心职责。

【一句话记法】

```text
内存管理不是只分内存，还要共享、保护、扩充、映射。
```

## 4. 分区管理与动态分配算法

### 4.1 分区类型速记

内存分区有两个维度：

| 维度 | 类型 | 含义 |
|---|---|---|
| 分区时刻 | 静态分区 | 系统初始化时预先划好 |
| 分区时刻 | 动态分区 | 进程申请时再划分 |
| 分区大小 | 等长分区 | 每块大小相同 |
| 分区大小 | 异长分区 | 按实际需要分配不同大小 |

常见对应：

```text
页式、段页式：静态等长页框
界地址、段式：动态异长区域
```

### 4.2 动态异长分区算法

【会判】四个算法：

| 算法 | 英文 | 选择规则 | 易错点 |
|---|---|---|---|
| 首次适应 | FF | 按首址递增，找第一个够大的空闲区 | 不是找最小 |
| 下次适应 | NF | 从上次位置之后继续找第一个够大的空闲区 | 指针位置很关键 |
| 最佳适应 | BF | 在所有够大的空闲区中找最小的 | 容易留下小碎片 |
| 最坏适应 | WF | 在所有够大的空闲区中找最大的 | 会切割大空闲区 |

分配后更新空闲区：

```text
若从空闲区首部切出 size：
空闲区首址 += size
空闲区长度 -= size
若长度变 0，删除该表项。
```

回收时看相邻空闲区：

| 相邻情况 | 空闲分区数变化 |
|---|---|
| 上下都不是空闲区 | +1 |
| 只有上方或只有下方空闲 | 不变 |
| 上下都是空闲区 | -1 |

### 4.3 对应例题：分配算法判断

题目：

```text
空闲区按首址递增为：
(100, 20KB), (200, 12KB), (300, 30KB), (500, 18KB)

现在申请 15KB。
分别用首次适应 FF、最佳适应 BF、最坏适应 WF，会选择哪个空闲区？
```

答案：

```text
FF 选择 (100, 20KB)
BF 选择 (500, 18KB)
WF 选择 (300, 30KB)
```

解答过程：

先删除不能满足 15KB 的空闲区：

```text
(200, 12KB) 不够，删掉。
可选：(100,20KB), (300,30KB), (500,18KB)
```

FF：

```text
按首址递增找第一个够大的 -> (100,20KB)
```

BF：

```text
在所有够大的空闲区中找最小的 -> (500,18KB)
```

WF：

```text
在所有够大的空闲区中找最大的 -> (300,30KB)
```

【易错排雷】

1. BF 不是地址最小，而是大小最贴近。
2. FF 不是最小够用，而是从低地址开始第一个够用。
3. WF 选最大空闲区，理由是希望切完后剩余区仍较大。

## 5. 碎片与紧凑

### 5.1 知识点速记

【必背】碎片分类：

| 类型 | 位置 | 典型方式 | 解释 |
|---|---|---|---|
| 内部碎片 | 已分配块内部 | 固定分区、页式、段页式 | 分给你了但你没用满 |
| 外部碎片 | 已分配块之间 | 动态分区、段式 | 空闲总量够但不连续 |

常见判断：

| 管理方式 | 内部碎片 | 外部碎片 |
|---|---|---|
| 固定分区 | 有 | 通常不强调 |
| 动态分区 | 无内部碎片 | 有外部碎片 |
| 页式 | 有内部碎片 | 无外部碎片 |
| 段式 | 无内部碎片 | 有外部碎片 |
| 段页式 | 有内部碎片 | 无外部碎片 |

紧凑：

```text
移动已占用区，把分散空闲区合成一大片。
```

紧凑主要解决外部碎片，但开销大，并且需要重定位支持。

### 5.2 对应例题：不会产生内部碎片的是谁

题目：

```text
不会产生内部碎片的存储管理是？
A. 页式
B. 段式
C. 固定分区式
D. 段页式
```

答案：

```text
B. 段式。
```

解答过程：

逐项排除：

1. 页式：按固定页框分配，最后一页可能用不满，有内部碎片。
2. 固定分区式：分区大小固定，作业小于分区大小时，分区内部剩余空间浪费，有内部碎片。
3. 段页式：段内仍然分页，每段最后一页可能用不满，有内部碎片。
4. 段式：按逻辑段实际大小动态分配，不产生内部碎片。

但段式会产生：

```text
外部碎片。
```

【一句话记法】

```text
页有内部碎片；段有外部碎片；段页式继承页的内部碎片。
```

## 6. 界地址管理

### 6.1 知识点速记

界地址管理是一维地址方式。

一个进程占用一个连续内存区：

```text
逻辑地址范围：0 ~ l-1
物理地址范围：b ~ b+l-1
```

寄存器：

| 寄存器 | 含义 |
|---|---|
| 基址寄存器 `b` | 当前进程在内存中的起始物理地址 |
| 限长寄存器 `l` | 当前进程长度 |

地址转换：

```text
若 0 <= a <= l-1：
    物理地址 = b + a
否则：
    越界中断
```

【会判】界地址管理代价最小，因为只需要基址/限长寄存器和一次加法。

### 6.2 对应例题：界地址映射

题目：

```text
某进程采用界地址管理。
基址寄存器 b = 1000，限长寄存器 l = 400。

求逻辑地址 a = 250 的物理地址。
逻辑地址 a = 400 是否合法？
```

答案：

```text
a = 250 合法，物理地址 = 1250。
a = 400 不合法，越界。
```

解答过程：

合法逻辑地址范围：

```text
0 <= a <= l-1 = 399
```

当 `a=250`：

```text
250 <= 399，合法
物理地址 = b + a = 1000 + 250 = 1250
```

当 `a=400`：

```text
400 > 399，越界
```

【易错排雷】

1. 最大合法地址是 `l-1`，不是 `l`。
2. 越界时不计算物理地址，而是产生越界中断。
3. 界地址管理要求进程连续装入。

### 6.3 对应例题：界地址管理代价最小

题目：

```text
操作系统为实现多道程序并发，对内存管理可以采用多种方式，
其中代价最小的是？
A. 界地址管理
B. 分页管理
C. 分段管理
D. 段页式管理
```

答案：

```text
A. 界地址管理。
```

解答过程：

界地址管理只需要：

```text
基址寄存器 + 限长寄存器 + 越界检查 + 加法
```

分页、分段、段页式都需要表结构：

| 方式 | 额外结构 |
|---|---|
| 分页 | 页表 |
| 分段 | 段表 |
| 段页式 | 段表 + 页表 |

所以代价最小的是界地址管理。

## 7. 交换与覆盖

### 7.1 知识点速记

【会判】交换 swapping：

```text
把暂时不能运行或暂时不用的进程从内存换出到外存；
需要时再换入内存。
```

特点：

1. 粒度通常是进程级。
2. 目的主要是缓解多进程内存不足。
3. 需要程序可重定位，因为换入位置可能变化。

【会判】覆盖 overlay：

```text
把程序划分成若干模块，不同时运行的模块共用同一块内存区域。
```

对比：

| 对比 | 交换 | 覆盖 |
|---|---|---|
| 粒度 | 进程级 | 程序模块级 |
| 目的 | 多进程内存调度 | 大程序在小内存中运行 |
| 管理 | OS 为主 | 程序结构/覆盖驱动参与 |

### 7.2 对应例题：交换与覆盖判断

题目：

```text
判断下列说法：
A. 交换是把整个进程或较大部分在内存和外存之间换进换出。
B. 覆盖是把互斥执行的程序模块放在同一内存区反复使用。
C. 交换要求程序具备可重定位能力。
D. 覆盖和交换完全相同。
```

答案：

```text
A 对，B 对，C 对，D 错。
```

解答过程：

1. A 对：交换面向内存中进程的调入调出。
2. B 对：覆盖利用“不会同时执行”的模块复用空间。
3. C 对：换出后再换入，物理位置可能变化，所以需要重定位支持。
4. D 错：交换偏 OS 进程级调度，覆盖偏程序模块级组织。

【一句话记法】

```text
交换换进程，覆盖换模块。
```

## 8. 页式存储管理

### 8.1 知识点速记

页式管理把两边都切成等长块：

| 概念 | 所在空间 | 含义 |
|---|---|---|
| 页面 page | 逻辑地址空间 | 进程被切成的固定大小块 |
| 页框 frame | 物理内存空间 | 主存被切成的固定大小块 |
| 页表 | 每个进程 | 记录逻辑页号到物理页框号的映射 |

【必背】分页主要特点：

```text
作业不要求装入主存的连续区域。
```

【易错】基本分页不等于请求分页。缺页中断、作业不必全部同时装入，属于第 7 章虚拟页式/请求分页重点。

### 8.2 页号和页内地址

若页面大小为 `page_size`，逻辑地址为 `A`：

```text
页号 p = A / page_size 取整
页内地址 d = A % page_size
```

若页面大小是 `2^i`：

```text
低 i 位 = 页内地址 d
高位 = 页号 p
```

物理地址：

```text
物理地址 = 页框号 f * page_size + 页内地址 d
```

### 8.3 对应例题：分页式存储管理主要特点

题目：

```text
分页式存储管理的主要特点是？
A. 要求处理缺页中断
B. 要求扩充主存容量
C. 不要求作业装入到主存的连续区域
D. 不要求作业全部同时装入主存
```

答案：

```text
C。
```

解答过程：

基本分页的核心是：

```text
逻辑空间分成页面，物理内存分成页框；
逻辑上连续的页面，可以放入物理上不连续的页框。
```

因此它不要求作业装入连续主存区域。

逐项排雷：

| 选项 | 判断 | 原因 |
|---|---|---|
| A | 错 | 缺页中断是请求分页/虚拟存储特征 |
| B | 错 | 分页本身不等于扩充主存容量 |
| C | 对 | 基本分页解决物理不连续分配 |
| D | 错 | 作业不必全部同时装入是请求分页/虚拟存储特征 |

【一句话记法】

```text
基本分页解决“不连续装入”；虚拟分页才强调“不必全装入”。
```

### 8.4 对应例题：分页地址转换

题目：

```text
页面大小为 4KB，逻辑地址为 8AC2H。
若页表显示逻辑页 8H 对应物理页框 20H，
求物理地址。
```

答案：

```text
物理地址 = 20AC2H。
```

解答过程：

页面大小：

```text
4KB = 4096B = 1000H = 2^12
```

所以低 12 位是页内地址。

逻辑地址：

```text
8AC2H
```

拆分：

```text
页号 p = 8H
页内地址 d = AC2H
```

查页表：

```text
p = 8H -> f = 20H
```

拼物理地址：

```text
物理地址 = f * 1000H + d
         = 20H * 1000H + AC2H
         = 20AC2H
```

【易错排雷】

1. 页面大小决定页内地址位数。4KB 对应低 12 位。
2. 页内地址 `d` 在转换前后不变。
3. 变的是页号 `p` 到页框号 `f`。

### 8.5 TLB 与有效访问时间

TLB，又称快表，用来缓存：

```text
页号 -> 页框号
```

基本分页无快表时：

```text
访问页表 1 次 + 访问数据 1 次 = 2 次访存
```

有快表时：

```text
快表命中：访问 TLB + 访问数据
快表未命中：访问 TLB + 访问页表 + 访问数据
```

有效访问时间公式：

```text
EAT = 命中率 * (TLB时间 + 内存时间)
    + 未命中率 * (TLB时间 + 2 * 内存时间)
```

### 8.6 对应例题：EAT 计算

题目：

```text
快表命中率为 98%，快表访问时间为 20ns，内存访问时间为 100ns。
求有效访问时间 EAT。
```

答案：

```text
EAT = 122ns。
```

解答过程：

命中时：

```text
TLB + 内存 = 20 + 100 = 120ns
```

未命中时：

```text
TLB + 页表内存访问 + 数据内存访问
= 20 + 100 + 100
= 220ns
```

代入公式：

```text
EAT = 0.98 * 120 + 0.02 * 220
    = 117.6 + 4.4
    = 122ns
```

【易错】未命中不是只多一次 TLB，而是还要访问页表，所以包含两次内存访问。

### 8.7 多级页表与反置页表

【会判】多级页表为什么引入：

```text
单级页表可能太大，并且要求较大的连续页表空间；
多级页表把页表也分页，只为实际使用的部分建立下级页表。
```

访存次数：

| 页表级数 | 无 TLB 访问一次数据需要 |
|---|---|
| 单级页表 | 页表 1 次 + 数据 1 次 = 2 次 |
| 二级页表 | 外页表 + 内页表 + 数据 = 3 次 |
| 四级页表 | 4 级页表 + 数据 = 5 次 |

【会判】反置页表：

```text
传统页表面向进程逻辑页；
反置页表面向物理页框，每个物理页框一项。
```

优点：页表大小由物理页框数决定，适合虚拟空间很大的系统。  
缺点：查找更复杂，通常需要散列和 TLB。

## 9. 段式存储管理

### 9.1 知识点速记

页式是机器视角的等长块，段式是程序视角的逻辑单位。

段可以是：

```text
主程序、子程序、函数、数据段、栈、符号表等。
```

段式逻辑地址是二维地址：

```text
(段号 s, 段内地址 d)
```

段表项包括：

| 内容 | 作用 |
|---|---|
| 段首址 `b'` | 该段在物理内存中的起始地址 |
| 段长度 `l'` | 判断段内地址是否越界 |
| 保护位 | 判断读/写/执行权限 |

段式地址转换：

```text
1. 检查段号 s 是否超过段表长度。
2. 查段表得到段首址 b' 和段长 l'。
3. 检查段内地址 d 是否越界。
4. 物理地址 = b' + d。
```

【易错】段式有两次越界检查：段号越界、段内地址越界。

### 9.2 对应例题：段式地址转换

题目：

```text
某进程段表如下：

| 段号 | 段首址 | 段长 |
|---|---:|---:|
| 0 | 1000 | 500 |
| 1 | 3000 | 800 |
| 2 | 5000 | 100 |

求逻辑地址 (1, 120) 的物理地址。
判断逻辑地址 (2, 100) 是否合法。
```

答案：

```text
(1,120) 的物理地址 = 3120。
(2,100) 不合法，段内地址越界。
```

解答过程：

对 `(1,120)`：

```text
段号 1 存在。
段 1 段长 = 800，合法段内地址范围 0~799。
120 <= 799，合法。
物理地址 = 段首址 + 段内地址 = 3000 + 120 = 3120。
```

对 `(2,100)`：

```text
段号 2 存在。
段 2 段长 = 100，合法段内地址范围 0~99。
100 不合法，越界。
```

【易错排雷】

1. 段内地址最大合法值是 `段长-1`。
2. 段式不是 `段号 * 段长 + 位移`，而是查段表得到段首址。
3. 不同段长度可以不同。

### 9.3 对应例题：段地址位数题

题目：

```text
采用分段存储管理系统中，若段地址用 24 位表示，
其中 8 位表示段号，则允许每段的最大长度为多少？
```

答案：

```text
2^16 B。
```

解答过程：

总逻辑地址位数：

```text
24 位
```

段号位数：

```text
8 位
```

所以段内地址位数：

```text
24 - 8 = 16 位
```

每段最大长度由段内地址决定：

```text
2^16 B
```

【易错排雷】

1. 不要选 `2^24B`，24 位是整个段地址长度。
2. 段号位数决定最多有多少段：`2^8` 段。
3. 段内位数决定每段最大多长：`2^16B`。

### 9.4 段的共享与保护

【必背】段式便于共享和保护，因为段是程序逻辑单位。

共享：

```text
多个进程的段表项指向同一个共享段。
```

保护：

```text
段表可设置 R/W/E 权限。
```

例子：

| 段 | 权限 |
|---|---|
| 代码段 | 可读、可执行、不可写 |
| 数据段 | 可读写，不一定可执行 |
| 共享库代码段 | 可共享执行 |

## 10. 段页式存储管理

### 10.1 知识点速记

段页式结合段式和页式：

```text
先按程序逻辑分段；
每段内部再分页；
内存仍按页框分配。
```

优点组合：

| 来源 | 优点 |
|---|---|
| 段式 | 便于共享和保护 |
| 页式 | 无外部碎片，分配简单 |

逻辑地址：

```text
(段号 s, 段内页号 p, 页内地址 d)
```

物理地址：

```text
(页框号 f, 页内地址 d)
```

段页式表结构：

| 表 | 内容 |
|---|---|
| 段表 | 段号、页表首址、页表长度 |
| 页表 | 段内页号、页框号 |

### 10.2 段页式地址转换

步骤：

```text
1. 检查段号 s 是否越界。
2. 查段表，得到该段页表首址和页表长度。
3. 检查段内页号 p 是否越界。
4. 查页表，得到页框号 f。
5. 物理地址 = f * 页大小 + d。
```

【易错】段页式既要检查段号，也要检查段内页号。

### 10.3 对应例题：段页式访存次数

题目：

```text
在段页式分配中，CPU 每次从内存中取一次数据需要几次访问内存？
不考虑快表。
```

答案：

```text
3 次。
```

解答过程：

无快表时：

1. 访问段表，找到该段页表的位置。
2. 访问页表，找到逻辑页对应的页框号。
3. 访问真正的数据。

所以：

```text
段表 1 次 + 页表 1 次 + 数据 1 次 = 3 次访存
```

【易错排雷】

1. 不是 2 次，因为段页式要先查段表，再查页表。
2. 如果题目没有提快表，默认按无快表计算。
3. 如果快表命中 `(s,p)->f`，才可能直接访问数据。

## 11. 页式、段式、段页式对照

### 11.1 核心对照表

| 对比 | 页式 | 段式 | 段页式 |
|---|---|---|---|
| 划分依据 | 等长页面 | 逻辑段 | 先分段，段内分页 |
| 逻辑地址 | 一维，可拆成 `(p,d)` | 二维 `(s,d)` | 二维/三元组 `(s,p,d)` |
| 内存分配 | 页框，不要求连续 | 每段连续 | 页框，不要求连续 |
| 碎片 | 内部碎片，无外部碎片 | 外部碎片，无内部碎片 | 内部碎片，无外部碎片 |
| 共享保护 | 不如段式直观 | 方便 | 方便 |
| 地址转换 | 查页表 | 查段表 | 查段表 + 页表 |

### 11.2 对应例题：三种方式判断

题目：

```text
判断下列说法：
A. 页式地址是一维地址。
B. 段式地址是二维地址。
C. 段式比页式更便于共享和保护。
D. 段页式结合了段式共享保护方便和页式无外部碎片的优点。
E. 页式没有任何碎片。
```

答案：

```text
A 对，B 对，C 对，D 对，E 错。
```

解答过程：

1. A 对：页式逻辑地址可以看作一个线性地址，硬件按页大小拆成页号和页内地址。
2. B 对：段式由程序员/编译系统给出段号和段内地址，是二维地址。
3. C 对：段是逻辑单位，比如代码段、数据段，天然适合共享和保护。
4. D 对：段页式先分段保留逻辑意义，再分页解决外部碎片。
5. E 错：页式没有外部碎片，但有内部碎片。

【一句话记法】

```text
页方便分配，段方便共享保护，段页式想两者都要。
```

## 12. 页表长度与页表首址

### 12.1 知识点速记

页表长度常指：

```text
页表项个数 = 逻辑页数
```

不是页表占用的字节数。

逻辑页数：

```text
ceil(进程长度 / 页面大小)
```

页表占用空间：

```text
页表项数 * 每个页表项大小
```

若页表本身也按页框分配：

```text
页表所需页框数 = ceil(页表占用空间 / 页面大小)
```

### 12.2 对应例题：页表首址与页表长度

题目：

```text
页面大小 2KB，进程 P 长度 9KB；
每个页表项 4B；
页表空间由系统动态分配。

问：PCB 中页表长度是多少？
页表本身至少需要多少个页框？
```

答案：

```text
页表长度 = 5。
页表本身至少需要 1 个页框。
```

解答过程：

先算逻辑页数：

```text
进程长度 / 页面大小 = 9KB / 2KB = 4.5
ceil(4.5) = 5
```

所以：

```text
页表长度 = 5
```

再算页表占用空间：

```text
页表项数 * 每项大小 = 5 * 4B = 20B
```

页面大小为 2KB：

```text
20B < 2KB
```

所以页表本身至少需要：

```text
1 个页框
```

【易错排雷】

1. `9KB/2KB=4.5` 要向上取整，因为最后不足一页也要占一页。
2. 页表长度通常写逻辑页数 `5`，不是 `20B`。
3. 页表本身也要占内存，考试问首址时要从空闲页框表中按算法分配页框。

## 13. 题型模板汇总

### 13.1 【题型】界地址映射

模板：

```text
给 b, l, a：
1. 判断 0 <= a <= l-1。
2. 合法：物理地址 = b + a。
3. 不合法：越界中断。
```

### 13.2 【题型】页式地址映射

模板：

```text
给逻辑地址 A、页面大小 page_size：
1. p = A / page_size 取整。
2. d = A % page_size。
3. 查页表：p -> f。
4. 物理地址 = f * page_size + d。
```

十六进制技巧：

```text
若 page_size = 2^i，低 i 位为页内地址，高位为页号。
```

### 13.3 【题型】段式地址映射

模板：

```text
给逻辑地址 (s,d)：
1. 检查 s 是否超过段表长度。
2. 查段表得到段首址 b' 和段长 l'。
3. 检查 0 <= d <= l'-1。
4. 合法：物理地址 = b' + d。
5. 不合法：越界中断。
```

### 13.4 【题型】段地址位数

模板：

```text
段内地址位数 = 总逻辑地址位数 - 段号位数
每段最大长度 = 2^(段内地址位数) B
最多段数 = 2^(段号位数)
```

### 13.5 【题型】段页式映射

模板：

```text
给逻辑地址 (s,p,d)：
1. 查段表，得到该段页表首址和页表长度。
2. 检查 p 是否超过该段页表长度。
3. 查页表，得到页框号 f。
4. 物理地址 = f * page_size + d。
```

无快表访存次数：

```text
段表 1 次 + 页表 1 次 + 数据 1 次 = 3 次。
```

### 13.6 【题型】动态分区分配

模板：

```text
1. 列空闲区 (首址, 长度)。
2. 去掉长度不足的空闲区。
3. 按算法选：
   FF：首址递增第一个够用。
   NF：从上次指针后第一个够用。
   BF：够用中最小。
   WF：够用中最大。
4. 分配后更新首址和长度。
5. 回收时与相邻空闲区合并。
```

### 13.7 【题型】页表长度

模板：

```text
逻辑页数 = ceil(进程长度 / 页面大小)
页表长度 = 逻辑页数
页表字节数 = 页表长度 * 页表项大小
页表所需页框数 = ceil(页表字节数 / 页面大小)
```

## 14. 易错判断总表

| 说法 | 判断 | 理由 |
|---|---|---|
| 逻辑地址就是物理地址 | 错 | 逻辑地址要经过映射 |
| 地址映射完全由操作系统软件完成 | 错 | 运行时主要依赖硬件 |
| 存储保护只防止越界，不管越权 | 错 | 越界、越权都要管 |
| FF 找最小够用空闲区 | 错 | FF 找按地址顺序第一个够用 |
| BF 找最小够用空闲区 | 对 | 最佳适应 |
| WF 找最大够用空闲区 | 对 | 最坏适应 |
| 页式无碎片 | 错 | 无外部碎片，有内部碎片 |
| 段式无内部碎片 | 对 | 但有外部碎片 |
| 段页式无外部碎片 | 对 | 但有内部碎片 |
| 紧凑主要解决内部碎片 | 错 | 主要解决外部碎片 |
| 界地址管理代价最小 | 对 | 基址/限长即可 |
| 界地址最大合法逻辑地址是 l | 错 | 是 `l-1` |
| 基本分页要求作业连续装入 | 错 | 不要求物理连续 |
| 基本分页一定有缺页中断 | 错 | 请求分页才强调缺页 |
| 页内地址位数由页大小决定 | 对 | 页面大小 `2^i`，页内地址 i 位 |
| 页表长度是页表字节数 | 错 | 常指页表项个数/逻辑页数 |
| TLB 命中还要查内存页表 | 错 | 命中直接得到页框号 |
| 段式地址是一维地址 | 错 | 段式是二维地址 `(s,d)` |
| 段式只检查段内地址越界 | 错 | 还要检查段号越界 |
| 段地址 24 位表示每段最大 2^24B | 错 | 要扣除段号位数 |
| 段页式无快表取数据 3 次访存 | 对 | 段表、页表、数据 |
| 页式比段式更便于共享保护 | 错 | 段式更符合逻辑单位 |
