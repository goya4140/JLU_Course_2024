## 3. 虚拟存储基本思想

### 3.1 知识点速记

【必背】虚拟存储的基础是局部性：

| 局部性 | 含义 |
|---|---|
| 时间局部性 | 刚访问过的内容，很可能近期再次访问 |
| 空间局部性 | 访问某地址后，很可能访问附近地址 |

虚拟存储思想：

```text
进程运行前不必全部装入内存；
部分页面/段在内存，其他部分在外存；
访问不在内存的部分时，发生缺页/缺段中断，再动态调入；
内存不够时，淘汰暂时不用的页/段。
```

【易错】虚拟存储不是让物理内存真实变大，而是用外存和动态调入制造“大地址空间”的效果。

虚拟存储通常依赖：

1. 离散分配。
2. 部分装入。
3. 请求调入。
4. 页面/段置换。

### 3.2 对应例题：虚拟存储判断

题目：

```text
判断下列说法：
A. 虚拟存储可以让进程逻辑地址空间大于实际物理内存。
B. 虚拟存储把整个程序一次性全部装入内存后才能运行。
C. 虚拟存储依赖程序访问的局部性。
D. 虚拟存储本质上是把物理内存条容量变大。
```

答案：

```text
A 对，B 错，C 对，D 错。
```

解答过程：

1. A 对：虚拟存储让程序看到较大的逻辑地址空间。
2. B 错：虚拟存储强调部分装入、按需调入。
3. C 对：如果没有局部性，频繁缺页会让系统性能很差。
4. D 错：物理内存没变大，只是用外存配合动态调入。

【一句话记法】

```text
虚拟存储不是内存变大，而是“用到再搬”。
```

## 4. 缺页中断与虚拟页式页表

### 4.1 知识点速记

虚拟页式：

```text
进程全部或大部分放在外存，部分页面装入内存；
访问页不在内存时，产生缺页中断。
```

【会写】缺页中断处理流程：

```text
1. 保存现场。
2. 查页表，得到该页外存块号。
3. 寻找空闲页框。
4. 若无空闲页框，按置换算法选择淘汰页。
5. 若淘汰页修改过，写回外存。
6. 将所需页从外存读入页框。
7. 修改页表、总页表、快表。
8. 恢复现场。
9. 重新执行导致缺页的指令。
```

【易错】缺页处理完后重新执行原指令，不是直接执行下一条指令。

虚拟页式页表字段：

| 字段 | 含义 |
|---|---|
| 逻辑页号 | 该页表项对应的逻辑页 |
| 页框号 | 页面在内存时对应的物理页框 |
| 外存块号 | 页面不在内存时所在外存块 |
| 内外标志 | 是否在内存 |
| 访问位/引用位 | 最近是否访问过 |
| 修改位 | 是否被写过，换出时是否要写回 |
| 访问权限 | 读/写/执行权限 |

### 4.2 对应例题：缺页处理流程

题目：

```text
某进程采用虚拟页式存储管理。CPU 访问逻辑页 p 时，
页表显示该页当前不在内存，但给出了外存块号。
请说明系统如何处理该访问。
```

答案：

```text
发生缺页中断，操作系统调入该页，更新页表，然后重新执行被中断指令。
```

解答过程：

按步骤写：

1. CPU 查页表发现 `p` 不在内存，产生缺页中断。
2. 操作系统保存当前进程现场。
3. 根据页表中的外存块号找到页面在外存的位置。
4. 查找空闲页框。
5. 若有空闲页框，直接读入；若无空闲页框，用页面置换算法选择淘汰页。
6. 若被淘汰页修改位为 1，要先写回外存。
7. 把所需页读入页框。
8. 修改页表：页框号、内外标志、访问位等。
9. 恢复现场，重新执行刚才导致缺页的指令。

【易错排雷】

1. 不在内存不是“地址越界”，而是缺页。
2. 修改位为 1 的被淘汰页必须写回。
3. 缺页处理属于中断处理，代价远大于普通访存。

## 5. 请求调页与预调页

### 5.1 知识点速记

【必背】请求调页：

```text
发生缺页中断时才把页面调入。
```

优点：不提前装入可能不用的页，节省内存和 I/O。  
缺点：第一次访问某页会缺页。

【必背】预调页：

```text
根据局部性或程序顺序，提前调入可能马上用到的页面。
```

课件强调：

```text
预调必须辅以请调。
```

因为预测不一定准，预调没调到的页面仍要靠请求调页处理。

### 5.2 对应例题：请求 + 预调规则

题目：

```text
某虚拟页式系统采用“请求 + 预调”：
每次发生页面请调时，
如果紧邻请调页面的下一逻辑页面已经加载到内存，则只加载当前请调页面；
否则同时加载当前请调页面及紧邻的下一逻辑页面。

若访问逻辑页 5 时发生缺页，且逻辑页 6 不在内存，应调入哪些页？
若逻辑页 6 已在内存，又应调入哪些页？
```

答案：

```text
页 6 不在内存：调入页 5 和页 6。
页 6 已在内存：只调入页 5。
```

解答过程：

按题目规则：

1. 当前访问页 5 不在内存，先确定页 5 必须调入。
2. 检查下一逻辑页 6。
3. 若页 6 不在内存，预调页 6，所以调入 5、6。
4. 若页 6 已在内存，不重复调入，所以只调入 5。

【易错排雷】

1. 预调页也占页框。
2. 如果固定页框数已满，调入当前页和预调页可能连续触发置换。
3. 不要只处理当前访问页，忘记下一逻辑页。

## 6. 页面置换算法

### 6.1 算法速记

页面置换目标：

```text
降低缺页率，减少换页 I/O。
```

常见算法：

| 算法 | 淘汰规则 | 特点 |
|---|---|---|
| OPT | 淘汰未来最长时间不用的页 | 理论最优，不可实现 |
| FIFO | 淘汰最早进入内存的页 | 简单，可能 Belady 异常 |
| LRU | 淘汰最长时间未被访问的页 | 性能较好，需要历史信息 |
| NUR | 淘汰最近一段时间未访问页 | LRU 近似 |
| LFU | 淘汰访问次数最少页 | 可能误淘刚装入页 |
| Clock | 访问位为 0 淘汰，为 1 清 0 后跳过 | 二次机会近似 |
| 改进 Clock | 同时考虑访问位和修改位 | 优先淘汰代价低的页 |

【必背】Belady 异常：

```text
分配更多页框，缺页次数反而增加。
```

FIFO 可能出现 Belady 异常；LRU 和 OPT 不会。

### 6.2 对应例题：Belady 异常

题目：

```text
访问序列：1,2,3,4,1,2,5,1,2,3,4,5。
课件给出 FIFO 算法：
3 个页框时缺页 9 次；
4 个页框时缺页 10 次。

这说明了什么现象？哪些算法可能/不可能出现这种现象？
```

答案：

```text
说明 FIFO 可能出现 Belady 异常。
FIFO 可能出现；LRU 和 OPT 不会出现。
```

解答过程：

正常直觉是：

```text
页框越多，能保留的页面越多，缺页次数应该不增加。
```

但 FIFO 只看“进入内存时间”，不看页面是否还会被频繁访问。页框数变化会改变 FIFO 队列中的淘汰顺序，导致更多页框反而缺页更多。

因此：

```text
3 页框缺页 9 次，4 页框缺页 10 次
```

就是 Belady 异常。

【一句话记法】

```text
Belady 异常：页框多了，FIFO 反而可能更差。
```

### 6.3 对应例题：LRU 缺页模拟

题目：

```text
内存有 3 个页框，初始为空。
访问序列：1, 2, 3, 2, 4, 1, 2。
采用 LRU 算法，统计缺页次数。
```

答案：

```text
缺页 5 次。
```

解答过程：

LRU 规则：

```text
每访问一页，把它更新为最近使用；
缺页且无空闲页框时，淘汰最长时间未访问的页。
```

逐步模拟：

| 访问 | 内存状态 | 是否缺页 | 说明 |
|---|---|---|---|
| 1 | 1 | 是 | 空页框，装入 1 |
| 2 | 1,2 | 是 | 空页框，装入 2 |
| 3 | 1,2,3 | 是 | 空页框，装入 3 |
| 2 | 1,3,2 | 否 | 2 变为最近使用 |
| 4 | 3,2,4 | 是 | 1 最久未用，淘汰 1 |
| 1 | 2,4,1 | 是 | 3 最久未用，淘汰 3 |
| 2 | 4,1,2 | 否 | 2 已在内存 |

缺页次数：

```text
5
```

【易错排雷】

1. LRU 淘汰“最久未访问”，不是“最早进入”。
2. 命中也要更新最近使用次序。
3. 初始装入空页框也算缺页。

## 7. Clock 与改进 Clock

### 7.1 Clock 知识点速记

Clock 算法把页面组织成环，指针指向某页。

每页有访问位 `r`：

| r | 处理 |
|---|---|
| 0 | 淘汰该页 |
| 1 | 清 0，给第二次机会，指针后移 |

【一句话记法】

```text
Clock 看到 1 不淘汰，先清 0；看到 0 才淘汰。
```

### 7.2 改进 Clock 知识点速记

改进 Clock 同时看访问位 `r` 和修改位 `m`。

优先级：

| r | m | 含义 | 淘汰优先级 |
|---|---|---|---|
| 0 | 0 | 最近未访问，未修改 | 第一优先 |
| 0 | 1 | 最近未访问，已修改 | 第二优先，要写回 |
| 1 | 0 | 最近访问，未修改 | 暂缓 |
| 1 | 1 | 最近访问，已修改 | 最后 |

标准扫描：

```text
1. 第一轮从指针处找 (0,0)，不改变 r。
2. 找不到时，第二轮找 (0,1)，扫描过程中把遇到的 r=1 清 0。
3. 若仍找不到，再回到第一轮，此时通常能找到。
```

【易错】第一轮找 `(0,0)` 时不清访问位；第二轮找 `(0,1)` 时才清。

### 7.3 对应例题：改进 Clock 选择淘汰页

题目：

```text
某进程有 4 个页框，改进 Clock 指针当前指向 A。
页面环顺序为 A -> B -> C -> D -> A。

| 页面 | r | m |
|---|---|---|
| A | 1 | 0 |
| B | 0 | 1 |
| C | 1 | 1 |
| D | 0 | 0 |

按改进 Clock 算法，应优先淘汰哪一页？
```

答案：

```text
淘汰 D。
```

解答过程：

第一轮寻找 `(r=0,m=0)`：

1. A 是 `(1,0)`，不是。
2. B 是 `(0,1)`，不是第一优先。
3. C 是 `(1,1)`，不是。
4. D 是 `(0,0)`，找到。

所以直接淘汰 D。第一轮找到了 `(0,0)`，不需要进入第二轮。

【易错排雷】

1. B 虽然 `r=0`，但 `m=1`，换出要写回，不如 D。
2. 改进 Clock 不是简单找第一个 `r=0`，还要看修改位。
3. 指针位置决定扫描顺序。

### 7.4 对应例题：作业五改进 Clock 题型

题目：

```text
某系统采用虚拟页式存储管理方式，页面大小为 2KB，
每个进程分配的页框数固定为 4 页。
采用局部置换策略，置换算法采用改进的时钟算法。
当有页面新装入内存时，页表的时钟指针指向新装入页面的下一个在内存的表项。
设当前进程 P 的页表如下（“时钟”指针指向逻辑页面 3 的表项）：

⑴ 当进程 P 依次对逻辑地址执行下述操作：
① 引用 4C7H；② 修改 19B4H；③ 修改 0C9AH；
写出进程 P 的页表内容。

⑵ 在 ⑴ 的基础上，当 P 对逻辑地址 27A8H 进行访问，
该逻辑地址对应的物理地址是多少？
```

说明：

```text
当前保存的 作业.md 中缺少“当前进程 P 的页表如下”的具体页表表格，
因此不能唯一算出最终页表和 27A8H 的物理地址。
```

但是这题的解题流程必须会。

解答过程模板：

第一步，按页面大小拆地址。

页面大小：

```text
2KB = 800H = 2^11
```

所以低 11 位是页内地址。

逐个地址拆分：

```text
4C7H:
页号 = 4C7H / 800H = 0
页内地址 = 4C7H

19B4H:
页号 = 19B4H / 800H = 3
页内地址 = 1B4H

0C9AH:
页号 = 0C9AH / 800H = 1
页内地址 = 49AH

27A8H:
页号 = 27A8H / 800H = 4
页内地址 = 7A8H
```

第二步，依次处理访问。

对“引用 4C7H”：

```text
访问逻辑页 0。
若页 0 在内存：访问位置 1，修改位不变。
若页 0 不在内存：缺页，按改进 Clock 找淘汰页，调入页 0，访问位置 1，修改位置 0。
```

对“修改 19B4H”：

```text
访问逻辑页 3。
若页 3 在内存：访问位置 1，修改位置 1。
若页 3 不在内存：缺页，调入后访问位置 1，修改位置 1。
```

对“修改 0C9AH”：

```text
访问逻辑页 1。
若页 1 在内存：访问位置 1，修改位置 1。
若页 1 不在内存：缺页，调入后访问位置 1，修改位置 1。
```

第三步，如果需要置换：

```text
从当前时钟指针开始；
先找 (r=0,m=0)；
找不到再找 (r=0,m=1)，同时清 r；
淘汰页若 m=1，要写回外存；
新页装入后，时钟指针指向新装入页面的下一个在内存的表项。
```

第四步，计算 `27A8H` 物理地址。

```text
27A8H -> 页号 4，页内地址 7A8H。
```

若经过前面处理后，逻辑页 4 在页框 `f` 中，则：

```text
物理地址 = f * 800H + 7A8H
```

若逻辑页 4 不在内存，则：

```text
先发生缺页，按改进 Clock 调入页 4；
再用调入后的页框号 f 计算 f * 800H + 7A8H。
```

【易错排雷】

1. `2KB=800H`，不是 `1000H`。
2. 引用只置访问位；修改要同时置访问位和修改位。
3. 局部置换只在本进程分配的 4 个页框中选淘汰页。
4. 新页面装入后，时钟指针按题目规则移动到“新装入页面的下一个在内存的表项”。
5. 当前题面缺失初始页表，不能凭空写最终页表。

## 8. 地址拆分与页式地址题

### 8.1 知识点速记

页式/虚拟页式地址题模板：

```text
页号 p = 逻辑地址 / 页大小
页内地址 d = 逻辑地址 % 页大小
```

若页面大小是 `2^i`：

```text
低 i 位是页内地址；
高位是页号。
```

物理地址：

```text
物理地址 = 页框号 f * 页大小 + 页内地址 d
```

若页不在内存：

```text
先缺页处理，再计算物理地址。
```

### 8.2 对应例题：2010 考研风格地址拆分

题目：

```text
进程空间和内存空间都是 64K；
页长 1K；
某进程 6 个页，内存分配 4 个页框；
访问逻辑地址 13B7H。

求该逻辑地址的逻辑页号和页内地址。
```

答案：

```text
逻辑页号 = 4。
页内地址 = 3B7H。
```

解答过程：

页长：

```text
1K = 1024B = 400H = 2^10
```

所以低 10 位是页内地址。

逻辑地址：

```text
13B7H = 4 * 400H + 3B7H
```

因此：

```text
页号 p = 4
页内地址 d = 3B7H
```

如果页 4 已在页框 `f`，则：

```text
物理地址 = f * 400H + 3B7H
```

如果页 4 不在内存，则发生缺页，先调入或置换，再计算物理地址。

【易错排雷】

1. 页大小决定页内地址位数。
2. 虚拟页式下页不在内存不是越界，而是缺页。
3. 页号合法但不在内存，处理方式是缺页调入。

## 9. 工作集、抖动与 PFFB

### 9.1 工作集

【必背】工作集：

```text
某进程在时间窗口 Δ 内访问过的不同页面集合。
```

记作：

```text
WS(t, Δ)
```

工作集大小：

```text
窗口内不同页面的个数。
```

### 9.2 对应例题：工作集大小

题目：

```text
某进程在时间窗口 Δ 内访问页面序列：
5, 7, 7, 1, 6, 2, 2。

该窗口内工作集是什么？工作集大小是多少？
```

答案：

```text
工作集 = {5, 7, 1, 6, 2}
工作集大小 = 5
```

解答过程：

工作集看的是“不同页面”，不是访问次数。

访问序列中不同页面为：

```text
5, 7, 1, 6, 2
```

所以工作集大小为：

```text
5
```

【易错】`7` 和 `2` 虽然各出现两次，但只计一次。

### 9.3 抖动/颠簸

【必背】抖动：

```text
页面在内存和外存之间频繁换入换出，系统大部分时间都在处理缺页和换页。
```

典型原因：

1. 分给进程的物理页框太少。
2. 置换算法不合理。
3. 多道程度过高，进程太多。

典型表现：

```text
CPU 利用率低；
交换空间磁盘利用率极高；
其他 I/O 设备利用率低。
```

处理：

1. 增加物理内存或页框数。
2. 减少并发进程数，降低多道程度。
3. 改进置换算法。
4. 用工作集或 PFFB 动态调页框。

### 9.4 对应例题：抖动判断

题目：

```text
某系统 CPU 利用率为 20%，交换空间所在磁盘利用率为 97.7%，
其他 I/O 设备利用率为 5%。
应如何提高系统性能？
```

答案：

```text
系统发生抖动/颠簸，应增加物理内存或减少并发进程数。
若选项中有“加内存条”，优先选它。
```

解答过程：

CPU 利用率很低，说明 CPU 大量时间在等待。  
交换区磁盘利用率极高，说明系统在频繁换页。  
其他设备利用率低，说明不是正常 I/O 密集，而是换页导致。

所以这是典型抖动。

【易错排雷】

1. 不要选“增加运行进程数”，那会让页框更紧张。
2. 只换更快磁盘不如增加内存直指根因。
3. 抖动不是 CPU 太忙，而是 CPU 因换页等待而低利用。

### 9.5 PFFB 页故障率反馈

PFFB：

```text
Page Fault Feed Back
```

页故障率：

```text
缺页次数 / 访问内存总次数
```

调节规则：

| 情况 | 说明 | 调整 |
|---|---|---|
| 页故障率超过上界 | 页框太少 | 增加页框 |
| 页故障率低于下界 | 页框可能过多 | 减少页框 |

## 10. 有效访问时间与缺页率

### 10.1 最大缺页率题

题目：

```text
页表保存在寄存器中。
有空页或被置换页未修改：处理缺页中断 8ms。
被置换页已修改：处理缺页中断增加写回时间，共 20ms。
一次内存访问时间 1ns。
70% 被置换页被修改过。

为保证有效访问时间不超过 12ns，可接受最大缺页率是多少？
```

答案：

```text
p <= 6.7 * 10^-7
```

解答过程：

先算平均缺页处理时间：

```text
30% 情况：8ms
70% 情况：20ms

平均 = 0.3 * 8ms + 0.7 * 20ms
     = 2.4ms + 14ms
     = 16.4ms
```

设缺页率为 `p`。

有效访问时间近似：

```text
EAT ≈ 1ns + p * 16.4ms
```

要求：

```text
EAT <= 12ns
```

所以：

```text
1ns + p * 16.4ms <= 12ns
p * 16.4ms <= 11ns
```

单位换算：

```text
16.4ms = 16.4 * 10^6 ns
```

因此：

```text
p <= 11 / (16.4 * 10^6)
  ≈ 6.7 * 10^-7
```

【易错排雷】

1. ms 和 ns 必须统一单位。
2. 缺页处理时间远大于普通访存，所以允许缺页率极低。
3. 70% 是“被置换页已修改”的概率，用来求平均缺页处理时间。

### 10.2 EAT 通用模板

如果没有缺页，只考虑快表：

```text
EAT_normal = 命中率 * (TLB + 内存)
           + 未命中率 * (TLB + 2 * 内存)
```

如果再考虑缺页：

```text
EAT = EAT_normal + 缺页率 * 缺页处理时间
```

【易错】缺页处理时间通常是 ms 级，必须换成 ns 后再与普通访存时间相加。

## 11. 虚拟段式、动态连接与虚拟段页式

### 11.1 虚拟段式

虚拟段式：

```text
进程运行前全部在外存，部分段装入内存；
访问段不在内存时，发生缺段中断。
```

逻辑地址：

```text
(段号 s, 段内地址 d)
```

若段在内存：

```text
物理地址 = 段内存首址 + d
```

若段不在内存：

```text
缺段中断。
```

虚拟段式特点：

1. 便于共享和保护。
2. 支持动态扩充和动态连接。
3. 段长不固定，调入时需要连续内存空间。
4. 可能产生外部碎片，必要时需要紧凑。

### 11.2 动态连接

静态连接：

```text
运行前由 link 完成连接。
```

动态连接：

```text
运行时用到某段时才连接。
```

优点：

1. 用到时再连接，减少不必要装入。
2. 便于共享库。
3. 减少目标代码长度。

动态连接中可能发生：

```text
连接中断。
```

【易错】动态连接要修改连接字，而共享代码要求纯代码，因此课件提出纯段共享、杂段私用。

### 11.3 虚拟段页式

虚拟段页式：

```text
先按逻辑分段，再对每段分页；
页面可按需调入；
兼顾段式共享保护和页式无外部碎片。
```

逻辑地址：

```text
(s, p, d)
```

可能中断：

| 中断 | 含义 |
|---|---|
| 连接中断 | 动态连接所需段/页表 |
| 缺页中断 | 所需页面不在内存 |
| 越界中断 | 段号或页号越界 |
| 越权中断 | 访问权限不允许 |

### 11.4 三种虚拟存储比较

| 方式 | 地址空间 | 存储分配 | 碎片 | 共享保护 | 动态扩充/连接 |
|---|---|---|---|---|---|
| 虚拟页式 | 一维 | 简单 | 无外部碎片 | 不方便 | 不方便 |
| 虚拟段式 | 二维 | 复杂 | 有外部碎片 | 方便 | 可以 |
| 虚拟段页式 | 二维 | 简单 | 无外部碎片 | 方便 | 可以 |

【易错】表中“无碎片”通常指无外部碎片；页式/段页式仍可能有内部碎片。

## 12. 真题虚拟页式大题模板

### 12.1 常见题目条件

真题/习题课中常见条件：

```text
一级页表；
页表空间由系统动态分配；
页框大小 4KB；
每个进程分配页框数固定；
局部置换；
LRU 或改进 Clock；
请求 + 预调；
空闲页框表；
最佳适应；
访问逻辑地址序列；
要求写页表、空闲页框表、物理地址。
```

### 12.2 完整答题流程

【会写】固定流程：

```text
1. 根据页大小拆逻辑地址：
   p = A / page_size
   d = A % page_size

2. 若要求页表长度：
   页表长度 = ceil(进程长度 / 页面大小)

3. 若页表本身要分配内存：
   页表字节数 = 页表项数 * 每项大小
   页表所需页框数 = ceil(页表字节数 / 页面大小)
   按空闲表和分配算法选页框。

4. 对每次访问：
   若页在内存：访问位置 1；写操作修改位也置 1。
   若页不在内存：缺页。

5. 缺页时：
   按请求/预调规则决定调入哪些页；
   有空闲页框则分配；
   无空闲页框则按局部置换算法淘汰本进程页面；
   淘汰页若修改位为 1，写回外存。

6. 更新页表：
   页框号、内外标志、访问位、修改位、外存块号。

7. 最后求物理地址：
   物理地址 = 页框号 * 页面大小 + 页内地址。
```

### 12.3 对应例题：模板型地址求解

题目：

```text
页面大小 4KB。
某逻辑地址为 2A3CH。
访问时发现逻辑页 2 已在页框 15H。
求物理地址。
```

答案：

```text
物理地址 = 15A3CH。
```

解答过程：

页面大小：

```text
4KB = 1000H
```

拆逻辑地址：

```text
2A3CH = 2 * 1000H + A3CH
页号 p = 2
页内地址 d = A3CH
```

查页表：

```text
p=2 -> f=15H
```

拼物理地址：

```text
物理地址 = 15H * 1000H + A3CH
         = 15A3CH
```

【易错】页内地址不变，逻辑页号换成物理页框号。

## 13. 题型模板汇总

### 13.1 【题型】缺页处理流程

模板：

```text
查页表发现不在内存；
产生缺页中断并保存现场；
查外存块号；
找空闲页框或按置换算法淘汰；
修改页若被淘汰则写回；
读入所需页；
更新页表/总页表/快表；
恢复现场；
重新执行原指令。
```

### 13.2 【题型】置换算法判断

模板：

```text
OPT：淘汰未来最晚使用页。
FIFO：淘汰最早进入内存页。
LRU：淘汰最久未访问页。
Clock：r=0 淘汰，r=1 清 0 后跳过。
改进 Clock：先找 00，再找 01，修改页换出要写回。
```

### 13.3 【题型】工作集选择题

模板：

```text
工作集 = 某时间窗口内访问过的不同页面集合。
工作集大小 = 不同页面个数。
```

### 13.4 【题型】抖动判断

模板：

```text
CPU 利用率低；
交换区磁盘利用率高；
缺页频繁；
说明抖动。

改进：增加页框/内存，减少并发进程，改进置换算法。
```

### 13.5 【题型】EAT 与缺页率

模板：

```text
先统一单位；
先算普通访问时间；
再算平均缺页处理时间；
EAT = EAT_normal + 缺页率 * 缺页处理时间。
```

## 14. 易错判断总表

| 说法 | 判断 | 理由 |
|---|---|---|
| 虚拟存储让物理内存真实变大 | 错 | 依靠外存和动态调入 |
| 虚拟存储不依赖局部性 | 错 | 局部性是基础 |
| 基本分页等于请求分页 | 错 | 请求分页才按需调入 |
| 页不在内存就是地址越界 | 错 | 页号合法但不在内存是缺页 |
| 缺页中断后执行下一条指令 | 错 | 重新执行导致缺页的指令 |
| 修改页换出前要写回 | 对 | 否则丢失修改 |
| 请求调页只在缺页时调入 | 对 | demand paging |
| 预调可以完全替代请求调页 | 错 | 预测不准，必须辅以请调 |
| OPT 可以实际实现 | 错 | 需要预知未来 |
| FIFO 可能 Belady 异常 | 对 | 页框更多可能缺页更多 |
| LRU 淘汰最早装入页 | 错 | 淘汰最久未访问页 |
| Clock 看到 r=1 就淘汰 | 错 | 清 0 后给第二次机会 |
| 改进 Clock 优先淘汰 (0,1) | 错 | 先淘汰 (0,0) |
| 工作集大小是访问次数 | 错 | 是不同页面个数 |
| 抖动时 CPU 利用率很高 | 错 | 通常 CPU 低、换页磁盘高 |
| 增加运行进程数可缓解抖动 | 错 | 通常会加重 |
| 虚拟段式便于共享保护 | 对 | 段是逻辑单位 |
| 虚拟段式无外部碎片 | 错 | 段需要连续空间 |
| 虚拟段页式只有缺页中断 | 错 | 还可能连接、越界、越权 |
