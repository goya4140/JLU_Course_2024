# 01 操作系统概述（深度版）

> 本章目标：从“为什么需要操作系统”开始，建立整门课的总地图。你需要理解操作系统的地位、作用、发展历史、基本特性、典型类型、运行环境、用户接口和运行机理，并能处理期末中常见的并发/共享、实时系统、特权指令、用户态/内核态、系统调用等选择题。
>
> 课内边界：本章是总论，不展开进程调度算法、PV 同步、死锁、内存分页、文件系统和 I/O 调度的细节；这些内容在后续章节深入。本章只负责把它们放到操作系统整体框架里。

## 1. 操作系统是什么：硬件之上的“总管家”

### 1.1 操作系统的地位

课件给出的地位：

```text
硬件 / HAL
  ↑
操作系统
  ↑
系统库、其他系统软件
  ↑
应用程序
  ↑
用户
```

更口语一点：

> 操作系统位于硬件之上，所有其他软件之下。

它向下管理硬件，向上提供服务。

【必会】习题课表述：

```text
地位：硬件之上，其他之下
作用：管理资源，服务用户
承上启下
```

考试判断：

- 操作系统不是普通应用程序。
- 操作系统也不是硬件本身。
- 操作系统是最靠近硬件的系统软件之一。
- 应用程序通常不能绕过操作系统直接控制关键硬件。

### 1.2 操作系统的作用

课件给出两个方向：

1. 管理系统中的软硬件资源。
2. 为用户和应用程序提供良好服务。

资源管理包括：

| 资源 | 操作系统要做什么 |
|---|---|
| CPU | 多个可运行程序如何分配处理机 |
| 内存 | 进程空间相对独立，必要时支持共享 |
| 设备 | 分配、驱动、控制、保护 |
| 文件 | 实现文件系统，支持文件操作 |

服务用户包括：

| 界面 | 例子 |
|---|---|
| API/系统调用 | `open`, `read`, `fork` |
| 命令行 | `ls`, `cd`, `cat`, `vi`, `rm`, `mount` |
| GUI | 图形界面 |
| JCL | 作业控制语言 |

【必会】一句话：

```text
操作系统对内管理资源，对外服务用户。
```

### 1.3 操作系统定义

课件定义可整理为：

> 操作系统是位于硬件层之上、所有其他软件层之下的系统软件，是管理系统中各种软硬件资源、方便用户使用计算机系统的程序集合。

考试表述时可以压缩成：

```text
操作系统是管理计算机软硬件资源，并为用户和应用程序提供接口与服务的系统软件。
```

【易错】不要把 OS 定义成“硬件的集合”或“用户程序的集合”。它是程序集合，但这些程序负责资源管理和服务提供。

## 2. 为什么会产生操作系统：从人工操作到自动管理

操作系统不是一开始就有的。它是因为计算机越来越快、设备越来越多、人工操作越来越浪费而逐步出现的。

### 2.1 手工操作阶段

课件时间：

```text
20 世纪 40 年代，无操作系统
```

典型步骤：

1. 程序和数据通过穿孔机打到纸带。
2. 纸带通过光电机装入机器内存。
3. 人在控制台开关启动第一条指令。
4. 运行结果从电传打字机输出。

缺点：

1. 资源独占。
2. 手工操作费时。

直觉解释：

机器运行很快，但人换作业、装纸带、启动、处理输出都很慢。机器越快，人的操作越显得浪费。

课件例子表达的直觉：

```text
机器速度提高后，程序运行时间变短，
但手工操作时间不变，
于是手工时间占比反而更大。
```

考试判断：

- 【重点】手工阶段没有 OS。
- 主要问题是人工干预多、资源独占、效率低。

### 2.2 批处理阶段

批处理的基本思想：

> 把一批作业组织起来，让系统自动连续处理，减少人工干预。

#### 联机批处理

特点：

- 作业由读卡机、磁带机等输入。
- 有监督程序自动控制作业过渡。
- 一个作业处理过程和作业之间切换减少人工干预。

优点：

- 摆脱大量人工干预。

缺点：

- I/O 操作慢，主机等待时间仍然长。

#### 脱机批处理

特点：

- 用卫星机处理输入输出。
- 主计算机更专注于计算。

优点：

- 减少主机等待 I/O 的时间。

缺点：

- 需要人工搬动磁带。
- 需要额外卫星机。

考试判断：

- 【重点】批处理解决了“人工干预”问题，但没有完全解决 CPU 与 I/O 并行问题。

### 2.3 执行系统阶段：通道与中断出现

课件强调：

```text
执行系统阶段是操作系统的初级阶段，
通道和中断技术出现。
```

通道：

- 专门用于控制 I/O 设备的处理机。
- 有自己的指令系统和运控部件。
- 与主机共享内存。
- 接受 CPU 委托执行通道程序。
- I/O 操作可与 CPU 并行。
- I/O 完成时向主机发中断。

中断：

- 设备完成后能通知 CPU。
- CPU 不必一直等待设备。

直觉解释：

以前是：

```text
CPU 等 I/O
```

有了通道和中断后：

```text
CPU 委托通道做 I/O
CPU 去做别的
I/O 完成后中断通知 CPU
```

考试判断：

- 【重点】通道和中断是 OS 能自动管理 I/O、实现并行的重要硬件基础。
- 通道细节第 9 章讲，本章知道“通道是 I/O 处理机”即可。

## 3. 操作系统的完善：多道、分时、实时、通用

### 3.1 多道批处理系统

课件给出的关键句：

```text
单道到多道：不是量的变化，是质的飞跃。
多道批处理出现，标志操作系统走向成熟。
```

直觉解释：

单道系统中：

```text
作业 A 等 I/O 时，CPU 闲着
```

多道系统中：

```text
作业 A 等 I/O 时，CPU 可以运行作业 B
```

目标：

1. 提高资源利用率。
2. 提高吞吐量。

特点：

1. 多道：系统中同时容纳多个作业。
2. 成批：作业分批进入系统。

多道带来的问题：

| 问题 | 后续章节 |
|---|---|
| 互斥 | 第 4 章 |
| 同步 | 第 4 章 |
| 通信 | 第 4 章 |
| 死锁 | 第 5 章 |
| 饥饿/饿死 | 第 5 章 |
| 调度 | 第 3 章 |
| 内存隔离 | 第 6、7 章 |

【重点】多道程序不是“程序越多越好”。程序过多会增加调度、内存和资源竞争开销，甚至导致抖动。

### 3.2 分时系统

背景：

程序员提出联机交互操作要求。

结构：

```text
一台主机 + 多个终端
```

特点：

| 特点 | 含义 |
|---|---|
| 多路性 | 一个主机连接多个终端 |
| 交互性 | 用户以对话方式使用系统 |
| 独占性 | 每个终端用户仿佛独占一台虚拟机 |

典型系统：

- CTSS
- Multics
- UNIX

考试判断：

- 【重点】分时系统强调交互性和响应时间。
- 分时系统通过时间片让多个用户感觉“同时”使用计算机。

### 3.3 实时系统

实时系统背景：

- 工业控制。
- 军事控制。
- 医疗控制。
- 航班订票。
- 联机情报检索。

核心要求：

> 必须在被控对象规定的时间内处理完外部事件。

【必会】真题：

```text
实时操作系统必须在______内处理完来自外部的事件。
答案：被控对象规定的时间。
```

考试判断：

- 实时不是“速度一定最快”。
- 实时的核心是满足时间约束。
- 超过规定时间，即使结果正确，也可能是失败。

### 3.4 通用操作系统

通用操作系统：

> 同时具有分时、实时、批处理功能。

常见组合：

```text
分时（前台） + 批处理（后台）
实时（前台） + 批处理（后台）
```

目标：

1. 提高处理能力。
2. 扩展应用领域。

考试判断：

- 【重点】通用 OS 是多类系统功能的结合，不是单独只服务一种模式。

## 4. 操作系统的发展类型

### 4.1 单用户操作系统

定义：

> 同一时刻仅有一个用户使用的系统。

特点：

```text
单用户，多进程，多线程
```

应用：

- 台式机。
- 笔记本。

【易错】单用户不等于单任务。一个用户也可以同时运行多个进程和线程。

### 4.2 网络操作系统

目标：

1. 相互通信。
2. 资源共享。
3. 提供网络服务。

网络服务例子：

- database server
- ftp server
- email server
- telnet server

特点：

- 资源共享依赖网络。
- 用户通常能意识到不同机器的存在。
- 课件强调 `No Transparent view`。

### 4.3 分布式操作系统

分布式系统特征：

1. 统一的操作系统。
2. 资源进一步共享。
3. 可靠性。
4. 透明性。

目标：

- 进一步共享资源。
- 负载均衡。
- 计算加速。

途径：

- 作业迁移。
- 进程迁移。

考试判断：

- 【重点】分布式 OS 强调透明性，让用户尽量感觉不到多台机器的分散性。
- 网络 OS 更像“多台机器联网并共享服务”，分布式 OS 更像“多台机器被统一 OS 管理成整体”。

### 4.4 多处理机操作系统

多处理机系统：

```text
多个 CPU + 公共内存
```

对称多处理机系统 SMP：

> 多个处理器没有主从关系。

多处理机 OS：

> 有效管理和使用多个 CPU 的操作系统。

复杂性：

- 多个主动体并发执行。
- 需要处理多处理器调度和并发控制。

### 4.5 嵌入式操作系统

定义：

> 嵌入在掌上电脑、通讯设备、车载系统、信息家电等非通用计算机设施上的操作系统。

课件提到特点：

- 常采用微内核结构。
- 核心小。
- 适应范围广。
- 可靠性高。
- 许多功能如文件系统、设备驱动可在应用程序模式运行。
- 可能效率较低。

### 4.6 多媒体操作系统和智能卡操作系统

多媒体操作系统：

- 具有一般操作系统功能。
- 支持多媒体信息采集、编辑、播放和传输。
- 课件备注：它不是一种完全独立的 OS 类型，更像现代 OS 的一种特性。

智能卡操作系统：

- 智能卡中有 CPU、ROM 等。
- 可支持 Java applet 下载和执行。
- 支持必要的资源管理。

考试判断：

- 这部分真题牵引弱，知道概念即可。

## 5. 操作系统的四个基本特性

课件列出：

```text
并发性
共享性
异步性（随机性）
虚拟性
```

真题问过：

> 现代操作系统中最基本的两个特征是？

答案：

```text
并发和共享
```

### 5.1 【必会】并发性

课件定义：

> 多个程序在宏观上同时向前推进。

并发 vs 并行：

| 概念 | 含义 |
|---|---|
| 并发 concurrent | 宏观上同时推进，微观上可能交替执行 |
| 并行 parallel | 物理上同时执行，通常需要多个处理器/核心 |

例子：

单 CPU 上多个进程轮流占用 CPU：

```text
P1 运行一会儿
P2 运行一会儿
P1 再运行一会儿
```

这是并发，不一定是并行。

【易错】并发不是“同时精确执行每条指令”。它强调一段时间内多个程序都在推进。

### 5.2 【必会】共享性

课件定义：

> 多个程序共用系统中的各种软硬件资源，并在操作系统协调和控制下使用。

共享对象：

- CPU。
- 内存。
- 文件。
- 设备。
- 数据结构。

直觉：

多个进程都想用打印机、文件、内存和 CPU，如果没有 OS 协调，就会冲突。

【重点】并发和共享互为条件：

- 因为多个程序并发，所以它们会共享资源。
- 因为资源共享，所以必须有并发控制、保护和调度。

### 5.3 异步性

课件定义：

> 多个程序以不可预知的速度向前推进。

直觉：

进程什么时候被调度、什么时候等待 I/O、什么时候被唤醒，都可能受外部事件和调度影响。

例子：

```text
进程 A 发出磁盘 I/O
磁盘什么时候完成不完全由进程 A 决定
进程 A 何时重新获得 CPU 也不完全确定
```

考试判断：

- 【重点】异步性又叫随机性。
- 它是并发环境中的重要特征。

### 5.4 虚拟性

课件例子：

| 实际资源 | 虚拟效果 |
|---|---|
| 单个实 CPU | 多个虚拟 CPU |
| 内存 + 外存 | 虚拟存储 |
| 独占设备 + 共享设备 | 虚拟设备 |

直觉：

虚拟性就是：

> 通过操作系统管理，把一个或有限个物理资源改造成多个逻辑资源，让用户感觉资源更多、更方便或更独立。

考试判断：

- 虚拟不是“假的没用”，而是“用软件管理形成逻辑资源”。
- 虚拟内存和 SPOOLing 虚拟设备都是典型例子。

## 6. 操作系统运行环境

习题课明确列出运行环境：

```text
定时装置、系统栈、特权指令与非特权指令、
处理机状态及状态转换、地址映射机构、存储保护设施
```

硬件支持：

```text
中断装置、通道与 DMA 控制器、I/O 保护
```

这些东西共同保证：

> 操作系统能获得控制权，能保护自己，能隔离用户程序，能安全控制硬件。

## 7. 定时装置：操作系统如何定期拿回 CPU

### 7.1 绝对时钟

课件：

- 记载实际时间。
- 不发中断。
- 系统操作员可以修改。
- 一般用户可以使用。

### 7.2 间隔时钟

课件：

- 定时发生中断。
- 一般间隔单位为毫秒。
- 是实现多道程序的基础。

关键句：

```text
间隔时钟保证操作系统获得控制权。
```

直觉解释：

如果没有时钟中断，用户程序一旦占用 CPU，就可能一直运行，操作系统很难强制收回控制权。

有了间隔时钟：

```text
每隔一段时间 -> 时钟中断 -> CPU 进入内核 -> OS 可调度其他进程
```

考试判断：

- 【重点】间隔时钟是多道程序和分时系统的重要硬件基础。
- 其他中断也能进入 OS，但不保证何时发生；间隔时钟能周期性提供控制机会。

## 8. 系统栈

系统栈作用：

1. 保存中断现场。
2. 保存函数调用返回点。
3. 保存参数、局部变量、返回值。

位置：

```text
操作系统区域
```

直觉：

用户程序有自己的用户栈；进入内核处理系统调用或中断时，内核需要可靠的栈保存上下文和执行内核代码，这就是系统栈的意义。

考试判断：

- 系统栈不是普通用户栈。
- 中断/系统调用进入内核时，需要保存现场。

## 9. 特权指令与非特权指令

### 9.1 特权指令

课件定义：

> 只有在管态才能执行的指令，通常会影响系统状态。

例子：

- 关中断。
- 置程序状态字。
- 停机。
- I/O 指令。
- 设置时钟。
- 设置中断屏蔽。
- 启动设备。

【必会】考试判断：

如果某指令会影响系统全局状态、硬件控制或系统安全，通常是特权指令。

### 9.2 非特权指令

课件定义：

> 所有程序可用，不影响系统状态。

例子：

- 取数。
- 四则运算。
- 普通逻辑运算。
- 读时钟指令常作为非特权操作考查。

真题：

> 以下属于非特权指令的是？
>
> A. 启动某设备  
> B. 设置中断屏蔽  
> C. 设置时钟指令  
> D. 读时钟指令

答案：

```text
D. 读时钟指令
```

解析：

- 启动设备、设置中断屏蔽、设置时钟都会影响系统状态或硬件控制。
- 读时钟只是读取信息，一般不破坏系统状态。

【易错】“读时钟”和“设置时钟”不同。读通常可开放，设置必须保护。

## 10. 处理机状态：用户态与内核态

### 10.1 两种状态

课件：

| 状态 | 也叫 | 权限 |
|---|---|---|
| 系统态 | 管态、核态 | 可执行特权指令 |
| 用户态 | 目态、常态 | 不能执行特权指令 |

现代 PC 支持多个权限环：

```text
R0 权限最强
R3 权限最弱
```

操作系统内核通常在高权限级运行，用户程序在低权限级运行。

### 10.2 状态转换

课件：

```text
管态 -> 目态：置程序状态字，特权指令
目态 -> 管态：中断，trap
```

直觉：

内核可以主动把 CPU 交给用户程序；用户程序不能随便把自己提升为内核，只能通过受控入口进入内核。

进入内核的典型事件：

1. 系统调用：用户主动请求服务。
2. 异常：如除零、缺页、非法指令。
3. 外部中断：如时钟中断、I/O 中断。

### 10.3 【必会】真题：哪些会导致用户态切换到内核态

题目：

```text
Ⅰ. 整数除以零
Ⅱ. sin() 函数调用
Ⅲ. read 系统调用
```

判断：

- 整数除以零：异常，会进入内核态。
- `sin()`：普通库函数调用，通常仍在用户态。
- `read`：系统调用，会进入内核态。

答案：

```text
仅 Ⅰ 和 Ⅲ
```

【易错】库函数不等于系统调用。库函数可能封装系统调用，但 `sin()` 这种数学库函数通常只是用户态计算。

## 11. 地址映射与存储保护

### 11.1 地址映射机构

课件：

```text
逻辑地址 -> 物理地址
```

| 地址 | 含义 |
|---|---|
| 逻辑地址/虚地址 | 程序中产生的地址 |
| 物理地址/实地址 | 主存储器中的真实地址 |

直觉：

用户程序以为自己在访问某个地址，但真正访问哪块物理内存，需要地址映射机构转换。

后续展开：

- 第 6 章讲页式、段式、段页式地址映射。
- 第 7 章讲虚拟存储和缺页/缺段。

### 11.2 存储保护设施

课件目标：

防止应用程序：

1. 侵犯操作系统空间。
2. 侵犯其他用户空间。

保护方式：

- 地址检查。
- 越界检查。
- 越权检查。

考试判断：

- 【重点】内存保护是多道程序能安全运行的前提。
- 没有保护，一个用户程序可能破坏内核或其他进程。

## 12. 中断装置、通道、DMA 与 I/O 保护

### 12.1 中断装置

课件定义：

> 发现并响应中断的硬件机构。

中断发生时：

```text
当前 PSW、PC -> 系统栈
中断向量中的 PSW、PC -> 寄存器
```

直觉：

中断装置负责把 CPU 从当前程序带到中断处理程序入口，并保存必要现场。

考试判断：

- 【重点】中断装置是硬件，中断处理程序是软件。
- 第 3 章会深入中断分类和中断向量。

### 12.2 通道与 DMA

课件在第 1 章已给出简要区分：

| 项 | 通道 | DMA |
|---|---|---|
| 定位 | 负责 I/O 操作的处理机 | 直接内存访问控制器 |
| 指令系统 | 有通道指令系统 | 没有独立指令系统 |
| 能力 | 读写、控制、转移等 | 简单块传输 |

考试判断：

- 第 9 章会深入。
- 第 1 章先记住：通道独立性更强，DMA 负责块传输。

### 12.3 I/O 保护

课件：

> 定义所有 I/O 指令为特权指令。

原因：

1. 方便系统统一管理设备。
2. 防止多个用户程序直接操作设备造成冲突。
3. 防止用户程序破坏系统状态。

考试判断：

- 【重点】用户程序不能直接执行 I/O 指令，需要通过系统调用请求 OS。

## 14. 操作系统运行机理

课件给出的运行机理可以理解为：

```text
程序 1 正在运行
发生中断
硬件保存关键现场
进入操作系统
中断处理
保存程序 1 完整现场
选择程序 2
恢复程序 2 现场
程序 2 继续运行
```

这就是后面“进程切换”的直觉基础。

【重点】关键动作：

1. 中断让 CPU 进入 OS。
2. OS 保存当前程序现场。
3. OS 可以调度选择另一个程序。
4. OS 恢复另一个程序现场。
5. CPU 从另一个程序继续执行。

考试判断：

- 用户程序不能自己完成真正的进程切换。
- 进程切换需要内核介入，涉及现场保存和恢复。

## 17. 课内题型归纳与逐题解析

### 17.1 【题型】现代操作系统最基本特征

题目：

```text
现代操作系统中最基本的两个特征是？
```

答案：

```text
并发和共享
```

解析：

- 并发：多个程序宏观上同时推进。
- 共享：多个程序共同使用系统资源。
- 虚拟和异步也重要，但常被看作建立在并发和共享基础上的特征。

### 17.2 【题型】实时操作系统时间约束

题目：

```text
实时操作系统必须在______内处理完来自外部的事件。
```

答案：

```text
被控对象规定的时间
```

解析：

实时系统的关键不是“一个机器周期”“周转时间”或“时间片”，而是外部控制对象规定的截止时间。

### 17.3 【题型】特权指令与非特权指令

题目：

```text
以下属于非特权指令的是：
启动设备、设置中断屏蔽、设置时钟、读时钟
```

答案：

```text
读时钟
```

判断模板：

```text
会改变系统状态或控制硬件 -> 特权指令
只是普通计算或读取信息 -> 非特权指令
```

### 17.4 【题型】用户态到内核态

题目：

```text
整数除以零、sin() 函数调用、read 系统调用，
哪些会导致用户进程从用户态切换到内核态？
```

答案：

```text
整数除以零 + read 系统调用
```

解析：

- 除零是异常。
- `read` 是系统调用。
- `sin()` 是普通库函数调用。

### 17.5 【题型】操作系统接口

题目：

```text
操作系统提供给编程人员的接口是什么？
```

答案：

```text
系统调用
```

解析：

命令行/GUI 面向普通用户；系统调用面向程序员；库函数可能封装系统调用，但不等于系统调用。

### 17.6 【题型】操作系统地位和作用

常见问法：

```text
操作系统在计算机系统中处于什么位置？
操作系统的主要作用是什么？
```

答题模板：

```text
操作系统位于硬件之上、其他软件之下，是承上启下的系统软件。
它一方面管理 CPU、内存、文件、设备等软硬件资源，
另一方面为用户和应用程序提供接口与服务。
```

### 17.7 【题型】分时、实时、批处理区分

快速判断：

| 关键词 | 系统类型 |
|---|---|
| 成批、吞吐量、资源利用率 | 多道批处理 |
| 多终端、交互、时间片、虚拟机感觉 | 分时系统 |
| 外部事件、截止时间、被控对象规定时间 | 实时系统 |
| 同时具有分时/实时/批处理功能 | 通用操作系统 |

### 17.8 【题型】系统运行环境

常见问法：

```text
为什么需要时钟中断？
为什么需要内核态和用户态？
为什么 I/O 指令要定义为特权指令？
```

答题模板：

```text
时钟中断保证操作系统能定期获得控制权；
内核态/用户态隔离普通程序和操作系统，防止用户程序执行危险操作；
I/O 指令定义为特权指令，使设备由操作系统统一管理，避免冲突和破坏。
```

## 18. 易混概念总表

| 易混概念 | 区别 |
|---|---|
| 操作系统 vs 应用程序 | OS 管理资源并提供服务；应用程序完成用户任务 |
| 管理资源 vs 服务用户 | 前者对内，后者对外 |
| 系统软件 vs 硬件 | OS 是系统软件，不是硬件 |
| 联机批处理 vs 脱机批处理 | 联机仍由主机直接处理 I/O；脱机用卫星机减少主机等待 |
| 执行系统 vs 多道批处理 | 执行系统是初级 OS；多道批处理标志 OS 走向成熟 |
| 多道 vs 多用户 | 多道强调多个作业/程序在系统中；多用户强调多个用户 |
| 并发 vs 并行 | 并发宏观同时推进；并行物理同时执行 |
| 并发 vs 共享 | 并发强调程序推进；共享强调资源共同使用 |
| 虚拟 vs 共享 | 虚拟把物理资源抽象成多个逻辑资源；共享让多个程序共同使用资源 |
| 异步 vs 随机 | 本章中异步性也称随机性，指推进速度不可预知 |
| 批处理 vs 分时 | 批处理重吞吐量；分时重交互响应 |
| 分时 vs 实时 | 分时追求及时响应用户；实时必须满足外部时间约束 |
| 网络 OS vs 分布式 OS | 网络 OS 用户能感知多机；分布式 OS 强调整体透明性 |
| 单用户 vs 单任务 | 单用户也可以多进程、多线程 |
| 绝对时钟 vs 间隔时钟 | 绝对时钟记实际时间；间隔时钟周期中断 |
| 特权指令 vs 非特权指令 | 特权指令只能内核态执行；非特权指令用户态也可执行 |
| 设置时钟 vs 读时钟 | 设置影响系统状态，是特权；读取通常是非特权 |
| 用户态 vs 内核态 | 用户态权限低；内核态可执行特权指令 |
| 系统调用 vs 库函数 | 系统调用进入内核；库函数通常是用户态函数，可能封装系统调用 |
| 中断装置 vs 中断处理程序 | 中断装置是硬件；中断处理程序是软件 |
| 通道 vs DMA | 通道是 I/O 处理机；DMA 是直接内存传输控制器 |
| shell vs 内核 | shell 是用户态命令解释程序；内核是 OS 核心 |
| 命令接口 vs 系统调用 | 命令接口面向用户；系统调用面向程序 |

- [ ] 为什么手工操作阶段效率低。
- [ ] 为什么批处理减少人工干预但仍有 I/O 等待。
- [ ] 为什么通道和中断推动了执行系统。
- [ ] 为什么间隔时钟能保证 OS 获得控制权。
- [ ] 系统栈保存什么。
- [ ] 地址映射为什么需要硬件支持。
- [ ] 存储保护为什么是多道程序安全运行的前提。
- [ ] 为什么所有 I/O 指令要定义为特权指令。
- [ ] shell 为什么不属于内核。
- [ ] 操作系统的进程观点、资源管理观点、虚拟机观点。

### 19.3 【易错】考前专门过一遍

- [ ] 并发不等于并行。
- [ ] 实时不等于速度最快，而是满足截止时间。
- [ ] 单用户不等于单任务。
- [ ] 分布式 OS 比网络 OS 更强调透明性。
- [ ] 用户程序不能直接执行 I/O 指令。
- [ ] 进程切换不能在纯用户态完成。
- [ ] 库函数不是系统调用本身。
- [ ] 中断装置不是中断处理程序。
- [ ] DMA 不是通道。
- [ ] 读时钟和设置时钟不要混。

## 20. 本章一句话总结

操作系统概述的主线是：

```text
操作系统位于硬件之上、其他软件之下，
对内管理 CPU、内存、文件、设备等资源，
对外通过命令、GUI、JCL、系统调用服务用户和程序。
它因人工操作和 I/O 等待造成的低效率而产生，
在通道、中断、多道程序支持下走向成熟，
并通过并发、共享、虚拟、异步组织现代计算机系统。
为了安全控制硬件，它依赖内核态/用户态、特权指令、
时钟中断、系统栈、地址映射和存储/I/O 保护等运行环境。
```

期末最值得抓住的硬考点是：

```text
OS 地位与作用、并发和共享、实时系统时间约束、
特权/非特权指令、用户态到内核态、系统调用 vs 库函数。
```
