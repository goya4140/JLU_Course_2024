## 从 0 理解

如果没有操作系统，用户程序要自己管理 CPU、内存、磁盘、打印机、键盘和错误处理。这样既难写，也不安全。操作系统把硬件包装成更容易使用的接口，并决定谁能在什么时候使用哪些资源。

“管理资源”是对内的：CPU 调度、内存分配、文件组织、设备分配。“服务用户”是对外的：提供系统调用、命令接口和程序运行环境。

多道程序之后，多个程序同时在系统中推进，于是操作系统必须处理四个基本特征：

| 特征 | 直觉 | 例子 |
|---|---|---|
| 并发 | 一段时间内多个程序都在推进 | CPU 在多个进程间切换 |
| 共享 | 多个程序共同使用资源 | 多个进程访问文件、内存、设备 |
| 虚拟 | 把一个物理资源变成多个逻辑资源 | 虚拟内存、SPOOLing 虚拟打印机 |
| 异步 | 每个程序推进速度不可预知 | I/O 完成时间不确定，进程时快时慢 |

## 核心机制/算法/公式

### 用户态与内核态

用户态运行普通用户程序，不能执行危险操作。内核态运行操作系统内核，可以执行特权指令、访问关键资源。

会导致用户态进入内核态的典型事件：

- 系统调用：用户主动请求内核服务，如 `read`。
- 异常：程序运行出错或需要内核介入，如除零、缺页。
- 外部中断：外设或时钟打断当前执行。

不会导致进入内核态的普通函数调用：如 `sin()`，它只是用户程序中的库函数调用。

### 特权指令与非特权指令

特权指令只能在内核态执行，典型包括启动设备、设置中断屏蔽、设置时钟、修改状态寄存器等。非特权指令用户态也可以执行，如读时钟指令通常可作为非特权操作考查。

### 系统调用

系统调用是操作系统提供给程序员的接口。库函数不是操作系统接口本身，库函数可能在内部封装系统调用。

## 易错点与区分表

| 易混点 | 正确理解 |
|---|---|
| 系统调用 vs 库函数 | 系统调用进入内核；库函数通常仍在用户态，可能封装系统调用 |
| 中断 vs 异常 | 中断多来自 CPU 外部；异常来自当前指令执行过程 |
| 用户态能否进程切换 | 进程切换由内核完成，不能在用户态完成 |
| 并发 vs 并行 | 并发是宏观同时推进；并行是物理上同时执行 |
| 共享 vs 虚拟 | 共享强调共同使用同一资源；虚拟强调把物理资源抽象成多个逻辑资源 |

## 题型与解析

### 选择题：操作系统接口

题干：操作系统提供给编程人员的接口是什么？

答案：系统调用。

解析：编程人员通过系统调用请求文件、进程、设备、内存等内核服务。库函数、高级语言、子程序都不是操作系统提供的核心接口。

### 选择题：用户态到内核态

题干：整数除以零、`sin()` 函数调用、`read` 系统调用，哪些会导致用户态切换到内核态？

答案：整数除以零和 `read` 系统调用。

解析：除零是异常，`read` 是系统调用；`sin()` 是普通库函数调用。

### 选择题：现代操作系统基本特征

真题出现“现代操作系统中最基本的两个特征”。按课内与常见教材口径，优先选并发和共享。虚拟、异步也重要，但通常是由并发共享进一步带来的特征。
