## 0. 本章定位与考试价值

文件系统给用户一个温和的世界：

```text
open -> read -> write -> close
```

设备管理面对的是比较粗糙的现实：

```text
设备很慢、种类很多、控制方式不同、一次只能服务有限请求、
还会中断、排队、缓存、出错、竞争、需要驱动程序。
```

所以第 9 章的核心问题是：

> 用户发出一个抽象 I/O 请求后，操作系统如何把它变成设备能执行的动作，并尽量让 CPU、内存和设备都高效工作？

本章考试价值很高，尤其是期末真题已经明确出现：

1. 通道、DMA、设备控制器的区分。
2. 缓冲与缓存的区分。
3. 用户程序发出磁盘 I/O 请求后的处理流程。
4. SPOOLing 虚拟设备。
5. 磁盘引臂调度大题：N-Step LOOK。
6. 磁头粘性问答。
7. 习题课文件与设备综合题：`read`、cache、I/O 链、缓冲区、中断唤醒。

## 1. 设备及其分类

### 1.1 按用途分类

课件中按用途把设备分成：

| 类型 | 例子 | 直觉 |
|---|---|---|
| 存储型设备 | 磁盘、磁带、光盘 | 保存大量数据 |
| I/O 型设备 | 键盘、鼠标、显示器、扫描仪、打印机 | 与用户或外部世界交互 |
| 网络设备 | 网卡、交换机等 | 进行网络通信 |

考试判断：

- 【重点】磁盘既是外存设备，也是 I/O 管理重点。
- 打印机通常是低速独占设备，是 SPOOLing 的经典例子。
- 网卡属于网络设备，但也会产生 I/O 中断和缓冲需求。

### 1.2 按管理方式分类

课件还按管理方式分为：

| 类型 | 特点 | 例子/理解 |
|---|---|---|
| 共享型设备 | 多个进程的 I/O 操作可以交叉进行 | 磁盘，常以块为单位排队 |
| 独占型设备 | 一段时间只适合由一个进程独占 | 打印机、磁带机等 |
| 字符型独占设备 | 以字符为单位，多个进程输出不宜交叉 | 终端、打印机等 |

直觉解释：

磁盘请求是：

```text
进程 A 读块 15
进程 B 读块 90
进程 C 写块 31
```

这些请求可以排队优化，交错服务。

但打印机如果让多个进程字符级交错输出，就可能变成：

```text
张三报告李四代码王五日志混在一起
```

所以打印机通常要独占或通过 SPOOLing 虚拟化。

考试判断：

- 【重点】共享型设备不等于“同时物理执行多个请求”，而是多个进程可以提交请求，由系统排队调度。
- 【易错】独占型设备不是永远不能共享，而是直接共享效率低或语义混乱，可通过虚拟设备技术改善。

## 2. 设备的物理特性

### 2.1 I/O 设备的基本物理差异

不同设备的差异包括：

1. 数据单位不同：字符、块、帧等。
2. 速度不同：键盘很慢，磁盘较快，内存更快。
3. 控制方式不同：端口、控制器、DMA、通道。
4. 访问方式不同：顺序访问、随机访问。
5. 是否可共享不同：磁盘可排队共享，打印机不适合直接交叉共享。

设备管理要屏蔽这些差异，让上层尽量使用统一接口。

### 2.2 磁带：一维地址与顺序结构

课件提到磁带：

- 结构类似：头标、信息块、信息块、尾标。
- 操作包括：反绕、正向查找、反向查找、读、写。
- 地址是一维的。
- 文件常采用顺序结构，一个文件占若干连续块。

考试判断：

- 【重点】磁带适合顺序访问，不适合随机访问。
- 磁盘调度大题通常不是考磁带，而是考磁盘。

### 2.3 磁盘：三维地址与一维块号

磁盘物理地址常包括：

```text
柱面号 i
盘面号 j
扇区号 k
```

课件给出三维地址到一维块号的转换：

```text
b = i * m * n + j * n + k
```

其中：

- `i`：柱面号。
- `j`：盘面号。
- `k`：扇区号。
- `m`：盘面数。
- `n`：每个盘面的扇区数。

一维块号到三维地址：

```text
i = b / (m * n)
j = (b mod (m * n)) / n
k = (b mod (m * n)) mod n
```

这里 `/` 表示整除。

【题型】如果给 `l=2, m=3, n=3`，块号 `b=14`：

```text
i = 14 / (3*3) = 1
余数 = 14 mod 9 = 5
j = 5 / 3 = 1
k = 5 mod 3 = 2
```

所以三维地址是：

```text
柱面 1，盘面 1，扇区 2
```

考试判断：

- 【重点】块号连续编号时，通常让相邻块物理上尽量近。
- 三维/一维转换不是本章最高频真题，但课件明确出现，会做公式即可。

### 2.4 扇区交错编号

课件展示了：

- 不考虑读写延迟的扇区编号。
- 单交错编号。
- 双交错编号。

直觉解释：

磁盘读完一个扇区后，CPU/控制器处理也需要一点时间。  
如果下一个逻辑扇区正好紧挨着，等系统准备好时，磁盘可能已经转过头了，需要再等一圈。

交错编号的目的：

> 让逻辑上连续的扇区在物理上隔开一点，使处理延迟之后，磁头刚好遇到下一个需要读写的扇区。

考试判断：

- 【重点】交错编号是为了减少旋转等待，不是为了减少寻道。

## 3. I/O 传输方式总览

课件列出四类：

| 方式 | CPU 参与程度 | 特点 |
|---|---|---|
| 程序查询方式 | CPU 全程盯着 | 简单但浪费 CPU |
| 中断驱动方式 | CPU 启动后去做别的，完成后被通知 | CPU 与设备可并行 |
| DMA 方式 | CPU 设置参数，DMA 控制器搬运块数据 | 减少 CPU 参与 |
| 通道方式 | 通道作为 I/O 处理机执行通道程序 | I/O 能力更强 |

考试主线：

```text
查询 -> 中断 -> DMA -> 通道
CPU 负担逐渐降低，I/O 并行能力逐渐增强。
```

## 4. 程序查询方式

### 4.1 直觉解释

程序查询像你在洗衣机旁边一直问：

```text
洗完了吗？
洗完了吗？
洗完了吗？
```

CPU 启动设备后反复检查设备状态寄存器，直到设备完成。

### 4.2 特点

优点：

- 实现简单。
- 对硬件要求低。

缺点：

1. CPU 与设备串行工作。
2. CPU 大量时间浪费在查询上。
3. 设备慢时尤其低效。

考试判断：

- 【重点】看到“CPU 反复查询设备状态”“忙等”，对应程序查询方式。
- 【易错】程序查询不是没有控制器，而是 CPU 没有释放出来做其他工作。

## 5. 中断驱动方式

### 5.1 直觉解释

中断驱动像你启动洗衣机后先去学习，洗衣机洗完会响铃通知你。

流程：

```text
CPU 启动设备
CPU 执行其他程序
设备完成后发中断
CPU 执行中断处理程序
```

### 5.2 优缺点

优点：

- CPU 与设备可以并行工作。
- 不用 CPU 一直忙等。

缺点：

- 设备很多或传输粒度很小时，中断次数太多。
- 中断处理本身有上下文保存、切换和恢复开销。

考试判断：

- 【重点】中断驱动提高 CPU 和设备并行性。
- 【易错】中断不是传输数据本身的唯一方式，它主要负责通知完成、错误或状态变化。

## 6. DMA 方式

### 6.1 为什么需要 DMA

如果每传一个字节都让 CPU 参与：

```text
设备 -> CPU -> 内存
```

CPU 会被大量数据搬运拖住。

DMA 的直觉：

> CPU 只负责设置传输参数，真正的数据块搬运由 DMA 控制器完成。

### 6.2 DMA 课件流程

课件中的 DMA 大致流程：

1. CPU 对 DMA 控制器编程，设置内存地址、计数、控制信息，同时启动设备。
2. DMA 控制器向磁盘控制器发出读写请求，并把内存地址放到地址总线上。
3. 磁盘控制器把数据传到内存指定单元。
4. 磁盘控制器向 DMA 控制器发送应答。
5. DMA 控制器修改地址寄存器和计数器。
6. 计数器为 0 时，DMA 控制器向 CPU 发出中断。

### 6.3 DMA 的考试判断

- 【重点】DMA 控制器负责块 I/O 的直接数据传输。
- CPU 不是完全不参与：启动前要设置 DMA，结束后要处理中断。
- DMA 减少的是 CPU 在数据搬运过程中的参与。

【易错】真题问：

> 独立于中央处理器，专门负责数据 I/O 传输工作的处理单元是什么？

答案是：

```text
通道
```

不是 DMA 控制器。

解释：

- DMA 控制器是硬件控制器，负责直接内存传输。
- 通道更像专门的 I/O 处理机，可以执行通道程序，独立性更强。

## 7. 通道方式

### 7.1 通道是什么

课件定义：

> 通道是负责 I/O 操作的处理机。

通道有自己的指令系统，基本操作包括：

- 控制
- 读
- 写
- 转移
- 结束

通道指令格式包括：

```text
(操作码, 传输量, 特征位, 地址)
```

### 7.2 通道相关部件

课件提到：

| 缩写 | 含义 |
|---|---|
| CAW | 通道地址字 |
| CCW | 通道命令字 |
| CSW | 通道状态字 |
| CDW | 通道数据字 |

通道程序由 CCW 指令序列组成，可以控制若干设备进行多次 I/O 传输。

### 7.3 通道程序执行过程

简化流程：

```text
按 CAW 取通道命令 -> CCW
CAW 加 1
判断是否通道结束命令
执行命令
结束时向 CPU 发中断
```

### 7.4 通道类型

课件列出三类通道：

| 类型 | 适用设备 | 特点 |
|---|---|---|
| 字节多路通道 | 多台低速外围设备 | 多个非分配型子通道 |
| 数组选择通道 | 多台高速设备 | 一个分配型子通道 |
| 数组多路通道 | 多台高速设备 | 多个非分配型子通道 |

考试判断：

- 【必会】通道是专门负责 I/O 的处理机。
- 【易错】通道比 DMA 独立性更强，能执行通道程序。

## 8. 用户 I/O 请求的正确处理流程

真题考过：

> 用户程序发出磁盘 I/O 请求后，系统的正确处理流程是？

正确顺序：

```text
用户程序 -> 系统调用处理程序 -> 设备驱动程序 -> 中断处理程序
```

### 8.1 为什么是这个顺序

1. 用户程序不能直接操作设备。
   - 启动设备通常是特权操作。
   - 用户程序只能通过系统调用进入内核。

2. 系统调用处理程序先接管。
   - 检查参数是否合法。
   - 检查权限、文件状态、缓冲区地址等。
   - 把抽象请求交给对应设备层。

3. 设备驱动程序启动设备。
   - 把通用 I/O 请求转换成设备命令。
   - 填写控制器寄存器、DMA 参数或通道程序。
   - 把请求挂入 I/O 队列。

4. 设备完成后才有中断处理程序。
   - 中断处理程序收尾。
   - 标记 I/O 完成。
   - 唤醒等待进程。

【必会】考试口诀：

```text
先 syscall 进内核，再 driver 启设备，完成以后 interrupt 收尾。
```

【易错】中断处理程序不是用户刚发请求时立刻执行，而是设备完成或出错后触发。

## 9. 设备分配与去配

### 9.1 为什么需要设备分配

设备是资源，多个进程可能竞争：

```text
进程 A 要打印
进程 B 也要打印
进程 C 要读磁盘
```

操作系统要决定：

1. 能不能分配。
2. 分配哪台设备。
3. 需要哪个控制器/通道。
4. 请求完成后如何释放。

### 9.2 相关数据结构

课件中出现：

| 结构 | 含义 | 作用 |
|---|---|---|
| SDT | 系统设备表 | 按设备类记录总数、等待队列、UCB 指针 |
| UCB | 设备控制块 | 描述某台设备的状态、标识、连接通道、占有进程 |
| CCB | 通道控制块 | 描述通道状态、类型、占有进程 |

直觉：

```text
SDT：系统有哪些类设备？
UCB：具体某台设备现在空闲还是忙？
CCB：通道现在能不能用？
```

### 9.3 独占型设备分配流程

课件给出独占型设备活动：

```text
申请 -> 使用 -> 使用 -> ... -> 使用 -> 释放
```

申请：

1. 根据设备类查 SDT。
2. 对该类设备信号量执行 `P(Sm)`。
3. 查 UCB 表，找空闲设备并分配。

使用：

1. 分配通道。
2. I/O 传输。
3. 去配通道。

释放：

1. 找 SDT 对应入口。
2. 查 UCB 表，去配设备。
3. 对信号量执行 `V(Sm)`。

考试判断：

- 【重点】独占型设备要有申请和释放。
- 【重点】共享型设备通常每次 I/O 分配通道、传输、释放通道，不长期独占设备。

## 10. 设备驱动

### 10.1 设备驱动的定位

设备驱动程序处在：

```text
通用 I/O 子系统
  -> 设备驱动程序
    -> 设备控制器/通道/DMA
      -> 真实设备
```

它的作用是：

> 把操作系统的抽象 I/O 请求转换成具体设备能理解的命令。

### 10.2 课件中的设备驱动工作

课件提到：

1. 编制通道程序。
2. 设备启动。
3. 通道启动。
4. 处理中断。

通道程序可以静态编制，也可以动态生成。

考试判断：

- 【必会】真题 I/O 流程中，设备驱动在系统调用处理程序之后、中断处理程序之前。
- 设备驱动不是应用程序，而是操作系统内核中和具体设备相关的部分。

## 11. 磁盘调度：为什么要调度磁头

### 11.1 磁盘访问时间由什么组成

课件给出磁盘块访问时间：

```text
访问时间 = 寻道时间 + 旋转延迟 + 传输时间
```

含义：

| 项 | 含义 |
|---|---|
| 寻道时间 | 磁头移动到指定柱面/磁道 |
| 旋转延迟 | 等目标扇区转到磁头下 |
| 传输时间 | 真正读写该扇区 |

其中最容易通过调度优化的是：

```text
寻道时间
```

因为不同请求服务顺序会导致不同磁头移动量。

### 11.2 磁盘调度考虑因素

课件列出：

1. 公平性。
2. 防止饿死。
3. 高效性。
4. 减少磁盘引臂移动量。

考试判断：

- 【重点】磁盘调度不是单纯追求移动量最小，也要避免饥饿。
- 真题选择考过 SSTF 可能饥饿。

---

## 12. FCFS 与 SSTF

### 12.1 FCFS：先来先服务

直觉：

请求按到达顺序排队：

```text
130 -> 42 -> 180 -> 15 -> ...
```

优点：

- 公平。
- 简单。
- 不会饥饿。

缺点：

- 磁头移动量可能很大。

考试判断：

- 【重点】FCFS 公平但效率可能差。

### 12.2 SSTF：最短寻找时间优先

直觉：

每次选择离当前磁头最近的请求。

优点：

- 通常比 FCFS 减少磁头移动。

缺点：

- 可能饥饿。
- 可能出现磁头粘性。

【必会】真题选择：

> 以下算法中，可能出现“饥饿”现象的是？

答案：

```text
最短寻找时间优先 SSTF
```

原因：

如果磁头附近不断有新请求到来，远处请求可能长期得不到服务。

## 13. SCAN、LOOK、C-SCAN、C-LOOK

### 13.1 SCAN：电梯算法

直觉：

像电梯一样，磁头沿一个方向移动，沿途服务请求，到尽头后反向。

特点：

- 移动到磁盘端点再反向。
- 比 SSTF 更公平。
- 不容易让远端请求长期饿死。

### 13.2 LOOK

LOOK 是 SCAN 的改进：

> 不必移动到没有请求的磁盘尽头，只移动到当前方向最远的请求处再反向。

例：

当前磁头 53，请求：

```text
15, 42, 68, 97, 108, 130, 180
```

若当前向小编号方向移动：

- SCAN 可能走到 0 再回头。
- LOOK 走到 15 就回头。

考试判断：

- 【必会】SCAN 到端点，LOOK 到最后一个请求。
- 【易错】LOOK 不是不改变方向，而是不走无请求的端点。

### 13.3 C-SCAN

C-SCAN 是循环扫描：

> 磁头只在一个方向上服务请求，到端点后快速回到另一端，再继续同方向服务。

特点：

- 各磁道等待时间更均匀。
- 回程是否计算移动量，要看题目和教材口径，考试中要说明假设。

### 13.4 C-LOOK

C-LOOK 是 LOOK 的循环版本：

> 只移动到当前方向最后一个请求，然后跳到另一端还有请求的位置，继续同方向服务。

考试判断：

- 【重点】C-SCAN 到端点，C-LOOK 到请求边界。
- 【易错】C-LOOK 的“跳转”在画序列时要写出来，是否计移动量按题目口径。

## 14. N-Step SCAN/LOOK

### 14.1 为什么需要 N-Step

普通 SCAN/LOOK 的问题：

如果扫描过程中不断有新请求插入当前队列，磁头可能被新请求“吸住”，导致某些请求等待过久。

N-Step 思想：

> 把请求队列按长度 N 分成若干子队列，每次只处理当前子队列，新来的请求进入后续队列。

### 14.2 课件例子说明

课件例子：

```text
磁道：0-99
当前：20
方向：向内
N = 4
请求：12, 5, 7, 30, 60, 77, 13, 26, 61, 80, 53, 66
```

分组：

```text
第 1 组：12, 5, 7, 30
第 2 组：60, 77, 13, 26
第 3 组：61, 80, 53, 66
```

课件给出的服务序列：

```text
20 -> 30 -> 12 -> 7 -> 5 -> 13 -> 26 -> 60 -> 77 -> 80 -> 66 -> 61 -> 53
```

可见：

- 每组内部按扫描/LOOK 思想服务。
- 当前组处理时，不把后续组插进来。
- `N` 很大时接近普通 SCAN。
- `N = 1` 时退化为 FCFS。

### 14.3 【必会】真题：N-Step LOOK 调度序列

真题条件：

```text
磁道编号：0, 1, 2, ..., 249
编号方向：由外向内递增
当前磁头：100
当前方向：向外移动
每磁道移动时间：0.8ms
每磁道 60 个扇区
转速：6000 r/min
算法：N-Step LOOK，N = 4
请求序列：90, 15, 200, 50, 10, 180, 70, 240, 120, 30
每个请求访问 1 个扇区
```

第一步：判断方向。

题目说：

```text
磁道由外向内编号递增
```

因此：

```text
向外移动 = 编号减小方向
向内移动 = 编号增大方向
```

第二步：按 N=4 冻结分组。

```text
第 1 组：90, 15, 200, 50
第 2 组：10, 180, 70, 240
第 3 组：120, 30
```

第三步：每组内部按 LOOK 处理。

当前在 100，方向向外，即先处理小于 100 的请求：

```text
第 1 组：90 -> 50 -> 15 -> 200
```

到 200 后方向为向内。处理第 2 组：

```text
第 2 组：240 -> 180 -> 70 -> 10
```

到 10 后方向为向外。第 3 组没有比 10 更小的请求，所以反向：

```text
第 3 组：30 -> 120
```

完整服务序列：

```text
100 -> 90 -> 50 -> 15 -> 200 -> 240 -> 180 -> 70 -> 10 -> 30 -> 120
```

第四步：计算移动量。

```text
|100-90|  = 10
|90-50|   = 40
|50-15|   = 35
|15-200|  = 185
|200-240| = 40
|240-180| = 60
|180-70|  = 110
|70-10|   = 60
|10-30|   = 20
|30-120|  = 90
```

总移动量：

```text
10 + 40 + 35 + 185 + 40 + 60 + 110 + 60 + 20 + 90 = 650
```

第五步：计算寻道时间。

```text
总寻道时间 = 650 * 0.8ms = 520ms
```

第六步：计算旋转延迟。

转速：

```text
6000 r/min = 100 r/s
一圈时间 = 1 / 100 s = 10ms
平均旋转延迟 = 10ms / 2 = 5ms
```

10 个请求：

```text
总旋转延迟 = 10 * 5ms = 50ms
```

第七步：计算传输时间。

每磁道 60 个扇区，读写 1 个扇区：

```text
单扇区传输时间 = 10ms / 60 = 1/6 ms
```

10 个请求：

```text
总传输时间 = 10 * 1/6 ms = 10/6 ms ≈ 1.67ms
```

第八步：总访问处理时间。

```text
总访问处理时间 = 520 + 50 + 1.67 = 571.67ms
```

【题型】答题模板：

```text
先看编号方向；
再按 N 个一组冻结；
每组内按 LOOK，不插入后续组；
移动量是相邻服务磁道差的绝对值求和；
总时间 = 寻道 + 平均旋转延迟总和 + 传输时间总和。
```

【易错】三点：

1. “向外”不是一定编号增大，要看题目如何定义编号。
2. N-Step 处理当前组时，后续请求不能插入当前组。
3. LOOK 不走无请求端点。

## 15. FSCAN/FLOOK：冻结扫描

习题课明确考过冻结电梯 FLOOK。

### 15.1 FSCAN/FLOOK 的思想

FSCAN：

> 把磁盘请求分成服务队列和请求队列。扫描服务队列时，新来的请求进入请求队列。服务队列扫描完后，两队列交换。

FLOOK：

> 用 LOOK 的“不走端点”思想处理冻结队列。

### 15.2 为什么叫冻结

当前服务队列一旦开始处理，就冻结：

```text
当前服务队列：本轮只服务这些请求
新到请求：进入请求队列，等下一轮
```

这样可以避免磁头在处理过程中被新请求不断干扰。

### 15.3 FLOOK 解题模板

1. 明确当前服务队列已有请求。
2. 明确新请求到达时刻。
3. 当前服务队列按 LOOK 规则处理。
4. 当前队列处理期间新到请求进入等待队列。
5. 当前队列处理完后，两队列交换。
6. 继续按 LOOK 处理新服务队列。

【必会】习题课提醒：

- FLOOK 的考点不是“会背算法名”，而是“当前队列冻结，新请求不能插队”。

## 16. 磁盘访问时间公式

### 16.1 寻道时间

课件公式：

```text
Ts = m * n + s
```

其中：

- `n`：跨越磁道数。
- `m`：跨越一个磁道所用时间。
- `s`：启动时间。

若题目说忽略启动时间，则：

```text
Ts = m * n
```

### 16.2 平均旋转延迟

课件公式：

```text
Tr = 1 / (2r)
```

其中 `r` 是转速，单位要统一。

常见转换：

```text
6000 r/min = 100 r/s
一圈时间 = 1/100 s = 10ms
平均旋转延迟 = 5ms
```

### 16.3 传输时间

课件公式：

```text
Tt = b / (rN)
```

其中：

- `b`：读写字节数。
- `r`：转速。
- `N`：一条磁道上的字节数。

如果题目按扇区给：

```text
一个扇区传输时间 = 一圈时间 / 每磁道扇区数
```

### 16.4 总访问时间

课件公式：

```text
Ta = Ts + Tr + Tt
```

按扇区访问时：

```text
Ta = m*n + s + 1/(2r) + 1/(rM)
```

其中 `M` 是每磁道扇区数。

考试判断：

- 【必会】大题通常先算移动量，再乘每磁道移动时间。
- 【必会】旋转延迟一般按平均半圈算。
- 【易错】传输时间不是平均半圈，而是目标扇区真正传过磁头所需时间。

## 17. 磁头粘性

真题简答问过：

> 何谓“磁头粘性”？假设每个磁道各有一个磁头，是否还存在磁头粘性问题？

### 17.1 什么是磁头粘性

磁头粘性可以理解为：

> 在某些调度算法下，磁头倾向于停留在请求密集区域附近，使远处请求长期等待。

最典型相关算法：

```text
SSTF
```

因为 SSTF 总是选择离当前磁头最近的请求。  
如果某个区域不断有新请求到达，磁头就容易在该区域来回服务，像被“粘住”一样。

### 17.2 每个磁道都有磁头时是否存在

如果每个磁道都有一个磁头：

```text
不需要移动磁头寻找磁道
```

那么由“磁头移动距离”造成的磁头粘性问题基本不存在。

答题方式：

```text
磁头粘性是由于磁头移动调度偏向当前位置附近请求而造成的。
若每个磁道各有一个磁头，则访问不同磁道不再需要移动磁头，
不再存在由寻道调度引起的磁头粘性问题。
```

【易错】不要回答“还有，因为设备仍然忙”。设备忙、队列等待和“磁头粘性”不是同一件事。

## 18. 缓冲技术

### 18.1 为什么需要缓冲

课件定义：

> 缓冲技术是设备处理数据到达与离开速度不一致所采用的技术。

直觉解释：

CPU 很快，设备很慢。  
如果两者必须一步一等：

```text
CPU 等设备
设备等 CPU
```

系统效率会很差。

缓冲区像中转站：

```text
快速一方先把数据放进缓冲区；
慢速一方按自己的速度取走。
```

### 18.2 引入缓冲的原因

真题问：

> 在设备管理中引入缓冲的主要原因不包括什么？

选项中“不包括”的是：

```text
实现设备独立性
```

因为缓冲主要用于：

1. 缓和 CPU 与 I/O 设备速度不匹配。
2. 减少中断次数。
3. 减少输入输出次数。
4. 提高 CPU 与设备、设备与设备之间的并行程度。

【必会】考试判断：

- 缓冲不是设备独立性的主要目的。
- 设备独立性更强调用户程序不依赖具体物理设备名和设备差异。

### 18.3 硬缓冲与软缓冲

| 类型 | 位置 | 说明 |
|---|---|---|
| 硬缓冲 | 设备或控制器中 | 硬件提供 |
| 软缓冲 | 内存系统空间中 | 操作系统管理 |

### 18.4 私用缓冲与公共缓冲

| 类型 | 特点 | 问题/优点 |
|---|---|---|
| 私用缓冲 | 一个缓冲区固定关联一个设备 | 简单但利用率低 |
| 公共缓冲 | 系统统一管理，按需分配 | 利用率高，管理复杂 |

### 18.5 缓冲池管理

课件用信号量描述空缓冲池：

```c
semaphore buf_num = n;  // 空缓冲区个数
semaphore mutex = 1;    // 互斥访问空缓冲链
```

申请缓冲区：

```c
P(buf_num);
P(mutex);
取链头空缓冲;
V(mutex);
```

释放缓冲区：

```c
P(mutex);
空缓冲入链尾;
V(mutex);
V(buf_num);
```

考试判断：

- 【重点】缓冲池是共享资源，需要互斥管理。
- 这里用到 P/V，但第 9 章重点不是复杂同步题，而是理解缓冲池如何申请/释放。

## 19. 缓冲 vs 缓存

真题直接考过：

> 关于缓冲和缓存，以下描述正确的是？

正确理解：

```text
缓冲主要用于协调数据传输速度不匹配；
缓存主要用于提高数据访问速度。
```

### 19.1 对比表

| 对比 | 缓冲 Buffer | 缓存 Cache |
|---|---|---|
| 主要目的 | 协调速度差、平滑传输 | 利用局部性，提高再次访问速度 |
| 典型场景 | CPU 和 I/O 设备之间 | 磁盘块缓存、页缓存 |
| 数据副本 | 通常关注传输中的一份数据 | 可能保存慢速设备数据的副本 |
| 关键词 | 生产者/消费者、暂存、减少中断 | 命中、替换、局部性 |

课件表述：

```text
buffering: one data copy
caching: multiple data copy (slow 多, fast 少)
```

【必会】考试判断：

- 缓冲和缓存不是完全相同概念。
- 缓冲不主要为了“命中率”，缓存才强调命中。
- 缓冲通常不是单纯硬件实现；软件缓冲很常见。

## 20. UNIX 块设备缓冲与 read 综合题

习题课文件与设备综合题要求描述：

> `open, seek, read` 读取文件数据时，如何根据逻辑块、物理块、cache、I/O 链、缓冲区，把数据读入进程空间。

这类题横跨第 8 章文件系统和第 9 章设备管理。

### 20.1 块型缓冲区结构

课件中的块型缓冲区包括：

```text
缓冲区头 + 缓冲区体
```

缓冲区头中常有：

| 字段 | 含义 |
|---|---|
| 设备号 | 哪个设备 |
| 设备块号 | 哪个磁盘块 |
| I/O 标识 | 读/写、忙、完成、延迟写等 |
| I/O 链指针 | 挂在设备 I/O 队列 |
| cache/free 链指针 | 挂在缓存链或空闲链 |
| 等待进程 | 哪些进程等这个块 |

### 20.2 常见标志

课件列出过：

| 标志 | 含义 |
|---|---|
| `B_BUSY` | 缓冲区正在使用 |
| `B_DONE` | I/O 已完成，数据有效 |
| `B_READ/B_WRITE` | 读/写 |
| `B_DELWRI` | 延迟写 |
| `B_ASYNC` | 异步写 |
| `B_WANTED` | 有进程等待该缓冲区 |
| `B_ERROR` | I/O 错误 |

### 20.3 UNIX 缓冲相关操作

课件列出：

| 操作 | 含义 |
|---|---|
| `getblk(dev, blkno)` | 为指定设备块取得缓冲区 |
| `bread(dev, blkno)` | 读入块，必要时启动 I/O |
| `breada(dev, blkno, rablkno)` | 读当前块，同时预读下一块 |
| `bwrite(bp)` | 写缓冲区并等待完成 |
| `bawrite(bp)` | 异步写 |
| `bdwrite(bp)` | 延迟写 |
| `brelse(bp)` | 释放缓冲区 |

### 20.4 【题型】read + cache + I/O 链综合步骤

习题课示例：

```text
从文件 f1 的 2.5K 起始读 3K 字节
磁盘块大小 1K
```

第一步：确定逻辑块范围。

从 2.5K 开始读 3K：

```text
覆盖逻辑块 2, 3, 4, 5
```

因为：

- 逻辑块 2 的后半部分：0.5K
- 逻辑块 3：1K
- 逻辑块 4：1K
- 逻辑块 5 的前半部分：0.5K

第二步：通过文件物理结构把逻辑块换成物理块。

习题课答案示例：

```text
逻辑块 2,3,4,5 -> 物理块 15,18,36,55
```

第三步：逐块判断是否在 cache。

如果在 cache：

```text
直接从缓冲区复制到进程空间
```

如果不在 cache：

```text
分配缓冲区
填写缓冲头部
挂入设备 I/O 链
启动设备或等待队列
进程等待 I/O 完成
```

第四步：I/O 完成后中断处理。

```text
设备完成 -> 中断 -> 标记缓冲区完成 -> 唤醒等待进程
```

第五步：复制数据并把缓冲区放入 cache。

```text
缓冲区数据 -> 进程空间
缓冲区进入 cache/free 链
read 返回
```

【必会】这种题的答题骨架：

```text
逻辑块范围
-> 物理块号
-> cache 命中直接复制
-> 未命中分配缓冲并入 I/O 链
-> 设备完成中断唤醒
-> 数据复制进进程空间
-> 缓冲入 cache
-> read 返回
```

【易错】不要把“cache 命中”和“设备 I/O”混为一谈。命中时不需要真正访问磁盘。

## 21. 输入输出进程

课件提到：

> 输入输出进程是专门负责 I/O 传输的进程。

它体现一种服务模式：

```text
用户进程 -> I/O 进程 -> 设备
```

优点：

- 界面清晰。
- 用户进程不直接处理设备细节。

缺点：

- 可能带来两次进程切换，影响速度。

考试判断：

- 【重点】I/O 进程是把 I/O 操作做成服务的一种模式。

## 22. RAID 技术

### 22.1 RAID 的目标

RAID：

```text
Redundant Array of Independent Disks
```

目标：

1. 提高访问速度。
2. 提高可靠性。
3. 扩大逻辑存储容量。

课件强调：

> 多个物理磁盘在操作系统看来可以是一个逻辑驱动器。

### 22.2 RAID 衡量指标

课件列出：

| 指标 | 问题 |
|---|---|
| 速度 | 是否支持多个访问同时进行 |
| 可靠性 | 是否能发现和改正错误 |
| 成本 | 是否有额外冗余开销 |

### 22.3 RAID 0：块级分条

特点：

- 数据按块分条，循环分布到多个磁盘。
- 速度快。
- 空间利用率 100%。
- 无容错能力。

考试判断：

- 【重点】RAID 0 没有冗余，所以任意一盘故障都可能导致数据丢失。

### 22.4 RAID 1：镜像

特点：

- 相同数据重复存放在两个盘上。
- 读可以较快。
- 写需要写两个盘。
- 可靠性高。
- 空间利用率 50%。

考试判断：

- 【重点】RAID 1 用空间换可靠性。

### 22.5 RAID 2/3/4/5

简要比较：

| 级别 | 分条粒度 | 校验/冗余 | 读并发 | 写并发 |
|---|---|---|---|---|
| RAID 2 | 位级 | 汉明纠错码 | 不支持 | 不支持 |
| RAID 3 | 位级 | 单个奇偶校验盘 | 不支持 | 不支持 |
| RAID 4 | 块级 | 单独校验盘 | 支持 | 不支持 |
| RAID 5 | 块级 | 分布式奇偶校验 | 支持 | 支持 |

RAID 5：

- 至少 3 个磁盘。
- 校验信息分散存储。
- 任意一盘故障可恢复。
- 若单盘容量为 `S`，磁盘数为 `N`：

```text
有效容量 = S * (N - 1)
磁盘利用率 = (N - 1) / N
```

考试判断：

- 【重点】RAID 5 比 RAID 4 改善了单独校验盘瓶颈。
- RAID 相关内容课件较多，但真题未明显重压，掌握比较表即可。

## 23. 虚拟设备与 SPOOLing

### 23.1 为什么需要虚拟设备

独占型设备直接使用的问题：

1. 速度慢，进程占有设备期间不一定一直有效使用。
2. 设备利用率低。
3. 多个进程竞争时等待长。

虚拟设备的思想：

> 利用共享型设备实现数量较多、速度较快的独占型设备。

典型共享设备：

```text
磁盘
```

典型独占设备：

```text
打印机
```

### 23.2 虚拟设备实现直觉

在进程和独占设备之间增加共享设备缓冲：

```text
进程 -> 虚拟设备/磁盘缓冲 -> 独占物理设备
```

这样进程不必长时间占有低速设备。

### 23.3 输入型虚拟设备

课件流程：

申请：

1. 分配虚设备。
2. 分配实设备。
3. 信息由实设备传到虚设备。
4. 去配实设备。

使用：

```text
信息由虚设备传到进程空间
```

释放：

```text
去配虚设备
```

### 23.4 输出型虚拟设备

课件流程：

申请：

```text
分配虚设备
```

使用：

```text
信息由进程空间传到虚设备
```

释放：

1. 分配实设备。
2. 信息由虚设备传到实设备。
3. 去配实设备。
4. 去配虚设备。

### 23.5 SPOOLing

SPOOLing：

```text
Simultaneous Peripheral Operation On-Line
```

课件例子：

| 类型 | 含义 |
|---|---|
| SPOOLing 输入 | 作业预输入：输入机 -> 输入井 |
| SPOOLing 输出 | 作业缓输出：输出井 -> 输出机 |

真题问：

> SPOOLing 技术把低速独占设备虚拟化为高速共享设备，其核心思想是？

正确答案：

```text
将独占设备的工作转移到高速的共享设备（如磁盘）上进行。
```

【必会】考试判断：

- SPOOLing 不是让用户绕过操作系统直接访问设备。
- SPOOLing 不是减少 CPU 与 I/O 的并行度，而是提高并行性和设备利用率。
- SPOOLing 也不能消除所有资源竞争，只是把独占设备虚拟成可排队共享的设备。

### 23.6 输出 SPOOLing 例子：打印

普通打印：

```text
进程长期占有打印机，慢慢打印
```

SPOOLing 打印：

```text
进程把输出写入磁盘输出井
进程很快返回
后台输出进程按顺序把输出井内容送打印机
```

直觉：

> 用户感觉自己有一台“虚拟打印机”，真实打印机由系统后台慢慢服务。

## 24. 稳定存储器

课件最后提到稳定存储器：

> 不丢失信息的存储器称为稳定存储器。

现实中不存在绝对可靠介质，所以实现策略是：

```text
冗余
```

### 24.1 保存策略

1. 把信息写到第一个存储块。
2. 成功后，把相同信息写到第二个存储块。
3. 第二次传输也成功，整个保存操作完成。

### 24.2 恢复策略

恢复时比较两个数据块：

1. 两块完全相同且无错误：正常。
2. 一块检测到错误：用另一块替代。
3. 两块均未检测到错误但内容不同：用第二块内容取代第一块内容。

考试判断：

- 【重点】稳定存储器的核心不是神奇硬件，而是冗余和恢复策略。
- 这部分真题牵引弱，理解即可。

## 25. 课内题型归纳与逐题解析

### 25.1 【题型】I/O 请求处理顺序

题目：

```text
用户程序发出磁盘 I/O 请求后，系统的正确处理流程是？
```

答案：

```text
用户程序 -> 系统调用处理程序 -> 设备驱动程序 -> 中断处理程序
```

解析：

- 用户程序通过系统调用进入内核。
- 系统调用处理程序完成通用检查。
- 设备驱动程序启动具体设备。
- 设备完成后才发生中断，由中断处理程序收尾。

### 25.2 【题型】通道 vs DMA

题目：

```text
独立于中央处理器，专门负责数据 I/O 传输工作的处理单元是？
```

答案：

```text
通道
```

区分：

| 对象 | 关键词 |
|---|---|
| DMA 控制器 | 直接内存传输，CPU 设置参数 |
| 通道 | I/O 处理机，执行通道程序 |
| I/O 设备控制器 | 控制具体设备 |
| 缓冲区 | 暂存数据，不是处理单元 |

### 25.3 【题型】缓冲引入原因

题目：

```text
在设备管理中引入缓冲的主要原因不包括？
```

答案倾向：

```text
实现设备独立性
```

解析：

缓冲主要解决速度不匹配、减少中断、减少 I/O 次数、提高并行；设备独立性是另一个层面的抽象目标。

### 25.4 【题型】缓冲 vs 缓存

题目：

```text
缓冲和缓存的正确描述是什么？
```

答案：

```text
缓冲主要用于协调数据传输速度不匹配；
缓存主要用于提高数据访问速度。
```

### 25.5 【题型】SPOOLing

题目：

```text
SPOOLing 把低速独占设备虚拟化为高速共享设备的核心思想是什么？
```

答案：

```text
把独占设备的工作转移到高速共享设备，如磁盘，上进行。
```

例子：

```text
打印输出先进入磁盘输出井，再由后台进程送打印机。
```

### 25.6 【题型】磁盘调度大题

答题模板：

```text
1. 判断磁道编号方向和磁头当前方向。
2. 按算法确定请求服务序列。
3. 计算相邻服务磁道差的绝对值之和，得到移动量。
4. 寻道时间 = 移动量 * 每磁道移动时间 + 启动时间。
5. 平均旋转延迟 = 请求数 * 半圈时间。
6. 传输时间 = 请求数 * 单扇区传输时间。
7. 总访问时间 = 总寻道 + 总旋转延迟 + 总传输。
```

N-Step/FLOOK 特别提醒：

- N-Step：按 N 个一组冻结。
- FLOOK：当前服务队列冻结，新请求入等待队列。

### 25.7 【题型】磁头粘性

答题模板：

```text
磁头粘性是指磁盘调度中磁头倾向于停留在请求密集区域，
使远处请求长期等待的现象，常与 SSTF 等按距离优先的算法有关。
若每个磁道都有一个磁头，则访问不同磁道不需要移动磁头，
由寻道距离导致的磁头粘性问题不再存在。
```

### 25.8 【题型】read/cache/I/O 链综合题

答题模板：

```text
根据 seek 后的当前指针和读取长度确定逻辑块；
根据文件物理结构得到物理块；
逐块查 cache；
命中则直接复制到进程空间；
未命中则申请缓冲区，填写设备号/块号/标志，挂入 I/O 链；
设备完成后发生中断，标记完成并唤醒等待进程；
进程被调度后把缓冲区内容复制到进程空间；
缓冲区进入 cache/free 链；
所有数据到达后 read 返回。
```

## 26. 易混概念总表

| 易混概念 | 区别 |
|---|---|
| 共享型设备 vs 独占型设备 | 共享型可让多个请求排队交错服务；独占型一段时间只适合一个进程使用 |
| 块设备 vs 字符设备 | 块设备按块传输，可随机访问；字符设备按字符流传输 |
| 查询方式 vs 中断方式 | 查询方式 CPU 忙等；中断方式设备完成后通知 CPU |
| 中断方式 vs DMA | 中断负责通知；DMA 负责数据块直接搬运 |
| DMA 控制器 vs 通道 | DMA 是直接内存传输控制器；通道是 I/O 处理机，可执行通道程序 |
| 系统调用处理程序 vs 设备驱动程序 | 前者处理通用内核入口；后者控制具体设备 |
| 设备驱动程序 vs 中断处理程序 | 驱动启动设备；中断处理程序在设备完成后收尾 |
| SCAN vs LOOK | SCAN 到磁盘端点；LOOK 到当前方向最后一个请求 |
| C-SCAN vs C-LOOK | C-SCAN 到端点循环；C-LOOK 到请求边界循环 |
| N-Step vs 普通 SCAN/LOOK | N-Step 按 N 个请求冻结分组，当前组不插入新请求 |
| FLOOK vs LOOK | FLOOK 有服务队列和请求队列，当前队列冻结 |
| SSTF vs LOOK | SSTF 每次选最近；LOOK 按方向扫描 |
| 饥饿 vs 磁头粘性 | 饥饿是长期得不到服务；磁头粘性是磁头被请求密集区域吸住的一种原因 |
| 寻道时间 vs 旋转延迟 | 寻道是移动磁头；旋转延迟是等扇区转到磁头下 |
| 旋转延迟 vs 传输时间 | 旋转延迟是等待；传输时间是真正读写 |
| 缓冲 vs 缓存 | 缓冲协调速度差；缓存提高重复访问速度 |
| 硬缓冲 vs 软缓冲 | 硬缓冲在设备/控制器；软缓冲在内存系统空间 |
| 私用缓冲 vs 公共缓冲 | 私用固定设备；公共由系统动态分配 |
| cache 命中 vs I/O 完成 | 命中可直接复制；I/O 完成需要中断唤醒 |
| RAID 0 vs RAID 1 | RAID 0 分条无冗余；RAID 1 镜像有冗余 |
| RAID 4 vs RAID 5 | RAID 4 单独校验盘；RAID 5 分布式校验 |
| 虚拟设备 vs 真实设备 | 虚拟设备通过共享设备模拟独占设备使用体验 |
| SPOOLing vs 直接独占设备 | SPOOLing 把独占设备工作转到磁盘井和后台进程 |

## 28. 本章一句话总结

设备与 I/O 管理的主线是：

```text
用户通过系统调用提出 I/O 请求，
内核检查后交给设备驱动，
驱动把请求挂入设备队列并启动设备/DMA/通道，
磁盘请求通过调度算法减少寻道，
数据通过缓冲和缓存减少等待与重复 I/O，
设备完成后以中断收尾并唤醒进程，
SPOOLing/RAID 等机制进一步改善独占设备效率和存储可靠性。
```

期末最值得抓住的硬考点是：

```text
I/O 请求流程、通道与 DMA、缓冲与缓存、N-Step LOOK/FLOOK、
磁盘访问时间公式、磁头粘性、SPOOLing。
```
