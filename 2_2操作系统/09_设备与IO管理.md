## 从 0 理解

I/O 管理的难点在于设备很慢、种类很多、控制方式不同。操作系统要把这些差异藏起来，让用户程序只需要发出 `read/write` 之类请求。

磁盘调度像电梯调度：磁头移动有成本，因此不能只按请求到达顺序服务。不同算法本质上是在“公平性”和“减少移动距离”之间取舍。

## 核心机制/算法/公式

### I/O 请求处理流程

用户程序发出磁盘 I/O 请求后的正确流程：

```text
用户程序 -> 系统调用处理程序 -> 设备驱动程序 -> 中断处理程序
```

解释：

1. 用户程序通过系统调用进入内核。
2. 系统调用处理程序检查参数、权限、文件状态。
3. 设备驱动程序把抽象请求转换成设备命令，启动 I/O。
4. 设备完成后发中断，中断处理程序收尾并唤醒等待进程。

### I/O 传输方式

| 方式 | 特点 |
|---|---|
| 查询 | CPU 反复检查设备状态，简单但浪费 CPU |
| 中断 | 设备完成后通知 CPU，提高并行性 |
| DMA | 数据在内存和设备间直接传输，CPU 只参与设置和完成处理 |
| 通道 | 专门负责 I/O 传输的处理单元，独立于 CPU |

真题中“独立于中央处理器，专门负责数据 I/O 传输工作的处理单元”选通道。

### 缓冲 vs 缓存

| 对比 | 缓冲 buffer | 缓存 cache |
|---|---|---|
| 目的 | 协调速度不匹配，减少中断/阻塞 | 提高重复访问速度 |
| 典型场景 | CPU 与 I/O 设备传输 | 磁盘块缓存、页缓存 |
| 关键词 | 生产消费、暂存、平滑速度 | 命中、替换、局部性 |

### 磁盘访问时间

```text
总访问时间 = 寻道时间 + 旋转延迟时间 + 传输时间
```

若转速 6000 r/min：

```text
6000 r/min = 100 r/s
一圈时间 = 1/100 s = 10ms
平均旋转延迟 = 5ms
```

若每磁道 60 个扇区，读写 1 个扇区：

```text
传输时间 = 10ms / 60 = 1/6 ms
```

### 磁盘调度算法

| 算法 | 思想 | 易错点 |
|---|---|---|
| FCFS | 按到达顺序 | 公平但移动量可能大 |
| SSTF | 选离当前磁头最近 | 可能饥饿，磁头粘性 |
| SCAN | 电梯到尽头再反向 | 要移动到端点 |
| LOOK | 到当前方向最后请求再反向 | 不必到磁盘尽头 |
| C-SCAN | 单方向扫描，到端点后回到另一端 | 回程通常也算移动 |
| C-LOOK | 单方向到最后请求，再跳到另一端请求 | 不必到端点 |
| N-Step LOOK | 把请求按 N 个一组冻结处理 | 新请求进下一批 |
| FLOOK | 两个队列，当前服务队列冻结 | 当前队列处理完再换新队列 |

## 易错点与区分表

| 易错点 | 正确理解 |
|---|---|
| 缓冲是为了设备独立性 | 缓冲主要解决速度不匹配、减少中断、提高并行 |
| DMA 是独立处理器 | DMA 控制器负责直接传输；通道才是专门 I/O 处理单元 |
| SSTF 一定最好 | SSTF 可能导致远处请求饥饿和磁头粘性 |
| SPOOLing 让用户直接访问设备 | SPOOLing 把独占低速设备虚拟成共享设备，通常借助磁盘 |
| LOOK 和 SCAN 相同 | LOOK 不移动到无请求的端点 |

## 题型与解析

### 真题题型：N-Step LOOK

题目：当前磁头在 100，向外移动，请求序列为 90，15，200，50，10，180，70，240，120，30，N=4。

解题关键：

1. 按到达顺序每 4 个请求分一批：
   - 第 1 批：90，15，200，50
   - 第 2 批：10，180，70，240
   - 第 3 批：120，30
2. 每批内部按 LOOK 规则处理，处理当前批时不插入后续批。
3. “向外移动”要先确认题目编号方向：课件说磁道由外向内编号递增，所以向外移动表示编号减小。
4. 计算移动量时，把每次相邻服务磁道差的绝对值相加。
5. 寻道时间 = 移动量 * 每磁道移动时间。
6. 总旋转延迟 = 请求数 * 平均旋转延迟。
7. 总传输时间 = 请求数 * 单扇区传输时间。

若题面没有明确“跳转是否计入移动量”，按教材算法口径说明假设。

### 习题课题型：FLOOK

FLOOK 有两个队列：当前服务队列和等待队列。处理当前服务队列时，新到请求进入等待队列；当前队列处理完后，等待队列变成新的服务队列。它的考点是“冻结当前队列”，不是简单 LOOK。

### 真题题型：虚拟设备 SPOOLing

SPOOLing 的核心思想：把低速独占设备的工作转移到高速共享设备上进行。例如打印请求先输出到磁盘上的输出井，再由后台进程慢慢送打印机。
