## 从 0 理解

文件系统给用户的错觉是：数据是按文件名组织的，可以 `open/read/write/close`。但磁盘真正认识的是块号。文件系统的核心工作，就是把“文件名 + 偏移量”转换成“磁盘块 + 块内偏移”，再配合设备管理完成 I/O。

目录不是神秘结构，它本质上是“文件名 -> 文件控制信息”的映射。UNIX 中目录项通常保存文件名和 inode 号，inode 保存文件的主要控制信息和数据块索引。

## 核心机制/算法/公式

### 文件访问方式

| 方式 | 含义 | 适合场景 |
|---|---|---|
| 顺序访问 | 按顺序从前往后读写 | 文本、日志、磁带式处理 |
| 随机访问 | 按记录号、关键字或偏移访问 | 数据库、索引文件 |

### 文件逻辑组织

- 记录式文件：由记录组成，可等长或不等长。
- 流式文件：字节序列，UNIX/Windows 常见。

### 文件物理组织

| 组织 | 优点 | 缺点 |
|---|---|---|
| 连续分配 | 顺序和随机访问快 | 文件增长困难，外部碎片 |
| 链接分配 | 易增长，无外部碎片 | 随机访问差，指针开销 |
| 索引分配 | 随机访问方便，易增长 | 索引块有空间开销 |

两级索引最大文件长度：

若磁盘块 1KB，每个盘块号 4B，则一个索引块可放 `1024/4=256` 个块号。两级索引最多指向 `256*256` 个数据块，每块 1KB，最大约 64MB。

### 打开文件表

打开文件后，系统不想每次读写都重新查路径，所以会建立打开文件表：

- 用户打开文件表：每个进程自己的文件描述符表。
- 系统打开文件表：系统范围的打开文件信息，如当前读写位置、访问模式、引用计数。
- inode/FCB：文件本身的元数据和物理位置。

## 易错点与区分表

| 易错点 | 正确理解 |
|---|---|
| 删除文件一定删除目录 | 删除文件通常删除目录项和 FCB/inode，释放数据块；不是删除文件所在目录 |
| 目录文件不是文件 | 目录也是文件，只是内容是目录项 |
| 设备不能作为文件管理 | UNIX 把设备作为特殊文件，统一 open/read/write 接口 |
| 缓存和文件表相同 | 缓存保存数据块，打开文件表保存打开状态 |
| 两级索引只算两块 | 两级索引是索引块指向索引块，再指向大量数据块 |

## 题型与解析

### 真题题型：删除文件不可能执行什么

用户删除某文件，操作系统不可能执行的是“删除此文件所在的目录”。它会删除目录中的该文件目录项、释放 FCB/inode 和数据块，必要时处理缓存，但不会把父目录删掉。

### 作业/习题课题型：`open, seek, read` 读取文件

题目给定从 2.5KB 起读 3KB，块大小 1KB。访问范围覆盖逻辑块：

```text
2.5KB 起始，读到 5.5KB
涉及逻辑块 2、3、4、5
```

完整答题流程：

1. `open`：路径查找目录，找到 FCB/inode，建立用户打开文件表和系统打开文件表。
2. `seek`：把当前读写指针设置到 2.5KB。
3. `read`：根据当前指针和读长度确定逻辑块 2、3、4、5。
4. 根据文件物理结构把逻辑块转换为物理块号。
5. 对每个物理块检查缓存。
6. 缓存命中：直接复制所需数据到进程空间。
7. 未命中：分配缓冲区，挂入设备 I/O 队列，启动设备或等待。
8. I/O 完成产生中断，唤醒等待进程。
9. 把缓冲区数据复制到进程空间，并把块加入缓存链。
10. 所有字节传完后，更新文件读写指针，`read` 返回。

答题时要写出“逻辑块 -> 物理块 -> cache/IO 链 -> 进程空间”的链路。
