## 3. 核心结论速记

### 3.1 操作系统是什么

一句话定义：

```text
操作系统是管理计算机软硬件资源，并为用户和应用程序提供接口与服务的系统软件。
```

考试压缩版：

```text
地位：硬件之上，其他软件之下。
作用：对内管理资源，对外服务用户。
```

资源管理对象：

| 资源 | 后续章节 |
|---|---|
| CPU/处理机 | 进程、线程、中断、调度 |
| 内存 | 存储管理、虚拟存储 |
| 文件 | 文件系统 |
| 设备 | I/O 管理 |
| 共享资源 | 同步互斥、死锁 |

接口对象：

| 接口 | 面向谁 | 例子 |
|---|---|---|
| 系统调用 | 程序员/应用程序 | `read`, `open`, `fork` |
| 命令接口 | 普通用户/管理员 | shell 命令 |
| GUI | 普通用户 | 图形界面 |
| JCL | 批处理作业 | 作业控制语言 |

【必背】系统调用是操作系统提供给编程人员的接口。库函数可能封装系统调用，但库函数不等于系统调用。

### 3.2 四个基本特征

| 特征 | 一句话 | 高频判断 |
|---|---|---|
| 并发 | 多个程序宏观上同时推进 | 不等于多 CPU 物理同时 |
| 共享 | 多个程序共同使用系统资源 | 并发必然带来共享管理问题 |
| 虚拟 | 把物理资源变成多个逻辑资源 | 如虚拟内存、虚拟设备 |
| 异步 | 进程推进速度不可预知 | 也常说不确定性 |

【真题必背】

```text
现代操作系统最基本的两个特征：并发和共享。
```

为什么不是“虚拟和异步”：

- 并发是多道程序存在的直接表现。
- 共享是并发环境下资源管理的核心问题。
- 虚拟和异步很重要，但不是“最基本的两个”。

### 3.3 操作系统类型

| 类型 | 关键词 | 高频结论 |
|---|---|---|
| 批处理系统 | 成批作业、自动连续处理、交互性差 | 提高吞吐量，用户不能及时交互 |
| 多道批处理 | 多个作业同时在系统中 | 提高 CPU 和 I/O 利用率 |
| 分时系统 | 多终端、时间片、交互 | 用户感觉独占机器 |
| 实时系统 | 时间约束、外部事件、及时响应 | 必须在被控对象规定时间内处理 |
| 通用系统 | 综合批处理、分时、实时等功能 | 现代系统常见方向 |

【真题必背】

```text
实时操作系统不是“越快越好”，而是必须在被控对象规定的时间内处理完事件。
```

常见错误选项：

- 一个机器周期：太机械，不是实时系统定义。
- 周转时间：是调度指标，不是实时约束。
- 时间片：是分时系统概念，不是实时系统定义。

### 3.4 发展历史线索

习题课给出的线索：

```text
产生：手工 -> 批处理 -> 执行系统
完善：多道 -> 分时 -> 实时 -> 通用
发展：网络、分布式、嵌入式、单用户、云
```

高频关键词：

| 阶段 | 关键问题/关键技术 |
|---|---|
| 手工操作 | 无 OS，人工干预多，资源独占 |
| 批处理 | 减少人工干预，但交互性差 |
| 执行系统 | 通道、中断出现，OS 初级形态 |
| 多道程序 | CPU 和 I/O 并行，提高资源利用率 |
| 分时系统 | 时间片轮转，强调交互 |
| 实时系统 | 强调响应截止时间 |

【易错】“通道、中断”是执行系统阶段的重要技术线索；“多道”是 OS 走向完善的核心关键词。

### 3.5 运行环境与硬件支持

习题课列出的运行环境：

```text
定时装置、系统栈、特权指令与非特权指令、
处理机状态及状态转换、地址映射机构、存储保护设施。
```

硬件支持：

```text
中断装置、通道与 DMA 控制器、I/O 保护。
```

作用速记：

| 机制 | 作用 |
|---|---|
| 定时装置/时钟 | 防止用户程序长期独占 CPU，触发时钟中断 |
| 系统栈 | 内核态保存调用/中断处理过程中的现场和参数 |
| 特权指令 | 保护关键硬件和系统状态 |
| 处理机状态 | 区分用户态和内核态 |
| 地址映射 | 将逻辑地址映射到物理地址 |
| 存储保护 | 防止进程越界访问或破坏 OS |
| 中断装置 | 识别中断、保存断点、引出中断处理程序 |
| 通道/DMA | 提高 I/O 与 CPU 并行能力 |
| I/O 保护 | 禁止用户程序直接控制关键设备 |

## 4. 真题选择精讲

### 题 1：现代 OS 最基本的两个特征

题型：

```text
现代操作系统中最基本的两个特征是______
A. 并发和不确定
B. 并发和共享
C. 共享和虚拟
D. 虚拟和不确定
```

答案：

```text
B. 并发和共享
```

精讲：

1. 多道程序让多个程序宏观上同时推进，这就是并发。
2. 多个程序并发存在，就必然共享 CPU、内存、文件、设备等资源。
3. OS 的基本任务就是在并发环境下管理共享资源。
4. 异步/不确定和虚拟都很重要，但不是这道题问的“最基本的两个”。

答题口诀：

```text
多道带来并发，并发要求共享管理。
```

### 题 2：实时 OS 的时间约束

题型：

```text
实时操作系统必须在______内处理完来自外部的事件。
A. 一个机器周期
B. 周转时间
C. 被控对象规定的时间
D. 时间片
```

答案：

```text
C. 被控对象规定的时间
```

精讲：

- 实时系统的核心不是平均速度，而是截止时间。
- 截止时间由被控制对象或外部环境决定，例如工业控制、导弹控制、医疗监护。
- 分时系统才强调时间片；批处理/调度题才常见周转时间。

一句话：

```text
实时 = 在规定期限内响应，规定期限来自被控对象。
```

### 题 3：非特权指令判断

题型：

```text
以下属于非特权指令的是______
A. 启动某设备
B. 设置中断屏蔽
C. 设置时钟指令
D. 读时钟指令
```

答案：

```text
D. 读时钟指令
```

精讲：

判断标准：

```text
会改变系统状态、控制硬件、影响其他进程安全的指令，一般是特权指令。
只读取信息、不破坏系统状态的普通操作，一般可作为非特权指令。
```

逐项分析：

| 选项 | 判断 | 原因 |
|---|---|---|
| 启动某设备 | 特权 | 直接控制 I/O 设备 |
| 设置中断屏蔽 | 特权 | 会影响系统响应中断 |
| 设置时钟 | 特权 | 会影响时间管理和调度 |
| 读时钟 | 非特权 | 通常只是读取时间信息 |

易错提醒：

```text
设置时钟 != 读时钟。
设置会改变系统状态，读通常只是查询。
```

### 题 4：用户态切换到内核态

题型：

```text
下列选项中，会导致用户进程从用户态切换到内核态的操作是______
I. 整数除以零
II. sin() 函数调用
III. read 系统调用
```

答案：

```text
仅 I 和 III
```

精讲：

用户态进入内核态的三类典型原因：

```text
系统调用：用户程序主动请求 OS 服务，如 read/open/fork。
异常：当前指令执行出错或需要 OS 处理，如除零、缺页、越界。
外部中断：CPU 外部事件，如时钟中断、I/O 中断。
```

逐项分析：

| 操作 | 是否切内核态 | 原因 |
|---|---|---|
| 整数除以零 | 是 | 异常，需要 OS 处理 |
| `sin()` 函数调用 | 否 | 普通库函数调用，通常仍在用户态 |
| `read` 系统调用 | 是 | 主动请求 OS 读文件/设备 |

答题口诀：

```text
系统调用、异常、中断进内核；普通函数调用不进内核。
```

### 题 5：程序员接口

作业同型题：

```text
操作系统提供给编程人员的接口是______
A. 库函数
B. 系统调用
C. 高级语言
D. 子程序
```

答案：

```text
B. 系统调用
```

精讲：

- 系统调用是 OS 暴露给程序的服务入口。
- 库函数可能调用系统调用，但也可能只是普通用户态代码。
- 高级语言和子程序不是 OS 提供的内核服务接口。

典型例子：

```text
printf() 是 C 库函数；
write() 是系统调用；
printf() 最终可能通过 write() 输出。
```

## 5. 高频题型模板

### 5.1 概念简答模板：OS 地位和作用

若问“什么是操作系统/操作系统作用是什么”，按三句写：

```text
操作系统是位于硬件之上、其他软件之下的系统软件。
它对内管理 CPU、内存、文件、设备等软硬件资源。
它对外为用户和应用程序提供接口与服务。
```

如果需要展开：

```text
管理资源包括处理机管理、存储管理、文件管理、设备管理；
提供服务包括系统调用、命令接口、图形界面、作业控制等。
```

### 5.2 选择题模板：判断系统类型

看到关键词就定位：

| 题干关键词 | 选什么 |
|---|---|
| 成批作业、自动连续处理、吞吐量 | 批处理 |
| 多个作业同时驻留、CPU/I/O 并行 | 多道批处理 |
| 多终端、交互、时间片 | 分时 |
| 外部事件、截止时间、被控对象 | 实时 |
| 综合多种功能 | 通用 |

### 5.3 选择题模板：判断特权指令

判断流程：

```text
1. 是否直接控制设备？
2. 是否改变中断、时钟、CPU 状态？
3. 是否修改地址映射或保护机制？
4. 是否影响其他进程安全？
```

只要答案是“是”，通常就是特权指令。

常见特权指令：

```text
启动 I/O 设备
设置中断屏蔽
设置时钟
修改 PSW/处理机状态
修改页表/段表基址
停机
```

常见非特权操作：

```text
普通算术逻辑运算
普通函数调用
读时钟
访问本进程允许访问的内存
```

### 5.4 选择题模板：判断用户态是否切内核态

会切内核态：

```text
系统调用：read/open/write/fork/exit
异常：除零、非法指令、越界、缺页
外部中断：时钟中断、I/O 中断、控制台中断
```

不会仅因此切内核态：

```text
普通函数调用：sin(), max(), 自己写的函数
普通加减乘除：除非发生异常
普通用户态变量赋值
```

注意：

```text
库函数不等于系统调用。
库函数中如果内部调用了系统调用，才会因为那次系统调用进入内核。
```

## 6. 易错点与扣分点

| 易错说法 | 正确判断 |
|---|---|
| 并发就是多个程序在多个 CPU 上物理同时运行 | 错。单 CPU 交替执行也可以是并发 |
| 现代 OS 最基本特征是共享和虚拟 | 错。最基本是并发和共享 |
| 实时系统要求在一个机器周期内响应 | 错。要求在被控对象规定时间内响应 |
| 时间片是实时系统的核心概念 | 错。时间片主要对应分时系统 |
| 启动设备是非特权指令 | 错。启动设备是特权指令 |
| 设置时钟是非特权指令 | 错。设置时钟是特权指令 |
| 读时钟一定是特权指令 | 错。真题中读时钟按非特权指令处理 |
| `sin()` 函数调用会导致用户态切内核态 | 错。普通库函数调用不等于系统调用 |
| `read()` 只是普通函数，所以不切内核态 | 错。`read` 是系统调用 |
| 系统调用是给普通用户点击图形界面的接口 | 不准确。系统调用是给程序员/应用程序的接口 |
| Shell 是操作系统内核 | 错。Shell 是命令解释程序，不是内核本身 |
| 用户程序可以直接执行 I/O 指令 | 错。I/O 指令通常是特权指令 |
