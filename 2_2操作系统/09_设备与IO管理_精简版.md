## 1. 本章考试定位

第 9 章主线：

```text
用户 I/O 请求
 -> 系统调用
 -> 设备驱动
 -> 设备控制器/DMA/通道
 -> 设备完成中断
 -> 唤醒进程
```

高频题型：

1. 选择题：查询/中断/DMA/通道，通道 vs DMA，缓冲 vs 缓存，SPOOLing。
2. 流程题：用户程序发出磁盘 I/O 请求后的处理顺序。
3. 计算题：磁盘调度序列、移动量、寻道时间、旋转延迟、传输时间。
4. 简答题：磁头粘性、SPOOLing 核心思想、缓冲引入原因。
5. 综合题：`read`、cache、I/O 链、缓冲区、中断唤醒。

本章最重要的一句话：

```text
CPU 很快，设备很慢；操作系统用驱动、调度、缓冲、缓存、中断、DMA/通道来协调它们。
```

## 3. 设备分类与磁盘地址

### 3.1 知识点速记

按用途：

| 类型 | 例子 |
|---|---|
| 存储型设备 | 磁盘、磁带、光盘 |
| I/O 型设备 | 键盘、显示器、打印机 |
| 网络设备 | 网卡等 |

按管理方式：

| 类型 | 特点 | 例子 |
|---|---|---|
| 共享型设备 | 多个请求可排队交错服务 | 磁盘 |
| 独占型设备 | 一段时间只适合一个进程使用 | 打印机、磁带机 |

【易错】独占型设备不是永远不能共享，而是直接共享语义混乱，可通过 SPOOLing 虚拟化。

### 3.2 磁盘三维地址

磁盘三维地址：

```text
柱面号 i，盘面号 j，扇区号 k
```

一维块号：

```text
b = i * m * n + j * n + k
```

其中：

```text
m = 盘面数
n = 每盘面扇区数
```

反算：

```text
i = b / (m*n)
余数 r = b mod (m*n)
j = r / n
k = r mod n
```

### 3.3 对应例题：块号转三维地址

题目：

```text
某磁盘有 m=3 个盘面，每个盘面 n=3 个扇区。
块号 b=14。
求柱面号 i、盘面号 j、扇区号 k。
```

答案：

```text
i=1, j=1, k=2。
```

解答过程：

```text
m*n = 3*3 = 9
i = 14 / 9 = 1
r = 14 mod 9 = 5
j = 5 / 3 = 1
k = 5 mod 3 = 2
```

所以三维地址为：

```text
柱面 1，盘面 1，扇区 2
```

【易错】这里的 `/` 是整除。

## 4. I/O 传输方式：查询、中断、DMA、通道

### 4.1 知识点速记

| 方式 | CPU 参与 | 特点 |
|---|---|---|
| 程序查询 | CPU 反复检查状态 | 简单但忙等，浪费 CPU |
| 中断驱动 | CPU 启动设备后做别的，完成后中断 | CPU 与设备可并行 |
| DMA | CPU 设置参数，DMA 控制器搬运数据块 | 减少 CPU 搬运数据负担 |
| 通道 | 通道执行通道程序，控制 I/O | 专门 I/O 处理机，独立性更强 |

发展方向：

```text
查询 -> 中断 -> DMA -> 通道
CPU 负担逐渐降低，I/O 并行能力逐渐增强。
```

### 4.2 DMA 流程

DMA 大致流程：

```text
1. CPU 设置 DMA 参数：内存地址、设备地址、传输方向、计数。
2. DMA 控制器接管数据搬运。
3. 数据在设备和内存之间直接传输。
4. 传输完成后 DMA 向 CPU 发中断。
```

【易错】DMA 不是 CPU 完全不参与；CPU 仍要启动和收尾。

### 4.3 通道

通道：

```text
负责 I/O 操作的处理机。
```

它能执行通道程序，通道程序由通道命令字 CCW 等组成。

通道比 DMA 更强：

| 对比 | DMA | 通道 |
|---|---|---|
| 定位 | 控制器 | I/O 处理机 |
| 能力 | 直接内存传输 | 执行通道程序，控制多个 I/O 操作 |
| CPU 负担 | 较低 | 更低 |

### 4.4 对应例题：通道 vs DMA

题目：

```text
独立于中央处理器，专门负责数据 I/O 传输工作的处理单元是什么？
A. DMA 控制器
B. 通道
C. 设备控制器
D. 缓冲区
```

答案：

```text
B. 通道。
```

解答过程：

逐项排雷：

| 选项 | 判断 | 原因 |
|---|---|---|
| DMA 控制器 | 错 | 负责直接内存传输，但不是课内所说 I/O 处理机 |
| 通道 | 对 | 独立于 CPU，执行通道程序，专门处理 I/O |
| 设备控制器 | 错 | 控制具体设备，不是通道处理机 |
| 缓冲区 | 错 | 暂存数据，不是处理单元 |

【一句话记法】

```text
DMA 管搬运，通道像 I/O 小处理机。
```

## 5. 用户 I/O 请求处理流程

### 5.1 知识点速记

真题核心流程：

```text
用户程序 -> 系统调用处理程序 -> 设备驱动程序 -> 中断处理程序
```

为什么：

1. 用户程序不能直接启动设备，只能系统调用进内核。
2. 系统调用处理程序检查参数、权限、缓冲区地址等。
3. 设备驱动程序把抽象请求变成设备命令，启动设备/DMA/通道。
4. 设备完成后发中断，中断处理程序收尾并唤醒进程。

【易错】中断处理程序在设备完成后执行，不是用户刚发出 I/O 请求时执行。

### 5.2 对应例题：I/O 请求顺序

题目：

```text
用户程序发出磁盘 I/O 请求后，系统的正确处理流程是什么？
```

答案：

```text
用户程序 -> 系统调用处理程序 -> 设备驱动程序 -> 中断处理程序。
```

解答过程：

用户程序发起 `read/write` 这样的系统调用，进入内核。  
系统调用处理程序先检查请求是否合法，并把请求交给设备相关层。  
设备驱动程序启动具体设备，可能填写控制器寄存器、DMA 参数或通道程序。  
设备完成后产生中断，中断处理程序标记完成、唤醒等待进程。

【一句话记法】

```text
先 syscall 进内核，再 driver 启设备，完成以后 interrupt 收尾。
```

## 6. 设备分配与设备驱动

### 6.1 设备分配数据结构

| 结构 | 含义 | 作用 |
|---|---|---|
| SDT | 系统设备表 | 记录设备类、总数、等待队列、UCB 指针 |
| UCB | 设备控制块 | 描述某台设备状态、标识、占有进程、连接通道 |
| CCB | 通道控制块 | 描述通道状态、类型、占有进程 |

独占型设备流程：

```text
申请 -> 使用 -> 释放
```

申请时查 SDT/UCB，使用时分配通道并传输，释放时去配设备并唤醒等待者。

### 6.2 设备驱动

设备驱动程序位置：

```text
通用 I/O 子系统 -> 设备驱动 -> 控制器/DMA/通道 -> 真实设备
```

作用：

```text
把抽象 I/O 请求转换成具体设备命令。
```

【必背】真题流程里，设备驱动在系统调用处理程序之后、中断处理程序之前。

## 7. 磁盘调度算法

### 7.1 算法速记

| 算法 | 规则 | 易错 |
|---|---|---|
| FCFS | 按到达顺序服务 | 公平但移动量大 |
| SSTF | 每次选离当前磁头最近的请求 | 可能饥饿/磁头粘性 |
| SCAN | 一个方向扫到磁盘端点再反向 | 电梯算法 |
| LOOK | 一个方向扫到最后一个请求再反向 | 不走无请求端点 |
| C-SCAN | 单方向扫到端点，回另一端继续 | 等待更均匀 |
| C-LOOK | 单方向到最后请求，跳到另一端请求 | 不走端点 |
| N-Step LOOK/SCAN | 按 N 个请求冻结分组，每组内扫描 | 后续请求不插队 |
| FLOOK/FSCAN | 服务队列冻结，新请求入请求队列 | 两队列交换 |

【必背】SSTF 可能饥饿，因为远处请求可能一直被近处新请求插队。

### 7.2 对应例题：算法判断

题目：

```text
判断下列说法：
A. FCFS 公平，但磁头移动量可能大。
B. SSTF 一定不会饥饿。
C. SCAN 一定走到磁盘端点再反向。
D. LOOK 只走到当前方向最后一个请求再反向。
E. FLOOK 当前服务队列处理期间，新请求可以插入当前队列。
```

答案：

```text
A 对，B 错，C 对，D 对，E 错。
```

解答过程：

1. A 对：FCFS 简单公平，但不优化寻道。
2. B 错：SSTF 可能让远处请求长期等待。
3. C 对：SCAN 是电梯算法，到端点再反向。
4. D 对：LOOK 是 SCAN 改进，不走无请求端点。
5. E 错：FLOOK 当前服务队列冻结，新请求进请求队列，下一轮处理。

## 8. N-Step LOOK 真题模板

### 8.1 题目

```text
磁道编号：0,1,2,...,249
编号方向：由外向内递增
当前磁头：100
当前方向：向外移动
每磁道移动时间：0.8ms
每磁道 60 个扇区
转速：6000 r/min
算法：N-Step LOOK，N=4
请求序列：90,15,200,50,10,180,70,240,120,30
每个请求访问 1 个扇区

求服务序列、移动量和总访问处理时间。
```

### 8.2 答案

```text
服务序列：
100 -> 90 -> 50 -> 15 -> 200 -> 240 -> 180 -> 70 -> 10 -> 30 -> 120

总移动量：650
总访问处理时间：约 571.67ms
```

### 8.3 解答过程

第一步，判断方向。

题目说：

```text
磁道由外向内编号递增
```

所以：

```text
向外 = 编号减小方向
向内 = 编号增大方向
```

第二步，按 `N=4` 冻结分组：

```text
第 1 组：90, 15, 200, 50
第 2 组：10, 180, 70, 240
第 3 组：120, 30
```

第三步，每组内部按 LOOK。

当前 100，方向向外，即先服务小编号：

```text
第 1 组：90 -> 50 -> 15 -> 200
```

到 200 后方向变为向内，处理第 2 组：

```text
第 2 组：240 -> 180 -> 70 -> 10
```

到 10 后方向变为向外，第 3 组没有更小请求，所以反向：

```text
第 3 组：30 -> 120
```

完整序列：

```text
100 -> 90 -> 50 -> 15 -> 200 -> 240 -> 180 -> 70 -> 10 -> 30 -> 120
```

第四步，算移动量：

```text
|100-90|  = 10
|90-50|   = 40
|50-15|   = 35
|15-200|  = 185
|200-240| = 40
|240-180| = 60
|180-70|  = 110
|70-10|   = 60
|10-30|   = 20
|30-120|  = 90
```

总移动量：

```text
10+40+35+185+40+60+110+60+20+90 = 650
```

第五步，算寻道时间：

```text
650 * 0.8ms = 520ms
```

第六步，算旋转延迟：

```text
6000 r/min = 100 r/s
一圈时间 = 1/100 s = 10ms
平均旋转延迟 = 10/2 = 5ms
10 个请求总旋转延迟 = 10 * 5 = 50ms
```

第七步，算传输时间：

```text
每磁道 60 扇区
1 个扇区传输时间 = 10ms / 60 = 1/6 ms
10 个请求总传输时间 = 10/6 ≈ 1.67ms
```

总访问处理时间：

```text
520 + 50 + 1.67 = 571.67ms
```

【易错排雷】

1. “向外”不一定编号增大，要看题目如何定义编号。
2. N-Step 当前组冻结，后续组不能插队。
3. LOOK 不走无请求端点。
4. 总访问时间不是只有寻道时间，还要加旋转延迟和传输时间。

## 9. 磁盘访问时间与磁头粘性

### 9.1 访问时间公式

总访问时间：

```text
Ta = Ts + Tr + Tt
```

| 项 | 含义 | 公式/做法 |
|---|---|---|
| `Ts` | 寻道时间 | 移动磁道数 * 每磁道时间，若有启动时间再加 |
| `Tr` | 平均旋转延迟 | 一圈时间 / 2 |
| `Tt` | 传输时间 | 一圈时间 / 每圈扇区数 * 扇区数 |

常用换算：

```text
6000 r/min = 100 r/s
一圈 = 10ms
平均旋转延迟 = 5ms
```

### 9.2 磁头粘性

磁头粘性：

```text
磁盘调度中，磁头倾向停留在请求密集区域，使远处请求长期等待。
```

常见于：

```text
SSTF 等按距离优先的算法。
```

### 9.3 对应例题：磁头粘性简答

题目：

```text
何谓“磁头粘性”？
假设每个磁道各有一个磁头，是否还存在磁头粘性问题？
```

答案：

```text
磁头粘性是磁头因调度策略偏向当前位置附近请求，长期停留在请求密集区域，使远处请求等待过久。
若每个磁道各有一个磁头，则不需要移动磁头寻找磁道，由寻道调度造成的磁头粘性基本不存在。
```

解答过程：

磁头粘性的根因是：

```text
调度算法根据磁头当前位置选择请求，尤其 SSTF 总选最近请求。
```

如果某区域不断有新请求，磁头就会反复服务近处请求，远处请求一直排不上。

若每个磁道都有磁头：

```text
访问不同磁道不再需要移动磁头。
```

所以由“磁头移动距离”导致的粘性问题消失。

【易错】设备仍可能忙，但“设备忙”和“磁头粘性”不是同一个概念。

## 10. 缓冲与缓存

### 10.1 缓冲 Buffer

缓冲用于：

```text
协调数据到达和离开速度不一致。
```

引入缓冲的主要原因：

1. 缓和 CPU 与 I/O 设备速度不匹配。
2. 减少中断次数。
3. 减少 I/O 次数。
4. 提高 CPU 与设备、设备与设备之间并行程度。

【必背】引入缓冲的主要原因不包括：

```text
实现设备独立性
```

### 10.2 缓存 Cache

缓存用于：

```text
利用局部性保存慢速设备数据副本，提高再次访问速度。
```

关键词：

```text
命中、替换、局部性。
```

### 10.3 对比表

| 对比 | 缓冲 Buffer | 缓存 Cache |
|---|---|---|
| 主要目的 | 协调速度差、平滑传输 | 提高重复访问速度 |
| 典型场景 | CPU 与 I/O 设备之间 | 磁盘块缓存、页缓存 |
| 关键词 | 暂存、生产者消费者、减少中断 | 命中、替换、局部性 |
| 真题易错 | 不是设备独立性 | 不是单纯缓冲 |

### 10.4 对应例题：缓冲引入原因

题目：

```text
在设备管理中引入缓冲的主要原因不包括？
A. 缓和 CPU 与 I/O 设备速度不匹配
B. 减少中断次数
C. 提高 CPU 与设备并行程度
D. 实现设备独立性
```

答案：

```text
D。
```

解答过程：

缓冲的核心是处理速度不匹配和传输效率问题。  
设备独立性是让用户程序不依赖具体设备名称、型号和差异，是 I/O 软件抽象层面的目标，不是引入缓冲的主要原因。

### 10.5 对应例题：缓冲 vs 缓存

题目：

```text
缓冲和缓存的正确区别是什么？
```

答案：

```text
缓冲主要用于协调数据传输速度不匹配；
缓存主要用于提高数据访问速度。
```

【一句话记法】

```text
缓冲管速度差，缓存管下次快。
```

## 11. read/cache/I/O 链综合题

### 11.1 知识点速记

UNIX 块缓冲区：

```text
缓冲区头 + 缓冲区体
```

缓冲区头常含：

```text
设备号、块号、状态标志、I/O 链指针、cache/free 链指针、等待进程。
```

常见标志：

| 标志 | 含义 |
|---|---|
| `B_BUSY` | 缓冲区正在使用 |
| `B_DONE` | I/O 已完成 |
| `B_READ/B_WRITE` | 读/写 |
| `B_DELWRI` | 延迟写 |
| `B_WANTED` | 有进程等待 |
| `B_ERROR` | I/O 错误 |

### 11.2 对应例题：read 综合流程

题目：

```text
从文件 f1 的 2.5K 起始读 3K 字节，磁盘块大小 1K。
说明系统如何通过逻辑块、cache、I/O 链和缓冲区把数据读入进程空间。
```

答案概要：

```text
逻辑块 2,3,4,5；
逻辑块映射到物理块；
逐块查 cache；
命中直接复制；
未命中分配缓冲，挂 I/O 链，启动设备，进程等待；
I/O 完成中断唤醒；
复制数据到进程空间，缓冲区进入 cache/free 链。
```

解答过程：

第一步，确定逻辑块范围。

```text
从 2.5K 开始读 3K：
逻辑块 2 后半部分 0.5K
逻辑块 3 全部 1K
逻辑块 4 全部 1K
逻辑块 5 前半部分 0.5K
```

所以覆盖：

```text
逻辑块 2,3,4,5
```

第二步，逻辑块到物理块。

```text
通过文件物理组织或 inode 地址表，把逻辑块号换成物理块号。
```

第三步，查 cache。

```text
若物理块已在 cache：直接从缓冲区复制到进程空间。
若不在 cache：调用 getblk/bread，分配缓冲区，填写设备号/块号，挂入设备 I/O 链。
```

第四步，设备完成。

```text
设备完成 -> 中断 -> 标记 B_DONE -> 唤醒等待进程。
```

第五步，返回数据。

```text
缓冲区内容复制到用户空间；
缓冲区进入 cache/free 链；
read 返回。
```

【易错】cache 命中时不需要真正访问磁盘，也不需要等待 I/O 完成中断。

## 12. RAID

### 12.1 知识点速记

RAID：

```text
多个物理磁盘在 OS 看来像一个逻辑驱动器。
```

目标：

1. 提高访问速度。
2. 提高可靠性。
3. 扩大逻辑存储容量。

常见级别：

| RAID | 特点 |
|---|---|
| RAID 0 | 块级分条，速度快，空间利用率 100%，无冗余 |
| RAID 1 | 镜像，可靠，空间利用率 50% |
| RAID 4 | 块级分条，单独校验盘，读并发好，写有校验盘瓶颈 |
| RAID 5 | 块级分布式校验，避免单校验盘瓶颈 |

RAID 5：

```text
N 块磁盘，每块容量 S
有效容量 = S * (N - 1)
磁盘利用率 = (N - 1) / N
```

【易错】RAID 0 没有容错能力。

## 13. 虚拟设备与 SPOOLing

### 13.1 知识点速记

虚拟设备思想：

```text
利用共享型设备实现数量较多、速度较快的独占型设备。
```

典型：

```text
用磁盘虚拟打印机。
```

SPOOLing：

```text
Simultaneous Peripheral Operation On-Line
```

核心：

```text
把低速独占设备的工作转移到高速共享设备（如磁盘）上进行。
```

打印例子：

```text
进程输出 -> 磁盘输出井 -> 后台输出进程 -> 打印机
```

### 13.2 对应例题：SPOOLing 核心思想

题目：

```text
SPOOLing 技术把低速独占设备虚拟化为高速共享设备，
其核心思想是什么？
```

答案：

```text
将独占设备的工作转移到高速共享设备，如磁盘，上进行。
```

解答过程：

以打印机为例：

1. 用户进程不直接长期占用打印机。
2. 进程把输出先写入磁盘输出井。
3. 进程较快返回。
4. 后台输出进程按顺序把输出井内容送到真实打印机。

这样用户感觉像拥有自己的虚拟打印机，真实打印机由系统排队服务。

【易错排雷】

1. SPOOLing 不是绕过 OS 直接访问设备。
2. SPOOLing 不是消除所有资源竞争，而是把独占设备虚拟成可排队共享。
3. SPOOLing 通常依赖磁盘这样的高速共享设备。

## 14. 题型模板汇总

### 14.1 【题型】I/O 请求流程

模板：

```text
用户程序 -> 系统调用处理程序 -> 设备驱动程序 -> 中断处理程序
```

### 14.2 【题型】通道/DMA 判断

模板：

```text
DMA：直接内存传输控制器，CPU 设置参数。
通道：I/O 处理机，执行通道程序，独立性更强。
```

### 14.3 【题型】磁盘调度

模板：

```text
1. 判断当前方向和编号方向。
2. 若 N-Step/FLOOK，先冻结分组/服务队列。
3. 写服务序列。
4. 移动量 = 相邻服务磁道差绝对值求和。
5. 寻道时间 = 移动量 * 每磁道移动时间。
6. 平均旋转延迟 = 请求数 * 半圈时间。
7. 传输时间 = 请求数 * 单扇区传输时间。
8. 总时间 = 寻道 + 旋转 + 传输。
```

### 14.4 【题型】缓冲/缓存判断

模板：

```text
缓冲：协调速度差、减少中断、提高并行。
缓存：利用局部性、命中后避免慢速访问。
```

### 14.5 【题型】SPOOLing

模板：

```text
进程先把数据写入磁盘输入井/输出井；
后台进程再慢慢与真实独占设备交互；
把低速独占设备虚拟成可共享设备。
```

## 15. 易错判断总表

| 说法 | 判断 | 理由 |
|---|---|---|
| 共享型设备可由多个进程直接同时物理执行请求 | 错 | 通常是排队交错服务 |
| 独占型设备不能通过任何方式共享 | 错 | 可用 SPOOLing 虚拟化 |
| 查询方式 CPU 不忙 | 错 | 查询方式 CPU 忙等 |
| 中断驱动由中断负责搬完整块数据 | 错 | 中断主要通知完成/错误 |
| DMA 中 CPU 完全不参与 | 错 | CPU 要设置参数和处理中断 |
| 通道是 I/O 处理机 | 对 | 能执行通道程序 |
| DMA 控制器就是通道 | 错 | 通道独立性更强 |
| 用户 I/O 请求一开始就进中断处理程序 | 错 | 完成后才中断 |
| 设备驱动程序属于应用程序 | 错 | 属于内核设备相关部分 |
| SSTF 可能饥饿 | 对 | 远处请求可能长期等待 |
| SCAN 和 LOOK 都一定走到端点 | 错 | LOOK 不走无请求端点 |
| N-Step 当前组可插入新请求 | 错 | 当前组冻结 |
| FLOOK 新请求进入当前服务队列 | 错 | 进入请求队列 |
| 平均旋转延迟是一圈时间 | 错 | 通常是一圈时间的一半 |
| 缓冲主要实现设备独立性 | 错 | 缓冲主要协调速度不匹配 |
| 缓冲和缓存完全相同 | 错 | 缓存强调命中和局部性 |
| cache 命中还必须访问磁盘 | 错 | 命中可直接复制 |
| RAID 0 有容错能力 | 错 | RAID 0 无冗余 |
| RAID 1 空间利用率通常 50% | 对 | 镜像 |
| SPOOLing 使用磁盘等共享设备虚拟独占设备 | 对 | 打印机是经典例子 |
| SPOOLing 可消除所有资源竞争 | 错 | 只是改善独占设备使用方式 |
