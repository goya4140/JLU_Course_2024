# 03 中断与处理机调度

- 理解中断系统由硬件中断装置和软件中断处理程序组成。

## 从 0 理解

中断让操作系统有机会重新取得 CPU 控制权。没有中断，用户程序一旦运行，操作系统就很难在合适时机介入；有了时钟中断，OS 才能实现分时；有了 I/O 中断，OS 才能在设备完成后唤醒等待进程。

调度解决的是“CPU 下一段时间给谁用”。调度算法不是背名字，而是看它偏向什么：先来先服务偏公平但可能拖累短作业；短作业优先降低平均等待但可能让长作业饥饿；时间片轮转适合分时；EDF 适合实时任务。

## 核心机制/算法/公式

### 中断系统

中断装置是硬件，负责识别中断源、保存关键现场、引出中断处理程序。中断处理程序是操作系统程序，开始执行时 CPU 处于内核态。

典型流程：

1. 当前程序运行。
2. 中断事件发生。
3. 硬件保存 PSW、PC 等关键现场。
4. 根据中断向量找到处理程序入口。
5. 进入内核态执行中断处理程序。
6. 处理结束后恢复现场，返回或触发调度。

### 调度指标

| 指标 | 公式或含义 |
|---|---|
| 周转时间 | 完成时间 - 到达/提交时间 |
| 平均周转时间 | 各进程周转时间求平均 |
| 带权周转时间 | 周转时间 / 实际运行时间 |
| 等待时间 | 在就绪队列等待 CPU 的时间 |
| 响应时间 | 从提交请求到首次响应 |

### SRTF 解题模板

最短剩余时间优先是抢占式短作业优先。

步骤：

1. 按到达时间列事件点。
2. 每到一个新进程，比较当前运行进程剩余时间和新进程运行时间。
3. 谁剩余时间短谁运行。
4. 画 Gantt 图。
5. 用完成时间计算周转时间，再算等待时间和带权周转时间。

作业给定：

| 进程 | 到达 | 运行 |
|---|---:|---:|
| P1 | 0 | 10 |
| P2 | 3 | 6 |
| P3 | 4 | 3 |
| P4 | 7 | 4 |

调度过程：

```text
0-3  P1   P1 剩 7
3-4  P2   P2 剩 5，P1 剩 7
4-7  P3   P3 最短，完成
7-11 P4   P4 最短，完成
11-16 P2  P2 完成
16-23 P1  P1 完成
```

完成时间：P1=23，P2=16，P3=7，P4=11。

周转时间：P1=23，P2=13，P3=3，P4=4，平均 10.75。

等待时间 = 周转时间 - 运行时间：P1=13，P2=7，P3=0，P4=0，平均 5。

带权周转时间：2.3、13/6、1、1，平均约 1.62。

### EDF 可调度性

周期实时任务的必要条件：

```text
sum(Ci / Ti) <= 1
```

作业题：A(30,10)、B(40,15)、C(50,5)。

```text
10/30 + 15/40 + 5/50 = 97/120 = 0.808 < 1
```

因此 EDF 可以调度。画 Gantt 图时，每个任务实例的截止期为释放时间 + 周期，始终选择截止期最早的可运行实例。

## 易错点与区分表

| 易混点 | 正确理解 |
|---|---|
| 中断处理程序属于谁 | 属于操作系统程序 |
| 外部中断 | 时钟中断、控制台中断等来自 CPU 外部 |
| 访管中断 | 系统调用相关，不是普通外部中断 |
| 缺页中断 | 异常/内部中断，由访问缺页触发 |
| FCFS | 对短进程不利，可能造成 convoy effect |
| HRRN | 同时考虑等待时间和服务时间 |
| SSTF/SRTF 饥饿 | 短者优先类算法可能让长任务长期等待 |

## 题型与解析

### 作业题：SJF 平均周转时间

三个作业同时到达，运行时间 T1<T2<T3。SJF 顺序是 J1、J2、J3。

完成时间分别为 T1、T1+T2、T1+T2+T3。

平均周转时间：

```text
(T1 + (T1+T2) + (T1+T2+T3)) / 3
= (3T1 + 2T2 + T3) / 3
```

### 真题题型：多道程序总运行时间

两个程序顺序运行 20s 和 30s，并发执行总时间不可能小于最长单个程序时间，即不可能为 20s。并发不是魔法，它只能重叠 CPU 和 I/O，不能让必须执行的工作消失。
