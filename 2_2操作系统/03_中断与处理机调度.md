# 03 中断与处理机调度（深度版）

> 本章目标：理解操作系统如何通过中断重新获得 CPU 控制权，以及如何决定下一个运行的进程。  
> 课内边界：本章深讲中断系统、处理机调度、多级调度、实时调度、多处理机调度。PV 同步、死锁、存储管理只在必要处点到为止。

---

## 0. 本章定位与考试价值

第 2 章告诉我们：系统中有多个进程，它们会在运行、就绪、等待之间转换。第 3 章回答两个更底层的问题：

1. 操作系统什么时候有机会介入？
2. 操作系统介入后，CPU 下一步给谁？

第一个问题靠中断。没有中断，CPU 可能一直跑用户程序，操作系统很难抢回控制权。时钟中断让分时系统成为可能，I/O 中断让等待进程能被唤醒，访管中断让用户程序能请求内核服务。

第二个问题靠处理机调度。系统中就绪进程往往多于 CPU，调度算法决定谁先运行、谁被抢占、系统响应速度和平均周转时间如何。

本章考试价值很高：

- 选择题常考：中断类型、中断向量、中断处理程序、用户态/内核态、外部中断、系统调用。
- 计算题常考：SJF/SRTF 的 Gantt 图、平均等待时间、平均周转时间、平均带权周转时间。
- 实时调度常考：EDF 可调度性条件和 Gantt 图。
- 理解题常考：调度时机、dispatcher 保存/恢复现场、多级调度。

本章主线：

```text
程序运行中发生事件
    ↓
中断装置识别、保存、引出
    ↓
中断处理程序在内核态处理事件
    ↓
若需要切换进程，进入 dispatcher
    ↓
调度算法选择上升进程
    ↓
恢复现场，CPU 转交给新进程
```

## 1. 中断的直觉：操作系统如何重新拿回控制权

### 1.1 为什么需要中断

假设没有中断，用户程序一旦开始运行，就会一直占用 CPU，除非它主动调用操作系统或结束。这样操作系统就很难实现：

- 分时：每个程序轮流运行一小段时间。
- I/O 并行：设备完成后通知系统。
- 错误处理：越界、除零、非法指令时及时介入。
- 系统调用：用户程序请求内核服务。

中断就是一种“打断当前执行流，让操作系统处理某个事件”的机制。

生活化理解：

```text
CPU 正在听一个人讲话。
突然门铃响了、闹钟响了、有人递交申请、或者讲话者说错了危险内容。
CPU 先记住刚才听到哪里，然后转去处理事件。
处理完以后，再决定回到原来的地方，还是换另一个人继续。
```

### 1.2 【必会】课内定义

课件定义：

```text
处理机在运行过程中，出现了某一事件，必须中止正在运行的程序，转去处理这个事件，然后再返回原来运行的程序，这一过程称为中断。
```

中断系统由两部分组成：

| 部分 | 性质 | 作用 |
|---|---|---|
| 中断装置 | 硬件 | 发现并响应中断 |
| 中断处理程序 | 软件，操作系统程序 | 分析中断原因并处理 |

【题型】考试判断：

- “中断处理程序属于操作系统程序”是对的。
- “中断装置是硬件，中断处理程序是软件”是对的。

## 2. 【必会】中断装置：识别、保存、引出

### 2.1 中断装置做什么

课件把中断装置的职责概括为三件事：

1. 识别中断源  
   哪个事件引起了中断？如果多个中断同时发生，按紧迫程度排队。

2. 保存现场  
   至少保存 PSW 和 PC，使得以后能回到中断发生处继续执行。

3. 引出中断处理程序  
   根据中断类型找到对应处理程序入口。

可以记成：

```text
识别 -> 保存 -> 引出
```

【重点】这是习题课直接点名的内容。选择题看到“中断装置”时，优先想到硬件职责，而不是中断处理程序的代码逻辑。

### 2.2 中断源、中断寄存器、中断字

课件定义：

- 中断源：引起中断的事件。
- 中断寄存器：保存与中断事件相关信息的寄存器。
- 中断字：中断寄存器的内容。

例如 I/O 中断可以通过设备状态寄存器记录设备完成、错误等信息。

### 2.3 【重点】PSW 和 PC 为什么重要

PC 是程序计数器，记录下一条要执行的指令地址。PSW 是程序状态字，记录处理机状态、条件码、中断屏蔽等状态。

中断发生时，如果不保存 PC，就不知道回到哪里；如果不保存 PSW，就不知道当时 CPU 是什么状态。

课后题问：“由核心返回目态程序时，进程的 PSW 和 PC 为何必须用一条机器指令同时恢复？”

答题思路：

- PSW 决定处理机状态，例如用户态/内核态、中断屏蔽等。
- PC 决定下一条执行地址。
- 如果先恢复 PSW，CPU 可能已经切回用户态，但 PC 还没恢复到用户程序地址，状态和地址不一致，可能导致安全问题。
- 如果先恢复 PC，还未恢复正确 PSW，也可能在错误状态下执行。
- 因此必须作为一个不可分割动作同时恢复，保证从内核返回用户程序时状态与地址一致。

## 3. 【必会】中断类型与中断向量

### 3.1 强迫性中断与自愿性中断

课件把中断分为两大类：

| 类型 | 直觉 | 例子 |
|---|---|---|
| 强迫性中断 | 程序不期望，被外部或错误事件打断 | 时钟中断、I/O 中断、控制台中断、硬件故障、越界、缺页、除零、非法指令 |
| 自愿性中断 | 程序主动请求操作系统服务 | 系统调用、访管指令 SVC |

强迫性中断又可细分：

- 外部中断：时钟中断、I/O 中断、控制台中断。
- 硬件故障中断：掉电、内存校验错。
- 程序性中断：越界、越权、缺页、溢出、除零、非法指令。

自愿性中断常见形式：

```text
准备参数 -> SVC n -> 取返回值
```

或高级语言形式：

```text
返回值 = 系统调用名称(实参1, ..., 实参n)
```

### 3.2 【必会】【易错】中断、异常、系统调用的考试口径

常见教材中还会这样区分：

| 概念 | 来源 | 是否主动 | 例子 |
|---|---|---|---|
| 中断 interrupt | CPU 外部事件 | 被动 | 时钟、I/O、控制台 |
| 异常 exception/trap | 当前指令执行导致 | 被动或同步 | 除零、越界、缺页、非法指令 |
| 系统调用 syscall/SVC | 用户程序主动请求 | 主动 | read、open、fork |

课内把这些都纳入中断系统讨论。做题时要看题目问的是“广义中断”还是“外部中断”。

真题例子：

```text
Ⅰ. 时钟中断
Ⅱ. 访管中断
Ⅲ. 缺页中断
Ⅳ. 控制台中断
问：能引起外部中断的是？
```

答案应是时钟中断和控制台中断。访管中断是系统调用相关，缺页中断是程序执行访问缺页导致，不属于外部中断。

应试口诀：

```text
外部中断看“CPU 外部来的”：时钟、I/O、控制台。
访管是主动请求内核，缺页是当前指令触发异常。
```

### 3.3 【必会】【易错】中断向量

课件定义：

```text
中断向量：中断处理程序的运行环境与入口地址（PSW, PC）。
```

要点：

- 每类中断事件有一个中断向量。
- 中断向量的存放位置由硬件规定。
- 中断向量的内容由操作系统在系统初始化时设置。
- 中断向量中的 mode 应为系统态。

考试陷阱：

如果题目说“中断向量保存中断处理程序的运行环境与入口地址”，按课内口径是对的。  
如果题目说“中断向量保存被中断程序的现场”，那就要警惕：被中断程序现场通常保存到系统栈或 PCB，不是中断向量本身。

## 4. 中断嵌套、系统栈、中断屏蔽

### 4.1 中断嵌套的直觉

中断处理程序执行时，又可能发生更紧急的中断。例如正在处理磁盘 I/O 中断时，突然发生电源故障中断。更紧急的中断应该能打断低优先级中断。

这就是中断嵌套。

课件原则：

```text
高优先级别中断可以嵌入低优先级中断。
```

实现方法：

```text
中断响应后立即屏蔽不高于当前中断优先级的中断源。
```

### 4.2 系统栈

嵌套中断需要保存多层现场。系统栈就是用来保存这些现场的内核栈结构。

如果发生多层中断，系统栈中可能依次压入：

```text
PSW1, PC1
PSW2, PC2
...
PSWn, PCn
```

每次中断返回时，按栈的后进先出顺序恢复。

### 4.3 关中断与开中断

中断处理程序典型过程：

```text
关中断
进一步保存现场到系统栈
开中断或开放高优先级中断
处理中断
关中断
恢复现场
开中断
中断返回
```

为什么一开始要关中断？

因为保存现场本身必须是安全的。如果保存一半又被打断，现场可能混乱。

为什么处理中断时又可能开中断？

为了允许更高优先级事件及时响应。

### 4.4 中断优先级与中断屏蔽

中断优先级是硬件规定的中断响应次序，通常依据：

- 紧迫程度。
- 处理时间。

中断屏蔽的作用：

- 防止低优先级中断打扰高优先级中断处理。
- 允许程序调整中断响应次序。

## 5. 【必会】中断处理程序

### 5.1 中断处理程序的一般流程

课件把中断处理程序分成强迫性中断和自愿性中断两条路。

强迫性中断：

```text
关中断
保存现场到系统栈
取中断字
分析中断原因
开放高优先级中断
处理中断
关中断
判断是否需要切换进程
    不需要：由系统栈恢复现场，返回上层中断或目态程序
    需要：转 dispatcher
```

自愿性中断：

```text
关中断
保存现场到系统栈
取调用号
分析系统调用类型
执行内核服务
判断是否需要切换进程
```

### 5.2 IO 中断处理

I/O 中断通常表示设备操作完成或出错。

正常结束时：

- 继续传输。
- 唤醒相关进程。

传输错误时：

- 可能重试若干次。
- 若仍失败，报告系统操作员或返回错误。

直觉：进程发起 I/O 后进入等待态；设备完成后发中断；中断处理程序唤醒该进程，使其从等待态回到就绪态。

### 5.3 时钟中断处理

时钟中断是分时系统的心跳。

课件列出时钟中断处理功能：

- housekeeping。
- 进程管理，例如重新计算进程调度参数。
- 实现软时钟，启动定时程序。
- 考虑进程切换。

如果没有时钟中断，一个 CPU 密集型用户进程可能一直运行，分时调度很难实现。

### 5.4 程序性中断处理

程序性中断包括越界、非法指令、缺页、除零、溢出等。

课件区分：

1. 只能由操作系统处理的中断  
   因为影响系统或其他进程，或需要系统管理。例如越界、非法指令、缺页、缺段。

2. 可以由用户自己处理的中断  
   不影响系统和其他进程。例如除零、溢出，有些语言或环境允许用户定义处理续元。

考试常见判断：

- 缺页中断需要操作系统处理，因为要动态调入页面。
- 非法指令、越界通常导致终止进程或调试。
- 除零可能由用户处理，也可由 OS 标准处理，具体看系统支持。

### 5.5 【必会】【易错】自愿性中断：系统调用

访管指令 SVC 是用户程序主动进入内核的方式。

一般过程：

```text
准备参数
执行 SVC n
内核根据调用号 n 查系统调用驱动表
执行对应服务程序
返回结果
```

真题常考：

```text
整数除以零、sin() 函数调用、read 系统调用，哪些会导致用户态切换到内核态？
```

答案：整数除以零和 read 系统调用。

解析：

- 整数除以零是异常/程序性中断。
- read 是系统调用。
- sin() 是普通库函数调用，通常不需要进入内核。

【题型】用户态切内核态题的判断顺序：系统调用一定进内核；异常通常进内核；普通库函数不一定进内核。

## 6. 考虑系统状态的进程状态转换

第 2 章讲过运行、就绪、等待。第 3 章加入用户态/内核态之后，状态转换更细：

```text
user 运行
    ↓ 中断
kernel 中断处理
    ↓ 需要调度
保存下降进程现场到 PCB
选择上升进程
由 PCB 恢复上升进程现场
    ↓ 返回，置 PSW
user 运行
```

关键直觉：

- 用户程序运行时发生中断，CPU 进入内核态。
- 中断处理程序可能只处理事件后返回原进程。
- 也可能因为时间片到、进程等待、I/O 完成唤醒高优先级进程等原因，引起进程切换。
- 进程切换一定由内核完成。

真题相关：

“进程切换不能在用户态发生”是对的。因为切换要保存/恢复 PCB、修改调度结构、改变处理机状态，这些都属于内核职责。

## 7. 【重点】处理机调度：CPU 下一步给谁

### 7.1 调度的直觉

系统中经常有多个就绪进程，但 CPU 数量有限。处理机调度就是从就绪进程集合中选一个，让它占用 CPU 运行。

课件将处理机调度分成三个问题：

| 问题 | 对应小节 |
|---|---|
| 按什么原则分配 | 处理机调度算法 |
| 何时重新分配 | 处理机调度时机 |
| 如何完成分配 | 处理机调度过程 |

### 7.2 【必会】调度评价指标

课件列出 scheduling criteria：

- CPU 利用率：越高越好。
- 吞吐量：单位时间完成作业数，越高越好。
- 周转时间：越短越好。
- 响应时间：越短越好。
- 系统开销：越小越好。

调度参数公式：

```text
周转时间 T = 完成时间 tf - 进入时间 ts

平均周转时间 = 所有进程周转时间之和 / n

带权周转时间 W = 周转时间 T / 运行时间 R

平均带权周转时间 = 所有带权周转时间之和 / n
```

等待时间：

```text
等待时间 = 周转时间 - CPU 实际运行时间
```

如果进程有 I/O burst，则等待时间的具体定义要看题目口径。课内调度计算题通常按“在就绪队列中等待 CPU 的时间”计算。

【题型】SRTF/SJF/RR 计算题一般都逃不开这三个量：周转、等待、带权周转。先用完成时间算周转，再倒推出等待，最稳。

### 7.3 CPU burst 与 I/O burst

进程运行行为通常是：

```text
CPU burst -> I/O burst -> CPU burst -> I/O burst -> ...
```

CPU burst 是进程使用 CPU 计算的阶段；I/O burst 是进程使用设备的阶段。

CPU 调度主要考虑处于 CPU burst 前、已经就绪的进程集合。

课件还给出 CPU burst 估计公式：

```text
tau(n+1) = alpha * t(n) + (1-alpha) * tau(n)
```

其中：

- `t(n)` 是最近一次实际 CPU burst 长度。
- `tau(n)` 是本次估计值。
- `alpha` 控制最近行为和历史估计的权重。
- `alpha=0` 时完全相信历史估计。
- `alpha=1` 时完全相信最近一次实际值。
- 通常取 `alpha=0.5`。

这个公式帮助 SJF/SRTF 估计“短作业”。

### 7.4 剥夺式与非剥夺式

| 类型 | 含义 | 进程运行到什么时候停止 |
|---|---|---|
| 剥夺式 | 就绪进程可以从运行进程手中抢占 CPU | 结束、等待、或被抢先 |
| 非剥夺式 | 就绪进程不能抢占运行进程 | 结束或等待 |

剥夺式适合响应要求高的系统，如分时系统、实时系统。非剥夺式简单，开销小，但响应可能慢。

## 8. 典型调度算法

### 8.1 【必会】FCFS：先到先服务

FCFS 按进程申请 CPU 或进入就绪队列的次序调度。

优点：

- 简单。
- 看起来公平。

缺点：

- 短作业可能等待很久。

课内例子：

| 进程 | 到达 | 运行 |
|---|---:|---:|
| P1 | 0 | 27 |
| P2 | 1 | 3 |
| P3 | 2 | 5 |

FCFS Gantt 图：

```text
0        27 30 35
|   P1   |P2|P3|
```

P2 和 P3 很短，但必须等 P1 长时间运行完，所以带权周转时间很大。

考试判断：

“FCFS 对短进程不利”是对的。

### 8.2 【必会】SJF：短作业优先

SJF 按 CPU burst 长度选择最短者优先。

若所有任务同时到达，SJF 可以使平均等待时间最短。

缺点：

- 长作业可能饥饿。
- 需要知道或估计 CPU burst 长度。

作业选择题：

三个作业 J1、J2、J3 同时到达，运行时间 T1<T2<T3，单道系统采用 SJF。

执行顺序：

```text
J1 -> J2 -> J3
```

完成时间：

```text
J1: T1
J2: T1 + T2
J3: T1 + T2 + T3
```

平均周转时间：

```text
(T1 + (T1+T2) + (T1+T2+T3)) / 3
= (3T1 + 2T2 + T3) / 3
```

### 8.3 【必会】【题型】SRTN/SRTF：最短剩余时间优先

课件称 SRTN，作业和真题常写 SRTF，本质都是“最短剩余时间优先”。

它是可剥夺式 SJF：

- 新进程到达时，比较它的运行时间和当前进程剩余时间。
- 谁剩余时间更短，谁运行。

作业题：

| 进程 | 到达时间 | 运行时间 |
|---|---:|---:|
| P1 | 0 | 10 |
| P2 | 3 | 6 |
| P3 | 4 | 3 |
| P4 | 7 | 4 |

调度过程：

```text
0-3   P1   P1 剩 7
3-4   P2   P2 剩 5，P1 剩 7
4-7   P3   P3 最短，完成
7-11  P4   P4 最短，完成
11-16 P2   P2 完成
16-23 P1   P1 完成
```

Gantt 图：

```text
0   3 4   7    11    16       23
| P1 |P2| P3 | P4 |  P2  |   P1   |
```

完成时间：

| 进程 | 完成时间 |
|---|---:|
| P1 | 23 |
| P2 | 16 |
| P3 | 7 |
| P4 | 11 |

计算：

| 进程 | 到达 | 运行 | 完成 | 周转 T | 等待 T-R | 带权 T/R |
|---|---:|---:|---:|---:|---:|---:|
| P1 | 0 | 10 | 23 | 23 | 13 | 2.30 |
| P2 | 3 | 6 | 16 | 13 | 7 | 2.17 |
| P3 | 4 | 3 | 7 | 3 | 0 | 1.00 |
| P4 | 7 | 4 | 11 | 4 | 0 | 1.00 |

结果：

```text
平均等待时间 = (13+7+0+0)/4 = 5
平均周转时间 = (23+13+3+4)/4 = 10.75
平均带权周转时间 = (2.30+2.17+1+1)/4 ≈ 1.62
```

考试解题口诀：

```text
每逢新进程到达，就比较“当前剩余”和“新进程总长”。
```

【易错】不要只在进程完成时比较；SRTF 是抢占式，任何新进程到达都可能打断当前进程。

### 8.4 【必会】HRN：最高响应比优先

课件公式：

```text
RR = (BT + WT) / BT = 1 + WT / BT
```

其中：

- BT = burst time，服务时间。
- WT = wait time，等待时间。

HRN 的直觉：

- 如果同时到达，短作业响应比高，短者优先。
- 长作业等待时间变长后，响应比也会提高，不容易一直饥饿。

作业选择题：

“既考虑作业等待时间，又考虑作业执行时间的调度算法”是最高响应比优先。

### 8.5 HPF：最高优先数算法

优先数可以是静态或动态。

| 类型 | 含义 | 特点 |
|---|---|---|
| 静态优先数 | 创建时分配，生存期不变 | 响应速度慢，开销小，适合批处理 |
| 动态优先数 | 生存期内可以修改 | 响应速度快，开销大 |

也可以分为：

- 非剥夺式静态优先数：运行到终止或等待。
- 剥夺式动态/静态优先数：出现更高优先级进程时可抢占。

注意不同系统中“优先数越大越高”还是“越小越高”不一定相同。课件 UNIX 例子里 `p_pri` 最小者被调度。

### 8.6 RR：循环轮转

RR 是分时系统常用算法。

基本思想：

- 每个进程获得固定长度时间片。
- 时间片用完还没完成，就回到就绪队列尾部。
- 所有进程等速向前推进。

时间片选择很重要：

| 时间片 | 结果 |
|---|---|
| 过长 | 响应速度慢，退化接近 FCFS |
| 过短 | 上下文切换频繁，系统开销大 |

课件说时间片通常为几十毫秒到几百毫秒。

### 8.7 MLQ：多级队列

多级队列算法有多个就绪队列，进程所属队列固定。

例子：

- 实时进程就绪队列：HPF。
- 分时进程就绪队列：RR。
- 批处理进程就绪队列：HPF。

直觉：不同类型进程使用不同规则。

### 8.8 FB：反馈排队算法

反馈排队算法有多个就绪队列，进程所属队列可变。

典型思想：

- 新进程或被唤醒进程进入高优先级队列。
- 若用完高优先级队列时间片仍未完成，下降到低优先级队列。
- 越低优先级，时间片可越长，调度频率越低。

调度效果：

- 资源利用率高。
- 响应速度快。
- 系统开销相对可控。

直觉：

```text
交互式进程经常等待用户输入，一旦被唤醒，应尽快响应。
CPU 密集型进程总是用完整个时间片，可以逐渐降级，减少切换开销。
```

## 9. 【重点】处理机调度时机与过程

### 9.1 调度时机

课件概括：

- 运行进程结束。
- 运行进程等待。
- 处理机被剥夺。

必然引起进程切换的中断：

- 进程自愿结束，如 `exit()`。
- 进程被强行终止，如非法指令、越界、`kill`。
- 进程等待。

可能引起进程切换的中断：

- 时钟中断。
- 设备 I/O 中断。
- 系统调用。

注意“可能”两个字：

- 时钟中断可能导致时间片到，从而切换。
- I/O 中断可能唤醒高优先级进程，但也可能当前进程继续运行。
- 系统调用可能阻塞当前进程，也可能执行完直接返回。

### 9.2 【重点】dispatcher：调度分派程序

dispatcher 做三件事：

1. 保存下降进程的现场。

```text
寄存器（PSW, PC, SP, 通用寄存器, 地址寄存器） -> PCB
```

2. 选择上升进程。

```text
按处理机调度算法选择
```

3. 恢复上升进程的现场。

```text
PCB -> 寄存器
先恢复通用寄存器和地址寄存器
最后恢复 PSW, PC
PSW 和 PC 必须用一条指令恢复
```

课后题“进程切换时需要保存哪些现场信息”可答：

- PC。
- PSW。
- SP。
- 通用寄存器。
- 地址寄存器。
- 与内存管理相关的寄存器或地址空间信息。
- 必要的调度/状态信息写回 PCB。

【易错】进程切换不是用户态动作；保存 PCB、选择上升进程、恢复 PSW/PC 都需要内核完成。

## 10. 【重点】调度级别与多级调度

### 10.1 低级调度：处理机调度

处理机调度就是 CPU scheduling，也称低级调度。它从就绪队列中选择进程或线程占用 CPU，发生频率最高。

### 10.2 中级调度：交换

中级调度又叫交换调度，目标是控制并发度。

为什么要控制并发度？

并发度过高会导致：

- 系统开销大。
- 响应速度慢。
- 内存等资源紧张。
- 进程频繁进入等待状态。
- 更容易发生死锁。

中级调度会把某些进程换出到外存，使其进入挂起状态；条件满足后再换入内存。

### 10.3 高级调度：作业调度

高级调度也叫作业调度。它决定后备作业什么时候进入内存执行。

课件给出作业状态：

```text
提交 -> 后备 -> 执行 -> 完成 -> 退出
```

转换：

- 提交 -> 后备：SPOOLing 输入进程完成。
- 后备 -> 执行：作业调度完成。
- 执行 -> 完成：作业调度完成善后。
- 完成 -> 退出：SPOOLing 输出进程完成。

适合批作业调度的算法：

- FCFS。
- HPF。
- SJF。
- HRN。

不适合批作业调度的算法：

- RR。
- SRTN。
- FB。

原因：RR、SRTN、FB 更适合运行期间频繁调度的进程级场景，而不是决定哪个后备作业进入内存。

## 11. 【必会】实时调度

### 11.1 实时任务的直觉

实时任务不是“运行得很快”的任务，而是“有明确时间约束”的任务。

例如：

- 某个时刻前必须开始处理。
- 某个时刻前必须处理完毕。

实时调度要合理安排就绪实时任务执行次序，满足每个任务的时间约束。

### 11.2 实时任务分类

| 分类 | 含义 |
|---|---|
| 硬实时 | 必须满足截止期要求 |
| 软实时 | 期望满足截止期要求，偶尔错过可接受 |
| 周期性 | 每隔固定时间发生一次 |
| 随机性 | 由随机事件触发，发生时刻不确定 |

术语：

- Ready time：就绪时间。
- Starting deadline：开始截止期。
- Processing time：处理时间。
- Completion deadline：完成截止期。
- Occurring period：发生周期。
- Occurring frequency：发生频率。

### 11.3 【必会】周期性实时任务可调度必要条件

课件公式：

```text
sum(Ci / Ti) <= 1
```

其中：

- `Ci` 是任务 `Pi` 的处理时间。
- `Ti` 是任务 `Pi` 的发生周期。

直觉：所有周期任务平均占用 CPU 的比例不能超过 100%。

这只是一般必要条件；对于 EDF，课件进一步说明它是充分条件。

### 11.4 【必会】【题型】EDF：最早截止期优先

EDF = Earliest Deadline First。

规则：

```text
优先选择截止期最早的实时任务。
```

课件口径：

- EDF 可抢先。
- 对 EDF 来说，`sum(Ci/Ti) <= 1` 是可调度充分条件。
- 在不可调度条件下，EDF 可使错过截止期任务最小化。

作业题：

| 任务 | 周期 Ti | 处理时间 Ci |
|---|---:|---:|
| A | 30 | 10 |
| B | 40 | 15 |
| C | 50 | 5 |

先算利用率：

```text
10/30 + 15/40 + 5/50
= 1/3 + 3/8 + 1/10
= 40/120 + 45/120 + 12/120
= 97/120
= 0.808 < 1
```

因此 EDF 可以调度。

前 100 个时间单位的一种 EDF 调度序列：

```text
0-10   A1   deadline 30
10-25  B1   deadline 40
25-30  C1   deadline 50
30-40  A2   deadline 60
40-55  B2   deadline 80
55-60  C2   deadline 100
60-70  A3   deadline 90
70-80  idle
80-95  B3   deadline 120
95-100 A4   deadline 120, 尚未完成但截止期在 120
```

注意：到 100 时，A4 的截止期是 120，因此前 100 单位内没有错过截止期。若要求画到 100，图中可以在 95-100 标出 A4 正在执行。

应试口诀：

```text
EDF 两步走：先算 sum(Ci/Ti)，再按最早截止期排队。
画图时给每个任务实例写 release 和 deadline，别只看任务名字。
```

### 11.5 RMS：速率单调调度

RMS = Rate Monotonic Scheduling。

课件口径：

- 面向周期性实时事务。
- 非剥夺式。
- 优先调度发生周期最短，也就是频度最高的任务。

可调度条件：

```text
sum(Ci / Ti) <= n(2^(1/n) - 1)
```

RMS 与 EDF 对比：

- RMS 条件强于 EDF，也就是更保守。
- RMS 实现较 EDF 简单。
- 当任务数趋向无穷时，RMS 上限趋近 `ln 2 ≈ 0.693`。

课后题给出：

```text
T1=100, C1=50
T2=200, C2=30
T3=500, C3=100
```

利用率：

```text
50/100 + 30/200 + 100/500
= 0.5 + 0.15 + 0.2
= 0.85
```

EDF：`0.85 < 1`，满足 EDF 充分条件，可调度。  
RMS：n=3 时上限约 0.779，`0.85 > 0.779`，不能由 RMS 充分条件保证可调度。注意“不能保证”不一定等于实际一定不可调度，若题目按条件判断，就写不满足 RMS 可调度充分条件。

## 12. 多处理机调度

### 12.1 问题背景

多处理机调度面对：

```text
M 个进程/线程
N 个处理机
```

课件提到 SMP：对称多处理器，所有处理器相同。

目标是 load-sharing，即负载共享。

### 12.2 自调度

自调度使用一个公共就绪队列，多个处理机从中取任务。

优点：

- 不需要专门处理机负责分派。
- 任务分配较均衡。

缺点：

- CPU 多时，就绪队列可能成为瓶颈。
- 同一线程两次调度可能在不同处理机上，影响缓存局部性。
- 不能保证同组线程同时调度。

### 12.3 组调度

组调度把一组相关或合作线程同时分派到多个处理机上运行。

优点：

- 避免合作线程之间相互等待。
- 降低开销，提高运行效率。

直觉：如果线程 A 和 B 频繁协作，只让 A 运行、B 不运行，A 很快可能等待 B；组调度让它们一起推进。

## 13. 操作系统是中断驱动的

课件最后强调：

```text
操作系统是中断驱动的。
```

这句话非常重要。它的含义是：操作系统的大量活动不是凭空发生的，而是由事件触发。

典型事件：

- 时钟中断触发调度。
- I/O 中断唤醒等待进程。
- 系统调用触发内核服务。
- 缺页中断触发页面调入。
- 非法指令触发异常处理。

如果把操作系统想象成一个一直主动轮询的“大循环”，会很难理解。更好的直觉是：

```text
平时用户程序运行。
事件发生。
中断进入内核。
内核处理事件。
必要时调度。
返回用户态。
```

## 14. 【题型】课内题型归纳与逐题解析

### 14.1 【必会】中断处理程序属于什么

题干：中断处理程序属于？

答案：操作系统程序。

解析：中断处理程序在内核态运行，负责分析中断原因、处理事件、唤醒进程或触发调度。它不是用户程序。

### 14.2 【必会】【易错】外部中断判断

题干：

```text
Ⅰ. 时钟中断
Ⅱ. 访管中断
Ⅲ. 缺页中断
Ⅳ. 控制台中断
```

能引起外部中断的是？

答案：Ⅰ 和 Ⅳ。

解析：

- 时钟中断来自外部时钟装置。
- 控制台中断来自外部设备。
- 访管中断是系统调用/自愿性中断。
- 缺页中断由当前指令访问不在内存的页面引起，属于程序性中断/异常。

### 14.3 【必会】【易错】中断、异常和系统调用错误项

真题中有说法：

```text
中断或异常发生时，CPU 一定处于内核态。
```

这句话容易出错。更严谨的说法是：

```text
中断或异常发生后，CPU 转入内核态执行处理程序。
```

中断发生前，CPU 可能正在用户态执行用户程序。若题目说“发生时一定处于内核态”，通常应判为错误。

### 14.4 【必会】SJF 平均周转时间

三个同时到达作业 T1<T2<T3，SJF 顺序为 J1、J2、J3。

平均周转时间：

```text
(3T1 + 2T2 + T3) / 3
```

### 14.5 【必会】HRN 选择题

题干：既考虑作业等待时间，又考虑作业执行时间的调度算法是？

答案：最高响应比优先。

公式：

```text
RR = 1 + WT / BT
```

### 14.6 【必会】对短进程不利的算法

题干：在进程调度算法中，对短进程不利的是？

答案：先来先服务 FCFS。

解析：如果短进程排在长进程后面，会长期等待，带权周转时间很大。

### 14.7 【必会】【题型】SRTF 大题模板

解题步骤：

1. 按到达时间排序。
2. 从 0 时刻开始，列出当前就绪进程。
3. 每当新进程到达或当前进程完成，重新比较剩余时间。
4. 画 Gantt 图。
5. 由完成时间算周转时间。
6. 等待时间 = 周转时间 - 运行时间。
7. 带权周转时间 = 周转时间 / 运行时间。

作业例题结果：

```text
Gantt: 0-3 P1, 3-4 P2, 4-7 P3, 7-11 P4, 11-16 P2, 16-23 P1
平均等待时间 = 5
平均周转时间 = 10.75
平均带权周转时间 ≈ 1.62
```

### 14.8 【必会】【题型】EDF 大题模板

解题步骤：

1. 写出每个任务的周期 Ti、处理时间 Ci。
2. 计算利用率 `sum(Ci/Ti)`。
3. 若使用 EDF，利用率不超过 1 则可调度。
4. 展开每个任务实例：释放时间、处理时间、截止期。
5. 每个时刻选择截止期最早的就绪任务。
6. 标注抢占点、空闲区间、是否错过截止期。

作业例题：

```text
A: T=30, C=10
B: T=40, C=15
C: T=50, C=5
利用率 = 97/120 < 1
EDF 可调度
```

## 15. 易混概念总表

| 易混概念 | 一句话区分 |
|---|---|
| 中断装置 vs 中断处理程序 | 前者是硬件，后者是 OS 软件 |
| 中断源 vs 中断字 | 中断源是事件，中断字是寄存器中记录的事件信息 |
| 外部中断 vs 程序性中断 | 外部来自 CPU 外部，程序性由当前指令执行导致 |
| 访管中断 vs 普通函数调用 | 访管进入内核，普通函数调用通常仍在用户态 |
| 中断向量 vs 被中断现场 | 中断向量指向处理程序，被中断现场保存在系统栈或 PCB |
| 剥夺式 vs 非剥夺式 | 剥夺式可抢占，非剥夺式运行到结束或等待 |
| SJF vs SRTF | SJF 常指非抢占短作业优先，SRTF 是抢占式最短剩余时间 |
| HRN vs SJF | HRN 同时考虑等待时间和服务时间 |
| RR vs FCFS | RR 按时间片轮转，FCFS 一来一服务 |
| 高级/中级/低级调度 | 作业进入内存 / 进程换入换出 / CPU 分配 |
| EDF vs RMS | EDF 看截止期，RMS 看周期频率 |
| 必要条件 vs 充分条件 | 必要是不满足则不行，充分是满足即可保证 |

## 17. 本章一句话收束

第 3 章的核心直觉是：操作系统靠中断进入内核、靠调度分配 CPU。中断回答“什么时候进入 OS”，调度回答“进入 OS 后 CPU 给谁”。掌握这条线，再去做 SRTF、EDF 和中断选择题，就不会只是在背公式，而是在模拟操作系统真实地接管和分配处理机。
