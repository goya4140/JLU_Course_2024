## 从 0 理解

虚拟存储的想法是：程序运行时通常只用到一部分代码和数据。既然如此，就没有必要把整个程序全部放进内存。先把当前需要的部分放入内存，其他部分放在外存；访问缺失部分时再调入。

这会带来一个新问题：如果访问的页不在内存，就会发生缺页中断。缺页中断不是普通 I/O 那么简单，它可能还要选择一个页面淘汰，并维护页表状态。

## 核心机制/算法/公式

### 缺页处理流程

1. CPU 访问某逻辑页。
2. 页表发现该页不在内存，产生缺页中断。
3. 操作系统找到该页在外存的位置。
4. 查找空闲页框。
5. 若没有空闲页框，按置换算法选择牺牲页。
6. 若牺牲页修改过，写回外存。
7. 把所需页读入内存。
8. 修改页表和相关状态位。
9. 重新执行引起缺页的指令。

### 页面置换算法

| 算法 | 思想 | 考点 |
|---|---|---|
| OPT | 淘汰未来最长时间不用的页 | 理论最优，用于比较 |
| FIFO | 淘汰最早进入内存的页 | 可能 Belady 异常 |
| LRU | 淘汰最长时间未访问的页 | 常考手算 |
| 二次机会 | FIFO + 访问位 | 访问位为 1 则清 0 并跳过 |
| 时钟 | 环形指针实现二次机会 | 指针移动和访问位变化 |
| 改进时钟 | 同时看访问位、修改位 | 优先淘汰未访问且未修改页 |

### 改进时钟算法优先级

通常按 `(访问位, 修改位)` 分类：

1. `(0,0)`：最近未访问，未修改，最适合淘汰。
2. `(0,1)`：未访问但修改过，淘汰需写回。
3. `(1,0)`：访问过但未修改。
4. `(1,1)`：访问过且修改过，最不优先。

扫描时会清除访问位，给页面第二次机会。

### 地址转换

页面大小为 `2^k`，则逻辑地址低 k 位是页内偏移，高位是页号。

例：页面大小 4KB = `2^12`，逻辑地址 `8AC2H`：

- 页号 = `8AC2H / 1000H = 8H`
- 页内偏移 = `AC2H`
- 若页 8 在页框 f，则物理地址 = `f * 1000H + AC2H`

## 易错点与区分表

| 易错点 | 正确理解 |
|---|---|
| 缺页中断是外部中断 | 缺页由当前指令访问缺页触发，属于异常/内部中断 |
| 页面越多缺页一定少 | FIFO 可能出现 Belady 异常 |
| 虚拟页式不需要页表 | 虚拟页式更依赖页表状态位 |
| 修改位无意义 | 修改位决定淘汰时是否需要写回外存 |
| CPU 低且交换区磁盘高 | 说明可能抖动，增加内存通常能改善 |

## 题型与解析

### 作业题：改进时钟 + 固定页框

页面大小 2KB 时，页内偏移占 11 位，即 `800H` 为一页。

地址拆分：

- `4C7H`：页号 0，偏移 `4C7H`。
- `19B4H`：页号 3，偏移 `1B4H`。
- `0C9AH`：页号 1，偏移 `49AH`。
- `27A8H`：页号 4，偏移 `7A8H`。

解题流程：

1. 先把每个逻辑地址拆成页号和偏移。
2. 查页表是否在内存。
3. 若在内存，引用置访问位，修改置访问位和修改位。
4. 若不在内存，用改进时钟从指针位置扫描选择牺牲页。
5. 新页装入后，指针指向新装入页面的下一个在内存表项。
6. 最后根据页框号 + 偏移得到物理地址。

### 真题题型：工作集

工作集大小 n 表示在窗口 `Delta` 内进程访问过的不同页面数。不是缺页次数，也不是淘汰页数。

### 真题题型：抖动判断

CPU 利用率很低，交换空间磁盘利用率很高，其他设备利用率低，说明系统频繁换页，出现抖动。提高性能的方向是增加物理内存或减少并发进程数；增加进程数会更糟。
