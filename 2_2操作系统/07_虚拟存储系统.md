```text
如果一个进程比内存还大，能不能运行？

如果访问的页面不在内存，系统怎样补救？
```


## 1. 虚拟存储为什么存在

### 1.1 无虚拟存储的问题

如果没有虚拟存储，进程必须全部装入内存才能运行。这会带来两个问题：

1. 不能运行比内存大的程序。
2. 即使程序很大，某一时刻真正活跃的部分可能很少，全部装入会浪费空间。

课件强调：

```text
进程活动具有局部性。
```

局部性意味着：

- 时间局部性：刚访问过的内容，很可能很快再访问。
- 空间局部性：访问某地址后，很可能访问附近地址。

所以没有必要一开始把整个进程都塞进内存。

### 1.2 【必会】虚拟存储的基本思想

虚拟存储：

```text
进程部分装入内存，部分或全部放在外存。
运行时访问在外存的部分，再动态调入；
内存不够时，淘汰暂时不用的部分。
```

直觉：

```text
内存像书桌，外存像书架。
你不需要把整套书都摊在桌上；
用到哪本拿哪本，桌子满了就把暂时不用的书放回书架。
```

【易错】虚拟存储不是让内存真的变大，而是让进程看到的地址空间可以大于实际内存，靠外存和动态调入支撑。

### 1.3 外存资源管理

课件先讲外存空间：

- 外存空间静态等长划分，大小为 `2^i`。
- 每块称为 block。
- 块是外存分配基本单位，也是 I/O 传输基本单位。

外存空间分配方法：

- 空闲块链。
- UNIX 空闲块表。
- 位示图。

进程与外存对应关系：

| 管理方式 | 外存对应 |
|---|---|
| 界地址 | 每进程一组连续外存块，双对界可两组 |
| 页式 | 内存一页，外存一块 |
| 段式 | 每段占外存若干连续块 |
| 段页式 | 内存一页，外存一块 |

## 2. 虚拟页式存储系统

### 2.1 【必会】基本原理

虚拟页式存储管理：

```text
进程运行前：全部装入外存，部分装入内存。
进程运行时：访问页不在内存，发生缺页中断。
```

缺页中断处理程序要做：

1. 找到访问页在外存的位置。
2. 在内存中找一个空闲页框。
3. 如果没有空闲页框，按置换算法淘汰一个页。
4. 如果被淘汰页被修改过，写回外存。
5. 修改页表和总页表。
6. 读入所需页面。
7. 更新页表/快表。
8. 恢复现场，重新启动发生中断的指令。

【易错】缺页中断处理完后，不是执行下一条指令，而是重新执行导致缺页的那条指令。

### 2.2 【必会】虚拟页式地址映射

逻辑地址：

```text
(p, d)
```

硬件先做：

1. 查快表，若命中得到页框号 `f`，形成物理地址 `(f,d)`。
2. 若未命中，检查页号 `p` 是否越界。
3. 查页表。
4. 若页在内存，得到 `f`，更新快表，形成物理地址。
5. 若页不在内存，产生缺页中断。

软件再做缺页处理：

```text
保存现场 -> 找外存块 -> 找空闲页框或淘汰页 -> 必要时写回 -> 读入页面 -> 更新页表/快表 -> 恢复现场
```

### 2.3 【必会】页表字段的改进

虚拟页式页表不再只记录页框号，还要记录：

| 字段 | 含义 |
|---|---|
| 逻辑页号 | 页表项对应的逻辑页 |
| 页框号 | 若在内存，对应物理页框 |
| 外存块号 | 若不在内存，页面在外存的位置 |
| 内外标识 | 标记页面是否在内存 |
| 访问权限 | `r/w/e` |
| 修改标志 | 页面是否被写过 |

快表也要改进：

- 逻辑页号。
- 页框号。
- 访问权限。
- 修改标志。

【题型】真题大题说“页表表项为：页框号、访问位、内外标志”，这就是虚拟页式页表字段的简化版本。

## 3. 页框与外存块分配策略

### 3.1 【重点】内存页框静态分配策略

课件列出：

| 策略 | 含义 |
|---|---|
| 平均分配 | 每个进程分配相同数量页框 |
| 按进程长度比例分配 | 进程越大，分配越多 |
| 按优先级比例分配 | 优先级高，分配越多 |
| 按长度和优先级比例分配 | 综合大小和优先级 |

按长度比例分配：

```text
Pi 有 si 个页面
S = Σsi
内存共有 m 个页框
ai = (si / S) * m
```

静态策略的问题：

```text
没有反映程序结构，也没有反映程序不同时刻的行为特性。
```

所以后面引入工作集、页故障率反馈等动态方法。

### 3.2 外存块分配策略

课件讲两类：

| 策略 | 含义 | 优点 | 缺点 |
|---|---|---|---|
| 静态分配 | 外存保持进程全部页面 | 淘汰未修改页面时不必写回，速度快 | 浪费外存 |
| 动态分配 | 外存仅保持不在内存的页面 | 节省外存 | 淘汰时通常要写回，速度慢 |

注意：

```text
静态/动态外存分配不是内存静态/动态分区。
```

## 4. 页面调入时机

### 4.1 【必会】请求调页 demand paging

请求调页：

```text
发生缺页中断时才把页面调入。
```

优点：

- 不提前装入可能不用的页面。
- 节省内存和 I/O。

缺点：

- 第一次访问某页时会缺页，产生中断开销。

### 4.2 【必会】预调 prepaging

预调：

```text
在真正访问前，根据预测提前调入页面。
```

依据通常是程序顺序行为或空间局部性。

课件强调：

```text
预调必须辅以请调。
```

原因：

预测不一定准。预调没调到的页面，仍要靠请求调页在缺页时补救。

### 4.3 【题型】真题中的请求 + 预调

真题存储管理大题给出规则：

```text
页面调度时机为请求 + 预调形式。
每次发生页面调请时：
如果紧邻调页面的下一逻辑页面已经加载到内存，则只加载当前请调页面；
否则同时加载当前请调页面及紧邻的下一逻辑页面。
```

答题时要按这个规则走：

1. 先算访问地址对应的逻辑页。
2. 若该页不在内存，发生缺页。
3. 查看下一逻辑页是否已在内存。
4. 若下一页不在内存，则当前页和下一页都要装入。
5. 每装入一个页面，都要分配页框；若页框数超过进程固定分配数量，就按局部置换算法淘汰。

【易错】预调加载的下一页也占用页框，也会影响后续置换。

## 5. 页面置换算法总览

### 5.1 置换算法用于哪里

课件说置换算法用于：

- 页淘汰。
- 段淘汰。
- 快表淘汰。

目标：

```text
降低 page-fault rate，降低缺页率。
```

### 5.2 【必会】OPT 最佳淘汰算法

OPT：

```text
淘汰将来最长时间以后才会用到的页面。
```

优点：

- 理论上缺页率最低。

缺点：

- 不可实现。

原因：

```text
操作系统无法预知未来页面访问序列。
```

考试定位：

- OPT 常作为理论最优基准。
- 真正系统不会直接实现 OPT。

### 5.3 【必会】FIFO

FIFO：

```text
淘汰最先调入内存的页面。
```

实现：

```text
队列。
```

优点：

- 简单。

缺点：

- 先进入的页面不一定不再使用。
- 可能出现 Belady 异常。

### 5.4 【必会】Belady 异常

Belady 异常：

```text
分配更多物理页框，缺页次数反而增加。
```

课件例子：

```text
访问序列：1,2,3,4,1,2,5,1,2,3,4,5
FIFO：
3 个页框，缺页 9 次；
4 个页框，缺页 10 次。
```

考试判断：

- FIFO 可能有 Belady 异常。
- LRU 和 OPT 没有这种异常。

### 5.5 【必会】LRU

LRU：

```text
淘汰最近一次访问距当前时间最长的页面。
```

英文：

```text
Least Recently Used
```

直觉：

```text
过去很久没用的页面，将来也可能暂时不用。
```

实现思想：

- 栈。
- 计时器。
- 近似算法。

做题模板：

```text
每访问一页，把它视为“最新”。
缺页时淘汰当前最久未访问的页。
```

### 5.6 【重点】NUR 最近不用

NUR：

```text
淘汰最近一段时间未用到的页面。
```

实现：

- 每页增加访问标志。
- 访问时置 1。
- 定时清 0。
- 淘汰时选访问标志为 0 的页。

它是 LRU 的近似。

### 5.7 LFU 与 MFU

LFU：

```text
淘汰使用次数最少的页面。
```

缺点：

- 前期使用多、以后不用的页面难以换出。
- 刚调入页面引用少，可能很快被换出。

MFU：

```text
淘汰使用次数最多的页面。
```

依据：

```text
使用多的页面可能已经用完。
```

缺点：

```text
有些程序成分整个运行期间都经常使用，不能简单淘汰。
```

## 6. 二次机会、时钟与改进时钟

### 6.1 【必会】二次机会算法

二次机会：

```text
淘汰装入最久且最近未被访问的页面。
```

每页有访问位 `r`：

- `r=0`：最近没访问，可以淘汰。
- `r=1`：最近访问过，给第二次机会，把 `r` 清 0，暂不淘汰。

二次机会通常用链表组织页面。

### 6.2 【必会】时钟算法

时钟算法把页面组织成环形，用一个指针指向当前位置。

淘汰时：

1. 看指针所指页面的访问位 `r`。
2. 若 `r=0`，淘汰该页。
3. 若 `r=1`，把 `r` 清 0，指针顺时针移动。
4. 重复直到找到 `r=0` 的页面。

课件强调：

```text
时钟算法与二次机会淘汰效果基本相同；
区别是数据结构不同。
```

时钟算法可直接利用页表中的引用位，加一个指针，速度快且省空间。

### 6.3 【必会】改进的时钟算法

改进时钟算法考虑修改标志 `m`。

四类页面：

| r | m | 含义 | 优先级 |
|---|---|---|---|
| 0 | 0 | 最近未访问，未修改 | 最佳淘汰 |
| 0 | 1 | 最近未访问，已修改 | 可淘汰，但要写回 |
| 1 | 0 | 最近访问，未修改 | 暂不淘汰 |
| 1 | 1 | 最近访问，已修改 | 暂不淘汰 |

直觉：

```text
最好淘汰既没被访问、又没被修改的页；
修改过的页淘汰前要写回，代价更高。
```

### 6.4 【必会】改进时钟步骤

课件步骤：

1. 从指针当前位置开始扫描，寻找第一个 `(r=0, m=0)` 的页面。扫描时不改变引用位。
2. 如果失败，再次从原位置开始扫描，寻找第一个 `(r=0, m=1)` 的页面；同时将扫描过页面的 `r` 清 0。
3. 如果仍失败，指针回到原位置，重新执行步骤 1 和步骤 2。此时一定能找到。

考试口诀：

```text
先找干净冷页 00；
再找脏冷页 01，边扫边清 r；
清完后再来一轮，必能找到。
```

【易错】第一轮找 `(0,0)` 时不清引用位；第二轮找 `(0,1)` 时才把扫描过页面的 `r` 清 0。

## 7. 置换算法例题模板

### 7.1 【题型】FIFO/LRU 缺页次数

做表格：

```text
访问序列：...
页框1：
页框2：
页框3：
缺页？ × / 空
```

每访问一个页：

1. 若页已在内存：不缺页，LRU 要更新最近使用次序。
2. 若页不在内存且有空页框：装入，缺页一次。
3. 若页不在内存且无空页框：按算法淘汰，装入，缺页一次。

### 7.2 【题型】2010 考研风格页式地址题

课件给例：

```text
进程空间和内存空间都是 64K；
页长 1K；
某进程 6 个页，内存分配 4 个页框；
访问逻辑地址 13B7H。
```

页长：

```text
1K = 2^10
```

所以低 10 位是页内地址。

`13B7H` 二进制为：

```text
0001001110110111
```

前 6 位是逻辑页号，后 10 位是页内地址。逻辑页号为：

```text
4
```

做题关键：

```text
页大小决定页内地址位数；
逻辑页号由高位得到。
```

若页 4 不在内存，发生缺页。题目再指定 FIFO 或 CLOCK，就按对应算法找被淘汰页框，再形成物理地址。

## 8. 颠簸、工作集与 PFFB

### 8.1 【必会】颠簸 thrashing

课件定义：

```text
页面在内存与外存之间频繁换入换出。
```

原因：

1. 分给进程的物理页框过少。
2. 淘汰算法不合理。

处理：

1. 增加分给进程的物理页框数。
2. 改进淘汰算法。

【题型】真题选择：

```text
CPU 利用率 20%，交换空间磁盘利用率 97.7%，其他设备 5%。
怎样提高性能？
```

这是典型抖动/颠簸：CPU 很闲，磁盘疯狂忙于换页。正确思路：

```text
加内存条来增加物理空间容量。
```

不要选“增加运行进程数”，那会让页框更紧张，颠簸更严重。

### 8.2 【必会】工作集 working set

工作集：

```text
进程在一段时间内所访问页面的集合。
```

记作：

```text
WS(t, Δ)
```

其中 `Δ` 是窗口尺寸。

例：

```text
在窗口 Δ 内访问序列为：
5, 7, 7, 1, 6, 2, 2
```

工作集是不同页面集合：

```text
{5, 7, 1, 6, 2}
```

【题型】真题选择：

```text
如果在 Δ 周期内确定某进程的工作集大小为 n，则 n 的含义是？
```

答案：

```text
该进程在 Δ 周期内访问不同页面的次数/个数。
```

更准确说：工作集大小是该周期访问的不同页面数。

### 8.3 窗口尺寸影响

窗口 `Δ` 过小：

```text
活动页面不能全部装入，页故障率高。
```

窗口 `Δ` 过大：

```text
包含太多不活跃页面，浪费内存。
```

课件提到：

```text
为使程序有效运行，工作集应能放入内存。
```

### 8.4 工作集估算

课件给公式：

```text
τ(n+1) = α * Wn + (1 - α) * τn
```

其中：

- `Wn`：第 n 个 Δ 周期实际工作集大小。
- `τn`：第 n 个 Δ 周期工作集估算值。
- `α`：通常取 0.5。

考试一般不重算复杂估计，但要知道它是用历史工作集大小估算未来需求。

### 8.5 【重点】PFFB 页故障率反馈

PFFB：

```text
Page Fault Feed Back
```

页故障率：

```text
访问内存缺页次数 / 访问内存总次数
```

思想：

```text
根据页故障率动态调整页框分配。
```

规则：

- 页故障率达到上界阈值：说明内存页面少，增加页框。
- 页故障率达到下界阈值：说明页框可能过多，减少页框。

## 9. 有效访问时间与缺页率

### 9.1 【题型】最大缺页率

课件作业 #5：

```text
页表保存在寄存器中。
有空页或被置换页未修改：处理缺页中断 8ms。
被置换页已修改：处理缺页中断增加写回时间，共 20ms。
一次内存访问时间 1ns。
70% 被置换页被修改过。
为保证有效访问时间不超过 12ns，可接受最大缺页率是多少？
```

缺页处理平均时间：

```text
0.3 * 8ms + 0.7 * 20ms
= 2.4ms + 14ms
= 16.4ms
```

设缺页率为 `p`。

有效访问时间近似：

```text
EAT = (1 - p) * 1ns + p * (16.4ms)
```

因为缺页时缺页处理时间远大于 1ns，也可写成：

```text
EAT ≈ 1ns + p * 16.4ms
```

要求：

```text
EAT <= 12ns
```

所以：

```text
p <= (12ns - 1ns) / 16.4ms
```

单位换算：

```text
16.4ms = 16.4 * 10^6 ns
```

于是：

```text
p <= 11 / (16.4 * 10^6)
  ≈ 6.7 * 10^-7
```

答题重点：

```text
缺页率必须极低，因为一次缺页的代价比一次内存访问大太多。
```

### 9.2 【题型】访问序列平均有效访问时间

课件作业 #5 另一题给出：

- LRU。
- 页面大小 4KB。
- 内存访问 100ns。
- 快表访问 20ns。
- 缺页中断处理 25ms。
- 进程长度 30KB。
- 分配 3 块。
- 快表命中率 20%。
- 页面访问序列 20 个。

解题框架：

1. 根据访问序列和 3 个页框，用 LRU 统计缺页次数 `F`。
2. 缺页率：

```text
p = F / 20
```

3. 非缺页访问时间：

```text
TLB 命中：20ns + 100ns
TLB 未命中：20ns + 2 * 100ns
```

4. 按命中率算正常访问平均时间：

```text
EAT_normal = 0.2 * (20 + 100) + 0.8 * (20 + 200)
```

5. 再把缺页处理时间纳入：

```text
EAT = (1 - p) * EAT_normal + p * (缺页处理时间 + EAT_normal)
```

可简化：

```text
EAT = EAT_normal + p * 缺页处理时间
```

【易错】缺页处理时间是 ms，普通访存是 ns，必须统一单位。

## 10. 虚拟段式存储系统

### 10.1 基本原理

虚拟段式：

```text
进程运行前，全部装入外存，部分装入内存；
访问段不在内存时，发生缺段中断。
```

逻辑地址：

```text
(s, d)
```

若段在内存：

```text
物理地址 = 段内存首址 b' + d
```

若段不在内存：

```text
缺段中断。
```

### 10.2 虚拟段式段表字段

段表要记录：

| 字段 | 含义 |
|---|---|
| 段号 | 逻辑段编号 |
| 段长 | 用于段内越界检查 |
| 内存首址 | 段在内存时的位置 |
| 外存首址 | 段在外存的位置 |
| 权限 | `r/w/e` |
| 内外标识 | 是否在内存 |
| 修改标志 | 是否被修改 |

快表也包含：

- 段号。
- 段长。
- 内存首址。
- 权限。
- 修改标志。

### 10.3 虚拟段式地址映射

步骤：

1. 给出 `(s,d)`。
2. 查快表，若命中得到 `b'` 和 `l'`。
3. 检查段号越界。
4. 查段表。
5. 检查段内地址越界。
6. 若段在内存，形成 `b' + d`。
7. 若段不在内存，发生缺段中断。
8. 缺段处理可能需要紧凑或淘汰某段。
9. 修改段表和快表，恢复现场。

虚拟段式的复杂点：

```text
段长不固定，调入时需要连续内存空间。
```

所以可能涉及紧凑。

## 11. 动态连接

### 11.1 【重点】静态连接 vs 动态连接

静态连接：

```text
运行前由 link 完成连接。
```

动态连接：

```text
运行时由操作系统完成连接。
```

静态连接缺点：

- 连接时间长。
- 目标代码长。
- 某些连接段可能根本不执行，但仍被连接进来。

动态连接优点：

- 用到时再连接。
- 便于共享库。
- 减少不必要装入。

### 11.2 动态连接的核心机制

编译/汇编时：

遇到访问外段指令，采用间接寻址，指向一个间接字。

间接字中有：

| 字段 | 含义 |
|---|---|
| `L=1` | 未连接 |
| `L=0` | 已连接 |
| `D` | 符号地址或逻辑地址 |

执行时：

如果遇到间接指令且 `L=1`，发生连接中断。

连接中断处理：

1. 由 `D` 取出符号地址。
2. 查段名-段号对照表。
3. 若已分配段号，查段表找到该段，再由符号表找到段内地址。
4. 把 `(段号, 段内地址)` 写回 `D`，把 `L` 置 0。
5. 若未分配段号，找到该段文件，读入内存/外存，分配段号，填写表，再重复。

### 11.3 动态连接与共享的矛盾

课件指出：

```text
动态连接要修改连接字；
段共享要求纯代码 pure code。
```

如果共享代码段被多个进程共享，某个进程修改连接字会破坏共享纯代码属性。

解决：

```text
共享代码分为纯段和杂段；
纯段共享，杂段私用。
```

## 12. 虚拟段页式存储系统

### 12.1 基本思想

虚拟段页式考虑：

- 段的动态连接。
- 段的共享。
- 段长度的动态变化。

逻辑地址：

```text
(s, p, d)
```

物理地址：

```text
(f, d)
```

### 12.2 所需表目

课件列出：

| 表 | 主要字段 |
|---|---|
| 段表 | 段号、页表长度、页表首址、访问权限、扩展标志、共享段入口 |
| 页表 | 逻辑页号、内存页号、外存块号、内外标志、修改标志 |
| 共享段表 | 段名、页表长度、页表首址、扩充标志、共享计数 |
| 段名-段号对照表 | 每进程一个，支持动态连接 |

### 12.3 虚拟段页式地址映射

步骤：

1. 由 `(s,p)` 查快表，若命中得到页框号 `f`。
2. 检查段号是否越界。
3. 检查访问权限。
4. 查段表得到页表首址 `b'` 和页表长度 `l'`。
5. 检查页号 `p` 是否越界。
6. 查页表。
7. 如果有障碍位/间址，可能发生连接中断。
8. 如果页不在内存，发生缺页中断。
9. 若都正常，形成物理地址 `(f,d)`。

### 12.4 虚拟段页式可能发生的中断

课件列出：

| 中断 | 处理 |
|---|---|
| 连接中断 | 建页表、读入外存/内存、分配段号、填段名-段号表、共享段表、段表 |
| 缺页中断 | 调入所需页面，更新页表和总页表 |
| 越界中断 | 段号越界或页号越界；若段可扩展则扩展，否则错误 |
| 越权中断 | 错误处理 |

### 12.5 【必会】三种虚拟存储比较

课件表：

| 方式 | 地址空间 | 存储分配 | 存储碎片 | 存储共享 | 存储保护 | 动态扩充 | 动态连接 |
|---|---|---|---|---|---|---|---|
| 虚拟页式 | 一维 | 简单 | 无外部碎片 | 不便 | 不便 | 不可 | 不可 |
| 虚拟段式 | 二维 | 复杂 | 有外部碎片 | 方便 | 方便 | 可以 | 可以 |
| 虚拟段页式 | 二维 | 简单 | 无外部碎片 | 方便 | 方便 | 可以 | 可以 |

【易错】表中“无”通常指无外部碎片。页式和段页式仍可能有内部碎片。

## 13. 真题虚拟页式大题模板

### 13.1 题目常见条件

真题给出的条件类似：

```text
一级页表；
页表空间由系统动态分配；
页框大小 4KB；
每个进程分配页框数固定为 6；
局部置换；
LRU；
页面调度时机为请求 + 预调；
空闲页框表；
最佳适应；
访问逻辑地址序列；
要求写页表、空闲页框表、物理地址。
```

这类题其实是第 6 章和第 7 章合体：

- 第 6 章：页表首址、页表长度、页号/页内地址、空闲页框表分配。
- 第 7 章：缺页、预调、置换、访问位/内外标志、页表变化。

### 13.2 【题型】完整解题流程

1. 计算页大小：

```text
4KB = 2^12，所以低 12 位为页内地址。
```

2. 计算进程逻辑页数：

```text
逻辑页数 = ceil(进程长度 / 页大小)
页表长度 = 逻辑页数
```

3. 为页表分配页框：

```text
页表项数 * 页表项大小 = 页表字节数
按空闲页框表和分配算法给页表本身分配页框。
```

4. 对每个访问地址：

```text
逻辑页号 p = 地址 / 页大小
页内地址 d = 地址 % 页大小
```

5. 若页在内存：

```text
更新访问位；
若是写操作，修改位也置 1；
按 LRU 更新最近使用顺序。
```

6. 若页不在内存：

```text
发生缺页；
按请求+预调规则决定装入哪些页；
若有空闲页框，分配页框；
若超过固定页框数，按局部置换算法淘汰本进程页面；
若被淘汰页修改过，写回外存；
更新页表和空闲表；
更新访问位/修改位/内外标志。
```

7. 最后访问的地址若页在内存，计算物理地址：

```text
物理地址 = 页框号 * 页大小 + 页内地址
```

### 13.3 【易错】请求 + 预调与固定页框数

如果一次缺页触发装入当前页和下一页，相当于可能连续装入 2 页。

这会带来两个后果：

1. 可能一次缺页消耗两个空闲页框。
2. 若进程固定页框数已满，可能需要多次置换。

答题时不要只处理被访问页，忘了预调页。

### 13.4 【易错】访问位、修改位、内外标志

常见规则：

| 操作 | 访问位 | 修改位 | 内外标志 |
|---|---|---|---|
| 读/引用已在内存页 | 置 1 | 不变 | 内 |
| 写/修改已在内存页 | 置 1 | 置 1 | 内 |
| 调入新页 | 置 1 或按题意 | 修改位通常 0，若本次是写则 1 | 内 |
| 淘汰页 | 可清空/保留题目格式 | 若修改过需写回 | 外 |

题目若给特殊规定，以题目为准。

## 14. 易混概念总表

| 易错点 | 正确理解 |
|---|---|
| 虚拟存储让内存真实变大 | 它用外存和动态调入制造“大地址空间”效果 |
| 基本分页等于请求分页 | 基本分页不一定缺页，请求分页才按需调入 |
| 缺页中断后执行下一条指令 | 应重新执行导致缺页的指令 |
| 预调可以替代请求调页 | 预调必须辅以请求调页 |
| OPT 可以实际实现 | OPT 需要预知未来，只能作理论基准 |
| FIFO 页框越多缺页一定越少 | FIFO 可能出现 Belady 异常 |
| LRU 淘汰最早装入页 | LRU 淘汰最久未被访问页 |
| 时钟算法看到 r=1 就淘汰 | r=1 时清 0 并给第二次机会 |
| 改进时钟先找 r=0,m=1 | 先找 r=0,m=0 |
| 修改位无所谓 | 修改页淘汰前要写回，代价更大 |
| 工作集大小是访问次数 | 是窗口内访问的不同页面数 |
| 页故障率高应增加进程数 | 通常应增加页框或减少负载 |
| 抖动时 CPU 很忙 | 抖动常表现为 CPU 利用率低、交换磁盘忙 |
| 虚拟段式不需要连续空间 | 段调入仍需要连续内存区，可能紧凑 |
| 虚拟段页式只有缺页中断 | 还可能有连接、越界、越权中断 |

## 15. 课内题型归纳与模板

### 15.1 【题型】缺页处理流程

答题模板：

```text
保存现场；
查页表得到外存块号；
寻找空闲页框；
若无空闲页框，按置换算法选淘汰页；
若淘汰页修改过，写回外存；
读入所需页；
更新页表/总页表/快表；
恢复现场；
重新执行中断指令。
```

### 15.2 【题型】置换算法

模板：

```text
OPT：淘汰未来最晚使用页。
FIFO：淘汰最早进入内存页。
LRU：淘汰最久未访问页。
CLOCK：r=0 淘汰，r=1 清零跳过。
改进 CLOCK：先 00，再 01，扫描中清 r。
```

### 15.3 【题型】工作集选择题

模板：

```text
工作集 = 某时间窗口内访问过的不同页面集合。
工作集大小 = 不同页面个数。
```

### 15.4 【题型】抖动判断

模板：

```text
CPU 利用率低；
交换区磁盘利用率高；
缺页频繁；
说明抖动。
改进：增加页框/内存，降低多道程度，改进置换算法。
```

### 15.5 【题型】有效访问时间

模板：

```text
先算正常访存时间；
再算缺页平均处理时间；
EAT = EAT_normal + 缺页率 * 缺页处理时间
注意统一单位。
```

## 17. 本章一句话总结

第 7 章的核心是：

```text
进程看到一个很大的地址空间，
系统只把当前需要的一小部分放入内存；
访问不到时靠缺页/缺段中断调入，
内存不够时靠置换算法腾位置。
```

会把“访问地址 -> 是否缺页 -> 调入/淘汰 -> 更新页表 -> 算物理地址”这条链条走顺，第 7 章的大题就稳了。
