## 1. 存储管理到底在管什么

### 1.1 内存为什么需要操作系统管理

程序运行时必须把指令和数据放进内存。问题是：

- 内存比外存小得多。
- 系统中有多个进程。
- 每个进程都希望自己像独占一片地址空间。
- 进程不能随便访问别人的内存。
- 有些代码或数据又希望被共享，节省内存。

所以操作系统要在“隔离”和“共享”之间做管理：

```text
该隔离的隔离，该共享的共享；
该保护的保护，该映射的映射。
```

### 1.2 【必会】存储管理五大功能

课件列出存储管理功能：

| 功能 | 直觉 | 课内要点 |
|---|---|---|
| 存储分配和去配 | 给进程分内存，用完回收 | 对象可包括内存、外存 |
| 存储共享 | 多个进程共用代码/数据 | 节省内存、进程通信 |
| 存储保护 | 防止乱访问 | 防止地址越界、防止操作越权 |
| 存储扩充 | 用外存扩展内存效果 | 内外存结合形成虚拟存储体系 |
| 地址映射 | 逻辑地址变物理地址 | 依赖硬件支持 |

习题课直接把这五点列入回顾，属于 `【必会】`。

### 1.3 【重点】分配和去配的时刻

课件提到存储分配/去配可能发生在：

- 进程创建。
- 进程撤销。
- 交换。
- 进程长度变化，例如栈溢出、`execl`。

理解：

```text
内存不是只在程序启动时分一次。
进程运行过程中，内存需求也可能变化。
```

### 1.4 地址映射需要硬件支持

课件列出硬件支持：

- 基址寄存器。
- 限长寄存器。
- 快表 TLB。

硬件使用这些寄存器完成地址映射；如果不能正常完成映射，就产生中断。

常见异常：

- 地址越界。
- 操作越权。
- 缺页/缺段，放第 7 章虚拟存储展开。

## 2. 逻辑地址与物理地址

### 2.1 【必会】两个地址空间

程序中看到的地址通常不是内存真实地址。

| 地址 | 含义 |
|---|---|
| 逻辑地址 | 程序生成的地址，也叫相对地址、虚拟地址语境下的程序地址 |
| 物理地址 | 内存条上的真实地址 |

地址映射就是：

```text
逻辑地址 -> 物理地址
```

### 2.2 为什么不能直接用物理地址

如果程序直接使用物理地址，会有很多问题：

1. 程序每次装入位置不同，地址就要重写。
2. 多个进程可能访问同一物理地址，互相破坏。
3. 进程交换进出内存后，位置变化，程序无法适应。
4. 难以做保护和共享。

所以现代系统让程序使用逻辑地址，再由硬件和操作系统做转换。

### 2.3 【易错】地址映射不是纯软件做的

如果每条指令访问内存都靠操作系统软件转换地址，速度会非常慢。

课件强调地址映射需要硬件支持。操作系统负责建立表和寄存器内容，硬件在程序运行时快速完成转换。

## 3. 内存资源管理

### 3.1 【必会】内存分区的两个维度

课件按两个维度分类：

| 维度 | 类型 | 含义 |
|---|---|---|
| 分区时刻 | 静态分区 | 系统初始化时划分 |
| 分区时刻 | 动态分区 | 申请时划分 |
| 分区大小 | 等长分区 | 每个分区大小相同，通常是 `2^i` |
| 分区大小 | 异长分区 | 按程序、程序单位、对象大小划分 |

通常组合：

```text
静态 + 等长：页式、段页式
动态 + 异长：段式、界地址
```

### 3.2 静态等长分区的管理方法

课件列出三种：

- 位示图。
- 空闲页面表。
- 空闲页面链。

这些主要用于页框/页面这类等长块管理。

### 3.3 【必会】位示图 bit map

位示图用一个 bit 表示一个页框是否空闲。

课件规定：

```text
0 表示空闲，1 表示占用。
```

分配：

```text
从头寻找第一个为 0 的位，改为 1，返回页号。
```

去配：

```text
把页号对应的 bit 置为 0。
```

优点：

- 结构紧凑。
- 适合管理大量等长块。

缺点：

- 查找连续空闲块时可能要扫描较多 bit。

### 3.4 【必会】空闲页面表

空闲页面表记录连续空闲页框区间：

```text
首页号 + 空页数
```

特点：

```text
可以分配连续页面。
```

课件特别提到 DMA 可能要求连续页面，因此空闲页面表在这类场景有意义。

### 3.5 【重点】空闲页面链

空闲页面链把空闲页框串成链表。

优点：

- 节省空间。
- 分配一个空闲页框很方便。

缺点：

- 不适合管理外存。
- 不方便快速找到连续空闲区。

### 3.6 动态异长分区的数据结构

动态异长分区用空闲区表：

```text
空闲区首址 + 空闲区长度
```

课件标准：

```text
尽量使空闲区域连续。
```

初始时整个空闲内存通常是一个连续空闲区。

## 4. 【必会】动态异长分区分配算法

### 4.1 最先适应 First Fit

规则：

```text
空闲区按首址递增排列；
申请时取第一个能满足大小要求的空闲区。
```

优点：

- 尽量使用低地址空间。
- 高地址区域可能保留较大的空闲区。
- 查找相对简单。

缺点：

- 可能不断分割低地址大空闲区。
- 低地址处容易形成很多小碎片。

考试判断：

```text
FF 不是找最小，也不是找最大，而是按地址顺序找第一个够用的。
```

### 4.2 下次适应 Next Fit

规则：

```text
从上次分配位置的下一个空闲区开始循环查找；
找到第一个可满足区域后分配；
分配后指针调整到该区域的下一个位置。
```

优点：

- 减少每次从表头扫描的时间开销。
- 空闲区分布更均匀。

缺点：

- 仍可能分割大空闲区。
- 可能破坏高地址大空闲区。

### 4.3 最佳适应 Best Fit

规则：

```text
取能满足申请的最小空闲区。
```

优点：

- 尽量使用小空闲区。
- 试图保留大空闲区。

缺点：

- 容易形成非常小、难以再利用的碎片。

【题型】真题存储管理大题中提到：

```text
主存空闲区管理采用空闲页框表结构，存储分配采用最佳适应算法。
```

看到“最佳适应”，就要在所有可满足空闲区中选最小的那个，而不是第一个。

### 4.4 最坏适应 Worst Fit

规则：

```text
取最大的可满足空闲区。
```

优点：

- 分割后剩余区域仍较大，不容易留下极小碎片。

缺点：

- 会不断切割大空闲区。
- 可能很快失去大块连续空间。

### 4.5 【题型】分配算法判断模板

做空闲区分配题时：

1. 列出空闲区：

```text
(首址, 长度)
```

2. 删掉不能满足申请大小的空闲区。

3. 按算法选择：

| 算法 | 选择 |
|---|---|
| FF | 首址递增时第一个够用 |
| NF | 从上次位置后第一个够用 |
| BF | 够用且长度最小 |
| WF | 够用且长度最大 |

4. 更新空闲区：

```text
若从首部切出 size：
空闲区首址 += size
空闲区长度 -= size
若长度为 0，删除该表项。
```

5. 回收时注意相邻空闲区合并。

## 5. 碎片与紧凑

### 5.1 【必会】碎片是什么

碎片是“总空间看起来够，但因为分布或块大小问题，难以有效使用”的空间。

分两类：

| 类型 | 产生位置 | 典型机制 |
|---|---|---|
| 内部碎片 | 已分配块内部 | 固定分区、页式、段页式 |
| 外部碎片 | 已分配块之间 | 动态分区、段式 |

### 5.2 【题型】不会产生内部碎片的是谁

习题课模拟选择：

```text
不会产生内部碎片的存储管理是？
A 页式
B 段式
C 固定分区式
D 段页式
```

答案：

```text
段式存储管理。
```

原因：

- 页式按固定页框分配，最后一页可能用不满，产生内部碎片。
- 固定分区也会有内部碎片。
- 段页式每段内分页，最后一页仍可能用不满。
- 段式按段实际长度动态异长分配，不产生内部碎片，但可能产生外部碎片。

### 5.3 紧凑 compaction

紧凑是：

```text
移动已占用区域，使所有空闲区域连成一片。
```

它主要用于处理外部碎片。

缺点：

- 开销很大。
- 需要移动内存内容。
- 需要重定位支持，否则移动后地址失效。

应试判断：

- 紧凑解决外部碎片。
- 页式通常不需要紧凑来解决外部碎片，因为物理页框可不连续。

## 6. 界地址管理方式

### 6.1 【必会】基本原理

界地址管理方式是一维地址方式。

内存空间：

```text
动态异长划分。
```

进程空间：

```text
一个进程一个连续区域，逻辑地址从 0 到 l-1。
```

进程空间和内存空间对应关系：

```text
逻辑地址 0       -> 物理地址 b
逻辑地址 a       -> 物理地址 b + a
逻辑地址 l - 1   -> 物理地址 b + l - 1
```

### 6.2 所需表目和寄存器

表目：

- 内存分配表，在 PCB 中。
- 空闲区域表，系统维护。

寄存器：

| 寄存器 | 含义 |
|---|---|
| 基址寄存器 `b` | 当前运行进程起始物理地址 |
| 限长寄存器 `l` | 当前运行进程长度 |

### 6.3 【必会】地址映射

给定逻辑地址 `a`：

1. 判断是否越界：

```text
0 <= a <= l - 1
```

不满足则越界中断。

2. 计算物理地址：

```text
物理地址 = b + a
```

例子：

```text
b = 1000
l = 400
a = 250
```

因为 `0 <= 250 <= 399`，合法：

```text
物理地址 = 1000 + 250 = 1250
```

若 `a = 400`，则越界，因为最大合法逻辑地址是 `399`。

### 6.4 【重点】双对界

经典 UNIX 把进程空间分为：

- I 空间：代码/指令空间。
- D 空间：数据空间。

每个空间各有一对界：

```text
代码：b1, l1
数据：b2, l2
```

硬件能识别逻辑地址属于哪个空间，然后使用对应寄存器映射。

### 6.5 真题选择：界地址管理代价最小

真题选择：

```text
操作系统为实现多道程序并发，对内存管理可以采用多种方式，其中代价最小的是？
A 界地址管理
B 分页管理
C 分段管理
D 段页式管理
```

答案：

```text
界地址管理。
```

原因：

界地址只需要基址/限长寄存器，地址映射简单；分页、分段、段页式都需要表结构和更复杂的查询。

## 7. 交换与覆盖

### 7.1 【重点】交换 swapping

交换又称换进换出：

```text
进程在内存和外存之间动态调度。
```

目的：

```text
缓解内存空间不足。
```

典型策略：

- 外存有可运行进程时，系统试图调入内存。
- 内存空间不足时，将某些暂时不可运行进程移出到外存。

交换要求程序可重定位，因为进程再次装入内存的位置可能和换出前不同。

### 7.2 可重定位程序

课件定义：

```text
程序编址与内存存放位置无关的程序，称为可重定位程序。
```

从 0 开始编址的程序满足重定位要求，也叫浮动程序。

### 7.3 【重点】覆盖技术

覆盖技术：

```text
将较大程序装入较小进程空间的技术。
```

做法：

- 全局代码和数据静态装入内存。
- 其他部分动态装入。
- 后装入的成分重复使用先装入成分的存储区，即覆盖先装入成分。

交换 vs 覆盖：

| 对比 | 交换 | 覆盖 |
|---|---|---|
| 粒度 | 进程级换进换出 | 程序模块级覆盖 |
| 目的 | 缓解多进程内存不足 | 让大程序在小空间运行 |
| 管理者 | 操作系统为主 | 程序结构/覆盖驱动参与 |

## 8. 页式存储管理

### 8.1 【必会】页框与页面

页式管理把内存和进程空间都切成等长块。

| 概念 | 所在空间 | 含义 |
|---|---|---|
| 页框 frame | 物理内存 | 固定大小的物理块 |
| 页面 page | 进程逻辑空间 | 固定大小的逻辑块 |

课件设页大小为 `2^i`。

物理地址可拆为：

```text
物理地址 = 页框号 f + 页内地址 d
```

逻辑地址可拆为：

```text
逻辑地址 = 逻辑页号 p + 页内地址 d
```

更具体：

```text
逻辑地址 = p * 页面大小 + d
物理地址 = f * 页面大小 + d
```

### 8.2 【必会】页号和页内地址计算

如果页面大小为 `page_size`，逻辑地址为 `A`：

```text
p = A / page_size 取整
d = A % page_size
```

若页面大小是 `2^i`，则：

- 低 `i` 位是页内地址。
- 高位是页号。

例子：

```text
页面大小 = 4KB = 2^12
逻辑地址 = 8AC2H
```

则：

```text
页内地址 = AC2H
逻辑页号 = 8H
```

【易错】页内地址位数由页面大小决定，不由页表长度决定。

### 8.3 【必会】页表

每个进程一张页表，页表记录：

```text
逻辑页号 -> 物理页框号
```

课件还提到：

- 页表首址寄存器。
- 页表长度寄存器。
- 快表 TLB。
- 总页表，系统一个。

【题型】习题课选择：

```text
操作系统采用分页存储管理方式，要求？
```

答案要点：

```text
每个进程拥有一张页表，且进程的页表驻留在内存中。
```

原因：

CPU 地址转换时需要查页表，如果执行进程的页表不在内存中，无法完成常规地址映射。更严格地说，传统基本分页要求页表在内存中；虚拟化页表或多级页表会优化空间，但不改变“进程有页表”的基本事实。

### 8.4 【必会】分页地址映射

逻辑地址：

```text
(p, d)
```

转换为物理地址：

```text
(f, d)
```

步骤：

1. 程序给出逻辑地址 `(p,d)`。
2. 先查快表 TLB，看是否有 `p -> f`。
3. 若快表命中，直接得到页框号 `f`。
4. 若快表未命中，检查 `0 <= p <= l - 1` 是否越界。
5. 用页表首址 `b` 和页号 `p` 查页表，得到页框号 `f`。
6. 将 `(p,f)` 写入快表，必要时淘汰旧项。
7. 物理地址为 `(f,d)`，即 `f * page_size + d`。

### 8.5 【必会】有效访问时间 EAT

课件给出公式：

```text
EAT = 命中率 * (快表访问时间 + 内存访问时间)
    + 未命中率 * (快表访问时间 + 2 * 内存访问时间)
```

例子：

```text
快表命中率 = 98%
快表访问时间 = 20ns
内存访问时间 = 100ns
```

则：

```text
EAT = 98% * (20 + 100) + 2% * (20 + 200)
    = 117.6 + 4.4
    = 122ns
```

为什么未命中是 `2 * 内存访问时间`？

```text
一次访问页表，一次访问真正数据。
```

### 8.6 【重点】多级页表

为什么需要多级页表？

课件背景：

- 内存空间增长。
- 进程虚拟空间增长。
- 单级页表需要很大连续内存空间。
- 进程虚拟空间可能有空洞。

例子：

```text
32 位地址空间，页面大小 4KB。
页内偏移 12 位，页号 20 位。
单级页表最多 2^20 个表项。
```

多级页表把页表也分页。二级页表示例：

```text
逻辑地址 = p_i | p_j | d
           10位  10位  12位
```

`p_i` 查外页表，`p_j` 查内页表，`d` 是页内偏移。

### 8.7 多级页表访存次数

如果没有 TLB：

- 单级页表：查页表一次 + 访问数据一次 = 2 次内存访问。
- 二级页表：查外页表 + 查内页表 + 访问数据 = 3 次。
- 四级页表：查 4 级页表 + 访问数据 = 5 次。

课件给四级页表 EAT：

```text
EAT = 命中率 * (TLB + 内存)
    + 未命中率 * (TLB + 5 * 内存)
```

### 8.8 【重点】反置页表

传统页表：

```text
面向进程空间，每个进程逻辑页面有一项。
```

反置页表：

```text
面向物理内存，每个物理页框有一项。
```

优点：

- 大小由物理内存页框数决定，固定。
- 当进程虚拟空间很大时，可节省页表空间。

缺点：

- 查找慢。
- 通常需要散列技术和 TLB 加速。

## 9. 段式存储管理

### 9.1 【必会】为什么引入段式

页式按固定大小切块，对机器很方便，但对程序员和编译器来说，一个程序天然由不同逻辑单位组成：

- 主程序。
- 子程序。
- 函数。
- 数据段。
- 符号表。
- 栈。

段式按程序逻辑单位划分。

直觉：

```text
页是机器视角的等长块；
段是程序视角的逻辑单位。
```

### 9.2 【必会】段式基本原理

内存空间：

```text
动态异长，每区一段。
```

进程空间：

```text
若干段，每段一个程序单位。
```

逻辑地址是二维地址：

```text
(段号 s, 段内地址 d)
```

物理地址：

```text
段首址 b' + 段内地址 d
```

### 9.3 段表

每个进程有一张段表：

| 段号 | 段首址 | 段长度 |
|---|---|---|

还需要：

- 段表首址寄存器 `b`。
- 段表长度寄存器 `l`。
- 快表 TLB，可缓存段号、段首址、段长度。

### 9.4 【必会】段式地址映射

逻辑地址：

```text
(s, d)
```

步骤：

1. 程序给出 `(s,d)`。
2. 查快表，看是否有 `s -> (b', l')`。
3. 若未命中，检查段号是否越界：

```text
0 <= s <= l - 1
```

4. 用段表首址 `b` 和段号 `s` 查段表，得到段首址 `b'` 和段长 `l'`。
5. 检查段内地址是否越界：

```text
0 <= d <= l' - 1
```

6. 物理地址：

```text
b' + d
```

【易错】段式有两次越界检查：

- 段号是否超过段表长度。
- 段内地址是否超过该段长度。

### 9.5 【题型】段地址位数题

习题课模拟选择：

```text
采用分段存储管理系统中，若段地址用 24 位表示，
其中 8 位表示段号，则允许每段的最大长度为多少？
```

分析：

```text
总位数 24 位
段号 8 位
段内地址 = 24 - 8 = 16 位
```

每段最大长度由段内地址位数决定：

```text
2^16 B
```

答案：

```text
2^16 B
```

【易错】不要选 `2^24B`。24 位是整个逻辑地址长度，不是段内地址长度。

### 9.6 【必会】段的共享

段式便于共享，因为段是逻辑单位。

例如两个进程共享同一个代码段：

```text
P1 段表某项 -> 共享段表 -> 共享段物理地址
P2 段表某项 -> 同一个共享段表项 -> 同一段物理地址
```

课件中的共享段表可记录：

- 段名。
- 共享计数。
- 段长。
- 段首址。
- 其他信息。

共享计数表示当前有多少进程正在引用该共享段。

### 9.7 【必会】段的保护

段表可加入访问权限：

```text
R W E
```

分别表示：

- Read，可读。
- Write，可写。
- Execute，可执行。

段式保护直观，因为不同段本来就是不同逻辑对象：

- 代码段可读、可执行，不可写。
- 数据段可读写，不一定可执行。
- 共享库代码段可共享执行。

## 10. 段页式存储管理

### 10.1 【必会】为什么要段页式

课件总结：

```text
段式优于页式：便于共享和保护。
页式优于段式：消除碎片问题。
段页式：结合二者优点。
```

直觉：

```text
先按程序逻辑分段；
再把每个段分页；
内存仍按页框分配。
```

### 10.2 基本原理

内存空间：

```text
同页式，静态等长页框。
```

进程空间：

```text
一个进程包含若干段；
一个段包含若干页。
```

逻辑地址是三元组：

```text
(s, p, d)
```

含义：

| 字段 | 含义 |
|---|---|
| `s` | 段号 |
| `p` | 段内逻辑页号 |
| `d` | 页内地址 |

物理地址：

```text
(f, d)
```

### 10.3 所需表目和寄存器

表：

| 表 | 每项内容 |
|---|---|
| 段表 | 段号、页表首址、页表长度 |
| 页表 | 逻辑页号、页框号 |
| 总页表 | 系统维护 |

寄存器：

- 段表基址寄存器。
- 段表限长寄存器。
- 快表，缓存 `(段号, 逻辑页号) -> 页框号`。

### 10.4 【必会】段页式地址映射

逻辑地址：

```text
(s, p, d)
```

步骤：

1. 查快表，看是否有 `(s,p) -> f`。
2. 若未命中，检查段号：

```text
0 <= s <= l - 1
```

3. 用段号查段表，得到该段页表首址 `b'` 和页表长度 `l'`。
4. 检查段内页号：

```text
0 <= p <= l' - 1
```

5. 用页表首址 `b'` 和页号 `p` 查页表，得到页框号 `f`。
6. 物理地址为：

```text
(f, d)
```

### 10.5 【题型】段页式访存次数

习题课模拟选择：

```text
在段页式分配中，CPU 每次从内存中取一次数据需要几次访问内存？
```

若不考虑快表：

1. 访问段表，找到页表。
2. 访问页表，找到页框。
3. 访问真正数据。

所以答案：

```text
3 次。
```

【易错】如果题目没有提 TLB/快表，按基本段页式三次访存答。

## 11. 页式、段式、段页式对照

### 11.1 【必会】一维地址与二维地址

| 管理方式 | 逻辑地址 | 地址维度 |
|---|---|---|
| 界地址 | `a` | 一维 |
| 页式 | `(p,d)`，本质可由一个线性地址拆出 | 一维 |
| 段式 | `(s,d)` | 二维 |
| 段页式 | `(s,p,d)` | 二维 |

课件和习题课都强调：

```text
界地址、页式是一维地址；
段式、段页式是二维地址。
```

### 11.2 【必会】碎片、共享、保护

| 方式 | 存储分配 | 碎片 | 共享保护 |
|---|---|---|---|
| 界地址 | 动态异长连续区 | 外部碎片 | 不方便 |
| 页式 | 静态等长页框 | 内部碎片，无外部碎片 | 不如段式直观 |
| 段式 | 动态异长段 | 外部碎片，无内部碎片 | 方便 |
| 段页式 | 段内分页 | 内部碎片，无外部碎片 | 方便 |

【易错】“页式无碎片”这句话不严谨。页式无外部碎片，但可能有内部碎片。

### 11.3 习题课虚拟存储对照表的前置理解

习题课给出虚拟页式、虚拟段式、虚拟段页式对比：

| 方式 | 地址空间 | 存储分配 | 存储碎片 | 存储共享 | 存储保护 | 动态扩充 | 动态连接 |
|---|---|---|---|---|---|---|---|
| 虚拟页式 | 一维 | 简单 | 无外部碎片 | 不便 | 不便 | 不可 | 不可 |
| 虚拟段式 | 二维 | 复杂 | 有外部碎片 | 方便 | 方便 | 可以 | 可以 |
| 虚拟段页式 | 二维 | 简单 | 无外部碎片 | 方便 | 方便 | 可以 | 可以 |

这张表本质来自第 6 章页/段/段页的差异，第 7 章会在“虚拟”语境下继续展开。

## 12. 真题与模拟题中的第 6 章考法

### 12.1 【题型】分页式存储管理主要特点

真题选择：

```text
分页式存储管理的主要特点是？
A 要求处理缺页中断
B 要求扩充主存容量
C 不要求作业装入到主存的连续区域
D 不要求作业全部同时装入主存
```

第 6 章基本分页的核心特点是：

```text
不要求作业装入主存的连续区域。
```

答案：

```text
C
```

解析：

- 缺页中断、作业不必全部装入，是虚拟页式/请求分页的特征，第 7 章展开。
- 基本分页解决的是物理内存不连续分配问题。

### 12.2 【题型】界地址管理代价最小

真题选择：

```text
实现多道程序并发的多种内存管理方式中，代价最小的是？
```

答案：

```text
界地址管理。
```

理由：

```text
只需基址/限长寄存器完成越界检查和地址相加。
```

### 12.3 【题型】段地址位数

见 9.5。

核心模板：

```text
段内地址位数 = 总逻辑地址位数 - 段号位数
每段最大长度 = 2^(段内地址位数)
```

### 12.4 【题型】段页式访存次数

见 10.5。

核心模板：

```text
无快表：段表一次 + 页表一次 + 数据一次 = 3 次。
```

### 12.5 【题型】页表首址与页表长度

习题课模拟题：

```text
页面大小 2KB；
进程 P 长度 9KB；
每个页表项 4B；
页表空间由系统动态分配；
空闲页面表 + 最先适应算法；
问 PCB 中页表首址和页表长度。
```

解题步骤：

1. 算逻辑页数：

```text
逻辑页数 = ceil(进程长度 / 页面大小)
```

例：

```text
9KB / 2KB = 4.5
ceil = 5 页
```

所以：

```text
页表长度 = 5
```

2. 算页表占用空间：

```text
页表项数 * 每项大小 = 5 * 4B = 20B
```

3. 页表本身也要放入内存。若按页框分配，20B 需要 1 个页框。

4. 根据空闲页框表和分配算法选择一个空闲页框作为页表首址。

习题课示例中，最先适应分配给页表的页框号为 55：

```text
页表首址 = 55 号页框
页表长度 = 5
```

【易错】页表长度通常指页表项个数，也就是逻辑页数，不是页表占多少字节。

### 12.6 【题型】逻辑地址映射物理地址

基本分页题模板：

```text
给定逻辑地址 A、页面大小 page_size。
先算 p = A / page_size，d = A % page_size。
查页表得到页框号 f。
物理地址 = f * page_size + d。
```

若是十六进制且页面大小为 `2^i`，直接切低 `i` 位。

例：

```text
页面大小 = 4KB = 1000H
逻辑地址 = 8AC2H
```

则：

```text
p = 8H
d = AC2H
```

若页表显示逻辑页 8 在页框 20H：

```text
物理地址 = 20H * 1000H + AC2H = 20AC2H
```

## 13. 易混概念总表

| 易错点 | 正确理解 |
|---|---|
| 逻辑地址就是物理地址 | 逻辑地址要经地址映射变成物理地址 |
| 地址映射由操作系统纯软件完成 | 运行时主要靠硬件，OS 建表和设置寄存器 |
| 页式没有碎片 | 页式无外部碎片，但可能有内部碎片 |
| 段式没有碎片 | 段式无内部碎片，但可能有外部碎片 |
| 分页一定有缺页中断 | 基本分页不一定，请求分页/虚拟存储才涉及缺页 |
| 分页要求作业连续装入 | 分页不要求物理连续装入 |
| 页表长度是页表字节数 | 常指页表项个数，也就是逻辑页数 |
| 页面大小决定页号位数 | 页面大小决定页内地址位数 |
| 段地址 24 位就表示每段最大 2^24B | 每段大小由段内地址位数决定 |
| 段式只查一次越界 | 段号越界和段内地址越界都要检查 |
| 段页式地址是一维地址 | 段页式逻辑地址是 `(s,p,d)`，属于二维逻辑地址 |
| 段页式访存 2 次 | 无快表时段表、页表、数据共 3 次 |
| TLB 命中还要查页表 | TLB 命中直接得到页框号，不必查页表 |
| Best Fit 找第一个够用 | Best Fit 找所有够用中最小的 |
| Worst Fit 防止所有碎片 | Worst Fit 可减少小碎片倾向，但会分割大空闲区 |
| 紧凑解决内部碎片 | 紧凑主要解决外部碎片 |

## 14. 课内题型归纳与模板

### 14.1 【题型】地址拆分

模板：

```text
页面/页框大小 = 2^i
低 i 位 = 页内地址 d
剩余高位 = 页号 p
```

段式：

```text
总地址位数 - 段号位数 = 段内地址位数
```

### 14.2 【题型】界地址映射

模板：

```text
给定 b, l, a
先判断 0 <= a <= l - 1
合法则物理地址 = b + a
否则越界
```

### 14.3 【题型】页式映射

模板：

```text
p = floor(A / page_size)
d = A % page_size
检查 p 是否越界
查页表 p -> f
物理地址 = f * page_size + d
```

### 14.4 【题型】段式映射

模板：

```text
逻辑地址 = (s,d)
检查 s 是否小于段表长度
查段表得到 b', l'
检查 d 是否小于 l'
物理地址 = b' + d
```

### 14.5 【题型】段页式映射

模板：

```text
逻辑地址 = (s,p,d)
检查 s
查段表得到该段页表首址和页表长度
检查 p
查页表得到 f
物理地址 = (f,d)
```

### 14.6 【题型】空闲区分配

模板：

```text
FF：按地址顺序找第一个够用。
NF：从上次位置后找第一个够用。
BF：找够用中最小。
WF：找够用中最大。
```

分配后：

```text
首址 += 申请大小
长度 -= 申请大小
长度为 0 则删除表项
```

回收后：

```text
若与前后空闲区相邻，要合并。
```
