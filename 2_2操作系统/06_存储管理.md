- 掌握存储管理的功能：分配、共享、保护、扩充、地址映射。
- 理解静态/动态分区，等长/异长分区。
- 掌握界地址、页式、段式、段页式管理的区别。

## 从 0 理解

程序使用的是逻辑地址，不应该也不能直接知道自己被放在物理内存哪里。地址映射机构负责把逻辑地址变成物理地址，并在越界或越权时触发中断。

内存分配的本质是两件事：

1. 把空闲内存记录清楚。
2. 当进程需要内存时，选择合适空闲区分给它，并在释放时回收。

分页和分段是两种不同的思路：分页从物理管理出发，把空间切成固定大小；分段从程序逻辑出发，把程序按代码段、数据段、栈段等逻辑单位组织。

## 核心机制/算法/公式

### 存储管理功能

| 功能 | 说明 |
|---|---|
| 分配和去配 | 进程创建、撤销、交换、长度变化时分配/回收 |
| 共享 | 多进程共享代码或数据 |
| 保护 | 防止地址越界和操作越权 |
| 扩充 | 内外存结合形成虚拟存储 |
| 地址映射 | 逻辑地址转换为物理地址 |

### 动态异长分区算法

| 算法 | 策略 | 特点 |
|---|---|---|
| 首次适应 FF | 从低地址开始找第一个够大的空闲区 | 简单快，低地址碎片多 |
| 最佳适应 BF | 找能满足请求的最小空闲区 | 易产生很多小碎片 |
| 最坏适应 WF | 找最大的空闲区 | 剩余空闲区较大，但大块消耗快 |

### 页式管理

逻辑地址分成页号和页内偏移：

```text
逻辑地址 = 页号 p + 页内偏移 d
物理地址 = 页框号 f * 页面大小 + d
```

特点：

- 页面和页框大小相同。
- 进程不必装入连续内存。
- 无外部碎片，但可能有页内碎片。
- 用户看到一维地址。

### 段式管理

逻辑地址分成段号和段内偏移：

```text
逻辑地址 = (段号 s, 段内偏移 d)
若 d < 段长，则物理地址 = 段基址 + d
否则越界中断
```

特点：

- 段有逻辑含义，便于共享和保护。
- 段长可变，会产生外部碎片。
- 用户看到二维地址。

### 段页式管理

先分段，再把段分页。优点是保留段的共享保护，又用分页消除外部碎片；缺点是地址转换复杂，系统开销较大。

## 易错点与区分表

| 对比 | 页式 | 段式 | 段页式 |
|---|---|---|---|
| 用户地址 | 一维 | 二维 | 二维 |
| 分配单位 | 固定大小页 | 可变长段 | 段内固定页 |
| 碎片 | 页内碎片 | 外部碎片 | 页内碎片 |
| 共享保护 | 不如段式自然 | 方便 | 方便 |
| 实现复杂度 | 较简单 | 中等 | 较复杂 |

## 题型与解析

### 真题题型：分页式存储管理主要特点

分页式存储管理的主要特点是：不要求作业装入主存的连续区域。

注意：是否要求缺页中断、是否要求作业全部装入内存，要看是不是“虚拟页式/请求分页”。普通分页管理本身强调离散分配，不等于一定有缺页。

### 真题题型：多道并发内存管理代价最小

界地址管理实现最简单，代价最小，但要求连续分配，灵活性和利用率不如分页/分段。
