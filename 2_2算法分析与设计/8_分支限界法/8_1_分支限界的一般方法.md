# 8.1 分支限界法的一般方法 — 专项复习资料

> 本节是分支限界法的概念总纲，建立"适用问题特点 → 基本思想 → 抽象算法8.1 → 检索方式 → FIFO实例"的完整链条，后续具体问题（8.3 / 8.5 / 8.6）的求解框架都建立在本节之上。对应**作业题1**（回溯法与分支限界法状态空间生成方式及基本思想的对比）。

---

## 一、方法适用的问题特点

### 1.1 与回溯法的相同点

分支限界法与回溯法一样，适用于求解**组合数较大的问题**，特别是**组合优化问题（求最优解）**。两者在解空间建模层面完全共享下列概念：

| 共享概念 | 说明 |
|---|---|
| 解向量表达 | n-元组 / k-元组（固定长 / 不定长） |
| 显式约束条件 | 每个 $x_i$ 的取值范围 |
| 隐式约束条件 | 各 $x_i$ 之间的关联约束（与问题实例相关） |
| 解空间 / 解空间树 | 与回溯法完全相同的定义 |

### 1.2 唯一的本质区别

> **两者主要区别在于 E-结点（即扩展结点）处理方式不同。**

这是贯穿整章的核心论断：解空间树的建模方式不变，仅仅是"在树上怎么走"不同（具体差异见第六节对比表）。

> 清华大学出版社《算法设计与分析》（屈婉玲等编著）认为："**分支限界是回溯算法的变种**"——这从另一个角度印证了两者共享设计框架、仅 E-结点处理策略不同的关系。

## 二、分支限界法的基本思想（★核心）

### 2.1 建模与验证

1. 针对问题定义解空间树结构：元组、显式约束条件、隐式约束条件（与回溯法相同）。
2. 检验问题是否满足**多米诺性质**（与回溯法的要求一致，是剪枝合法性的前提）。

### 2.2 搜索过程（假设目标是寻找"一个"答案结点）

| 步骤 | 操作 |
|---|---|
| ① | 若根结点 T 是答案结点，输出 T，结束；否则令 T 为当前扩展结点 E |
| ② | 生成 E 的**所有**儿子结点，对每个儿子 X 判断 |
| ②-a | 若 X 是答案结点，输出从 X 到根的路径，结束 |
| ②-b | 若 X 满足约束函数 $B$，将 X 加入**活结点表**；否则舍弃 X |
| ③ | 从活结点表中选出下一个结点作为新的 E-结点，重复步骤②；若活结点表为空，算法以失败结束 |

> **约束函数 B 的剪枝作用：** 避免生成那些不包含可行解的子树（可行解剪枝）。

> **关键标注：** 以上流程在课件中标注为"**找一个可行解**"——这是本节（8.1）的目标。若问题是求**最优解**（组合优化问题），需要 8.4 节引入成本估计函数 $\hat{c}$ 和上界 $U$ 做进一步改进。

## 三、算法 8.1：分支限界法的抽象化描述（★重点）

```
procedure BB(T)
  if T 是答案结点 then 输出 T; return endif
  E ← T
  将活结点表初始化为空
  loop
    for E 的每个儿子 X do
      if X 是答案结点 then 输出从 X 到 T 的那条路径; return; endif
      if B(X) then call ADD(X); PARENT(X) ← E endif
    repeat
    if 表中不再有活结点 then print("no answer node"); return; endif
    call LEAST(E)
  repeat
end BANDB
```

| 子过程 | 功能 |
|---|---|
| **ADD(X)** | 将 X 添加到活结点表中 |
| **LEAST(E)** | 从活结点表中选中一个结点赋值给 E，并将该结点从表中删除 |

> 算法本身**不规定** `LEAST` 具体怎样选——这正是第四节"不同检索方式"的切入点：`LEAST` 的实现方式决定了算法是 FIFO-检索还是 LC-检索。

### 3.1 思考题：如果想获得所有可行解，算法怎样设计？

**标准答案：** 算法 8.1 一旦在 `for` 循环中发现某个儿子 X 是答案结点，就立即 `输出; return`，终止整个算法——这只能找到**第一个**遇到的答案结点。若要找出**所有**可行解，需要做两处改动：

1. **去掉提前终止：** 将 `if X 是答案结点 then 输出...; return; endif` 改为 `if X 是答案结点 then 输出...; 计数器加一 endif`（不再 `return`），使循环可以继续处理 E 的其余儿子。
2. **改变终止条件：** 算法不再以"找到一个答案结点"为终止信号，而是改为与原算法"无解"分支一致——**直到活结点表为空**才终止整个搜索。

> 直观理解：把算法从"找到即停"改造成"穷尽活结点表"，这与回溯法中"求全部解"需要去掉提前 `return` 的改造思路是一致的。

## 四、分支限界法的不同检索方式

### 4.1 两类活结点表的实现策略

| 策略 | 选取规则 | 活结点表实现 |
|---|---|---|
| **顺序队列式** | 活结点依照**进入活结点表的顺序**被选出 | 队列（最常见为**先进先出 FIFO**） |
| **优先队列式** | 活结点依照**成本估计函数 $\hat{c}$** 排序，$\hat{c}$ 最小/最大者优先被选出 | 极小堆 / 极大堆 |

### 4.2 LEAST 规则与算法命名

| 若 LEAST 遵循 | 算法即为 |
|---|---|
| FIFO-顺序队列式 | **FIFO-检索**（FIFO-分支限界法） |
| 极小堆优先队列式 | **LC-检索**（LC-分支限界法，详见 8.2 节） |

> **要点：** 算法 8.1 的总体框架（BANDB 主循环）完全不变，FIFO-检索与 LC-检索仅仅是 `LEAST(E)` 子过程的不同实现——这进一步印证了"分支限界法是一类策略框架，检索方式是可插拔的组件"。

## 五、理解 FIFO-检索：4-皇后问题实例（★重点，含完整推导）

### 5.1 问题设置

与第七章 7.3 节相同的 4-皇后问题：$X_i \in \{1,2,3,4\}$ 表示第 $i$ 行皇后所在列号。

| 约束类型 | 内容 |
|---|---|
| 显式约束 | $X_1, X_2, X_3, X_4$ 互不相同（列号不重复） |
| 隐式约束（约束函数 B） | 任意两皇后不在同一斜线：$\lvert X_i - X_k \rvert \neq \lvert i-k \rvert$ |

唯一区别在于：回溯法按 DFS 深入；分支限界法（本例用 FIFO-检索）按层扩展、用队列维护活结点表。

### 5.2 FIFO-检索的逐层展开（完整推导）

**第 0 层（根结点 1）：** 生成 4 个儿子（$X_1=1,2,3,4$），均无冲突，结点 2、3、4、5 全部进入活结点表。

**第 1 层（依次处理结点 2、3、4、5）：**

| E-结点 | $X_1$ | 候选 $X_2$ | 杀死（冲突） | 存活 |
|---|---|---|---|---|
| 2 | 1 | 2,3,4 | $X_2=2$（结点6） | 7（$X_2=3$），8（$X_2=4$） |
| 3 | 2 | 1,3,4 | $X_2=1$（9），$X_2=3$（10） | 11（$X_2=4$） |
| 4 | 3 | 1,2,4 | $X_2=2$（13），$X_2=4$（14） | 12（$X_2=1$） |
| 5 | 4 | 1,2,3 | $X_2=3$（17） | 15（$X_2=1$），16（$X_2=2$） |

本层共生成 **12 个**结点（编号 6–17），杀死 6 个，存活 6 个：**7、8、11、12、15、16**。

**第 2 层（依次处理结点 7、8、11、12、15、16）：**

| E-结点 | $(X_1,X_2)$ | 候选 $X_3$ | 杀死（冲突） | 存活 |
|---|---|---|---|---|
| 7 | (1,3) | 2,4 | 二者均与 $X_2$ 冲突（18，19） | 无 |
| 8 | (1,4) | 2,3 | $X_3=3$（21） | 20（$X_3=2$） |
| 11 | (2,4) | 1,3 | $X_3=3$（23） | 22（$X_3=1$） |
| 12 | (3,1) | 2,4 | $X_3=2$（24） | 25（$X_3=4$） |
| 15 | (4,1) | 2,3 | $X_3=2$（26） | 27（$X_3=3$） |
| 16 | (4,2) | 1,3 | 二者均与 $X_2$ 冲突（28，29） | 无 |

本层共生成 **12 个**结点（编号 18–29），杀死 8 个，存活 4 个：**20（1,4,2）、22（2,4,1）、25（3,1,4）、27（4,1,3）**。

**第 3 层（按 FIFO 顺序依次处理结点 20、22）：**

- 处理结点 20（局面 1,4,2）：唯一剩余列 $X_4=3$ → 结点 30。检验：与 $X_3=2$ 冲突（$\lvert 3-2\rvert=1$）→ 舍弃。
- 处理结点 22（局面 2,4,1）：唯一剩余列 $X_4=3$ → 结点 31。检验：与 $X_3=1$（$|3-1|=2$）、$X_2=4$（$|3-4|=1\neq2$）、$X_1=2$（$|3-2|=1\neq3$）**均无冲突** → **结点 31 是答案结点！** 输出解 $(2,4,1,3)$，**算法立即终止**。

> 活结点表中此时还残留结点 25（局面 3,1,4）与 27（局面 4,1,3）——沿着各自分支本可以推出 4-皇后问题的**第二个解** $(3,1,4,2)$，但由于算法在找到第一个答案结点后立即返回，这两个结点**永远不会被取出处理**。

### 5.3 结点数汇总

| 层级 | 处理的 E-结点 | 本层生成数 | 被杀死 | 存活 / 进入下一轮 |
|---|---|---|---|---|
| 0 | 1（根） | 4 | 0 | 4 |
| 1 | 2,3,4,5 | 12 | 6 | 6 |
| 2 | 7,8,11,12,15,16 | 12 | 8 | 4 |
| 3 | 20,22（找到答案即终止） | 2 | 1 | — |
| **合计** | | | | **31 个结点** |

> **FIFO-分支限界法一共处理了 31 个结点，回溯法一共处理了 16 个结点。**

### 5.4 为什么 FIFO-检索处理的结点数明显更多？

- **回溯法（DFS）：** 每次只**延迟生成**一个候选值去尝试，一旦发现冲突立即回溯，同层中尚未尝试的候选值要等到真正需要时才会被生成——不会提前批量生成。
- **FIFO-分支限界法：** 算法 8.1 要求"生成 E 的**所有**儿子结点"，哪怕某个儿子注定会被约束函数 B 杀死，它也必须先被**生成并检验**才能被丢弃——这个生成动作本身就计入处理的结点总数。

> 这正是 8.2 节引入"**有智力的**"成本估计函数 $\hat{c}$、设计 LC-检索的动机：FIFO-检索"对下一个 E-结点的选择是死板的、盲目的"，无法对更有希望的活结点给予优先权。

## 六、回溯法与分支限界法对比总结（★高频考点）

| 维度 | 回溯法 | 分支限界法 |
|---|---|---|
| 适用问题 | 组合数较大的问题（含组合优化问题） | 与回溯法相同 |
| 搜索策略 | **深度优先**（DFS） | 广度优先 / 最小代价优先（依检索方式而定） |
| E-结点处理 | 生成**一个**儿子后立即转移为新 E-结点 | 生成**全部**儿子后才变为死结点 |
| 活结点表 | **不需要** | **需要**（队列或堆） |
| 检索方式细分 | 仅 DFS 一种 | 顺序队列式（FIFO）/ 优先队列式（LC） |
| 剪枝依据 | 约束函数 $B$ | $B$（可行解剪枝）+ $\hat{c}$ 与上界 $U$（最优解剪枝，见 8.4 节） |
| 求最优解效率 | 一般 | 更高（可借助 $\hat{c}$ 优先选择更有希望的结点） |
| 存储空间 | 较小 | 较大（需额外维护活结点表） |
| 终止条件（找一个解） | 沿当前路径穷尽或找到答案 | 找到答案结点 / 活结点表为空 |

## 七、作业题1标准答案：状态空间生成方式与基本思想对比

**题目：回溯法和分支限界法的状态空间生成方式有何不同？简述两种方法的基本思想。**

**状态空间生成方式的不同：**

两种方法的本质区别在于 E-结点（扩展结点）的处理方式不同。回溯法中，E-结点生成一个儿子结点后，该儿子立即成为新的 E-结点（纵深推进）；其整棵子树处理完毕、回溯之后，原 E-结点才重新生成下一个儿子——这本质上是**深度优先搜索**，不需要维护活结点表。分支限界法中，E-结点一次性生成**所有**儿子结点，逐一判断可行性后加入**活结点表**，自身随即变为死结点；再依据某种选取策略（FIFO 或 $\hat{c}$ 最小）从活结点表中选出下一个 E-结点。

**两种方法的基本思想：**

回溯法针对问题定义解空间树后，验证问题满足多米诺性质，随后从根结点出发做深度优先搜索：扩展结点满足约束函数 $B$ 则纵深前进，否则回溯到最近的活结点尝试其他分支；约束函数 $B$ 的作用是剪去不含可行解的子树。

分支限界法同样先定义解空间树并验证多米诺性质，区别在于：每次将当前 E-结点满足约束函数 $B$ 的全部儿子一次性加入活结点表，再依照活结点表的选取规则（顺序队列式或优先队列式）决定下一个 E-结点；如果问题要求**最优解**（组合优化问题），还会引入成本估计函数 $\hat{c}$ 与最小成本上界 $U$（详见 8.4 节），对活结点表做进一步剪枝，从而比单纯枚举更高效地定位最优解。

## 八、易错点汇总

| 易错点 | 正确理解 |
|---|---|
| 误以为分支限界法与回溯法的解空间建模不同 | 二者解向量、显式/隐式约束、解空间树的**定义完全相同**，唯一区别是 E-结点处理方式 |
| 混淆"找一个可行解"与"求最优解"的算法版本 | 算法 8.1（BANDB）只负责找**一个**答案结点；求最优解需要 8.4 节引入 $\hat{c}$ 和 $U$ 的改进算法（FIFOBB / LCBB） |
| 认为 FIFO-检索和 LC-检索是两套完全不同的算法 | 二者共用算法 8.1 的主体框架，仅 `LEAST(E)` 子过程（活结点表选取规则）不同 |
| 误以为算法找到答案结点后会继续找出全部解 | 算法 8.1 默认找到**第一个**答案结点就 `return` 终止；要找全部解需去掉提前 `return`，并改终止条件为"活结点表为空" |
| 低估 FIFO-检索的结点数开销 | FIFO 必须**先生成全部儿子才能判断哪些被剪枝**，即使被杀死的结点也计入"处理过的结点"，因此通常比回溯法处理更多结点（4-皇后例：31 vs 16） |
| 误以为分支限界法一定比回溯法慢 | 这只是 FIFO-检索（顺序队列式）的特点；LC-检索借助 $\hat{c}$ 优先扩展更有希望的结点，求最优解时通常**更快**（详见 8.2 节） |
| 混淆"约束函数 B"与"成本估计函数 $\hat{c}$"的剪枝时机 | $B$ 用于**可行解剪枝**（8.1 节即出现）；$\hat{c}$ 和 $U$ 用于**最优解剪枝**，是 8.4 节才引入的概念 |

## 九、考点速查

| 考点 | 核心内容 |
|---|---|
| 与回溯法的核心区别 | E-结点（扩展结点）处理方式不同；共享解空间树等全部建模概念 |
| 分支限界法基本思想 | 建模 → 验证多米诺性质 → 生成 E 的**全部**儿子 → B 检验后入活结点表 → LEAST 选下一个 E-结点 |
| 算法 8.1（BANDB）核心子过程 | `ADD(X)`：加入活结点表；`LEAST(E)`：选出并删除一个活结点 |
| 求全部可行解的改造 | 去掉提前 `return`，终止条件改为"活结点表为空" |
| 顺序队列式 vs 优先队列式 | 队列（FIFO）vs 极小/极大堆（按 $\hat{c}$ 排序） |
| LEAST 规则对应算法名 | FIFO-顺序队列式 → FIFO-检索；极小堆优先队列式 → LC-检索 |
| FIFO-检索的局限 | 选取下一个 E-结点"死板、盲目"，不区分活结点的潜力，是 LC-检索（8.2节）的引入动机 |
| 4-皇后 FIFO 实例结点数 | FIFO-分支限界法处理 **31** 个结点；回溯法处理 **16** 个结点 |
| 算法终止条件（找一个解） | 找到答案结点 `return`，或活结点表为空 `print("no answer node")` |
| 作业题1标准答案 | 状态空间生成方式区别（E-结点处理）+ 两种方法基本思想（B剪枝 vs B+$\hat{c}$+U剪枝） |
