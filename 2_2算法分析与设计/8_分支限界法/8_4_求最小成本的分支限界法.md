# 8.4 求最小成本的分支限界法 — 复习资料

> 本节解决的核心问题：8.1 节的算法 8.1（BB）只能找**一个**可行解，并不保证这个解是**最优**的；本节在 8.2 节 $\hat{c}$ 框架的基础上，先证明"简单套用 LC-检索"仍然找不到最小成本答案结点，进而给出正确的改进算法（算法8.2），再引入成本上界 $U$ 与上界函数 $u(X)$ 进一步加速剪枝（算法8.3 界函数 UB），最终给出通用的 FIFOBB（算法8.4）与 LCBB（算法8.5）——这两个算法正是 **8.5（作业调度）、8.6（货郎担问题）** 两节具体执行实例所直接套用的框架。

---

## 一、铺垫：可行解与最优解

| 概念 | 定义 |
|---|---|
| **可行解** | 即答案结点（满足约束条件的解） |
| **最优解** | 使**目标函数**取极值的答案结点（在所有可行解中代价最小/最大的那个） |

8.1 节的算法只关心"找到一个答案结点就算完成"；本节关心的是"找到的这个答案结点，必须是**所有**答案结点里代价最小的那个"——这是质的提升，也是组合**优化**问题（而非单纯组合搜索问题）的核心要求。

---

## 二、寻找最小成本：原始算法 BB 为什么不可靠（★核心动机★）

**问题：** 基于 LC-检索的算法 8.1（BB）是否一定能找到具有最小成本的答案结点？

**反例树（完整推导）：** 设有如下解空间树，每个结点标注 $(c,\hat{c})$：

```
                          R (c=10, ĉ=0)
                  ┌─────────────┴─────────────┐
              L (c=20, ĉ=2)              Ri (c=10, ĉ=4)
              ┌───────┴───────┐          ┌───────┴───────┐
       LL(c=20,ĉ=20)   LR(c=∞,ĉ=∞)  RL(c=∞,ĉ=∞)   RR(c=10,ĉ=10)
        ★答案结点                                      ★答案结点(真正最优)
```

逐结点验证 $c$ 的递归定义（与 8.2 节方法一致）：$c(L)=\min(20,\infty)=20$；$c(Ri)=\min(\infty,10)=10$；$c(R)=\min(20,10)=10$。**全树真正的最小成本答案结点是 RR，cost=10。**

### 2.1 用原始算法 8.1（BB）逐步追踪

回忆算法 8.1 的逻辑：**在 `for E 的每个儿子 X` 循环内部**，一旦发现某个新生成的儿子 X 是答案结点，**立即** `输出; return`。

| 步骤 | 当前 E | 生成的儿子 | 即时检查"是否答案" | 结果 |
|---|---|---|---|---|
| 1 | R | L（ĉ=2）、Ri（ĉ=4） | 均非叶结点，不是答案 | 两者通过 B 检验后加入活结点表 |
| 2 | LEAST 选 ĉ 最小者 → **L** | LL（ĉ=20）、LR（ĉ=∞） | **LL 是答案结点！** | **立即 `return`，输出 LL，cost = 20** |

> **错误发生了：** 算法在 E=L 的儿子循环里一碰到 LL 是答案结点就直接终止，根本没有去比较活结点表里还留着的 Ri（ĉ=4，比 LL 所在分支更有希望）。最终输出的 cost=20，但全树真正的最小成本是 **10**（在 RR 处）。

**补救思路（追加要求）：** 要求对任意结点 X、Y，当 $c(X)<c(Y)$ 时必有 $\hat{c}(X)<\hat{c}(Y)$（即 $\hat{c}$ 与 $c$ 的大小顺序完全一致）。但**这个条件很难实现**——一般只能找到满足"$\hat{c}(X)\leq c(X)$"等较弱条件的 $\hat{c}$（见 8.2 节三点要求），无法保证严格的顺序对应。

## 三、改进算法：先选 E、再判断"是否答案"

**改进思路：** 不在生成儿子的瞬间就检查"是否答案"，而是**等到该结点真正凭 $\hat{c}$ 最小被 `LEAST` 选为新的 E-结点之后**，才检查它是否是答案结点；如果是，则结束操作。**该思路适用于 n-元组表达。**

### 3.1 用改进算法重新追踪同一棵反例树

| 步骤 | 当前 E | 是否答案？ | 生成的儿子（仅加入活结点表，不立即判断） | 活结点表（按 ĉ） |
|---|---|---|---|---|
| 1 | R | 否 | L(2)、Ri(4) | {L:2, Ri:4} |
| 2 | **L**（LEAST 选中 ĉ=2 最小） | 否（L 非叶） | LL(20)、LR(∞) | {Ri:4, LL:20, LR:∞} |
| 3 | **Ri**（LEAST 选中 ĉ=4 最小） | 否（Ri 非叶） | RL(∞)、RR(10) | {LL:20, LR:∞, RL:∞, RR:10} |
| 4 | **RR**（LEAST 选中 ĉ=10 最小） | **是！** | — | 输出 RR，**cost = 10**（正确！） |

> E-结点访问顺序：$R\to L\to Ri\to RR$，恰好是 8.4 节配图中标出"E"的四个结点。把"判断答案"挪到"选出 E 之后"，结果就从错误的 20 修正为正确的 **10**。

### 3.2 思考题：如何证明改进后找到的答案结点一定是最小成本？

**证明：** 设 LEAST 选出的当前结点为 E，活结点表中剩余的任意结点记为 L。由 `LEAST` 的选取规则，必有 $\hat{c}(E)\leq \hat{c}(L)$（E 是表中 $\hat{c}$ 最小者）。又因为 E 此刻被判定为答案结点，按照 8.2 节 $\hat{c}$ 的第三条要求，叶答案结点处 $\hat{c}(E)=c(E)=\text{cost}(E)$。于是：

$$\text{cost}(E)=c(E)=\hat{c}(E)\leq \hat{c}(L)\leq c(L)\qquad(\text{对表中任意 } L)$$

也就是说 E 的真实成本不大于活结点表中**任何**结点子树里能找到的最优答案成本，因此 **E 就是全局最小成本的答案结点**。$\blacksquare$

## 四、算法 8.2：基于 $\hat{c}$ 求最小成本的 LC-检索算法

```
procedure LC(T, ĉ)              //找出最小成本答案结点
  E ← T,  将活结点表初始化为空
  loop
    if E 是答案结点 then 输出从 E 到 T 的那条路径; return; endif
    for E 的每个儿子 X do
      if B(X) then call ADD(X); PARENT(X) ← E; endif
    repeat
    if 表中不再有活结点 then print("no answer node"); return; endif
    call LEAST(E)            // 选择 ĉ 最小的
  repeat
end LC
```

> 与算法 8.1（BB）相比，**唯一的结构性差异**就是把"是否答案"的判断，从 `for` 循环内部移到了循环**外部、最顶端**（即先 `LEAST` 选出 E，下一轮循环开头才检查 E）。

### 4.1 算法 8.2 对 n-元组和 k-元组的处理区别（★重点考查★）

| 表达方式 | 答案结点的位置 | 是否满足 $\text{cost}=c=\hat{c}$ | 算法 8.2 是否一定终止于最小成本答案结点 |
|---|---|---|---|
| **n-元组**（固定长） | 答案结点必为**叶结点** | 满足**等式** $\hat{c}(\cdot)=c(\cdot)=\text{cost}(\cdot)$（8.2节第三条要求保证） | **一定**（第三节证明成立） |
| **k-元组**（不定长） | 答案结点**可能是非叶结点** | 只能满足**不等式** $\hat{c}(\cdot)\leq c(\cdot)\leq \text{cost}(\cdot)$（非叶答案结点的真实代价可能不是其子树最优） | **不一定** |

> 下一步需要寻找更通用、剪枝能力更强的算法——这正是引入成本上界 $U$ 的动机。

## 五、加速：成本上界 $U$ 与上界函数 $u(X)$

### 5.1 基本思想

> 在寻找最优解过程中，不止判断 $B(X)$，若发现结点 X **不能导致最小成本答案结点**，也不必再搜索子树 X，子树 X 被剪枝。

**设计方案：** 设置一个最小成本**上界** $U$（问题的最小成本绝不会超过这个上界）。如果 $\hat{c}(X)>(\text{或}\geq)U$，算法无需检索以 X 为根的子树。

> **两种剪枝：** ① **可行解剪枝**（依据约束函数 $B$，避免生成不含可行解的子树——8.1节已有）；② **最优解剪枝**（依据 $\hat{c}(X)$ 和上界 $U$，本节新增——避免生成"肯定不会更优"的子树）。

### 5.2 上界 $U$ 的含义、取值与作用

| 方面 | 说明 |
|---|---|
| **含义** | $U$ 是**当前算法生成过的所有状态结点**对最小成本上界估计的**最小值** |
| **取值** | 初值可取 $\infty$，或通过启发式方法（如贪心法）预先得到一个合理初值；初值须 $\geq$ 最小成本答案结点的真实成本；随着结点的访问**不断改小** |
| **作用** | ① 界定当前结点的死活（剪枝判据）；② 通过剪枝加快找到最优解 |

### 5.3 成本上界函数 $u(X)$

$u(X)$ 是对成本 $c(X)$ 的上界估计，简称**成本上界函数**。可仿照 $\hat{c}(X)$ 的方式定义，由两部分构成：

- 根到 $X$ 的**真实**成本
- $X$ 到最小成本答案结点的成本**上界**估计

**对 $u(X)$ 的要求：**

1. 易于计算
2. $c(X)\leq u(X)$（**上界性质**——与 $\hat{c}$ 的下界性质恰好相反）
3. 当答案结点 $X$ 是叶结点时，$c(X)=u(X)$

> 综合 $\hat{c}$ 与 $u$，对任意结点 $X$ 都有这条贯穿全局的不等式链：
> $$\boxed{\hat{c}(X)\leq c(X)\leq u(X)}$$
> $U$ 可以借助 $u(X)$ 加速变小——一旦发现某个状态结点的 $u(X)$ 比当前 $U$ 还小，说明这棵子树里"保证能找到"的最优解一定不会超过 $u(X)$，应立刻把 $U$ 收紧到 $u(X)$。

### 5.4 区分上界值 $U$ 和函数 $u(X)$（易混淆点）

| | $U$ | $u(X)$ |
|---|---|---|
| **本质** | 一个**全局**的、随时间推进而单调收紧的**数值**（算法运行状态的一部分） | 针对**某个特定结点 $X$**（沿从根到 $X$ 的这条**指定路径**）计算出的**函数值** |
| **更新方式** | 检查某结点的 $\hat{c}$ 值来判断死活；也会被某结点的 $u(X)$ 值"拉低" | 不被 $U$ 影响，只依赖 $X$ 自身在树中的位置和已知信息计算 |

### 5.5 区分四个函数：$\text{cost}$、$\hat{c}$、$c$、$u$（完整实例验证）

```
                         结点1：ĉ=3 c=4 u=7 cost=10
                                  │
                         结点2：ĉ=2 c=4 u=7 cost=8
                          ┌───────┴───────┐
                结点3：ĉ=4 c=5 u=6 cost=7   结点4：ĉ=4 c=4 u=4 cost=4 【叶结点】
                          │
                结点5：ĉ=5 c=5 u=5 cost=5 【叶结点】
```

| 函数 | 含义 | 验证 |
|---|---|---|
| $\text{cost}(X)$ | 根到答案结点 $X$ 的**真实成本**（这里每个结点都假设满足约束、可作为候选答案，体现 k-元组场景下非叶结点也可为答案） | 结点4、5为叶，$\text{cost}=u=c=\hat{c}$（完全收口）；结点1、2、3 为非叶候选，$\text{cost}$ 是它们"如果自己就是答案"时的真实代价 |
| $c(X)$ | 子树 $X$ 中**所有答案结点的最小成本值** | $c(3)=\min(\text{cost}(3)=7,\,c(5)=5)=5$；$c(2)=\min(c(3)=5,\,c(4)=4)=4$；$c(1)=c(2)=4$ |
| $\hat{c}(X),u(X)$ | 对 $c(X)$ 的下界 / 上界估计 | 处处满足 $\hat{c}(X)\leq c(X)\leq u(X)$，如结点2：$2\leq4\leq7$ ✓ |

> **关键结论：** 当 $\text{cost}(X)>u(X)$ 时，意味着**以 X 为根的子树中一定有更优解**。例如结点1：$\text{cost}(1)=10>u(1)=7$，说明结点1自身（即便它本身满足约束、可作答案）绝不是它子树里最好的选择——果然 $c(1)=4$ 远小于 10。这正是 $u(X)$ 在算法中的实际用途：哪怕还不知道精确的 $c(X)$，只要发现 $u(X)$ 比某个候选自身的真实代价还小，就能断定"还有更好的藏在下面"，应继续往下搜索而非满足于当前结点。

## 六、基于 $\hat{c}(X)$ 和 $U$ 求最小成本的分支限界法基本思想

> 分支限界法求**极小化**问题时，问题转化为：在解空间树中寻找**最小成本答案结点**。

| 步骤 | 内容 |
|---|---|
| ① | 对于极小化问题，把**目标函数**作为成本函数 $c$ |
| ② | 约束条件作为约束函数 $B$ |
| ③ | 问题转化为寻找解空间树中最小成本答案结点 |
| ④ | 设计成本估计函数 $\hat{c}(X)$，满足 $\hat{c}(X)\leq c(X)$（8.2 节内容） |
| ⑤ | 还可以设计最小成本的**上界** $U$（本节内容，$U$ 是界） |
| ⑥ | 基于 $\hat{c}(X)$ 和 $U$ 进行分支限界搜索 |

> **思考题：极大化问题如何转化？**——见第十节。

## 七、细谈界 $U$ 的改值与剪枝规则

### 7.1 $U$ 的改值

| 情形 | 操作 |
|---|---|
| 发现一个**答案结点** $X$，且 $\text{cost}(X)<U$ | $U\leftarrow \text{cost}(X)$（用**真实**值收紧） |
| 发现一个**状态结点** $X$，且 $u(X)<U$ | $U\leftarrow u(X)$（用**估计**值收紧） |

### 7.2 $U$ 剪枝（界定死活）

| $U$ 当前的来源 | 杀死规则 | 原因 |
|---|---|---|
| 来自一个**真实成本**（已有确认答案结点 `ans`） | $\hat{c}(X)\geq U$ 的活结点都可以被杀死（**含等号**） | 此时 $U$ 记录的是**已确认**的最小成本答案结点；想找的是**更小**的，打平也没意义 |
| 来自一个**成本上界估计**（`ans` 为空，尚无确认答案） | $\hat{c}(X)>U$ 的活结点才可以被杀死（**不含等号**） | 此时只能确定"存在不超过 U 的解"，但还没找到一个**真正**成本 $\leq U$ 的答案结点；打平的 X 仍有机会就是那个真正的答案 |

> **思考题：答案结点 X 是非叶结点时，如果 $\text{cost}(X)$ 和 $u(X)$ 都优于当前 $U$ 值，该怎样选择？**
>
> **答案：选值小的。** 因为 $\text{cost}(X)$ 是 X 自身的真实代价，$u(X)$ 是 X 子树里可能存在更优解的估计上界；两者谁更小，就用谁去收紧 $U$（如果是 $u(X)$ 更小，说明子树里有希望找到比 X 自己更好的答案，此时 `ans` 应该清空为 0，因为我们还没找到那个真正更好的结点；如果 $\text{cost}(X)$ 更小，说明 X 自己就是当前已知最好的，`ans←X`）。

## 八、算法 8.3：界函数 UB

```
procedure UB(X, U, ans)
//ans 是当前最小成本的答案结点，0 表示无结点
//X 是当前活结点，U 是当前上界估计值，ĉ(X) ≤ c(X) ≤ u(X)
//当 X 是答案结点时，cost(X) 是真实成本值，否则 ∞
  if ĉ(X)>U or (ans 有结点 and ĉ(X)=U) then return false endif      ← 界定杀死
  if X 是答案结点 and cost(X)<U then U←cost(X); ans←X endif         ┐
  if u(X)<U then U←u(X); ans←0 endif                                ┘ 更改 U 值
  return true
end UB
```

### 8.1 逐行解读

| 行 | 作用 |
|---|---|
| 第1行 | **界定杀死**：若 $\hat{c}(X)>U$，或（已有确认答案 `ans` **且** $\hat{c}(X)=U$），X 必死，直接 `return false`（呼应第七节"7.2 U 剪枝"的两条规则） |
| 第2行 | 若 X 本身恰好是答案结点，且它的真实代价比当前 $U$ 还好，**用真实值**收紧 $U$，并记录 `ans←X` |
| 第3行 | 若 X 的上界估计 $u(X)$ 比当前 $U$ 还好（哪怕 X 自己不是答案，子树里可能藏着更好的），**用估计值**收紧 $U$，但此时还没找到真正的答案结点，置 `ans←0` |
| 第4行 | X 通过了死活检验，`return true`，允许被加入活结点表 |

> $\hat{c}(X)<U$ 或 ($ans$ **无**结点 且 $\hat{c}(X)=U$) 时，**界定通过**（即第1行 if 条件为假的情形）。

## 九、算法 8.4 / 8.5：求最小成本的 FIFO-/LC-分支限界算法

两个算法都假定状态空间树 $T$ 至少包含一个解结点，**不可行结点的估计值 $\hat{c}(X)=\infty$**。

### 9.1 算法 8.4：FIFOBB

```
procedure FIFOBB(T, ĉ, u, cost)
  E←T; PARENT(E)←0; U←∞，ans←0
  UB(E,U,ans); 将队列初始化为空
  loop
    for E 的每个儿子 X do
      if B(X) and UB(X,U,ans) then call ADDQ(X); PARENT(X)←E endif
    repeat
    loop
      if 队列为空 then print('least cost = ', U); 输出从 ans 到 T 的路径; return endif
      call DELETEQ(E)
      if ĉ(E)<U or (ans 无结点 and ĉ(E)=U) then exit endif        ← 界定通过
    repeat
  repeat
end FIFOBB
```

### 9.2 算法 8.5：LCBB

```
procedure LCBB(T, ĉ, u, cost)
//函数 ADD：加一个结点到 min-堆中；函数 LEAST：从 min-堆中删去堆顶结点
  E←T; PARENT(E)←0; U←∞，ans←0
  UB(E,U,ans); 将活结点表初始化为空
  loop
    for E 的每个儿子 X do
      if B(X) and UB(X,U,ans) then call ADD(X); PARENT(X)←E endif
    repeat
    if 表中不再有活结点 or
       堆顶结点 ĉ 值>U or (ans 有结点 and 堆顶结点 ĉ 值=U)
    then print('least cost = ', U); 输出从 ans 到 T 的那条路径; return endif      ← 界定杀死
    call LEAST(E)
  repeat
end LCBB
```

### 9.3 为什么 FIFOBB 多了一层内层循环，LCBB 却不需要？（★易混淆点，深入理解★）

两者的差异**不是偶然**，根源在于活结点表所用数据结构的性质不同：

| | 数据结构 | 终止判断方式 |
|---|---|---|
| **LCBB（min-堆）** | 堆顶**始终**是当前活结点表中 $\hat{c}$ 最小的结点 | 只需检查**堆顶一个结点**：若它都已经被 $U$ 判死（$\hat{c}>U$，或 `ans` 有结点且 $\hat{c}=U$），那么表里**其余所有**结点的 $\hat{c}$ 必然更大，**全表皆死**，可以直接终止 |
| **FIFOBB（队列）** | 队首只是**最早入队**的结点，**不代表** $\hat{c}$ 最小 | 队首被判死，并不意味着队列后面没有还活着的结点（可能是更早加入、当时 U 还较松时放进来的"过时"结点）；因此必须用**内层 loop**逐个 `DELETEQ` 并重新核验 $\hat{c}(E)<U$（或 `ans` 无结点且相等），直到找到第一个仍然存活的结点才 `exit` 继续主循环，或队列耗尽才彻底终止 |

> 这正是 8.1、8.2 节反复强调的"FIFO-检索死板盲目"在本节算法里的具体体现：队列本身不维护任何优先级，所以**连"判断是否该结束"这件事，都必须额外多做一层过滤**。

## 十、求极大化问题的两种转化方法

很多最优解问题求**目标函数极大值**，可以用两种方式套用本节的极小化框架：

| 方法 | 做法 | 不等式方向 |
|---|---|---|
| **方法1：取负转换** | 取目标函数的**相反数**作为成本函数 $c$，直接复用本节全部极小化算法（最直接、最常用） | 转换后仍是 $\hat{c}(X)\leq c(X)\leq u(X)$ |
| **方法2：镜像修改** | **不取负**，仍把目标函数本身作为成本函数 $c$，但问题改为寻找解空间树中**最大成本**答案结点 | 此时角色互换：$u(X)\leq c(X)\leq \hat{c}(X)$（$\hat{c}$ 变成上界估计，$u$ 变成下界估计） |

> **方法2 下 $U$ 的角色也随之镜像：** $U$ 变为**下界**（当前已知的最大成本估计），随搜索推进**只增不减**；剪枝规则相应反转——$\hat{c}(X)\leq U$（已不可能超过当前下界）的活结点被杀死，"入活结点表待选"和"杀死"的区域在 $\hat{c}$ 值轴上前后对调。

## 十一、总结：求最优解问题分支限界法的剪枝体系

| 维度 | 内容 |
|---|---|
| **剪枝依据** | 约束函数 $B$：限定**是否存在可行解**（可行解剪枝）；成本估计函数 $\hat{c}(X)$ 和界 $U$：界定**是否存在最优解**（最优解剪枝） |
| **剪枝发生点** | X **入**活结点表时：接受 $B$ 检验和 $U$ 检验（即 `UB` 函数）；X **出**活结点表时：再接受一次 $U$ 检验（应对 $U$ 在等待期间被其他分支收紧的情况） |
| **算法终止条件** | 活结点表为空；或活结点表中再没有通过 $U$ 检验的活结点（即"全表皆死"） |

> **思考题：用回溯法解决最优解问题时，$\hat{c}(X)$ 和界 $U$ 能否用于提高算法效率？**
>
> **答案：能。** 回溯法虽然是深度优先、不维护显式的活结点表，但在每次准备**纵深前进**到某个候选儿子之前，同样可以现场计算它的 $\hat{c}(X)$（或等价的下界估计），与当前已经记录到的最优解上界 $U$ 比较；一旦 $\hat{c}(X)$ 已经不可能优于 $U$，就直接放弃这个分支（不再递归深入），从而起到与分支限界法剪枝完全同样的加速效果——这说明 $\hat{c}$ 和 $U$ 是一种**通用的最优解剪枝思想**，并不依赖于"是否维护活结点表"这一具体实现细节。

## 十二、易错点汇总

| 易错点 | 正确理解 |
|---|---|
| 以为算法 8.1（BB）套上 LC-检索的 $\hat{c}$ 排序，就自动能找到最小成本答案结点 | **不能**！反例：在 E 的儿子循环里一旦碰到答案结点就立即 `return`，会抢在更优分支（活结点表里 $\hat{c}$ 更小的其他结点）被处理之前抛出错误答案（本节反例：错误输出 20，正确答案是 10） |
| 混淆"判断答案的时机" | 算法 8.2 的关键修正是：**先用 `LEAST` 选出 ĉ 最小的 E，再判断 E 是否是答案结点**；不是在生成儿子的瞬间判断 |
| 认为算法 8.2 对所有问题都保证找到最优解 | 仅对**n-元组**（答案结点必为叶结点，$\hat{c}=c=\text{cost}$）成立；**k-元组**（答案结点可能非叶）不保证，需要本节后续的 $U$、$u(X)$ 进一步加强 |
| 把 $\hat{c}$ 和 $u$ 的方向搞反 | $\hat{c}(X)\leq c(X)$（**下界**估计）；$u(X)\geq c(X)$（**上界**估计）；合起来是 $\hat{c}(X)\leq c(X)\leq u(X)$ |
| 混淆全局上界值 $U$ 与结点函数 $u(X)$ | $U$ 是算法运行过程中的一个**全局数值**（会越来越小）；$u(X)$ 是针对某结点、沿特定路径计算出的**函数值**，二者互相影响但不是同一个东西 |
| 杀死条件的等号判断弄反或漏掉 | $U$ 来自**真实成本**（`ans` 有结点）时，$\hat{c}(X)\geq U$ 杀死（**含等号**，因为已有真凡的答案，打平没意义）；$U$ 来自**上界估计**（`ans` 为空）时，仅 $\hat{c}(X)>U$ 杀死（**不含等号**，打平的结点仍有机会成为真正答案） |
| 以为 FIFOBB 和 LCBB 的主循环结构应该完全一样 | FIFOBB 因为队列**不具备**自动排序能力，多了一层内层 `DELETEQ` 循环来跳过"过时已死"的队首结点；LCBB 用 min-堆，堆顶必然最小，**一次检查**即可判断全表死活 |
| 认为求最大化问题必须先转化为最小化 | 不一定，可以直接镜像修改（方法2），只是此时 $\hat{c}$、$u$、$U$ 的不等式方向、剪枝方向全部反转 |
| 误以为 $\hat{c}$、$U$ 的剪枝思想只能用在分支限界法（需要活结点表）里 | 回溯法同样可以借助 $\hat{c}(X)$ 与 $U$ 的比较来提前放弃没希望的分支，提高效率 |

## 十三、考点速查

| 考点 | 核心内容 |
|---|---|
| 算法 8.1（BB）直接套 LC-检索的缺陷 | 在儿子生成瞬间判断答案并立即 `return`，可能漏掉活结点表中更优的分支 |
| 改进核心 | 先 `LEAST` 选出 ĉ 最小的 E，**再**判断 E 是否答案结点 |
| 改进正确性证明关键不等式 | $\text{cost}(E)=c(E)=\hat{c}(E)\leq \hat{c}(L)\leq c(L)$（对表中任意 L） |
| 算法 8.2 适用边界 | n-元组：一定终止于最优；k-元组：不一定终止于最优 |
| 两种剪枝 | 可行解剪枝（约束函数 $B$）；最优解剪枝（$\hat{c}(X)$ 和上界 $U$） |
| 上界函数 $u(X)$ 三要求 | 易计算；$c(X)\leq u(X)$；叶答案结点处 $c(X)=u(X)$ |
| 四函数不等式链 | $\hat{c}(X)\leq c(X)\leq u(X)$（$\text{cost}(X)$ 仅在 X 是答案结点时有意义） |
| $\text{cost}(X)>u(X)$ 的含义 | 以 X 为根的子树中**一定有更优解** |
| U 的两种改值 | 答案结点 $\text{cost}(X)<U\Rightarrow U\leftarrow \text{cost}(X)$, `ans←X`；状态结点 $u(X)<U\Rightarrow U\leftarrow u(X)$, `ans←0` |
| U 剪枝的等号规则 | `ans` 有结点（真实成本）→ $\hat{c}(X)\geq U$ 杀死；`ans` 为空（估计上界）→ $\hat{c}(X)>U$ 杀死 |
| 算法 8.3（UB）四步骤 | 界定杀死 → 答案结点更新 U（真实值） → 一般结点更新 U（估计值） → 通过 |
| FIFOBB 与 LCBB 结构差异原因 | 队列无序需内层循环逐个核验；min-堆堆顶天然最小，一次检查即可判定全表死活 |
| 极大化问题两种转化 | 取负转极小化（最常用）；或镜像修改，方向全部反转（$U$ 变下界、只增不减） |
| 算法终止条件 | 活结点表为空；或表中再没有通过 U 检验的活结点 |
| 回溯法能否用 $\hat{c}/U$ 提速 | 能，属于通用的最优解剪枝思想，与是否维护活结点表无关 |
