# 8.5 带有期限的作业调度问题 — 专项复习资料

> 本节对应**作业题5**，考查内容：静态树构建、$\hat{c}(X)$ 与 $u(X)$ 定义、FIFOBB 与 LCBB 动态树绘制、最优解与罚款数。

---

## 一、问题描述

**输入：** $n$ 个作业，每个作业 $i$ 由三元组 $(p_i, d_i, t_i)$ 表示：

| 符号 | 含义 |
|------|------|
| $p_i$ | 若作业 $i$ **未按时完成**所需缴纳的**罚款** |
| $d_i$ | 作业 $i$ 的**期限**（截止时间），必须在时间 $d_i$ 前完成 |
| $t_i$ | 作业 $i$ 所需的**处理时间** |

**目标：** 从 $n$ 个作业中选取子集 $J$，使 $J$ 中作业都能在各自期限内完成，且**不在 $J$ 中的作业罚款总数最小**。

> 这是一个**极小化**问题：成本 = 未选中作业的罚款之和。

---

## 二、问题建模

### 2.1 解空间表示

采用**不定长 k-元组**表示（因为 $J$ 的大小不固定）：

$$\text{解向量} = (X_1, X_2, \ldots, X_k), \quad k \leq n$$

| 约束类型 | 内容 |
|---------|------|
| **显式约束** | $X_j$ 表示选中作业的**下标**，且 $X_1 < X_2 < \cdots < X_k$（下标升序） |
| **隐式约束** | 选中的作业集合 $J = \{X_1, \ldots, X_k\}$ 能在各自期限内全部完成 |
| **目标函数** | $\sum_{i \notin J} p_i$（未选中作业的罚款总和）最小 |

> **为何是不定长 k-元组？** 因为最优子集 $J$ 的大小不固定，可能选 0 到 $n$ 个作业。

### 2.2 可行性判断（约束函数 B）

**判断作业集合 $J$ 是否可行的方法（EDF 算法）：**

将 $J$ 中的作业**按截止时间 $d_i$ 升序排列**，然后顺序执行。若所有作业都能在截止时间前完成，则 $J$ 可行。

**形式化：** 设排好序后 $J = \{j_1, j_2, \ldots, j_k\}$，$d_{j_1} \leq d_{j_2} \leq \cdots \leq d_{j_k}$，则 $J$ 可行当且仅当：
$$\forall l \in \{1, \ldots, k\}: \sum_{i=1}^{l} t_{j_i} \leq d_{j_l}$$

## 三、状态空间树

### 3.1 静态树结构

- **共 $2^n$ 个结点**（每个作业选或不选，集合树）
- **根结点**：空集（未选任何作业），层次遍历编号为 1
- **第 $l$ 层结点**：通过选取编号为 $X_l$ 的作业从父结点扩展而来
- **叶结点**：选定了完整子集 $J$（答案结点为满足约束的叶结点）

> **与回溯法不同：** 分支限界法中状态空间树按**层次遍历**编号，动态树只含实际生成的结点。

### 3.2 作业题5 的静态树（n=5）

作业实例：$(p_1,d_1,t_1)=(6,3,2)$，$(p_2,d_2,t_2)=(3,1,1)$，$(p_3,d_3,t_3)=(4,3,2)$，$(p_4,d_4,t_4)=(8,2,1)$，$(p_5,d_5,t_5)=(5,3,1)$

**静态树按层次遍历编号（共 $2^5=32$ 个结点，$n=5$ 时选最多5个作业）：**

```

  2(X₁=1), 3(X₁=2), 4(X₁=3), 5(X₁=4), 6(X₁=5)
```

## 四、函数设计

### 4.1 成本估计函数 $\hat{c}(X)$（下界，$\hat{c}(X) \leq c(X)$）

设 $S_X$ 为根到结点 $X$ 时已选中的作业集合，令 $m = \max\{i \mid i \in S_X\}$（已选作业中最大下标）：

$$\hat{c}(X) = \begin{cases} \displaystyle\sum_{\substack{i < m \\ i \notin S_X}} p_i & X \text{ 是可行结点} \\ \infty & X \text{ 是不可行结点} \end{cases}$$

> **通俗理解：** 已经决定了下标 $\leq m$ 的作业的去留，$\hat{c}$ 只统计其中**未被选中**的作业的罚款（下标 $< m$ 且不在 $S_X$ 中）。下标 $> m$ 的作业尚未决策，乐观地假设它们都能被选中（不产生罚款），因此是下界。

**为何满足 $\hat{c}(X) \leq c(X)$？**  
$c(X)$ 包含了下标 $< m$ 未选中的罚款（已确定，无法更改），还可能包含下标 $> m$ 未选中的罚款（尚未决策）。$\hat{c}$ 只计前者，后者取乐观估计0，故 $\hat{c} \leq c$。

### 4.2 成本上界函数 $u(X)$（上界，$u(X) \geq c(X)$）

$$u(X) = \sum_{i \notin S_X} p_i \quad (\text{所有未选中作业的罚款之和})$$

> **通俗理解：** 悲观地假设下标 $> m$ 的所有作业都**无法**被选中，全部缴纳罚款，这是当前路径下成本的上界估计。

**关系总结：**

$$\hat{c}(X) \leq c(X) \leq u(X)$$

| 函数 | 方向 | 含义 |
|------|------|------|
| $\hat{c}(X)$ | 下界（乐观） | 仅统计已确定未选且下标 $< m$ 的罚款 |
| $c(X)$ | 真实值（未知） | 以 $X$ 为根的子树中最小罚款 |
| $u(X)$ | 上界（悲观） | 所有当前未选作业的罚款总和 |

### 4.3 全局上界 U

- **初始值：** $\infty$（或通过贪心法得到更好的初始上界）
- **更新时机：**
  - 发现答案结点 $X$，若 $\text{cost}(X) < U$，则 $U \leftarrow \text{cost}(X)$
  - 发现结点 $X$，若 $u(X) < U$，则 $U \leftarrow u(X)$（此时 $\text{ans} \leftarrow 0$，尚无实际答案）
- **剪枝规则：**
  - 若 $\hat{c}(X) > U$：杀死结点 $X$（**不可能改善当前最优**）
  - 若 $\hat{c}(X) = U$ 且 ans 已有结点：也杀死（已找到同等或更好的解）

## 七、LCBB 算法运行（作业题5 实例）

**LCBB（最小成本分支限界法）：** 活结点表用**极小堆**实现，每次选 $\hat{c}$ 最小的结点作为 E-结点。

### LCBB 与 FIFOBB 的关键区别

| 特征 | FIFOBB | LCBB |
|------|--------|------|
| 活结点表结构 | 队列（FIFO） | 极小堆（按 $\hat{c}$ 排序） |
| 选 E-结点规则 | 先进先出 | $\hat{c}$ 最小者优先 |
| 终止条件 | 队列为空 | 堆空 **或** 堆顶 $\hat{c} \geq U$（ans有结点时） |
| 最终结果 | 相同最优解 | 相同最优解，但通常**访问结点更少** |

### LCBB 运行示意

初始：$U=26$，活结点表为空，$E=1$（根）。

1. 扩展根结点1，生成儿子 2,3,4,5,6，计算各 $\hat{c}$，入堆（不可行或 $\hat{c}\geq U$ 的杀死）。
2. 堆顶（$\hat{c}$ 最小的结点）出堆，成为新 E-结点。
3. 不断扩展，更新 $U$，直到堆空或堆顶 $\hat{c} \geq U$（且 ans 有结点）。

> **LCBB 的优势：** 因为优先扩展 $\hat{c}$ 最小的结点，更快到达最优解方向，剪枝更多。

## 八、作业题5 解题指南

### 第(1)题：画静态树（n=5）

- $2^5 = 32$ 个结点，按层次遍历编号。
- 根（编号1）为空集；第一层5个结点分别对应 $X_1 \in \{1,2,3,4,5\}$；以此类推。
- **静态树无需标注可行性**，仅表示解空间结构。

### 第(2)题：给出 $\hat{c}(X)$ 和 $u(X)$ 定义

**$\hat{c}(X)$（成本下界估计函数）：**

设 $S_X$ 为到达结点 $X$ 时已选中的作业集合，$m = \max\{i \mid i \in S_X\}$：
$$\hat{c}(X) = \sum_{\substack{i < m \\ i \notin S_X}} p_i \quad (X \text{ 可行}); \quad \hat{c}(X) = \infty \quad (X \text{ 不可行})$$

**$u(X)$（成本上界估计函数）：**
$$u(X) = \sum_{i \notin S_X} p_i$$

### 第(3)题：画 FIFOBB 和 LCBB 的动态树

**步骤：**

1. 从根结点出发，计算根的 $\hat{c}$ 和 $u$，初始化 $U \leftarrow u(\text{根}) = \sum p_i = 26$。
2. **FIFOBB：** 用队列管理活结点，依次扩展。
   - 每个新结点 $X$：先检验 $B(X)$（可行性），再调用 $\text{UB}(X, U, \text{ans})$：
     - 若 $\hat{c}(X) > U$ 或（ans有结点且 $\hat{c}(X)=U$）→ 杀死
     - 否则：若 $X$ 是答案结点且 $\text{cost}(X) < U$ → $U \leftarrow \text{cost}(X)$，$\text{ans} \leftarrow X$
     - 若 $u(X) < U$ → $U \leftarrow u(X)$，$\text{ans} \leftarrow 0$
     - 入队
   - 出队条件：队列非空且队首 $\hat{c} < U$（或 ans 无结点且 $\hat{c}=U$）
3. **LCBB：** 用极小堆管理活结点（按 $\hat{c}$ 排序），扩展规则相同，选堆顶结点为 E。

**动态树标注要求：**
- 每个结点标注：静态树编号 + $\hat{c}$ 值
- 被杀死的结点标注"kill"或不画出（题目要求有时不同）
- 标出**答案结点**（最优解对应结点）
- 给出**最小罚款数**

## 十、易错点汇总

| 易错点 | 正确处理方式 |
|--------|------------|
| **$\hat{c}$ 的统计范围搞错** | 只统计下标 $< m$（已决策范围内）且**未选中**的罚款；下标 $> m$ 的乐观假设为0 |
| **$m$ 的含义** | $m$ 是已选集合 $S_X$ 中**最大**的作业下标，不是集合大小 |
| **根结点的 $\hat{c}$** | 根结点 $S_X = \emptyset$，无 $m$，$\hat{c}(\text{根}) = 0$ |
| **不可行结点的 $\hat{c}$** | 不可行结点 $\hat{c} = \infty$，直接杀死，不入活结点表 |
| **$u(X)$ 的统计范围** | 是**所有未选中**的作业罚款（包含下标 $> m$ 的），悲观估计 |
| **$U$ 更新来源** | $U$ 可以从 $\text{cost}(X)$（发现真实答案结点）或 $u(X)$（上界估计）两种来源更新 |
| **FIFOBB 终止条件** | 队列为空（不是 $\hat{c} \geq U$）；每次出队后还要检验该结点是否已超出 $U$ |
| **LCBB 终止条件** | 堆空，**或**堆顶 $\hat{c} \geq U$（且 ans 有结点时）；两者都要检查 |
| **静态树编号** | 按**层次遍历**（BFS顺序）编号，动态树中的编号与静态树一致 |
| **EDF 可行性验证** | 一定要先**按 $d_i$ 升序排序**，再顺序执行，检验每步是否超期 |

## 十一、考点速查

| 考点 | 核心内容 |
|------|---------|
| **问题建模** | k-元组、显式约束（下标升序）、隐式约束（可行性）、极小化目标 |
| **可行性判断** | EDF：按 $d_i$ 升序执行，逐步检验 $\sum t \leq d$ |
| **$\hat{c}(X)$ 公式** | $\sum_{i<m, i\notin S_X} p_i$（下界）；不可行时 $=\infty$ |
| **$u(X)$ 公式** | $\sum_{i\notin S_X} p_i$（上界） |
| **$U$ 的更新** | 发现答案结点取 $\text{cost}$；发现 $u(X)<U$ 取 $u(X)$ |
| **FIFOBB 活结点表** | 队列（先进先出） |
| **LCBB 活结点表** | 极小堆（$\hat{c}$ 最小优先） |
