## 一、分支限界法一般方法（8.1）

### 1.1 适用问题

分支限界法与回溯法类似，适用于**组合数较大的问题**，特别是**组合优化问题（求最优解）**。两种方法共用的概念：

| 概念 | 说明 |
|------|------|
| 解向量表达 | n-元组 / k-元组 |
| 显式约束条件 | 每个 $x_i$ 的取值范围 |
| 隐式约束条件 | 各 $x_i$ 之间的关联约束 |
| 解空间 / 解空间树 | 与回溯法完全相同 |

**两者核心区别：E-结点（扩展结点）的处理方式不同。**

| 特征 | 回溯法 | 分支限界法 |
|------|--------|-----------|
| 搜索方式 | 深度优先（DFS） | BFS / 优先队列 |
| E-结点 | 生成一个儿子后立即转移 | 生成**所有**儿子后才变为死结点 |
| 活结点表 | 不需要 | 需要维护 |
| 求最优解效率 | 一般 | 更高（借助 $\hat{c}$ 选择） |
| 存储空间 | 小 | 较大（需维护活结点表） |

> 注：清华大学出版的《算法设计与分析》认为，"分支限界是回溯算法的变种"。

### 1.2 分支限界法基本思想

算法在解空间树上按如下方式搜索：

1. 若根结点 T 是答案结点，输出 T，结束。
2. 令 T 为当前扩展结点 E。
3. **生成 E 的所有儿子结点**，对每个儿子 X 判断：
   - 若 X 是答案结点，输出路径，结束；
   - 若 X 满足约束函数 B(X)，则将 X 加入**活结点表**，否则舍弃 X。
4. 从活结点表中选出下一个结点作为新的 E-结点，重复。
5. 若活结点表为空，算法以失败结束。

> **约束函数的作用：** 避免生成不包含可行解的子树（可行解剪枝）。

### 1.3 算法 8.1 抽象描述

```
procedure BB(T)
  if T 是答案结点 then 输出 T; return endif
  E ← T
  将活结点表初始化为空
  loop
    for E 的每个儿子 X do
      if X 是答案结点 then 输出从 X 到 T 的那条路径; return; endif
      if B(X) then call ADD(X); PARENT(X) ← E endif
    repeat
    if 表中不再有活结点 then print("no answer node"); return; endif
    call LEAST(E)
  repeat
end BANDB
```

- **ADD(X)**：将 X 添加到活结点表中
- **LEAST(E)**：从活结点表中选中一个结点赋值给 E，并从表中删除
- **B(X)**: 界限函数,用于计算结点 X 的下界，判断是否值得继续往下搜索（剪枝依据）
- **PARENT(X)**: 父结点指针.记录 X 的父结点是谁，用于最终找到答案时回溯整条路径

### 1.4 两种检索方式

| 检索方式 | 活结点选取规则 | 数据结构 |
|---------|-------------|---------|
| **FIFO-检索** | 先进先出（BFS） | 队列 |
| **LC-检索（最小代价优先）** | 成本估计函数 $\hat{c}$ 最小的优先 | 极小堆 |

**FIFO-检索特点：**
- 能找到**离根最近**的答案结点
- 不关心问题的具体实例，按千篇一律的顺序处理
- 4-皇后问题中 FIFO 处理了 **31 个结点**，回溯法处理了 **16 个**

## 二、LC-检索（8.2）

### 2.1 LC-检索的动机

FIFO-检索对下一个 E-结点的选择是**盲目的**，不区分哪些活结点更可能快速到达答案结点。

理想做法：用**"有智力的"成本函数 $c$** 给活结点排序，优先选择更可能导向答案结点的活结点。

### 2.2 成本函数 c 的量化方法

令 X 是当前结点，$c(X)$ 关注以 X 为根的子树中的**最小成本值**：

$$c(X) = \begin{cases} \text{cost}(X) & \text{叶结点 X 是答案结点} \\ \min\{\text{cost}(P) \mid \text{答案结点 } P \in \text{子树 X}\} & \text{子树 X 中包含答案结点} \\ \infty & \text{子树 X 不包含答案结点} \end{cases}$$

其中 $\text{cost}(X)$ 表示**根到达答案结点 X 的真实成本**（如路径长度、目标函数值等），非答案结点时为 $\infty$。

**三种量化方法：**

| 方法 | 目标 | 定义依据 |
|------|------|---------|
| 方法1 | 寻找生成数目最少的答案结点 | 到达答案结点前需要生成的结点数 |
| 方法2 | 寻找路径长度最短的答案结点 | 答案结点的路径长度 |
| 方法3 | 寻找使目标函数取极值的答案结点（**最优解**） | 目标函数值 |

> **规则：** 问题不存在目标函数时，用方法2定义 $c$ 函数。

### 2.3 成本估计函数 $\hat{c}$

直接计算 $c(X)$ 的难度等同于原问题本身。因此，设计一个**易于计算的成本估计函数** $\hat{c}$：

$$\hat{c}(X) = f(h(X)) + \hat{g}(X)$$

| 分量 | 含义 |
|------|------|
| $h(X)$ | 根结点到结点 X 的**已产生成本** |
| $\hat{g}(X)$ | X 到最小成本答案结点的**估计成本**（未来） |
| $f$ | 调整 $h$ 和 $\hat{g}$ 影响比例的非负函数 |

**对 $\hat{c}$ 的要求：**

1. 易于计算
2. $\hat{c}(X) \leq c(X)$（下界估计）
3. 当叶结点 X 是答案结点时，$c(X) = \hat{c}(X)$

> **为何不能只用 $\hat{g}(X)$？**
> 若 $\hat{c}(X) = \hat{g}(X)$（即 $f=0$），只考虑未来代价，则子结点 $\hat{g}$ 值往往 $\leq$ 父结点，算法将一直纵深搜索而不切换分支，退化为深度优先——"乐观的冒进"，可能丢失更靠近根的最优答案。

### 2.4 LC-检索定义

**LC-检索（Least Cost Search）**：选取 $\hat{c}$ 值最小的活结点作为下一个 E-结点。

**BFS 是 LC-检索的特殊情况：** 令 $f(h(X))$ 为根到结点 X 的路径长度，$\hat{g}(X)=0$，则退化为广度优先搜索。

### 2.5 LC-检索总结

- 算法优先选择**更靠近答案结点、同时又离根较近**的结点
- 对于每个结点 X，有 $\hat{c}(X) \leq c(X)$（即估计值不高于真实成本）
- 当答案结点 X 是叶结点时，$c(X) = \hat{c}(X)$（n-元组表达时算法必终止于最小成本）

## 三、15-谜问题（8.3）

### 3.1 问题描述

在 4×4 方格棋盘上放 15 块编号牌（1-15），要求通过合法移动（当前牌邻接有空位置时，可将牌移至空位）到达目标排列：

```
目标排列 (b)：     初始排列 (a)：
 1  2  3  4        1  2  3  4
 5  6  7  8        5  6     8
 9 10 11 12        9 10  7 11
13 14 15           13 14 15 12
```

### 3.2 状态空间树

- **问题状态：** 棋牌布局（棋盘上各牌的排列方式）
- **初始状态：** 初始排列；**目标状态：** 目标排列
- **E-结点的儿子：** 通过一次合法移动到达的布局状态
- **移动方向顺序：** 上、右、下、左（空格的移动方向）

### 3.3 判定定理（关键！）

对棋盘方格位置编号 1~16（与目标排列一致，空格为位置 16）：

- $\text{POSITION}(i)$：牌面 $i$ 在**初始状态**时的位置号
- $\text{LESS}(i)$：牌面上 $j < i$ 且 $\text{POSITION}(j) > \text{POSITION}(i)$ 的 $j$ 的数目（即反序数）

**判定定理：** 当且仅当初始状态的 $\sum_{1 \leq i \leq 16} \text{LESS}(i) + X$ 是**偶数**时，图(b)所示目标状态可由此初始状态到达。

> 其中 $X$ 的取值：若空格在棋盘阴影位置（棋盘中的偶数行列格），$X=1$；否则 $X=0$。

**初始排列(a)验证：**

| 牌面 $i$ | 8 | 9 | 10 | 13 | 14 | 15 | 16（空格） |
|---------|---|---|----|----|----|----|-----------|
| $\text{LESS}(i)$ | 1 | 1 | 1 | 1 | 1 | 1 | 9 |

其余牌面 LESS 值为 0，故 $\sum \text{LESS}(i) = 15$；空格在位置 7（行2列3，奇数行），$X=1$。

$15 + 1 = 16$（偶数）→ **可达目标状态(b)**。

### 3.4 三种检索方式比较

| 检索方式 | 行为特点 | 缺点 |
|---------|---------|------|
| **FIFO** | 能找到离根最近的答案结点 | 不关心实例具体情况，千篇一律 |
| **深度优先** | 始终沿最左路径纵深 | 呆板盲目，可能远离目标 |
| **LC-检索** | 基于 $\hat{c}$ 智能选择 | 需设计合适的 $\hat{c}$ |

### 3.5 15-谜的 $\hat{c}(X)$ 设计

目标：寻找从初始排列到目标排列**棋牌最少移动次数**。

$$\hat{c}(X) = f(X) + \hat{g}(X)$$

| 分量 | 含义 |
|------|------|
| $f(X)$ | 从初始排列到达状态 X 时，棋牌**已经移动的次数** |
| $\hat{g}(X)$ | 当前状态 X 中**不在目标位置的非空白棋牌数目** |

**示例（初始排列(a)）：**
不在目标位置的非空白棋牌为 $\{7, 11, 12\}$，故 $\hat{g}(\text{根}) = 3$，$f(\text{根}) = 0$，$\hat{c}(\text{根}) = 3$。

**LC-检索效果：** 对于本实例仅生成 **12 个结点**就找到答案，远优于 FIFO（23 个结点），接近使用精确函数 $c$ 的效果。

## 四、求最小成本的分支限界法（8.4）

### 4.1 寻找最小成本的思路

**问题：** 算法 BB（基于 FIFO 或 LC-检索）一旦发现儿子结点 X 是答案结点就立即结束，但这不能保证找到**最小成本**的答案结点。

**改进方法：** 从活结点表选出 E-结点后，**再判断 E 是否是答案结点**，若是则输出路径。

> 此改进适用于 **n-元组表达**。对于 n-元组，答案结点必为叶结点，满足 $\hat{c}(E) = c(E) = \text{cost}(E)$，故 E 必为最小成本。

### 4.2 算法 8.2 基于 $\hat{c}$ 求最小成本的 LC-检索

```
procedure LC(T, ĉ)   // 找出最小成本答案结点
  E ← T，将活结点表初始化为空
  loop
    if E 是答案结点 then 输出从 E 到 T 的那条路径; return; endif
    for E 的每个儿子 X do
      if B(X) then call ADD(X); PARENT(X) ← E; endif
    repeat
    if 表中不再有活结点 then print("no answer node"); return; endif
    call LEAST(E)   // 选取 ĉ 最小的结点
  repeat
end LC
```

**n-元组 vs k-元组 的差异：**

| 表达类型 | 答案结点特征 | 算法 8.2 结论 |
|---------|------------|-------------|
| **n-元组** | 答案结点必为叶结点，$\hat{c}(.) = c(.) = \text{cost}(.)$ | 必终止于最小成本答案结点 ✓ |
| **k-元组** | 答案结点可能非叶结点，仅满足 $\hat{c}(.) \leq c(.) \leq \text{cost}(.)$ | 不一定终止于最小成本 ✗ |

### 4.3 加速：引入成本上界 U

**思想：** 借助 $\hat{c}$ 和界 $U$，同时进行两种剪枝：

| 剪枝类型 | 依据 | 作用 |
|---------|------|------|
| **可行解剪枝** | 约束函数 $B(X)$ | 排除不含可行解的子树 |
| **最优解剪枝** | $\hat{c}(X) \geq U$（或 $> U$） | 排除不含最优解的子树 |

**最小成本上界 U 的性质：**

- $U$ 是当前算法生成过的所有状态结点对最小成本**上界估计的最小值**
- 初值为 $\infty$，或通过启发式方法（如贪心法）得到
- 随着结点的访问**不断减小**

### 4.4 成本上界函数 $u(X)$

$u(X)$ 是对成本 $c(X)$ 的**上界估计**（$c(X) \leq u(X)$），类比 $\hat{c}(X)$ 是下界估计：

$$\hat{c}(X) \leq c(X) \leq u(X)$$

对 $u(X)$ 的要求：
1. 易于计算
2. $c(X) \leq u(X)$
3. 当叶结点 X 是答案结点时，$c(X) = u(X)$

**$U$ 的更新规则：**
- 发现答案结点 X，且 $\text{cost}(X) < U$：$U \leftarrow \text{cost}(X)$
- 发现状态结点 X，且 $u(X) < U$：$U \leftarrow u(X)$

**$U$ 的剪枝规则：**

| $U$ 来源 | 剪枝条件 | 说明 |
|---------|---------|------|
| 真实成本（已找到答案结点） | $\hat{c}(X) \geq U$ 的活结点被杀死 | 已有真实最优，更差的不需要 |
| 成本上界（u 函数） | $\hat{c}(X) > U$ 的活结点被杀死 | 上界确保了存在更好解，等于时保留 |

### 4.5 界函数 UB（算法 8.3）

```
procedure UB(X, U, ans)
  // ans：当前最小成本的答案结点，0 表示无结点
  // 当 X 是答案结点时，cost(X) 是真实成本值，否则 ∞
  if ĉ(X) > U or (ans 有结点 and ĉ(X) = U) then return false endif
  if X 是答案结点 and cost(X) < U then U ← cost(X); ans ← X endif
  if u(X) < U then U ← u(X); ans ← 0 endif
  return true
end UB
```

| 条件 | 结果 |
|------|------|
| $\hat{c}(X) > U$，或（ans有结点 且 $\hat{c}(X) = U$） | **界定杀死**，返回 false |
| $\hat{c}(X) < U$，或（ans无结点 且 $\hat{c}(X) = U$） | **界定通过**，返回 true |

### 4.6 算法 8.4 求最小成本的 FIFO-分支限界

```
procedure FIFOBB(T, ĉ, u, cost)
  E ← T; PARENT(E) ← 0; U ← ∞; ans ← 0
  UB(E, U, ans); 将队列初始化为空
  loop
    for E 的每个儿子 X do
      if B(X) and UB(X, U, ans) then call ADDQ(X); PARENT(X) ← E endif
    repeat
    loop
      if 队列为空 then print('least cost =', U); 输出从 ans 到 T 的路径; return endif
      call DELETEQ(E)
      if ĉ(E) < U or (ans 无结点 and ĉ(E) = U) then exit endif   // 界定通过
    repeat
  repeat
end FIFOBB
```

> 假定状态空间树 T 至少包含一个解结点，不可行结点的估计值 $\hat{c}(X) = \infty$。

### 4.7 算法 8.5 求最小成本的 LC-分支限界

```
procedure LCBB(T, ĉ, u, cost)
  // ADD：加结点到 min-堆；LEAST：从 min-堆删去堆顶结点
  E ← T; PARENT(E) ← 0; U ← ∞; ans ← 0
  UB(E, U, ans); 将活结点表初始化为空
  loop
    for E 的每个儿子 X do
      if B(X) and UB(X, U, ans) then call ADD(X); PARENT(X) ← E endif
    repeat
    if 表中不再有活结点
       or 堆顶结点 ĉ 值 > U
       or (ans 有结点 and 堆顶结点 ĉ 值 = U)
      then print('least cost =', U); 输出从 ans 到 T 的路径; return
    endif
    call LEAST(E)
  repeat
end LCBB
```

### 4.8 求极大化问题

| 方法 | 处理方式 |
|------|---------|
| **方法1**（推荐） | 取目标函数相反数，转化为极小化问题 |
| **方法2** | 对照极小化做镜像修改，此时 $u(X) \leq c(X) \leq \hat{c}(X)$，$U$ 是**下界**且不断**增大** |

### 4.9 算法剪枝总结

| 类型 | 时机 | 依据 |
|------|------|------|
| **可行解剪枝** | X 入活结点表时 | 约束函数 $B(X)$ |
| **最优解剪枝（入表）** | X 入活结点表时 | $\hat{c}(X)$ 与 $U$ 比较（UB 函数） |
| **最优解剪枝（出表）** | X 出活结点表时 | $\hat{c}(E)$ 与 $U$ 比较 |

**算法终止条件：**
1. 活结点表为空；
2. 活结点表中再没有通过 $U$ 检验的活结点（堆顶 $\hat{c}$ 值 $\geq U$）。

## 五、带有期限的作业调度问题（8.5）—— 作业题5

### 5.1 问题描述

有 $n$ 个作业和一台处理机，作业 $i$ 由三元组 $(p_i, d_i, t_i)$ 表示：
- $p_i$：若未在期限内完成，需交付的**罚款**
- $d_i$：**期限**（截止时间）
- $t_i$：处理所需**时间**

**目标：** 选取作业子集 J，使 J 中作业都在期限内完成，且**未选中作业的罚款总数最小**（极小化问题）。

### 5.2 问题建模

- **元组表达：** 不定长 k-元组 $(X_1, \ldots, X_k)$，$k \leq n$
- **显式约束：** $X_j$ 表示选中的作业下标，且 $X_i < X_{i+1}$（$1 \leq i < k$）
- **隐式约束（B 函数）：** 选中的作业能在各自期限前完成

### 5.3 成本函数 $\hat{c}(X)$（下界估计）

设 $S_X$ 是根结点到结点 X 时已选中的作业集合，令 $m = \max\{i \mid i \in S_X\}$：

$$\hat{c}(X) = \begin{cases} \sum_{i < m, \, i \notin S_X} p_i & \text{X 是可行解} \\ \infty & \text{X 是不可行解} \end{cases}$$

**含义：** 在下标 $m$ 之前且未被选中的作业，其罚款一定会产生（因为即使之后所有更大下标的作业都选，这些也无法回头选）。

### 5.4 成本上界函数 $u(X)$

$$u(X) = \sum_{i \notin S_X} p_i$$

**含义：** X 路径确定后，所有未选中作业的罚款总和（上界）。

### 5.5 实例运行 (n=5)

**(题5实例)** $n=5$，$(p_1,d_1,t_1)=(6,3,2)$，$(p_2,d_2,t_2)=(3,1,1)$，$(p_3,d_3,t_3)=(4,3,2)$，$(p_4,d_4,t_4)=(8,2,1)$，$(p_5,d_5,t_5)=(5,3,1)$。

**构建静态树（按层次遍历编号）：**

根结点（1）的每层分支对应"选第几个作业"，共 $2^5 = 32$ 个叶结点。状态空间树的路径表示所选作业的下标集合。

**判断约束 B：** 利用 EDF（最早截止时间优先）判定作业集合是否能在期限内完成——按 $d_i$ 升序排列后逐一模拟执行时间，若存在作业超出截止时间则不可行。

**FIFOBB 算法步骤（示意）：**

从根结点出发，逐层生成儿子结点，同时：
1. 对每个儿子 X，先用 $B(X)$ 检验可行性（$\hat{c}(X) = \infty$ 时舍弃）；
2. 再用 $UB(X, U, \text{ans})$ 检验最优性；
3. 通过检验的结点加入 FIFO 队列；
4. 每次从队列取出 E-结点，更新 $U$ 和 ans，直至队列为空。

**最优解分析（n=4 课件实例）：**

以课件中 $n=4$，$(5,1,1)$，$(10,3,2)$，$(6,2,1)$，$(3,1,1)$ 为例：

| 编号 | 路径 | $\hat{c}$ | $u$ | 说明 |
|------|------|-----------|-----|------|
| 1（根） | 空 | 0 | 24 | $U \leftarrow 24$ |
| 2 | $\{1\}$ | 0 | 19 | $U \leftarrow 19$ |
| 3 | $\{2\}$ | 5 | 14 | $U \leftarrow 14$ |
| 4 | $\{3\}$ | 15 | 18 | 未更新 |
| 5 | $\{4\}$ | 21 | 21 | kill（$\hat{c} = 21 > U = 14$） |
| 6 | $\{1,2\}$ | 0 | 9 | $U \leftarrow 9$ |
| 7 | $\{1,3\}$ | 10 | — | kill |
| 9 | $\{2,3\}$ | 5 | 8 | $U \leftarrow 8$，ans=9 |
| 10 | $\{2,4\}$ | 11 | — | kill |

队列为空，算法结束，最优解 ans=9（即选 $\{2,3\}$），最小罚款 $U=8$（未选作业1和4的罚款 $5+3=8$）。

## 六、货郎担问题（8.6）

### 6.1 问题描述

设 $G(V, E)$ 是有向图，$|V|=n$，$c_{ij}>0$ 表示边 $\langle i, j \rangle$ 的成本。寻找一条**最小成本**的周游路线（从结点1出发，经过所有结点各一次，返回1）。

- 动态规划法：$\Theta(n^2 2^n)$（任何实例）
- **分支限界法：** 最坏情况也是 $O(n^2 2^n)$，但对许多具体实例**花费时间远少于动态规划**

### 6.2 问题建模

- **元组表达：** 固定长 $(n-1)$-元组 $(X_1, \ldots, X_{n-1})$，表示结点 $2 \sim n$ 的访问排列
- **显式约束：** $X_i \in \{2, 3, \ldots, n\}$ 且互不相同
- **隐式约束：** 存在周游路线 $1, X_1, X_2, \ldots, X_{n-1}, 1$（相邻两点有边）
- **目标函数：** 周游路线成本最小

### 6.3 成本函数 $c(X)$

$$c(X) = \begin{cases} \text{cost}(X) = \text{周游路线 } 1,X_1,\ldots,X_{n-1},1 \text{ 的总成本} & \text{X 是叶结点} \\ \min\{\text{cost}(P) \mid \text{叶结点 } P \in \text{子树 X}\} & \text{X 是非叶结点} \end{cases}$$

### 6.4 成本下界函数 $\hat{c}(X)$

设结点 X 对应部分路径 $1, X_1, \ldots, X_k$，令 $S = V - \{1, X_1, \ldots, X_k\}$ 为剩余结点集合：

$$\hat{c}(X) = \begin{cases} \displaystyle\sum_{0 \leq i < k} a_i + l_{X_k} + \sum_{d \in S} l_d & k < n-1 \\ \displaystyle\sum_{0 \leq i < n-1} a_i + c_{X_{n-1}, 1} & k = n-1 \end{cases}$$

其中：
- $a_i = c_{X_i, X_{i+1}}$：已选路段 $\langle X_i, X_{i+1} \rangle$ 的成本（$X_0 = 1$）
- $l_d$：结点 $d$ **出发**的最小边成本（$d \in S \cup \{X_k\}$），**不关心到达哪个点**

**直觉：** 路径中每个结点（包括 $X_k$ 和剩余结点）至少贡献一条出边，取各自最小出边之和作为下界。

### 6.5 成本上界 U

初始上界 $U$ 可通过**贪心法**（每次选最近未访问城市）获得：
- 课件实例（$n=4$）从 1 出发，贪心法得路线 $1 \to 2 \to 4 \to 3 \to 1$，成本 $10+10+9+6=35$，故 $U$ 初值可设为 39（或直接用 39 作为某路线成本），之后随搜索更新。

> 本节未设计 $u(X)$，因此 $U$ 仅靠找到答案结点时的真实成本 $\text{cost}$ 来更新。

### 6.6 实例运行（$n=4$）

**邻接矩阵：**

$$C = \begin{pmatrix} 0 & 10 & 15 & 20 \\ 5 & 0 & 9 & 10 \\ 6 & 13 & 0 & 12 \\ 8 & 8 & 9 & 0 \end{pmatrix}$$

各结点最小出边：$l_1=10,\ l_2=5,\ l_3=6,\ l_4=8$。

**部分 $\hat{c}$ 值计算：**

| 结点编号 | 路径 | $f(X)$ | $S$ | $l_{X_k}$ | $\sum_{d \in S} l_d$ | $\hat{c}(X)$ |
|---------|------|--------|-----|-----------|---------------------|-------------|
| 1（根） | 空 | 0 | $\{2,3,4\}$ | 10（$l_1$） | 19（$5+6+8$） | **29** |
| 2 | $1\to2$ | 10 | $\{3,4\}$ | 5（$l_2$） | 14（$6+8$） | **29** |
| 3 | $1\to3$ | 15 | $\{2,4\}$ | 6（$l_3$） | 13（$5+8$） | **34** |
| 4 | $1\to4$ | 20 | $\{2,3\}$ | 8（$l_4$） | 11（$5+6$） | **39** |
| 5 | $1\to2\to3$ | 19 | $\{4\}$ | 6（$l_3$） | 8（$l_4$） | **33** |
| 6 | $1\to2\to4$ | 20 | $\{3\}$ | 8（$l_4$） | 6（$l_3$） | **34** |
| 12（叶）| $1\to2\to4\to3$ | — | 空 | — | — | **35** |

**LCBB 运行过程（初始 $U=39$，ans=0）：**

| E | 儿子 X | $U$ | ans | 活结点表（堆） |
|---|--------|-----|-----|--------------|
| 1 | 2,3,4弃 | 39 | — | 2(29), 3(34) |
| 2 | 5(33),6(34) | 39 | — | 3(34), 5(33), 6(34) |
| 5 | 11弃 | 39 | — | 3(34), 6(34) |
| 3 | 7(36),8(35) | 39 | — | 6(34), 7(36), 8(35) |
| 6 | 12(35) | 35 | 12 | 7(36), 8(35) |
| 8 | — | 35 | 12 | 7(36) |

堆顶结点 $\hat{c}(7) = 36 > U = 35$（若取等则 ans 有结点），算法终止。

**最优解：** ans=12，路线 $1 \to 2 \to 4 \to 3 \to 1$，成本 $U = 35$。

## 七、小结（8.7）

### 7.1 分支限界法 vs 回溯法

| 特性 | 分支限界法 | 回溯法 |
|------|-----------|--------|
| 适用场景 | 组合优化（求最优解）为主 | 组合搜索（求可行解）为主 |
| 搜索顺序 | BFS / 最优优先 | DFS |
| E-结点处理 | 生成**所有**儿子后变为死结点 | 生成**一个**儿子后转移 |
| 活结点表 | 需要（队列/堆） | 不需要 |
| 存储需求 | 较大 | 较小 |
| 求最优解 | 效率更高（借助 $\hat{c}$ 选择） | 需额外处理 |
| 与回溯法关系 | 回溯法的变种 | 基础方法 |

### 7.2 分支限界法设计步骤

对于极小化问题：

1. **建模：** 确定元组表达、显式/隐式约束
2. **目标函数 → 成本函数 $c$**
3. **设计 $\hat{c}(X)$**（下界估计，$\hat{c} \leq c$）
4. **设计 $u(X)$**（上界估计，可选，$c \leq u$）
5. **初始化 $U$**（可用贪心法）
6. **运行 FIFOBB 或 LCBB**

### 7.3 各函数对比速查

| 函数 | 类型 | 关系 | 作用 |
|------|------|------|------|
| $\text{cost}(X)$ | 真实成本 | — | 答案结点 X 到根的路径成本 |
| $c(X)$ | 精确成本 | = $\min\{\text{cost}\}$ | 子树 X 中最小 cost（上帝视角） |
| $\hat{c}(X)$ | 下界估计 | $\hat{c} \leq c$ | 选择 E-结点（LC-检索），剪去最优解 |
| $u(X)$ | 上界估计 | $c \leq u$ | 加速减小 $U$，提前剪枝 |
| $U$ | 全局上界 | $\geq$ 最小成本 | 界定所有活结点的死活 |

### 7.4 区分 cost / c / $\hat{c}$ / $u$ / $U$

**以货郎担问题为例：**

- $\text{cost}(X)$：叶结点 X 对应的某条完整周游路线的成本
- $c(X)$：以 X 为根的子树中，所有可行周游路线的最小成本（不可直接算，需遍历）
- $\hat{c}(X)$：用已选路段成本 + 剩余结点最小出边之和估计（$\leq c(X)$）
- $u(X)$：若未设计则不使用；如设计则满足 $\geq c(X)$
- $U$：当前找到的最短周游路线长度（初值可为贪心解）

## 八、作业题精讲

### 作业题4：15-谜问题

**题目：** 已知某初始状态，(1) 判断是否能达到目标状态；(2) 用 LC-检索画出状态空间树并标注 $\hat{c}$ 值。

**解题步骤：**

**(1) 判断可达性：**

1. 对 16 个位置编号（与目标排列对应，空格为位置 16）；
2. 计算每个牌面 $i$ 的 $\text{LESS}(i)$（在 $i$ 之前且位置号更大的牌的数量）；
3. 判断 $\sum \text{LESS}(i) + X$ 是否为偶数（$X$：空格在阴影格取1，否则取0）；
4. 偶数 → 可达；奇数 → 不可达。

**(2) LC-检索作图：**

1. 根结点 = 初始状态，$\hat{c}(\text{根}) = 0 + \hat{g}(\text{根})$；
2. 生成根的所有儿子（上/右/下/左，注意不回到父状态）；
3. 计算每个儿子的 $\hat{c}$，选 $\hat{c}$ 最小者为新 E-结点；
4. 若 $\hat{c}$ 值相同，按约定顺序（上/右/下/左）选择；
5. 重复直至找到目标状态（$\hat{g} = 0$）。

### 作业题5：带期限的作业调度问题

**题目：** $n=5$，$(6,3,2)$，$(3,1,1)$，$(4,3,2)$，$(8,2,1)$，$(5,3,1)$。画出 FIFOBB 和 LCBB 的动态树。

**关键步骤：**

**(1) 静态树结构：** k-元组解空间树，根结点1，每个结点的儿子对应选择下标更大的作业。

**(2) $\hat{c}(X)$ 计算：** $\sum_{i < m, i \notin S_X} p_i$；$u(X)$ 计算：$\sum_{i \notin S_X} p_i$。

**(3) 约束函数 B 验证（EDF算法）：**

将 $S_X$ 中的作业按截止时间 $d_i$ 升序排列，依次执行，若某作业完成时刻超过 $d_i$ 则不可行。

**对于 n=5 实例的关键判断：**

| 候选集合 | 执行顺序（EDF）| 执行时刻 | 可行？|
|---------|--------------|---------|------|
| $\{2\}$ | 作业2（d=1,t=1）| 完成于时刻1 ≤ 1 | ✓ |
| $\{4\}$ | 作业4（d=2,t=1）| 完成于时刻1 ≤ 2 | ✓ |
| $\{2,4\}$ | 作业2→作业4 | 1,2 ≤ 1,2 | ✓ |
| $\{1,3,5\}$ | 按d排序：作业3(d=3,t=2),作业1(d=3,t=2),作业5(d=3,t=1) | 完成于2,4→超期 | ✗ |
| $\{4,5\}$ | 作业4→作业5 | 1,2 ≤ 2,3 | ✓ |
| $\{2,4,5\}$ | 作业2→作业4→作业5 | 1,2,3 ≤ 1,2,3 | ✓ |

**最优解：** 选 $\{2,4,5\}$（作业2、4、5），未选作业为 $\{1,3\}$，罚款 $= 6 + 4 = 10$。

**(4) FIFOBB 与 LCBB 的差异：**

| 算法 | 选 E-结点方式 | 动态树形状 |
|------|-------------|----------|
| FIFOBB | 按入队顺序（BFS层序） | 按层展开，较宽 |
| LCBB | 按 $\hat{c}$ 最小（堆） | 沿最小代价路径，较窄 |

两者最终结果相同，但 LCBB 通常访问结点数更少（约束更紧）。

## 九、考点速查

| 考点 | 核心内容 |
|------|---------|
| **FIFO vs LC-检索** | FIFO = BFS，LC = 成本估计函数 $\hat{c}$ 最小优先 |
| **$\hat{c}$ 的三个要求** | 易计算；$\hat{c}(X) \leq c(X)$；叶节点答案结点时 $c(X) = \hat{c}(X)$ |
| **$u$ 的三个要求** | 易计算；$c(X) \leq u(X)$；叶节点答案结点时 $c(X) = u(X)$ |
| **算法8.2 适用范围** | **n-元组**表达时保证最小成本；k-元组不保证 |
| **UB 函数两个动作** | ①界定杀死（返回false）；②更改 U 值（减小上界） |
| **15-谜判定定理** | $\sum\text{LESS}(i) + X$ 为偶数才可达 |
| **货郎担 $\hat{c}$ 公式** | $\sum a_i + l_{X_k} + \sum_{d \in S} l_d$（取各结点最小出边） |
| **作业调度 $\hat{c}$ 公式** | $\sum_{i < m, i \notin S_X} p_i$（m之前未选中的罚款） |
| **作业调度可行性判断** | EDF算法：按截止时间升序执行，检验是否超期 |
| **极大化转极小化** | 取目标函数的相反数作为成本函数 $c$ |
| **算法终止条件** | 活结点表为空，或堆顶 $\hat{c} \geq U$（LCBB）|
| **两种剪枝** | 可行解剪枝（$B$ 函数）+ 最优解剪枝（$\hat{c}$ 与 $U$）|

## 十、易错点汇总

| 易错点 | 正确理解 |
|--------|---------|
| **$\hat{c}$ 与 $u$ 的方向混淆** | $\hat{c}$ 是**下**界估计（$\leq c$）；$u$ 是**上**界估计（$\geq c$）|
| **U 的初值设置** | 初值须 $\geq$ 最小成本（一般取 $\infty$ 或贪心解），不能取得太小 |
| **n-元组 vs k-元组** | n-元组答案结点必为叶结点，算法 8.2 保证最优；k-元组不保证 |
| **UB 函数中等号处理** | ans 有结点时 $\hat{c}=U$ 也杀死；ans 无结点时 $\hat{c}=U$ 通过 |
| **货郎担 $l_d$ 的含义** | $l_d$ 是结点 $d$ **出发**的最小边（非到达），且不关心目标结点 |
| **15-谜中 X 的取值** | 查阴影图（偶数行列格），空格在阴影中 $X=1$，否则 $X=0$ |
| **作业调度 $\hat{c}$ 含义** | 只统计下标 $< m$ 且**未被选中**的作业罚款，不是所有未选的 |
| **FIFO 与 LC 的结果** | 最优解相同，但 LC 通常访问结点更少（更高效）|
