# 6.1 一般方法 — 复习资料

---

## 一、方法适用的问题特点

### 1.1 多阶段决策问题

动态规划适用于解决一类**多阶段决策问题**，其特征为：

- 问题可以被分成**若干个阶段**，每一个阶段都需要作出**决策**
- 第 $i$ 阶段的决策**仅依赖于** $i$ 阶段当前的状态（不依赖于之前是如何到达这个状态的）
- 当每个阶段的决策都确定后，问题得到一个**决策序列**，进而计算出一个**解**

### 1.2 适用于求最优解

动态规划方法适用于解决**求最优解**的多阶段决策问题：

- 问题描述中一定存在**目标函数**
- 求解目标是寻找**最优解和最优决策序列**
- 这类问题也称为**多阶段决策优化问题**，或**组合优化问题**

> **直观理解：** 把一个复杂问题看成一连串需要做出选择的"关卡"，每一关的选择会影响后续状态，而每一关该怎么选，只取决于当前所处的状态，不取决于走到这一步具体经过了哪些路。

## 二、最优性原理（Principle of Optimality）

### 2.1 历史背景

20 世纪 50 年代，贝尔曼（Bellman）等人根据多阶段决策问题的特性，提出了解决这类问题的**"最优性原理"**，从而创建了一种新的算法设计方法——**动态规划**。

### 2.2 原理表述

设决策过程为：

$$S_0 \xrightarrow{x_1} S_1 \xrightarrow{x_2} S_2 \xrightarrow{x_3} \cdots \xrightarrow{x_n} S_n$$

**过程的最优决策序列具有如下性质：** 无论过程的初始状态和初始决策是什么，**其余的决策都必须相对于初始决策所产生的状态构成一个最优决策序列**。

### 2.3 与最优子结构的关系

- 最优性原理表明，**如果原问题包含子问题，那么原问题的部分最优决策序列一定是其子问题的最优决策序列**
- **问题满足最优性原理，则称其具有最优子结构性质**

> 也就是说，"最优性原理"和"最优子结构性质"是**同一件事的两种说法**：前者描述的是决策序列层面的性质，后者是对这一性质的命名。

### 2.4 思考题

**思考1：任意子问题的最优决策序列都是原问题的部分最优决策序列么？**

> 不是。最优性原理只保证：**原问题最优解中涉及到的那个子问题**，其解必须是该子问题的最优解。但反过来，**任意一个**子问题的最优解，不一定能拼接成原问题的最优解的一部分——因为该子问题可能根本不出现在原问题的最优决策路径上。

**思考2：最优性原理的逆否命题是什么？**

> 原命题：若过程整体最优，则其余决策相对于初始决策产生的状态也最优。
> 逆否命题：**若某一阶段之后的决策不是相对于当前状态的最优决策序列，则整个过程一定不是最优的。**
> 这正是证明"最优性原理成立"时常用反证法的理论基础：假设存在某个更优的后续子序列，就能拼出一个比原"最优解"更优的解，从而导出矛盾。

## 三、最优性原理证明思想（通用模板）

证明一个问题满足最优性原理，通常按以下**四个步骤**展开：

$$S_0 \xrightarrow{x_1} S_1 \xrightarrow{x_2} S_2 \xrightarrow{x_3} \cdots \xrightarrow{x_n} S_n$$

| 步骤 | 内容 |
|------|------|
| ① | 确定**最优决策序列的形式**（设 $x_1,x_2,\ldots,x_n$ 是最优决策序列） |
| ② | 确定**初始状态和初始决策**（$S_0$ 和 $x_1$） |
| ③ | 确定**初始决策所产生的状态**（即 $S_1$，把问题转化为以 $S_1$ 为起点的子问题） |
| ④ | **证明其余决策相对于③是最优决策序列**（通常用反证法：假设存在更优的子序列，推出与"原序列最优"矛盾） |

> 这一模板贯穿全书各章节的最优性原理证明（多段图、货郎担、0/1背包、可靠性设计等），是历次证明题的**通用解题框架**。

## 四、以 0/1 背包问题为例的完整证明

### 4.1 问题描述与符号约定

对于 $n$ 个物品、容量为 $M$ 的背包，要求物品**或者整件装入**背包，**或者根本不装入**，即 $x_i$ 限定只能取 $0$ 或 $1$ 值：

$$\text{极大化} \quad \sum_{1\leq i\leq n} p_i x_i$$
$$\text{约束条件} \quad \sum_{1\leq i\leq n} w_i x_i \leq M, \quad x_i=0\text{ 或 }1,\ 1\leq i\leq n$$

用 $\text{KNAP}(l,j,X)$ 表示这个问题，**函数值表示最大效益和**。对于 $n$ 个物品、容量为 $M$ 的背包，原问题可表示为：

$$\text{KNAP}(1,n,M)$$

> $\text{KNAP}(l,j,X)$ 三个参数的含义：$l$ 是物品编号下界，$j$ 是物品编号上界（即考虑第 $l$ 到第 $j$ 件物品），$X$ 是背包容量约束。函数值代表"问题最优解"，参数集代表"问题边界和约束条件"。

### 4.2 完整证明（套用四步模板）

**① 确定最优决策序列的形式：** 设 $x_1,x_2,\ldots,x_n$ 是最优序列。

**②③ 确定初始决策及其产生的状态：** 按 $x_1$ 的取值分两种情况讨论。

**④ 证明其余决策相对于初始决策构成最优序列：**

**情况一：若 $x_1=0$**

则 $x_2,x_3,\ldots,x_n$ **必须**相对于 $\text{KNAP}(2,n,M)$ 问题构成一个最优序列。

> **反证：** 如若不然，则 $x_1,x_2,\ldots,x_n$ 就不是 $\text{KNAP}(1,n,M)$ 的最优序列（因为可以替换为 $\text{KNAP}(2,n,M)$ 的真正最优序列，得到更大的效益值，与 $x_1,\ldots,x_n$ 最优矛盾）。

**情况二：若 $x_1=1$**

则 $x_2,x_3,\ldots,x_n$ **必须**是 $\text{KNAP}(2,n,M-w_1)$ 的最优序列。

> **反证：** 如若不然，则必有另一个 $0/1$ 序列 $z_2,z_3,\ldots,z_n$ 使得 $\sum w_iz_i \leq M-w_1\ (2\leq i\leq n)$ 且 $\sum p_iz_i > \sum p_ix_i\ (2\leq i\leq n)$。因此，序列 $x_1,z_2,z_3,\ldots,z_n$ 是一个对问题 $\text{KNAP}(1,n,M)$ **具有更大效益值**的序列，这与"$x_1,\ldots,x_n$ 是最优序列"的假设相矛盾。

**结论：** 所以 0/1 背包问题的**最优性原理成立**。

### 4.3 思考题：最优性原理如何帮助提高算法效率？

> 最优性原理保证了"求解原问题的最优解，等价于求解某个特定子问题的最优解，再结合当前决策"。这意味着我们**不需要枚举所有可能的决策序列**（指数级数量），而是可以把问题**递归地分解为规模更小的子问题**，通过子问题的最优解直接推出原问题的最优解。再结合"自底向上 + 备忘录"的实现方式（见第六节），相同的子问题只需计算一次，从而把原本指数级的穷举枚举，优化为多项式级（或伪多项式级）的迭代计算。

## 五、递推关系式设计

确定了最优性原理之后，下一步是设计**递推关系式**，把"原问题最优解"用"子问题最优解"表示出来。设计递推关系式有两种方向：

### 5.1 向前处理法（forward approach）

**从最后阶段开始，逐步向前递推**，即根据 $x_{i+1},\ldots,x_n$ 的那些最优决策序列，来列出求取 $x_i$ 决策值的关系式。

### 5.2 向后处理法（backward approach）

**从初始阶段开始，逐步向后递推**，即根据 $x_1,\ldots,x_{i-1}$ 的那些最优决策序列，来列出求取 $x_i$ 决策值的关系式。

> **两种方法的本质区别在于"子问题是如何定义的"：** 向前处理法的子问题是"剩余的尾部决策"，向后处理法的子问题是"已完成的前缀决策"。两者通常都能推出正确的递推式，具体选用哪一种，取决于问题本身用哪种方向定义子问题更自然（参见 6.2 节中多段图的向前/向后处理对比）。

## 六、0/1 背包问题的两种递推关系式

### 6.1 向前处理法的递推关系式

根据最优性原理：

$$\text{KNAP}(1,n,M) = \max\{\text{KNAP}(2,n,M),\ \text{KNAP}(2,n,M-w_1)+p_1\}$$

> 这正是上一节证明中"$x_1=0$"和"$x_1=1$"两种情况分别对应的子问题，取两者中效益更大的一个。

参数 $n$ 在递推过程中不起变化作用，舍弃后可以简化函数。设：

$$g_j(X) = \text{KNAP}(j+1, n, X)$$

则 $g_0(M) = \text{KNAP}(1,n,M)$，上式简化为：

$$g_0(M) = \max\{g_1(M),\ g_1(M-w_1)+p_1\}$$

推知一般的递推关系式：

$$\boxed{g_j(X) = \max\{g_{j+1}(X),\ g_{j+1}(X-w_{j+1})+p_{j+1}\}}$$

**边界条件：**

$$g_n(X) = \begin{cases} -\infty & X<0 \\ 0 & X\geq0 \end{cases}$$

### 6.2 向后处理法的递推关系式

设 $f_i(X) = \text{KNAP}(1,i,X)$，则 $f_n(M)=\text{KNAP}(1,n,M)$。

那么 $f_n(M)$ 的递归关系式可表示为：

$$f_n(M) = \max\{f_{n-1}(M),\ f_{n-1}(M-w_n)+p_n\}$$

对于任意的 $f_i(X)$，$i>0$，有以下递推关系式：

$$\boxed{f_i(X) = \max\{f_{i-1}(X),\ f_{i-1}(X-w_i)+p_i\}}$$

**边界条件：**

$$f_0(X) = \begin{cases} -\infty & X<0 \\ 0 & X\geq0 \end{cases}$$

> 这正是 6.4 节中实际使用的递推关系式——可以看到，**6.4 节直接采用的是向后处理法**的形式。

### 6.3 思考题：递归 vs 迭代

**思考：如何设计算法？根据递推关系式如此顺畅的自顶向下递归求解，为什么还要自底向上迭代求解？**

> 关键原因是**重叠子问题性质**（见下一节）：自顶向下递归求解时，同一个子问题会被**多次重复计算**，造成大量冗余；而自底向上迭代求解，配合"备忘录"记录每个子问题已经算过的结果，可以保证**每个子问题只计算一次**，从而大幅提升效率。

## 七、重叠子问题性质

### 7.1 递归求解的冗余示例

以向后处理递推关系式 $f_i(X) = \max\{f_{i-1}(X),\ f_{i-1}(X-w_i)+p_i\}$ 为例：

**实例：** $n=4$，$(p_1,p_2,p_3,p_4)=(1,2,5,6)$，$(w_1,w_2,w_3,w_4)=(1,1,1,1)$，$M=4$

若设计递归算法 $f(i,X)$，直接按递推式展开调用，会发现调用树中出现大量**重复的子问题调用**：

```
                              (4,4)
                  /                          \
              (3,4)                          (3,3)
           /         \                    /         \
       (2,4)         (2,3)            (2,3)         (2,2)
      /     \       /     \          /     \       /     \
  (1,4)   (1,3)  (1,3)   (1,2)    (1,3)   (1,2)  (1,2)   (1,1)
   / \     / \    / \     / \      / \     / \    / \     / \
(0,4)(0,3)(0,3)(0,2)(0,3)(0,2)(0,2)(0,1) (0,2)(0,1)(0,1)(0,0)
```

> 图中可以清楚看到，**$(2,3)$ 这个子问题被调用了两次**，$(1,3)$、$(1,2)$、$(0,3)$、$(0,2)$ 等也都被重复计算——这就是"重叠子问题"。

### 7.2 重叠子问题性质的定义

**重叠子问题性质：** 求解一个问题时，**同一个子问题被多次重用**。

### 7.3 由重叠子问题性质引出的结论

- **自底向上迭代求解效率更高**：因为可以借助"备忘录"避免重复计算
- **重叠子问题越多，（动态规划相对于直接递归的）效率提升越明显**

## 八、动态规划的设计思想（总结流程）

动态规划方法求解问题时，遵循以下流程：

| 步骤 | 内容 |
|------|------|
| ① **证明最优性原理** | 证明多阶段决策（最优）问题满足最优性原理，**刻画问题规模**（确定状态如何表示） |
| ② **设计递推关系式** | 设计函数表达形式：**函数值代表问题最优解**；**参数集代表问题边界和约束条件**；根据最优性原理确定原问题和子问题的关系 |
| ③ **自底向上方式设计程序** | 从最小的子问题开始迭代计算，直到达到原始问题规模；计算过程中保存所有子问题的最优解和最优决策（即**备忘录**和**标记函数**） |

> 针对一些问题，动态规划法比蛮力法（穷举）在时间复杂度上得到了实质的改进。

---

## 九、动态规划的迭代实现

### 9.1 自底向上计算子问题

- **每个子问题只计算一次**：先计算出所有规模为 1 的子问题，再计算所有规模为 2 的子问题，……，直到规模 $n$ 为止
- 这种"按规模从小到大"的计算顺序，正是 6.2～6.8 各节"求解顺序"部分的共同模式（如 6.3 节按 $|S|$ 从小到大）

### 9.2 备忘录与标记函数

- **备忘录：** 定义一个存储空间（通常是表格形式），保存**每个子问题的最优解**（和最优决策）的表格
- **标记函数：** 与备忘录配合，记录每个子问题对应的**最优决策**（用于事后回溯出完整的决策序列）
- **重用子问题时，直接查询备忘录**，避免重复计算

## 十、动态规划的特点

### 10.1 两大性质

| 性质 | 说明 |
|------|------|
| **最优子结构性质** | 满足最优性原理 |
| **重叠子问题性质** | 同一子问题被多次重用 |

### 10.2 优点

- 设计思想**优于蛮力法**（不需要穷举所有决策序列）
- 实现方法**优于递归法**（避免重叠子问题的重复计算）

### 10.3 缺点

- 如果**子问题重叠程度低**，动态规划的时间复杂度改进不明显
- **过大的空间需求**成为动态规划算法效率低下的主要因素（备忘录需要保存所有子问题的解，可能占用大量存储空间）

## 十一、易错点汇总

| 易错点 | 正确理解 |
|--------|---------|
| **最优性原理 ≠ 贪心选择性质** | 最优性原理（最优子结构）是动态规划的必要条件；贪心法还需要额外的"贪心选择性质"，两者不可混淆（见 6.5 节TSP的反例：TSP有最优子结构但不能用贪心） |
| **任意子问题最优解可拼接成原问题最优解** | 错误。只有"原问题最优解路径上经过的"那个子问题，其解才必须是子问题最优解；其余不在最优路径上的子问题最优解不一定能用上 |
| **向前/向后处理法的子问题定义方向不同** | 向前处理：子问题是"剩余尾部决策"；向后处理：子问题是"已完成前缀决策"，两者是对偶关系，不要混淆方向 |
| **递归与动态规划的区别** | 递归（自顶向下）和动态规划（自底向上+备忘录）使用的是**同一个**递推关系式，区别仅在于**实现方式**：是否利用备忘录避免重叠子问题的重复计算 |
| **重叠子问题不是动态规划的必要条件** | 即使没有重叠子问题（如普通分治法），递推关系式依然可以成立；但只有**存在大量重叠子问题**时，动态规划相对于直接递归才有显著的效率优势 |

## 十二、考点速查

| 考点 | 关键内容 |
|------|---------|
| **适用问题特点** | 多阶段决策问题：可分阶段，每阶段决策仅依赖当前状态，存在目标函数，求最优解 |
| **最优性原理** | 无论初始状态和初始决策是什么，其余决策都必须相对于初始决策产生的状态构成最优决策序列 |
| **最优性原理 = 最优子结构性质** | 两者是同一性质的不同表述 |
| **证明四步法** | ①确定最优序列形式 ②确定初始状态和决策 ③确定初始决策产生的状态 ④证明其余决策相对于③最优（反证法） |
| **0/1背包最优性原理证明** | 分 $x_1=0$ 和 $x_1=1$ 两种情况，各自用反证法证明 |
| **向前处理法** | 子问题 = 剩余尾部决策，从最后阶段向前递推 |
| **向后处理法** | 子问题 = 已完成前缀决策，从初始阶段向后递推 |
| **重叠子问题性质** | 同一子问题被多次重用；自底向上迭代+备忘录可避免重复计算 |
| **动态规划设计三步骤** | ①证明最优性原理（刻画问题规模） ②设计递推关系式（函数值=最优解，参数=边界约束） ③自底向上+备忘录实现 |
| **动态规划两大性质** | 最优子结构性质 + 重叠子问题性质 |
| **优缺点** | 优于蛮力法和递归法；但子问题重叠少则提升不明显，空间需求可能过大 |
