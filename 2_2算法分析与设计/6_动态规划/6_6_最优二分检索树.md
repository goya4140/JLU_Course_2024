## 一、背景知识：二分检索树

### 1.1 定义

**二分检索树（Binary Search Tree, BST）** 是一棵二叉树，满足：

- 左子树所有元素 **小于** 根结点元素
- 右子树所有元素 **大于** 根结点元素
- 左、右子树也分别是二分检索树
- 所有结点元素**互异**

### 1.2 检索算法

```
procedure SEARCH(T, X, i)
  i ← T
  while i ≠ 0 do
    case
      : X < IDENT(i) : i ← LCHILD(i)     // 往左子树走
      : X = IDENT(i) : return             // 检索成功
      : X > IDENT(i) : i ← RCHILD(i)     // 往右子树走
    endcase
  repeat
  // 循环结束 i=0，表示检索失败
end SEARCH
```

**检索成本（比较次数）：**

- **成功检索**：在内结点（$l$ 级）处终止，循环执行 **$l$ 次**
- **不成功检索**：在外结点（$l$ 级）处终止，循环执行 **$l-1$ 次**

> 📌 **由此引出问题：** 同一组标识符，可以构造出**多种不同形状**的二分检索树（根结点选哪个、左右子树怎么分都不同）。形状不同，检索的平均比较次数也不同。如果我们知道每个标识符被检索的概率，自然会问：**哪种树形让平均检索成本最低？**

## 二、问题描述

### 2.1 结点设置

给定有序标识符集 $(a_1, a_2, \ldots, a_n)$，$a_1 < a_2 < \cdots < a_n$：

| 结点类型 | 符号 | 数量 | 含义 |
|---|---|---|---|
| **内结点（圆结点）** | $a_i$ | $n$ 个 | 成功检索（$X = a_i$） |
| **外结点（方结点）** | $E_i$ | $n+1$ 个 | 不成功检索（$a_i < X < a_{i+1}$） |

> 外结点 $E_0$ 表示 $X < a_1$，$E_n$ 表示 $X > a_n$。

### 2.2 概率设置

- $P(i)$：对 $X = a_i$ 成功检索的概率（$1 \leq i \leq n$）
- $Q(i)$：第 $i$ 个间隔不成功检索的概率（$0 \leq i \leq n$）
- 满足：$\sum_{1 \leq i \leq n} P(i) + \sum_{0 \leq i \leq n} Q(i) = 1$

### 2.3 预期成本公式（给树形"计分"的方法）

$$\text{cost}(T) = \sum_{1 \leq i \leq n} P(i) \cdot \text{level}(a_i) + \sum_{0 \leq i \leq n} Q(i) \cdot (\text{level}(E_i) - 1)$$

**目标：** 在所有可能的二分检索树中，找到使预期成本**最小**的那棵，即**最优二分检索树**。

### 2.4 为什么不能用贪心？（$n=3$ 的对比实验）

设 $(a_1, a_2, a_3) = (\text{do, if, stop})$，比较两种概率分布下的最优树形：

- **情景1**（等概率 $P(i)=Q(i)=1/7$）：以 $a_2$ 为根最优，成本 $= 13/7$
- **情景2**（$P=(0.5, 0.1, 0.05)$，$Q=(0.15,0.1,0.05,0.05)$）：以 $a_1$ 为根最优，成本 $= 1.5$

**关键观察：** 情景2 中 $a_1$ 的访问概率最高（$P(1)=0.5$），但最优树并不总是"把概率最大的放根结点"这么简单——根的选择还要综合考虑它对**所有其他结点层数**的连带影响。

> 📌 **结论：** 局部看起来"最该放根"的标识符未必是全局最优的根，**贪心策略不可靠**，必须枚举/递推所有可能的根，即采用动态规划。

## 三、给递推式的下标体系"热身"：先看一个最小例子

在写出正式递推式之前，先固定一套记号习惯，避免后面被下标绕晕。

### 3.1 约定：用 $C(i,j)$ 表示哪一段标识符

我们规定：

$$C(i,j) = \text{由内结点 } a_{i+1},\ldots,a_j \text{ 和外结点 } E_i,\ldots,E_j \text{ 构成的最优子树的成本}$$

**为什么内结点从 $a_{i+1}$ 开始而不是 $a_i$？** 因为外结点编号是 $0\sim n$（共 $n+1$ 个），内结点编号是 $1\sim n$（共 $n$ 个），两者数量差1。若让 $C(i,j)$ 覆盖外结点 $E_i,\ldots,E_j$（共 $j-i+1$ 个），则它夹着的内结点自然就是 $a_{i+1},\ldots,a_j$（共 $j-i$ 个）——**外结点比内结点多一个，下标错位正是为了让两者一一对应、首尾各挂一个外结点**。

### 3.2 最小例子：$n=1$（只有一个标识符 $a_1$）

此时只有一种树形：$a_1$ 做根，左孩子 $E_0$，右孩子 $E_1$。

$$C(0,1) = P(1)\cdot 1 + Q(0)\cdot 0 + Q(1)\cdot 0 = P(1)$$

（$a_1$ 在第1层，$\text{level}=1$；外结点在第2层，$\text{level}-1=1-1=0$，故外结点项系数为 0，整体退化为单纯的 $P(1)$。这与"空树成本为0、加入一个内结点后成本恰为其概率"的直觉一致。）

### 3.3 这给我们的启示

把"由两个外结点夹着的空隙"记作 $C(i,i)$（不含任何内结点），它的成本应为：

$$C(i,i) = 0$$

这就是后续递推的**初始条件**——下标体系热身完毕，下面可以放心地处理一般情况。

## 四、合并子树时，额外代价从哪里来？（$W$ 函数的来源）

### 4.1 直觉：把两棵独立的树接到新根下面，层数都要 +1

设当前树 $T$ 以 $a_k$ 为根，左子树 $L$ 单独存在时覆盖 $a_1,\ldots,a_{k-1}$ 和 $E_0,\ldots,E_{k-1}$，右子树 $R$ 单独存在时覆盖 $a_{k+1},\ldots,a_n$ 和 $E_k,\ldots,E_n$。

**关键事实：** $L$、$R$ 单独成树时，根在第1层；但接到 $a_k$ 下面变成子树后，原来的根变成第2层，**$L$、$R$ 内部所有结点的层数都比原来多1**。

由于预期成本公式中每个结点的"层数"都要乘以它的访问概率，层数集体 +1，意味着 $L$ 和 $R$ 各自原有成本之外，还要**额外**加上"$L$（或 $R$）中所有结点的概率之和"——因为每个结点多走一层，就多产生一次该结点概率大小的代价。

### 4.2 定义权重函数 $W(i,j)$

把"$C(i,j)$ 这一段所有结点（内结点+外结点）的概率之和"定义为：

$$\boxed{W(i, j) = Q(i) + \sum_{l=i+1}^{j} \big(Q(l) + P(l)\big)}$$

这正是 4.1 节中"接到新根下面要额外付出的代价"。

### 4.3 子树成本的合并公式

$$\text{cost}(T) = P(k) + \text{cost}(L) + \text{cost}(R) + W(0, k-1) + W(k, n)$$

- $P(k)$：根结点 $a_k$ 自身的成本（层数为1）
- $\text{cost}(L), \text{cost}(R)$：左右子树**单独成树**时的最优成本
- $W(0,k-1), W(k,n)$：左右子树**因为多了一层**而额外产生的代价

### 4.4 $W$ 的递推（方便逐步计算）

直接按定义累加效率不高，可以递推：

$$W(i,j) = W(i, j-1) + P(j) + Q(j)$$

（在 $W(i,j-1)$ 基础上，多了一个内结点 $a_j$ 和一个外结点 $E_j$）

**初始值：** $W(i,i) = Q(i)$（区间内没有内结点，只有一个外结点 $E_i$）

## 五、最优性原理与递推关系式

### 5.1 最优性原理

**命题：** 若树 $T$ 是包含 $a_1, \ldots, a_n$ 的最优二分检索树，则其左子树 $L$ 和右子树 $R$ 分别是对应子集的最优二分检索树。

**证明（反证法）：** 若 $L$ 不是最优的，设 $L'$ 是 $L$ 对应子集的最优树且 $\text{cost}(L') < \text{cost}(L)$，则将 $L$ 替换为 $L'$ 后得到的新树成本更小，与 $T$ 最优矛盾。故满足最优性原理。✓

### 5.2 递推关系式（核心）——水到渠成

有了 4.3 节的合并公式、5.1 节的最优性原理，只需把"选哪个 $k$ 做根"这一层决策加上 $\min$，就得到完整递推式：

$$\boxed{C(i,j) = \min_{i < k \leq j} \{ C(i, k-1) + C(k, j) \} + W(i, j)}$$

**初始条件（$j-i=0$，空树）：**

$$C(i,i) = 0, \quad W(i,i) = Q(i), \quad 0 \leq i \leq n$$

**标记函数：** $R(i,j) = $ 使 $C(i,j)$ 取最小值时的 $k$ 值（即最优根结点编号）

> ⚠️ **下标含义回顾**（第三节已热身，这里再次确认）：
> - $C(i,j)$ 覆盖的内结点是 $a_{i+1}, \ldots, a_j$（**不含** $a_i$）
> - $C(i,j)$ 覆盖的外结点是 $E_i, \ldots, E_j$
> - 整棵树的最优成本是 $C(0,n)$
> - $k$ 的范围是 $i < k \leq j$（左开右闭）：若 $k=i$，则左子树对应 $C(i,i-1)$ 没有意义，所以 $k$ 不能取到 $i$

## 六、计算过程（课件完整实例）

$$P(1:4) = (3, 3, 1, 1), \quad Q(0:4) = (2, 3, 1, 1, 1)$$

## 七、朴素算法有多慢？为什么值得优化？

### 7.1 朴素算法：$O(n^3)$

计算 $C(i,j)$ 时，遍历所有 $k \in (i, j]$：

- $j-i=m$ 时，每个 $C(i,j)$ 需要 $O(m)$ 时间
- 共有 $n-m+1$ 个这样的 $C(i,j)$，总时间 $O(m(n-m+1))$
- 对 $m=1,2,\ldots,n$ 求和：

$$\sum_{m=1}^{n} m(n-m+1) = O(n^3)$$

### 7.2 观察实例中的 $R$ 表，寻找优化的线索

回看第六节的 $R(i,j)$ 表，逐行逐列看一下数字变化：

- 固定 $i=0$ 这一行：$R(0,1)=1,\ R(0,2)=1,\ R(0,3)=2,\ R(0,4)=2$ —— **随 $j$ 增大，根的编号不减**
- 固定 $j=4$ 这一列：$R(1,4)=2,\ R(2,4)=3,\ R(3,4)=4$ —— **随 $i$ 增大，根的编号也不减**

**这绝非偶然！** 这意味着最优根的位置具有某种"单调性"，如果能证明这一规律对所有实例都成立，计算 $C(i,j)$ 时就不必把 $k$ 的整个区间 $(i,j]$ 都试一遍，只需在一个**更小的范围**里搜索——这正是 Knuth 优化的出发点。

---

## 八、Knuth 优化：$O(n^2)$

### 8.1 单调性定理

**关键定理（Knuth，1971）：**

$$R(i, j-1) \leq R(i, j) \leq R(i+1, j)$$

即最优根结点的位置随着区间扩张单调不减：$R(i,j)$ 的搜索范围可以从原来的 $(i, j]$ **缩小**到 $[R(i,j-1),\ R(i+1,j)]$。

**直觉解释：** $R(i,j-1)$ 是"去掉右端一个结点"时的最优根，$R(i+1,j)$ 是"去掉左端一个结点"时的最优根；区间从两端收缩或扩张时，最优根的移动也是有方向、不会"反弹"的，所以真正的 $R(i,j)$ 必然落在这两者之间。

### 8.2 优化后的递推式

$$\boxed{C(i,j) = \min_{R(i,j-1) \leq k \leq R(i+1,j)} \{ C(i, k-1) + C(k, j) \} + W(i, j)}$$

### 8.3 时间复杂度证明（对应作业题6）

对固定的 $i$，$k$ 的搜索区间长度之和为：
$$\sum_{j=i+1}^{n} \big( R(i+1,j) - R(i,j-1) \big) = R(i+1,n) - R(i,i) \leq n$$

这是一个**望远镜求和**（telescoping sum）——绝大多数中间项相互抵消，只剩首尾两项。

故对所有 $i$ 求和：$\sum_{i=0}^{n-1} O(n) = O(n^2)$，总比较次数降至 $O(n^2)$。

## 九、算法描述（OBST）

```
procedure OBST(P, Q, n)
  // 初始化 j-i=0 和 j-i=1 两种情况
  for i ← 0 to n-1 do
    W(i,i) ← Q(i);  R(i,i) ← 0;  C(i,i) ← 0
    W(i,i+1) ← Q(i)+Q(i+1)+P(i+1)
    R(i,i+1) ← i+1
    C(i,i+1) ← W(i,i+1)
  repeat
  W(n,n) ← Q(n);  R(n,n) ← 0;  C(n,n) ← 0

  // 自底向上，按 j-i=m=2,3,...,n 逐层计算
  for m ← 2 to n do
    for i ← 0 to n-m do
      j ← i + m
      W(i,j) ← W(i,j-1) + P(j) + Q(j)
      // 在 Knuth 优化范围内找最优 k
      k ← 在 [R(i,j-1), R(i+1,j)] 中使 {C(i,l-1)+C(l,j)} 最小的 l
      C(i,j) ← C(i,k-1) + C(k,j) + W(i,j)
      R(i,j) ← k
    repeat
  repeat
end OBST
```

**复杂度对比：**

| 版本 | 时间复杂度 | 空间复杂度 |
|---|---|---|
| 朴素动态规划 | $O(n^3)$ | $O(n^2)$ |
| Knuth 优化 | $O(n^2)$ | $O(n^2)$ |

### 作业题6（Knuth 优化 $O(n^2)$ 证明）

> 构造最优二分检索树的优化算法，把根结点 $k$ 的检索区间限制在 $R(i,j-1) \leq k \leq R(i+1,j)$，证明此时算法计算时间为 $O(n^2)$。

证明思路即第八节8.3，核心是望远镜求和：

$$\sum_{j=i+1}^{n} \big[R(i+1,j) - R(i,j-1) + 1\big] = R(i+1, n) - R(i, i) + (n - i) = O(n)$$

对所有 $i$ 求和：$\sum_{i=0}^{n-1} O(n) = O(n^2)$。

## 十二、易错点汇总

1. **$C(i,j)$ 的范围是 $a_{i+1}, \ldots, a_j$**，内结点编号从 $i+1$ 开始，不是从 $i$ 开始。$C(0,n)$ 才是完整的 $n$ 个标识符的最优树。这一错位编号的来源见第三节——是为了让数量多1的外结点和内结点对齐。

2. **$k$ 的遍历范围是 $i < k \leq j$**（左开右闭），不是 $i \leq k \leq j$，因为若 $k=i$ 则左子树为空（$C(i,i-1)$ 无意义）。

3. **$W(i,j)$ 的递推**是 $W(i,j) = W(i,j-1) + P(j) + Q(j)$，新增的是内结点 $a_j$ 和外结点 $E_j$。注意 $W$ 衡量的是"一段区间内所有结点的概率总和"，本质上和层数无关，只在合并子树时因为层数+1才"借用"它来表示额外代价。

4. **外结点的检索成本**是 $Q(i) \cdot (\text{level}(E_i) - 1)$，注意是层数**减1**，因为不成功检索在外结点处终止时循环已经结束，不计最后一级的比较。

5. **回溯最优树结构**：用 $R$ 表自顶向下递归。$R(0,n)$ 是根，$R(0, R(0,n)-1)$ 是左子树的根，$R(R(0,n), n)$ 是右子树的根，以此类推。

6. **Knuth 优化的边界初始值**：需要单独初始化 $j-i=1$ 的情况（$R(i,i+1) = i+1$），这是 $j-i \geq 2$ 递推的基础。

7. **Knuth 优化不是凭空冒出来的**：它来自对 $R$ 表数据的观察（单调性），不是孤立的数学技巧。记忆时建议把"观察单调性 → 证明定理 → 缩小搜索区间"这条逻辑链一起记，比单独硬记不等式更不容易遗忘。

## 十三、考点速查

| 考点 | 核心内容 |
|---|---|
| 预期成本公式 | $\sum P(i)\cdot\text{level}(a_i) + \sum Q(i)\cdot(\text{level}(E_i)-1)$ |
| 下标体系 | $C(i,j)$ 覆盖内结点 $a_{i+1}\ldots a_j$，外结点 $E_i\ldots E_j$ |
| 权重函数 $W$ 的来源 | 子树合并入新根时，所有结点层数 +1 产生的额外代价 |
| $W$ 递推 | $W(i,j)=W(i,j-1)+P(j)+Q(j)$ |
| 递推关系式 | $C(i,j) = \min_{i<k\leq j}\{C(i,k-1)+C(k,j)\}+W(i,j)$ |
| 初始条件 | $C(i,i)=0$，$W(i,i)=Q(i)$ |
| 朴素复杂度 | $O(n^3)$ 时间，$O(n^2)$ 空间 |
| 优化线索 | 观察 $R$ 表呈现的单调性 |
| Knuth 优化条件 | $R(i,j-1)\leq k\leq R(i+1,j)$ |
| Knuth 优化复杂度 | $O(n^2)$ 时间，$O(n^2)$ 空间 |
| 树结构回溯 | 递归用 $R$ 表，$R(i,j)$ 为子树根的编号 |
