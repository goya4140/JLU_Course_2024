# 6.6 最优二分检索树（OBST）— 复习资料

---

## 一、背景知识：二分检索树

### 1.1 定义

**二分检索树（Binary Search Tree, BST）** 是一棵二叉树，满足：

- 左子树所有元素 **小于** 根结点元素
- 右子树所有元素 **大于** 根结点元素
- 左、右子树也分别是二分检索树
- 所有结点元素**互异**

### 1.2 检索算法

```
procedure SEARCH(T, X, i)
  i ← T
  while i ≠ 0 do
    case
      : X < IDENT(i) : i ← LCHILD(i)     // 往左子树走
      : X = IDENT(i) : return             // 检索成功
      : X > IDENT(i) : i ← RCHILD(i)     // 往右子树走
    endcase
  repeat
  // 循环结束 i=0，表示检索失败
end SEARCH
```

**检索成本（比较次数）：**

- **成功检索**：在内结点（$l$ 级）处终止，循环执行 **$l$ 次**
- **不成功检索**：在外结点（$l$ 级）处终止，循环执行 **$l-1$ 次**

## 二、问题描述

### 2.1 结点设置

给定有序标识符集 $(a_1, a_2, \ldots, a_n)$，$a_1 < a_2 < \cdots < a_n$：

| 结点类型 | 符号 | 数量 | 含义 |
|---|---|---|---|
| **内结点（圆结点）** | $a_i$ | $n$ 个 | 成功检索（$X = a_i$） |
| **外结点（方结点）** | $E_i$ | $n+1$ 个 | 不成功检索（$a_i < X < a_{i+1}$） |

> 外结点 $E_0$ 表示 $X < a_1$，$E_n$ 表示 $X > a_n$。

### 2.2 概率设置

- $P(i)$：对 $X = a_i$ 成功检索的概率（$1 \leq i \leq n$）
- $Q(i)$：第 $i$ 个间隔不成功检索的概率（$0 \leq i \leq n$）
- 满足：$\sum_{1 \leq i \leq n} P(i) + \sum_{0 \leq i \leq n} Q(i) = 1$

### 2.3 预期成本公式

$$\text{cost}(T) = \sum_{1 \leq i \leq n} P(i) \cdot \text{level}(a_i) + \sum_{0 \leq i \leq n} Q(i) \cdot (\text{level}(E_i) - 1)$$

**目标：** 在所有可能的二分检索树中，找到使预期成本**最小**的那棵，即**最优二分检索树**。

### 2.4 直观示例（$n=3$ 的5种树形）

设 $(a_1, a_2, a_3) = (\text{do, if, stop})$：

- **情景1**（等概率 $P(i)=Q(i)=1/7$）：图b（以 $a_2$ 为根）最优，成本 $= 13/7$
- **情景2**（$P=(0.5, 0.1, 0.05)$，$Q=(0.15,0.1,0.05,0.05)$）：图c（以 $a_1$ 为根）最优，成本 $= 1.5$

> 📌 **启示：** 同一组标识符，概率分布不同，最优树形可能完全不同。贪心策略（概率最大的做根）不能保证全局最优，必须使用动态规划。

## 三、递推关系式推导

### 3.1 子树的预期成本分析

设当前树 $T$ 以 $a_k$ 为根，左子树 $L$ 包含 $a_1, \ldots, a_{k-1}$ 和 $E_0, \ldots, E_{k-1}$，右子树 $R$ 包含 $a_{k+1}, \ldots, a_n$ 和 $E_k, \ldots, E_n$：

$$\text{cost}(T) = P(k) + \text{cost}(L) + \text{cost}(R) + W(0, k-1) + W(k, n)$$

其中 $W(0,k-1)$ 和 $W(k,n)$ 是因为 $L$ 和 $R$ 中所有结点的层数在合并进 $T$ 时各增加了1。

### 3.2 权重函数 $W(i,j)$

$$\boxed{W(i, j) = Q(i) + \sum_{l=i+1}^{j} \big(Q(l) + P(l)\big)}$$

递推计算：$W(i,j) = W(i, j-1) + P(j) + Q(j)$

**初始值：** $W(i,i) = Q(i)$

### 3.3 最优性原理

**命题：** 若树 $T$ 是包含 $a_1, \ldots, a_n$ 的最优二分检索树，则其左子树 $L$ 和右子树 $R$ 分别是对应子集的最优二分检索树。

**证明（反证法）：** 若 $L$ 不是最优的，设 $L'$ 是 $L$ 对应子集的最优树且 $\text{cost}(L') < \text{cost}(L)$，则将 $L$ 替换为 $L'$ 后得到的新树成本更小，与 $T$ 最优矛盾。故满足最优性原理。✓

### 3.4 递推关系式（核心）

设 $C(i,j)$ 为包含 $a_{i+1}, \ldots, a_j$ 和 $E_i, \ldots, E_j$ 的最优二分检索树的成本：

$$\boxed{C(i,j) = \min_{i < k \leq j} \{ C(i, k-1) + C(k, j) \} + W(i, j)}$$

**初始条件（$j-i=0$，空树）：**

$$C(i,i) = 0, \quad W(i,i) = Q(i), \quad 0 \leq i \leq n$$

**标记函数：** $R(i,j) = $ 使 $C(i,j)$ 取最小值时的 $k$ 值（即最优根结点）

> ⚠️ **下标含义：**
> - $C(i,j)$ 覆盖的内结点是 $a_{i+1}, \ldots, a_j$（**不含** $a_i$）
> - $C(i,j)$ 覆盖的外结点是 $E_i, \ldots, E_j$
> - 整棵树的最优成本是 $C(0,n)$

## 四、计算过程（课件完整实例）

$$P(1:4) = (3, 3, 1, 1), \quad Q(0:4) = (2, 3, 1, 1, 1)$$

（注：此处概率值未归一化，作为权重使用，结论不变）

### 第(0)步：初始化（$j-i=0$）

$$C(i,i) = 0, \quad W(i,i) = Q(i), \quad R(i,i) = 0 \quad (0 \leq i \leq 4)$$

$$W(0,0)=2,\ W(1,1)=3,\ W(2,2)=1,\ W(3,3)=1,\ W(4,4)=1$$

### 第(1)步：$j-i=1$（单个内结点）

每次只有一个 $k$ 可选，根必为 $a_{i+1}$。

| $i,j$ | $W(i,j)$ | 计算 | $C(i,j)$ | $R(i,j)$ |
|---|---|---|---|---|
| $C(0,1)$ | $Q(0)+P(1)+Q(1) = 2+3+3=8$ | $C(0,0)+C(1,1)+W = 0+0+8$ | **8** | 1 |
| $C(1,2)$ | $3+3+1=7$ | $0+0+7$ | **7** | 2 |
| $C(2,3)$ | $1+1+1=3$ | $0+0+3$ | **3** | 3 |

### 第(2)步：$j-i=2$（比较2个候选根）

**$C(0,2)$：** $W(0,2)=W(0,1)+P(2)+Q(2)=8+3+1=12$

$$k=1: C(0,0)+C(1,2) = 0+7 = 7$$
$$k=2: C(0,1)+C(2,2) = 8+0 = 8$$

**$C(1,3)$：** $W(1,3)=W(1,2)+P(3)+Q(3)=7+1+1=9$

## 五、时间复杂度分析

### 5.1 朴素算法：$O(n^3)$

计算 $C(i,j)$ 时，遍历所有 $k \in (i, j]$：

- $j-i=m$ 时，每个 $C(i,j)$ 需要 $O(m)$ 时间
- 共有 $n-m+1$ 个这样的 $C(i,j)$，总时间 $O(m(n-m+1))$
- 对 $m=1,2,\ldots,n$ 求和：

$$\sum_{m=1}^{n} m(n-m+1) = O(n^3)$$

### 5.2 Knuth 优化：$O(n^2)$

**关键定理（Knuth，1971）：**

$$R(i, j-1) \leq R(i, j) \leq R(i+1, j)$$

即最优根结点具有**单调性**：$R(i,j)$ 的搜索范围可从 $(i, j]$ 缩小到 $[R(i,j-1),\ R(i+1,j)]$。

**优化后的递推式：**

$$\boxed{C(i,j) = \min_{R(i,j-1) \leq k \leq R(i+1,j)} \{ C(i, k-1) + C(k, j) \} + W(i, j)}$$

**时间复杂度证明（作业题6）：**

对固定的 $i$，$k$ 的搜索上界之和为：
$$\sum_{j=i+1}^{n} \big( R(i+1,j) - R(i,j-1) \big) = R(i+1,n) - R(i,i) \leq n$$

故对所有 $i$ 求和，总比较次数为 $O(n^2)$，计算时间降至 $O(n^2)$。

## 六、算法描述（OBST）

```
procedure OBST(P, Q, n)
  // 初始化 j-i=0 和 j-i=1 两种情况
  for i ← 0 to n-1 do
    W(i,i) ← Q(i);  R(i,i) ← 0;  C(i,i) ← 0
    W(i,i+1) ← Q(i)+Q(i+1)+P(i+1)
    R(i,i+1) ← i+1
    C(i,i+1) ← W(i,i+1)
  repeat
  W(n,n) ← Q(n);  R(n,n) ← 0;  C(n,n) ← 0

  // 自底向上，按 j-i=m=2,3,...,n 逐层计算
  for m ← 2 to n do
    for i ← 0 to n-m do
      j ← i + m
      W(i,j) ← W(i,j-1) + P(j) + Q(j)
      // 在 Knuth 优化范围内找最优 k
      k ← 在 [R(i,j-1), R(i+1,j)] 中使 {C(i,l-1)+C(l,j)} 最小的 l
      C(i,j) ← C(i,k-1) + C(k,j) + W(i,j)
      R(i,j) ← k
    repeat
  repeat
end OBST
```

**复杂度对比：**

| 版本 | 时间复杂度 | 空间复杂度 |
|---|---|---|
| 朴素动态规划 | $O(n^3)$ | $O(n^2)$ |
| Knuth 优化 | $O(n^2)$ | $O(n^2)$ |

### 作业题6（Knuth 优化 $O(n^2)$ 证明）

> 构造最优二分检索树的优化算法，把根结点 $k$ 的检索区间限制在 $R(i,j-1) \leq k \leq R(i+1,j)$，证明此时算法计算时间为 $O(n^2)$。

**证明思路：**

固定 $i$，对 $j$ 从 $i+1$ 到 $n$ 累计搜索长度：

$$\sum_{j=i+1}^{n} \big[R(i+1,j) - R(i,j-1) + 1\big]$$

这是一个**望远镜求和**（telescoping sum），$R(i,j-1)$ 和 $R(i+1,j)$ 的下标"错位"，大量项相消：

$$= R(i+1, n) - R(i, i) + (n - i) = O(n)$$

对所有 $i$ 求和：$\sum_{i=0}^{n-1} O(n) = O(n^2)$。

## 八、易错点汇总

1. **$C(i,j)$ 的范围是 $a_{i+1}, \ldots, a_j$**，内结点编号从 $i+1$ 开始，不是从 $i$ 开始。$C(0,n)$ 才是完整的 $n$ 个标识符的最优树。

2. **$k$ 的遍历范围是 $i < k \leq j$**（左开右闭），不是 $i \leq k \leq j$，因为若 $k=i$ 则左子树为空（$C(i,i-1)$ 无意义）。

3. **$W(i,j)$ 的递推**是 $W(i,j) = W(i,j-1) + P(j) + Q(j)$，新增的是内结点 $a_j$ 和外结点 $E_j$。

4. **外结点的检索成本**是 $Q(i) \cdot (\text{level}(E_i) - 1)$，注意是层数**减1**，因为不成功检索在外结点处终止时循环已经结束，不计最后一级的比较。

5. **回溯最优树结构**：用 $R$ 表自顶向下递归。$R(0,n)$ 是根，$R(0, R(0,n)-1)$ 是左子树的根，$R(R(0,n), n)$ 是右子树的根，以此类推。

6. **Knuth 优化的边界初始值**：需要单独初始化 $j-i=1$ 的情况（$R(i,i+1) = i+1$），这是 $j-i \geq 2$ 递推的基础。

## 九、考点速查

| 考点 | 核心内容 |
|---|---|
| 预期成本公式 | $\sum P(i)\cdot\text{level}(a_i) + \sum Q(i)\cdot(\text{level}(E_i)-1)$ |
| 递推关系式 | $C(i,j) = \min_{i<k\leq j}\{C(i,k-1)+C(k,j)\}+W(i,j)$ |
| 初始条件 | $C(i,i)=0$，$W(i,i)=Q(i)$ |

| Knuth 优化条件 | $R(i,j-1)\leq k\leq R(i+1,j)$ |

| 树结构回溯 | 递归用 $R$ 表，$R(i,j)$ 为子树根的编号 |
