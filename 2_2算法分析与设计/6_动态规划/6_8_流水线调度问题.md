# 6.8 流水线调度问题 — 复习资料（讲解逻辑重构版）

> **本节讲解主线：** 问题有多难 → 状态怎么定义 → 递推式从何而来 → 递推式太复杂，能否化简 → 化简的目标是什么 → 数学推演 → 推出排序规则 → 用规则求解实例 → 验证规则

---

## 一、问题描述

### 1.1 基本设定

- 有 $n$ 个**作业**，$m$ 台**设备**（$P_1, P_2, \ldots, P_m$）
- 每个作业 $i$ 须依次执行 $m$ 个任务：$T_{1i}, T_{2i}, \ldots, T_{mi}$
- 任务 $T_{ji}$ **只能在设备 $P_j$ 上执行**，耗时 $t_{ji}$
- **顺序约束：** 同一作业内，$T_{j-1,i}$ 完成前不能开始 $T_{ji}$
- **设备约束：** 同一台设备同一时刻只能处理一个任务

### 1.2 目标

确定调度方案 $S$（每台设备上任务的执行顺序），使所有作业的**最大完成时间最小**：

$$F(S) = \max_{1 \leq i \leq n} f_i(S) \ \longrightarrow\ \min$$

### 1.3 调度类型与问题难度

| 调度类型 | 含义 |
|---|---|
| **非抢先调度** | 任务一旦开始不能中断 |
| **抢先调度** | 允许任务被中断后恢复 |

**关键事实：** 当设备数 $m > 2$ 时，无论哪种调度方式都是**难于计算的问题**（NP-Hard）。

> 📌 这就是为什么本节只能处理一个**特殊情形**：$m=2$ 的非抢先调度，记作 **OFT（Optimal Flowshop Time）问题**。只有缩小到这个特例，动态规划才能给出多项式时间的解法。

---

## 二、$m=2$ 时问题为什么"看起来"很难

### 2.1 符号简化

$m=2$ 时每个作业只有两个任务，记：

$$a_i = t_{1i}\ (\text{在 } P_1\text{ 上耗时}), \qquad b_i = t_{2i}\ (\text{在 } P_2\text{ 上耗时})$$

并假定 $a_i \neq 0$（$a_i=0$ 的作业可单独处理，见第七节末尾说明）。

**调度的自由度：** 由于每个任务都应尽早开始执行（这是非抢先调度下的显然最优策略），**整个调度方案完全由 $P_1$ 上 $n$ 个作业的执行顺序决定**——一旦 $P_1$ 的顺序确定，$P_2$ 上各任务的开始时间也随之唯一确定。

### 2.2 朴素思路的代价：$O(n!)$

于是问题等价于：在 $n!$ 种作业排列中，找出使 $F(S)$ 最小的那个排列。**直接穷举不可行。**

### 2.3 直观对比例子

设 $n=2$ 个作业，3台设备的耗时矩阵 $J=\begin{pmatrix}2&0\\3&3\\5&2\end{pmatrix}$（仅用于说明顺序敏感性，下文聚焦 $m=2$）：

- 先作业1后作业2：$F=12$
- 先作业2后作业1：$F=11$

**结论：作业的相对顺序直接决定总完成时间**，这正是需要动态规划/排序策略来确定最优顺序的原因。

---

## 三、状态定义：如何把"排列问题"转化为可递推的子问题

### 3.1 关键观察——最优子结构

> **最优调度的排列具有这样的性质：** 一旦确定了排列中的第一个作业，剩下的作业相对于"两台设备完成第一个作业后所处的状态"而言，仍然构成一个最优排列。

这就是最优性原理在本问题中的体现：**子问题 = 剩余作业集合 + 两台设备的当前状态**。

### 3.2 如何刻画"两台设备的当前状态"？

设第一个作业是 $i$：$P_1$ 花 $a_i$ 时间执行完后，立刻可以开始下一个作业；但 $P_2$ 此刻可能还在忙（处理 $a_i$ 之前遗留的工作，或者正在等待 $P_1$）。

**关键简化：** 两台设备的状态可以压缩为**一个数字** $t$ —— "$P_1$ 完成当前任务、可以立刻开始下一个作业时，$P_2$ 还需要额外等待多久才能空出来"。

- $t=0$：$P_2$ 已经空闲，可以随时开始
- $t>0$：$P_2$ 还在忙，下一个作业的 $b$ 任务要等 $t$ 时间后才能开始

> 这一步是本节**最容易被忽略但最关键**的设计：把"两台设备各自的时间线"这个看似二维的状态，压缩成了一个一维标量 $t$，子问题才能递推下去。

### 3.3 状态定义

$$g(S, t) = \text{作业集合 } S\text{（尚未调度的作业）在 } P_2\text{ 还需等待 } t\text{ 时间的条件下，完成 } S\text{ 中所有作业所需的最短总时长}$$

**原问题** 即为：

$$g(\{1,2,\ldots,n\},\ 0)$$

## 四、递推关系式的建立

### 4.1 一步展开

设当前从 $S$ 中选作业 $i$ 作为下一个执行的作业：

- $P_1$ 立刻开始执行 $a_i$，耗时 $a_i$；此后 $P_1$ 空出
- $P_2$ 这边：若 $t \leq a_i$，等 $P_1$ 做完 $a_i$ 时，$P_2$ 早已空闲，于是 $b_i$ 紧接 $a_i$ 结束后开始；若 $t > a_i$，$P_2$ 还要再等 $(t - a_i)$，才能开始 $b_i$
- 因此 $b_i$ 的开始时刻相对于 "$a_i$ 结束时刻" 还要延迟 $\max\{t-a_i,\ 0\}$
- 执行完 $b_i$ 后，"$P_2$ 还需要的额外等待时间"变为新的状态 $x = b_i + \max\{t-a_i,\ 0\}$，传递给剩余作业集合 $S-\{i\}$

### 4.2 递推关系式（核心）

$$\boxed{g(S, t) = \min_{i \in S} \left\{ a_i + g\big(S - \{i\},\ b_i + \max\{t - a_i,\ 0\}\big) \right\}}$$

**初始值：**

$$g(\emptyset, t) = t \quad (t \geq 0)$$

（作业集合为空时，只需让 $P_2$ 把剩下的 $t$ 时间耗完即可）

### 4.3 这个递推式的问题：状态空间太大

$g(S,t)$ 的状态由"子集 $S$"和"实数 $t$"共同决定——子集有 $2^n$ 个，$t$ 取值连续，**直接按此递推式编程求解的代价依然极高**，并不比穷举好多少。

> 📌 **这就是本节真正的难点和教学重点：** 递推式本身写出来不难，但它**不能直接拿来用**。课件接下来做的事情，是通过纯数学推演，把这个递推式"消化掉"，最终提炼出一条不需要任何递推、只需排序一次即可执行的规则。这才是"流水线调度"区别于其他动态规划问题的独特之处。

---

## 五、数学推演：从递推式到排序规则

### 5.1 推演的目标（先讲清楚在做什么，再做）

我们想知道：**在最优排列中，任意两个相邻的作业应该按什么相对顺序排列？**

如果能找到一个只依赖于 $a_i,b_i,a_j,b_j$（与 $t$、与其他作业都无关）的简单判断条件，就能把"找最优排列"整体转化为"按某种规则排序"——从递推问题变成排序问题。

### 5.2 展开两个相邻作业

设 $i, j$ 是最优排列 $R$ 中**相邻**的两个作业（$i$ 排在 $j$ 前面），对 $g(S,t)$ 连续展开两步：

$$g(S,t) = a_i + a_j + g\big(S-\{i,j\},\ t_{ij}\big)$$

其中 $t_{ij}$ 是依次执行 $i$、$j$ 后传递下去的状态：

$$t_{ij} = b_j + \max\big\{\,b_i + \max\{t-a_i,0\} - a_j,\ 0\,\big\}$$

### 5.3 化简 $t_{ij}$

逐步把括号展开、把常数项分离出来：

$$t_{ij} = b_j + b_i - a_j + \max\big\{\max\{t-a_i,0\} - b_i + b_i,\ a_j\big\} - a_j \ \ (\text{中间步骤，逐项配凑})$$

最终化简为（与课件一致的标准形式）：

$$\boxed{t_{ij} = (b_i + b_j - a_i - a_j) + \max\{\,t,\ \ a_i+a_j-b_i,\ \ a_i\,\}}$$

### 5.4 交换顺序得到 $t_{ji}$

若把 $i,j$ 的顺序对调（先 $j$ 后 $i$），同样的推导给出：

$$t_{ji} = (b_i + b_j - a_i - a_j) + \max\{\,t,\ \ a_i+a_j-b_j,\ \ a_j\,\}$$

**观察：** $t_{ij}$ 和 $t_{ji}$ 只有一个共同的常数前缀 $(b_i+b_j-a_i-a_j)$ 加上一个 $\max\{\cdots\}$ 项的区别——这意味着**比较 $t_{ij}$ 与 $t_{ji}$ 大小，等价于比较两个 $\max\{\cdots\}$ 项的大小**。

### 5.5 建立最优性判据

由于 $g(S,t)$ 中的剩余部分 $g(S-\{i,j\}, \cdot)$ 对状态参数是单调不减的，"先 $i$ 后 $j$" 不劣于 "先 $j$ 后 $i$" 当且仅当：

$$t_{ij} \leq t_{ji}$$

代入化简结果，两侧消去公共项 $(b_i+b_j-a_i-a_j)$：

$$\max\{t,\ a_i+a_j-b_i,\ a_i\} \leq \max\{t,\ a_i+a_j-b_j,\ a_j\}$$

两侧同时消去与 $t$ 无关的部分（两边都减去 $a_i+a_j$）：

$$\max\{-b_i,\ -a_j\} \leq \max\{-b_j,\ -a_i\}$$

两边取负数，$\max$ 变 $\min$，不等号方向翻转：

$$\boxed{\min\{b_i,\ a_j\} \ \geq\ \min\{b_j,\ a_i\}}$$

> ✅ **推演完成。** 这正是第五节一开始设定的目标：一个只依赖 $a_i,b_i,a_j,b_j$ 四个数、与 $t$ 和其他作业完全无关的简单不等式。

## 六、从判据到全局规则

### 6.1 局部判据 ⇒ 全局最优

**命题：** 若一个排列中**每一对相邻作业** $(i,j)$（$i$ 在前）都满足 $\min\{b_i,a_j\} \geq \min\{b_j,a_i\}$，则该排列是全局最优排列。

这是因为：任意两个相邻作业若不满足该条件，交换后必不会变差（由 5.5 节推导），故最优排列中必然处处满足该条件；反之，处处满足该条件的排列也不可能再通过任何一次相邻交换变得更优，因此就是最优排列本身。

### 6.2 判据的两个直接推论

把 $j=i$ 之外所有作业的 $a,b$ 值都纳入考虑，可得：

- 设 $M^* = \min\{a_1,\ldots,a_n,\ b_1,\ldots,b_n\}$
- 若 $M^* = a_i$：作业 $i$ 应排在**最前面**（$P_1$ 上最短的任务应尽早开始）
- 若 $M^* = b_j$：作业 $j$ 应排在**最后面**（$P_2$ 上最短的任务应尽量靠后，让它"压轴"）

### 6.3 完整排序规则（Johnson 算法）

将 6.2 的"确定首尾"推广为一套可以确定**整个排列**的规则：

> 把全部 $a_i$ 和 $b_j$（共 $2n$ 个数）合并后按**非降序**排列，依次扫描：
> - 若当前数是某个 $a_j$，且作业 $j$ 尚未安排：将 $j$ 放入**当前最左的空位**
> - 若当前数是某个 $b_j$，且作业 $j$ 尚未安排：将 $j$ 放入**当前最右的空位**
> - 若作业 $j$ 已经安排过，跳过，继续扫描下一个数
> - 直到 $n$ 个作业全部被安排

**直觉解释：** $a$ 值小的作业，意味着它在 $P_1$ 上很快让出，应该尽早上场；$b$ 值小的作业，意味着它在 $P_2$ 上很快做完，不急于提前，可以放到后面收尾。

该规则同样适用于存在 $a_i=0$ 的作业（$a_i=0$ 的作业自动会在排序中最先被取出，从而自然地排在最前面，与最初"$a_i=0$ 的作业放最前"的特殊处理一致）。

## 七、完整计算实例

### 实例参数

$n=4$，$(a_1,a_2,a_3,a_4)=(3,4,8,10)$，$(b_1,b_2,b_3,b_4)=(6,2,9,15)$

### 第(1)步：合并 $a,b$ 值并非降排序

| 值 | 2 | 3 | 4 | 6 | 8 | 9 | 10 | 15 |
|---|---|---|---|---|---|---|---|---|
| 来源 | $b_2$ | $a_1$ | $a_2$ | $b_1$ | $a_3$ | $b_3$ | $a_4$ | $b_4$ |

### 第(2)步：依规则填入调度序列（4个位置）

| 当前元素 | 类型 | 操作 | 序列状态 |
|---|---|---|---|
| $b_2=2$ | $b$ 类，作业2未排 | 放最右空位 | $[\ ,\ ,\ ,2]$ |
| $a_1=3$ | $a$ 类，作业1未排 | 放最左空位 | $[1,\ ,\ ,2]$ |
| $a_2=4$ | 作业2已排 | 跳过 | $[1,\ ,\ ,2]$ |
| $b_1=6$ | 作业1已排 | 跳过 | $[1,\ ,\ ,2]$ |
| $a_3=8$ | $a$ 类，作业3未排 | 放最左空位 | $[1,3,\ ,2]$ |
| $b_3=9$ | 作业3已排 | 跳过 | $[1,3,\ ,2]$ |
| $a_4=10$ | $a$ 类，作业4未排 | 放最左空位 | $[1,3,4,2]$ |

**最优调度序列：$\sigma=(1,3,4,2)$**

### 第(3)步：按序列计算实际完成时间

每个任务尽早开始，$P_2$ 上某作业的开始时刻 $= \max\{P_1\text{完成该作业的时刻},\ P_2\text{上一个作业的结束时刻}\}$：

| 作业 | $P_1$区间 | $P_2$最早可用 | $P_2$区间 | 完成时间 $f_i$ |
|---|---|---|---|---|
| 1（$a=3,b=6$） | $[0,3]$ | 0 | $[3,9]$ | 9 |
| 3（$a=8,b=9$） | $[3,11]$ | 9 | $[11,20]$ | 20 |
| 4（$a=10,b=15$） | $[11,21]$ | 20 | $[21,36]$ | 36 |
| 2（$a=4,b=2$） | $[21,25]$ | 36 | $[36,38]$ | 38 |

$$\boxed{F(S) = \max\{9,20,36,38\} = 38}$$

## 八、规则验证：回头检验判据是否真的处处满足

| 相邻对 $(i,j)$ | $\min\{b_i,a_j\}$ | $\min\{b_j,a_i\}$ | 是否满足 $\geq$ |
|---|---|---|---|
| $(1,3)$ | $\min\{6,8\}=6$ | $\min\{9,3\}=3$ | $6\geq3$ ✓ |
| $(3,4)$ | $\min\{9,10\}=9$ | $\min\{15,8\}=8$ | $9\geq8$ ✓ |
| $(4,2)$ | $\min\{15,4\}=4$ | $\min\{2,10\}=2$ | $4\geq2$ ✓ |

三对相邻作业均满足判据，验证 $\sigma=(1,3,4,2)$ 确为最优。

## 九、整体逻辑回顾（一图理清全节脉络）

```
m>2 时调度问题 NP-Hard
        │
        ▼
聚焦特例：m=2，非抢先调度（OFT）
        │
        ▼
洞察：顺序由 P1 上的排列唯一确定 → 问题 = 找最优排列（O(n!) 朴素做法不可行）
        │
        ▼
最优性原理：确定第一个作业后，剩余作业相对"当前设备状态"仍最优
        │
        ▼
状态压缩：把两台设备的状态压缩为一个标量 t（P2 还需等待的时间）
        │
        ▼
写出递推式 g(S,t) = min{ ai + g(S-{i}, bi+max{t-ai,0}) }
        │
        ▼
问题：状态空间 (2^n 个子集 × 连续的 t) 太大，递推式不能直接用
        │
        ▼
目标转换：寻找只依赖 (ai,bi,aj,bj) 的相邻交换判据
        │
        ▼
数学推演：展开 g(S,t)，比较 t_ij 与 t_ji，化简得 min{bi,aj} ≥ min{bj,ai}
        │
        ▼
局部判据 → 全局规则：Johnson 排序算法（a 类靠左，b 类靠右）
        │
        ▼
O(n log n) 排序，问题解决
```

**这条主线说明：** 6.8 节与前面几节（OBST、矩阵链）的画风不同——它不是"写出递推式 → 直接按 $O(n^2)/O(n^3)$ 编程实现"，而是把递推式当作一个**中间产物**，通过代数推演把它彻底转化掉，最终落地为一条比任何动态规划实现都更快的排序规则。**这正是本节在全章中的独特教学意义。**

## 十、易错点汇总

1. **状态 $t$ 的方向**：$t$ 是"$P_1$完成后，$P_2$还需要等待"的时间（面向未来），不是"$P_2$已经等待了多久"（面向过去）。

2. **$x$ 的两种情形**：$x=b_i+\max\{t-a_i,0\}$ —— 若 $t\le a_i$，$P_2$ 早已空闲，$x=b_i$；若 $t>a_i$，$P_2$ 还要再等 $(t-a_i)$，$x=b_i+(t-a_i)$。

3. **数学推演的目的**：所有代数化简都是为了**消去 $t$ 和其他作业的影响**，得到一个只含 $a_i,b_i,a_j,b_j$ 四个数的判据。如果中途忘了这个目标，很容易迷失在 $\max\{\cdots\}$ 的展开细节里。

4. **排序方向不要搞反**：$a$ 类（$P_1$任务短）→ 尽早执行 → 放最左；$b$ 类（$P_2$任务短）→ 尽量收尾 → 放最右。

5. **等值情况**：若 $\min\{\cdots\}$ 中出现 $a_i=b_j$ 的并列，两种排列结果相同，任取一种即为最优。

6. **适用范围**：此规则仅对 $m=2$ 成立，$m\geq3$ 时调度问题是 NP-Hard，不能套用。

## 十二、考点速查

| 考点 | 核心内容 |
|---|---|
| 适用范围 | 仅 $m=2$，非抢先调度（OFT） |
| 状态定义 | $g(S,t)$：作业集合 $S$、$P_2$ 还需等待 $t$ 时的最优总时长 |
| 递推关系式 | $g(S,t)=\min_{i\in S}\{a_i+g(S-\{i\},b_i+\max\{t-a_i,0\})\}$ |
| 初始值 | $g(\emptyset,t)=t$ |
| 推演目标 | 找到与 $t$ 无关的相邻交换判据 |
| 最优相邻判据 | $\min\{b_i,a_j\}\geq\min\{b_j,a_i\}$（$i$ 在 $j$ 前） |
| 首作业判据 | $\min(a_i)$ 最小 ⇒ $i$ 排最前 |
| 末作业判据 | $\min(b_j)$ 最小 ⇒ $j$ 排最后 |
| 排序规则 | 合并 $a_i/b_j$ 非降排序；$a$ 类→最左空位，$b$ 类→最右空位 |
