# 6.2 多段图问题 — 复习资料

---

## 一、问题描述

### 1.1 多段图的定义

**多段图** $G=(V,E)$ 是一个有向带权图，具有以下特性：

- 图中结点被划分成 $k \geq 2$ 个**不相交的集合** $V_i$，$1 \leq i \leq k$
- $V_1$ 和 $V_k$ 分别只有**一个结点** $s$（源点）和 $t$（汇点）
- 所有边 $\langle u,v \rangle$ 满足：若 $u \in V_i$，则 $v \in V_{i+1}$（即边只能从第 $i$ 段指向第 $i+1$ 段）
- 每条边 $\langle u,v \rangle$ 附有成本 $c(u,v)$

**目标：** 求由 $s$ 到 $t$ 的**最小成本路径**（路径成本 = 路径上各边成本之和）。

### 1.2 课件标准实例（12结点，5段图）

结点编号：$V_1=\{1\}$，$V_2=\{2,3,4,5\}$，$V_3=\{6,7,8\}$，$V_4=\{9,10,11\}$，$V_5=\{12\}$

## 二、最优性原理证明

**命题：** 多段图问题满足最优性原理。

**证明（反证法）：**

1. 设 $s, v_2, v_3, \ldots, v_{k-1}, t$ 是由 $s$ 到 $t$ 的一条**最短路径**。
2. 从源点 $s$ 出发，已作出到结点 $v_2$ 的决策，$v_2$ 即初始决策产生的状态。
3. 将 $v_2$ 视为子问题的初始状态，子问题为：求 $v_2$ 到 $t$ 的最短路径。
4. 子问题的最短路径显然是 $v_2, v_3, \ldots, v_{k-1}, t$。
5. **反证：** 若存在 $v_2$ 到 $t$ 的更短路径 $v_2, q_3, \ldots, q_{k-1}, t$，则 $s, v_2, q_3, \ldots, q_{k-1}, t$ 比 $s, v_2, v_3, \ldots, v_{k-1}, t$ 更短，与假设矛盾。

✅ **结论：最优性原理成立**，可以使用动态规划求解。

## 三、向前处理法（Forward Approach）

### 3.1 核心思想

**从汇点倒推到源点**。设 $\text{COST}(i, j)$ 为从 $V_i$ 中结点 $j$ 到汇点 $t$ 的最小成本路径的成本。

### 3.2 递推关系式

$$\text{COST}(i, j) = \min_{l \in V_{i+1},\, \langle j,l \rangle \in E} \{ c(j, l) + \text{COST}(i+1, l) \}$$

**初始条件（$i = k-1$，即倒数第二段到汇点）：**

$$\text{COST}(k-1, j) = \begin{cases} c(j, t) & \text{若 } \langle j,t \rangle \in E \\ \infty & \text{否则} \end{cases}$$

> **注意区分方向：** 向前处理中 $\text{COST}(i,j)$ 表示从结点 $j$ **到汇点** $t$ 的最小成本，计算方向从 $i=k-1$ 向 $i=1$ 递推（倒推）。

### 3.3 决策记录（标记函数）

在计算每个 $\text{COST}(i,j)$ 的同时，记录使其取到最小值的下一个结点 $l$：

$$D(i, j) = l \quad \text{（结点 } j \text{ 在第 } i \text{ 段的最优决策）}$$

路径回溯方式：$P(1) = s \to P(2) = D(1, P(1)) \to \cdots \to P(k) = t$

### 3.4 算法伪代码（FGRAPH）

```
procedure FGRAPH(E, k, n, P)
  // P(1:k) 保存最小成本路径上各段结点
  // COST(n) 为备忘录，D(n-1) 为标记函数
  real COST(n);  int D(n-1), P(k), r, j, k, n
  
  COST(n) ← 0                              // 汇点成本为 0
  for j ← n-1 to 1 by -1 do
    寻找结点 r，满足 <j,r> ∈ E 且使 c(j,r)+COST(r) 最小
    COST(j) ← c(j,r) + COST(r)
    D(j) ← r
  repeat
  
  P(1) ← 1;  P(k) ← n                     // 路径起点和终点
  for j ← 2 to k-1 do
    P(j) ← D(P(j-1))                       // 逐段回溯路径
  repeat
end FGRAPH
```

**复杂度分析：**
- 时间复杂度：$\Theta(n + e)$（$n$ 为结点数，$e$ 为边数）
- 空间复杂度：$\Theta(n + e)$（邻接表存储 + COST、D、P 数组）

> **存储优化：** 对结点按段从左到右统一编号，$\text{COST}$ 和 $D$ 可以简化为一维数组（省略段号下标 $i$）。

## 五、向后处理法（Backward Approach）

### 5.1 核心思想

**从源点正推到汇点**。设 $\text{BCOST}(i, j)$ 为从源点 $s$ 到 $V_i$ 中结点 $j$ 的最小成本路径的成本。

> **方向对比：向前处理从汇点倒推，向后处理从源点正推，两者是对偶的。**

### 5.2 递推关系式

$$\text{BCOST}(i, j) = \min_{l \in V_{i-1},\, \langle l,j \rangle \in E} \{ \text{BCOST}(i-1, l) + c(l, j) \}$$

**初始条件（$i = 2$，即源点直接到达的结点）：**

$$\text{BCOST}(2, j) = \begin{cases} c(s, j) & \text{若 } \langle s,j \rangle \in E \\ \infty & \text{否则} \end{cases}$$

### 5.3 向后处理完整计算过程（12结点实例）

**$V_2$ 各结点：**

| 结点 | BCOST | D |
|------|-------|---|
| 2 | $c(1,2)=9$ | 1 |
| 3 | $c(1,3)=7$ | 1 |
| 4 | $c(1,4)=3$ | 1 |
| 5 | $c(1,5)=2$ | 1 |

**$V_3$ 各结点：**

| 结点 | 计算过程 | BCOST | D |
|------|----------|-------|---|
| 6 | $\min\{\text{BCOST}(2)+c(2,6),\ \text{BCOST}(3)+c(3,6)\}=\min\{9+4,7+2\}=9$ | **9** | 3 |
| 7 | $\min\{\text{BCOST}(2)+c(2,7),\ \text{BCOST}(3)+c(3,7)\}=\min\{9+2,7+7\}=11$ | **11** | 2 |
| 8 | $\min\{\text{BCOST}(2)+c(2,8),\ \text{BCOST}(4)+c(4,8),\ \text{BCOST}(5)+c(5,8)\}=\min\{9+1,3+11,2+8\}=10$ | **10** | 2 |

**$V_4$ 各结点：**

| 结点 | 计算过程 | BCOST | D |
|------|----------|-------|---|
| 9 | $\text{BCOST}(6)+c(6,9)=9+6=15$ | **15** | 6 |
| 10 | $\min\{\text{BCOST}(6)+c(6,10),\ \text{BCOST}(7)+c(7,10),\ \text{BCOST}(8)+c(8,10)\}=\min\{9+5,11+3,10+5\}=14$ | **14** | 6 |
| 11 | $\text{BCOST}(8)+c(8,11)=10+6=16$ | **16** | 8 |

**汇点 $t=12$：**

$$\text{BCOST}(12) = \min\{\text{BCOST}(9)+c(9,12),\ \text{BCOST}(10)+c(10,12),\ \text{BCOST}(11)+c(11,12)\}$$
$$= \min\{15+4,\ 14+2,\ 16+5\} = \min\{19,\ 16,\ 21\} = 16$$

$D(12) = 10$

**路径回溯（逆向追踪 $D$）：**

$t=12 \leftarrow D(12)=10 \leftarrow D(10)=6 \leftarrow D(6)=3 \leftarrow s=1$

$$\boxed{\text{最小成本路径：} 1 \to 3 \to 6 \to 10 \to 12，\text{最小成本} = 16}$$

> ⚠️ **注意：向前和向后处理求出的最小成本相同（均为16），但最优路径可能不同！** 本例中向前得到 $1\to2\to7\to10\to12$，向后得到 $1\to3\to6\to10\to12$，两者均是最优解（存在多个最优路径）。

## 六、向前 vs 向后处理对比

| 对比维度 | 向前处理（FGRAPH） | 向后处理（BGRAPH） |
|---------|-----------------|-----------------|
| **$\text{COST}$ 含义** | 结点 $j$ **到汇点** $t$ 的最小成本 | 源点 $s$ **到结点** $j$ 的最小成本 |
| **计算方向** | 从 $i=k-1$ 倒推至 $i=1$ | 从 $i=2$ 正推至 $i=k$ |
| **初始条件** | $\text{COST}(k-1, j) = c(j, t)$ | $\text{BCOST}(2, j) = c(s, j)$ |
| **路径回溯** | 从 $P(1)=s$ 正向追踪 $D$ | 从 $P(k)=t$ 反向追踪 $D$ |
| **时间复杂度** | $\Theta(n+e)$ | $\Theta(n+e)$ |

## 七、多段图的应用（资源分配问题）

多段图是**资源分配问题**的经典建模工具：

**问题：** 把 $n$ 个资源分配给 $r$ 个项目，$N(i,j)$ 表示把 $j$ 个资源分配给项目 $i$ 所获的利润，求使总利润最大的分配方案。

**建模：** 构造 $r+1$ 段图
- $V(i,j)$：前 $i-1$ 个项目共分配了 $j$ 个资源的状态
- 边 $\langle V(i,j), V(i+1,l) \rangle$：项目 $i$ 分配 $l-j$ 个资源，获利 $N(i, l-j)$
- 源点：$V(1,0)$，汇点：$V(r+1, n)$
- 求源点到汇点的**最大成本路径** = 最大总利润分配方案

| 结点 $j$ | $\text{COST}(j)$ | $D(j)$ |
|---------|-----------------|--------|

## 九、易错点汇总

| 易错点 | 正确理解 |
|--------|---------|
| **向前/向后处理混淆** | 向前处理：$\text{COST}(i,j)$ = 结点 $j$ 到**汇点**的成本，从 $k-1$ **倒推**；向后处理：$\text{BCOST}(i,j)$ = **源点**到 $j$ 的成本，从 $2$ **正推** |
| **初始条件搞错** | 向前：初始为 $i=k-1$（第 $k-1$ 段直接到汇点）；向后：初始为 $i=2$（源点直接可达的结点）|
| **路径回溯方向** | 向前处理路径从 $P(1)=s$ 正向查 $D$；向后处理路径从 $t$ 反向查 $D$ |
| **两种方法路径不同** | 两种方法最小成本相同，但路径可能不同（存在多条最优路径时） |
| **$\text{COST}$ 下标含义** | $\text{COST}(i,j)$ 中 $i$ 是段号，$j$ 是结点编号；简化为一维时省略 $i$（因为每个结点编号唯一确定所在段） |
| **无边时取 $\infty$** | 若结点 $j$ 在 $V_{k-1}$ 段但 $\langle j,t \rangle \notin E$，则 $\text{COST}(k-1,j)=\infty$，不可达 |

## 十、考点速查

| 考点 | 关键内容 |
|------|---------|
| **向前处理递推式** | $\text{COST}(i,j)=\min_{l}\{c(j,l)+\text{COST}(i+1,l)\}$，初值 $\text{COST}(k-1,j)=c(j,t)$ |
| **向后处理递推式** | $\text{BCOST}(i,j)=\min_{l}\{\text{BCOST}(i-1,l)+c(l,j)\}$，初值 $\text{BCOST}(2,j)=c(s,j)$ |
| **算法名称** | 向前：FGRAPH；向后：BGRAPH（需自行设计） |
| **时间复杂度** | $\Theta(n+e)$ |
| **空间复杂度** | $\Theta(n+e)$（邻接表 + COST/D/P数组） |
| **最优性原理** | 反证法：若子路径非最优，则整体路径非最短，矛盾 |
| **应用场景** | 资源分配问题（$r+1$ 段图建模） |
