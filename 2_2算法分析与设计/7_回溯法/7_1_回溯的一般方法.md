# 7.1 回溯的一般方法 — 复习资料

> **对应作业：** 平时作业题1（回溯法与分支限界法状态空间生成方式对比）

---

## 一、方法适用的问题特点

### 1.1 问题特征

回溯法适用于解决**多阶段决策问题**，也称为**组合问题**。

- 问题的解向量用**元组**来表示，元素 $x_i$ 通常取自于某个有穷集 $S_i$，$1\leq i\leq n$，$n$ 表示问题规模；
- **固定长 n-元组：** $(x_1,\ldots,x_n)$；
- **不定长 k-元组：** $(x_1,\ldots,x_k)$，$k\leq n$；
- 问题满足**多米诺性质**（见第二节）；
- 问题寻找满足约束条件的一个解或所有解；有时则要寻找最优解，此时问题中还存在**目标函数**。

### 1.2 组合搜索 vs 组合优化

多阶段决策问题可细分为两类：

| 类型 | 关注点 | 问题描述特征 | 求解目标 |
|------|--------|-------------|---------|
| **组合搜索** | 解的存在性 | 仅存在**约束条件** | 找到一个可行解 / 所有可行解 |
| **组合优化** | 解的最优性 | 还存在**目标函数** | 找到具有极小值/极大值的可行解，即最优解（也称极小化/极大化问题） |

## 二、多米诺性质

### 2.1 定义

设 $P(x_1,\ldots,x_i)$ 是关于向量 $(x_1,\ldots,x_i)$ 的某种性质的判定：

$$\text{当 } P(x_1,\ldots,x_{i+1}) \text{ 为真时，一定有 } P(x_1,\ldots,x_i) \text{ 为真} \quad (0<i<n)$$

**逆命题（用于剪枝）：** 根据多米诺性质，如果 $P(x_1,\ldots,x_i)$ 不成立，则 $P(x_1,\ldots,x_{i+1})$ 亦不成立。

一个满足多米诺性质的组合问题，是指能够根据约束条件和目标函数**不断检验**正在构造的部分解向量 $(x_1,\ldots,x_i)$（$0<i<n$），一旦发现不成立，则**无需考虑**后续 $x_{i+1},\ldots,x_n$ 的取值。

### 2.2 多米诺性质如何提高算法效率

**思考：利用多米诺性质能够提高算法效率，它会如何起作用？**

设约束函数 $B$ 实现问题的约束条件，对于 n-元组 $(x_1,\ldots,x_n)$，$x_i\in S_i$，$|S_i|=m_i$：

| 处理方式 | 做法 | 检测代价 |
|---------|------|---------|
| **硬性处理（不利用多米诺性质）** | 向量个数 $m=m_1\times m_2\times\cdots\times m_n$，对这 $m$ 个 n-元组**逐一**检测是否满足 $B(x_1,\ldots,x_n)$ | 必须构造完整的 n-元组才能检验 |
| **回溯法（利用多米诺性质）** | 约束函数 $B$ 可以**提前**检验正在构造中的部分向量 $(x_1,\ldots,x_i)$，一旦发现不能导致问题的解，立即**终止该向量继续构造** | 不再构造 $x_{i+1},\ldots,x_n$ 的取值，减少了 $m_{i+1}\times\cdots\times m_n$ 个向量 |

> **结论：问题的多米诺性质是回溯法提高算法效率的关键。** 它使得约束检验可以"提前介入"，对一棵子树整体剪枝，而不必等到生成完整的叶结点之后才发现该路径无效。

## 三、解空间树与基本概念

### 3.1 显式约束与隐式约束

| 概念 | 定义 |
|------|------|
| **显式约束** | 每个 $x_i$ 的取值集合 $S_i$，可能与问题实例有关，也可能无关 |
| **隐式约束** | 描述了 $x_i$ 彼此相关的情况，**与问题实例有关** |

### 3.2 解空间与解空间树

| 概念 | 定义 |
|------|------|
| **解空间** | 满足**显式约束**条件的所有元组 |
| **解空间树** | 基于解空间画成的树形状（即"静态树"，与具体实例无关） |
| **问题状态** | 解空间树中的**所有**结点 |
| **解状态 X** | 由根到结点 X 的路径，确定了解空间中的一个元组（**只需满足显式约束**） |
| **答案状态 X** | 由根到解状态结点 X 的路径，确定了问题的**一个解**（**还需满足隐式约束**） |

> **回溯法解决问题的过程，就是在解空间树上搜索答案状态结点的过程。**

### 3.3 三者之间的对应关系

```
问题状态  ⊇  解状态（满足显式约束，对应约束函数）  ⊇  答案状态（还满足隐式约束）
```

## 四、问题建模

### 4.1 建模四步骤

1. **确定元组表达形式：** 固定长元组（$n$-元组）或不定长元组（$k$-元组，$k\leq n$）；
2. **确定约束条件：** 显式约束和隐式约束；
3. **检验问题满足多米诺性质**；
4. **确定解空间树：** 问题状态、解状态和答案状态。

### 4.2 建模示例：子集和问题

**问题描述：** 已知 $n+1$ 个正数 $w_i\,(1\leq i\leq n)$ 和 $M$，要求找出 $w_i$ 的和是 $M$ 的所有子集。

**问题实例：** $n=4$，$(w_1,w_2,w_3,w_4)=(11,13,24,7)$，$M=31$。

- 可行解1：$11,13,7$
- 可行解2：$24,7$

> 关于子集和问题约束函数的优化设计与完整回溯算法，详见 **7.4 节**复习资料；本节只用它来演示两种元组表达方式对解空间树结构的影响。

#### （一）固定长元组表达（n-元组）

用 $n$-元组 $(x_1,\ldots,x_n)$ 表示：

| 项目 | 内容 |
|------|------|
| 显式约束 | $x_i\in\{0,1\}$，$1\leq i\leq n$；选择 $w_i$ 则 $x_i=1$，否则 $x_i=0$ |
| 隐式约束 | $\sum w_i x_i = M$，$1\leq i\leq n$ |
| 多米诺性质 | 若部分解向量的和已大于 $M$，则包含它的解向量也大于 $M$（可剪枝） |
| 解空间规模 | $2^n$ 个元组 |

按本实例（$n=4$）：解空间 $2^4=16$ 个元组；可行解1表示为 $(1,1,0,1)$，可行解2表示为 $(0,0,1,1)$。

**对应解空间树（n=4）：**

| 类型 | 数量 |
|------|------|
| 问题状态（全部结点） | 31 个 |
| 解状态（叶结点） | 16 个 |
| 答案状态（当前实例） | 2 个 |

> 这是一棵深度为 4 的**完全二叉树**，每个结点恰有两个儿子（$x_i=1$ 或 $x_i=0$），总结点数 $=2^0+2^1+2^2+2^3+2^4=31$。**只有叶结点（完整的 4 元组）才是解状态**——因为固定长表达要求 $n$ 个分量全部确定才构成一个候选解。

#### （二）不定长元组表达（k-元组）

用 $k$-元组 $(x_1,\ldots,x_k)$，$k\leq n$ 表示（$x_i$ 直接记录"被选中的下标"，而非 0/1 标记）：

| 项目 | 内容 |
|------|------|
| 显式约束 | $x_i\in\{j \mid j\text{是}w_j\text{的下标}, 1\leq j\leq n\}$，$1\leq i\leq k$，且 $x_i<x_{i+1}$（$1\leq i<k$） |
| 隐式约束 | 相应的 $w_{x_i}$ 的和等于 $M$ |
| 多米诺性质 | 若部分解向量的和已大于 $M$，则包含它的解向量也大于 $M$ |
| 解空间规模 | $2^n$ 个元组 |

> **为什么要求 $x_i<x_{i+1}$？** 是为了**避免重复情况**，如 $(1,2,4)$ 和 $(1,4,2)$ 本质上是同一个子集 $\{w_1,w_2,w_4\}$，规定下标严格递增可保证每个子集只对应唯一的一条路径。

按本实例：可行解1表示为 $(1,2,4)$，可行解2表示为 $(3,4)$。

**对应解空间树（n=4）：**

| 类型 | 数量 |
|------|------|
| 问题状态（全部结点） | 16 个 |
| 解状态（全部结点） | **16 个** |
| 答案状态（当前实例） | 2 个 |

> 与固定长表达不同，这里**所有结点（不仅是叶结点）都是解状态**——因为不定长表达中，从根到**任何**结点的路径都已经确定了解空间中的一个完整候选子集（不要求达到长度 $n$），不需要再补齐到第 $n$ 层才算一个候选解。

### 4.3 思考题：两种表达形式解空间树结点数的渐进比较

**思考：比较两种元组表达形式下的解空间树，请基于 $n$ 的渐进表示来描述结点个数。**

| 表达形式 | 结点总数公式 | 渐进阶 | $n=4$ 时的值 |
|---------|-------------|--------|-------------|
| **固定长 n-元组**（完全二叉树） | $\sum_{i=0}^{n}2^i = 2^{n+1}-1$ | $\Theta(2^n)$ | $31$ |
| **不定长 k-元组**（子集计数树） | $\sum_{k=0}^{n}\binom{n}{k} = 2^n$ | $\Theta(2^n)$ | $16$ |

**结论：**

- 两种表达形式下，解空间树的结点数**渐进阶相同**，均为 $\Theta(2^n)$（指数级）；
- 但**常数因子不同**：固定长表达的结点数 $2^{n+1}-1\approx 2\times 2^n$，恰好是不定长表达 $2^n$ 的约 **2 倍**（验证：$n=4$ 时 $31/16\approx1.94$）；
- 原因在于固定长表达对**每个**元素都显式区分"选中/不选中"两个分支（包括对应"不选中"的冗余分支结构），而不定长表达**只为被选中的元素建立结点**，不为"跳过"操作单独建立分支，因此结构更紧凑。

## 五、动态树中的结点类型

### 5.1 静态树与动态树

| 概念 | 定义 |
|------|------|
| **静态树** | 即解空间树，树结构与所要解决的**问题实例无关** |
| **动态树** | 树结构与**实例相关**，在求解过程中**动态生成**结点 |

### 5.2 三类结点

| 结点类型 | 定义 |
|---------|------|
| **活结点** | 自己已经生成，而其儿子结点还没有全部生成的结点 |
| **E-结点（正在扩展的结点）** | 当前正在生成其儿子结点的活结点 |
| **死结点** | 不再进一步扩展，或者其儿子结点已全部生成的结点 |

## 六、动态树结点生成方式与回溯法设计思想

### 6.1 两种结点生成方式（回溯法 ↔ 分支限界法的本质区别）

| 方式 | 描述 | 对应方法 |
|------|------|---------|
| **第一种方式** | 当前的 E-结点 R 一旦生成一个新的儿子结点 C，这个 C 结点就**立即变成新的 E-结点**；当检测完了子树 C 后，R 结点**再次成为 E-结点**，生成下一个儿子结点 | **回溯法**（基于深度优先） |
| **第二种方式** | 当前结点一旦成为 E-结点，就**一直处理到变成死结点为止**（即生成其全部儿子）。生成的儿子结点加入到**活结点表**中，然后再从活结点表中选择下一个新的 E-结点 | **分支限界法** |

> 两种方式的核心差异在于：回溯法每次只生成**一个**儿子就立即下探（深度优先，无需维护活结点表）；分支限界法生成**全部**儿子后才离开当前结点（需要维护活结点表，从中选择下一个 E-结点，可以是 BFS 或最小代价优先）。

### 6.2 回溯法的设计思想

1. 针对问题定义解空间树结构：**元组、显式约束条件、隐式约束条件**；
2. 检验问题满足**多米诺性质**；
3. 以**深度优先**方式搜索解空间树，在搜索过程中使用**约束函数 $B$** 避免无效搜索：
   - 对当前 E-结点 R，一旦生成一个新的儿子结点 C，**C 就成为新的 E-结点**，搜索向纵深方向移动；
   - 如果当前 E-结点不能满足约束函数 $B$，或者再没有儿子结点，则其成为**死结点**，搜索不再向纵深方向移动，转而**向上回溯**至最近的一个活结点，该活结点成为新的 E-结点；
4. 搜索过程直至**找到所要求的解**或**解空间树中已没有活结点**时为止。

### 6.3 回溯法的形式化描述

假设要找出所有的答案结点：

- $(x_1,x_2,\ldots,x_{k-1})$ 是状态空间树中由根出发的一条路径，到达结点 $Y$；
- $T(x_1,\ldots,x_{k-1})$ 是元素 $x_k$ 的集合，对于每一个 $x_k$，$(x_1,x_2,\ldots,x_{k-1},x_k)$ 是一条由根到结点 $Y$ 的一个儿子结点的路径；
- 对于约束函数 $B_k$：如果路径 $(x_1,x_2,\ldots,x_{k-1},x_k)$ **不可能**延伸到一个答案结点，则 $B_k(x_1,x_2,\ldots,x_k)$ 取**假值**，否则取**真值**。

## 七、回溯法的通用算法

### 7.1 算法7.1：非递归算法 BACKTRACK

```
procedure BACKTRACK(n)
  int k, n
  local X(1:n)
  k ← 1
  while (k>0) do
    if (还剩有没检验的X(k)使得 X(k)∈T(X(1)…X(k-1))
        and B(X(1)…X(k))=TRUE)
    then if ((X(1)…X(k))是一条抵达答案结点的路径)
           then print(X(1)…X(k))
         endif
         k ← k+1
    else k ← k-1
    endif
  repeat
end BACKTRACK
```

**要点：**

- `k ← k+1`：当前 $X(k)$ 通过检验，搜索向纵深移动（生成新 E-结点）；
- `k ← k-1`：当前层已无满足条件的 $X(k)$ 可选，**回溯**至上一层。

### 7.2 算法7.2：递归算法 RBACKTRACK

```
procedure RBACKTRACK(k)
  global X(1:n);
  int k, n;
  for (满足下式的每个X(k), X(k)∈T(X(1)…X(k-1))
       and B(X(1),…X(k))=true) do
    if (X(1),…,X(k))是一条抵达答案结点的路径 then
      print(X(1)…X(k))
    endif
    call RBACKTRACK(k+1)
  repeat
end RBACKTRACK
```

> **注意：** 进入算法时，解向量 X 中的前 $k-1$ 个分量 $X(1)\ldots X(k-1)$ **已经被赋值**；`for` 循环本身就蕴含了"枚举所有满足显式约束且通过 $B$ 检验的儿子结点"，递归调用 `RBACKTRACK(k+1)` 对应着进入下一层继续深度优先搜索，函数返回则自然对应"回溯"。

### 7.3 两种算法的对比

| 维度 | 算法7.1（非递归） | 算法7.2（递归） |
|------|------------------|-----------------|
| 控制结构 | `while` 循环 + 显式的 `k←k+1` / `k←k-1` | `for` 循环 + 递归调用 |
| 回溯实现方式 | 显式修改层号 $k$ | 函数调用返回自动完成 |
| 状态保存 | 需要局部数组手动维护 | 由调用栈自动维护 |
| 可读性/常用程度 | 较低 / 教学价值高 | 较高 / 实际编程常用 |

## 八、作业题1精解

> **题目：** 回溯法和分支限界法的状态空间生成方式有何不同？简述两种方法的基本思想。

### 8.1 状态空间生成方式的不同

| 方法 | 状态空间生成方式 |
|------|-----------------|
| **回溯法** | 当前的 E-结点 R 一旦生成一个新的儿子结点 C，这个 C 结点就变成一个新的 E-结点，当检测完了子树 C 后，R 结点就再次成为 E-结点，生成下一个儿子结点 |
| **分支限界法** | 当前结点一旦成为 E-结点，就一直处理到变成死结点为止。其生成的儿子结点加入到活结点表中，然后再从活结点表中选择下一个新的 E-结点 |

### 8.2 两种方法的基本思想

**回溯法：** 回溯法从根结点出发，按**深度优先**策略搜索解空间树。判断 E-结点是否包含问题的解：如果不包含，则跳过以该结点为根的子树，逐层向其祖先结点回溯；否则就进入该子树，继续按深度优先的策略进行搜索。

**分支限界法：** 分支限界法从根结点出发，生成当前 E-结点**全部**儿子之后，再选择新的活结点成为 E-结点，重复上述过程。为提高算法效率，会使用**限界函数**和**成本估计函数**等进行剪枝。

> 两种方法均可视为对解空间树的系统化搜索，区别在于"何时离开当前 E-结点"（生成1个儿子就走 vs 生成全部儿子才走）以及是否需要维护活结点表。分支限界法的详细内容见**第8章**复习资料。

## 九、易错点汇总

| 易错点 | 正确理解 |
|--------|---------|
| **多米诺性质的逆命题方向** | 多米诺性质本身是"长串为真 ⟹ 短串为真"；剪枝利用的是其**逆否命题**："短串为假 ⟹ 长串为假"，二者逻辑等价但叙述方向不同，考试中常考逆否命题的应用 |
| **解状态 vs 答案状态的判定层级因表达方式而异** | 固定长 n-元组表达中，**只有叶结点**是解状态；不定长 k-元组表达中，**所有结点**都是解状态——这取决于元组是否要求"凑满 $n$ 位"才算一个候选解 |
| **解空间规模的渐进阶相同 ≠ 结点数相同** | 固定长与不定长表达下子集和问题解空间树结点数渐进阶都是 $\Theta(2^n)$，但具体数值不同（$2^{n+1}-1$ vs $2^n$），不能混用 |
| **E-结点定义中"生成儿子"的时序** | 回溯法中 E-结点每次只生成**一个**儿子就立即转移为该儿子（深度优先）；分支限界法中 E-结点要生成**全部**儿子后才变成死结点，二者不能混淆 |
| **活结点表是分支限界法的专属概念** | 回溯法不需要维护活结点表（因为它总是沿着唯一一条"当前路径"深度优先前进，回溯时直接修改层号 $k$）；活结点表是分支限界法为了支持"从多个候选中选最优 E-结点"而引入的数据结构 |
| **非递归算法的 `k←k-1` 含义** | 表示**回溯到上一层**，并不是"放弃整个搜索"，下一次循环会继续在上一层尝试该层剩余的候选值 |

## 十、考点速查

| 考点 | 核心内容 |
|------|---------|
| **适用问题特征** | 多阶段决策问题（组合搜索 关注存在性 / 组合优化 关注最优性 + 目标函数） |
| **多米诺性质** | $P(x_1,\ldots,x_{i+1})$ 为真 $\Rightarrow$ $P(x_1,\ldots,x_i)$ 为真；逆否命题用于剪枝 |
| **显式约束 / 隐式约束** | 显式约束确定取值集合 $S_i$；隐式约束描述 $x_i$ 间关联，与实例有关 |
| **解状态 / 答案状态** | 解状态满足显式约束；答案状态还满足隐式约束（即问题的解） |
| **静态树 / 动态树** | 静态树即解空间树（与实例无关）；动态树是实际生成的结点（与实例相关） |
| **活结点 / E-结点 / 死结点** | 活结点=已生成但儿子未生成完；E-结点=正在扩展的活结点；死结点=不再扩展 |
| **回溯法 vs 分支限界法** | 回溯法：生成1个儿子即转移（深度优先，无需活结点表）；分支限界法：生成全部儿子才转移（需活结点表） |
| **回溯法两种通用算法** | 算法7.1非递归（`k←k+1`/`k←k-1`显式控制）；算法7.2递归（`for`+递归调用，回溯由函数返回完成） |
| **固定长 vs 不定长元组解空间树结点数** | 固定长：$2^{n+1}-1=\Theta(2^n)$，仅叶结点为解状态；不定长：$2^n=\Theta(2^n)$，所有结点为解状态 |
