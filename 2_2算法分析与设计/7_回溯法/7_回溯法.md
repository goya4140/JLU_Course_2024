## 一、回溯法的适用问题

### 1.1 问题特征

回溯法适用于**多阶段决策问题**（也称组合问题），其解向量用**元组**表示：

- **固定长 n-元组：** $(x_1, x_2, \ldots, x_n)$，每个 $x_i$ 取自有穷集 $S_i$
- **不定长 k-元组：** $(x_1, x_2, \ldots, x_k)$，$k \leq n$

问题分为两类：

| 类型 | 关注点 | 问题特征 |
|------|--------|---------|
| **组合搜索** | 解的存在性 | 仅有约束条件，寻找一个或所有可行解 |
| **组合优化** | 解的最优性 | 有约束条件 + 目标函数，寻找最优解 |

### 1.2 多米诺性质（核心）

设 $P(x_1, \ldots, x_i)$ 是关于部分解向量的某种性质判定：

> **若 $P(x_1,\ldots,x_{i+1})$ 为真，则 $P(x_1,\ldots,x_i)$ 必为真**（$0 < i < n$）

**逆命题（用于剪枝）：** 若 $P(x_1,\ldots,x_i)$ 不成立，则 $P(x_1,\ldots,x_{i+1})$ 也不成立——无需再考虑后续 $x_{i+1},\ldots,x_n$ 的取值。

> 多米诺性质是回溯法提高效率的**核心关键**。

## 二、解空间树与问题建模

### 2.1 基本概念

| 概念 | 定义 |
|------|------|
| **显式约束** | 每个 $x_i$ 的取值集合 $S_i$（可与问题实例有关或无关） |
| **隐式约束** | 描述各 $x_i$ 之间的关联，与具体实例有关 |
| **解空间** | 满足显式约束的所有元组的集合 |
| **解空间树（静态树）** | 将解空间画成树形，与问题实例无关 |
| **问题状态** | 解空间树中的所有结点 |
| **解状态 X** | 由根到结点 X 的路径确定解空间中的一个元组（满足显式约束） |
| **答案状态 X** | 由根到解状态结点 X 的路径确定问题的一个解（同时满足隐式约束） |

> **结论：** 回溯法解决问题的过程 = 在解空间树上搜索答案状态结点的过程

### 2.2 解空间树的两种类型

| 类型 | 结构 | 代表问题 |
|------|------|---------|
| **集合树** | 解是对集合元素的取舍（0/1选择） | 子集和问题、0/1背包问题 |
| **排列树** | 解是对集合元素的排列 | n-皇后问题、图着色问题 |

### 2.3 问题建模步骤

1. 确定元组表达形式（固定长 n-元组 或 不定长 k-元组）
2. 确定显式约束和隐式约束
3. 验证问题满足多米诺性质
4. 确定解空间树（问题状态、解状态、答案状态）
5. 设计约束函数 $B$

## 三、动态树与回溯法设计思想

### 3.1 动态树中的结点类型

- **活结点：** 自己已生成，但儿子结点还未全部生成
- **E-结点（扩展结点）：** 当前正在生成儿子结点的活结点
- **死结点：** 不再扩展或儿子结点已全部生成的结点

### 3.2 两种生成方式

| 方式 | 描述 | 对应算法 |
|------|------|---------|
| **第一种** | E-结点 R 生成儿子 C 后，C 立即成为新 E-结点；C 子树处理完后，R 再次成为 E-结点 | **回溯法**（深度优先） |
| **第二种** | E-结点处理到变成死结点为止，儿子加入活结点表，再从表中选新 E-结点 | **分支限界法** |

### 3.3 回溯法设计思想

1. 针对问题定义解空间树结构（元组、显式/隐式约束）
2. 验证多米诺性质
3. **深度优先**搜索解空间树，用约束函数 $B$ 剪枝
4. 当前 E-结点 R 生成儿子 C → C 成为新 E-结点，搜索纵深移动
5. 若 E-结点不满足约束函数 $B$ 或无儿子结点 → 变为死结点，**回溯**至最近活结点
6. 直到找到所需解或解空间树中无活结点为止

## 四、回溯法通用算法

### 4.1 非递归算法（算法7.1）

```
procedure BACKTRACK(n)
  k ← 1
  while (k > 0) do
    if (还剩有未检验的 X(k) 使得 X(k)∈T(X(1)…X(k-1)) and B(X(1)…X(k))=TRUE)
      then if (X(1)…X(k) 是一条抵达答案结点的路径)
             then print(X(1)…X(k))
           endif
           k ← k+1          // 向纵深扩展
    else k ← k-1             // 回溯
    endif
  repeat
end BACKTRACK
```

### 4.2 递归算法（算法7.2）

```
procedure RBACKTRACK(k)
  // 进入时 X(1)…X(k-1) 已赋值
  for (满足 X(k)∈T(X(1)…X(k-1)) and B(X(1),…X(k))=true 的每个 X(k)) do
    if (X(1),…,X(k) 是一条抵达答案结点的路径) then
      print(X(1)…X(k))
    endif
    call RBACKTRACK(k+1)
  repeat
end RBACKTRACK
```

## 五、回溯法效率分析

### 5.1 决定效率的四个因素

1. **生成下一个 $X(k)$ 的时间**（生成子结点的时间）
2. **满足显式约束的 $X(k)$ 数目**（子结点个数）
3. **约束函数 $B_k$ 的计算时间**（检验子结点的时间）
4. **使 $B_k$ 为真的 $X(k)$ 数目**（通过检验的子结点个数）← 体现不同实例差异

> **约束函数剪枝作用：** 避免生成不包含可行解的子树。$B_k$ 越强，剪枝越多，但计算时间也越长，需权衡。

### 5.2 蒙特卡罗效率估计（算法7.3）

**思想：** 在解空间树中沿随机路径生成一条路径，估计不受限界结点数。

设 $m_i$ = 第 $i$ 级结点平均未受限界的儿子结点数，则：

$$\hat{m} = 1 + m_1 + m_1 m_2 + m_1 m_2 m_3 + \cdots$$

```
Procedure ESTIMATE()
  m ← 1;  r ← 1;  k ← 1
  loop
    T_k ← {X(k): X(k)∈T(X(1),…,X(k-1)) and B_k(X(1),…,X(k))}
    if SIZE(T_k) = 0 then exit endif
    r ← r × SIZE(T_k)          // 第 k 级的结点总数
    m ← m + r                   // 前 k 级的结点总数
    X(k) ← CHOOSE(T_k)         // 从 T_k 中随机挑选一个元素
    k ← k+1
  repeat
  return m
end ESTIMATE
```

**特点：**
- 优点：适用于找所有答案结点的情形；约束函数固定，对同层结点均适用
- 缺点：仅求一个解时实际结点数远小于估计值；随着搜索进行约束函数应更强

## 六、n-皇后问题

### 6.1 问题建模

在 $n \times n$ 棋盘上放 $n$ 个皇后，使任意两个皇后不在同一行、同一列、同一斜角线上。

| 要素 | 内容 |
|------|------|
| **解向量** | n-元组 $(x_1,\ldots,x_n)$，$x_i$ = 第 $i$ 个皇后所在列 |
| **显式约束** | $x_i \in \{1,2,\ldots,n\}$，且任意两个 $x_i$ 不同（不同行、不同列） |
| **隐式约束** | 任意两个皇后不在同一斜角线上 |
| **解空间大小** | $n!$ 个元组 |

### 6.2 斜角线判断

位于 $(i, x_i)$ 和 $(k, x_k)$ 的两个皇后在同一斜角线上，当且仅当：

$$|x_i - x_k| = |i - k|$$

（左斜线：$i - x_i = k - x_k$；右斜线：$i + x_i = k + x_k$）

### 6.3 约束函数 PLACE(k)（算法7.4）

```
procedure PLACE(k)
  // 检验第 k 行皇后放在 X(k) 列是否合法
  i ← 1
  while (i < k) do
    if (X(i) = X(k) or ABS(X(i)-X(k)) = ABS(i-k))
      then return false
    endif
    i ← i+1
  repeat
  return true
end PLACE
```

### 6.4 n-皇后回溯算法（算法7.5）

```
procedure NQUEENS(n)
  X(1) ← 0;  k ← 1
  while (k > 0) do
    X(k) ← X(k)+1
    while (X(k) ≤ n and not PLACE(k)) do X(k) ← X(k)+1 repeat
    if (X(k) ≤ n) then
      if (k = n) then print(X)
      else k ← k+1;  X(k) ← 0     // 准备求解下一个皇后
      endif
    else k ← k-1                     // 没有合适位置，回溯
    endif
  repeat
end NQUEENS
```

### 6.5 效率估计（8-皇后）

- 状态空间树结点数：$1+8+8\times7+\cdots+8! = 109601$
- 蒙特卡罗估计（多次平均）：约 **1625** 个不受限结点
- 比例：$1625 / 109601 \approx \mathbf{1.48\%}$

## 七、子集和问题

### 7.1 问题建模

给定 $n$ 个不同正数 $W(1:n)$ 和目标值 $M$，找出所有使和等于 $M$ 的子集。

| 要素 | 内容 |
|------|------|
| **解向量** | n-元组 $(x_1,\ldots,x_n)$，$x_i \in \{0,1\}$ |
| **显式约束** | $x_i \in \{0,1\}$，$1 \leq i \leq n$ |
| **隐式约束** | $\sum_{i=1}^n w_i x_i = M$ |
| **解空间大小** | $2^n$ 个元组 |

### 7.2 约束函数（需 W 按非降序排列）

设 $s = \sum_{i=1}^{k-1} W(i)X(i)$（当前已选总和），$r = \sum_{i=k}^{n} W(i)$（剩余元素总和）。

约束函数 $B_k(X(1),\ldots,X(k))=\text{true}$ 当且仅当同时满足：

$$\underbrace{s + r}_{\text{当前和 + 剩余所有元素}} \geq M \quad \text{且} \quad \underbrace{s + W(k+1)}_{\text{加入下一元素后}} \leq M$$

- **第一个条件：** 即使加入所有剩余元素，也能凑够 $M$（必要条件：有可能达到）
- **第二个条件（需预排序）：** 当前和加入下一个最小元素不超过 $M$（若连最小的都超了，此分支无解）

### 7.3 递归算法（算法7.6）

初始调用：`SUMOFSUB(0, 1, ∑W(i))`

```
Procedure SUMOFSUB(s, k, r)
  // s = 前 k-1 项已选总和；r = 第 k 到 n 项权值之和
  
  // 生成左儿子（选 W(k)）
  X(k) ← 1
  if s + W(k) = M
    then print(X)                                    // 找到一个解
    else if s + W(k) + W(k+1) ≤ M
           then call SUMOFSUB(s+W(k), k+1, r-W(k))  // 进入左子树
         endif
  endif
  
  // 生成右儿子（不选 W(k)）
  if s + r - W(k) ≥ M and s + W(k+1) ≤ M
    then X(k) ← 0
         call SUMOFSUB(s, k+1, r-W(k))              // 进入右子树
  endif
end SUMOFSUB
```

> **思考题（作业题3(1)）：** 若不将 W 预排序，需将第二个剪枝条件 `s+W(k+1)≤M` 改为 `s+r-W(k)≥M`（只保留左侧条件），无法利用最小元素提前剪枝，效率下降。

### 7.4 作业题3详解

解空间树是完全二叉树，共 $2^4=16$ 个叶结点，$2^4-1=15$ 个内部结点，总共 31 个结点。按深度优先（先左后右）编号：

## 八、图着色问题

### 8.1 问题描述

给定无向连通图 $G=(V,E)$，用 $m$ 种颜色给顶点着色，使任意两个相邻顶点颜色不同。

- **图着色优化问题：** 求最少需要多少种颜色（属于 NP-完全问题）
- **图的 m-着色判定问题：** 给定 $m$，判定是否存在合法的 $m$-着色方案（本节用回溯法求解）

### 8.2 问题建模

| 要素 | 内容 |
|------|------|
| **解向量** | n-元组 $(x_1,\ldots,x_n)$，$x_i$ = 顶点 $i$ 的颜色编号 |
| **显式约束** | $x_i \in \{1, 2, \ldots, m\}$ |
| **隐式约束** | 若顶点 $i$ 和 $j$ 之间有边，则 $x_i \neq x_j$ |
| **解空间大小** | $m^n$ 个叶结点 |

### 8.3 约束函数 OK(k)（算法7.7）

```
procedure OK(k)
  i ← 1
  while (i < k) do
    if (i 和 k 之间有边存在 and C(i) = C(k))
      then return false
    endif
    i ← i+1
  repeat
  return true
end OK
```

### 8.4 图着色回溯算法（算法7.8）

```
Procedure MCOLORING(V, E, C, n, m)
  C(1:n) ← 0;  k ← 1
  while (k ≥ 1)
    C(k) ← C(k) + 1
    while (not OK(k) and C(k) ≤ m) do C(k) ← C(k)+1 repeat
    if C(k) ≤ m then
      if k = n then print(C); return true   // 全部着色成功
      else k ← k+1;  C(k) ← 0             // 准备为下一顶点着色
      endif
    else k ← k-1                            // 无合法颜色，回溯
    endif
  repeat
  return false                               // k=0，遍历完整棵树，无解
end MCOLORING
```

### 8.5 求最小着色数（作业6中的图着色优化）

基于判定算法求最优解：从 $m=1$ 开始逐步增大，直到判定算法返回 true，此时的 $m$ 即为图的**色数**。时间复杂度：$O(n \cdot m^n)$。

## 九、作业题6：工作分配问题

### 9.1 问题描述

$n$ 件工作分配给 $n$ 个人，工作 $i$ 分配给第 $j$ 个人的费用为 $c(i,j)$，求使总费用最小的分配方案。

### 9.2 问题建模

| 要素 | 内容 |
|------|------|
| **解向量** | n-元组 $(x_1,\ldots,x_n)$，$x_i$ = 第 $i$ 件工作分配给的人的编号 |
| **显式约束** | $x_i \in \{1, 2, \ldots, n\}$ |
| **隐式约束** | 任意两个 $x_i$ 互不相同（每人只做一件工作） |
| **目标函数** | 最小化 $\sum_{i=1}^n c(i, x_i)$ |
| **解空间大小** | $n!$（排列树） |

### 9.3 n=4 的解空间树

根 → 第1层：$x_1 \in \{1,2,3,4\}$，4个分支  
第1层 → 第2层：$x_2 \in \{1,2,3,4\} \setminus \{x_1\}$，每个3个分支  
第2层 → 第3层：2个分支；第3层 → 第4层（叶）：1个分支  
叶结点共 $4! = 24$ 个

### 9.4 算法设计

```
procedure JOB_ASSIGN(k, current_cost)
  // 当前已分配前 k-1 件工作，花费 current_cost
  global X(1:n), best_cost, best_X
  
  if k = n+1 then
    if current_cost < best_cost then
      best_cost ← current_cost;  best_X ← X
    endif
    return
  endif
  
  for j ← 1 to n do
    if j 未被使用 then
      X(k) ← j
      // 剪枝：当前费用已超过最优解
      if current_cost + c(k, j) < best_cost then
        call JOB_ASSIGN(k+1, current_cost + c(k, j))
      endif
    endif
  repeat
end JOB_ASSIGN

// 主调用
best_cost ← +∞;  call JOB_ASSIGN(1, 0)
print(best_X, best_cost)
```

## 十、作业题7：最大团问题

### 10.1 问题描述

给定无向图 $G=\langle V, E \rangle$，**团**是一个完全子图（子图内任意两点之间都有边）。求图 $G$ 的**最大团**（包含结点数最多的团）。

**转化思路：** 判定是否存在大小为 $k$ 的团（从 $k=|V|$ 开始向下搜索，找到的第一个有解的 $k$ 即为最大团大小）。

### 10.2 问题建模

**方法：** 枚举顶点子集，判断是否构成团（类似子集和的集合树）。

| 要素 | 内容 |
|------|------|
| **解向量** | n-元组 $(x_1,\ldots,x_n)$，$x_i \in \{0,1\}$（顶点 $i$ 是否在团中） |
| **显式约束** | $x_i \in \{0,1\}$ |
| **隐式约束** | 所有选中的顶点两两之间均有边；且 $\sum x_i = k$（判定大小 $k$） |

**n=5 时判定大小3的团：解空间树结构同子集和问题（完全二叉树，$2^5=32$ 个叶结点）**

### 10.3 约束函数（剪枝条件）

设当前路径为 $(x_1,\ldots,x_k)$，已选顶点集 $S = \{i \mid x_i=1\}$：

1. **可行性剪枝：** 新加入顶点 $k$ 必须与 $S$ 中所有顶点相连，即 $\forall j \in S: (k,j) \in E$
2. **最优性剪枝（上界剪枝）：** 当前已选 $|S|$ 个顶点，剩余未决策顶点有 $(n-k)$ 个，若 $|S| + (n-k) \leq \text{当前最优解大小}$，则剪枝

### 10.4 算法设计

```
procedure MAX_CLIQUE(k, current_clique)
  // current_clique: 当前选中的顶点集合
  global best_clique
  
  if k = n+1 then
    if |current_clique| > |best_clique| then
      best_clique ← current_clique
    endif
    return
  endif
  
  // 上界剪枝：即使后续全选也不如当前最优
  if |current_clique| + (n-k+1) ≤ |best_clique| then return endif
  
  // 选顶点 k（若与 current_clique 中所有顶点相邻）
  if ∀j∈current_clique: (k,j)∈E then
    call MAX_CLIQUE(k+1, current_clique ∪ {k})
  endif
  
  // 不选顶点 k
  call MAX_CLIQUE(k+1, current_clique)
  
end MAX_CLIQUE

// 主调用
best_clique ← {};  call MAX_CLIQUE(1, {})
print(best_clique)
```

## 十三、易错点汇总

| 易错点 | 正确理解 |
|--------|---------|
| **回溯法 vs 分支限界法** | 回溯法是**深度优先**；分支限界法是从活结点表（BFS/优先队列）中选 E-结点。题4(15谜)、题5(带期限作业调度)属于分支限界法！ |
| **静态树 vs 动态树** | 静态树是解空间树（与实例无关）；动态树是回溯过程中实际生成的结点（与实例相关，远少于静态树） |
| **解状态 vs 答案状态** | 解状态满足显式约束（到达叶结点的路径）；答案状态同时满足隐式约束（即问题的解） |
| **子集和中 W 必须预排序** | 利用 $s+W(k+1)\leq M$ 的剪枝条件需要 W 按非降序排列，否则 $W(k+1)$ 不是剩余中最小值，条件失效 |
| **n-皇后解空间大小** | 是 $n!$（满足不同列的排列），而非 $n^n$（允许重复列的情形） |
| **图着色解空间大小** | 叶结点（完全方案）有 $m^n$ 个；总结点数为 $1 + m + m^2 + \cdots + m^n = (m^{n+1}-1)/(m-1)$ |
| **子集和约束函数左子树简化** | 进入左子树时 $B_{k-1}=\text{true}$，已保证 $s+W(k)\leq M$，因此只需检验 $s+W(k)+W(k+1)\leq M$ |
| **蒙特卡罗估计的局限性** | 约束函数固定，不随搜索进行而增强；只求一个解时估计值偏大 |

## 十四、考点速查

| 考点 | 核心内容 |
|------|---------|
| **多米诺性质** | $P(x_1,\ldots,x_{i+1})$ 为真 ⟹ $P(x_1,\ldots,x_i)$ 为真；其逆用于剪枝 |
| **回溯法搜索方式** | 深度优先，约束函数 $B$ 剪枝 |
| **E-结点特征** | 当前正在生成儿子的活结点；生成儿子后儿子立即成为新 E-结点 |
| **蒙特卡罗公式** | $\hat{m} = 1 + m_1 + m_1 m_2 + m_1 m_2 m_3 + \cdots$ |
| **n-皇后约束** | $x_i=x_j$ 或 $|x_i-x_j|=|i-j|$ 时冲突 |
| **子集和约束函数** | $s+r\geq M$ 且 $s+W(k+1)\leq M$（需预排序） |
| **子集和初始调用** | SUMOFSUB(0, 1, $\sum W(i)$) |
| **图着色约束** | 相邻顶点颜色不同，算法时间复杂度 $O(n \cdot m^n)$（含找最优） |
| **解空间树分类** | 集合树（$2^n$叶，子集和/背包）vs 排列树（$n!$叶，皇后/TSP） |
| **回溯 vs 分支限界** | 深度优先 vs 广度优先/最小代价优先；回溯空间需求小 |
