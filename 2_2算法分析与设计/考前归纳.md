    - 渐进上界：如果存在两个正常数 c 和 n₀，对于所有的 n ≥ n₀，有 |f(n)| ≤ c|g(n)|，则记作：**f(n) = O(g(n))**。
    - 渐进下界：如果存在两个正常数 c 和 n₀，对于所有的 n ≥ n₀，有 |f(n)| ≥ c|g(n)|，则记作：**f(n) = Ω(g(n))**。
    - 渐进紧界：如果存在正常数 c₁, c₂ 和 n₀，对于所有的 n ≥ n₀，有 c₁|g(n)| ≤ |f(n)| ≤ c₂|g(n)|，则记作：**f(n) = Θ(g(n))**。

# 4 分治法【算法题】
## 前置知识
- 分治法适用的问题：（1）问题规模较大，难以直接求解；（2）问题可以被分解；（3）子问题的解可以合并。
- 分治法的求解思想：将一个**规模较大、难以直接求解**的问题，分解为若干个**规模较小、结构相同**的子问题，分别求解后再合并成原问题的解。
    - 分：子问题和原问题具有相同特征但规模更小
    - 治：递归地求解每个子问题，直到子问题规模足够小可以直接求解为止。
    - 合：将子问题的解组合成原问题的子解
- 时间计算公式：$g(n)$：直接求解小规模问题的时间，$f(n)$：COMBINE 合并两子问题解的时间
$$T(n) = \begin{cases} g(n) & n \text{ 足够小} \\ 2T(n/2) + f(n) & \text{否则} \end{cases}$$
- 分支法的3种优化策略
    - 子问题个数过多：寻找子问题间的依赖关系，减少需递归求解的子问题数。
    - 递归内工作量过多：将重复性工作提取到递归外**预处理**
- 主定理分析时间复杂度：对于递归式 $T(n) = aT(n/b) + f(n)$（其中 $a \geq 1, b > 1$），若 $f(n) = \Theta(n^d)$，则：$$\boxed{T(n) = \begin{cases} \Theta(n^d) & \text{若 } a < b^d \\ \Theta(n^d \log n) & \text{若 } a = b^d \\ \Theta(n^{\log_b a}) & \text{若 } a > b^d \end{cases}}$$


## 二分查找问题
- 分支法基本思想
    - 分
    - 治
    - 合
- 时间复杂度分析：$$T(n) = \begin{cases}\Theta(1) & n = 1,\ \text{（常数时间）} \\T(n/2) + \Theta(1) & n > 1,\ \text{（常数时间比较）}\end{cases}$$
由主定理，$a=1,\ b=2,\ f(n)=\Theta(1)$，计算比较基准：$$n^{\log_b a} = n^{\log_2 1} = n^0 = 1$$
由于 $f(n) = \Theta(1) = \Theta(n^{\log_b a})$，属于情形 2（$k=0$）：$$\boxed{T(n) = \Theta(\log n)}$$

- 以比较为基础的查找算法时间下界证明
    - n个元素=>至少n个内节点
    - k层级的二叉树（k=根到最远结点的路径长度）=>至多有2^k-1个节点
    - n <= 树的内结点数 <= 2^k-1
    - **结论**：不存在最坏情况下计算时间低于 $\Theta(\log n)$ 的基于比较的查找算法，**二分查找是解决查找问题最坏情况下的最优算法**。

## 排序问题：自顶向下的归并排序
- 分治法基本思想
    - **分（Divide）：** 将 $A(1), \ldots, A(n)$ 平均分为两个子集：$A(1), \ldots, A(\lfloor n/2 \rfloor)$ 和 $A(\lfloor n/2 \rfloor + 1), \ldots, A(n)$
    - **治（Conquer）：** 递归调用，分别将两个子集排序,直到只有1个元素
    - **合（Combine）：** MERGE(low,mid,high)设两个已排好序的子集分别为 $A(\text{low}:\text{mid})$ 和 $A(\text{mid}+1:\text{high})$合并为一个有序集合
- 时间复杂度分析：递推关系为：$$T(n) = \begin{cases} a & n = 1,\ a \text{ 为常数} \\ 2T(n/2) + cn & n > 1,\ c \text{ 为常数} \end{cases}$$，由主定理，$a=2,\ b=2,\ f(n)=\Theta(n)$，$n^{\log_2 2} = n$，属于情形2：$$\boxed{T(n) = O(n \log n)}$$

- 优化策略：自底向上版本**直接从最小子段（长度1）出发**，用变量 `step` 控制当前子段长度，每轮将 `step` 翻倍，逐轮对整个数组从左到右两两合并相邻子段，无递归调用

- 以比较为基础的排序算法时间下界证明
    - $n$ 个关键字共有 $n!$ 种排列，因此比较树至少有 $n!$ 个外节点。设树高（最坏比较次数）为 $T(n)$，则：$2^{T(n)} \geq n!$
    - 推导：$\begin{aligned} 2^{T(n)} &\geq n! \geq n(n-1)(n-2)\cdots\left(\left\lfloor \frac{n}{2} \right\rfloor\right) \geq \left(\frac{n}{2}\right)^{n/2} \end{aligned}$
    - 两边取对数：$T(n) \geq \frac{n}{2} \log \frac{n}{2} \geq \frac{n}{4} \log n \quad (n \geq 4)$
    - $\therefore \quad T(n) = \Omega(n \log n)$

## 二维极大值点问题
- 分治法设计基本思想
    - 分：设计垂直于 $X$ 轴的中位线 $L$，将点集 $S$ 划分为两个子集 $S_L$（左半部分）和 $S_R$（右半部分），每个子集含 $n/2$ 个点

## 假币问题

## 棋盘问题
- 算法思想：将棋盘四等分，黑色方格必在某一个 $2^{k-1} \times 2^{k-1}$ 的子棋盘中。在其余三个子棋盘各自靠近中心的角落**人为放置一个 L 型牌**，使每个子棋盘恰好各有一个"特殊格"（黑格或被 L 型牌覆盖的格），然后对四个子棋盘递归求解。
- 时间复杂度分析：棋盘共 $2^k \times 2^k = 4^k$ 个格，其中 $1$ 个黑格，需要 $(4^k - 1)/3$ 个 L 型牌。每次调用分为 4 个子问题，规模减半：$T(n) = 4T(n/4) + O(1) \Rightarrow T(n) = O(n)$

# 5 贪心法【证明题】
## 前置知识
- 贪心法适用的问题特点：从一组候选输入中，按某种规则逐步构造出一个子集，使该子集既满足约束、又使目标函数取得最优值。
    - 目标函数
    - 约束条件
    - 可行解
    - 最优解：能使目标函数取得极值的可行解
- 贪心法的基本思想：贪心方法是一种**"只顾眼前"**的分级处理方法：
    - 1. 根据题意选取一种**量度标准**，对原问题进行排序操作；
    - 2. 按该标准**一次选中一个输入**；
    - 3. 如果这个输入和当前的部分解加在一起满足约束条件，则将其加入到部分解中；否则舍弃掉。
- 贪心解为最优解的证明
    - 涉及问题
        - 1.背包问题的最优性原理证明
        - 2.带期限的作业排序的贪心解最优性证明以及可调度性证明
        - 3.最小生成树的贪心解最优性证明
    - 证明思路
        - 1.定义贪心解X
        - 2.假设存在更优解Y
        - 3.对比X与Y的差异：根据贪心性质，x_i必然优于y_i
        - 4.构建新解Z：根据差异基于Y去构造更接近于X的解
        - 5.证明Z相较于Y的优越性
        - 6.得出结论，不断使Y接近于X，则解更优，即X为最优

## 多段图问题
- 最优性原理证明
- 求解过程：按照集合划分，V_i
- 向前处理的递推关系式：$$\text{COST}(i, j) = \min_{l \in V_{i+1},\, \langle j,l \rangle \in E} \{ c(j, l) + \text{COST}(i+1, l) \}$$
- 向后处理的递推关系式：$$\text{BCOST}(i, j) = \min_{l \in V_{i-1},\, \langle l,j \rangle \in E} \{ \text{BCOST}(i-1, l) + c(l, j) \}$$
- 时间复杂度分析：$\Theta(n + e)$（$n$ 为结点数，$e$ 为边数）

## 货郎担问题
- 最优性原理证明
- 求解过程：|S|从0递增
- 递推关系式：从 $i$ 出发，下一步去往 $S$ 中的某个结点 $j$，剩余的待访问集合随之缩小为 $S-\{j\}$；枚举所有可能的 $j$，取总成本最小的方案。$$\boxed{g(i, S) = \min_{j \in S} \{ c_{ij} + g(j, S - \{j\}) \}}$$
- 时间复杂度分析：$O(n^2 2^n)$

## 最优二分检索树
- 求解过程：按照j-i=0，1，……，n的顺序，每个情况计算所有满足ij关系的C(i,j)的最小值
- 递推关系式：C(i,j)代表以节点 i到 j为子树构成的最优期望搜索代价​，W(i, j)代表”内节点i+1到j“和”外结点i到j“的概率总和。枚举每个可能的根，递归地求左右子树最优解，然后用 W(i,j)一次性把"所有节点深度+1"的成本补上。
$$\boxed{C(i,j) = \min_{i < k \leq j} \{ C(i, k-1) + C(k, j) \} + W(i, j)}$$ $$W(i,j) = W(i, j-1) + P(j) + Q(j)$$
初始条件：$$C(i,i) = 0, \quad W(i,i) = Q(i), \quad 0 \leq i \leq n$$
- 标记函数：$R(i,j) = $ 使 $C(i,j)$ 取最小值时的 $k$ 值（即最优根结点编号）
- 时间复杂度分析：计算 $C(i,j)$ 时，遍历所有 $k \in (i, j]$，$$\sum_{m=1}^{n} m(n-m+1) = O(n^3)$$

- 优化策略：Knuth定理：$$R(i, j-1) \leq R(i, j) \leq R(i+1, j)$$，即最优根结点的位置随着区间扩张单调不减：$R(i,j)$ 的搜索范围可以从原来的 $(i, j]$ **缩小**到 $[R(i,j-1),\ R(i+1,j)]$。故对所有 $i$ 求和：$\sum_{i=0}^{n-1} O(n) = O(n^2)$，总比较次数降至 $O(n^2)$。

## 矩阵连乘积问题
- 最优性原理证明
- 求解过程：按照子链长度 $w = j - i + 1（w=1,2,……,n）$ 从小到大
- 递推关系式：$A_t$ 是 $p_{t-1} \times p_t$ 的矩阵（$t = 1, \ldots, n$），$m(i,j)$ 为计算矩阵链 $A_i A_{i+1} \cdots A_j$ 所需的**最少标量乘法次数**。$$\boxed{m(i,j) = \begin{cases} 0 & i = j \\ \displaystyle\min_{i \leq k < j} \left\{ m(i, k) + m(k+1, j) + p_{i-1} \cdot p_k \cdot p_j \right\} & i < j \end{cases}}$$
- 时间复杂度分析：$O(n^3)$


## 流水线调度问题
- Johnson算法
    > 把全部 $a_i$ 和 $b_j$（共 $2n$ 个数）合并后按**非降序**排列，依次扫描：
    > - 若当前数是某个 $a_j$，且作业 $j$ 尚未安排：将 $j$ 放入**当前最左的空位**
    > - 若当前数是某个 $b_j$，且作业 $j$ 尚未安排：将 $j$ 放入**当前最右的空位**
    > - 若作业 $j$ 已经安排过，跳过，继续扫描下一个数
    > - 直到 $n$ 个作业全部被安排

# 9 NP问题

## P类问题
- 定义：所有可在**多项式时间**内用**确定**算法求解的判定问题的集合
- 关系：**P ⊆ NP**：P 是 NP 的特例（`choice(S)` 中 $S$ 只有一个元素）。

## NP类问题
- 定义：所有可在**多项式时间**内用**不确定**算法求解的判定问题的集合
- 关系：NP类问题包含NP-完全问题

## NP-完全问题
- 判定条件：① $L \in \text{NP}$；② 对所有 $L' \in \text{NP}$，有 $L' \leq_p L$

## NP-难问题
- 判定条件：对所有 $L' \in \text{NP}$，有 $L' \leq_p L$（不要求 $L \in \text{NP}$）
- 关系：NP-难问题包含NP-完全问题

## 问题实例：SAT可满足性问题
- Cook定理：即任何 NP 问题都可以在多项式时间内归约为 SAT 问题。SAT 是**第一个**被发现的 NP-完全问题（1971年，S. Cook）。
