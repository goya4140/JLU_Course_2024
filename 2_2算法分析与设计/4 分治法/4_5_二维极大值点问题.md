## 五、二维极大点问题（4.5）

### 5.1 问题描述

**基本定义：**

- **支配规则：** 在二维空间中，若 $x_1 > x_2$ 且 $y_1 > y_2$，则称点 $(x_1, y_1)$ **支配**点 $(x_2, y_2)$。
- **极大点：** 若一个点没有被其他任何点支配，则称其为**极大点**。
- **问题目标：** 给定 $n$ 个点的集合 $S$，找出其中所有的极大点。

**直接方法：** 对每一对点进行比较，判断是否存在支配关系，时间复杂度为 $O(n^2)$。

---

### 5.2 分治法设计思想

将分治的三个步骤应用于极大点问题：

- **分（Divide）：** 设计垂直于 $X$ 轴的中位线 $L$，将点集 $S$ 划分为两个子集 $S_L$（左半部分）和 $S_R$（右半部分），每个子集含 $n/2$ 个点
- **治（Conquer）：** 递归分别找出 $S_L$ 和 $S_R$ 的极大点
- **合（Combine）：** 基于支配规则合并 $S_L$ 和 $S_R$ 的极大点

**关键观察：** 由于中位线垂直于 $X$ 轴，$S_L$ 中所有点的 $x$ 值**总是小于** $S_R$ 中所有点的 $x$ 值。因此：

- $S_R$ 的极大点**不可能**被 $S_L$ 中的点支配（$x$ 坐标已经更大）
- $S_L$ 的极大点**可能**被 $S_R$ 中的极大点支配（当其 $y$ 值更小时）

**合并策略：** 对于 $S_L$ 中的每个极大点 $p$，若 $y_p$ 小于 $S_R$ 中所有极大点的最大 $y$ 值，则 $p$ 被支配，予以舍弃；否则保留。

---

### 5.3 问题分析

**三个关键设计决策：**

**① 如何确定中位线？**

中位线理论上可以是任何曲线，但应选择最简单的形式以降低计算代价。对集合 $S$ 中所有点**预先按 $x$ 值排序**，之后每次直接查询排序结果确定中间位置，无需每次重新排序。

**② 如何设计递归出口？**

若集合 $S$ 中只有一个点，则该点必为极大点（无其他点可支配它），直接返回。

**③ 如何高效实现合并中的支配规则？**

$S_R$ 的极大点按 $y$ 值排列后，其最大 $y$ 值即为"门槛"。$S_L$ 中 $y$ 值低于此门槛的极大点全部舍弃，高于门槛的保留。这一步只需 $O(|S_L \text{ 的极大点数}|)$ 的时间，整体合并为 $O(n)$。

---

### 5.4 算法描述

```
procedure MAXPOINTS(S)
    // 前提：S 中各点已按 x 值预排序
    // input：n 个平面点的集合 S
    // output：S 的所有极大点

    if |S| = 1
        then 输出 S 中唯一的点为极大点; return
    endif

    // 分：按 x 值中位线将 S 划分为 SL 和 SR
    mid ← ⌊|S| / 2⌋
    SL ← S 中 x 值排名前 mid 的点的集合
    SR ← S 中 x 值排名后 |S|-mid 的点的集合

    // 治：递归求两个子集的极大点
    call MAXPOINTS(SL)    // 得到 SL 的极大点集 ML
    call MAXPOINTS(SR)    // 得到 SR 的极大点集 MR

    // 合：舍弃 ML 中被 MR 支配的点
    ymax ← MR 中所有极大点的最大 y 值
    for each point p in ML do
        if yp < ymax
            then 舍弃 p   // p 被 SR 中某极大点支配
            else 输出 p 为极大点
        endif
    repeat

    // SR 的极大点全部保留
    输出 MR 中所有点为极大点

end MAXPOINTS
```

**调用方式：** 预先对 $S$ 按 $x$ 值排序后，调用 `MAXPOINTS(S)`。

### 5.5 算法实例【作业题5】

**输入点集：** $(1,2),\ (2,8),\ (3,5),\ (4,1),\ (5,4),\ (6,7),\ (7,6),\ (8,3)$

点集已按 $x$ 值排好序，中位线取 $x = 4.5$，划分为：

$$S_L = \{(1,2),\ (2,8),\ (3,5),\ (4,1)\}, \quad S_R = \{(5,4),\ (6,7),\ (7,6),\ (8,3)\}$$

#### 递归求 $S_L$ 的极大点

$S_L$ 再次二分，中位线取 $x = 2.5$：

$$S_{LL} = \{(1,2),\ (2,8)\}, \quad S_{LR} = \{(3,5),\ (4,1)\}$$

**求 $S_{LL}$ 的极大点：**
- 继续二分：$\{(1,2)\}$ 和 $\{(2,8)\}$，各自只有一个点，均为极大点
- 合并：$S_{LL}$ 的极大点为 $\{(1,2),\ (2,8)\}$；$y_{\max}(\{(2,8)\}) = 8$，$(1,2)$ 的 $y=2 < 8$，舍弃
- $S_{LL}$ 极大点：$\{(2,8)\}$

**求 $S_{LR}$ 的极大点：**
- 继续二分：$\{(3,5)\}$ 和 $\{(4,1)\}$，各自只有一个点，均为极大点
- 合并：$y_{\max}(\{(4,1)\}) = 1$，$(3,5)$ 的 $y=5 > 1$，保留；$(4,1)$ 为 $S_{LR}$ 右半极大点，保留
- $S_{LR}$ 极大点：$\{(3,5),\ (4,1)\}$

**合并 $S_{LL}$ 与 $S_{LR}$ 的极大点：**
- $M_{LL} = \{(2,8)\}$，$M_{LR} = \{(3,5),\ (4,1)\}$
- $y_{\max}(M_{LR}) = 5$，$(2,8)$ 的 $y=8 > 5$，保留
- $S_L$ 极大点：$\{(2,8),\ (3,5),\ (4,1)\}$

#### 递归求 $S_R$ 的极大点

$S_R$ 再次二分，中位线取 $x = 6.5$：

$$S_{RL} = \{(5,4),\ (6,7)\}, \quad S_{RR} = \{(7,6),\ (8,3)\}$$

**求 $S_{RL}$ 的极大点：**
- 继续二分：$\{(5,4)\}$ 和 $\{(6,7)\}$，各自只有一个点，均为极大点
- 合并：$y_{\max}(\{(6,7)\}) = 7$，$(5,4)$ 的 $y=4 < 7$，舍弃
- $S_{RL}$ 极大点：$\{(6,7)\}$

**求 $S_{RR}$ 的极大点：**
- 继续二分：$\{(7,6)\}$ 和 $\{(8,3)\}$，各自只有一个点，均为极大点
- 合并：$y_{\max}(\{(8,3)\}) = 3$，$(7,6)$ 的 $y=6 > 3$，保留；$(8,3)$ 为右半极大点，保留
- $S_{RR}$ 极大点：$\{(7,6),\ (8,3)\}$

**合并 $S_{RL}$ 与 $S_{RR}$ 的极大点：**
- $M_{RL} = \{(6,7)\}$，$M_{RR} = \{(7,6),\ (8,3)\}$
- $y_{\max}(M_{RR}) = 6$，$(6,7)$ 的 $y=7 > 6$，保留
- $S_R$ 极大点：$\{(6,7),\ (7,6),\ (8,3)\}$

#### 合并 $S_L$ 与 $S_R$ 的极大点

- $M_L = \{(2,8),\ (3,5),\ (4,1)\}$，$M_R = \{(6,7),\ (7,6),\ (8,3)\}$
- $y_{\max}(M_R) = 7$
- $(2,8)$：$y=8 > 7$，**保留**
- $(3,5)$：$y=5 < 7$，**舍弃**
- $(4,1)$：$y=1 < 7$，**舍弃**

**最终极大点：** $\{(2,8),\ (6,7),\ (7,6),\ (8,3)\}$

#### 结果汇总

| 点 | 是否为极大点 | 原因 |
|----|------------|------|
| $(1,2)$ | 否 | 被 $(2,8)$ 支配 |
| $(2,8)$ | **是** | 无点支配它 |
| $(3,5)$ | 否 | 被 $(6,7)$ 支配 |
| $(4,1)$ | 否 | 被 $(6,7)$ 支配 |
| $(5,4)$ | 否 | 被 $(6,7)$ 支配 |
| $(6,7)$ | **是** | 无点支配它 |
| $(7,6)$ | **是** | 无点支配它 |
| $(8,3)$ | **是** | 无点支配它 |

### 5.6 时间复杂度分析

设使用预排序优化后的时间复杂度递推关系为：

$$T(n) = \begin{cases} 1 & n = 1 \\ 2T(n/2) + O(n) + O(n) & n > 1 \end{cases}$$

其中两个 $O(n)$ 分别来自：
- **分：** 按预排序结果确定中位线并划分子集，$O(n)$
- **合：** 基于支配规则扫描 $S_L$ 的极大点，$O(n)$

由主定理，$a=2,\ b=2,\ f(n)=\Theta(n)$，$n^{\log_2 2} = n$，属于情形2：

$$\boxed{T(n) = O(n \log n)}$$

加上预排序的 $O(n \log n)$，总时间复杂度仍为 $O(n \log n)$，优于直接法的 $O(n^2)$。

> **优化关键：** 将排序工作提取到递归过程之外作为预处理，避免在每次递归内部重复排序，将分治内部的分/合代价从 $O(n \log n)$ 降至 $O(n)$。这是分治法优化策略2（减少递归内部工作量）的典型应用。
