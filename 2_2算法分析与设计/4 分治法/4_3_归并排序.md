## 三、归并排序（4.3）

### 3.1 问题描述与直接插入法

**问题描述：** 给定一个含有 $n$ 个元素的集合，要求把它们按非降次序排列。

**一般方法（直接插入法）：**

```
for j ← 2 to n do
    将 A(j) 放入已排序的 A(1:j-1) 中
repeat
```

| 情况 | 时间复杂度 |
|------|-----------|
| 最好情况 | $O(n)$ |
| 最坏情况 | $O(n^2)$ |

直接插入法在最坏情况下效率较低，考虑用分治法改进。

---

### 3.2 分治策略设计归并排序

归并排序将分治的三个步骤应用于排序问题：

- **分（Divide）：** 将 $A(1), \ldots, A(n)$ 平均分为两个子集：$A(1), \ldots, A(\lfloor n/2 \rfloor)$ 和 $A(\lfloor n/2 \rfloor + 1), \ldots, A(n)$
- **治（Conquer）：** 递归调用，分别将两个子集排序
- **合（Combine）：** 将两个已排好序的子集合并为一个有序集合

---

### 3.3 MERGESORT 算法描述

```
procedure MERGESORT(low, high)
    integer low, high, mid;
    if low < high
        then mid ← ⌊(low + high) / 2⌋
             call MERGESORT(low, mid)
             call MERGESORT(mid + 1, high)
             call MERGE(low, mid, high)
    endif
end MERGESORT
```

**调用方式：** 对数组 $A(1:n)$ 排序时，调用 `MERGESORT(1, n)`。

> **注意：** 递归出口为 `low ≥ high`，即子集只含单个元素时直接返回（单个元素天然有序）。

---

### 3.4 MERGE 合并算法

#### 算法思想

设两个已排好序的子集分别为 $A(\text{low}:\text{mid})$ 和 $A(\text{mid}+1:\text{high})$，用指针 $i_{\text{Left}}$ 和 $i_{\text{Right}}$ 分别指向两段的当前元素，$i_{\text{Write}}$ 指向辅助数组 $B$ 的当前写入位置：

1. 若 $i_{\text{Left}} \leq \text{mid}$ 且 $i_{\text{Right}} \leq \text{high}$，则 $B(i_{\text{Write}}) \leftarrow \min(A(i_{\text{Left}}), A(i_{\text{Right}}))$，对应指针前移
2. 若 $i_{\text{Left}} > \text{mid}$ 且 $i_{\text{Right}} \leq \text{high}$，则将 $A(i_{\text{Right}}:\text{high})$ 的剩余元素依次写入 $B$
3. 若 $i_{\text{Left}} \leq \text{mid}$ 且 $i_{\text{Right}} > \text{high}$，则将 $A(i_{\text{Left}}:\text{mid})$ 的剩余元素依次写入 $B$

最后将 $B(\text{low}:\text{high})$ 复制回 $A(\text{low}:\text{high})$。

#### 算法描述

```
procedure MERGE(low, mid, high)
    integer iLeft, iRight, iWrite, k, low, mid, high;
    global A(low : high);
    local B(low : high);
    
    // 初始化指针
    iLeft  ← low;      // 左段起始位置
    iWrite ← low;      // 写入位置从 low 开始
    iRight ← mid + 1;  // 右段起始位置
    
    // 两段均未耗尽时，取较小值写入 B，一直持续，直到其中一段被完全取完
    while iLeft ≤ mid and iRight ≤ high do
        if A(iLeft) ≤ A(iRight)
            then B(iWrite) ← A(iLeft);  iLeft  ← iLeft  + 1
            else B(iWrite) ← A(iRight); iRight ← iRight + 1
        endif
        iWrite ← iWrite + 1
    repeat
    
    // 处理剩余元素（逻辑不变，变量名替换）
    if iLeft > mid  then //左半段已空，处理右半段
        for k ← iRight to high do
            B(iWrite) ← A(k);  iWrite ← iWrite + 1
        repeat
    else    //右半段已空，处理左半段
        for k ← iLeft to mid do
            B(iWrite) ← A(k);  iWrite ← iWrite + 1
        repeat
    endif
    
    // 将 B 复制回 A
    for k ← low to high do
        A(k) ← B(k)
    repeat
end MERGE
```

**MERGE 的时间复杂度：** 合并两个共 $n$ 个元素的有序子集，元素比较次数在 $\lceil n/2 \rceil$ 到 $n-1$ 之间，故 MERGE 的时间复杂度为 $O(n)$。

---

### 3.5 算法实例（$n = 5$）

以 $A = [6, 3, 9, 4, 1]$ 为例，展示递归调用树：

```
MERGESORT 调用树（节点标注 low, high）：

              (1,5)
            /       \
         (1,3)      (4,5)
         /   \      /   \
       (1,2) (3,3)(4,4) (5,5)
       /   \
     (1,1) (2,2)
```

MERGE 调用顺序（标注 low, mid, high）：

| 调用顺序 | low | mid | high | 合并内容 |
|---------|-----|-----|------|---------|
| 1 | 1 | 1 | 2 | $[6]$ 与 $[3]$ → $[3,6]$ |
| 2 | 1 | 2 | 3 | $[3,6]$ 与 $[9]$ → $[3,6,9]$ |
| 3 | 4 | 4 | 5 | $[4]$ 与 $[1]$ → $[1,4]$ |
| 4 | 1 | 3 | 5 | $[3,6,9]$ 与 $[1,4]$ → $[1,3,4,6,9]$ |

> **思考：** 算法的调用过程（递归树结构）是否受问题实例（输入数据的具体值）影响？
>
> **结论：** 不受影响。MERGESORT 的递归树结构完全由 $n$ 决定，与数组中具体数值无关，因此归并排序在最好、最坏和平均情况下的时间复杂度相同。

---

### 3.6 归并排序的时间复杂度

设归并排序的时间复杂度递推关系为：

$$T(n) = \begin{cases} a & n = 1,\ a \text{ 为常数} \\ 2T(n/2) + cn & n > 1,\ c \text{ 为常数} \end{cases}$$

其中 $cn$ 为 MERGE 合并一次的时间代价。

**求解（以 $n = 2^k$ 为例）：**

$$\begin{aligned} T(n) &= 2T(n/2) + cn \\ &= 2\bigl(2T(n/4) + c \cdot n/2\bigr) + cn = 2^2 T(n/4) + 2cn \\ &= 2^3 T(n/8) + 3cn \\ &\vdots \\ &= 2^k T(1) + kcn \\ &= an + cn\log n \end{aligned}$$

若 $2^k < n \leq 2^{k+1}$，则 $T(n) \leq T(2^{k+1})$，故：

$$\boxed{T(n) = O(n \log n)}$$

**归并排序最好、最坏和平均情况下，时间复杂度均为 $O(n \log n)$。**

---

### 3.7 以比较为基础的排序算法时间下界

**定理：** 任何以关键字比较为基础的排序算法，最坏情况下的时间下界都是 $\Omega(n \log n)$。

因此从数量级角度看，**归并排序是最坏情况下的最优算法**。

#### 证明思路

用二元比较树描述以比较为基础的排序算法：

- 树的每个**内节点**表示一次关键字比较 $A(i) : A(j)$
- 树的每个**外节点（叶节点）** 对应一种可能的排列结果

**关键事实：** $n$ 个关键字共有 $n!$ 种排列，因此比较树至少有 $n!$ 个外节点。设树高（最坏比较次数）为 $T(n)$，则：

$$2^{T(n)} \geq n!$$

**推导：**

$$\begin{aligned} 2^{T(n)} &\geq n! \geq n(n-1)(n-2)\cdots\left(\left\lfloor \frac{n}{2} \right\rfloor\right) \geq \left(\frac{n}{2}\right)^{n/2} \end{aligned}$$

两边取对数：

$$T(n) \geq \frac{n}{2} \log \frac{n}{2} \geq \frac{n}{4} \log n \quad (n \geq 4)$$

（其中用到 $n \geq 4$ 时 $n/2 \geq n^{1/2}$，故 $\log(n/2) \geq \frac{1}{2}\log n$）

$$\therefore \quad T(n) = \Omega(n \log n)$$

### 3.8 归并排序的优化

归并排序存在两类主要开销，对应两种优化方向：

#### 优化1：减少递归时间（针对小规模子问题）

**问题：** 子集规模很小时，递归调用本身的开销相对较大。

**策略：** 当子集元素个数小于某阈值时，切换为直接插入算法，避免递归开销。

#### 优化2：减少辅助空间（取消辅助数组 $B$）

**问题：** MERGE 使用了辅助数组 $B(1:n)$，且每次调用 MERGE 时需将 $B$ 中结果复制回 $A$，消耗额外时间和空间。

**策略：** 用**链接数组** $\text{LINK}(1:n)$ 代替辅助数组 $B$。$\text{LINK}(i)$ 存储 $A$ 中当前元素的下一个元素的下标（即链表指针），避免元素的实际移动和复制。

**算法描述：**
ERGESORT_LINK 算法描述

```
procedure MERGESORT_LINK(low, high)
    integer low, high, mid, headLeft, headRight;
    global A(1:n), LINK(1:n);
    
    if low ≥ high
        then return(low)
        else mid ← ⌊(low + high) / 2⌋
             headLeft  ← MERGESORT_LINK(low, mid)
             headRight ← MERGESORT_LINK(mid + 1, high)
             return(MERGELIST(headLeft, headRight))
    endif
end MERGESORT_LINK
```
MERGELIST 算法描述（逐行对照 MERGE）
```
procedure MERGELIST(headLeft, headRight)
    integer pLeft, pRight, pTail, head;
    global A(1:n), LINK(1:n);
    
    pLeft  ← headLeft;   // 左段当前指针，对应 iLeft
    pRight ← headRight;  // 右段当前指针，对应 iRight
    
    // ---- 确定合并后链表的头节点 head，并令 pTail 指向它 ----
    if A(pLeft) ≤ A(pRight)
        then head ← pLeft;  pTail ← pLeft;  pLeft  ← LINK(pLeft)
        else head ← pRight; pTail ← pRight; pRight ← LINK(pRight)
    endif
    
    // ---- 两段均未耗尽时，取较小值接到 pTail 之后 ----
    // 与 MERGE 的 while 循环结构完全一致，0 表示链表结束（对应 MERGE 中下标超出范围）
    while pLeft ≠ 0 and pRight ≠ 0 do
        if A(pLeft) ≤ A(pRight)
            then LINK(pTail) ← pLeft;  pTail ← pLeft;  pLeft  ← LINK(pLeft)
            else LINK(pTail) ← pRight; pTail ← pRight; pRight ← LINK(pRight)
        endif
    repeat
    
    // ---- 处理剩余元素 ----
    // MERGE 中用 for 循环逐个复制剩余段；
    // 这里剩余段本身已是一条有序链，只需把 pTail 直接接到该链的头部即可
    if pLeft = 0
        then LINK(pTail) ← pRight   // 左段已空，接入右段剩余链
        else LINK(pTail) ← pLeft    // 右段已空，接入左段剩余链
    endif
    
    // ---- 没有"复制回 A"这一步：数据始终留在原位，只是 LINK 改变了遍历顺序 ----
    
    return(head)
end MERGELIST
```


**示例（$n=5$，$A = [16, 13, 19, 14, 11]$）：**

排序后链表（起始指针指向下标 5）：

| $A$ 下标 | 1 | 2 | 3 | 4 | 5 |
|---------|---|---|---|---|---|
| $A$ 值 | 16 | 13 | 19 | 14 | 11 |
| LINK | 3 | 4 | 0 | 1 | 2 |

遍历链表（从起始指针 5 出发）：$5 \to 2 \to 4 \to 1 \to 3 \to 0$（终止），对应元素值 $11, 13, 14, 16, 19$，即排序结果。

#### 优化3：自底向上的归并排序（取消栈空间）【作业题4】

**问题：** 递归实现的归并排序需要系统栈空间，栈深度为 $O(\log n)$。

**策略：** 改用**自底向上**的迭代实现，直接从长度为 1 的子段开始逐轮合并，完全取消对系统栈空间的依赖。
- 核心思想
    - 递归版归并排序"自顶向下"分解问题，依赖系统栈记录每层的 $(low, high)$，栈深度 $O(\log n)$
    - 自底向上版本**直接从最小子段（长度1）出发**，用变量 `step` 控制当前子段长度，每轮将 `step` 翻倍，逐轮对整个数组从左到右两两合并相邻子段，无递归调用，栈空间 $O(1)$

- 算法结构
    - **外层循环：** `step` 从 $1$ 增长到 $\geq n$，共 $\lceil \log_2 n \rceil$ 轮，每轮子段长度翻倍
    - **内层循环：** 每轮内从左到右扫描数组，每次合并 $[\text{low},\ \text{mid}]$ 和 $[\text{mid}+1,\ \text{high}]$ 两段，然后 $\text{low} \leftarrow \text{high}+1$ 跳到下一对

- 边界处理（关键，易错）
    - **内层循环进入条件：** `low + step - 1 < n`，即右段至少存在1个元素，否则当前子段已是最后一段无需合并
    - **mid 的计算：** $\text{mid} \leftarrow \text{low} + \text{step} - 1$，固定为左段的末尾
    - **high 的计算：** $\text{high} \leftarrow \min(\text{mid} + \text{step},\ n)$，用 $\min$ 防止右段越界（当 $n$ 不是2的幂次时，最后一段可能不足 `step` 长度）

- MERGE 子程序
    - **完全不变**，直接复用原 MERGE（`iLeft / iRight / iWrite` 版本），自底向上只改变了**调用 MERGE 的顺序和参数**，MERGE 内部逻辑不受影响


**算法描述：**

```
procedure MERGESORT_BU(n)
    integer step, low, mid, high;
    global A(1:n);
    
    step ← 1;   // 当前待合并子段的长度
    
    // 当 step ≥ n 时，整个数组已是一个有序段，结束
    while step < n do
        low ← 1;
        
        // 在本轮内，从左到右扫描，每次合并相邻的两个长度为 step 的子段
        while low + step ≤ n do   // 确保右段至少存在1个元素
            mid  ← low + step - 1;            // 左段范围 [low, mid]
            high ← min(mid+step, n);  // 右段范围 [mid+1, high]，注意最后一段可能不足 step
            call MERGE(low, mid, high);
            low ← low + 2 * step;             // 跳到下一对待合并子段
        repeat
        
        step ← step * 2;   // 子段长度翻倍，进入下一轮
    repeat
end MERGESORT_BU
```

**执行过程示例（$A = [6, 3, 9, 4, 1]$）：**

| 轮次 | step | 合并操作 | 结果 |
|------|------|---------|------|
| 第1轮 | 1 | MERGE(1,1,2)、MERGE(3,3,4)、A(5)单独留 | $[3,6,\ 4,9,\ 1]$ |
| 第2轮 | 2 | MERGE(1,2,4)、A(5)单独留 | $[3,4,6,9,\ 1]$ |
| 第3轮 | 4 | MERGE(1,4,5) | $[1,3,4,6,9]$ |

**时间与空间复杂度：**

| 指标 | 递归归并排序 | 自底向上归并排序 |
|------|------------|----------------|
| 时间复杂度 | $O(n \log n)$ | $O(n \log n)$ |
| 辅助数组空间 | $O(n)$ | $O(n)$ |
| 栈空间 | $O(\log n)$ | $O(1)$ |

自底向上归并排序在时间复杂度上与递归版本相同，但完全消除了递归栈的空间开销，在栈空间受限的场景下更为实用。

### 3.9 归并排序小结

| 指标 | 结果 |
|------|------|
| 最好情况时间复杂度 | $O(n \log n)$ |
| 最坏情况时间复杂度 | $O(n \log n)$ |
| 平均情况时间复杂度 | $O(n \log n)$ |
| 辅助空间 | $O(n)$（辅助数组） |
| 是否稳定 | 是（$A(h) \leq A(j)$ 时优先取左段） |
