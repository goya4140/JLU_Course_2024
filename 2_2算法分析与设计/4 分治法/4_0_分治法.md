# 第四章 分治法 — 复习资料

## 一、分治法一般方法（4.1）

### 1.1 核心思想

分治法的核心是将一个**规模较大、难以直接求解**的问题，分解为若干个**规模较小、结构相同**的子问题，分别求解后再合并成原问题的解。

三个关键步骤：

- **分（Divide）**：将原问题分解为 k 个规模更小的子问题，子问题与原问题具有相同的结构。
- **治（Conquer）**：递归地求解每个子问题，直到子问题规模足够小可以直接求解为止。
- **合（Combine）**：将各子问题的解合并，得到原问题的解。

> 分治法天然适合递归实现。

### 1.2 DANDC (Divide AND Conquer) 抽象控制框架

DANDC 是分治法的通用抽象框架，适用于将问题 A(1:n) 分为两个子问题的情形。

```
procedure DANDC(p, q)
  global n, A(1:n); integer m, p, q;
  // 1 ≤ p ≤ q ≤ n
  if SMALL(p, q)
    then return(G(p, q))
    else m ← DIVIDE(p, q)
         return(COMBINE(DANDC(p, m), DANDC(m+1, q)))
  endif
end DANDC
```

各函数说明：

| 函数 | 作用 |
|---|---|
| `SMALL(p,q)` | 判断当前规模 q-p+1 是否足够小，可直接求解 |
| `G(p,q)` | 直接求解小规模问题 |
| `DIVIDE(p,q)` | 将问题分割为两个子问题，返回分割点 m |
| `COMBINE(s1, s2)` | 将两个子问题的解合并为原问题的解 |

调用方式：原问题为 A(1:n)，调用 `DANDC(1, n)`。

### 1.3 计算时间递归式

当两个子问题规模大致相等时，DANDC 的计算时间为：

>$$T(n) = \begin{cases} g(n) & n \text{ 足够小} \\ 2T(n/2) + f(n) & \text{否则} \end{cases}$$

- $g(n)$：直接求解小规模问题的时间
- $f(n)$：COMBINE 合并两子问题解的时间

---

### 【作业题1】求解递归关系式

**题目**：求解以下递归关系式，并分析：

$$T(n) = \begin{cases} g(n) & n \text{ 足够小} \\ 2T(n/2) + f(n) & \text{否则} \end{cases}$$

利用逐层展开法（设 $n = 2^k$）

**情况①**：$g(n) = O(1)$，$f(n) = O(n)$

**解**：：

$$T(n) = 2T(n/2) + cn$$
$$= 2(2T(n/4) + cn/2) + cn = 4T(n/4) + 2cn$$
$$= 2^k T(1) + kcn$$
$$= an + cn\log n = O(n\log n)$$

**结论**：$T(n) = O(n\log n)$

**情况②**：$g(n) = O(1)$，$f(n) = O(1)$

**解**：

$$T(n) = 2T(n/2) + c$$
$$= 2(2T(n/4) + c) + c = 4T(n/4) + 3c$$
$$= 2^k T(1) + (2^k - 1)c$$
$$= an + (n-1)c = O(n)$$

**结论**：$T(n) = O(n)$

**分析**：合并操作的代价 $f(n)$ 直接决定了算法整体效率。$f(n) = O(n)$ 时，每层的合并开销相同，故总代价为 $O(n\log n)$；$f(n) = O(1)$ 时，合并几乎没有开销，总代价仅为 $O(n)$。

## 二、二分查找（4.2）

### 2.1 问题描述

已知一个**非降序排列** $a_1, a_2, \ldots, a_n$，判定给定元素 $x$ 是否在其中。若是，将下标赋给变量 $j$；否则，令 $j = 0$。

### 2.2 分治法建模

将查找问题表示为 $I = (n, a_1, \ldots, a_n, x)$：

- **分**：选取中间下标 $k = \lfloor(low+high)/2\rfloor$，将问题分为三个子问题：
  - $I_1 = (k-1,\ a_1,\ \ldots,\ a_{k-1},\ x)$
  - $I_2 = (1,\ a_k,\ x)$
  - $I_3 = (n-k,\ a_{k+1},\ \ldots,\ a_n,\ x)$
- **治**：
  - 若 $x = A[k]$，令 $j = k$，查找成功，结束。
  - 若 $x < A[k]$，在子问题 $I_1$（左半部分）中继续查找。
  - 若 $x > A[k]$，在子问题 $I_3$（右半部分）中继续查找。
- **合**：当前子问题的解即为原问题的解，无需额外合并步骤。

> 为什么可以忽略另外两个子问题？因为数组已排序，$x < A[k]$ 时 $x$ 不可能在 $k$ 之后，$x > A[k]$ 时亦然，每次查找只需进入一个子问题。

---

### 2.3 BINSRCH 算法（迭代版）

```
procedure BINSRCH(A, n, x, j)
  integer low, high, mid, j, n;
  low ← 1;  high ← n
  while low ≤ high do
    mid ← ⌊(low + high) / 2⌋      // 取中间下标
    case
      : x < A(mid) : high ← mid - 1  // 搜索前半段
      : x > A(mid) : low  ← mid + 1  // 搜索后半段
      : else        : j ← mid; return // 查找成功
    endcase
  repeat
  j ← 0                              // 查找失败
end BINSRCH
```

---

### 2.4 算法正确性证明

证明需覆盖参数的每类取值，以及算法的每个分支：

- **$n = 0$**：不进入循环，直接令 $j = 0$，算法正确终止。
- **成功情况**：若 $x = A[mid]$，则 $j = mid$，查找成功终止。若 $x < A(mid)$，则 $x$ 不可能在 $mid \sim high$ 范围内，查找范围缩小到 $low \sim mid-1$；$x > A(mid)$ 时亦然。
- **不成功情况**：`low`、`high`、`mid` 均为整型变量，按算法描述的方式缩小查找范围，总可以在有限步内使 $low > high$，说明 $x$ 不在 $A$ 中，退出循环，令 $j = 0$，算法正确终止。

---

### 2.5 算法实例

以 $n = 9$，数组 $A = [-15,\ -6,\ 0,\ 7,\ 9,\ 23,\ 54,\ 82,\ 101]$ 为例：

| 查找 $x$ | 比较序列 | 比较次数 | 结果 |
|---|---|---|---|
| $x = 9$（$A[5]$） | ①$A[5]$ | 1 次 | 成功 |
| $x = -15$（$A[1]$） | ①$A[5]$ → ②$A[2]$ → ③$A[1]$ | 3 次 | 成功 |
| $x = 101$（$A[9]$） | ①$A[5]$ → ②$A[7]$ → ③$A[8]$ → ④$A[9]$ | 4 次 | 成功 |
| $x = 8$（不存在） | ①$A[5]$ → ②$A[7]$ → ③$A[6]$ → ④$A[4]$ → 失败 | 4 次 | 失败 |

> 思考：为什么对称位置的元素（如 $A[1]$ 和 $A[9]$）比较次数不同？因为 mid 取下整 $\lfloor(low+high)/2\rfloor$，树并非严格对称，右侧子树在某些情况下会多一层。

### 2.6 时间复杂度分析

#### 2.6.1 二元比较树

算法的执行过程实质上是 $x$ 与一系列中间元素 $A(mid)$ 的比较过程，可以用**二元比较树**来具象描述：

- **内节点**（圆形）：记录一个 mid 值，表示一次元素比较，对应一种成功查找情况。
- **外节点**（方形）：表示一种查找失败的情况，共 $n+1$ 种。
- **从根到某节点的路径**：表示一次查找所经历的全部比较序列。
- **成功查找**在内节点终止；**失败查找**在外节点终止。

仍以 $n = 9$ 为例，二元比较树结构如下：

```
              (5)                      ← 第1级
            /     \
          (2)     (7)                  ← 第2级
         /   \   /   \
        (1) (3) (6) (8)               ← 第3级
              \       \
              (4)     (9)             ← 第4级
```

外节点（方形）挂在每个内节点的空子树位置，共 10 个，分布在第 4、5 级。

> **思考：二元比较树的结构受问题实例影响吗？** 不受影响。树的结构只取决于 $n$，与数组的具体数值无关，对所有大小为 $n$ 的查找问题，二元比较树的形状完全相同。

#### 2.6.2 通过具体例子计算平均比较次数

**成功查找**（9 个内节点，各自所在级数即为比较次数）：

| 下标 | 1 | 2 | 3 | 4 | 5 | 6 | 7 | 8 | 9 |
|---|---|---|---|---|---|---|---|---|---|
| 级数（比较次数） | 3 | 2 | 3 | 4 | 1 | 3 | 2 | 3 | 4 |

成功查找总次数 $= 1+2+2+3+3+3+3+4+4 = 25$，平均：

$$S(9) = \frac{25}{9} \approx 2.78 \text{ 次}$$

**失败查找**（10 个外节点，比较次数 = 所在级数 $-1$，分布在第 4、5 级）：

失败查找总次数 $= 3+3+3+4+4+3+3+3+4+4 = 34$，平均：

$$U(9) = \frac{34}{10} = 3.4 \text{ 次}$$

汇总：

| 情况 | 最好 | 最坏 | 平均 |
|---|---|---|---|
| 成功查找 | 1 次 | 4 次 | $25/9 \approx 2.78$ |
| 失败查找 | 3 次 | 4 次 | $34/10 = 3.4$ |

#### 2.6.3 内外路径长度的关系与 $E = I + 2n$ 的证明

**定义**：
- **内部路径长度 $I$**：根到所有内节点的距离之和。
- **外部路径长度 $E$**：根到所有外节点的距离之和。

**定理（思考题）：$E = I + 2n$**

**证明**（对 $n$ 归纳）：

- **基础情况** $n = 0$：树只有一个外节点（根），$E = 0$，$I = 0$，$E = I + 0 = I + 2 \times 0$ ✓
- **归纳步骤**：设对所有节点数小于 $n$ 的二叉树结论成立。对于有 $n$ 个内节点的二叉树，设根的左子树有 $k$ 个内节点，右子树有 $n-k-1$ 个内节点（$0 \leq k \leq n-1$）。

  设左子树的内、外部路径长度分别为 $I_L$、$E_L$，右子树为 $I_R$、$E_R$。由于从根到子树中每个节点的距离都比子树内距离多 1，故：

  $$I = I_L + I_R + (n-1), \quad E = E_L + E_R + (n+1)$$

  由归纳假设 $E_L = I_L + 2k$，$E_R = I_R + 2(n-k-1)$，代入得：

  $$E = (I_L + 2k) + (I_R + 2(n-k-1)) + (n+1) = I_L + I_R + 2n - 2 + n + 1 = I + 2n \quad \checkmark$$

由此推导成功与失败查找平均比较次数的关系：

$$S(n) = \frac{I}{n} + 1, \quad U(n) = \frac{E}{n+1}$$

由 $E = I + 2n$，得 $(n+1) \cdot U(n) = n \cdot (S(n)-1) + 2n$，整理得：

$$\boxed{S(n) = \frac{n+1}{n} \cdot U(n) - 1 = \left(1 + \frac{1}{n}\right) U(n) - 1}$$

因此 $S(n) = \Theta(U(n))$，两者同阶，只需分析一个即可得到另一个。

**以 $n = 9$ 验证**：$S(9) = \dfrac{10}{9} \times 3.4 - 1 = \dfrac{34}{9} - 1 = \dfrac{25}{9} \approx 2.78$ ✓

#### 2.6.4 定理 4.1：BINSRCH 时间复杂度

**定理 4.1**：若 $n$ 在区间 $[2^{k-1},\ 2^k)$ 中，则对于一次成功的查找，BINSRCH 至多作 $k$ 次比较；对于一次不成功的查找，或者作 $k-1$ 次比较，或者作 $k$ 次比较。

**证明**：

当 $2^{k-1} \leq n < 2^k$ 时，考察二元比较树：
- 内节点最大级数 $\leq k$（前 $k-1$ 层必为满二叉树）。
- 外节点级数在 $k$ 级或 $k+1$ 级。
- 成功查找比较次数 = 内节点级数 $\leq k$。
- 失败查找比较次数 = 外节点级数 $- 1$，即 $k-1$ 或 $k$。

**推论**：此外，由 $2^{k-1} \leq n < 2^k$ 得：

$$k - 1 \leq \log n < k \implies k = \lfloor \log n \rfloor + 1, \quad k = \Theta(\log n)$$
又因为：
$$S(n) = \frac{I}{n} + 1, \quad U(n) = \frac{E}{n+1}，且 I =\Theta(n\log n)$$

完整时间复杂度汇总：

| 计算时间 | 最好情况 | 最坏情况 | 平均情况 |
|---|---|---|---|
| 成功查找 | $\Theta(1)$ | $\Theta(\log n)$ | $\Theta(\log n)$ |
| 失败查找 | $\Theta(\log n)$ | $\Theta(\log n)$ | $\Theta(\log n)$ |

**空间复杂度**：$n$ 个位置存放数组 $A$，另有 `low, high, mid, x, j` 五个变量，共 $n + 5 = \Theta(n)$。

#### 2.6.5 定理 4.2：查找问题的时间下界

至此，自然引出一个问题：**能否设计出最坏情况下优于二分查找的算法？**

**定理 4.2**：设 $A(1:n)$ 含有 $n \geq 1$ 个不同的元素，排序为 $A(1) < \cdots < A(n)$。以比较为基础判断是否 $x \in A(1:n)$ 的任何算法，在最坏情况下所需的最小比较次数满足：

$$\text{FIND}(n) \geq \lceil \log(n+1) \rceil$$

**证明**：对任意基于比较的查找算法，都可以用一棵二元比较树描述其执行过程。该树必须包含至少 $n+1$ 个外节点（对应 $x$ 不在 $A$ 中的 $n+1$ 种情况）。若树高（内节点最大级数）为 $k$，则外节点数最多为 $2^k$，故 $2^k \geq n+1$，即 $k \geq \lceil\log(n+1)\rceil$。最坏情况下的比较次数即为树高 $k$，故 $\text{FIND}(n) \geq \lceil\log(n+1)\rceil$。证毕。

**结论**：不存在最坏情况下计算时间低于 $\Theta(\log n)$ 的基于比较的查找算法，**二分查找是解决查找问题最坏情况下的最优算法**。

### 2.7 BINSRCH 变型

定理 4.1 分析了 BINSRCH 的时间复杂度，但这里需要进一步思考：**二分思想下查找算法的时间复杂度是固定不变的吗？会受算法实现细节影响吗？**

原版 BINSRCH 使用 `case` 语句，每次循环最多比较**两次**（先判断 $<$，再判断 $>$）。当 $x$ 恰好等于 $A[mid]$ 时提前返回，但在其他情况下实际做了两次比较。改用嵌套 `if`，可以让循环内**每次只做一次比较**，将等号判断推迟到循环结束后：

```
procedure BINSRCH1(A, n, x, j)
  integer low, high, mid, j, n;
  low ← 1;  high ← n + 1    // high 总比可能取值大 1
  while low < high - 1 do
    mid ← ⌊(low + high) / 2⌋
    if x < A(mid)
      then high ← mid        // 搜索前半段
      else low  ← mid        // x ≥ A(mid)，含等号情况
    endif
  repeat
  if x = A(low)
    then j ← low             // 退出循环后再判等一次
    else j ← 0
end BINSRCH1
```

**思考题：证明 BINSRCH1 的最好、最坏和平均情况时间对于成功和不成功的查找都是 $\Theta(\log n)$**

**证明**：

BINSRCH1 在循环内每次只做一次比较，且循环条件为 $low < high - 1$，即区间长度大于 1 时继续循环。每次迭代后区间长度约减半，故循环次数恰好为 $\lfloor\log n\rfloor$ 次，循环结束后额外做一次判等。

- **成功查找**：无论目标在何位置，循环始终执行至区间长度为 1，比较次数固定为 $\lfloor\log n\rfloor + 1$，最好、最坏、平均均为 $\Theta(\log n)$。
- **失败查找**：同理，循环执行至区间为空，比较次数同为 $\lfloor\log n\rfloor + 1$，最好、最坏、平均均为 $\Theta(\log n)$。

**回答思考**：时间复杂度的**数量级不变**，但实现细节影响实际比较次数的常数因子。BINSRCH 成功查找最好情况为 $\Theta(1)$（第一次就命中），BINSRCH1 则始终需要 $\Theta(\log n)$ 次比较，两种版本各有取舍。

### 【作业题2】递归版二分查找

**题目**：根据 4.2 节的二分查找策略，写一个二分查找的递归过程。

**算法思想**：将查找区间 $[low, high]$ 作为参数传入，每次取中间元素比较，根据结果递归地在左半或右半继续查找，直到找到目标或区间为空为止。

```
procedure BINSRCH_REC(A, x, low, high, j)
  integer mid, low, high, j;

  // 递归出口：区间为空，查找失败
  if low > high
    then j ← 0; return
  endif

  mid ← ⌊(low + high) / 2⌋         // 取中间下标
  case
    : x = A(mid) : j ← mid; return                        // 查找成功
    : x < A(mid) : call BINSRCH_REC(A, x, low, mid-1, j)  // 递归左半
    : x > A(mid) : call BINSRCH_REC(A, x, mid+1, high, j) // 递归右半
  endcase
end BINSRCH_REC

// 调用入口：
procedure BINSRCH_MAIN(A, n, x, j)
  call BINSRCH_REC(A, x, 1, n, j)
end BINSRCH_MAIN
```

**复杂度分析**：

递归关系为 $T(n) = T(n/2) + O(1)$，由主定理（$a=1,\ b=2,\ d=0$，$a = b^d = 1$），解得 $T(n) = O(\log n)$。

与迭代版对比：

| 对比项 | 迭代版（BINSRCH） | 递归版（BINSRCH_REC） |
|---|---|---|
| 时间复杂度 | $O(\log n)$ | $O(\log n)$ |
| 额外空间 | $O(1)$ | $O(\log n)$（递归调用栈） |
| 实现风格 | 循环，节省栈空间 | 递归，结构更直观 |

### 【作业题3】三分查找算法

**题目**：设计一个"三分"查找算法，首先检查 $n/3$ 处的元素，然后检查 $2n/3$ 处的元素，分析各种情况下的计算复杂度。

**算法思想**：每次将查找区间三等分，先检查 $1/3$ 处元素，再检查 $2/3$ 处元素，每次最多两次比较后将范围缩小为原来的 $1/3$。

```
procedure TRISRCH(A, n, x, j)
  integer low, high, m1, m2, j, n;
  low ← 1;  high ← n
  while low ≤ high do
    m1 ← low  + ⌊(high - low + 1) / 3⌋      // 1/3 处
    m2 ← high - ⌊(high - low + 1) / 3⌋      // 2/3 处
    case
      : x = A(m1) : j ← m1; return           // 在 1/3 处找到
      : x = A(m2) : j ← m2; return           // 在 2/3 处找到
      : x < A(m1) : high ← m1 - 1            // 搜索前 1/3 段
      : x > A(m2) : low  ← m2 + 1            // 搜索后 1/3 段
      : else       : low  ← m1 + 1            // 搜索中间 1/3 段
                     high ← m2 - 1
    endcase
  repeat
  j ← 0                                       // 查找失败
end TRISRCH
```

**复杂度分析**：

每次迭代最多 **2 次比较**，查找范围缩小为原来的 $1/3$，递归关系为 $T(n) = T(n/3) + 2$，解得：

$$T(n) = 2\log_3 n = \frac{2}{\log 3} \cdot \log n \approx 1.26 \log_2 n$$

| 情况 | 最好 | 最坏 | 平均 |
|---|---|---|---|
| 成功查找 | 1 次（第一次比较命中 $m1$） | $\Theta(\log n)$ | $\Theta(\log n)$ |
| 失败查找 | $\Theta(\log n)$ | $\Theta(\log n)$ | $\Theta(\log n)$ |

**思考题：三分（多分）查找会得到更优的结果吗？**

| 算法 | 每次缩小范围 | 每次最多比较次数 | 最坏总比较次数 |
|---|---|---|---|
| 二分查找 | $1/2$ | 2 次 | $\approx \log_2 n$ |
| 三分查找 | $1/3$ | 2 次 | $\approx 2\log_3 n \approx 1.26\log_2 n$ |
| $k$ 分查找 | $1/k$ | $k-1$ 次 | $\approx (k-1)\log_k n$ |

对于 $k$ 分查找，最坏比较次数为 $(k-1)\log_k n = \dfrac{(k-1)}{\ln k} \cdot \ln n$，函数 $\dfrac{k-1}{\ln k}$ 在 $k \geq 2$ 时单调递增，因此 $k$ 越大，最坏情况下总比较次数越多。

**结论**：三分乃至多分查找**并不能得到更优的结果**。虽然每次缩小的范围更大，但每次所需的比较次数同步增加，且增加得更快，导致最坏情况下总比较次数反而变多。这与定理 4.2 的结论完全一致——基于比较的查找下界为 $\Theta(\log n)$，二分查找已经达到这一下界，是最优的。

## 三、归并排序（4.3）

### 3.1 问题描述

给定一个含有 $n$ 个元素的集合，要求按**非降次序**排列。

直接插入法：最好 $O(n)$，最坏 $O(n^2)$。分治法可以做到 $O(n\log n)$。

### 3.2 分治法建模

- **分**：将 $A(1), \ldots, A(n)$ 平均分为两个子集：$A(1,\ldots,\lfloor n/2\rfloor)$ 和 $A(\lfloor n/2\rfloor+1, \ldots, n)$。
- **治**：递归调用，分别对两个子集排序。
- **合**：将两个已排好序的子集合并为一个有序集合（调用 MERGE）。

### 3.3 MERGESORT 算法

```
procedure MERGESORT(low, high)
  integer low, high, mid;
  if low < high
    then mid ← ⌊(low + high) / 2⌋
         call MERGESORT(low, mid)        // 递归排序左半段
         call MERGESORT(mid+1, high)     // 递归排序右半段
         call MERGE(low, mid, high)      // 合并两个有序子段
  endif
end MERGESORT

// 调用入口：
call MERGESORT(1, n)
```

### 3.4 MERGE 算法

**算法思想**：用两个指针 $h$、$j$ 分别指向左右两个子数组的当前元素，每次取较小者放入辅助数组 $B$，一方耗尽后将另一方剩余元素直接复制，最后将 $B$ 复制回 $A$。

```
procedure MERGE(low, mid, high)
  integer h, i, j, k, low, mid, high;
  global A(low:high);  local B(low:high);
  h ← low;  i ← low;  j ← mid + 1;

  // 两段均未耗尽时，逐一比较取小者
  while h ≤ mid and j ≤ high do
    if A(h) ≤ A(j)
      then B(i) ← A(h);  h ← h + 1
      else B(i) ← A(j);  j ← j + 1
    endif
    i ← i + 1
  repeat

  // 左段有剩余，直接复制
  if h > mid
    then for k ← j to high do
           B(i) ← A(k);  i ← i + 1
         repeat
    else for k ← h to mid do
           B(i) ← A(k);  i ← i + 1
         repeat
  endif

  // 将辅助数组 B 的结果复制回 A
  for k ← low to high do
    A(k) ← B(k)
  repeat
end MERGE
```

### 3.5 时间复杂度分析

合并两个大小为 $n/2$ 的子数组，元素比较次数在 $n/2$ 到 $n-1$ 之间，故合并时间为 $\Theta(n)$。

递归关系：

$$T(n) = \begin{cases} a & n = 1 \\ 2T(n/2) + cn & n > 1 \end{cases}$$

解（设 $n = 2^k$）：

$$T(n) = 2^k T(1) + kcn = an + cn\log n = O(n\log n)$$

**结论**：归并排序在**最好、最坏和平均**情况下，时间复杂度均为 $O(n\log n)$。

归并排序是**以比较为基础排序的最坏情况最优算法**：任何以关键字比较为基础的排序算法，最坏情况时间下界为 $\Omega(n\log n)$。

### 【作业题4】自底向上归并排序

**题目**：实现"自底向上"的归并排序算法，从而取消对栈空间的利用。

**算法思想**：自底向上无需递归。先将数组视作 $n$ 个长度为 1 的有序子段，然后逐轮将相邻的两个子段两两合并，每轮结束后子段长度翻倍，重复直至整个数组有序。全程只需循环，无递归调用，不占用系统栈空间。

```
procedure MERGESORT_BU(A, n)
  integer size, low, mid, high, n;
  // size：当前每个有序子段的长度，从 1 开始翻倍
  size ← 1
  while size < n do
    low ← 1
    // 遍历所有相邻子段对，依次合并
    while low ≤ n do
      mid  ← min(low + size - 1, n)         // 左段末尾
      high ← min(low + 2*size - 1, n)       // 右段末尾（可能不足 size）
      if mid < high                          // 存在右段才合并
        then call MERGE(low, mid, high)
      endif
      low ← low + 2 * size                  // 移到下一对子段
    repeat
    size ← size * 2                          // 子段长度翻倍
  repeat
end MERGESORT_BU
```

**与自顶向下版本的对比**：

| 对比项 | 自顶向下（递归） | 自底向上（迭代） |
|---|---|---|
| 时间复杂度 | $O(n\log n)$ | $O(n\log n)$ |
| 额外栈空间 | $O(\log n)$（递归调用栈） | $O(1)$（无递归） |
| 实现复杂度 | 代码简洁 | 稍复杂，需处理边界 |

**关键边界处理**：最后一轮合并时，右段可能不足 `size` 个元素，需用 `min` 防止数组越界。

## 四、斯特拉森矩阵乘法（4.4）

### 4.1 问题描述

$n \times n$ 矩阵相乘：$C[i][j] = \sum_{k=1}^{n} A[i][k] \cdot B[k][j]$

直接法时间复杂度：$O(n^3)$。

### 4.2 分治法直接建模的局限

将矩阵分为 4 个 $n/2 \times n/2$ 的子矩阵，得到 8 个子矩阵乘法：

$$\begin{bmatrix}C_{11}&C_{12}\\C_{21}&C_{22}\end{bmatrix} = \begin{bmatrix}A_{11}&A_{12}\\A_{21}&A_{22}\end{bmatrix}\begin{bmatrix}B_{11}&B_{12}\\B_{21}&B_{22}\end{bmatrix}$$

$C_{11} = A_{11}B_{11} + A_{12}B_{21}$，$C_{12} = A_{11}B_{12} + A_{12}B_{22}$，以此类推。

递推：$T(n) = 8T(n/2) + O(n^2)$，解得 $T(n) = O(n^3)$，**未能改善**。

原因：**子问题个数（8个）过多**。

### 4.3 斯特拉森算法

斯特拉森通过代数技巧，用 **7 次乘法** 代替原来的 8 次，计算 7 个中间矩阵：

$$M_1 = A_{11}(B_{12} - B_{22})$$
$$M_2 = (A_{11} + A_{12})B_{22}$$
$$M_3 = (A_{21} + A_{22})B_{11}$$
$$M_4 = A_{22}(B_{21} - B_{11})$$
$$M_5 = (A_{11} + A_{22})(B_{11} + B_{22})$$
$$M_6 = (A_{12} - A_{22})(B_{21} + B_{22})$$
$$M_7 = (A_{11} - A_{21})(B_{11} + B_{12})$$

用 8 次加减法得到结果：

$$C_{11} = M_5 + M_4 - M_2 + M_6, \quad C_{12} = M_1 + M_2$$
$$C_{21} = M_3 + M_4, \quad C_{22} = M_5 + M_1 - M_3 - M_7$$

### 4.4 时间复杂度

$$T(n) = 7T(n/2) + O(n^2)$$

解：$T(n) = O(n^{\log_2 7}) \approx O(n^{2.81})$

**优于** $O(n^3)$，但需 $n$ 足够大（实验表明 $n \approx 120$ 时才有显著优势）。

**核心启示**：**减少子问题个数**是降低分治算法时间复杂度的有效途径。

## 五、二维极大点问题（4.5）

### 5.1 问题描述

- **支配规则**：若 $x_1 > x_2$ 且 $y_1 > y_2$，则点 $(x_1, y_1)$ 支配点 $(x_2, y_2)$。
- **极大点**：若一个点不被其他任何点支配，则称其为极大点。
- **目标**：在 $n$ 个点的集合中找出所有极大点。
- 直接比较所有点对：$O(n^2)$。

### 5.2 分治法设计

```
procedure MAXPOINTS(S)
  // S：平面点的集合
  // 返回 S 中所有极大点的集合

  // 递归出口
  if |S| = 1
    then return S   // 单个点必为极大点
  endif

  // 分：按 x 坐标排序后取中位线 L，将 S 分为 SL 和 SR
  将 S 按 x 坐标从小到大排序（可预处理）
  m ← ⌊|S| / 2⌋
  SL ← S 中 x 值最小的 m 个点
  SR ← S 中 x 值最大的 m 个点

  // 治：递归求左右子集的极大点
  ML ← MAXPOINTS(SL)
  MR ← MAXPOINTS(SR)

  // 合：过滤 ML 中被 MR 支配的点
  // 由于 SL 中所有点的 x 值 < SR 中所有点的 x 值，
  // 故只需比较 y 值：
  ymax_R ← max{ p.y | p ∈ MR }
  result ← MR
  for each p in ML do
    if p.y > ymax_R          // p 的 y 值大于 MR 中所有极大点，则 p 不被支配
      then result ← result ∪ {p}
    endif
  repeat
  return result
end MAXPOINTS
```

**关键洞察**：$S_L$ 中所有点的 $x$ 值均小于 $S_R$，故 $S_L$ 中的点 $p$ 要不被 $S_R$ 的极大点支配，当且仅当 $p.y$ 大于 $S_R$ 所有极大点的 $y$ 值最大值。

### 5.3 预排序优化

若每次分治都重新排序，额外开销为 $O(n\log n)$，导致总时间：

$$T(n) = 2T(n/2) + O(n\log n) \Rightarrow T(n) = O(n\log^2 n)$$

**优化策略**：在递归开始前**预先对所有点按 $x$ 值排序一次**，之后每层直接利用已排好的序列确定中位线，每层合并代价降为 $O(n)$：

$$T(n) = 2T(n/2) + O(n) \Rightarrow T(n) = O(n\log n)$$

总复杂度（含预排序）：$O(n\log n) + O(n\log n) = O(n\log n)$，**优于直接法的 $O(n^2)$**。

### 【作业题5】二维极大点问题求解过程

**题目**：对点集 $(1,2)\ (2,8)\ (3,5)\ (4,1)\ (5,4)\ (6,7)\ (7,6)\ (8,3)$ 用分治策略求极大点，给出求解过程及结果。

**预排序**（按 $x$ 值升序，已满足）：

$(1,2),\ (2,8),\ (3,5),\ (4,1),\ (5,4),\ (6,7),\ (7,6),\ (8,3)$

---

**第一层分割**：取中位线，将 8 个点分为左右各 4 个：

- $S_L = \{(1,2),\ (2,8),\ (3,5),\ (4,1)\}$
- $S_R = \{(5,4),\ (6,7),\ (7,6),\ (8,3)\}$

---

**递归求 $S_L$ 的极大点**：

- 再分：$S_{LL} = \{(1,2),\ (2,8)\}$，$S_{LR} = \{(3,5),\ (4,1)\}$

  - $S_{LL}$：比较 $(1,2)$ 和 $(2,8)$：$x=2 > 1$，$y=8 > 2$，故 $(2,8)$ 支配 $(1,2)$。$M_{LL} = \{(2,8)\}$
  - $S_{LR}$：比较 $(3,5)$ 和 $(4,1)$：$x=4 > 3$，$y=1 < 5$，互不支配（$x$ 大但 $y$ 小）。$M_{LR} = \{(3,5),\ (4,1)\}$

- 合并 $M_{LL}$ 和 $M_{LR}$：$S_{LR}$ 的极大点 $y$ 值最大为 $\max\{5, 1\} = 5$
  - $(2,8)$：$y=8 > 5$，**保留**
  - $M_L = \{(2,8),\ (3,5),\ (4,1)\}$

---

**递归求 $S_R$ 的极大点**：

- 再分：$S_{RL} = \{(5,4),\ (6,7)\}$，$S_{RR} = \{(7,6),\ (8,3)\}$

  - $S_{RL}$：$(6,7)$ 支配 $(5,4)$（$x$ 和 $y$ 均大）。$M_{RL} = \{(6,7)\}$
  - $S_{RR}$：$(7,6)$ 和 $(8,3)$：$x=8>7$，$y=3<6$，互不支配。$M_{RR} = \{(7,6),\ (8,3)\}$

- 合并 $M_{RL}$ 和 $M_{RR}$：$S_{RR}$ 极大点 $y$ 最大为 $\max\{6, 3\} = 6$
  - $(6,7)$：$y=7 > 6$，**保留**
  - $M_R = \{(6,7),\ (7,6),\ (8,3)\}$

---

**第一层合并**：将 $M_L$ 和 $M_R$ 合并

$S_R$ 极大点 $y$ 值最大为 $\max\{7, 6, 3\} = 7$

对 $M_L$ 中每个点判断 $y$ 是否大于 7：

| 点 | $y$ 值 | $y > 7$？ | 结论 |
|---|---|---|---|
| $(2,8)$ | 8 | 是 | **保留** |
| $(3,5)$ | 5 | 否 | 舍弃（被 $(6,7)$ 支配） |
| $(4,1)$ | 1 | 否 | 舍弃（被 $(6,7)$ 支配） |

---

**最终结果**：

$$\text{极大点} = \{(2,8),\ (6,7),\ (7,6),\ (8,3)\}$$

**验证**：对每个极大点，不存在另一个点同时在 $x$ 和 $y$ 两个维度上都更大，结果正确。

## 六、分治法优化（4.6）与主定理（4.7）

### 6.1 效率低的原因与优化策略

| 原因 | 发生阶段 | 优化策略 | 典型例子 |
|---|---|---|---|
| 子问题个数过多 | 治 | 寻找子问题间的依赖关系，减少需递归求解的子问题数 | 斯特拉森矩阵乘法（8→7个子问题） |
| 递归内工作量过多 | 分/合 | 将重复性工作提取到递归外**预处理** | 二维极大点（预排序）、最近点对问题 |

### 6.2 主定理

对于递归式 $T(n) = aT(n/b) + f(n)$（其中 $a \geq 1, b > 1$），若 $f(n) = \Theta(n^d)$，则：

$$T(n) = \begin{cases} \Theta(n^d) & \text{若 } a < b^d \\ \Theta(n^d \log n) & \text{若 } a = b^d \\ \Theta(n^{\log_b a}) & \text{若 } a > b^d \end{cases}$$

**直观理解**：

- $a < b^d$：合并代价主导，$f(n)$ 的增长速度压过了子问题数的增长，总代价由顶层决定。
- $a = b^d$：两者平衡，各层代价相同，多出一个 $\log n$ 因子。
- $a > b^d$：子问题数主导，叶子层代价之和决定总代价。

**用主定理验证本章各算法**：

| 算法 | $a$ | $b$ | $f(n)$ | $d$ | $a$ vs $b^d$ | 结论 |
|---|---|---|---|---|---|---|
| 二分查找 | 1 | 2 | $O(1)$ | 0 | $1 = 2^0$ | $\Theta(\log n)$ |
| 归并排序 | 2 | 2 | $\Theta(n)$ | 1 | $2 = 2^1$ | $\Theta(n\log n)$ |
| 斯特拉森 | 7 | 2 | $O(n^2)$ | 2 | $7 > 2^2=4$ | $\Theta(n^{\log_2 7}) \approx \Theta(n^{2.81})$ |
| 极大点（优化后） | 2 | 2 | $O(n)$ | 1 | $2 = 2^1$ | $\Theta(n\log n)$ |

### 【作业题6】假币问题

**题目**：N 个硬币放在袋子里，其中一枚是假币，假币比真币轻，设计一个分治算法找到那枚假币。

**算法思想**：

利用分治法，每次将硬币三等分（每份 $n/3$ 枚），用天平称量其中两份：

- 若左边轻，假币在左边那份；
- 若右边轻，假币在右边那份；
- 若两边相等，假币在未称量的第三份。

每次将范围缩小为原来的 $1/3$，递归求解。

```
procedure FAKE_COIN(coins, low, high)
  // coins[low..high] 中恰有一枚假币
  integer low, high, size, m1, m2;

  // 递归出口：只剩一枚
  if low = high
    then return low           // 该枚即为假币
  endif

  size ← ⌊(high - low + 1) / 3⌋

  m1 ← low  + size - 1       // 第一份：coins[low .. m1]
  m2 ← m1   + size            // 第二份：coins[m1+1 .. m2]
                               // 第三份：coins[m2+1 .. high]

  // 称量第一份与第二份
  w1 ← WEIGH(coins, low, m1)
  w2 ← WEIGH(coins, m1+1, m2)

  case
    : w1 < w2 : return FAKE_COIN(coins, low,    m1)     // 假币在第一份
    : w1 > w2 : return FAKE_COIN(coins, m1+1,   m2)     // 假币在第二份
    : else     : return FAKE_COIN(coins, m2+1, high)     // 假币在第三份
  endcase
end FAKE_COIN

// 调用入口：
call FAKE_COIN(coins, 1, N)
```

**复杂度分析**：

每次称量一次，将问题规模缩小为 $n/3$：

$$T(n) = T(n/3) + 1 \Rightarrow T(n) = O(\log_3 n) = O(\log n)$$

这是**最优算法**，因为 $n$ 枚硬币中找一枚假币，至少需要 $\lceil\log_3 n\rceil$ 次称量（每次称量有三种结果）。

### 【作业题7】棋盘覆盖问题

**题目**：有一个 $2^k \times 2^k$ 的方格棋盘，恰有一个方格为黑色，其余为白色。用包含 3 个方格的 L 型牌覆盖所有白色方格，黑色方格不能被覆盖，白色方格不能被重复覆盖。

**算法思想（分治）**：

将棋盘四等分，黑色方格必在某一个 $2^{k-1} \times 2^{k-1}$ 的子棋盘中。在其余三个子棋盘各自靠近中心的角落**人为放置一个 L 型牌**，使每个子棋盘恰好各有一个"特殊格"（黑格或被 L 型牌覆盖的格），然后对四个子棋盘递归求解。

```
procedure CHESS_COVER(board, tr, tc, dr, dc, size)
  // board：棋盘
  // (tr, tc)：当前子棋盘左上角坐标
  // (dr, dc)：当前子棋盘中黑色（特殊）格的坐标
  // size：当前子棋盘边长
  integer tr, tc, dr, dc, size, t, s;

  // 递归出口：1×1 的棋盘，直接返回
  if size = 1 then return endif

  t    ← tile_number++        // 当前 L 型牌的编号
  s    ← size / 2              // 子棋盘边长

  // 左上子棋盘
  if dr < tr+s and dc < tc+s  // 黑格在左上子棋盘
    then call CHESS_COVER(board, tr, tc, dr, dc, s)
    else board[tr+s-1][tc+s-1] ← t   // 在右下角放 L 型牌
         call CHESS_COVER(board, tr, tc, tr+s-1, tc+s-1, s)
  endif

  // 右上子棋盘
  if dr < tr+s and dc ≥ tc+s  // 黑格在右上子棋盘
    then call CHESS_COVER(board, tr, tc+s, dr, dc, s)
    else board[tr+s-1][tc+s] ← t     // 在左下角放 L 型牌
         call CHESS_COVER(board, tr, tc+s, tr+s-1, tc+s, s)
  endif

  // 左下子棋盘
  if dr ≥ tr+s and dc < tc+s  // 黑格在左下子棋盘
    then call CHESS_COVER(board, tr+s, tc, dr, dc, s)
    else board[tr+s][tc+s-1] ← t     // 在右上角放 L 型牌
         call CHESS_COVER(board, tr+s, tc, tr+s, tc+s-1, s)
  endif

  // 右下子棋盘
  if dr ≥ tr+s and dc ≥ tc+s  // 黑格在右下子棋盘
    then call CHESS_COVER(board, tr+s, tc+s, dr, dc, s)
    else board[tr+s][tc+s] ← t       // 在左上角放 L 型牌
         call CHESS_COVER(board, tr+s, tc+s, tr+s, tc+s, s)
  endif

end CHESS_COVER
```

**复杂度分析**：

棋盘共 $2^k \times 2^k = 4^k$ 个格，其中 $1$ 个黑格，需要 $(4^k - 1)/3$ 个 L 型牌。

每次调用分为 4 个子问题，规模减半：

$$T(4^k) = 4T(4^{k-1}) + O(1) \Rightarrow T(n) = O(n)$$

其中 $n = 4^k$ 为格子总数，算法时间复杂度为 $O(4^k)$，即与棋盘格子数成正比，是**最优的**。

### 8.2 易错点汇总

1. **递归式求解**：展开时注意 $n = 2^k$ 的假设，最终结论需说明对一般 $n$ 的适用性（$2^k < n \leq 2^{k+1}$ 时有 $T(n) \leq T(2^{k+1})$）。

2. **二分查找的比较次数**：成功查找最好 $O(1)$（第一次就命中中间元素），失败查找最好也是 $O(\log n)$（不存在 $O(1)$ 情况）。

3. **归并排序辅助空间**：MERGE 需要 $O(n)$ 的额外数组，这是归并排序相对于原地排序（如堆排序）的主要缺点。

4. **三分查找不优于二分查找**：虽然每次缩小为 $1/3$（优于 $1/2$），但每次比较次数增加（最多 2 次），整体效率并未提升，常数因子约为 $1.26\log_2 n > \log_2 n$。

5. **斯特拉森实际效果**：理论上 $O(n^{2.81})$，但由于常数因子大，$n < 120$ 时通常不如直接法；且数值稳定性较差。

6. **极大点的合并条件**：合并时只需比较 $y$ 值（因为 $S_L$ 的所有 $x$ 值已经小于 $S_R$，$S_L$ 中的点不可能在 $x$ 方向上支配 $S_R$ 的点）。

7. **主定理的使用条件**：$T(n) = aT(n/b) + f(n)$ 中要求 $a \geq 1$，$b > 1$，$f(n)$ 渐近正。注意主定理的三种情况要求是**严格的多项式级差异**，即 $f(n) = O(n^{\log_b a - \varepsilon})$ 或 $f(n) = \Omega(n^{\log_b a + \varepsilon})$，不满足时需用其他方法。

8. **自底向上归并的边界处理**：最后一趟合并时右段可能不足 `size` 个元素，必须用 `min` 限制 `high` 不超过 `n`，否则数组越界。
