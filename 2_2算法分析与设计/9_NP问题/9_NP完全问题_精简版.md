# 第九章 NP-完全问题（精简版）

---

## 一、背景：为什么研究问题难度？

不是所有问题都能被高效求解。评价算法好坏的标准是运行时间：

- **好算法（多项式时间）**：$O(1),\ O(\log n),\ O(n),\ O(n\log n),\ O(n^2),\ O(n^3)$
- **坏算法（指数时间）**：$O(2^n),\ O(n!),\ O(n^n)$

现实中有大量问题（如图着色、TSP、0/1背包）——目前只有指数级算法，但**没有人能证明它们一定没有多项式算法**。NP-完全理论就是研究这类问题难度关系的工具。

---

## 二、判定问题

为了统一比较问题难度，将优化问题转化为只回答"是/否"的**判定问题**：

| 优化问题 | 判定问题 |
|---------|---------|
| 最大团：最大完全子图有多大？ | 图G是否存在大小为 $k$ 的团？ |
| 0/1背包：效益值最大是多少？ | 是否存在选法使效益值之和 $\geq X$？ |

> 优化问题与其对应判定问题的难度等价：一个有多项式算法，当且仅当另一个也有。

---

## 三、P 类问题

$$\boxed{P = \text{所有已经找到多项式算法的（判定）问题的集合}}$$

例：排序、最短路径、最小生成树等。

---

## 四、NP 类问题

$$\boxed{NP = \text{所有可在多项式时间内由不确定算法（验证）求解的判定问题的集合}}$$

**关键理解**：NP 问题不要求能在多项式时间内**找到**解，只要求能在多项式时间内**验证**一个给定的解是否正确。

- 不确定算法分两步：**猜想**一个解（用 `choice` 瞬间猜中）→ **验证**该解是否合法（多项式时间）
- 显然 $P \subseteq NP$（确定算法是不确定算法的特例）
- **核心未解问题**：$P \stackrel{?}{=} NP$（是否所有 NP 问题都有多项式算法？）

---

## 五、可满足性（SAT）问题

**问题描述**：给定布尔变量 $x_1, x_2, \ldots, x_n$ 构成的合取范式（CNF）：

$$F = A_1 \wedge A_2 \wedge \cdots \wedge A_k$$

其中每个子句 $A_j$ 是若干文字（变量或其非）的析取。问：是否存在一组真值指派使 $F$ 为真？

**示例**：
- $F_1 = (x_1 \vee x_2) \wedge (\neg x_1 \vee x_2 \vee x_3) \wedge \neg x_2$
  → 令 $(x_1,x_2,x_3)=(1,0,1)$，$F_1$ 为真 ✓ **可满足**
- $F_2 = (x_1 \vee \neg x_2 \vee x_3) \wedge (\neg x_1 \vee \neg x_2 \vee x_3) \wedge x_2 \wedge \neg x_3$
  → 无论如何赋值均不满足 ✗ **不可满足**

SAT 问题属于 NP：只需猜一组赋值，代入验证是 $O(n)$ 的。

---

## 六、归约

**归约**（$L_1 \leq_p L_2$）：若能在多项式时间内将 $L_1$ 的任意实例转化为 $L_2$ 的等价实例，则称 $L_1$ 可多项式归约到 $L_2$。

**含义**：$L_1 \leq_p L_2$ 意味着"$L_2$ 至少和 $L_1$ 一样难"——$L_2$ 能解则 $L_1$ 也能解。

**传递性**：若 $L_1 \leq_p L_2$，$L_2 \leq_p L_3$，则 $L_1 \leq_p L_3$。

---

## 七、NP-难 与 NP-完全

| 概念 | 条件 | 直觉 |
|------|------|------|
| **NP-难（NP-Hard）** | SAT $\leq_p L$ | $L$ 至少和所有 NP 问题一样难（不要求 $L \in NP$） |
| **NP-完全（NPC）** | SAT $\leq_p L$，**且** $L \in NP$ | NP 问题中最难的一批 |

**证明某问题 $L$ 是 NP-完全的两步：**
1. 证明 $L \in NP$（设计多项式时间可验证的不确定算法）
2. 证明 SAT $\leq_p L$（找到从 SAT 到 $L$ 的多项式时间归约）

---

## 八、Cook 定理

$$\boxed{\text{SAT 是 NP-完全的} \iff \text{SAT} \in P \iff P = NP}$$

完整表述：**可满足性问题在 P 内，当且仅当 P = NP。**

- SAT 是历史上**第一个**被证明为 NP-完全的问题（Cook，1971）
- 它是整个 NPC 归约链的**源头**：从 SAT 出发，可归约出数千个 NPC 问题

```
SAT
├── 3SAT → 图着色、独立集、最大团 ...
└── 恰好覆盖 → 子集和 → 0/1背包 ...
```

---

## 九、关系总结

```
         NP-Hard
    ┌────────────────┐
    │  NP-Complete   │
    │  ┌──────────┐  │
    │  │    NP    │  │
    │  │  ┌────┐  │  │
    │  │  │ P  │  │  │
    │  │  └────┘  │  │
    │  └──────────┘  │
    └────────────────┘
```

| 结论 | 状态 |
|------|------|
| $P \subseteq NP$ | ✅ 已证 |
| $NPC \subseteq NP\text{-Hard}$ | ✅ 已证 |
| $P \stackrel{?}{=} NP$ | ❓ 未解决 |
| 若任意 NPC 问题 $\in P$，则 $P=NP$ | ✅ 已证 |

---

## 十、考点速查 ★

| 考点 | 核心结论 |
|------|---------|
| P 定义 | 多项式时间**确定**算法可解 |
| NP 定义 | 多项式时间**验证**（不确定算法）可解 |
| P ⊆ NP | 成立；P ?= NP 未解决 |
| SAT 问题 | 第一个 NPC 问题（Cook 1971） |
| Cook 定理 | SAT ∈ P $\iff$ P = NP |
| NP-难 | SAT $\leq_p L$（不要求 $L \in NP$） |
| NP-完全 | SAT $\leq_p L$ 且 $L \in NP$ |
| 证明 NPC 两步 | ① $L \in NP$；② 已知 NPC $\leq_p L$ |
| 归约方向易错 | $L_1 \leq_p L_2$：$L_2$ **更难**（不比 $L_1$ 容易） |
| NP ≠ 非多项式 | NP = Nondeterministic Polynomial，与"指数时间"无关 |
