# 第九章 NP-完全问题 — 复习资料

---

## 一、好算法与多项式时间算法

### 1.1 算法复杂性 vs 问题复杂性

| 概念 | 定义 | 说明 |
|------|------|------|
| **算法的复杂性** | 解决问题的某一具体算法的执行时间 | 是**算法**的性质 |
| **问题的复杂性** | 该问题所有求解算法中，最好算法的复杂性 | 是**问题**的性质 |

例：排序问题的复杂性是 $O(n\log n)$（快速排序），而冒泡排序是 $O(n^2)$——是算法复杂性，不是问题复杂性。

### 1.2 多项式时间算法（好算法）

| 类型 | 增长阶 |
|------|--------|
| **多项式时间（好算法）** | $O(1) < O(\log n) < O(n) < O(n\log n) < O(n^2) < O(n^3)$ |
| **指数时间（坏算法）** | $O(2^n) < O(n!) < O(n^n)$ |

### 1.3 三类算法对比（每秒 $10^9$ 次计算机）

| 算法 | 时间复杂度 | 问题规模 | 计算时间 |
|------|-----------|---------|---------|
| 快速排序 | $O(n\log n)$ | $10^5$ 个数据 | $1.7 \times 10^{-3}$ 秒 |
| Dijkstra | $O(n^2)$ | $10^4$ 个顶点 | $0.1$ 秒 |
| 最大团回溯 | $O(n \cdot 2^n)$ | $10^2$ 个顶点 | $\approx 5.7 \times 10^{15}$ 年 |

> **结论：多项式时间算法是好算法。**

## 二、现实世界问题的分类

| 类型 | 描述 | 示例 |
|------|------|------|
| **多项式级** | 已设计出多项式算法 | 排序、最小生成树、单源最短路径 |
| **不可计算** | 根本不存在求解算法 | 希尔伯特第十问题 |
| **指数级** | 已证明不存在多项式算法 | 带幂运算的正则表达式全体性 |
| **尚不确定** ⭐ | 只有指数算法，但未证明不存在多项式算法 | 图着色、货郎担（TSP）、0/1背包 |

> 很多优化问题属于**"尚不确定"**类，这是本章研究的核心。

## 三、判定问题

### 3.1 定义

**判定问题**：只需回答"是"或"否"的问题。

### 3.2 优化问题 → 判定问题 转化示例

| 优化问题 | 对应判定问题 |
|---------|------------|
| 图着色：最少需要多少种颜色？ | 图G能否用 $m$ 种颜色着色？ |
| 最大团：最大完全子图有多大？ | 图G是否存在大小为 $k$ 的团？ |
| 0/1背包：效益值之和最大是多少？ | 是否存在选法使效益值之和 $\geq X$？ |

### 3.3 相互转化（以最大团为例）

**判定问题 → 优化问题：**
设 DCLIQUE(G,k) 为判定算法，检验 $k=n, n-1, n-2, \ldots$ 直到返回1，时间 $O(n \cdot f(n))$。

**优化问题 → 判定问题：**
求出最大团大小 $j$，若 $j < k$ 返回否，否则返回是，时间 $O(g(n))$。

> **结论：最大团问题与其判定问题可在多项式时间内相互转换。**
> 转化为判定问题的好处：消除不同优化问题输出的差异性，统一输出"是/否"。

## 四、不确定算法

### 4.1 基本概念

**不确定算法**：取消运算"确定性"限制，允许运算结果受限于某个特定集合。

SPARKS 中引入三个新操作（时间复杂度均为 $O(1)$）：

| 操作 | 含义 |
|------|------|
| `choice(S)` | 按题意从集合 $S$ 中选取一个元素 |
| `failure` | 发出不成功完成的信号 |
| `success` | 发出成功完成的信号 |

### 4.2 不确定机

**不确定机**：能执行不确定算法的理论机器（现实不存在）。

- 具有"**上帝视角**"：若问题有解，总能在 $O(1)$ 内猜中正确答案
- 若问题无解，随机返回一个元素

> 不确定机是为分析时间复杂度难题而**虚构的理论模型**。

### 4.3 不确定算法的设计步骤

| 阶段 | 别名 | 内容 |
|------|------|------|
| **猜想阶段** | 不确定阶段 | 基于 `choice` 猜出一个解 |
| **验证阶段** | 确定阶段 | 用确定语句验证猜出的解是否正确 |

**时间复杂度计算：**
- 返回 `failure`：视为 $O(1)$（不关注失败情况）
- 返回 `success`：= 不确定阶段 + 确定阶段的时间之和

**多项式时间可验证**：验证阶段时间复杂度是多项式级的。

### 4.4 不确定的判定算法

只产生 0/1 输出：
- **输出 0**：当且仅当没有一种选择可导致 success
- **输出 1**：当且仅当至少存在一种选择导致 success

### 4.5 最大团判定问题的不确定算法

```
procedure DCK(G, n, k)
  S ← ∅                          // 求得的点集
  for i ← 1 to k do              // 不确定阶段 O(k)
    t ← choice(1:n)
    if t ∈ S then failure endif
    S ← S ∪ {t}
  repeat
  for 使得 i, j ∈ S 的每一对(i,j) and i≠j do  // 确定阶段 O(k²)
    if (i,j) 不是此图的边 then failure endif
  repeat
  success
end DCK
```

> **总时间复杂度：** $O(k + k^2) = O(n^2)$

### 4.6 问题规模的二进制表示

为统一度量，输入参数均转换为**二进制形式**，时间复杂度基于**二进制输入长度** $m$ 来考虑。

DCK 的输入二进制长度：
$$m = n^2 + \lfloor\log_2 k\rfloor + \lfloor\log_2 n\rfloor + 2$$

算法 DCK 的时间复杂度：$O(n^2) = O(m)$，为多项式级。

## 五、P 问题与 NP 问题

### 5.1 定义

| 问题类 | 定义 |
|--------|------|
| **P 问题** | 所有可在**多项式时间**内用**确定**算法求解的判定问题的集合 |
| **NP 问题** | 所有可在**多项式时间**内用**不确定**算法求解的判定问题的集合 |

**P ⊆ NP**：P 是 NP 的特例（`choice(S)` 中 $S$ 只有一个元素）。

### 5.2 P vs NP 的核心问题

$$P \subseteq NP \quad \text{（已证）}, \quad P \stackrel{?}{=} NP \quad \text{（未解决）}$$

> 这是计算机科学最著名的未解问题之一。

## 六、归约

### 6.1 归约定义

$L_1$ **归约到** $L_2$（记作 $L_1 \leq L_2$ 或 $L_1 \propto L_2$）是指：存在转换函数 $T(x)$，使得：
1. $L_1$ 的任意合法输入 $x$ 可转换为 $L_2$ 的合法输入 $T(x)$
2. $L_1$ 对 $x$ 的输出为 yes $\Longleftrightarrow$ $L_2$ 对 $T(x)$ 的输出为 yes

> **直观意义：** $L_1 \leq L_2$，则 $L_2$ 的时间复杂度不低于 $L_1$，即 $L_1$ 不比 $L_2$ 难。

### 6.2 归约示例

- $L_1$：判定 $n$ 个布尔值中是否至少有一个为 true
- $L_2$：判定 $n$ 个整数中的最大元素是否为正数
- 转换函数 $T$：$Y_i = 1$（若 $X_i = \text{true}$），$Y_i = 0$（若 $X_i = \text{false}$）

### 6.3 多项式时间归约

若计算转换函数 $T$ 的算法是多项式级的，则称 $L_1$ 可**多项式时间归约**到 $L_2$，记为 $L_1 \leq_p L_2$。

**性质：**
- **传递性**：若 $L_1 \leq_p L_2$，$L_2 \leq_p L_3$，则 $L_1 \leq_p L_3$
- 若 $L_2$ 多项式级可解，则 $L_1$ 也多项式级可解
- 若 $L_2$ 指数级可解，则 $L_1$ 也指数级可解

## 七、NP-完全问题与 NP-难问题

### 7.1 定义

| 概念 | 条件 |
|------|------|
| **NP-完全（NPC）** | ① $L \in \text{NP}$；② 对所有 $L' \in \text{NP}$，有 $L' \leq_p L$ |
| **NP-难（NP-Hard）** | 对所有 $L' \in \text{NP}$，有 $L' \leq_p L$（不要求 $L \in \text{NP}$） |

### 7.2 关系图

```
NP-Hard
  ├── NP-Complete  ⊆  NP
  │       ├── P（若 P=NP）
  │       └── ...
  └── 不属于 NP 的问题（如停机问题）
```

> NP-难比 NP-完全范围更广，NP-难问题不一定是 NP 问题。

### 7.3 NP-完全的意义

- 若任意一个 NPC 问题有多项式算法，则**所有 NP 问题**都有多项式算法（即 P=NP）
- $P = NP$ 的充要条件：存在一个 NP-完全问题 $L \in P$

### 7.4 证明问题 $L$ 是 NP-完全的步骤

1. 证明 $L \in \text{NP}$（设计多项式时间可验证的不确定算法）
2. 找到一个已知的 NP-完全问题 $L'$，证明 $L' \leq_p L$

## 八、可满足性（SAT）问题

### 8.1 基本概念

- **布尔变量** $x_i$：取值 true/false
- **文字** $\sigma$：$x_i$ 或 $\neg x_i$
- **子句** $A_j$：若干文字的析取 $(\sigma_1 \vee \sigma_2 \vee \cdots)$
- **合取范式（CNF）**：$F = A_1 \wedge A_2 \wedge \cdots \wedge A_k$

**SAT 问题**：给定 CNF 公式 $F$，是否存在一组对变量的赋值使 $F$ 为真？

### 8.2 实例

| 公式 | 结论 |
|------|------|
| $F_1 = (x_1 \vee x_2) \wedge (\neg x_1 \vee x_2 \vee x_3) \wedge \neg x_2$ | 可满足：$(x_1,x_2,x_3)=(1,0,1)$ |
| $F_2 = (x_1 \vee \neg x_2 \vee x_3) \wedge (\neg x_1 \vee \neg x_2 \vee x_3) \wedge x_2 \wedge \neg x_3$ | **不可满足** |

### 8.3 SAT 的不确定算法

```
procedure EVAL(F, n)
  Boolean x(1:n)
  for i ← 1 to n do           // 不确定阶段
    x_i ← choice(true, false)
  repeat
  if F(x_1,...,x_n) = true    // 确定阶段
    then success
    else failure
  endif
end EVAL
```

> **时间复杂度：** $O(n)$，SAT $\in$ NP

### 8.4 Cook 定理（核心定理）

$$\text{SAT 是 NP-完全的}$$

即任何 NP 问题都可以在多项式时间内归约为 SAT 问题。SAT 是**第一个**被发现的 NP-完全问题（1971年，S. Cook）。

## 九、恰好覆盖问题（Exact Cover）

### 9.1 问题描述

给定有穷集 $A = \{a_1, a_2, \ldots, a_n\}$ 和 $A$ 的子集集合 $W = \{S_1, S_2, \ldots, S_m\}$。

求 $U \subseteq W$，使得 $U$ 中各集合**两两不相交**且**并集等于** $A$（称 $U$ 为 $A$ 的恰好覆盖）。

### 9.2 示例

$A=\{1,2,3,4,5\}$，$W=\{S_1,S_2,S_3,S_4\}$：
- $S_1=\{1,2\}$，$S_2=\{1,3,4\}$，$S_3=\{2,4\}$，$S_4=\{2,5\}$
- $\{S_2, S_4\}$ 是 $A$ 的恰好覆盖（$S_2 \cup S_4 = A$，且 $S_2 \cap S_4 = \emptyset$）

> **恰好覆盖问题是 NP-完全的。**

## 十、子集和问题（Subset Sum）

### 10.1 问题描述

给定正整数集合 $X = \{x_1, x_2, \ldots, x_n\}$ 及正整数 $N$，问是否存在 $X$ 的子集 $T$，使得 $T$ 中元素之和等于 $N$？

### 10.2 证明子集和是 NP-完全的

**思路：** 证明恰好覆盖 $\leq_p$ 子集和（即从恰好覆盖实例构造子集和实例）

**构造方法：**

给定恰好覆盖实例 $(A, W)$，$|A|=n$，$|W|=m$，令 $k = \lceil\log_2(m+1)\rceil$。

- 将每个 $x_j$ 和 $N$ 表示成 $nk$ 位二进制数，分 $n$ 段，每段 $k$ 位
- $N$：每段最右位为 1，其余为 0
- $x_j$：若 $a_i \in S_j$，第 $i$ 段最右位为 1，否则第 $i$ 段全为 0

**为什么每段需要 $k$ 位？**  
求和时最多 $m$ 个数相加，$k = \lceil\log_2(m+1)\rceil$ 位能保证不会产生**进位**到相邻段。

### 10.3 构造举例

$A=\{a_1,a_2,a_3,a_4\}$，$W=\{S_1,S_2,S_3\}$，$n=4$，$m=3$，$k=2$：

| 集合 | 包含元素 | 对应 $x_j$（8位，4段×2位） |
|------|---------|--------------------------|
| $N$（目标） | — | `01 01 01 01` |
| $x_1$（$S_1=\{a_1,a_2\}$） | $a_1,a_2$ | `01 01 00 00` |
| $x_2$（$S_2=\{a_1,a_3,a_4\}$） | $a_1,a_3,a_4$ | `01 00 01 01` |
| $x_3$（$S_3=\{a_2\}$） | $a_2$ | `00 01 00 00` |

验证：$x_2 + x_3 = $ `01 01 01 01` $= N$ ✓，对应 $S_2 \cup S_3 = A$ 的恰好覆盖。

## 十一、P、NP、NP-完全、NP-难关系总结



| 关系 | 结论 |
|------|------|
| $P \subseteq NP$ | 已证 |
| $NP\text{-Complete} \subseteq NP$ | 定义即满足 |
| $NP\text{-Complete} \subseteq NP\text{-Hard}$ | 定义即满足 |
| $P \stackrel{?}{=} NP$ | 未解决 |
| 若某 NPC 问题 $\in P$ | $\Rightarrow P = NP$ |

## 十三、易错点汇总

1. **NP ≠ "非多项式"**：NP 是 Nondeterministic Polynomial 的缩写，表示可用不确定算法多项式可解，**不是**"非多项式时间"。

2. **P ⊆ NP（已证），P ?= NP（未证）**：不要混淆。目前只知道 $P \subseteq NP$，不知道是否 $P = NP$。

3. **NP-难不一定是NP问题**：NP-完全 = NP-难 ∩ NP；NP-难范围更广。

4. **归约方向**：$L_1 \leq_p L_2$ 表示 $L_1$ 可归约到 $L_2$，意味着 $L_2$ **不比** $L_1$ 容易（$L_2$ 至少和 $L_1$ 一样难）。方向常被搞反。

5. **证明NPC需要两步**：① 证 $L \in NP$；② 找已知NPC问题 $L'$ 证 $L' \leq_p L$。只做一步不够。

6. **不确定算法的时间复杂度**：只计算 success 路径的时间；failure 视为 $O(1)$。

7. **子集和构造中k位的作用**：是防止进位，$k = \lceil\log_2(m+1)\rceil$ 保证 $m$ 个数相加不会进位到相邻段。

8. **SAT是NP完全的方向**：Cook定理证明的是"任何NP问题 $\leq_p$ SAT"，即SAT是所有NP问题的"上界"，不是SAT可以归约到别的NP问题。

## 十四、考点速查

| 考点 | 核心内容 |
|------|---------|
| P 定义 | 多项式时间确定算法可解的判定问题集合 |
| NP 定义 | 多项式时间不确定算法可解的判定问题集合 |
| 不确定算法两阶段 | 猜想（choice）+ 验证（确定语句） |
| 不确定机特点 | 有"魔力"，$O(1)$ 内猜中；理论模型，现实不存在 |
| 归约 $L_1 \leq_p L_2$ 含义 | $L_2$ 不比 $L_1$ 容易；$L_2$ 可解 $\Rightarrow$ $L_1$ 可解 |
| NPC 定义两条件 | ① $L \in NP$；② 所有 $NP$ 问题 $\leq_p L$ |
| Cook 定理 | SAT 是第一个 NPC 问题 |
| 证明某问题是NPC | 证明 $L \in NP$，且已知NPC问题 $\leq_p L$ |
| 子集和 NPC 证明核心 | 恰好覆盖 $\leq_p$ 子集和；二进制构造，$k$ 位防进位 |
| P=NP 充要条件 | 存在一个 NPC 问题属于 P |
