## 一、贪心法一般方法（5.1）

### 1.1 适用问题特点

贪心法适用于这样一类问题：有 n 个输入，而问题的解就是这 n 个输入的**某个子集**，该子集必须满足某些事先给定的条件。

| 术语 | 定义 |
|---|---|
| **约束条件** | 子集必须满足的条件 |
| **可行解** | 满足约束条件的子集 |
| **目标函数** | 衡量可行解优劣的标准，以函数形式给出 |
| **最优解** | 使目标函数取极值（极大或极小）的可行解 |

### 1.2 贪心法求解思想

贪心方法是一种 **"只顾眼前"** 的分级处理方法：

1. 根据题意选取一种**量度标准**；
2. 按该标准**一次选中一个输入**；
3. 若该输入与当前部分解合并后**满足约束条件**，则加入部分解；否则**舍弃**。

> 核心问题：选择能产生问题最优解的量度标准，即**最优量度标准**。最优量度标准需要经过严格证明。

### 1.3 GREEDY 抽象控制框架

```
Procedure GREEDY(A, n)
  // A(1:n) 包含 n 个输入
  solution ← Φ        // 解向量初始化为空
  for i ← 1 to n do
    x ← SELECT(A)     // 按最优量度标准从 A 中选一个输入，并从 A 中删除
    if FEASIBLE(solution, x)
      then solution ← UNION(solution, x)
    endif
  repeat
  return (solution)
end GREEDY
```

三个核心操作：
- **SELECT**：按某种最优量度标准从 A 中选择一个输入赋值给 x，并从 A 中消去它
- **FEASIBLE**：判定 x 是否可以包含在解向量中（约束条件检验）
- **UNION**：将 x 与解向量结合并修改约束判定

### 1.4 贪心法的优缺点

**优点：**
- 一旦最优量度标准证明成立，就是一种高效算法
- 策略构造简单易行
- 对许多问题能产生整体最优解或近似最优解

**缺点：**
- 不是对所有问题都能得到最优解
- 基于目标函数制定的度量标准不一定是最优的
- 最优量度标准需要经过证明

## 二、背包问题（5.2）

### 2.1 问题描述

已知有 n 种物品和一个可容纳 M 重量的背包，每种物品 i 的重量为 $w_i$，将物品 i 的某一部分 $x_i$ 放入背包就会得到 $p_i x_i$ 的效益（$0 \leq x_i \leq 1$，$p_i > 0$）。

$$\text{极大化} \quad \sum_{1 \leq i \leq n} p_i x_i \qquad 0 \leq x_i \leq 1,\ p_i > 0$$

$$\text{约束条件} \quad \sum_{1 \leq i \leq n} w_i x_i \leq M \qquad w_i > 0,\ 1 \leq i \leq n$$

> 注意：这是**分数背包**，物品可以取一部分（$x_i$ 可以是小数）。与 0/1 背包不同。

### 2.2 三种量度标准对比

以 n=3，M=20，$(p_1,p_2,p_3)=(25,24,15)$，$(w_1,w_2,w_3)=(18,15,10)$ 为例：

| 量度标准 | $(x_1,x_2,x_3)$ | $\sum w_i x_i$ | $\sum p_i x_i$ | 问题 |
|---|---|---|---|---|
| 按效益值 $p_i$ 非增次序 | $(1,\ 2/15,\ 0)$ | 20 | **28.2** | 容量消耗过快 |
| 按重量 $w_i$ 非降次序 | $(0,\ 2/3,\ 1)$ | 20 | **31** | 效益值增长缓慢 |
| 按 $p_i/w_i$ 非增次序 | $(0,\ 1,\ 1/2)$ | 20 | **31.5** ✓ | 每消耗单位容量获得最大效益 |

**结论**：按单位重量效益值 $p_i/w_i$ 的非增次序排列，是背包问题的最优量度标准。

### 2.3 GREEDY-KNAPSACK 算法

```
procedure GREEDY-KNAPSACK(P, W, M, X, n)
  // 先将物品按 pi/wi 比值的非增次序排序（降序）
  // P(1:n)：效益值；W(1:n)：重量；X(1:n)：解向量；M：背包容量
  real P(1:n), W(1:n), X(1:n), M, cu
  integer i, n
  X ← 0
  cu ← M                           // cu 表示背包剩余容量
  for i ← 1 to n do
    if W(i) > cu then exit endif
    X(i) ← 1
    cu ← cu - W(i)
  repeat
  if i ≤ n then X(i) ← cu / W(i)  // 物品 i 只放入一部分
end GREEDY-KNAPSACK
```

> 时间复杂度：排序 $O(n\log n)$，贪心选取 $O(n)$，总体 $O(n\log n)$。

### 2.4 定理5.1：背包问题贪心解最优性证明

**定理5.1**：如果 $p_1/w_1 \geq p_2/w_2 \geq \cdots \geq p_n/w_n$，则算法 GREEDY-KNAPSACK 对于给定的背包问题实例生成一个最优解。

**证明思路（替换论证法）**：

设 $X=(x_1,\ldots,x_n)$ 是贪心解，形如 $(1,\ldots,1,\ x_j,\ 0,\ldots,0)$（j 是第一个 $x_j \neq 1$ 的下标）。

若 X 不是最优解，则存在最优解 $Y=(y_1,\ldots,y_n)$ 使 $\sum p_i y_i > \sum p_i x_i$。

设 k 是使 $y_k \neq x_k$ 的最小下标，可证 $y_k < x_k$（三种情况分析）。

构造新解 $Z$：将 $y_k$ 增加到 $x_k$，同时从 $k$ 之后减去等量重量，使总重量仍为 M。

利用物品按 $p_i/w_i$ 非增排列，可推出 $\sum p_i z_i \geq \sum p_i y_i$。

若 $\sum p_i z_i > \sum p_i y_i$，则 Y 不是最优解，矛盾。故每次替换后目标函数值不变，反复替换最终将 Y 转换为 X，证明 X 也是最优解。

## 三、带有期限的作业调度问题（5.3）

### 3.1 问题描述

- 一台机器处理 n 个作业，每个作业均可在**单位时间**内完成
- 每个作业 i 有截止期限 $d_i > 0$（整数），在期限前完成获得效益 $p_i > 0$
- **可行解**：作业子集 J，J 中每个作业都能在各自截止期限前完成
- **最优解**：效益值 $\sum_{j \in J} p_j$ 最大的可行解

**实例**：n=4，$(p_1,p_2,p_3,p_4)=(100,10,15,20)$，$(d_1,d_2,d_3,d_4)=(2,1,2,1)$

| 可行解 | 处理顺序 | 效益值 |
|---|---|---|
| {1} | 1 | 100 |
| {1,2} | 2,1 | 110 |
| {1,3} | 1,3 | 115 |
| **{1,4}** | **4,1** | **120** ✓ |
| {2,3} | 2,3 | 25 |
| {3,4} | 4,3 | 35 |

### 3.2 GREEDY_JOB 算法

**量度标准**：将各作业按效益值 $p_i$ 降序排列（$p_1 \geq p_2 \geq \cdots \geq p_n$），依次加入若仍可行。

```
procedure GREEDY_JOB(D, J, n)
  // 作业已按 p1 ≥ p2 ≥ … ≥ pn 的次序输入；期限值 D(1:n) ≥ 1；J 是最优解
  J ← {1}
  for i ← 2 to n do
    if (J ∪ {i} 的所有作业都能在它们的截止期限前完成)
      then J ← J ∪ {i}
    endif
  repeat
end GREEDY_JOB
```

### 3.3 定理5.2：贪心解最优性证明

**定理5.2**：算法 GREEDY_JOB 所描述的贪心方法总是得到一个最优解。

**证明思路**：设 J 是贪心解，I 是最优解，不妨设 I ≠ J。

- 若 $I \subset J$：I 的效益 < J 的效益，与 I 是最优解矛盾，故 $I \subset J$ 不可能。
- 若 $J \subset I$：存在作业 $a \in I$ 且 $a \notin J$，$\forall b \in J$ 有 $b \in I$。
  - 若 a 的效益 < J 中所有作业效益，则 $J \cup \{a\}$ 是可行解，贪心法可将 a 纳入 J，矛盾。
  - 若 a 比 J 中某作业 c 效益大，贪心法会先于 c 考虑 a 并纳入 J，矛盾。
  - 故 $J \subset I$ 不可能。
- 因此 I 和 J 互不包含，利用替换论证：找 J 中效益最高的 $a \notin I$ 替换 I 中与 a 同时间片的作业 b（$p_a \geq p_b$），得到不减效益的新解 I'，反复替换最终将 I 转换为 J，故 J 也是最优解。

### 3.4 定理5.3：可行调度判断

**定理5.3**：设 J 是 k 个作业的集合，$\sigma = i_1, i_2, \ldots, i_k$ 是 J 中作业的排列，使得 $d_{i_1} \leq d_{i_2} \leq \cdots \leq d_{i_k}$。J 是一个可行解，**当且仅当** J 中的作业可以按照 σ 的次序而又不违反任何一个期限的情况来处理。

**证明思路**（充分性 ← 方向显然；必要性 → 方向）：

若 J 是可行解，则存在允许调度 $\sigma' = r_1,r_2,\ldots,r_k$（$d_{r_j} \geq j$）。

若 $\sigma' \neq \sigma$，令 a 是使 $r_a \neq i_a$ 的最小下标，设 $r_b = i_a$（b > a）。

在 $\sigma'$ 中交换 $r_a$ 与 $r_b$ 的位置得 $\sigma''$：由于 $d_{r_b} = d_{i_a} \leq d_{r_a}$，$r_a$ 延后执行仍满足期限；反复交换，将 $\sigma'$ 转换为 $\sigma$ 而不违反任何期限。

> **实用推论**：检验 J 是否可行，只需按期限非降次序排列 J 中的作业，逐一验证 $d_{i_j} \geq j$（第 j 个作业在时间片 j 完成，需 $d_{i_j} \geq j$）。

### 3.5 算法5.4（JS）：基于直接插入排序的贪心算法

**思想**：维护已选作业集合 J，保持 J 按期限非降有序。对每个新作业 i，用直接插入法找到合适位置，验证插入后仍满足所有期限约束。

```
procedure JS(D, J, n, k)
  integer D(0:n), J(0:n), i, k, n, r
  D(0) ← J(0) ← 0
  k ← 1; J(1) ← 1
  for i ← 2 to n do
    r ← k
    // 从后向前寻找插入位置 r+1
    while D(J(r)) > D(i) and D(J(r)) ≠ r do
      r ← r - 1
    repeat
    // 若位置 r+1 合法，则将作业 i 插入
    if D(J(r)) ≤ D(i) and D(i) > r then
      for l ← k to r+1 by -1 do
        J(l+1) ← J(l)         // 位置 r+1 到 k 的作业依次后移一位
      repeat
      J(r+1) ← i
      k ← k + 1
    endif
  repeat
end JS
```

**时间复杂度**：O(n²)（外层 for 执行 n-1 次，内层 while 至多循环 k ≤ n 次）

### 3.6 优化思路：集合树

**问题**：JS 算法内层 while 循环导致 O(n²)，能否降低复杂度？

**优化策略**：用**空间换时间**。

- 定义数组 **F**：$F(d_j)$ 表示期限值为 $d_j$ 的作业 j 当前可用的最大空时间片
- 定义数组 **P**：模拟集合树，P(i) < 0 表示 i 是根节点（树中结点个数的负数），P(i) > 0 表示 i 是分支/叶结点，值为父节点下标

**集合树操作**：

```
// 查找操作（压缩规则）：找到结点 i 所在树的根节点 j，
// 并将路径上所有结点的 PARENT 直接指向 j
procedure FIND(i)
  integer i, j, k
  j ← i
  while PARENT(j) > 0 do
    j ← PARENT(j)
  repeat
  k ← i
  while k ≠ j do
    t ← PARENT(k); PARENT(k) ← j; k ← t
  repeat
  return j
end FIND
```

```
// 合并操作（加权规则）：结点数少的树挂到结点数多的树下
procedure UNION(i, j)
  integer i, j, x
  x ← PARENT(i) + PARENT(j)   // 两棵树结点数之和（取负）
  if PARENT(i) > PARENT(j)    // 树 i 的结点数少于树 j
    then PARENT(i) ← j; PARENT(j) ← x
    else PARENT(j) ← i; PARENT(i) ← x
  endif
end UNION
```

### 3.7 算法5.5（FJS）：基于集合树的贪心算法

**变量**：
- $b = \min\{n, \max\{d_j\}\}$：可能分配的时间片范围
- P(i)：集合树，节点 i 对应期限值 i
- F(i)：期限值为 i 的作业当前可用的最大时间片（仅当 i 是根节点时有效）

```
procedure FJS(D, n, b, J, k)
  // 找最优解 J=J(1),…,J(k)，已知 p1 ≥ p2 ≥ … ≥ pn，b = min{n, max{dj}}
  integer b, D(n), J(n), F(0:b), P(0:b)
  for i ← 0 to b do
    F(i) ← i; P(i) ← -1     // 初始化：每个节点独立，F(i) = i
  repeat
  k ← 0
  for i ← 1 to n do
    j ← FIND(min(n, D(i)))   // 找期限 D(i) 所在树的根节点
    if F(j) ≠ 0              // 存在可用时间片
      then k ← k+1; J(k) ← i
           l ← FIND(F(j)-1)  // 找时间片 F(j)-1 所在树的根节点
           call UNION(l, j)   // 合并两棵树
           F(j) ← F(l)       // 更新根节点的最大可用时间片
    endif
  repeat
End FJS
```

> **时间复杂度**：接近 O(n)，因为集合树的 FIND 和 UNION 操作均摊时间接近 O(1)（压缩规则 + 加权规则）。

## 四、带有期限的作业调度 — 作业题详解

### 【作业题2】定理5.3 推广到不同时间片情况

**题目**：已知定理5.3面向时间片相同的作业调度问题。设 J 是 k 个作业的集合，$\sigma = i_1,i_2,\ldots,i_k$，使得 $d_{i_1} \leq d_{i_2} \leq \cdots \leq d_{i_k}$。J 是一个可行解，当且仅当 J 中的作业可以按照 σ 的次序而又不违反任何一个期限的情况来处理。证明即使作业有不同时间片，该定理亦真（规定作业 i 的效益 $p_i > 0$，要用的处理时间 $t_i > 0$，期限 $d_i \geq t_i$）。

**证明思路**：

**充分性**（$\Leftarrow$）：若作业可按 σ 顺序处理而不违反期限，则 J 显然是可行解。

**必要性**（$\Rightarrow$）：若 J 是可行解，存在允许调度 $\sigma' = r_1, r_2, \ldots, r_k$。

设 $\sigma' \neq \sigma$，令 a 是使 $r_a \neq i_a$ 的最小下标，设 $r_b = i_a$（b > a）。

在 $\sigma'$ 中交换 $r_a$ 与 $r_b$ 的位置，得到新排列 $\sigma''$。

需验证交换后 $r_a$ 不违反期限：

设 $r_a$ 原完成时刻为 $T_a$，交换后完成时刻变为 $T_a + t_{r_b}$（推迟了 $t_{r_b}$）。

由于 $d_{r_b} = d_{i_a} \leq d_{r_a}$（按期限排序），且 $r_b$ 在 $\sigma'$ 中在 b 位置满足期限：

$$T_b' = T_{a-1} + \sum_{j=a}^{b} t_{r_j} \leq d_{r_b}$$

而 $T_{a}' = T_{a-1} + t_{r_a}$，交换后 $r_a$ 的完成时刻为 $T_{a-1} + t_{r_b} + t_{r_a} = T_b'$，仍满足 $d_{r_a} \geq d_{r_b}$，故 $r_a$ 不违反期限。

反复交换，将 $\sigma'$ 转换为 $\sigma$ 而不违反任何期限，证毕。

### 【作业题3】n=7，用算法5.4（JS）求解

**题目**：n=7，$(p_1,\ldots,p_7)=(3,5,20,18,1,6,30)$，$(d_1,\ldots,d_7)=(1,3,4,3,2,1,2)$

**第一步：按效益值降序排列作业**

| 排序后编号 | 原编号 | $p_i$ | $d_i$ |
|---|---|---|---|
| 1 | 7 | 30 | 2 |
| 2 | 3 | 20 | 4 |
| 3 | 4 | 18 | 3 |
| 4 | 6 | 6 | 1 |
| 5 | 2 | 5 | 3 |
| 6 | 1 | 3 | 1 |
| 7 | 5 | 1 | 2 |

**第二步：逐步运行 JS 算法（J 中作业按期限非降有序存储）**

**初始**：J = [7]（作业7，$d=2$），k=1

**考虑作业3**（$d=4$）：
- r=1，D(J(1))=2 ≤ D(3)=4，且 D(3)=4 > r=1，满足条件
- 插入 r+1=2 位置
- J = [7, 3]（期限 2, 4），k=2 ✓

**考虑作业4**（$d=3$）：
- r=2，D(J(2))=4 > D(4)=3，且 D(J(2))=4 ≠ r=2 → r=1
- D(J(1))=2 ≤ D(4)=3，且 D(4)=3 > r=1，满足条件
- J(3) ← J(2)（后移），插入 r+1=2 位置
- J = [7, 4, 3]（期限 2, 3, 4），k=3 ✓

**考虑作业6**（$d=1$）：
- r=3，D(J(3))=4 > 1，且 ≠3 → r=2；D(J(2))=3 > 1，且 ≠2 → r=1；D(J(1))=2 > 1，且 ≠1 → r=0
- D(J(0))=0 ≤ D(6)=1，且 D(6)=1 > r=0，满足条件
- 后移所有，插入 r+1=1 位置
- J = [6, 7, 4, 3]（期限 1, 2, 3, 4），k=4 ✓

**考虑作业2**（$d=3$）：
- r=4，D(J(4))=4 > 3，且 ≠4 → r=3；D(J(3))=3 ≤ 3 且 D(2)=3 > r=3？3 > 3 不成立
- 条件不满足，**舍弃作业2**

**考虑作业1**（$d=1$）：
- r=4，向前找，最终 r=0
- D(J(0))=0 ≤ 1，D(1)=1 > 0，插入位置1
- 但插入后 J = [1, 6, 7, 4, 3]，位置1处 $d=1$ 满足 $d \geq 1$，位置2处 $d=1 \geq 2$？**不满足**
- 实际上 D(J(1))=1 = r=1，条件 D(J(r)) ≠ r 不成立，退出循环
- 条件不满足，**舍弃作业1**

**考虑作业5**（$d=2$）：
- r=4，D(J(4))=4>2，≠4 → r=3；D(J(3))=3>2，≠3 → r=2；D(J(2))=2 ≤ 2，D(5)=2 > 2？不成立
- 条件不满足，**舍弃作业5**

**最终结果**：

J = {7, 3, 4, 6}（原编号），处理顺序：作业6 → 作业7 → 作业4 → 作业3

效益值 = 30 + 20 + 18 + 6 = **74**

### 【作业题4】n=7，用算法5.5（FJS）求解

**题目**：n=7，$(p_1,\ldots,p_7)=(40,12,30,20,7,15,10)$，$(d_1,\ldots,d_7)=(2,4,4,3,2,1,6)$

**第一步：按效益值降序排列**

| i（排序后） | 原编号 | $p_i$ | $d_i$ |
|---|---|---|---|
| 1 | 1 | 40 | 2 |
| 2 | 3 | 30 | 4 |
| 3 | 4 | 20 | 3 |
| 4 | 6 | 15 | 1 |
| 5 | 2 | 12 | 4 |
| 6 | 7 | 10 | 6 |
| 7 | 5 | 7 | 2 |

计算 b = min{7, max{2,4,4,3,2,1,6}} = min{7,6} = **6**

**初始化**（F 和 P 数组）：

| 下标 | 0 | 1 | 2 | 3 | 4 | 5 | 6 |
|---|---|---|---|---|---|---|---|
| F | 0 | 1 | 2 | 3 | 4 | 5 | 6 |
| P | -1 | -1 | -1 | -1 | -1 | -1 | -1 |

**考虑作业1**（$d_1=2$）：
- j = FIND(min(7,2)) = FIND(2) = 2，F(2)=2 ≠ 0
- 纳入 J：J(1) = 1，k=1
- l = FIND(F(2)-1) = FIND(1) = 1

## 五、最小生成树问题（5.4）

### 5.1 问题回顾

**最小生成树**：设 $G=(V,E)$ 是加权无向连通图，$|V|=n$。G 的生成树 $T=(V, E')$（$E' \subseteq E$）中权值之和最小者为最小生成树。

**贪心分析**：
- 约束条件：$|V|=n$；$|E'|=n-1$，$E' \subseteq E$；T 中无环
- 目标函数：权值和最小
- 量度标准：**边的权值从小到大排列**（把目标函数作为度量标准）

### 5.2 Kruskal 算法

```
// input：加权连通无向图 G=(V,E)
// output：G 的一棵最小生成树

T ← Φ                          // 存储选中的边
while T 所含的边数少于 n-1 do
  从 E 中选择一条最小权值的边(v,w)
  从 E 中删除该边
  if 将(v,w)加入 T 中没有形成回路
    then 将(v,w)加入 T 中
    else 放弃(v,w)
  endif
repeat
```

**时间复杂度**：$O(e \log e)$，其中 e 为边数。连通图最多 $n(n-1)/2$ 条边，故为 $O(n^2 \log n)$。

### 5.3 Kruskal 算法实例

图：A-B(300), A-E(50), A-D(80), B-C(70), B-D(75), C-D(60), D-E(90), B-E(200)

边按权值排序：AE(50) → CD(60) → BC(70) → BD(75) → AD(80) → DE(90) → BE(200) → AC(300)

| 步骤 | 选边 | 是否成环 | T |
|---|---|---|---|
| 1 | AE(50) | 否 | {AE} |
| 2 | CD(60) | 否 | {AE,CD} |
| 3 | BC(70) | 否 | {AE,CD,BC} |
| 4 | BD(75) | **是**（B-C-D 已连通） | 舍弃 |
| 5 | AD(80) | 否 | {AE,CD,BC,AD} |
| — | 已有 n-1=4 条边 | 结束 | — |

**最小生成树权值** = 50 + 60 + 70 + 80 = **260**

### 5.4 Kruskal 贪心解最优性证明

**定理**：Kruskal 算法对于每一个无向连通图 G 产生一棵最小生成树。

**证明**（替换论证法）：

设 T 是 Kruskal 产生的生成树，T' 是 G 的最小成本生成树。

若 $E(T) \neq E(T')$，设 e 是使 $e \in E(T)$ 但 $e \notin E(T')$ 的最小成本边。

将 e 加入 T'，产生一个环 $e, e_1, e_2, \ldots, e_k$。

其中至少有一条 $e_j \notin E(T)$（否则 T 包含环，与算法矛盾）。

**关键**：若 $c(e_j) < c(e)$，则 Kruskal 算法会在 e 之前考虑 $e_j$，并将 $e_j$ 纳入 T，矛盾。故 $c(e_j) \geq c(e)$。

删去 $e_j$，得新树 $T''$：$c(T'') = c(T') + c(e) - c(e_j) \leq c(T')$，故 $T''$ 也是最小成本树。

反复替换，将 T' 转换为 T 而成本不增，故 T 也是最小成本生成树，证毕。

## 六、货郎担问题（5.5）

### 6.1 问题描述

**TSP（Traveling Salesman Problem）**：从某城市出发，经过每个城市恰好一次，最后回到出发城市，要求总路程最短。

### 6.2 贪心策略（最近邻居法）

```
procedure TSP_GREEDY(dist, n, start, path, total)
  // dist(i,j)：城市 i 到 j 的距离；n：城市数；start：出发城市
  boolean visited(1:n) ← false
  path(1) ← start; visited(start) ← true
  current ← start; total ← 0
  for step ← 2 to n do
    min_dist ← ∞; next ← -1
    for j ← 1 to n do
      if not visited(j) and dist(current, j) < min_dist
        then min_dist ← dist(current, j); next ← j
      endif
    repeat
    path(step) ← next; visited(next) ← true
    total ← total + min_dist; current ← next
  repeat
  total ← total + dist(current, start)  // 返回出发城市
end TSP_GREEDY
```

**时间复杂度**：$O(n^2)$（外层 n-1 次，内层每次扫描 n 个城市）

### 6.3 贪心只得近似解

**实例**：5个城市，从城市1出发

贪心解（城市1出发）：$1 \to 4 \to 3 \to 5 \to 2 \to 1$，总距离 = **14**

实际最优解：$1 \to 2 \to 5 \to 4 \to 3 \to 1$，总距离 = **13**

> 原因：TSP 问题**不具有最优子结构性质**，从城市1出发贪心选择了距离最近的城市4（距离2），但最优解却从城市1先去城市2（距离3）。局部最优不等于全局最优。

### 6.4 贪心法与最优子结构性质

| 性质 | 说明 |
|---|---|
| **最优子结构性质** | 一个问题的最优解中包含其子问题的最优解 |
| 贪心法 ↔ 最优子结构 | 贪心法能求最优解 ⟺ 问题具有最优子结构性质 |
| **TSP** | 不具有最优子结构，贪心只得近似解 |
| **背包/作业调度/MST** | 具有最优子结构，贪心得最优解 |

## 七、拓展：汽车加油问题

### 7.1 问题描述

汽车加满油后可行驶 N 公里，旅途中有 k 个加油站，使沿途加油次数最少。

**贪心策略**：尽可能走远，只在**不得不加油时**才加油（即当前油量不足以到达下一站时，在上一个能到达的站加油）。

### 7.2 Gas 算法

```
Procedure GAS(A, n, k)
  // A(i)：第 i 个加油站与第 i+1 个加油站之间的距离（A(0) 为起点到第1站）
  // n：加满油可行驶距离；k：沿途加油站个数
  integer num, i; real s
  num ← 0; s ← 0; B(0:k) ← 0
  for i ← 0 to k do
    if A(i) > n then Print("no solution") endif  // 相邻站距离超过满箱续航
  repeat
  for i ← 0 to k do
    s ← s + A(i)               // 假设在第 i 个加油站不加油，继续行驶
    if s > n                   // 需要行驶里程超过满箱续航
      then num ← num + 1
           B(i) ← 1            // 在第 i 个加油站加油
           s ← A(i)            // 从第 i 站重新开始计距
    endif
  repeat
end GAS
```

**实例**：距离序列 = [1,2,3,4,5,1,6,6]，N=7

| 步骤 | 累计 s | 是否加油 | 加油站编号 |
|---|---|---|---|
| A(0)=1 | s=1 | 否 | — |
| A(1)=2 | s=3 | 否 | — |
| A(2)=3 | s=6 | 否 | — |
| A(3)=4 | s=10>7 | **是** | 3（第1次） |
| A(4)=5 | s=4+5=9>7 | **是** | 4（第2次） |
| A(5)=1 | s=5+1=6 | 否 | — |
| A(6)=6 | s=6+6=12>7 | **是** | 6（第3次） |
| A(7)=6 | s=6+6=12>7 | **是** | 7（第4次） |

结果：加油4次，在加油站3、4、6、7处加油。

## 八、其余作业题详解

### 【作业题5】磁带程序存放问题（ERT 最小化）

**题目**：将长为 $l_1, l_2, \ldots, l_n$ 的 n 个程序存入磁带，程序被检索的频率是 $f_i$。若程序按 $i_1, i_2, \ldots, i_n$ 次序存放，则期望检索时间（ERT）为：

$$ERT = \left[\sum_j \left(f_{i_j} \sum_{k=1}^{j} l_{i_k}\right)\right] / \sum f_i$$

**① 证明按 $l_i$ 非降次序存放程序不一定得到最小的 ERT**

**反例**：n=2，$l_1=1, l_2=2, f_1=1, f_2=100$

- 按 $l_i$ 非降：存放顺序 1,2（$l_1 < l_2$）
  $ERT = (f_1 \cdot l_1 + f_2 \cdot (l_1+l_2)) / (f_1+f_2) = (1\cdot1 + 100\cdot3)/101 = 301/101 \approx 2.98$

- 按另一顺序 2,1：
  $ERT = (100\cdot2 + 1\cdot(2+1))/101 = 203/101 \approx 2.01$

故按 $l_i$ 非降不一定最优。

**② 证明按 $f_i$ 非增次序存放程序不一定得到最小的 ERT**

**反例**：n=2，$l_1=100, l_2=1, f_1=2, f_2=1$

- 按 $f_i$ 非增：存放顺序 1,2（$f_1 > f_2$）
  $ERT = (2\cdot100 + 1\cdot(100+1))/3 = 301/3 \approx 100.3$

- 按另一顺序 2,1：
  $ERT = (1\cdot1 + 2\cdot(1+100))/3 = 203/3 \approx 67.7$

故按 $f_i$ 非增不一定最优。

**③ 证明按 $f_i/l_i$ 非增次序存放时 ERT 取最小值**

**证明**（相邻交换论证）：

不妨设 $\sum f_i = F$ 为常数，最小化 $\sum_j f_{i_j} \sum_{k=1}^{j} l_{i_k}$，展开得：

$$\text{目标} = f_{i_1}l_{i_1} + f_{i_2}(l_{i_1}+l_{i_2}) + \cdots = \sum_j f_{i_j}\sum_{k\leq j}l_{i_k} = \sum_j l_{i_j}\sum_{k \geq j} f_{i_k}$$

考虑相邻两个程序 $i_j = a, i_{j+1} = b$，其贡献为：

$$C(a,b) = (\cdots) + f_a \cdot L_a + f_b \cdot (L_a + l_a + l_b) + (\cdots)$$

其中 $L_a$ 为 a 之前所有程序长度之和（与 a,b 顺序无关）。

将 $a, b$ 互换，贡献变为：

$$C(b,a) = (\cdots) + f_b \cdot L_a + f_a \cdot (L_a + l_b + l_a) + (\cdots)$$

$C(a,b) \leq C(b,a)$ 当且仅当 $f_a l_b \leq f_b l_a$，即 $f_a/l_a \geq f_b/l_b$。

因此，当相邻程序满足 $f_a/l_a \geq f_b/l_b$ 时顺序最优，即按 $f_i/l_i$ 非增排列时总代价最小，ERT 取最小值，证毕。

### 【作业题6】集装箱装船问题

**题目**：有编号为 1,2,…,n 的集装箱准备装上轮船，集装箱重量为 $w_i$，轮船最大承重量为 C，无体积限制，如何选择使装上集装箱数量最多。

**① 贪心方法可以求得最优解**

**贪心策略**：按重量 $w_i$ **非降次序**选择集装箱，直到无法再装为止。

**② 最优量度标准证明（替换论证）**

**定理**：按重量非降次序贪心选择，得到的是装箱数量最多的最优解。

**证明**：设贪心解为 $J = \{j_1, j_2, \ldots, j_k\}$（$w_{j_1} \leq w_{j_2} \leq \cdots \leq w_{j_k}$），最优解为 $I = \{i_1, i_2, \ldots, i_m\}$（$m \geq k$）。

若 $m > k$：最优解装了更多箱子，总重量 $\sum_{t=1}^{m} w_{i_t} \leq C$。

由贪心法，$j_{k+1}$ 是未被选入 J 的重量最小的箱子，$w_{j_{k+1}} \leq w_{i_1}$（贪心已选了最轻的 k 个）。

则 $\sum_{t=1}^{k} w_{j_t} \leq \sum_{t=1}^{k} w_{i_t} \leq \sum_{t=1}^{m} w_{i_t} - w_{i_{k+1}} \leq C - w_{j_{k+1}}$。

即 $\sum_{t=1}^{k} w_{j_t} + w_{j_{k+1}} \leq C$，贪心法还可以继续装入 $j_{k+1}$，矛盾。

故 $m = k$，贪心解是最优解，证毕。

**③ 贪心算法描述**

```
procedure SHIP(W, C, n, J, k)
  // W(1:n)：集装箱重量，已按非降次序排列
  // C：最大承重；J：解集合；k：装箱数量
  integer i, k, n; real cu, W(1:n)
  sort W in non-decreasing order
  cu ← C; k ← 0
  for i ← 1 to n do
    if W(i) ≤ cu
      then k ← k+1; J(k) ← i; cu ← cu - W(i)
      else exit
    endif
  repeat
end SHIP
```

**时间复杂度**：排序 $O(n\log n)$，选择 $O(n)$，总体 $O(n\log n)$。

### 【作业题7】顾客服务问题

**题目**：n 个顾客同时等待服务，服务时间 $t_i$（i=1,…,n），从时刻0开始，在时刻 t 开始对顾客 i 服务，顾客 i 的等待时间为 t。安排 n 个顾客的服务次序，使总等待时间（每个顾客等待时间之和）最少。

**① 最优贪心策略及证明**

**贪心策略**：按服务时间 $t_i$ **非降次序**安排服务。

**证明（相邻交换）**：设服务顺序为 $\sigma = i_1, i_2, \ldots, i_n$，顾客 $i_j$ 在时刻 $\sum_{k=1}^{j-1} t_{i_k}$ 开始等待，等待时间为 $\sum_{k=1}^{j-1} t_{i_k}$。

总等待时间：
$$W(\sigma) = \sum_{j=1}^{n} \sum_{k=1}^{j-1} t_{i_k} = \sum_{j=1}^{n} (n-j) t_{i_j}$$

考虑相邻顾客 $i_j = a, i_{j+1} = b$，对总等待时间的贡献（忽略与顺序无关的部分）：

$$\text{贡献}(a,b) = (n-j) t_a + (n-j-1) t_b$$

交换后：

$$\text{贡献}(b,a) = (n-j) t_b + (n-j-1) t_a$$

$\text{贡献}(a,b) \leq \text{贡献}(b,a)$ 当且仅当 $(n-j)t_a + (n-j-1)t_b \leq (n-j)t_b + (n-j-1)t_a$，即 $t_a \leq t_b$。

因此当 $t_a \leq t_b$ 时顺序最优，即按 $t_i$ 非降排列时总等待时间最少。

**② 贪心算法描述**

```
procedure SERVICE(T, n, order, total_wait)
  // T(1:n)：服务时间；order：服务顺序；total_wait：总等待时间
  sort T in non-decreasing order, record order
  total_wait ← 0; current_time ← 0
  for i ← 1 to n do
    total_wait ← total_wait + current_time   // 顾客 order(i) 等待 current_time
    current_time ← current_time + T(order(i))
  repeat
end SERVICE
```

**时间复杂度**：排序 $O(n\log n)$，计算 $O(n)$，总体 $O(n\log n)$。

### 9.2 贪心法最优解证明方法总结

**替换论证法**（通用框架）：

1. 设贪心解为 X，假设存在更优解 Y（$X \neq Y$）
2. 找到 X 与 Y 第一个不同的分量（下标 k）
3. 构造新解 Z：将 $y_k$ 替换为 $x_k$，同时调整其余分量以保持可行性
4. 证明 Z 的目标函数值 **≥ Y 的目标函数值**（利用量度标准的性质）
5. 若 Z 严格优于 Y，则 Y 不是最优解，矛盾
6. 反复替换，将 Y 转换为 X 且目标函数值不减，故 X 也是最优解

**相邻交换论证法**（适用于排列型问题）：
- 假设最优顺序中有相邻元素不满足贪心条件，交换后目标函数值**不增**
- 说明满足贪心条件的排列与最优排列目标函数值相同，故贪心排列也是最优解

### 9.3 易错点汇总

1. **分数背包 vs 0/1背包**：$p_i/w_i$ 非增策略对分数背包给出最优解，对 0/1背包只给近似解（物品不能拆分时，放满背包不一定最优）。

2. **作业调度可行性检验**：对作业集合 J，按期限非降排序后，第 j 个作业的期限必须 $\geq j$（时间片编号），两个条件缺一不可。

3. **FJS 中的 F 数组含义**：F(j) 表示**以 j 为根的集合树所对应的最大可用时间片**，UNION 后新根的 F 值要更新为合并后更小的那个。

4. **Kruskal 判环**：实际实现中用并查集（集合树）判断两端点是否已连通，时间复杂度远优于 DFS/BFS。

5. **TSP 无最优子结构**：贪心每步选最近城市，不能保证全局最优，因为路径是闭合的，后期选择会影响最终回路代价。

6. **量度标准需要证明**：贪心法的"好"量度标准不能仅凭直觉，必须通过替换论证法或相邻交换论证法严格证明。

7. **集合树中 UNION 的加权规则方向**：节点数少的树挂到节点数多的树下（P 值取负数表示根，绝对值为节点数），合并后更新根节点的 F 值。
