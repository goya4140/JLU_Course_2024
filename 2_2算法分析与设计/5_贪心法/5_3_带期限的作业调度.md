## 三、带有期限的作业调度问题（5.3）

### 3.1 问题描述

- 一台机器处理 n 个作业，每个作业均可在**单位时间**内完成
- 每个作业 i 有截止期限 $d_i > 0$（整数），在期限前完成获得效益 $p_i > 0$
- **可行解**：作业子集 J，J 中每个作业都能在各自截止期限前完成
- **最优解**：效益值 $\sum_{j \in J} p_j$ 最大的可行解

**实例**：n=4，$(p_1,p_2,p_3,p_4)=(100,10,15,20)$，$(d_1,d_2,d_3,d_4)=(2,1,2,1)$

| 可行解 | 处理顺序 | 效益值 |
|---|---|---|
| {1} | 1 | 100 |
| {1,2} | 2,1 | 110 |
| {1,3} | 1,3 | 115 |
| **{1,4}** | **4,1** | **120** ✓ |
| {2,3} | 2,3 | 25 |
| {3,4} | 4,3 | 35 |

---

### 3.2 GREEDY_JOB 算法

**量度标准**：将各作业按效益值 $p_i$ 降序排列（$p_1 \geq p_2 \geq \cdots \geq p_n$），依次尝试加入，若 $J \cup \{i\}$ 仍是可行解则纳入。

```
procedure GREEDY_JOB(D, J, n)
  // 作业已按 p1 ≥ p2 ≥ … ≥ pn 的次序输入
  // 期限值 D(1:n) ≥ 1；J 是最优解
  J ← {1}
  for i ← 2 to n do
    if (J ∪ {i} 的所有作业都能在它们的截止期限前完成)
      then J ← J ∪ {i}
    endif
  repeat
end GREEDY_JOB
```

---

### 3.3 定理5.2：贪心解最优性证明

**定理5.2**：算法 GREEDY_JOB 所描述的贪心方法总是得到一个最优解。

**证明**：设 J 是贪心解，I 是最优解，若 I = J 则结论显然成立。以下设 I ≠ J，分三种情况：

**情况一：$I \subset J$**

J 比 I 多至少一个作业，J 的效益 > I 的效益，与 I 是最优解矛盾，故 $I \subset J$ 不可能。

**情况二：$J \subset I$**

存在作业 $a \in I$ 且 $a \notin J$，对 $\forall b \in J$ 有 $b \in I$。

- 若 $\forall b \in J$，$p_a < p_b$：则 $J \cup \{a\}$ 是可行解（a 在 I 中满足期限），贪心法应将 a 纳入 J，与 $a \notin J$ 矛盾。
- 若 a 比 J 中某作业 c 效益大：贪心法按效益降序处理，a 排在 c 之前被考虑，必先于 c 纳入 J，与 $a \notin J$ 矛盾。

故 $J \subset I$ 不可能。

**情况三：I 和 J 互不包含**

设 $a$ 是 J 中效益最高且 $a \notin I$ 的作业。由贪心法可知，对所有 $b \in I$ 且 $b \notin J$ 的作业，均有 $p_a \geq p_b$（若 $p_b > p_a$，贪心法会先于 a 考虑 b 并将其纳入 J，矛盾）。

设 $S_J$ 和 $S_I$ 分别是 J 和 I 的可行调度表，通过相邻交换令两表中相同作业在相同时间片执行，然后用 J 中作业 a 替换 I 中占用同一时间片的作业 b，得到新集合 $I' = I - \{b\} \cup \{a\}$。

由于 $p_a \geq p_b$，I' 的效益值不小于 I 的效益值，且 I' 比 I 少了一个与 J 不同的作业。反复替换，最终将 I 转换为 J 且效益值不减，故 **J 也是最优解**。$\square$

### 3.4 定理5.3：可行调度判断

**定理5.3**：设 J 是 k 个作业的集合，$\sigma = i_1, i_2, \ldots, i_k$ 是 J 中作业按期限非降排列（$d_{i_1} \leq d_{i_2} \leq \cdots \leq d_{i_k}$）。J 是一个可行解，**当且仅当** J 中的作业可以按照 $\sigma$ 的次序处理而不违反任何期限。

**证明**：

**充分性**（$\Leftarrow$）：若作业可按 $\sigma$ 顺序处理而不违反期限，则 J 显然是可行解。

**必要性**（$\Rightarrow$）：若 J 是可行解，存在允许调度 $\sigma' = r_1, r_2, \ldots, r_k$（满足 $d_{r_j} \geq j$）。若 $\sigma' \neq \sigma$，令 a 是使 $r_a \neq i_a$ 的最小下标，设 $r_b = i_a$（$b > a$）。

在 $\sigma'$ 中交换 $r_a$ 与 $r_b$ 的位置，得新排列 $\sigma''$。由于 $d_{r_b} = d_{i_a} \leq d_{r_a}$（$\sigma$ 按期限非降），$r_a$ 延后到位置 b 后 $d_{r_a} \geq d_{r_b} \geq b$，仍满足期限。反复交换，将 $\sigma'$ 转换为 $\sigma$ 而不违反任何期限。$\square$

> **实用推论**：检验 J 是否可行，只需按期限非降次序排列 J 中的作业，逐一验证第 j 个作业满足 $d_{i_j} \geq j$。

### 3.5 算法5.4（JS）：基于直接插入排序的贪心算法

**思想**：维护已选作业集合 J，保持 J 按期限非降有序。对每个新作业 i，从后向前寻找插入位置 r+1，验证插入后 J 中各作业仍满足 $d_{J(l)} \geq l$。

> **关键**：向后移动的条件是 $D(J(l)) > l$（该位置"未紧绷"），若 $D(J(l)) = l$ 则说明该位置期限已紧绷，无法腾出空间，停止搜索。

```
procedure JS(D, J, n, k)
    integer D(0:n), J(0:n), i, k, n, r     // k：当前已选作业的数量（也是集合 J 的有效长度）。
    D(0) ← 0; J(0) ← 0                     // 初始化：设置哨兵节点，0号位置和0号截止时间
    k ← 1; J(1) ← 1                         // 初始化：默认选中第1个作业，已选集合大小为1
    for i ← 2 to n do                       // 外层循环：从第2个作业开始，依次尝试加入集合
        r ← k                               // 初始化游标 r 为当前已选集合的末尾位置
        while D(J(r)) > D(i) and D(J(r)) ≠ r do  // 从后向前查找插入位置
            r ← r - 1                       // 如果当前位置的截止时间大于新作业，且未紧绷，则继续前移
        repeat
        if D(J(r)) ≤ D(i) and D(i) > r then  // 检查是否可插入：新作业截止时间 ≤ 当前位置，且位置未占用
            for l ← k to r+1 by -1 do       // 从后向前腾出插入空间（将r+1到k的元素后移）
                J(l+1) ← J(l)
            repeat
            J(r+1) ← i                      // 将新作业 i 插入到位置 r+1
            k ← k + 1                       // 已选作业数量加1
        endif
    repeat
end JS
```

**时间复杂度**：$O(n^2)$（外层 for 执行 n-1 次，内层 while 至多循环 k ≤ n 次）

### 3.6 优化思路：集合树

**问题**：JS 算法内层 while 循环导致 $O(n^2)$，希望通过"空间换时间"降低复杂度。

**数组 F**：$F(i)$ 表示期限值为 i 的作业当前可用的**最大空时间片**（i 为根节点时有效）。

**数组 P**：模拟集合树，同一棵树的节点共享相同的最大可用时间片（根节点的 F 值）：
- $P(i) < 0$：i 是根节点，$|P(i)|$ 为树中节点个数
- $P(i) > 0$：i 是分支/叶节点，$P(i)$ 为父节点下标

```
procedure FIND(i)
  // 查找结点 i 的根节点，并对路径上所有结点做路径压缩
  integer i, j, k
  j ← i
  while PARENT(j) > 0 do
    j ← PARENT(j)
  repeat
  k ← i
  while k ≠ j do
    t ← PARENT(k); PARENT(k) ← j; k ← t
  repeat
  return j
end FIND
```

```
procedure UNION(i, j)
  // 加权规则：结点数少的树挂到结点数多的树下
  integer i, j, x
  x ← PARENT(i) + PARENT(j)
  if PARENT(i) > PARENT(j)
    then PARENT(i) ← j; PARENT(j) ← x
    else PARENT(j) ← i; PARENT(i) ← x
  endif
end UNION
```

> **为什么复杂度接近 O(n)**：采用"加权规则合并 + 压缩规则查找"后，n 个元素上的 m 次 FIND/UNION 操作总时间为 $O(m \cdot \alpha(m,n))$，$\alpha$ 为增长极缓慢的 Ackermann 反函数，实际使用中接近常数，故整体接近 $O(n)$。

### 3.7 算法5.5（FJS）：基于集合树的贪心算法

**变量**：$b = \min\{n, \max\{d_j\}\}$；F(i)：期限 i 的最大可用时间片；P(i)：集合树父指针。

**逻辑**：对作业 i，用 FIND 找 $\min(n, d_i)$ 所在树的根 j，查 F(j)：若 $F(j) \neq 0$ 则纳入，将时间片 F(j) 占用后与 F(j)-1 所在树合并；若 $F(j) = 0$ 则舍弃。

```
procedure FJS(D, n, b, J, k)
  // 已知 p1 ≥ p2 ≥ … ≥ pn，b = min{n, max{dj}}
  integer b, D(n), J(n), F(0:b), P(0:b)
  for i ← 0 to b do
    F(i) ← i; P(i) ← -1
  repeat
  k ← 0
  for i ← 1 to n do
    j ← FIND(min(n, D(i)))
    if F(j) ≠ 0
      then k ← k+1; J(k) ← i
           l ← FIND(F(j)-1)
           call UNION(l, j)
           F(j) ← F(l)
    endif
  repeat
end FJS
```

**时间复杂度**：接近 $O(n)$。

## 四、带有期限的作业调度 — 作业题详解

### 【作业题2】定理5.3 推广到不同时间片情况

**题目**：规定作业 i 的效益 $p_i>0$，处理时间 $t_i>0$，期限 $d_i \geq t_i$。证明即使作业有不同时间片，定理5.3 亦真。

**证明**：

**充分性**（$\Leftarrow$）：显然。

**必要性**（$\Rightarrow$）：若 J 是可行解，存在允许调度 $\sigma' = r_1, r_2, \ldots, r_k$。设 $\sigma' \neq \sigma$，令 a 是使 $r_a \neq i_a$ 的最小下标，$r_b = i_a$（$b > a$）。

在 $\sigma'$ 中交换 $r_a$ 与 $r_b$，得 $\sigma''$。$r_a$ 在 $\sigma''$ 中的完成时刻为：

$$T_{a-1} + t_{r_b} + t_{r_a}$$

由于 $r_b$ 在 $\sigma'$ 的位置 b 满足期限：

$$T_{a-1} + \sum_{j=a}^{b} t_{r_j} \leq d_{r_b}$$

故 $r_a$ 的完成时刻 $\leq d_{r_b} \leq d_{r_a}$（因 $\sigma$ 按期限非降，$d_{i_a} = d_{r_b} \leq d_{r_a}$），$r_a$ 满足期限。反复交换，将 $\sigma'$ 转换为 $\sigma$ 而不违反任何期限。$\square$

### 【作业题3】n=7，用算法5.4（JS）求解

**数据**：$(p_1,\ldots,p_7)=(3,5,20,18,1,6,30)$，$(d_1,\ldots,d_7)=(1,3,4,3,2,1,2)$

**第一步：按效益值降序排列**

| 考虑顺序 | 原编号 | $p_i$ | $d_i$ |
|---|---|---|---|
| 1 | 7 | 30 | 2 |
| 2 | 3 | 20 | 4 |
| 3 | 4 | 18 | 3 |
| 4 | 6 | 6  | 1 |
| 5 | 2 | 5  | 3 |
| 6 | 1 | 3  | 1 |
| 7 | 5 | 1  | 2 |

**第二步：逐步运行 JS（J 内作业按期限升序存储，哨兵 D(0)=J(0)=0）**

**初始**：J = [7]（$d=2$），k=1

**考虑作业3**（$d=4$）：r=1，$D(J(1))=2 \leq 4$ 停止；$D(3)=4>1$ ✓，插入位置2。
J = [7, 3]（期限 2,4），k=2 ✓

**考虑作业4**（$d=3$）：r=2，$D(J(2))=4>3$ 且 $\neq 2$ → r=1；$D(J(1))=2 \leq 3$ 停止；$D(4)=3>1$ ✓，后移位置2，插入位置2。
J = [7, 4, 3]（期限 2,3,4），k=3 ✓

**考虑作业6**（$d=1$）：r=3，$D(J(3))=4>1,\neq 3$ → r=2；$D(J(2))=3>1,\neq 2$ → r=1；$D(J(1))=2>1,\neq 1$ → r=0；$D(J(0))=0 \leq 1$ 停止；$D(6)=1>0$ ✓，后移全部，插入位置1。
J = [6, 7, 4, 3]（期限 1,2,3,4），k=4 ✓

**考虑作业2**（$d=3$）：r=4，$D(J(4))=4>3,\neq 4$ → r=3；$D(J(3))=3 \leq 3$ 停止；$D(2)=3>3$？**不成立**，舍弃。

**考虑作业1**（$d=1$）：r=4 → r=3 → r=2 → r=1；$D(J(1))=d_6=1$，$1>1$？**不成立**，while 退出（$D(J(r))=r=1$ 条件触发）；$D(1)=1>1$？**不成立**，舍弃。

**考虑作业5**（$d=2$）：r=4 → r=3 → r=2；$D(J(2))=d_7=2$，$2>2$？**不成立**，while 退出（$D(J(r))=r=2$ 条件触发）；$D(5)=2>2$？**不成立**，舍弃。

**J 变化过程汇总**：

| 考虑作业（原编号） | $p_i$ | $d_i$ | 操作 | J（原编号，期限升序） |
|---|---|---|---|---|
| 作业7 | 30 | 2 | 纳入 | [7]，期限(2) |
| 作业3 | 20 | 4 | 纳入 | [7, 3]，期限(2,4) |
| 作业4 | 18 | 3 | 纳入 | [7, 4, 3]，期限(2,3,4) |
| 作业6 | 6  | 1 | 纳入 | [6, 7, 4, 3]，期限(1,2,3,4) |
| 作业2 | 5  | 3 | 舍弃 | 不变 |
| 作业1 | 3  | 1 | 舍弃 | 不变 |
| 作业5 | 1  | 2 | 舍弃 | 不变 |

**最终结果**：处理顺序 作业6 → 作业7 → 作业4 → 作业3，效益值 = 6+30+18+20 = **74**
