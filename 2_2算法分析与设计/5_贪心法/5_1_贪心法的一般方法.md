# 第五章 贪心方法 · 5.1 一般方法

---

## 一、贪心方法适用的问题特点

有这样一类问题：它有 $n$ 个输入，而它的解就是这 $n$ 个输入的**某个子集**，这些子集必须满足某些事先给定的条件。

| 术语 | 定义 |
|---|---|
| **约束条件** | 必须满足的条件 |
| **可行解** | 满足约束条件的子集 |
| **目标函数** | 为了衡量可行解的优劣，预先给定的衡量标准，以函数形式给出 |
| **最优解** | 使目标函数取极值（极大值或极小值）的可行解 |

### 与实例的对应（找零钱问题）

以"找零钱、张数最少"为例：
- **输入**：可用的各种面额的硬币/纸币
- **约束条件**：所选币值之和必须恰好等于待找零钱数
- **可行解**：任意满足"总和等于待找零钱数"的币值组合
- **目标函数**：所用张数
- **最优解**：使张数最少的可行解

> 这类问题的共同特点是：从一组候选输入中，按某种规则逐步构造出一个子集，使该子集既满足约束、又使目标函数取得最优值。

## 二、贪心法的求解思想

贪心方法是一种**"只顾眼前"**的分级处理方法：

1. 根据题意选取一种**量度标准**；
2. 按该标准**一次选中一个输入**；
3. 如果这个输入和当前的部分解加在一起满足约束条件，则将其加入到部分解中；否则舍弃掉。

### 思考：贪心法得到的可行解是否一定是问题的最优解？

**不一定**。这取决于贪心法中的**量度标准**是否选取得当——只有当量度标准被证明为"最优量度标准"时，贪心法才能保证得到最优解。

> 这是贪心法的核心难点：算法框架本身很简单，真正的设计工作在于**找到正确的量度标准并加以证明**。

## 三、算法5.1：贪心法的抽象化控制

```
Procedure GREEDY(A, n)
// A(1:n) 包含 n 个输入
solution ← Φ                          // 解向量 solution 初始化为空
for i ← 1 to n do
    x ← SELECT(A)
    if FEASIBLE(solution, x)
        then solution ← UNION(solution, x)
    endif
repeat
return (solution)
end GREEDY
```

### 三个核心子过程

| 子过程 | 功能 |
|---|---|
| **SELECT** | 按某种最优量度标准从 $A$ 中选择一个输入赋值给 $x$，并从 $A$ 中消去它 |
| **FEASIBLE** | 判定 $x$ 是否可以包含在解向量中（即加入后是否仍满足约束条件） |
| **UNION** | 将 $x$ 与解向量结合，并修改约束判定 |

### 框架解读

GREEDY 算法本身只是一个**通用骨架**：它对所有满足"5.1节问题特点"的问题都适用，但 SELECT、FEASIBLE、UNION 三个子过程的具体实现因问题而异。设计一个贪心算法，本质上就是为这三个子过程填充具体内容——尤其是 SELECT 中所采用的量度标准。

## 四、贪心法的特点

### 4.1 核心问题

贪心法设计求解的核心问题是：**选择能产生问题最优解的量度标准，即最优量度标准**。

### 4.2 缺点

- 不是对所有问题都能得到最优解；
- 基于目标函数制定的度量标准不一定是最优的；
- 最优量度标准需要经过证明。

### 4.3 优点

- 一旦证明成立后，它就是一种高效的算法；
- 策略的构造简单易行；
- 对许多问题都能产生整体最优解或者近似最优解。

---

## 五、补充说明

贪心方法是指在对问题求解时，总是做出在当前看来是最好的选择。贪心方法的思想可以追溯到1823年 J.C. Warnsdorff 解决**马踏棋盘问题**时给出的一个著名算法：将马放到国际象棋的 $8\times8$ 棋盘 board 上的某个方格中，马按走棋规则移动，要求每个方格进入且只进入一次，走遍棋盘上的64个方格，将数字 $1,2,\ldots,64$ 依次填入一个 $8\times8$ 的方阵，找出一种可行方案。

贪心方法应用的**典型问题**有：背包问题、带有期限的作业排序、最小生成树、单源点最短路径等问题。

## 六、本节小结与考点

| 考点 | 内容 |
|---|---|
| 四个核心术语 | 约束条件、可行解、目标函数、最优解，要能准确定义并能在具体问题中对号入座 |
| 贪心法求解步骤 | 选量度标准 → 逐个选输入 → 可行则纳入，否则舍弃 |
| 决定最优性的关键环节 | **量度标准**——贪心法是否得到最优解，完全取决于量度标准是否最优 |
| GREEDY 抽象框架 | 三个子过程 SELECT / FEASIBLE / UNION 的职责划分要清楚 |
| 贪心法的根本局限 | 只做局部最优决策，不回溯，因此"最优量度标准需要证明"是贯穿全章的核心任务（5.2~5.4节的定理5.1、5.2、5.3均是此类证明，5.5节则是反例） |

> **承上启下**：5.1节给出的是贪心法的"通用骨架"，后续 5.2（背包问题）、5.3（作业调度）、5.4（最小生成树）分别展示了三个贪心法**成功**得到最优解的实例（均给出了严格证明），5.5（货郎担问题）则展示了贪心法**失败**只能得到近似解的反例。理解这一对比，是把握全章逻辑的关键。
