## 一、算法（2.1）

### 1.1 什么是算法

**算法（Algorithm）**：解决一确定类问题的任意一种**特殊**的方法。

- **数值计算方法**：求解数值问题，如插值计算、数值积分等。
- **非数值计算方法**：求解非数值问题，主要进行判断比较。

> **Knuth 给出的非形式描述**：算法就是一组**有穷的规则**，它规定了解决某一特定类型问题的一系列运算。

### 1.2 算法的五个重要特性（★重点考查★）

| 特性 | 英文 | 含义 | 反例 |
|------|------|------|------|
| **确定性** | Definiteness | 每一种运算必须有确切定义，无二义性 | "6或7与x相加" |
| **能行性** | Effectiveness | 运算都是基本运算，原理上能用纸和笔在有限时间完成 | 除以零（5/0）|
| **输入** | Input | 有 0 个或多个输入；在算法开始前，从特定对象集合中取值 | — |
| **输出** | Output | 一个或多个输出，与输入有特定关系 | — |
| **有穷性** | Finiteness | 在执行了有穷步运算后终止（必须在现代计算机上有效） | 无限循环 |

> **注意**：能行性中"有限时间"是**非常有限**，必须在现代计算机上有效才行。

### 1.3 计算过程与算法的区别

- **计算过程**（Computational Procedure）：可以**不满足**算法的有穷性。
  - 例：操作系统——当没有任务时处于等待状态，直到有新任务启动，因而**不是**算法。
- **程序 = 算法 + 数据结构**（By Niklaus Wirth）

### 1.4 问题的求解过程

```
理解问题
    ↓
选择数据结构和算法设计策略（核心决策）
    ↓
设计算法
    ↓
证明正确性
    ↓
分析算法  ←──── 作时空性能分布图（事后测试）
    ↓
设计程序
```

> 若分析算法或证明正确性发现问题，需返回重新选择数据结构和算法设计策略。

### 1.5 算法学习的五个基本内容

| 内容 | 说明 |
|------|------|
| **如何设计算法** | 运用基本设计策略（被实践证明有用的方法）规划算法 |
| **如何表示算法** | 采用结构程序设计方式；用 SPARKS 语言简单明了地描述 |
| **如何确认算法** | 证明对所有合法输入都能给出正确答案（穷举法、数学归纳法、反证法、构造性证明、测试）|
| **如何分析算法** | 对计算时间和存储空间做定量分析 |
| **如何测试程序** | 调试 + 时空分布图（事后测试） |

## 二、分析算法（2.2）

### 2.1 算法分析的目的

**算法分析（Analysis of Algorithms）**：对一个算法需要多少**计算时间**和**存储空间**作定量的分析。

- 确定算法在什么样的环境下能够有效地运行
- 分析在**最好、最坏和平均**情况下的执行情况
- 对同一问题不同算法的有效性作出比较

### 2.2 时间复杂性的形式化定义

设 n 为问题规模（输入量、输出量，或两者之和，或其中之一的某种测度）：

- **算法的时间复杂性** T(n)
- **算法的空间复杂性** S(n)

三种情况下的时间复杂性：

| 情况 | 形式定义 |
|------|---------|
| **最坏情况** | $T_{\max}(n) = \max\{T(I) \mid \text{size}(I) = n\}$ |
| **最好情况** | $T_{\min}(n) = \min\{T(I) \mid \text{size}(I) = n\}$ |
| **平均情况** | $T_{\text{avg}}(n) = \sum_{\text{size}(I)=n} p(I) \cdot T(I)$ |

其中 $I$ 是问题规模为 $n$ 的实例，$p(I)$ 是实例 $I$ 出现的概率。

### 2.3 算法分析过程

算法的全面分析分两个阶段：

**阶段一：准备工作（最重要、最富于创造性）**
1. 确定算法所涉及的运算（基本运算 / 复杂运算）
2. 确定出能反映算法在各种情况下工作的数据集（最好/最坏/有代表性情况）

**阶段二：事前分析（a priori analysis）**
- 确定每条语句的执行次数（频率计数）
- 求出该算法的时间限界函数（关于问题规模 n 的函数）

**阶段三：事后测试（a posterior testing）**
- 用各种给定数据执行调试后的程序
- 测定计算时间和空间，作时空性能分布图

### 2.4 基本运算与复杂运算

**基本运算**：时间囿界于常数的运算（每种运算只花费一个固定量的时间）
- 加、减、乘、除整数算术运算
- 浮点算术、字符比较、对变量赋值、过程调用等

**复杂运算**：由基本运算的任意长序列组成
- 例：两个字符串的比较——时间总量取决于字符串的长度

### 2.5 频率计数与算法数量级

**频率计数（Frequency Count）**：同一条语句在算法中的执行次数。
- 由算法直接确定，与所用的机器无关，独立于程序设计语言

**语句的时间总量** = 频率计数 × 执行一次该语句所需的时间

**算法的执行时间** = 构成算法的所有语句的执行时间总量之和

**三种程序段的频率计数示例：**

| 程序段 | `x ← z+y` 的执行次数 |
|--------|---------------------|
| 程序段1（单独一句）| 1 次 |
| 程序段2（单层 for 循环）| n 次 |
| 程序段3（双重 for 循环）| n² 次 |

> **算法的数量级**（Order of Magnitude）：算法所有语句执行频率之和，本质上反映了算法所需的计算时间。

## 三、计算时间的渐进表示（★核心考查★）

### 3.1 符号约定

- **n**：问题规模
- **f(n)**：算法的计算时间（与机器和语言有关）
- **g(n)**：事前分析中确定的某个形式简单的函数（**独立于**机器和语言）

> f(n) 与机器和语言有关，而 g(n) 是独立于机器和语言的。

### 3.2 定义 2.1：大 O 记号（上界）

> **定义**：如果存在两个正常数 c 和 n₀，对于所有的 n ≥ n₀，有 |f(n)| ≤ c|g(n)|，则记作：**f(n) = O(g(n))**。

**含义**：
- 当 n 充分大时，f(n) 有上界，c×g(n) 是 f(n) 的一个上界
- f(n) 的阶**不高于** g(n) 的阶
- 在确定 f(n) 的数量级时，总是试图求出**最小的** g(n)

**证明关键**：找出 c 和 n₀。

**常见判断示例：**

| f(n) | g(n) | 是否 f(n) = O(g(n)) | 证明/说明 |
|------|------|---------------------|---------|
| 3n | n | ✅ 是 | 取 c=3, n₀=1，∀n≥1 有 3n ≤ 3n |
| n+1024 | 1025n | ✅ 是 | 取 c=1, n₀=1 |
| 2n²+11n-10 | 3n² | ✅ 是 | 取 c=1, n₀≥一定值 |
| n² | n³ | ✅ 是 | n² ≤ n³ 对 n≥1 成立（c=1, n₀=1）|
| **n³** | n² | ❌ 否 | 假设∃c,n₀，n≥n₀时 n³≤cn²；构造n₁=max{n₀,c}+1，n₁³>cn₁²，矛盾 |

### 3.3 O 记号的性质（对非负 f(n) 和 g(n)）

| 性质 | 公式 |
|------|------|
| 1 | O(f(n)) + O(g(n)) = **O(max{f(n), g(n)})** |
| 2 | O(f(n)) + O(g(n)) = O(f(n) + g(n)) |
| 3 | O(f(n)) · O(g(n)) = O(f(n) · g(n)) |
| 4 | 若 g(n) = O(f(n))，则 O(f(n)) + O(g(n)) = O(f(n)) |
| 5 | O(c·f(n)) = O(f(n))，c 为正常数 |
| 6 | f(n) = O(f(n)) |

**性质1 的证明思路**（考试可能考）：

设 p(n) = max{f(n), g(n)}，f'(n) = O(f(n))，g'(n) = O(g(n))，则：

当 n ≥ max{n₁, n₂} 时：
$$f'(n) + g'(n) \leq c_1 \cdot f(n) + c_2 \cdot g(n) \leq c_1 \cdot p(n) + c_2 \cdot p(n) = (c_1 + c_2) \cdot p(n)$$

取 c₃ = c₁ + c₂，n₃ = max{n₁, n₂}，证毕。

### 3.4 定义 2.2：大 Ω 记号（下界）

> **定义**：如果存在两个正常数 c 和 n₀，对于所有的 n ≥ n₀，有 |f(n)| ≥ c|g(n)|，则记作：**f(n) = Ω(g(n))**。

**含义**：
- f(n) 的阶**不低于** g(n) 的阶
- 在确定 f(n) 的下界时，总是试图求出**最大的** g(n)

**Ω 记号的性质**（与 O 性质对称）：

| 性质 | 公式 |
|------|------|
| 1 | Ω(f(n)) + Ω(g(n)) = **Ω(min{f(n), g(n)})** |
| 4 | 若 g(n) = Ω(f(n))，则 Ω(f(n)) + Ω(g(n)) = Ω(f(n)) |

> O 性质1 用 **max**，Ω 性质1 用 **min**，注意区分！

### 3.5 定义 2.3：大 Θ 记号（紧确界）

> **定义**：如果存在正常数 c₁, c₂ 和 n₀，对于所有的 n ≥ n₀，有 c₁|g(n)| ≤ |f(n)| ≤ c₂|g(n)|，则记作：**f(n) = Θ(g(n))**。

**含义**：一个算法的 f(n) = Θ(g(n)) 意味着该算法在最好和最坏情况下的计算时间，就一个常数因子范围内而言是**相同的**。

### 3.6 渐近表示函数的若干性质

**传递性：**
- f(n) = Θ(g(n)) 且 g(n) = Θ(h(n)) ⟹ f(n) = Θ(h(n))
- f(n) = O(g(n)) 且 g(n) = O(h(n)) ⟹ f(n) = O(h(n))
- f(n) = Ω(g(n)) 且 g(n) = Ω(h(n)) ⟹ f(n) = Ω(h(n))

**反身性：**
- f(n) = Θ(f(n))
- f(n) = O(f(n))
- f(n) = Ω(f(n))

### 3.7 定理 2.1（★重点证明★）

> **定理**：若 A(n) = aₘnᵐ + ··· + a₁n + a₀ 是一个 m 次多项式，则 **A(n) = O(nᵐ)**。

**证明**：取 n₀ = 1，当 n ≥ n₀ 时：

$$|A(n)| = |a_m n^m + \cdots + a_1 n + a_0| \leq |a_m|n^m + \cdots + |a_1|n + |a_0|$$

$$= \left(|a_m| + \frac{|a_{m-1}|}{n} + \cdots + \frac{|a_0|}{n^m}\right) n^m \leq \left(|a_m| + |a_{m-1}| + \cdots + |a_0|\right) n^m$$

取 $c = |a_m| + |a_{m-1}| + \cdots + |a_0|$，有 $|A(n)| \leq cn^m$，即 A(n) = O(nᵐ)。 $\square$

**推论**：若一个算法有数量级为 c₁nᵐ¹, c₂nᵐ², ..., cₖnᵐᵏ 的 k 个语句，则算法的计算时间为：

$$O(n^m), \quad m = \max\{m_i \mid 1 \leq i \leq k\}$$

### 3.8 数量级对算法有效性的影响

**两类算法：**

| 类型 | 计算时间 | 增长阶（从小到大）|
|------|---------|-----------------|
| **多项式时间算法** | 多项式函数限界 | O(1) < O(log n) < O(n) < O(n log n) < O(n²) < O(n³) |
| **指数时间算法** | 指数函数限界 | O(2ⁿ) < O(n!) < O(nⁿ) |

**不同时间复杂性对比（每微秒处理一次）：**

| T(n) | n=10 | n=40 | n=60 |
|------|------|------|------|
| log n | 3.3 μs | 5.3 μs | 5.9 μs |
| n | 10 μs | 40 μs | 60 μs |
| n log n | 33 μs | 213 μs | 354 μs |
| n² | 100 μs | 1600 μs | 3600 μs |
| n³ | 1 ms | 64 ms | 216 ms |
| 2ⁿ | 1 ms | 12.7 天 | 366 世纪 |
| n! | 3.6 s | 10³ 世纪 | 10⁶⁶ 世纪 |

> **对数函数的优越属性**：对数函数增长极慢，具有对数级操作次数的算法对任何实际规模的输入几乎都能在瞬间完成。

### 3.9 常用整数求和公式

$$\sum_{1 \leq i \leq n} 1 = \Theta(n)$$

$$\sum_{1 \leq i \leq n} i = \frac{n(n+1)}{2} = \Theta(n^2)$$

$$\sum_{1 \leq i \leq n} i^2 = \frac{n(n+1)(2n+1)}{6} = \Theta(n^3)$$

**通式**：$\displaystyle\sum_{1 \leq i \leq n} i^k = \frac{n^{k+1}}{k+1} + \frac{n^k}{2} + \text{低次项} = \Theta(n^{k+1})$

### 5.6 算法定义（过程）

**输入/输出：**
```

```

**函数（过程）格式：**
```
procedure NAME(<参数列表>)
  parameters  形式参数（及其数据类型）
  global      全局变量
  local       局部变量
  S           算法主体
  return (<表达式>)
end NAME
```

> 函数名通常用**大写字母**。

### 5.7 算法示例：MAX（求数组最大值）

```
procedure MAX(A, n, j)
  parameters real A(1:n)
  parameters integer n, j
  global real xmax
  local integer i
  xmax ← A(1)
  for i ← 2 to n do
    if A(i) > xmax then
      xmax ← A(i);  j ← i
    endif
  repeat
  return xmax
end MAX
```

**分析**：`if` 语句执行 n-1 次，时间复杂度为 **O(n)**。

### 5.8 算法的同质异相

同一个算法，变量的声明方式可以不同（使用 `parameters`/`global`/`local` 或简化写法），但描述的算法逻辑相同。这体现了**算法独立于具体程序设计语言**的特点。

## 六、作业题解析

### 题1：递推关系求解

**题目：**

$$T(n) = \begin{cases} g(n) & n \text{ 足够小} \\ 2T(n/2) + f(n) & \text{否则} \end{cases}$$

**情况① g(n) = O(1)，f(n) = O(n)：**

展开递推（设 n = 2ᵏ）：

$$T(n) = 2T(n/2) + cn$$
$$= 2[2T(n/4) + c\cdot\frac{n}{2}] + cn = 4T(n/4) + 2cn$$
$$= \cdots = 2^k T(1) + k \cdot cn$$

当 n = 2ᵏ 时，k = log₂n，T(1) = O(1)：

$$T(n) = O(1) \cdot n + cn\log n = O(n \log n)$$

> **结论①：T(n) = O(n log n)**（典型例子：归并排序）

**情况② g(n) = O(1)，f(n) = O(1)：**

$$T(n) = 2T(n/2) + c_1$$

展开（n = 2ᵏ）：

$$T(n) = 2^k T(1) + c_1(2^{k-1} + 2^{k-2} + \cdots + 1) = n \cdot O(1) + c_1(n-1) = O(n)$$

> **结论②：T(n) = O(n)**

### 题2：二分查找的递归过程

**思路**：将有序数组 A(1:n)，用递归描述"将搜索范围对半缩小"的策略。

```
procedure BINSEARCH(A, low, high, x)
  // 在 A(low:high) 中查找 x，返回下标；未找到返回 -1
  parameters integer A(1:n), low, high, x
  if low > high then return(-1) endif
  mid ← (low + high) / 2
  if x = A(mid) then
    return(mid)
  else if x < A(mid) then
    return BINSEARCH(A, low, mid-1, x)
  else
    return BINSEARCH(A, mid+1, high, x)
  endif
end BINSEARCH
```

**时间复杂度分析**：每次将搜索范围缩小一半，最多递归 ⌊log₂n⌋ + 1 次，故 T(n) = **O(log n)**。

### 题3："三分"查找算法

**思路**：先检查 n/3 处元素，再检查 2n/3 处元素，每次把搜索集合缩小到原来的 1/3。

```
procedure TRISEARCH(A, low, high, x)
  if low > high then return(-1) endif
  m1 ← low + (high - low) / 3
  m2 ← low + 2*(high - low) / 3
  if x = A(m1) then
    return(m1)
  else if x < A(m1) then
    return TRISEARCH(A, low, m1-1, x)
  else if x = A(m2) then
    return(m2)
  else if x < A(m2) then
    return TRISEARCH(A, m1+1, m2-1, x)
  else
    return TRISEARCH(A, m2+1, high, x)
  endif
end TRISEARCH
```

**计算复杂度分析：**

设 T(n) 为三分查找的时间复杂度。递推关系为：

$$T(n) = T(n/3) + c \quad (\text{每次做2次比较，为常数})$$

展开（设 n = 3ᵏ）：

$$T(n) = T(n/3) + c = T(n/9) + 2c = \cdots = T(1) + k \cdot c = O(\log_3 n) = O(\log n)$$

**三种情况：**

| 情况 | 发生条件 | 时间复杂度 |
|------|---------|-----------|
| **最好情况** | x = A(m1)（第一次比较即命中）| O(1) |
| **最坏情况** | x 不存在或在最后一段 | O(log n) |
| **平均情况** | 均匀分布 | O(log n) |

> **与二分查找的对比**：三分查找每次缩减到 1/3，但每步需要**2次**比较；二分查找每次缩减到 1/2，每步只需**1次**比较。
> - 三分查找：T(n) = O(log₃n) ≈ O(0.63 log₂n)
> - 二分查找：T(n) = O(log₂n)
>
> **三分查找并不比二分查找快**（在比较次数上甚至更多），因为每次虽然规模缩减更快，但比较次数增加了。

### 题4：自底向上的归并排序（取消对栈空间的利用）

**核心思路**：将递归的"自顶向下"改为迭代的"自底向上"，先合并大小为1的子数组，再合并大小为2的，依此类推。

```
procedure BOTTOMUP_MERGESORT(A, n)
  // 自底向上归并排序，不使用递归（无栈空间）
  parameters integer n; real A(1:n)
  local integer size, low, mid, high
  size ← 1
  while size < n do
    low ← 1
    while low + size - 1 < n do
      mid ← low + size - 1
      high ← min(low + 2*size - 1, n)
      call MERGE(A, low, mid, high)
      low ← low + 2*size
    repeat
    size ← size * 2
  repeat
end BOTTOMUP_MERGESORT
```

**时间复杂度**：O(n log n)，与递归归并排序相同。
**空间复杂度**：O(n)（仅需合并时的临时数组），消除了 O(log n) 的栈空间。

### 题5：二维极大点问题（分治策略）

**问题描述**：给定 n 个点，若点 (a, b) 满足不存在另一个点 (c, d) 使得 c ≥ a 且 d ≥ b（(c,d) ≠ (a,b)），则 (a, b) 是极大点。

**给定实例**：(1,2) (2,8) (3,5) (4,1) (5,4) (6,7) (7,6) (8,3)

**分治策略**：按 x 坐标排序后，将点集对半分为左半部和右半部，递归求解。

**求解过程**：

第一步：按 x 坐标升序排序（已排好）。

第二步：分治求解。

**递归分组：**

```
所有点（按x排序）：
  左半（x=1~4）：(1,2) (2,8) (3,5) (4,1)
  右半（x=5~8）：(5,4) (6,7) (7,6) (8,3)
```

**处理右半部**：
- 右子集 (5,4) (6,7) (7,6) (8,3)
- 极大点：(6,7)（y最大，其他点均被其在x方向支配或y较小），(7,6)，(8,3)
- 实际上右半极大点需要：没有右半其他点在两个维度上均不劣于它
  - (5,4)：(6,7) 的 x>5 且 y>4，被支配，不是极大点
  - (6,7)：无右半点支配它，是极大点 ✓
  - (7,6)：(6,7) 的 y>6，但 x<7；(8,3) 的 x>7 但 y<6；无点支配，是极大点 ✓
  - (8,3)：x 最大，一定是极大点 ✓
- 右半极大点集：{(6,7), (7,6), (8,3)}，右半 **y 最大值 = 7**

**处理左半部**：
- 左子集 (1,2) (2,8) (3,5) (4,1)
- 右半 y 最大值 = 7，因此左半点的 y 必须 > 7 才能是全局极大点
  - (1,2)：y=2 < 7，不是全局极大点
  - (2,8)：y=8 > 7，是全局极大点 ✓
  - (3,5)：y=5 < 7，不是全局极大点
  - (4,1)：y=1 < 7，不是全局极大点

**最终极大点结果**：

| 点 | 是否极大点 | 原因 |
|---|----------|------|
| (2,8) | ✅ | y=8 是全局最大，无点支配 |
| (6,7) | ✅ | 右半无点支配，y=7 |
| (7,6) | ✅ | 无点在两维均≥ |
| (8,3) | ✅ | x=8 是最大横坐标 |

**极大点集：{(2,8), (6,7), (7,6), (8,3)}**

### 题6：N 个硬币中找假币（分治算法）

**问题**：N 个硬币中有一枚假币，假币比真币轻，用天平（每次比较两组）找出假币。

**算法思想（分治——折半）**：

每次将硬币分为三组：
- 若 N 为偶数：左组 N/2，右组 N/2（每次用一次天平）；
- 若 N 为奇数：左组 ⌊N/2⌋，右组 ⌊N/2⌋，余下1枚单独放置。

```
procedure FIND_FAKE(coins, low, high)
  // coins[low..high] 中有一枚假币，假币较轻
  if low = high then return(low) endif    // 只剩一枚即为假币
  
  mid ← (low + high) / 2
  if (high - low + 1) 为偶数 then
    // 均分两组，各 (high-low+1)/2 枚
    if weight(coins[low..mid]) < weight(coins[mid+1..high]) then
      return FIND_FAKE(coins, low, mid)
    else
      return FIND_FAKE(coins, mid+1, high)
    endif
  else
    // 奇数枚：左组 mid-low+1 枚，右组同，最后一枚单独
    if weight(coins[low..mid]) < weight(coins[mid+1..high-1+1]) then
      return FIND_FAKE(coins, low, mid)
    else if weight(coins[low..mid]) > weight(coins[mid+1..high-1+1]) then
      return FIND_FAKE(coins, mid+1, high-1)
    else
      return(high)                        // 余下的那枚是假币
    endif
  endif
end FIND_FAKE
```

**时间复杂度分析**：

每次将问题规模减半，需要 1 次称量（天平比较）：

$$T(n) = T(n/2) + 1 \Rightarrow T(n) = O(\log_2 n)$$

**更优策略（三分）**：每次分为三组各 ⌊N/3⌋，两次称量确定假币在哪一组：

$$T(n) = T(n/3) + 2 \Rightarrow T(n) = O(\log_3 n)$$

两种策略比较次数：
- 二分：log₂n 次
- 三分：2·log₃n = 2·log₂n/log₂3 ≈ 1.26·log₂n 次

> **结论**：三分策略次数略多于二分，二分策略更优。

## 八、易错点汇总

1. **能行性反例**：`5/0` 违反确定性（除零无定义），`"6或7与x相加"` 违反确定性（有二义性）。注意分清哪条特性被违反。

2. **有穷性的"有限"**：不是理论上终止就够，必须要在**现代计算机上有效**才算满足有穷性。

3. **O和Ω的方向混淆**：O是上界（f不超过cg），Ω是下界（f不低于cg）；O性质1取max，Ω性质1取min。

4. **定理2.1的证明方向**：是证明"≤"（即是O），不是"≥"（Ω）；关键是不等式 1/nⁱ ≤ 1 当 n ≥ 1 时成立。

5. **频率计数 vs 执行时间**：频率计数与机器无关；执行时间与机器、语言、编译程序有关。

6. **三分查找并不优于二分查找**：虽然每次规模减少到1/3，但每步要2次比较，总比较次数约为二分的1.26倍。

7. **SPARKS 中 loop...until 的终止条件**：`until 条件 repeat` 表示**条件为真时终止**（与while相反）。

8. **回溯定理2.1的证明技巧**：取n₀=1是因为此时 1/n ≤ 1 成立，可以直接放缩；c取所有系数绝对值之和。
