## 二、课件补充·无对应原题

### 补充A：8255A C口状态字格式 ⚠️考点

C口在方式1/方式2下会被部分占用作联络信号，剩余位可作普通I/O。课件给出的C口引脚分配规律：

**方式1输入时（A口为例）：**

```
PC7-PC6 → I/O（普通使用）
PC5     → IBF  （输入缓冲器满，8255A输出）
PC4     → STB  （外设选通输入，低有效）
PC3     → INTRA（中断请求）
```

**方式1输出时（A口为例）：**

```
PC7     → OBF  （输出缓冲器满，低有效）
PC6     → ACK  （外设响应，低有效）
PC5-PC4 → I/O
PC3     → INTRA（中断请求）
```

记忆口诀：**输入时 STB→IBF（外进内满）；输出时 OBF→ACK（先满后应）**。

---

### 补充B：串行通信传送方向 ⚠️考点

三种方向分类：
- **单工**：仅一个方向传输（A→B），不可反向
- **半双工**：双向但不同时（A→B 或 B→A，交替）
- **全双工**：双向同时传输（需两条独立信道）

8251A 支持全双工（TXD 发送线、RXD 接收线各自独立）。

### 模块1：8255A 基础——端口选址与控制字写法

理解8255A的前提是先搞清楚"怎么选口"和"怎么告诉它每个口干什么"。这两件事分别由 **A1/A0地址线** 和 **方式选择控制字** 完成。

#### 1.1 端口选择规则 ⚠️考点

| A1 | A0 | 选中端口 |
|----|----|----|
| 0 | 0 | A口（数据口）|
| 0 | 1 | B口（数据口）|
| 1 | 0 | C口（数据口）|
| 1 | 1 | 控制口 |

规律：**地址偏移0/1/2/3 → A口/B口/C口/控制口**。已知芯片基地址（如60H），则控制口 = 基地址+3 = 63H。

#### 1.2 方式选择控制字格式 ⚠️🔥考点

```
D7  D6  D5  D4  D3  D2  D1  D0
 1  [A组方式]  A口IO  C高IO  [B组方式]  B口IO  C低IO
     D6D5:           D4:         D2:          D1:       D0:
     00=方式0        0=输出      0=输出    0=方式0    0=输出
     01=方式1        1=输入      1=输入    1=方式1    1=输入
     1X=方式2
```

**记忆关键：D7=1 是方式选择控制字的特征位；D7=0 是C口置位/复位控制字的特征位。** 这是区分两种控制字的唯一方法。

**方式0控制字快速推导模板：**
全部方式0，全部输出 → `1 00 0 0 0 00 0 0` = `10000000B` = `80H`

#### Q1 ⚠️🔥 8255A方式0应用：产生波形

> **原题（课件例7.1.1）：** 利用8255A在方式0下工作，使其在PC0、PC3引脚产生如图所示波形（PC3周期2ms，PC0周期4ms，相位关系见课件图）。设8255A各端口地址分别为60H、61H、62H和63H，波形延时可调用延时1ms子程序（D1ms）实现，试编写相应程序段。

**知识铺垫：** C口在方式0下整体作普通输出口使用。要同时控制PC0和PC3，每次向62H（C口地址）写入一个完整字节，其中PC3对应D3位，PC0对应D0位，其他位写0即可。

根据波形分析，4个时间片各1ms，C口输出值依次为：

| 时刻 | PC3 | PC0 | C口值（D7~D0）|
|-----|-----|-----|-----|
| 0ms | 0 | 1 | 00000001B = **01H** |
| 1ms | 1 | 1 | 00001001B = **09H** |
| 2ms | 0 | 0 | 00000000B = **00H** |
| 3ms | 1 | 0 | 00001000B = **08H** |

**✅ 参考答案要点：**

```asm
START: MOV AL, 80H        ; 方式选择控制字：方式0，全部输出，D7=1
       OUT 63H, AL        ; 写入控制口

A1:    MOV AL, 01H        ; PC3=0, PC0=1
       OUT 62H, AL
       CALL D1ms

       MOV AL, 09H        ; PC3=1, PC0=1
       OUT 62H, AL
       CALL D1ms

       MOV AL, 00H        ; PC3=0, PC0=0
       OUT 62H, AL
       CALL D1ms

       MOV AL, 08H        ; PC3=1, PC0=0
       OUT 62H, AL
       CALL D1ms

       JMP A1             ; 循环产生连续波形
```

### 模块2：8255A 方式1——握手信号配合逻辑

方式0是"无条件"直接读写；方式1引入了**握手联络信号**，允许外设告诉CPU"我准备好了"，是带同步机制的选通方式。

#### 2.1 方式1输入关键信号 📝画图题

```
外设 ──STB↓──→ 8255A ──IBF↑──→ 外设（告知"数据已锁存"）
                  ↓
              （STB、IBF、INTE三者都高时）
                  ↓
              INTR↑ ──→ CPU（中断请求）
```

- **STB**（低有效）：外设给8255A的选通脉冲，低电平时把数据锁入A口或B口锁存器
- **IBF**（高有效）：8255A输出给外设，表示"缓冲器已满，请等待"
- **INTR**（高有效）：当 STB=1（回高）、IBF=1、INTE=1 时，向CPU发中断请求
- **INTE**：A口由 PC4 置/复位控制；B口由 PC2 置/复位控制

#### 2.2 方式1输出关键信号 📝画图题

```
CPU 写数据 ──→ 8255A ──OBF↓──→ 外设（"数据已准备好，请取走"）
                         ↓
             外设取走数据后 ──ACK↓──→ 8255A
                         ↓
                     INTR↑ ──→ CPU（"可以再输出下一个"）
```

- **OBF**（低有效）：8255A输出，表示CPU已将数据写入端口，通知外设取走
- **ACK**（低有效）：外设确认已取走数据
- **INTR**（高有效）：外设取走数据后，向CPU再次请求
- **INTE**：A口由 PC6 置/复位控制；B口由 PC2 置/复位控制

#### Q2 📝 画出8255A方式1输入/输出的时序波形并说明工作过程

> **原题：** 画出8255A工作于方式1输入的波形，并说明其工作过程。画出8255A工作于方式1输出的波形，并说明其工作过程。

**✅ 参考答案要点（方式1输入过程）：**

1. 外设将数据送到PA口引脚，随后拉低 STB
2. STB 下降沿将数据锁入A口输入缓冲器，同时 IBF 变高，通知外设"请别再发"
3. STB 回高后，若 INTE=1，则 INTR 变高，向CPU发中断
4. CPU响应中断，执行 `IN AL, 60H` 读取数据，INTR自动清零，IBF清零

**✅ 参考答案要点（方式1输出过程）：**

1. CPU执行 `OUT 60H, AL`，数据写入A口输出锁存器，同时 OBF 变低，通知外设
2. 外设读取数据后，拉低 ACK
3. ACK 下降沿清除 OBF（变高），并在 ACK 回高后、INTE=1 时，INTR 变高
4. CPU响应中断，再次写入下一个数据

### 模块3：8255A C口置位/复位——产生脉冲

C口除了可以整体输出字节，还可以**逐位**置0或置1，这靠的是另一种控制字（D7=0的BSR控制字）。

#### C口置位/复位控制字格式 ⚠️考点

```
D7  D6-D4  D3-D1  D0
 0   不用   位号   1=置位/0=复位

位号编码：000=PC0, 001=PC1, ..., 111=PC7
```

举例：将PC5置1 → D3D2D1=101, D0=1 → `00001011B` = `0BH`
将PC5清0 → D3D2D1=101, D0=0 → `00001010B` = `0AH`

#### Q3 🔥 编程使PC5输出一个负脉冲

> **原题：** 并行接口芯片8255A的A口-控制口的端口地址依次为60H-63H。编一段程序使从PC5输出一个负脉冲。另外，若脉冲宽度不够，应如何解决？

**知识铺垫：** 负脉冲 = 先高 → 变低 → 回高。平时PC5=1，拉低一段时间后恢复。使用C口BSR控制字（D7=0），写入控制口63H。

PC5的位号=5，二进制101：
- 置1（PC5=1）：`0_xxx_101_1` → `00001011B` = `0BH`
- 复位（PC5=0）：`0_xxx_101_0` → `00001010B` = `0AH`

**✅ 参考答案要点：**

```asm
; 先初始化8255A（C口输出）
MOV AL, 80H     ; 方式0，全部输出
OUT 63H, AL

; 产生负脉冲
MOV AL, 0BH     ; PC5 置1（高电平）
OUT 63H, AL

MOV AL, 0AH     ; PC5 复位→0（低电平，脉冲开始）
OUT 63H, AL

; （若需增加脉冲宽度，在此处调用延时子程序 CALL Dxms）

MOV AL, 0BH     ; PC5 置1（脉冲结束，恢复高电平）
OUT 63H, AL
```

❌**易错：** 置位/复位控制字写入的端口是**控制口（63H）**，而不是C口数据口（62H）。两种控制字都写控制口，但D7=1是方式选择，D7=0是BSR置位复位，8255A内部靠D7区分。

### 模块4：8253 基础——控制字与计数初值计算

8253是计数器/定时器，核心操作就两步：**写控制字**（告诉它用哪个通道、哪种方式、读写格式）→ **写计数初值**。

#### 4.1 8253控制字格式 ⚠️🔥考点

```
D7D6   D5D4    D3D2D1    D0
SC1SC0  RL1RL0  M2M1M0    BCD
计数器  读写格式  工作方式   计数方式
选择    00=锁存  000=方式0  0=二进制
00=C0  01=只低8  001=方式1  1=BCD
01=C1  10=只高8  X10=方式2
10=C2  11=先低后高 X11=方式3
11=非法         100=方式4
                101=方式5
```

**端口地址对应（以基址40H为例）：**
- 40H = 计数器0数据口
- 41H = 计数器1数据口
- 42H = 计数器2数据口
- 43H = 控制口（唯一一个控制口管三个通道）

#### 4.2 计数初值计算公式 ⚠️🔥考点

**定时方式：**
$$N = \frac{OUT时间}{CLK周期} = OUT时间 \times CLK频率$$

**计数方式：** N = 要计数的次数

❌**易错：** 写入N=0时，实际计数值为最大值（二进制=65536，BCD=10000），因为计数器是先减1再判断，0减1后溢出到65535，继续减到0才触发。

#### Q4 ✅真题🔥 8253综合应用：产生20ms周期信号

> **原题（真题计算题2）：** 若要用8253A产生20ms周期的信号，作为8259A中断请求输入，其中8253A的时钟CLK频率为1MHz，问8253A计数器需要工作在哪两个工作方式之一？并计算出8253A的计数初值是多少？

**知识铺垫：** 要产生周期性重复信号，需要工作方式**重复有效**——只有方式2（脉冲频率发生器）和方式3（方波发生器）满足。

计数初值：
$$N = 20\text{ms} \times 1\text{MHz} = 20 \times 10^{-3} \times 10^6 = 20000$$

20000 = 0x4E20，需要先写低8字节（20H），再写高字节（4EH）。

**✅ 参考答案要点：**

工作方式：**方式2（脉冲频率发生器）或方式3（方波发生器）**，两者都重复有效。

计数初值：**N = 20000**（二进制计数，两字节写入：先低字节20H，再高字节4EH）

示例初始化程序（以计数器0，端口40H-43H，方式2为例）：

```asm
MOV AL, 00110100B  ; 00=C0, 11=先低后高, 010=方式2, 0=二进制
OUT 43H, AL        ; 写控制口

MOV AX, 20000      ; 计数初值
OUT 40H, AL        ; 写低8位（20H）
MOV AL, AH
OUT 40H, AL        ; 写高8位（4EH）
```

#### Q5 🔥 8253三通道综合初始化编程

> **原题：** 某系统中有一片8253，其计数器0至控制口地址依次为40H-43H，请按如下要求编程：
> （1）通道0：方式3，CLK0=2MHz，要求在OUT0输出1KHz方波。
> （2）通道1：方式2，CLK1=1MHz，要求OUT1输出1KHz脉冲波。
> （3）通道2：方式4，CLK2=OUT1，计数值为1000，计数到0时输出一个控制脉冲。

**计数初值计算：**

| 通道 | 方式 | CLK | 目标 | N |
|-----|-----|-----|-----|---|
| 0 | 3（方波）| 2MHz | 1KHz | 2MHz/1KHz = **2000** |
| 1 | 2（脉冲）| 1MHz | 1KHz | 1MHz/1KHz = **1000** |
| 2 | 4（软件触发选通）| OUT1 | 计数1000 | **1000** |

**控制字推导：**
- 通道0方式3二进制：`00 11 011 0` = `36H`
- 通道1方式2二进制：`01 11 010 0` = `74H`
- 通道2方式4二进制：`10 11 100 0` = `B8H`

**✅ 参考答案要点：**

```asm
; 通道0：方式3，N=2000
MOV AL, 36H        ; 00=C0, 11=先低后高, 011=方式3, 0=二进制
OUT 43H, AL
MOV AX, 2000
OUT 40H, AL        ; 低8位 D0H
MOV AL, AH
OUT 40H, AL        ; 高8位 07H

; 通道1：方式2，N=1000
MOV AL, 74H        ; 01=C1, 11=先低后高, 010=方式2, 0=二进制
OUT 43H, AL
MOV AX, 1000
OUT 41H, AL        ; 低8位 E8H
MOV AL, AH
OUT 41H, AL        ; 高8位 03H

; 通道2：方式4，N=1000
MOV AL, 0B8H       ; 10=C2, 11=先低后高, 100=方式4, 0=二进制
OUT 43H, AL
MOV AX, 1000
OUT 42H, AL        ; 低8位
MOV AL, AH
OUT 42H, AL        ; 高8位
```

### 模块5：8253六种工作方式对比

六种方式的核心差异可以从三个维度归纳：**启动方式、计数值有效期、OUT输出波形**。

| 方式 | 名称 | 启动 | 计数值有效期 | OUT波形特征 |
|-----|-----|-----|-----|-----|
| 0 | 计数结束中断 | 软 | 一次有效 | 计数期间低，计到0后变高 |
| 1 | 可编程单稳 | **硬**（GATE↑）| 多次有效 | GATE触发后，低电平持续N个CLK |
| 2 | 脉冲频率发生器 | 软/硬 | **重复有效** | 连续输出，每N个CLK出一个低脉冲（1个CLK宽）|
| 3 | 方波发生器 | 软/硬 | **重复有效** | 连续方波，高N/2个CLK，低N/2个CLK |
| 4 | 软件触发选通 | 软 | 一次有效 | 计到0时出一个低脉冲（1个CLK宽）|
| 5 | 硬件触发选通 | **硬**（GATE↑）| 多次有效 | 同方式4，但由GATE触发，可重复触发 |

**GATE的作用（通用规则）：**
- 软件启动的方式（0/2/3/4）：GATE=1 允许计数，GATE=0 暂停计数
- 硬件启动的方式（1/5）：GATE上升沿触发，GATE=0 不影响已运行的计数

#### Q6 🔥 方式1 vs 方式2对比

> **原题：** 从方式名称、启动方法、计数值有效期限角度，说明8253的方式1与方式2的工作特点。

**✅ 参考答案要点：**

- **方式1**（可编程单稳）：硬件启动（GATE上升沿触发），计数值多次有效（每次GATE触发可再次单稳）。计数期间OUT保持低电平，计到0时OUT变高。
- **方式2**（脉冲频率发生器）：软件/硬件均可启动，计数值重复有效（自动重装计数值循环计数）。每计完一轮，在计到1时OUT低一个CLK宽度，然后自动重新从N开始，连续产生等间隔脉冲。

---

#### Q7 🔥 方式2 vs 方式3对比

> **原题：** 说明8253的方式2与方式3的工作特点。

**✅ 参考答案要点：**

两者相同点：都是软件/硬件可启动，计数值重复有效，能产生连续周期波形。

差异：
- **方式2**：OUT波形不对称，高电平持续(N-1)个CLK，低电平仅1个CLK（窄脉冲）
- **方式3**：OUT波形对称（方波），高低各占约N/2个CLK（N为奇数时高多一个CLK）

---

#### Q8（延伸）方式1 vs 方式5对比

> **原题：** 说明8253的方式1与方式5的工作特点。

**✅ 参考答案要点：**

两者都是**硬件启动**（GATE上升沿），计数值**多次有效**（可被GATE再次触发重新计数），区别在于输出波形：
- **方式1**：触发后OUT立即变低，持续N个CLK后变高（单稳，宽脉冲低电平）
- **方式5**：触发后OUT不立即变化，计数到0时才出一个低脉冲（1个CLK宽），类似方式4的硬件版

### 模块6：8251A 串行通信——帧格式与计算

8251A是可编程串行通信芯片，支持同步和异步两种方式。考试重点是异步方式的帧格式和传输速率计算。

#### 6.1 异步通信帧格式 ⚠️🔥考点

```
空闲位  起始位  数据位(5~8位)  校验位(可选)  停止位(1/1.5/2)  空闲位  起始位...
 高     低=0   D0...D7(低先) 奇/偶/无       高              高      低
```

**每字符总位数 = 1（起始）+ 数据位数 + 校验位数（0或1）+ 停止位数**

**每秒传送字节数：**
$$字节数/秒 = \frac{波特率}{每字符总位数}$$

#### 6.2 波特率因子 ⚠️考点

```
收发时钟频率 = 波特率 × 波特率因子（1、16、64之一）
```

接收时钟常用16倍频：在起始位开始时中间采样（等待8个时钟后采样），抗干扰能力强。

#### Q8 ✅真题🔥 8251A传输速率计算与字符帧格式

> **原题（真题计算题3）：** 8251A工作于异步通信方式，通信格式为9600、8、N、1，即波特率为9600、8个数据位、无奇偶校验位和一个停止位。计算每秒可以传送多少字节？并画出传送字符'A'的帧格式。要求写出计算过程。

**知识铺垫：** 格式"9600、8、N、1"解读：
- 波特率 = 9600 Baud
- 数据位 = 8位
- 校验 = N（无，0位）
- 停止位 = 1位

每字符总位数 = 1 + 8 + 0 + 1 = **10位**

**✅ 参考答案要点：**

$$每秒字节数 = \frac{9600}{10} = 960 \text{ 字节/秒}$$

字符'A'的ASCII码 = 41H = `0100 0001B`，低位先发（D0先）：

```
帧格式（时间从左到右）：

| 空闲 | 起始位 | D0 | D1 | D2 | D3 | D4 | D5 | D6 | D7 | 停止位 | 空闲 |
|  1  |   0   |  1 |  0 |  0 |  0 |  0 |  0 |  1 |  0 |   1   |  1  |
                ↑                                           ↑
              低电平起始                               高电平停止
```

注：'A'=0100 0001B，低位D0=1先发，依次为：1,0,0,0,0,0,1,0

#### Q9 🔥 8251A初始化编程

> **原题：** 设8251A工作在异步方式，波特率因子为16，每个字符7个数据位，采用偶校验，2个停止位，允许发送和接收，设8251A数据端口地址为200H，8251A控制端口地址为201H。试编写8251A的初始化程序。

**方式选择控制字解析（D7~D0）：**

```
D7D6  D5D4  D3  D2D1  D0D1
停止  字长  EP  PEN   通信方式
 11    11   1   1     10
(2位)(7+1)偶  使用  ×16异步
```

按课件控制字格式：
- D1D0 = 10 → 异步×16
- D3D2 = 11 → 字符长度8位（注：课件里"11=字符长度8位"，但题目说7位，D3D2=10）

重新推导（7位字符）：
- D1D0 = 10（×16异步）
- D3D2 = 10（7位字符）
- D4 = 1（使用奇偶校验）
- D5 = 1（偶校验）
- D7D6 = 11（2位停止位）

方式控制字 = `1 1 1 1 1 0 1 0` = `0FAH`

命令控制字（允许发送TXEN=1，允许接收RXE=1，清出错标志ER=1）= `00110111B` = `37H`

**✅ 参考答案要点：**

```asm
MOV AL, 0FAH        ; 方式选择控制字：异步×16，7位/字符，偶校验，2停止位
MOV DX, 201H        ; 控制端口地址
OUT DX, AL

MOV AL, 37H         ; 命令控制字：TXEN=1，RXE=1，ER=1（清出错标志）
OUT DX, AL          ; 命令控制字紧跟方式控制字后写入同一控制端口
```

❌**易错：** 方式控制字和命令控制字都写入**控制端口（201H）**，数据端口（200H）只用于收发实际数据。8251A靠接收顺序区分两种控制字：复位后第一个写入的是方式控制字，之后写入的是命令控制字。

### 模块7：8253计数初值为0的特殊含义 ❌易错勘误

> **原题：** 为什么对8253A写入计数值0是最大的计数值？在二进制计数方式下计数值相当于多少？如果8253A的时钟CLK频率为1.193MHz，工作在周期工作方式时，计算最大可以产生每秒多少次的周期信号（保留一位小数）？

**✅ 参考答案要点：**

8253计数器是**减1计数**（每个CLK脉冲到来时减1）。写入0时，第一个CLK到来减1后变成65535（0FFFFh），之后继续从65535减到0，整整经历65536个CLK周期，所以0代表最大计数值65536。

二进制最大计数值 = **65536**

最大周期信号频率：
$$f = \frac{1.193\text{MHz}}{65536} \approx 18.2 \text{ Hz}$$
