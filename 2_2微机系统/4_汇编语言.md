## 2. 核心知识点精讲

### 2.1 有效地址（EA）公式与存储器寻址方式

**核心定义**：有效地址EA是操作数在内存中的**段内偏移地址**。

**原理解析**：

$$EA = 基址 + (变址 \times 比例因子) + 位移量$$

| 成分 | 16位寻址 | 32位寻址 |
|------|----------|----------|
| 位移量 | 0、8、16位 | 0、8、32位 |
| 基址寄存器 | BX、BP | 任何32位通用寄存器 |
| 变址寄存器 | SI、DI | 除ESP外的任何32位通用寄存器 |
| 比例因子 | 无 | 1、2、4、8 |

**七种存储器寻址方式（16位，考试重点）**：

```
1. 直接寻址         MOV AX, [2000H]        EA = 2000H
2. 寄存器间接寻址   MOV AX, [BX]           EA = BX
3. 寄存器相对寻址   MOV AX, [BX+4]         EA = BX+4
4. 基址变址寻址     MOV AX, [BX+SI]        EA = BX+SI
5. 相对基址变址寻址 MOV AX, 24[BX][SI]     EA = BX+SI+24
6. (32位)比例变址  MOV AX, [ESI*4]
7. (32位)基址比例变址 MOV AX, [EBX+ESI*2]
```

**段默认规则**：
- 使用 BP/SP（EBP/ESP）寻址 → 默认 **SS段**
- 其他通用寄存器参与寻址 → 默认 **DS段**
- 取指令 → 默认 **CS段**
- 串操作目的地址 → 默认 **ES段**

⚠️ 考点：选择题/简答题 | ⭐⭐⭐
❌ 易错：16位寻址中，基址只能用BX或BP，变址只能用SI或DI，不能用CX、AX等。`[AX]`是非法寻址。

---

### 2.2 数据传送指令

**MOV 规则（必背限制）**：

```
✅ 合法：
    MOV AX, BX           ; 寄存器→寄存器
    MOV AX, [BX]         ; 存储器→寄存器
    MOV [BX], AX         ; 寄存器→存储器
    MOV AX, 1234H        ; 立即数→寄存器

❌ 非法：
    MOV [BX], [SI]       ; 两个操作数不能同时为存储器
    MOV CS, AX           ; CS不能作为目的操作数
    MOV DS, 1000H        ; 段寄存器不能直接装入立即数
    MOV AH, BX           ; 操作数长度不匹配（8位 vs 16位）
```

**PUSH/POP 操作（16位）**：

```
PUSH SRC：  SP←SP-2，[SP+1:SP]←SRC（先减SP再存数）
POP  DST：  DST←[SP+1:SP]，SP←SP+2（先取数再增SP）
```

❌ 易错：PUSH/POP 操作数必须是16位或32位，不能是8位寄存器（如AL）。

**LEA vs MOV区别**：

```asm
MEM DW 1234H          ; 假设MEM的偏移地址是3100H，内容是1234H
MOV AX, MEM           ; AX = 1234H（取内容）
LEA AX, MEM           ; AX = 3100H（取地址）
```

**IN/OUT指令**：

```asm
; 端口地址 ≤ 255，直接寻址
IN  AL, 60H           ; 从60H端口读一字节到AL
OUT 60H, AL           ; 将AL写到60H端口

; 端口地址 > 255，间接寻址（地址放DX）
MOV DX, 0300H
IN  AL, DX
OUT DX, AL
```

⚠️ 考点：选择题/编程题 | ⭐⭐⭐

### 2.3 算术运算指令与标志位

**标志位一览**：

| 标志 | 名称 | 置1条件 |
|------|------|---------|
| CF | 进位标志 | 最高位有进位/借位 |
| ZF | 零标志 | 结果为0 |
| SF | 符号标志 | 结果最高位为1（负数） |
| OF | 溢出标志 | 有符号数溢出 |
| PF | 奇偶标志 | 结果低8位中1的个数为偶数 |
| AF | 辅助进位 | 低4位向高4位有进位 |

**各指令对标志位的影响**：

```
ADD/SUB：影响 CF、PF、AF、ZF、SF、OF（全影响）
INC/DEC：影响 AF、OF、PF、SF、ZF，【不影响CF】 ← 常考易错点
CMP：    同SUB，影响所有标志，但结果不保存
AND/OR/XOR/TEST：CF=0，OF=0，影响SF、ZF、PF，AF无定义
MUL：    若高半部分不为0，则CF=OF=1，否则CF=OF=0
```

**CMP 后跳转条件判断**：

```
无符号数比较 CMP A, B：
  A > B → JA (CF=0且ZF=0)
  A < B → JB (CF=1)
  A = B → JE (ZF=1)

有符号数比较 CMP A, B：
  A > B → JG (SF=OF且ZF=0)
  A < B → JL (SF≠OF)
  A = B → JE (ZF=1)
```

⚠️ 考点：计算题 | ⭐⭐⭐ 🔥 高频

### 2.4 逻辑运算与移位指令

**AND 常用技巧**：
- 清零某位：`AND AL, 0FH`（清高4位）
- 判断某位：`TEST AL, 01H`（测试最低位，不改变AL）

**OR 常用技巧**：
- 置1某位：`OR AL, 80H`（将最高位置1）

**XOR 常用技巧**：
- 清零寄存器：`XOR AX, AX`（比MOV AX,0更快）

**移位指令图示**：

```
SHL/SAL（逻辑/算术左移）：
  CF ← [D7...D0] ← 0     （低位补0，最高位移入CF）

SHR（逻辑右移）：
  0 → [D7...D0] → CF     （高位补0，最低位移入CF）

SAR（算术右移）：
  [D7] → [D7...D0] → CF  （高位补符号位，保持符号）

ROL（循环左移）：CF←最高位，最高位→最低位
ROR（循环右移）：CF←最低位，最低位→最高位
RCL（带进位循环左移）：CF参与循环
RCR（带进位循环右移）：CF参与循环
```

❌ 易错：8086中移位次数 > 1时，必须先 `MOV CL, 次数`，再 `SHL AX, CL`。不能直接写 `SHL AX, 3`（Pentium以后才支持立即数）。

### 2.5 控制转移指令

**无条件跳转**：

```asm
JMP SHORT 标号     ; 短跳转，8位偏移，范围-128~+127
JMP NEAR PTR 标号  ; 近跳转，16位偏移，段内
JMP FAR PTR 标号   ; 远跳转，段间，修改CS:IP
JMP BX             ; 间接跳转，IP←BX
```

**条件跳转（重点）**：

```asm
JZ/JE   标号  ; ZF=1，结果为零/相等
JNZ/JNE 标号  ; ZF=0，结果不为零/不相等
JS      标号  ; SF=1，结果为负
JNS     标号  ; SF=0，结果为正
JC      标号  ; CF=1，有进位/借位
JNC     标号  ; CF=0，无进位/借位（冒泡中用于判断大小）
JA      标号  ; CF=0且ZF=0，无符号A>B
JB/JNAE 标号  ; CF=1，无符号A<B
JG      标号  ; 有符号A>B
JL      标号  ; 有符号A<B
```

**LOOP指令**：

```asm
LOOP 标号
; 等价于：
; DEC CX
; JNZ 标号
; 注意：CX=0时不执行循环（先减后判）
```

❌ 易错：条件跳转只能使用**段内相对寻址**（8位偏移），跳转范围限制在-128~+127字节内。

### 2.6 子程序调用与返回

**CALL/RET机制**：

```asm
; 段内近调用（16位）
CALL 子程序名：
    SP←SP-2
    [SP]←IP          ; 保存返回地址
    IP←子程序偏移地址

RET：
    IP←[SP]
    SP←SP+2
```

**子程序标准框架**：

```asm
SUB1 PROC NEAR
    PUSH AX           ; 保护现场（保存用到的寄存器）
    PUSH BX
    PUSH CX
    ; ===== 子程序功能代码 =====
    ; ...
    ; ===== 功能结束 =====
    POP CX            ; 恢复现场（顺序与PUSH相反！）
    POP BX
    POP AX
    RET               ; 返回主程序
SUB1 ENDP
```

⚠️ 考点：编程题/简答题 | ⭐⭐⭐
❌ 易错：POP恢复现场的顺序必须与PUSH保护现场**完全相反**（堆栈LIFO特性）。

### 2.7 中断指令

**INT n 执行步骤（需背诵）**：

```
① SP←SP-2，PUSH FLAGS（标志寄存器压栈）
② TF←0，IF←0（禁止单步/禁止中断）
③ SP←SP-2，PUSH CS（断点段地址压栈）
④ SP←SP-2，PUSH IP（断点偏移地址压栈）
⑤ IP←[n×4]，CS←[n×4+2]（从中断向量表取入口地址）
```

**IRET 执行步骤**：

```
① POP IP（弹出断点偏移地址）
② POP CS（弹出断点段地址）
③ POP FLAGS（弹出标志寄存器）
```

**INT 21H常用功能**：

```asm
; 功能1：等待键盘输入
MOV AH, 1
INT 21H               ; 输入字符的ASCII码在AL中

; 功能2：输出字符到屏幕
MOV DL, 41H           ; 'A'的ASCII码
MOV AH, 2
INT 21H

; 功能4CH：终止程序
MOV AH, 4CH
INT 21H
```

### 2.8 伪指令

**段定义结构（程序框架）**：

```asm
DATA SEGMENT                    ; 数据段开始
    变量名 DB/DW/DD 初值        ; 定义数据
DATA ENDS                       ; 数据段结束

STACK SEGMENT STACK 'STACK'     ; 堆栈段
    DW 100H DUP(?)
STACK ENDS

CODE SEGMENT                    ; 代码段开始
    ASSUME CS:CODE, DS:DATA     ; 段寄存器分配
START PROC FAR
    MOV AX, DATA
    MOV DS, AX                  ; 初始化DS（注意：CS由系统初始化，DS需手动）
    ; ... 程序主体 ...
    MOV AH, 4CH
    INT 21H
START ENDP
CODE ENDS
END START                       ; 程序结束，指定入口
```

**数据定义伪指令**：

```asm
DB：字节（8位）
DW：字（16位），低字节在低地址（小端存储）
DD：双字（32位）

; 示例
A1 DB 12H, 34H           ; 占2字节
A2 DW 1234H              ; 低字节12H在低地址
A3 DD 12345678H          ; 4字节，78H在最低地址
A4 DB 5 DUP(0)           ; 5个0，占5字节
A5 DW 10 DUP(?)          ; 10个未初始化的字，占20字节
```

**常用运算符**：

```asm
OFFSET 变量名   ; 返回变量的段内偏移地址
SEG    变量名   ; 返回变量所在段的段地址
TYPE   变量名   ; DB返回1，DW返回2，DD返回4
PTR    类型声明 ; 临时改变操作数类型属性
$ 当前地址计数器

; 例：
N EQU $-ARRAY  ; N = 数组ARRAY的字节数
LEA BX, ARRAY  ; BX = ARRAY的偏移地址
MOV AX, OFFSET ARRAY  ; 同上效果
```

❌ 易错：`END`（程序汇编结束）、`ENDS`（段定义结束）、`ENDP`（过程定义结束）含义不同！

## 3. 高频计算题型与模板

### 3.1 标志位计算（执行指令后的FLAGS状态）

**公式**：（无统一公式，按位运算规则计算）

**解题步骤**：
1. 将操作数转换为二进制（或直接十六进制计算）
2. 执行运算，观察结果
3. 根据结果判断各标志位

**CF判断**：无符号数加法最高位有进位→CF=1；减法有借位→CF=1

**OF判断**：两个正数相加结果为负，或两个负数相加结果为正→OF=1

$$OF = C_n \oplus C_{n-1}$$（最高位进位 XOR 次高位进位）

**示例**（练习题第27题）：

```asm
MOV AX, 35E5H
MOV BX, 7832H
ADD AX, BX
```

Step 1：35E5H + 7832H = AE17H

```
  0011 0101 1110 0101   (35E5H)
+ 0111 1000 0011 0010   (7832H)
= 1010 1110 0001 0111   (AE17H)
```

Step 2：判断标志位

- CF=0（最高位加法无进位，16位范围内）
- ZF=0（结果AE17H ≠ 0）
- SF=1（最高位为1，结果为负数补码表示）
- OF=1（正数+正数=负数，发生溢出）
- PF=1（AE17H低8位=17H=00010111B，含三个1，奇数→PF=0？）

> 重新验证：17H = 0001 0111B，1的个数=4（偶数）→ PF=1 ✓

**答**：CF=0，ZF=0，SF=1，OF=1，PF=1

### 3.2 NOT/ADD组合计算（练习题第28题）

**示例**：

```asm
MOV AX, BX       ; AX = BX（设BX = X）
NOT AX            ; AX = ~X（按位取反）
ADD AX, BX        ; AX = ~X + X
INC AX            ; AX = ~X + X + 1
```

**分析**：对任意X，`~X + X = 0FFFFH`（所有位取反后相加，每位都得1）

所以 `NOT AX + BX = FFFFH`，再 `INC AX` → AX = 0000H，产生进位CF=0（循环溢出到0），CF=0。

**答**：AX=0，CF=0

❌ 易错：FFFFH + 1 = 10000H，16位结果为0000H，**最高位有进位**故CF=1，不是CF=0。

> 修正答案：AX=0，CF=1（注意INC不影响CF，但此处是ADD等价）实际练习题答案为CF=0，因为题目用INC，INC不影响CF。

## 4. 练习题精析

### 题目23（指令错误判断）

**题目**：指出下列指令的错误：
(1) MOV AH, BX  (2) ADD 15H, BX  (3) MOV CS, AX  (4) MOV AX, [SI][DI]  (5) MOV BYTE PTR [BX], 1000

**解答**：

(1) **MOV AH, BX**：错误。AH是8位寄存器，BX是16位寄存器，操作数长度不匹配。

(2) **ADD 15H, BX**：错误。ADD指令目的操作数不能是立即数（立即数不能作为目的）。

(3) **MOV CS, AX**：错误。CS寄存器只能读出（作为源），不能被MOV写入（只能通过JMP/CALL/INT修改）。

(4) **MOV AX, [SI][DI]**：错误。16位寻址中，变址寄存器只能选SI或DI之一，不能同时使用两个变址寄存器（SI和DI不能同时作变址）。正确格式是基址+变址：`[BX+SI]`或`[BX+DI]`。

(5) **MOV BYTE PTR [BX], 1000**：错误。用了BYTE PTR指定字节操作，但立即数1000超出一字节范围（0~255）。

**考察点**：MOV指令的使用限制，是选择题/简答题高频考点。

### 题目28（NOT+ADD结果）

**题目**：阅读下列程序段，将执行结果填入空格。

```asm
MOV AX, BX
NOT AX
ADD AX, BX
INC AX
```

**解答**：

设BX = N，则：
- `MOV AX, BX` → AX = N
- `NOT AX` → AX = ~N（按位取反，即 0FFFFH - N）
- `ADD AX, BX` → AX = ~N + N = 0FFFFH（16位全1）
- `INC AX` → AX = 0FFFFH + 1 = 10000H，16位截断后 = 0000H

**INC不影响CF**，所以CF保持为0（ADD的CF：FFFFH本身不超16位范围，CF=0；INC不改CF）。

**答**：AX = 0，CF = 0

### 题目29（源操作数寻址方式）

**题目**：说明下列指令中源操作数的寻址方式。

**解答**：

(1) `MOV CX, 24[BX][SI]`：**相对基址变址寻址**，EA = BX + SI + 24

(2) `MOV BX, [2000H]`：**直接寻址**，EA = 2000H

(3) `INC BYTE PTR[BX]`：**寄存器间接寻址**，EA = BX

(4) `MOV AX, [BP+DI]`：**基址变址寻址**，EA = BP + DI（注意：使用BP，默认段为SS）

**考察点**：8086的存储器寻址方式识别。

### 题目30（PUSH指令工作过程）

**题目**：解释PUSH DI指令是怎样工作的。

**解答**：

① SP ← SP - 1，调整SP指针（指向新的栈顶字节）。

② 将DI内容的**高8位**压入SP所指存储单元（DI高字节先入栈，地址较高）。

③ SP ← SP - 1，再次调整SP指针。

④ 将DI内容的**低8位**压入SP所指存储单元（DI低字节，地址较低）。

最终SP指向DI低字节所在地址（即新栈顶）。

**考察点**：8086堆栈操作原理，"先减SP再存数"。

### 题目31（伪指令TYPE/OFFSET/LENGTH/SIZE）

**数据定义**：

```asm
ORG 2000H
A1 DB ?
A2 DW 20 DUP(?)
A3 DB 'ABCD'
A4 DW 10 DUP(?)
```

**解答**：

(1) `MOV AX, TYPE A1`：A1是DB（字节变量），TYPE=1，**AX = 1**

(2) `MOV AX, OFFSET A2`：A2紧跟A1，A1在2000H占1字节，故A2偏移 = 2001H，**AX = 2001H**

(3) `MOV AX, LENGTH A3`：LENGTH返回用DUP定义时重复次数，A3不是DUP定义，返回1，**AX = 1**

(4) `MOV AX, SIZE A4`：A4是 DW 10 DUP(?)，SIZE = LENGTH × TYPE = 10 × 2 = **20**

### 题目36（提取正数并排序）

**题目**：从BUFFER中提取正的字节数据存入PLUS，再由大到小排序。

**核心思路**：
1. SUB1：遍历BUFFER，用TEST AL, 80H判断符号位，80H=0说明是正数（最高位为0）
2. SUB2：对PLUS中的正数进行冒泡排序
3. 主程序：CALL SUB1，CALL SUB2

**关键代码段（提取正数）**：

```asm
SUB1 PROC NEAR
    PUSH AX
    PUSH BX
    LEA SI, PLUS          ; SI指向存储正数的缓冲区
    LEA BX, BUFFER        ; BX指向原数据
    XOR DX, DX            ; DX计数正数个数
AA1:
    MOV AL, [BX]
    CMP AL, 24H           ; 检查是否到'$'结束标志
    JZ  AA3
    TEST AL, 80H          ; 测试最高位（符号位）
    JNZ AA2               ; 最高位=1，是负数，跳过
    MOV [SI], AL          ; 最高位=0，是正数，存入PLUS
    INC SI
    INC BX
    INC DX                ; 正数计数+1
    JMP AA1
AA2:
    INC BX                ; 跳过负数
    JMP AA1
AA3:
    POP BX
    POP AX
    RET
SUB1 ENDP
```

**考察点**：TEST指令判断符号位，子程序设计，冒泡排序。

## 5. 简答题备考模板

### 【简答1】子程序与宏指令的区别

**规范答法**：

子程序（过程）是可被多次调用的独立代码段，每次调用通过CALL指令跳转执行，执行完后通过RET返回，**代码只存在一份**，每次调用有跳转和返回的时间开销。

宏指令是在汇编时将宏定义**展开**为相应的代码序列，每次引用宏都会产生一份代码副本，不存在调用跳转开销，但程序体积更大。

区别对比：

| 比较项 | 子程序 | 宏指令 |
|--------|--------|--------|
| 代码占用 | 只有一份 | 每次展开都有一份 |
| 执行速度 | 有CALL/RET开销 | 无跳转开销，更快 |
| 适用场景 | 代码较长、多次调用 | 代码较短、追求速度 |

---

### 【简答2】RET、END、HLT的区别

**规范答法**：

- **RET**：子程序返回指令。从堆栈中弹出返回地址（IP或CS:IP），恢复程序执行流，跳回调用处继续执行。
- **END**：汇编伪指令，表示源程序汇编结束，是汇编程序处理的，不是CPU执行的。不影响程序运行。
- **HLT**：CPU暂停指令，停止CPU的程序执行。有三种方式退出暂停：接收到中断、硬件复位、DMA操作。

---

### 【简答3】冒泡排序程序（完整框架）🔥 高频

📝 画图要点：考试需要能写出双重循环结构代码。

**由大到小排序（无符号数）标准代码**：

```asm
DATA SEGMENT
NUMBER1 DB 100, 3, 90, 80, 99, 77, 44, 66, 50  ; N个数据
N EQU $-NUMBER1-1              ; 数据个数-1（=8），$是当前地址计数器
DATA ENDS

CODE SEGMENT
ASSUME CS:CODE, DS:DATA
START:
    MOV AX, DATA
    MOV DS, AX                 ; 初始化数据段寄存器

    MOV DX, N                  ; 外循环计数：遍数（共N-1遍）
A1:
    LEA BX, NUMBER1            ; 每遍从头开始
    MOV CX, DX                 ; 内循环计数：比较次数（每遍递减）
A2:
    MOV AL, [BX]               ; 取前一个数
    CMP AL, [BX+1]             ; 与后一个数比较
    JNC A3                     ; 前≥后（无进位），不交换，跳到A3
    XCHG AL, [BX+1]            ; 前<后，交换：AL与[BX+1]互换
    MOV [BX], AL               ; 将较大数写回[BX]
A3:
    INC BX                     ; 指向下一对
    LOOP A2                    ; 内循环（CX-1，不为0则继续）
    DEC DX                     ; 外循环计数-1
    JNZ A1                     ; 外循环（还有遍数则继续）

    MOV AH, 4CH
    INT 21H                    ; 程序结束，返回DOS
CODE ENDS
END START
```

❌ 易错：JNC是"无进位转移"，即CF=0时跳转。CMP AL,[BX+1]结果，当AL ≥ [BX+1]（无符号比较），CF=0，故JNC表示"前≥后，不需要交换"。

### 【简答4】子程序设计的一般结构

**规范答法**：

```asm
子程序名 PROC NEAR（或FAR）
    ; ① 保护现场：将子程序用到的、但主程序也用的寄存器压栈
    PUSH AX
    PUSH BX
    ; ...
    
    ; ② 取入口参数（主程序通过寄存器或存储器传递参数）
    
    ; ③ 子程序功能主体
    ; ...
    
    ; ④ 存出口参数（结果放到约定的寄存器或存储器）
    
    ; ⑤ 恢复现场：按PUSH的反序弹出
    POP BX
    POP AX
    
    ; ⑥ 返回主程序
    RET
子程序名 ENDP
```

主程序调用：`CALL 子程序名`

### 【简答5】伪指令END/ENDS/ENDP/ENDM的区别

**规范答法**：

- **END [标号]**：源程序汇编结束伪指令，标志整个源程序的结束，标号指示程序执行的起始地址。必须是程序最后一条语句，且只能有一个。
- **ENDS**：段定义结束，与SEGMENT配对使用，标志段定义的结束。
- **ENDP**：过程（子程序）定义结束，与PROC配对使用。
- **ENDM**：宏定义结束，与MACRO配对使用。

## 6. 易错点汇总

- ❌ 易错：`MOV CS, AX`合法
  ✅ 正确：CS不能作为MOV的目的操作数，只能读出（出），不能写入（入）。

- ❌ 易错：`INC AX`会影响CF
  ✅ 正确：INC/DEC指令**不影响CF**，但影响AF、OF、PF、SF、ZF。

- ❌ 易错：8086中`SHL AX, 3`合法
  ✅ 正确：8086中移位次数只能是1或CL，必须写`MOV CL, 3` / `SHL AX, CL`。Pentium以后才支持立即数形式。

- ❌ 易错：`MOV AX, [SI][DI]`合法（两个变址寄存器）
  ✅ 正确：16位寻址中，SI和DI都是变址寄存器，不能同时使用。合法组合是基址(BX/BP)+变址(SI/DI)。

- ❌ 易错：`PUSH AL`合法
  ✅ 正确：PUSH/POP操作数必须是16位或32位，8位寄存器（AL/AH等）不能直接PUSH。

- ❌ 易错：`ADD VAR1, VAR2`（两个内存操作数）合法
  ✅ 正确：绝大多数指令的源和目的操作数不能**同时**为存储器操作数。必须借助寄存器中转。

- ❌ 易错：`JNZ VAR12`（JNZ后跟变量名）
  ✅ 正确：条件跳转指令后跟的必须是**指令标号**，不能是数据变量名。

- ❌ 易错：OFFSET与LEA效果完全相同
  ✅ 正确：`MOV AX, OFFSET VAR`在汇编时确定（静态），`LEA AX, VAR`在运行时计算（可用于复杂地址表达式如`LEA AX, [BX+SI+4]`，OFFSET不能处理）。

- ❌ 易错：`N EQU $-ARRAY`得到的是数组元素个数
  ✅ 正确：`$-ARRAY`得到的是从ARRAY到当前位置的**字节数**。若ARRAY是DW定义，元素个数=字节数÷2。

- ❌ 易错：AND/OR/XOR执行后OF=1（有溢出）
  ✅ 正确：AND/OR/XOR/TEST执行后，**CF=0，OF=0**（强制清零），AF无定义。

## 7. 一页纸速记

**核心公式**：
- EA = 基址 + 变址 + 位移量（16位：基址用BX/BP，变址用SI/DI）
- PUSH：SP←SP-2，存数；POP：取数，SP←SP+2
- INT n：标志→CS→IP压栈；IRET：IP→CS→标志出栈

**必背结论**：
- CS不能被MOV写入；INC/DEC不影响CF；AND/OR/XOR清零CF和OF
- 条件跳转只能段内短跳转（±127字节）
- `$`是汇编时地址计数器，`N EQU $-数组名`计算字节数
- 子程序保护现场用PUSH，恢复用POP，顺序必须相反

**最易混淆的区别**：

| 指令 | 区别 |
|------|------|
| JA vs JG | JA无符号大于（看CF/ZF），JG有符号大于（看SF/OF/ZF） |
| JNC vs JNS | JNC：CF=0（无进位）；JNS：SF=0（结果非负） |
| MOV vs LEA | MOV取内容；LEA取地址（运行时计算） |
| END vs ENDS vs ENDP | 程序结束/段结束/过程结束 |
| SHR vs SAR | SHR高位补0；SAR高位补符号位 |
| PUSH vs CALL | PUSH只压数据；CALL压IP（或CS:IP）并跳转 |
