## 二、知识重排正文

本章两大主题——键盘接口与LED显示器接口——之间的逻辑关系是：它们都依赖8255A这块可编程并行接口芯片作为中间人，键盘通过8255A的输入口接收行列信号，LED通过8255A的输出口送出段码与位码。因此，把握8255A的配置方法是本章一切程序设计的前提。

---

### 模块一：键盘接口基础——为什么直接读引脚不可靠

#### 1.1 抖动问题与消抖原理 ⚠️考点

机械按键按下或弹起的瞬间，触点并不会干净地完成一次闭合/断开，而是在10～20ms内反复抖动几次，才最终稳定。这意味着：**如果CPU在抖动期间读取引脚电平，一次按键会被误识别为多次按键**。

解决办法分两类：

**硬件滤波**：对每个按键加RC滤波电路或RS去抖电路，成本高，每键都需要额外电路。

**软件消抖**（课件9.1.1，考试常用方法）：
```
第一次检测到低电平（疑似有键按下）
    ↓
等待约20ms（CALL D20ms延时子程序）
    ↓
再次读取引脚
    ↓
仍为低电平 → 确认有键按下，开始识别
仍为高电平 → 刚才是干扰，重新查询
```

**✅参考答案要点：消抖的本质是"两次采样确认"，中间间隔20ms左右，使抖动区间完全落在两次采样之间，从而滤掉干扰。**

---

#### 1.2 编码键盘 vs 非编码键盘 ⚠️考点

| 类型 | 编码方式 | 特点 |
|---|---|---|
| 编码键盘 | 硬件自动产生ASCII码或其他编码 | 使用方便，成本高 |
| 非编码键盘 | 仅提供行列矩阵，软件识别 | 设计灵活，成本低，需解决抖动/重键/识别/编码四个问题 |

课件指出非编码键盘需解决的四个问题：
1. 清除键接触时的抖动干扰
2. 防止重键错误（两键同时按下）
3. 键盘结构及按键识别
4. 产生被按键相应的编码

---

#### 1.3 键盘结构：线性键盘 vs 矩阵键盘 ⚠️考点🆕（学习要点原文已列）

**线性键盘（独立式按键）**：每个按键独占一根I/O口线，直接和8255A的某一位相连。优点是逻辑简单，缺点是按键多时占用大量I/O口线。

**矩阵键盘（行列式键盘）**：按键设置在行线和列线的交点上，N根行线×M根列线可以支持N×M个按键，仅用N+M根I/O线。

---

**真题原题：** ✅真题🔥

> **选择题第15题：设有14个按键组成矩阵键盘，识别这14个按键最少需要（ ）根线。**  
> A. 14  B. 8  C. 7  D. 6

**解题逻辑**：寻找满足"行数×列数 ≥ 14"且"行数+列数最小"的正整数组合。

| 组合 | 行×列 | 覆盖键数 | 总线数 |
|---|---|---|---|
| 2×7 | 14 | ✅ | 9 |
| 3×5 | 15 | ✅ | 8 |
| 4×4 | 16 | ✅ | **8** |
| 7×2 | 14 | ✅ | 9 |

最小总线数为 **8根**（3行5列或4行4列均可）。

**✅参考答案要点：答案B，8根。原则是找行数+列数最小且乘积≥按键数的整数组合，4×4=16≥14，行+列=8最小。**

### 模块二：矩阵键盘扫描算法——行扫描法与线反转法 ⚠️考点🔥

矩阵键盘需要软件来确定"哪根行线和哪根列线的交叉点有键按下"。课件和题库介绍了两种扫描算法，这也是本章最重要的程序设计方法。

---

**练习题Q2（核心原题）：**

> **什么是键盘的行扫描法和线反转法？其实现过程有哪些区别？**

#### 2.1 行扫描法（步进扫描方式）🔥

**硬件要求**：行线接输出口，列线接输入口，方向固定不变。

**工作流程**（以4行4列为例）：

```
Step 1：预检——行全输出0，列全输入读取
         若列全为1 → 无键按下，继续轮询
         若列有0  → 有键按下，执行软件消抖（20ms）

Step 2：逐行扫描——行扫描值从 11111110B 开始，每次循环左移1位
         第0行：行输出 11111110B（仅第0行为低）
         第1行：行输出 11111101B
         第2行：行输出 11111011B
         第3行：行输出 01111111B

Step 3：每输出一个行扫描值，立即读列线
         若列线不全为1 → 找到被按键所在行（行扫描值）和列（列线读值）

Step 4：形成位置码 = 行号（高4位）OR 列号（低4位）
         在键值表（TABLE）中查表 → 得到键号（0~F等）
```

**课件代码核心片段**（行扫描识别阶段）：
```asm
MOV AH, 11111110B  ; 初始行扫描值（第0行为低）
MOV CX, 4          ; 共4行
A2: MOV DX, 200H   ; 行输出端口地址
    MOV AL, AH
    OUT DX, AL     ; 输出行扫描值
    INC DX         ; 列输入端口地址 201H
    IN  AL, DX     ; 读列线状态
    AND AL, 0FH    ; 取低4位
    CMP AL, 0FH
    JNZ A3         ; 列不全1 → 找到按键所在行列
    ROL AH, 1      ; 准备下一行
    LOOP A2
```

形成位置码（找到行列后）：
```asm
A3: MOV CL, 4
    SHL AH, CL     ; 行号移到高4位
    OR  AL, AH     ; 行号|列号 → 位置码
    ; 查TABLE表得到键号
```

#### 2.2 线反转法（行列反转扫描法）✅真题🔥🆕

**硬件要求**：行线和列线都需要能动态改变输入/输出方向（如8255A工作于方式0，通过重写控制字切换A口方向），这是线反转法的核心前提。

**工作流程**：

```
Step 1（行输出阶段）：
    将行口设为输出，列口设为输入
    行输出全0（0000...）
    读列口 → 若全1，无键；若有0，保存列值CL

Step 2（线反转阶段）：
    将行口改为输入，列口改为输出（重写8255A控制字）
    将上一步读到的列值从列口输出
    读行口 → 保存行值RW

Step 3（定位）：
    由读入的列值CL和行值RW即可唯一确定按下的按键位置
```

**ASCII示意图**：
```
第一步（行→列）          第二步（列→行，即"反转"）
┌───────┐                ┌───────┐
│ 行口  │───输出全0──→   │ 行口  │←──读行值──
│（输出）│               │（输入）│
└───────┘                └───────┘
┌───────┐                ┌───────┐
│ 列口  │←──读列值──     │ 列口  │───输出上步列值──→
│（输入）│               │（输出）│
└───────┘                └───────┘
```

**真题综合设计题**明确要求"键盘以行列反转方式工作"，这意味着8255A的C口高4位（PC7～PC4）接列，C口低4位（PC3～PC0）接行，两次切换时需重写8255A控制字以改变PC口高低半字节的输入/输出方向。

#### 2.3 两种方法对比 ⚠️考点

| 对比维度 | 行扫描法 | 线反转法 |
|---|---|---|
| 硬件要求 | 行输出、列输入，方向固定 | 行列方向可互换，接口需支持方向切换 |
| 硬件复杂度 | **简单** | 复杂（需可编程I/O口如8255A） |
| 软件复杂度 | **复杂**（逐行循环扫描，查表） | 简单（两步操作即可定位） |
| 识别速度 | 较慢（最坏需扫描所有行） | 较快（两步即定位） |

**✅参考答案要点：行扫描法逐行输出低电平扫描字、读列值查表；接口简单但软件复杂。线反转法先行输出全0读列值，再反转方向输出列值读行值；需要接口方向可变（硬件复杂），但软件简单。两者均需软件消抖。**

### 模块三：键盘工作方式——CPU何时响应按键 （课件补充·无对应原题）

课件9.1.4节介绍了键盘的三种工作方式，虽然练习题无直接对应，但选择题背景知识：

| 工作方式 | 触发时机 | 特点 |
|---|---|---|
| 程序控制扫描 | CPU空余时主动调用扫描子程序 | 最简单，但CPU始终参与扫描 |
| 定时扫描 | 定时器中断触发，CPU响应后扫描 | 本质是中断方式，但定时响应而非实时响应 |
| 中断扫描 | 有键按下时键盘主动向CPU发中断请求 | CPU效率最高，实时响应 |

### 模块四：BIOS与DOS键盘中断服务 ⚠️考点✅真题

键盘的中断号考点集中在09H/16H/21H三者的定位区别，是高频选择题素材。

---

**真题原题：** ✅真题

> **选择题第7题：（ ）是系统功能调用。**  
> A. INT 9H  B. INT 10H  C. INT 16H  D. INT 21H

#### 4.1 三种中断号的定位

| 中断号 | 名称 | 角色 | 数据流方向 |
|---|---|---|---|
| **INT 09H** | 键盘硬件中断服务 | BIOS级，硬件中断 | 从键盘控制器（60H端口）读扫描码 → 写入键盘缓冲区**队尾** |
| **INT 16H** | BIOS键盘服务 | BIOS级，软件调用 | 从键盘缓冲区**队首**读取数据 → 送AL（ASCII码）+AH（扫描码） |
| **INT 21H** | DOS系统功能调用 | DOS级，软件调用 | 提供键盘输入/字符输出等高级功能，是编程题最常用的接口 |

**记忆口诀**：09写队尾（硬件中断写缓冲），16读队首（BIOS读缓冲），21是DOS（应用程序用这个）。

#### 4.2 INT 09H 处理过程

1. 从键盘接口输出缓冲寄存器（端口60H）读取系统扫描码
2. 判断该键分类
3. 将ASCII码和扫描码写入键盘缓冲区队尾

键盘缓冲区位于BIOS数据区，占32字节，存放16次击键信息，以FIFO循环队列方式工作。

#### 4.3 INT 16H 常用功能号

| AH值 | 功能 | 无键时行为 |
|---|---|---|
| AH=0 | 从键盘读一字符到AL，AH=扫描码 | **等待**（阻塞） |
| AH=1 | 检测缓冲区是否有字符；有键→ZF=0，AL=字符；无键→ZF=1 | **不等待**（非阻塞） |

#### 4.4 INT 21H 常用功能号（编程必备）🔥

| AH值 | 功能 |
|---|---|
| AH=1 | 从键盘输入一字符并**回显**到屏幕，AL=字符 |
| AH=2 | 显示DL中的字符 |
| AH=7 | 从键盘输入一字符，**不回显**，AL=字符 |
| AH=9 | 显示DS:DX指向的字符串（以$为结束标志） |
| AH=4CH | 程序返回DOS |

**✅参考答案要点：INT 09H是键盘硬件中断（写缓冲区队尾）；INT 16H是BIOS软件调用（读缓冲区队首）；INT 21H是DOS系统功能调用（应用程序使用），选D。**

### 模块五：LED七段显示器基础——段码与极性 ⚠️考点

键盘解决了"如何输入"，LED显示器解决"如何输出"。在程序设计层面，二者都通过8255A连接，是本章综合设计题的两个并列模块。

---

**练习题Q3：**

> **LED显示器的工作原理是什么？何谓共阳极？何谓共阴极？**

#### 5.1 七段显示器结构

LED七段显示器由a/b/c/d/e/f/g七个发光二极管（加小数点h共8个）组成，不同段的组合点亮形成不同数字。

```
      a
   ┌─────┐
 f │     │ b
   ├─────┤
 e │  g  │ c
   └─────┘
      d      ●h
```

**共阴极（Common Cathode）**：所有LED的阴极（负极）连接在一起接地（COM接GND）。要点亮某段，对应段脚输出**高电平1**。"要亮给1"。

**共阳极（Common Anode）**：所有LED的阳极（正极）连接在一起接+5V（COM接VCC）。要点亮某段，对应段脚输出**低电平0**（拉低）。"要亮给0"。

❌易错提醒：**共阴极和共阳极的段码互为反码**。做综合设计题第一步必须先确认题目给的是哪种极性。

#### 5.2 段码速查表

课件表9.2.1（共阴极）与动态显示例题（共阳极）的对应关系：

| 数字 | 共阴极段码（H） | 共阳极段码（H） |
|---|---|---|
| 0 | 3FH | C0H |
| 1 | 06H | F9H |
| 2 | 5BH | A4H |
| 3 | 4FH | B0H |
| 4 | 66H | 99H |
| 5 | 6DH | 92H |
| 6 | 7DH | 82H |
| 7 | 07H | F8H |
| 8 | 7FH | 80H |
| 9 | 6FH | 98H |

> 验证方法：共阴极段码 OR 共阳极段码 = FFH（互为反码，因为第7位h通常为0，部分数字可能有差异，但规律成立）

**真题综合设计题**要求写出"0"对应的七段数码管译码值（共阳极）：  
**答案：0C0H**

**✅参考答案要点：共阴极各段接正电平点亮（给1），共阳极各段接负电平点亮（给0），两者段码互为反码。"0"共阴极段码3FH，共阳极段码C0H。**

### 模块六：显示方式——静态 vs 动态 ⚠️考点🔥

---

**练习题Q4：**

> **LED显示器接口有哪几种方式？**

#### 6.1 静态显示

每个显示位独占一个8位输出口，当前要显示的段码持续输出，发光二极管恒定导通或截止。

- 优点：显示稳定，程序简单
- 缺点：显示位数多时占用大量I/O口（每位需1个端口）

课件图9.2.2：8255A的PA口驱动第1位LED，PB口驱动第2位，PC口驱动第3位，各自独立输出段码，互不影响。

#### 6.2 动态显示（扫描显示）🔥

多个LED共用同一组段码线（8位数据线），但每次只点亮其中一位（通过位码选择），快速循环扫描所有位，利用人眼视觉暂留效应形成"同时显示"的效果。

**两条控制线**：
- **段码**（Segment Code）：决定当前亮起的位显示什么字形，送到所有LED共用的段线（如PB口）
- **位码**（Digit Select Code）：决定哪一位LED被点亮（如PA口输出选通信号）

```
循环流程（每一位）：
  ① 送段码到PB口（该位要显示的字形）
  ② 送位码到PA口（选中该位LED）
  ③ 延时一小段时间（约1~5ms）
  ④ 切换到下一位
```

- 优点：节省I/O口（N位LED只需段码口+位码口）
- 缺点：程序复杂，亮度较低（每位实际点亮时间 = 总时间/位数）

**✅参考答案要点：LED显示接口分静态显示（每位独占端口，段码持续输出，稳定但占用I/O多）和动态显示（多位共用段码口，位码轮流选通，节省I/O但程序复杂）两种方式。**

### 模块七：综合应用——8255A驱动LED动态显示接口设计 ⚠️考点🔥📝

---

**练习题Q5：**

> **设计一个通过8255A芯片控制的一个两位LED的动态显示接口电路。**

#### 7.1 硬件设计思路

两位动态显示，需要：
- **段码口**（8位输出）：送a～g/h段选信号，两位LED共用
- **位码口**（至少2位输出）：依次选通第0位和第1位

用8255A：A口（PA0～PA7）接段码；B口（PB0～PB1）接位码（仅用2位）；控制字配置A口和B口均为方式0输出。

控制字计算（A口输出，B口输出，C口不用）：
```
D7  D6  D5  D4  D3  D2  D1  D0
 1   0   0   0   0   0   0   0  = 80H
 ↑特征  A组方式0  A口出  C高出  B组方式0  B口出  C低出
```
控制字 = **80H**

#### 7.2 显示子程序逻辑

```asm
; 假设显示缓冲区：BUF0=个位数字，BUF1=十位数字
; 8255A端口：PA=PORT_A, PB=PORT_B, CTR=PORT_CTR

DISPLAY PROC NEAR
    ; 显示第0位（个位）
    MOV AL, BUF0        ; 取个位字形值（0~9）
    LEA BX, LEDTAB
    XLAT                ; 查表得段码（共阴极）
    OUT PORT_A, AL      ; 输出段码
    MOV AL, 01H         ; 位码：选通第0位（PB0=1，如共阴极驱动）
    OUT PORT_B, AL
    CALL DELAY          ; 延时约1~5ms
    
    ; 显示第1位（十位）
    MOV AL, BUF1
    LEA BX, LEDTAB
    XLAT
    OUT PORT_A, AL
    MOV AL, 02H         ; 位码：选通第1位（PB1=1）
    OUT PORT_B, AL
    CALL DELAY
    RET
DISPLAY ENDP
```

**✅参考答案要点：两位动态显示需段码口（A口输出段选码）+位码口（B口输出位选通信号），8255A控制字80H（A口B口均方式0输出）；显示子程序循环输出每位的段码与位码，各延时几毫秒，利用视觉暂留产生同时显示效果。**

### 模块八：真题综合设计题全解——湿度闭环控制系统 ✅真题🔥📝

这道综合设计题是本章所有知识的汇总，也是对第7章（8255A/8253A）和第8章（ADC/DAC）知识的综合运用。

**真题原题（综合设计题，15分）：**

> 设计一个湿度闭环采集控制系统（乐器保存湿度控制）。  
> 组成：两位七段数码管 + 4×4按键 + 湿度检测（ADC0809） + 输出控制加湿器（DAC0832）。  
> 要求：(1) 利用一片8255连接两位共阳极七段数码管，以**静态方式**显示；**键盘以行列反转方式工作**；湿度采集ADC0809；输出控制DAC0832；用8253发出1kHz按键提示音。

#### 8.1 "0"的共阳极七段段码 ❌易错

共阳极下"0"的段码：查表得 **0C0H**。

验证：共阴极"0"= 3FH = 0011 1111B，取反 = 1100 0000B = 0C0H（最高位dp不亮）。

#### 8.2 8255A初始化 ⚠️考点

题目要求：
- PA口：静态显示，接两位数码管各自的段码（一个口接一位，需要两个口，用PA和PB）
- PC口：4×4键盘（行列反转），高4位（PC7～PC4）接列线，低4位（PC3～PC0）接行线

由于键盘行列反转时需要切换PC口高低半字节的方向，一种典型分配方案：
- PA口（8位）→ 数码管十位（静态输出）
- PB口（8位）→ 数码管个位（静态输出）
- PC高4位（PC7～PC4）→ 键盘列线
- PC低4位（PC3～PC0）→ 键盘行线

**键盘扫描第一步**（行输出列输入）：PC低4位输出，PC高4位输入  
控制字 = `1000 1001B` = **89H**（A组方式0，A口输出，C口高输入；B组方式0，B口输出，C口低输出）

**键盘扫描第二步**（线反转：行输入列输出）：PC低4位输入，PC高4位输出  
控制字 = `1000 0010B` = **82H**（A口输出，C口高输出；B口输出，C口低输入）

```asm
init_8255 PROC NEAR
    MOV DX, PORT_CTR_8255
    MOV AL, 89H          ; 初始状态：A口B口输出（显示），PC高输入（列），PC低输出（行）
    OUT DX, AL
    RET
init_8255 ENDP
```

#### 8.3 8253初始化（1kHz按键提示音）⚠️考点

CLK=1MHz，需产生1kHz方波，工作方式3（方波发生器）：

```
计数初值 N = CLK / 目标频率 = 1MHz / 1kHz = 1000
```

控制字（以计数器1为例，先低后高，方式3，二进制）：
```
D7 D6 | D5 D4 | D3 D2 D1 | D0
 0  1 |  1  1 |  0  1  1 |  0   = 76H
计数1  先低后高  方式3      二进制
```

```asm
init_8253 PROC NEAR
    MOV DX, PORT_CTR_8253
    MOV AL, 76H          ; 计数器1，先低后高，方式3，二进制
    OUT DX, AL
    MOV DX, PORT_CTR1_8253
    MOV AX, 1000         ; 计数初值 1000
    OUT DX, AL           ; 写低8位
    MOV AL, AH
    OUT DX, AL           ; 写高8位
    RET
init_8253 ENDP
```

#### 8.4 按键识别子程序（线反转法）✅真题🔥

```asm
key PROC NEAR
    ; --- 第一步：行输出全0，读列值 ---
    MOV DX, PORT_CTR_8255
    MOV AL, 89H          ; PC低输出（行），PC高输入（列）
    OUT DX, AL
    MOV DX, PORT_C_8255
    MOV AL, 00H          ; 行全输出0
    OUT DX, AL
    NOP                  ; 稳定信号
    IN  AL, DX           ; 读PC口
    AND AL, 0F0H         ; 取高4位（列值）
    CMP AL, 0F0H
    JE  no_key           ; 全1则无键按下
    
    ; --- 软件消抖 ---
    CALL DELAY_20ms
    IN  AL, DX
    AND AL, 0F0H
    CMP AL, 0F0H
    JE  no_key
    
    ; --- 保存列值 ---
    MOV BH, AL           ; BH保存列值（高4位有效）
    
    ; --- 第二步：线反转，列输出上步读到的列值，读行值 ---
    MOV DX, PORT_CTR_8255
    MOV AL, 82H          ; PC高输出（列），PC低输入（行）
    OUT DX, AL
    MOV DX, PORT_C_8255
    MOV AL, BH           ; 输出上步读到的列值
    OUT DX, AL
    NOP
    IN  AL, DX           ; 读PC口
    AND AL, 0FH          ; 取低4位（行值）
    
    ; --- BH高4位=列值，AL低4位=行值，共同确定按键位置 ---
    OR  AL, BH           ; 合并行列值 → 键位置码
    ; 此处可查表转换为键号，或直接判断
    
    ; --- 等待键释放 ---
    ; ... （略）
    
    CALL BEEP_1kHz       ; 调用8253发出提示音
    RET
no_key:
    RET
key ENDP
```

#### 8.6 ADC采集子程序（查询方式）

```asm
adc PROC NEAR
    ; 启动ADC0809转换（通道0=湿度传感器）
    MOV DX, PORT_ADC_START
    MOV AL, 00H          ; ALE有效，选通道0
    OUT DX, AL
    
    ; 等待EOC变高（转换完成）
wait_eoc:
    MOV DX, PORT_ADC_EOC
    IN  AL, DX
    TEST AL, 01H         ; 检查EOC引脚
    JZ  wait_eoc
    
    ; 读取转换结果
    MOV DX, PORT_ADC_DATA
    IN  AL, DX
    MOV [adc_val], AL    ; 保存数字量（0~255对应0~100%湿度）
    RET
adc ENDP
```

#### 8.7 DAC输出子程序（控制加湿器）

控制逻辑：当前湿度与设定湿度比较，输出0V/1V/2V三档控制电压。

DAC0832，Vref=5V，输出范围0～5V（单极性），公式：
```
Vout = Vref × D / 256 = 5 × D / 256
```

- 强加湿（2V）：D = 2 × 256 / 5 = 102（取整，约102）= 66H
- 中等加湿（1V）：D = 1 × 256 / 5 = 51 = 33H
- 关闭（0V）：D = 0 = 00H

```asm
dac PROC NEAR
    ; CL = adc_val（当前湿度数字量），DH = target_humidity（设定湿度）
    MOV DX, PORT_DAC
    MOV AL, CL
    
    ; 比较判断输出哪档
    MOV BL, DH
    SUB BL, 10           ; target - 10%
    CMP CL, BL
    JB  strong_hum       ; 当前湿度 < 设定-10%，强加湿
    
    MOV BL, DH
    ADD BL, 10           ; target + 10%
    CMP CL, BL
    JA  no_hum           ; 当前湿度 > 设定+10%，关加湿
    
    ; 中等加湿：1V
    MOV AL, 33H
    OUT DX, AL
    RET
strong_hum:
    MOV AL, 66H          ; 2V
    OUT DX, AL
    RET
no_hum:
    MOV AL, 00H          ; 0V
    OUT DX, AL
    RET
dac ENDP
```

**✅参考答案要点：综合设计题要素——8255A初始化（静态显示+行列反转键盘）；8253产生1kHz方波（方式3，计数初值1000）；"0"共阳段码0C0H；线反转法两步骤（行输出全0读列值→反转读行值）；ADC查询EOC标志；DAC按三档阈值输出0/1/2V控制电压。**
