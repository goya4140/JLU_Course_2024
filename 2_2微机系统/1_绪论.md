## 2. 核心知识点精讲

### 2.1 三个层次的概念辨析

**核心定义**：

- **微处理器**：用一片或少数几片大规模集成电路组成的中央处理器（CPU 芯片本身）。
- **微型计算机**：由 CPU、主存储器、I/O 接口电路和系统总线构成的硬件整体。
- **微型计算机系统**：在微型计算机基础上，加上 I/O 设备、系统软件、电源、面板和机架，构成完整可用的系统。

**原理解析**（包含关系）：

```
微型计算机系统
  └─ 微型计算机 (硬件核心)
       └─ 微处理器 (CPU)
            + 主存储器
            + I/O 接口
            + 系统总线 (DB / AB / CB)
  + I/O 设备 (键盘、显示器等)
  + 系统软件 (OS、驱动等)
  + 电源、机架等
```

**考试写法**：三者是包含关系，微处理器是微型计算机的核心部件，微型计算机是微型计算机系统的硬件基础。

⚠️ 考点：简答 | ⭐⭐

### 2.2 微型计算机的三总线结构

**核心定义**：微型计算机内部通过三类总线连接 CPU、主存和 I/O 接口：

| 总线 | 英文 | 方向 | 作用 |
|------|------|------|------|
| 数据总线 DB | Data Bus | 双向 | 传输数据 |
| 地址总线 AB | Address Bus | 单向（CPU→外） | 指定访问地址 |
| 控制总线 CB | Control Bus | 双向 | 传输控制信号（RD/WR等） |

**结构图**：

```
         地址线 AB ─────────────────────────→
微处理器  数据线 DB ←──────────────────────→  主存M │ I/O接口 ↔ I/O设备
  CPU    控制线 CB ←──────────────────────→
```

### 2.3 CISC 与 RISC 技术特点

⚠️ 考点：简答 | ⭐⭐⭐ 🔥 高频

**核心定义**：

- **CISC**（Complex Instruction Set Computer，复杂指令系统计算机）：指令集庞大、功能强，硬件实现特殊功能。代表：Intel x86 系列。
- **RISC**（Reduced Instruction Set Computer，精简指令系统计算机）：只保留高频常用指令，复杂功能由基本指令组合完成。代表：ARM、MIPS、PowerPC。

**三维度对比**：

| 对比维度 | RISC | CISC |
|----------|------|------|
| **指令系统** | 简单，指令短、等长，数量少 | 复杂，指令长、不等长，数量多 |
| **内存访问** | 专有 Load/Store 指令，访问简单 | 大量指令可直接操作内存，控制逻辑复杂 |
| **电路结构** | 简单，面积小，功耗低，设计周期短 | 复杂，面积大，功耗高，技术更新难 |
| **中断响应** | 在一条指令执行的适当地方响应中断 | 在一条指令执行**结束后**响应中断 |
| **特殊任务** | 效率较低（需组合基本指令） | 效率高（有专用指令） |
| **易用性** | 结构简单，易学易用，性能易把握 | 功能强大，实现特殊功能容易 |

**典型产品**：

- CISC：Intel x86、AMD、TI（德州仪器）、Cyrix、VIA（威盛）
- RISC：IBM PowerPC/Power2、Sun SPARC、HP PA-RISC 7000、MIPS R系列、**ARM 微处理器**

**考试写法（简答题可直接套用）**：

> CISC 是复杂指令系统计算机，RISC 是精简指令系统计算机。两者主要区别如下：
>
> ① **指令系统**：RISC 指令系统简单，指令短且等长，只含常用指令；CISC 指令系统复杂，指令长且不等长，含大量特殊功能指令。
>
> ② **内存访问**：RISC 对存储器操作简单，只有专用 Load/Store 指令；CISC 有大量指令直接操作存储器，控制逻辑复杂。
>
> ③ **中断响应**：RISC 可在一条指令执行的适当地方响应中断；CISC 在一条指令执行结束后响应中断。
>
> ④ **电路结构**：RISC 电路简单，面积小、功耗低、设计周期短；CISC 电路复杂，面积大、功耗高、技术更新难。
>
> ⑤ **使用特点**：RISC 易学易用，处理特殊功能时需组合基本指令，效率较低；CISC 功能强大，处理特殊任务效率高。

❌ 易错：不要混淆"RISC中断响应时机"——RISC 是在指令执行**适当地方**（不是结束后）响应中断，CISC 才是在指令**结束后**。

### 2.4 微处理器发展各代特点（了解）

| 代别 | 时间 | 典型产品 | 特点 |
|------|------|----------|------|
| 4位 | 1970年代初 | Intel 4004, 4040 | 指令简单，运算功能差 |
| 8位 | 1974—1977 | Intel 8080/8085, Z80 | 指令完善，有中断和DMA |
| 16位 | 1978—1980 | **Intel 8086**（DB16/AB20）, 80286 | 多级中断，段式存储 |
| 32位 | 1980年代初 | 80386, 80486, **Pentium** | Cache，多任务，超标量 |
| 64位 | 2001年起 | Itanium, Core i系列 | 多核，超线程，睿频 |

> **记忆要点**：8086 是 16 位数据总线、20 位地址总线；Pentium(80586) 是 32 位地址总线、**64 位数据总线**，首次超标量结构。

## 4. 练习题精析

### 题目1（练习题第1题）

**题目**：微处理器、微型计算机、微型计算机系统的区别是什么？

**解答**：
- **微处理器**：即 CPU，是用大规模集成电路制造的中央处理器芯片，只是一个芯片级器件。
- **微型计算机**：以微处理器为核心，加上主存储器、I/O 接口电路和系统总线，构成的硬件系统。
- **微型计算机系统**：在微型计算机基础上，配备 I/O 外部设备、系统软件、电源、面板和机架，构成完整、可独立运行的计算机系统。

三者是层层包含的关系：微处理器 ⊂ 微型计算机 ⊂ 微型计算机系统。

**考察点**：概念辨析，三层次包含关系。

---

### 题目2（练习题第2题）

**题目**：微型计算机有哪些基本部分构成？

**解答**：
微型计算机由以下四部分构成：
1. **微处理器（CPU）**：执行指令，进行运算和控制。
2. **主存储器（RAM/ROM）**：存储程序和数据。
3. **I/O 接口电路**：连接外部设备与系统总线的中间电路。
4. **系统总线**：包括数据总线（DB）、地址总线（AB）、控制总线（CB），是各部件间信息传输的公共通道。

**考察点**：微型计算机四大基本组成。

---

### 题目3（练习题第3题）

**题目**：CISC、RISC 的技术特点。

**解答**：见第 2.3 节完整对比表和"考试写法"。

关键补充（来自练习题答案原文要点）：
- RISC 不经常使用的功能通过基本指令**组合**完成，处理特殊功能时效率相对较低。
- CISC 指令系统丰富，处理特殊任务效率较高。
- 中断响应时机：RISC 在指令执行**适当地方**响应；CISC 在指令执行**结束后**响应。
- RISC CPU 面积小、功耗低、设计周期短；CISC 相反。

**考察点**：CISC/RISC 三维度对比，中断响应时机差异，典型产品举例。

## 5. 简答题备考模板

### 简答：CISC 与 RISC 的主要技术特点及区别

**规范答法**（考试写法）：

CISC（复杂指令系统计算机）与 RISC（精简指令系统计算机）的主要区别体现在以下几方面：

**① 指令系统**：RISC 指令系统简单，指令短、等长，仅含常用指令，不常用功能通过基本指令组合实现；CISC 指令系统复杂，指令长且不等长，含大量特殊功能指令，处理特殊任务效率高。

**② 存储器访问**：RISC 对存储器操作简单，只有专用的 Load/Store 指令访问内存；CISC 有大量指令可直接操作存储器，控制逻辑复杂。

**③ 中断响应**：RISC 可在一条指令执行的适当地方响应中断；CISC 在一条指令执行结束后响应中断。

**④ 电路结构**：RISC 电路简单，芯片面积小、功耗低、设计周期短，易于采用并行技术；CISC 电路复杂，面积大、功耗高、技术更新难度大。

**⑤ 使用特点**：RISC 结构简单，易学易用；CISC 功能强大，实现特殊功能容易。

---

### 简答：微型计算机的基本构成

**规范答法**：

微型计算机由四部分构成：微处理器（CPU）、主存储器、I/O 接口电路和系统总线。系统总线包含三类：数据总线（DB，双向传输数据）、地址总线（AB，单向，CPU 发出地址）、控制总线（CB，双向传输控制信号）。

## 6. 易错点汇总

- ❌ 易错：认为 RISC 在指令"结束后"响应中断。
  ✅ 正确：RISC 在指令执行**适当地方**（即可以响应的时刻）响应中断；**CISC** 才是在指令**执行结束后**响应中断。

- ❌ 易错：将"微型计算机"与"微型计算机系统"混用。
  ✅ 正确：微型计算机是纯硬件（CPU+主存+I/O接口+总线）；微型计算机系统还包含 I/O 外设、操作系统等软件和机架电源等。

- ❌ 易错：认为 ARM 是 CISC 架构。
  ✅ 正确：ARM 是典型 **RISC** 架构，Intel x86 才是 CISC。

- ❌ 易错：Pentium 的数据总线是 32 位。
  ✅ 正确：Pentium(80586) 地址总线 32 位，**数据总线 64 位**（外部总线宽度）。

- ❌ 易错：将 8086 和 8088 混淆。
  ✅ 正确：8086 内外部数据总线均为 16 位；8088（准16位）内部 16 位、外部 **8 位**数据总线（IBM PC/XT 用的是 8088）。

## 7. 一页纸速记

**核心结论（必背）**：

```
三层次：微处理器 ⊂ 微型计算机 ⊂ 微型计算机系统
三总线：DB（双向/数据）、AB（单向/地址）、CB（双向/控制）

RISC vs CISC 口诀：
  指令：RISC 简短等长，CISC 复杂不等长
  内存：RISC 专用指令，CISC 多指令直接访问
  中断：RISC 适当地方，CISC 指令结束后
  电路：RISC 简单省电，CISC 复杂功能强
  代表：RISC→ARM/MIPS/PowerPC，CISC→Intel x86/AMD
```

**最易混淆区别**：

| | RISC | CISC |
|---|---|---|
| 中断响应时机 | 指令执行**适当地方** | 指令执行**结束后** |
| 内存操作指令 | 少（专用 Load/Store） | 多（可直接操作内存） |
| 电路复杂度 | 简单，面积小功耗低 | 复杂，面积大功耗高 |

**典型处理器速记**：

- 8086：16位CPU，DB=16位，AB=20位（寻址1MB）
- Pentium：32位CPU，DB=**64位**，AB=32位，首次**超标量**
- ARM：RISC，广泛用于移动设备
