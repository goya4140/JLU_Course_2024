## 1. 知识体系

> 本章核心问题：**CPU如何与外部设备高效地交换数据？**
>
> 首先，由于外设种类繁多、速度各异、信号格式不同，CPU不能直接连接外设，必须通过**接口**（Interface）进行桥接。接口内部有若干**端口**（寄存器），CPU通过读写端口来与外设通信。
>
> 其次，接口的数据传送有四种控制方式，效率依次递增：**无条件传送→查询方式→中断方式→DMA方式**，速度要求越高、数据量越大，就选用越高级的方式。
>
> 最后，本章重点介绍两颗核心芯片：**8259A**（可编程中断控制器，管理中断优先级）和**8237A**（DMA控制器，实现高速数据直传）。

## 2. 核心知识点精讲

### 2.1 接口与端口的区别

**核心定义**：

- **接口（Interface）**：CPU与外部设备之间的连接电路及控制软件的总称，是一套硬件+软件的综合体。
- **端口（Port）**：接口电路内部，可由程序寻址并进行读写操作的**寄存器**。

**原理解析**：
```
                   ┌────────────────────────┐
                   │        I/O 接口         │
CPU ─── 数据总线 ──→│  数据输出寄存器（端口）  │──→ 外设
CPU ←── 数据总线 ──│  数据输入寄存器（端口）  │←── 外设
        地址总线 ──→│  地址译码/选择逻辑      │
        控制总线 ──→│  状态寄存器（端口）      │←── 外设状态
                   │  控制寄存器（端口）      │──→ 外设控制
                   └────────────────────────┘
```

**考试写法**：
> 接口是CPU与外部设备之间的硬件连接电路和控制软件的综合体；端口是接口电路中可被程序寻址并进行读写操作的寄存器。两者关系：一个接口内部包含多个端口（通常有数据口、状态口、控制口）。

⚠️ 考点：简答题 | ⭐⭐⭐  
❌ 易错：不要将接口和端口混用，接口是整体概念，端口是接口内部的寄存器。

---

### 2.2 接口的4大功能

**考试写法**（四点，每点须点到关键词）：

1. **地址译码/设备选择**：对CPU发出的地址进行译码，产生设备选择信号，使CPU与指定外设通信。
2. **数据缓冲和锁存**：数据总线是公用双向总线，各设备分时复用，接口提供缓冲/锁存，避免长期占用总线。
3. **信息格式与电平转换**：完成并行↔串行、数字↔模拟、TTL电平↔其他电平的转换。
4. **数据传送的协调**：采用查询、中断、DMA等方式协调CPU与外设的速度差异。

⚠️ 考点：简答题 | ⭐⭐⭐

### 2.3 I/O接口一般编程结构 📝

**画图要点**（必须标注的元素）：

```
        ┌─────────────────────────────────────────┐
数据总线 │       ┌─────────────────┐ 数据          │
─────→  │       │ 数据输出寄存器   │──────────────→│→ I/O设备
←─────  │       ├─────────────────┤               │
地址总线 │地址   │ 数据输入寄存器   │←─────────────│← I/O设备
─────→  │译码   ├─────────────────┤ 状态          │
        │选择   │ 状态寄存器       │←─────────────│← I/O设备
控制总线 │逻辑   ├─────────────────┤ 控制          │
─────→  │       │ 控制寄存器       │──────────────→│→ I/O设备

```

从用户角度看，接口有4个寄存器（端口）：数据输出、数据输入、状态、控制。  
接口由三部分组成：**寄存器组、地址译码与控制逻辑、连接线**。

### 2.4 四种数据传送控制方式

**核心对比表**：

| 方式 | 原理 | CPU效率 | 适用场合 |
|------|------|---------|---------|
| 无条件传送 | CPU直接IN/OUT，默认外设随时准备好 | 最低（串行） | 外设始终就绪（如LED等简单外设） |
| 查询（轮询）方式 | CPU不断读状态端口，等待外设就绪 | 低（CPU忙等待） | 外设速度慢，实时性要求不高，数据量不大 |
| 程序中断方式 | 外设就绪后主动发INTR，CPU响应后执行ISR | 较高（CPU与外设可并行） | 外设速度较慢，请求随机，响应实时性要求较高 |
| DMA方式 | 硬件（DMAC）接管总线，数据直接在内存与外设间传输，不经过CPU | 最高 | 外设速度快，一次传输数据量大（磁盘、网卡等） |

**考试写法（查询方式程序结构）**：
```asm
A1: IN  AL, 状态端口地址    ; 读状态字
    TEST AL, 状态位掩码     ; 测试状态位
    JZ  A1                  ; 未就绪则继续查询
    IN  AL, 数据端口地址    ; 就绪则读入数据
    ……                      ; 处理数据
    JMP A1                  ; 处理下一个数据
```

⚠️ 考点：简答题 | ⭐⭐⭐ 🔥 高频  
❌ 易错：DMA方式不是完全不需要CPU，初始化阶段仍需CPU；只是传输过程不经过CPU寄存器。

### 2.5 8259A 可编程中断控制器

#### 2.5.1 8259A主要功能

**考试写法**：
1. 每片管理**8级**优先权中断源，级联最多管理**64级**。
2. 可单独对每级进行屏蔽。
3. 向CPU提供可编程的**中断类型码**。
4. 有**10种**工作方式，可通过编程选择。
5. 可与8086～Pentium CPU**直接连接**，不需外加硬件电路。

#### 2.5.2 8259A内部关键寄存器

```
IR0(最高优先级)
IR1           ┌─────┐   ┌────┐   ┌─────┐
IR2    ──────→│ IRR │──→│ PR │──→│ ISR │──→ INT（送CPU）
IR3           └─────┘   └────┘   └─────┘
IR4                                 ↑↓
IR5           ┌──────────────────┐
IR6    ←──────│ IMR（中断屏蔽）   │
IR7(最低优先级)└──────────────────┘
```

| 寄存器 | 名称 | 功能 |
|--------|------|------|
| **IRR** | 中断请求寄存器 | 寄存所有IR输入的中断请求，对应位置1表示该级有请求 |
| **PR** | 优先级分辨器 | 确定IRR中各位的优先等级，决定能否向CPU申请中断 |
| **ISR** | 中断服务寄存器 | 保存当前正在被CPU服务的中断级，对应位置1 |
| **IMR** | 中断屏蔽寄存器 | 某位=1则对应中断被屏蔽；即OCW1 |

⚠️ 考点：简答题 | ⭐⭐⭐ 🔥 高频  

#### 2.5.3 8259A响应中断过程（两个INTA脉冲）

**考试写法**（必须按步骤写）：

CPU响应中断时，向8259A发出**两个INTA负脉冲**：

**第一个INTA负脉冲到达时**，8259A完成：
1. 使ISR中对应位**置1**（记录正在服务的中断）。
2. 使IRR中对应位**清0**（撤销中断请求锁存）。

**第二个INTA负脉冲到达时**，8259A完成：
1. 将**ICW2的内容**送到D7-D0，CPU读取此值作为**中断类型码**。
2. 若ICW4的AEOI位=1（自动结束方式），则在第二个脉冲结束时将ISR对应位**清0**。

⚠️ 考点：简答题 | ⭐⭐⭐ 🔥 高频  

#### 2.5.4 8259A中断类型码的形成

**关键规则**：
- ICW2的高5位（T7-T3）由程序设定。
- 低3位由8259A根据引脚编号**自动填入**：IR0→000，IR1→001，...，IR7→111。

$$中断类型码 = ICW2_{高5位} \| IR引脚编号_{低3位}$$

**示例**：若ICW2=20H（0010_0000B），IR1有中断请求：
- 中断类型码 = 0010_0001B = **21H**
- 中断向量表地址 = 4×21H = **84H**（在0000:0000段）

⚠️ 考点：计算题/简答题 | ⭐⭐⭐ 🔥 高频  

#### 2.5.5 8259A初始化流程（ICW写入顺序）

```
┌──────────────────────────────────────────┐
│  8259A初始化流程                          │
│                                          │
│  写ICW1 (偶地址) ← D4=1是特征标志位       │
│      ↓                                   │
│  写ICW2 (奇地址) ← 设置中断类型码高5位    │
│      ↓                                   │
│  级联方式? ─→ Y → 写ICW3 (奇地址)        │
│      ↓N         ↓                        │
│  需要ICW4? ─→ Y → 写ICW4 (奇地址)       │
│      ↓N         ↓                        │
│  初始化完成                               │
└──────────────────────────────────────────┘
```

**注意**：
- ICW1和ICW2是**必须设置**的。
- ICW3（级联配置）和ICW4（工作方式）根据需要选择。
- ICW使用**奇地址**（除ICW1用偶地址）。
- ICW写入必须**严格按顺序**。

⚠️ 考点：综合题/简答题 | ⭐⭐⭐ 🔥 高频  

#### 2.5.6 各ICW格式要点

**ICW1（偶地址，A0=0）**：

| 位 | 名称 | 功能 |
|----|------|------|
| D4 | — | **恒为1**（ICW1特征标志位） |
| D3 | LTIM | 触发方式：1=电平触发，0=边沿触发 |
| D1 | SNGL | 1=单片，0=级联方式 |
| D0 | IC4 | 1=需要ICW4，0=不需要 |

**ICW2（奇地址，A0=1）**：高5位(T7-T3)设置中断类型码的高5位，低3位自动填入。

**ICW3（奇地址，仅级联）**：
- 主片：S7-S0各位对应IRx，=1表示该引脚接有从片。
- 从片：D2-D0 = 从片INT连接到主片的哪个IRx编号。

**ICW4（奇地址，A0=1）**：

| 位 | 名称 | 功能 |
|----|------|------|
| D4 | SFNM | 1=特殊全嵌套方式，0=全嵌套方式 |
| D3 | BUF | 1=缓冲方式，0=非缓冲方式 |
| D2 | M/S | 缓冲方式下：1=主片，0=从片 |
| D1 | AEOI | 1=自动中断结束，0=正常中断结束 |
| D0 | μPM | 1=8086~Pentium CPU，0=其他 |

#### 2.5.7 8259A工作方式（10种，考试重点4种）

1. **全嵌套方式**（默认）：IR0优先级最高，IR7最低，固定不变。ISR置位期间不响应同级及低级中断，可响应更高级中断。
2. **循环优先级方式**：某级服务后自动降为最低优先级，其后续级变为最高，实现轮流服务（公平调度）。
3. **特殊屏蔽方式**：所有未被屏蔽的中断（无论优先级高低）都可申请并得到服务。
4. **多片级联方式**：一个主片最多带8个从片，可扩展至64级；非缓冲方式下SP=1是主片，SP=0是从片。

❌ 易错：特殊屏蔽方式**不是**只允许高优先级，而是允许所有未屏蔽级（包括低优先级）都可中断。

#### 2.5.8 8259A中断结束方式（3种）

1. **自动中断结束（AEOI）**：第二个INTA脉冲后沿自动将ISR对应位清0。
2. **非特殊EOI命令**：全嵌套方式下用，清除ISR中优先级**最高的非零位**。
3. **特殊EOI命令**：非全嵌套方式下用，通过L2-L0指定要清除的ISR位。

### 2.6 8237A DMA控制器

#### 2.6.1 8237A主要特点

**考试写法**（至少说出以下要点）：
- 一片8237A内有**4个独立的DMA通道**。
- 每通道DMA请求可分别编程允许或禁止。
- 优先级有两种：**固定优先级**（通道0最高，通道3最低）和**循环优先级**。
- 可在外设与存储器、**存储器与存储器**之间传送数据。
- 四种工作（传送）方式：单字节、数据块、请求、级连。
- DMA操作**结束**有两种方法：①字节计数器减1由0变为FFFFH；②外界通过EOP输入负脉冲强制结束。
- DMA操作**启动**有两种方法：①外设输入DREQ；②软件编程内部启动。

#### 2.6.2 8237A四种传送方式

| 方式 | 特点 |
|------|------|
| **单字节传送方式** | 每次获得总线控制权后仅传送**1个字节**，之后释放总线 |
| **数据块传送方式** | 获得总线控制权后**连续传完整个数据块**，才归还总线 |
| **请求传送方式** | 类似数据块传送，但每传1字节后检测DREQ，DREQ无效则暂停 |
| **级连方式** | 将从片的HRQ/HLDA分别接主片通道的DREQ/DACK，扩展通道数 |

#### 2.6.3 8237A三种传送类型

1. **DMA读**：8237A输出有效的 $\overline{MEMR}$ 和 $\overline{IOW}$，把**存储器→I/O设备**。
2. **DMA写**：8237A输出有效的 $\overline{IOR}$ 和 $\overline{MEMW}$，把**I/O设备→存储器**。
3. **DMA校验**：伪传输，控制线无效，禁止实际传送，用于检验芯片内部读写功能。

⚠️ 考点：简答题 | ⭐⭐⭐  
❌ 易错：DMA读是"从存储器读到外设"，DMA写是"从外设写到存储器"，方向容易搞反。

## 3. 高频计算题型与模板

### 3.1 中断类型码与中断向量表地址计算

**公式**：

$$中断类型码\ n = ICW2_{高5位}\ |\ IR引脚编号$$

$$中断向量表地址 = 4 \times n$$

**解题步骤**：
1. 写出ICW2的值，取高5位（D7-D3）。
2. 确定哪个IR引脚有中断请求，取其编号（0-7）作为低3位。
3. 拼合得中断类型码n。
4. 计算4n，得到中断向量表中ISR入口地址存放的位置（在段0000H）。

**示例**（练习题第79题）：  
题目：ICW2写入20H，IR1连接中断源，ISR_1从4000H:5678H开始。说明从中断源到中断向量表到中断服务程序的对应关系。

Step 1：ICW2=20H=0010_0000B，高5位=00100  
Step 2：IR1引脚编号=001  
Step 3：中断类型码 n = 0010_0001B = **21H**  
Step 4：中断向量表地址 = 4×21H = **0084H**（即0000:0084H处存放ISR_1的CS:IP）  
Step 5：0000:0084H存放偏移78H，0000:0085H存放56H，0000:0086H存放00H，0000:0087H存放40H  
答：CPU取得中断类型码21H，到0000:0084H读出入口地址4000H:5678H，跳转执行ISR_1。

❌ 易错：中断向量表地址是 4×n（字节），而不是n或2×n。每个向量占4字节（IP低2字节+CS高2字节，小端存储）。

### 3.2 8253计数初值计算（与第7章相关，此处也可出现）

**公式**：

$$N = \frac{f_{CLK}}{f_{OUT}}$$

**解题步骤**：
1. 确定工作方式（输出方波用**方式3**，输出脉冲用**方式2**等）。
2. 由 $N = f_{CLK} / f_{OUT}$ 计算计数初值。
3. 写控制字（通道号+读写方式+工作方式+计数制）。
4. 写初值低8位，再写高8位。

**示例**（练习题第91题之通道0）：  
题目：通道0，方式3，CLK0=2MHz，输出1KHz方波。

Step 1：方式3（方波发生器）✓  
Step 2：$N = 2\text{MHz} / 1\text{KHz} = 2000$  
Step 3：控制字 = 00_11_011_0B = 36H（通道0，先低后高，方式3，二进制）  
Step 4：写端口43H←36H；写端口40H←D0H（低8位）；写端口40H←07H（高8位，2000=07D0H）

### 题目51（练习题第51题）

**题目**：CPU与外部设备之间为什么要使用接口？

**解答**：
外部设备种类繁多，与CPU不能直接互联，主要原因有四点：
1. 外设信号类型不同（数字/模拟/开关量），与CPU使用的数据类型不同。
2. 外设工作速度变化范围大，与CPU速度不匹配。
3. 外设信息格式复杂（并行/串行），与CPU数据格式不一致。
4. 外设种类繁多，控制方式各异，需要统一的接口标准协调。

**考察点**：接口存在的必要性。

### 题目52（练习题第52题）

**题目**：什么是I/O接口？什么是I/O端口？

**解答**：
- **I/O接口**：位于I/O设备与CPU之间的电路，是完成CPU与外设连接任务的软件和硬件的综合体。
- **I/O端口**：接口电路中那些完成信息传送、可由程序寻址并进行读写操作的**寄存器**。

两者关系：接口是整个连接机构，端口是接口内部可编程访问的寄存器。

**考察点**：接口与端口的区别（高频简答）。

### 题目58（练习题第58题）

**题目**：CPU与外设之间的数据传送控制方式。

**解答**：
1. **程序查询方式**：CPU通过查询I/O设备的状态，判断是否就绪，就绪则进行数据传送，否则继续查询。CPU全程占用，效率低。
2. **程序中断方式**：I/O设备就绪时主动发INTR，CPU中断当前程序，转入中断服务程序完成数据传送，服务完毕返回。CPU与外设可并行工作，效率较高。
3. **DMA方式**：在DMA控制器的管理下，I/O设备和存储器直接交换数据，不需CPU介入。速度最快，适合大批量高速数据传输。
4. **I/O处理机方式**：引入专用I/O处理机，全部I/O操作由I/O处理机独立承担，与CPU完全并行，适合复杂系统。

**考察点**：四种传送方式（核心考点）。

### 题目70（练习题第70题）

**题目**：说明8259A中断控制器中IRR、ISR和IMR三个寄存器的功能。

**解答**：
- **IRR（中断请求寄存器）**：保存所有IR输入线的中断请求信号。接收到某引脚的中断请求后，IRR对应位置1（锁存）。
- **ISR（中断服务寄存器）**：保存当前正在被CPU服务的中断级。CPU响应中断时，对应位置1，一直保持到收到中断结束命令（EOI）后清0。
- **IMR（中断屏蔽寄存器）**：保存中断屏蔽字。某位=1，对应中断请求被屏蔽，无法传递给CPU；某位=0，对应中断被允许。

**考察点**：8259A内部三个寄存器功能区分（必考简答）。

### 题目79（练习题第79题）

**题目**：ICW2写入20H，IR1连接中断源ISR_1（从4000H:5678H开始）。画图说明从中断源到中断向量表到中断服务程序的对应关系。

**解答**：
1. IR1有中断请求 → 8259A向CPU发INT信号。
2. CPU响应，发两个INTA：
   - 第一个INTA：ISR[1]置1，IRR[1]清0。
   - 第二个INTA：8259A将中断类型码送数据总线。
3. 中断类型码：ICW2=20H（高5位=00100），IR1引脚编号=001，类型码=0010_0001B=**21H**。
4. CPU用4×21H=84H，到**0000:0084H**取中断向量：
   - [0000:0084H]=78H，[0000:0085H]=56H（IP=5678H）
   - [0000:0086H]=00H，[0000:0087H]=40H（CS=4000H）
5. CPU跳转到**4000H:5678H**执行ISR_1。

**考察点**：中断类型码形成+中断向量表查找全过程（综合考点）。

### 题目80（练习题第80题）

**题目**：8259A响应中断过程中会连续执行两个INTA中断响应周期，说明每个周期的功能是什么？

**解答**：  
**第一个INTA周期**：
1. 使IRR的锁存功能失效（准备接收新中断请求）。
2. 使当前中断服务寄存器ISR的**相应位置1**。
3. 使IRR寄存器中**对应位清0**。

**第二个INTA周期**：
1. 将**ICW2**的内容送到数据总线D7-D0，CPU读取作为中断类型码。
2. 若ICW4的AEOI位=1，在第二个INTA结束时，自动将ISR相应位**清0**（自动结束中断服务）。

**考察点**：8259A中断响应两个INTA周期的具体动作（高频简答）。

### 题目81（练习题第81题）

**题目**：8259A的中断结束是指什么？简要说明8259A的3种中断结束方式的特点。

**解答**：
中断结束是指将**ISR中被响应中断的对应位清0**的操作，这样8259A才能再次响应同级或低级中断。

三种中断结束方式：
1. **自动中断结束（AEOI）**：第二个INTA响应信号的后沿，8259A自动清除ISRx位，无需CPU发EOI命令。
2. **非特殊EOI命令**：用于全嵌套方式。CPU发出非特殊EOI命令，8259A将ISR中**优先级最高的非零位**清0。
3. **特殊EOI命令**：用于非全嵌套方式。命令中含L2-L0编码，指定清除ISR中哪一位，可精确结束任意优先级的中断服务。

**考察点**：中断结束方式（ISR清0机制）。

### 题目60/64（练习题第60、64题）

**题目**：说明8237A四种基本传送方式的特点。

**解答**：

①**单字节传送方式**：8237A每次获得总线控制权后仅传送1个字节，传送完毕立即释放总线，若还有DMA请求则重新申请。

②**数据块传送方式**：8237A一旦获得总线控制权，将连续传送完一整个数据块（直到计数器减为0或收到EOP），才归还总线给CPU。

③**请求传送方式**：类似数据块传送，但每传1字节后都采样DREQ信号；若DREQ无效则暂停传送，释放总线，等DREQ再次有效时继续。三种情况停止：DREQ无效、计数器减至0(TC)、外部EOP信号。

④**级联传送方式**：允许多片8237A级联扩展通道。将从片的HRQ和HLDA分别接到主片某通道的DREQ和DACK，主片该通道只做优先级和信号转接，不做实际DMA传送。

**考察点**：8237A四种传送方式（高频）。

### 题目61（练习题第61题）

**题目**：说明8237A三种传送类型。

**解答**：
1. **DMA读**：8237A输出有效的 $\overline{MEMR}$ 和 $\overline{IOW}$，将**存储器**中的数据传送到**I/O设备**（存储器→外设）。
2. **DMA写**：8237A输出有效的 $\overline{IOR}$ 和 $\overline{MEMW}$，将**I/O设备**的数据传送到**存储器**（外设→存储器）。
3. **DMA校验**：伪传输，8237A产生地址信号等操作，但存储器和I/O设备的控制线均无效，不进行实际数据传送，用于检验芯片内部读写功能。

**考察点**：三种传送类型的方向和控制信号。

## 5. 简答题备考模板

### [简答1] 接口与端口的区别

**规范答法**：
> 接口是CPU与外部设备之间的硬件连接电路和控制软件的综合体，负责完成CPU与外设之间信息传送的协调工作。端口是接口电路中那些可由程序寻址并进行读写操作的寄存器。两者关系：一个接口内通常包含数据端口、状态端口、控制端口等多个端口。

---

### [简答2] 四种数据传送控制方式的特点及适用场合

**规范答法**（考试按表格或分点）：

① **无条件传送方式**：外设随时准备好，CPU直接执行IN/OUT，适用于简单、始终就绪的外设（如LED、开关）。

② **程序查询方式**：CPU不断读取状态端口，判断外设是否就绪，就绪再传输。适合外设速度慢、实时性要求不高、数据量不大的场合，缺点是CPU利用率低。

③ **程序中断方式**：外设就绪后主动发INTR，CPU中断当前程序转入ISR完成数据传送，服务完毕返回。CPU与外设可并行工作，适合响应实时性要求较高、数据量不大的场合。

④ **DMA方式**：由DMA控制器接管总线，数据直接在外设与存储器间传输，不经CPU。适合外设速度快、一次传输数据量大的场合（如磁盘、高速A/D采集）。

### [简答3] 8259A初始化编程过程 📝

**规范答法**：
8259A初始化必须按固定顺序写入ICW，步骤如下：

1. 向**偶地址**写入**ICW1**（D4=1为特征标志），设置触发方式和是否级联。
2. 向**奇地址**写入**ICW2**，设置中断类型码的高5位（T7-T3）。
3. 若为级联方式（ICW1的SNGL=0），向奇地址写入**ICW3**（主片记录从片连接位，从片记录其在主片的连接编号）。
4. 若ICW1的IC4=1，向奇地址写入**ICW4**，设置CPU类型（μPM=1表示8086）、中断结束方式、缓冲方式等。
5. 初始化完成后，可根据需要写入操作命令字OCW（OCW1设置屏蔽字，OCW2发EOI命令，OCW3设查询/读状态方式）。

### [简答4] 8259A两个INTA周期的功能 

**规范答法**：
CPU响应中断时，发出两个INTA负脉冲：

**第一个INTA**：8259A将ISR中相应位**置1**，同时将IRR中相应位**清0**（取消中断请求锁存）。

**第二个INTA**：8259A将ICW2内容送数据总线，CPU将此值作为**中断类型码**；若设置了自动结束方式（AEOI=1），则同时将ISR相应位**清0**。

### [简答5] 8237A四种传送方式与三种传送类型

**规范答法（传送方式）**：
- **单字节方式**：每次传1字节，立即释放总线。
- **数据块方式**：一次连续传完整个数据块。
- **请求方式**：连续传输但每字节后检查DREQ，无效则暂停。
- **级联方式**：扩展通道数，不做实际传输。

**规范答法（传送类型）**：
- **DMA读**：存储器→I/O设备（MEMR+IOW有效）。
- **DMA写**：I/O设备→存储器（IOR+MEMW有效）。
- **DMA校验**：伪传输，控制线无效，仅验证芯片功能。

## 6. 易错点汇总

- ❌ 易错：将接口和端口混淆，认为端口=接口。  
  ✅ 正确：接口是整体硬件+软件，端口只是接口内部的寄存器（可寻址的那部分）。

- ❌ 易错：认为DMA方式完全不需要CPU参与。  
  ✅ 正确：DMA**初始化阶段**仍需CPU编程设置参数，只有**数据传送过程**不经过CPU寄存器。

- ❌ 易错：中断向量表地址=中断类型码×2。  
  ✅ 正确：中断向量表地址=中断类型码×**4**（每个向量占4字节：IP占2字节+CS占2字节）。

- ❌ 易错：认为ICW2直接就是中断类型码。  
  ✅ 正确：ICW2的**高5位**是类型码的高5位，**低3位**由8259A根据IRx引脚编号自动填入，合并后才是中断类型码。

- ❌ 易错：搞混DMA读和DMA写的方向。  
  ✅ 正确："DMA读"是从存储器读到外设（存储器→外设）；"DMA写"是从外设写到存储器（外设→存储器）。以**存储器**为基准，"读"指读存储器送出。

- ❌ 易错：认为8259A中IMR某位=1表示允许该中断。  
  ✅ 正确：IMR某位=**1**表示**屏蔽**该中断；=0表示允许。

- ❌ 易错：ICW写入顺序混乱，ICW1写到奇地址。  
  ✅ 正确：ICW1写到**偶地址**（A0=0）；ICW2、ICW3、ICW4写到**奇地址**（A0=1）。

- ❌ 易错：全嵌套方式下，认为正在服务一个中断时，任何中断都不能进来。  
  ✅ 正确：全嵌套方式下，同级和低级中断被禁止，但**更高优先级**的中断仍可响应（若CPU开放中断IF=1）。

- ❌ 易错：8237A级联方式时认为主片的那个通道也参与数据传输。  
  ✅ 正确：级联方式下，主片连接从片的那个通道**只做优先级管理和信号转接**，不进行实际DMA数据传送。

## 7. 一页纸速记

**接口 vs 端口**：接口=整体（硬件+软件）；端口=接口内可寻址的寄存器。

**四种传送方式（效率从低到高）**：
```
无条件传送 < 查询方式 < 中断方式 < DMA方式
（外设速度快、数据量大→选更高级方式）
```

**8259A必背**：
- 三寄存器：IRR（请求锁存）/ ISR（正在服务）/ IMR（屏蔽，=1屏蔽）
- 类型码公式：`n = ICW2高5位 | IRx引脚编号低3位`
- 向量表地址：`4 × n`
- 两个INTA：第1个→ISR置1+IRR清0；第2个→送类型码（+AEOI则ISR清0）
- ICW写入顺序：**ICW1（偶）→ICW2（奇）→ICW3（奇，级联时）→ICW4（奇，需要时）**

**8237A必背**：
- 4通道，固定优先级（通道0最高）
- 4种传送方式：单字节/数据块/请求/级连
- 3种传送类型：DMA读（存储器→外设）/ DMA写（外设→存储器）/ DMA校验（伪传输）
- 结束方法：计数器减为0（TC）或外部EOP
- 启动方法：DREQ外部请求 或 软件内部启动

**最易混淆的区别**：
- IRR vs ISR：IRR是"排队等待"，ISR是"正在服务"
- DMA读 vs DMA写：以存储器为基准，读=从存储器取出送外设；写=从外设写入存储器
- ICW vs OCW：ICW是初始化（开机设一次，顺序严格）；OCW是操作命令（随时可写，不限顺序）
