# 第8章 模拟接口 · 自学理解手册

> **这份资料的目标**：不是让你背公式，而是让你理解"为什么计算机需要一套专门的电路才能和模拟世界对话"，以及 DAC 和 ADC 这两类器件在做同一件事的正反两面。读完之后，你应该能在脑子里建立一张从"传感器测出一个温度"到"电机转速被控制"的完整信号流图，理解每一个环节存在的理由。

## 分层知识树

```
第8章 模拟接口
│
├─ 【第一层】回答"数字系统为什么要处理模拟信号"的问题
│   ├─ 物理世界的量（温度、压力、湿度）都是连续变化的模拟量
│   ├─ CPU 只能处理离散的二进制数字量
│   ├─ 传感器将非电量转为电压/电流，仍是模拟量
│   └─ 必须有 ADC（模→数）和 DAC（数→模）才能形成完整的闭环系统
│
├─ 【第二层】回答"转换器能做到多精细、多快"的问题
│   ├─ 分辨率 = FS/2ⁿ，位数n越大分辨率越高（DAC0832为8位，DAC1210/AD574为12位）
│   ├─ DAC 建立时间：输出模拟量稳定到 ±ε 范围所需时间（DAC0832约1μs）
│   ├─ ADC 转换时间：从启动到数字量稳定输出的时间（ADC0809在640kHz时约100μs）
│   └─ 核心数学关系：Vout/Vref = N/2ⁿ（D/A 正向），Vin/Vref = N/2ⁿ（A/D 反向）
│
├─ 【第三层】回答"DAC0832 怎么把一个数变成电压"的问题
│   ├─ R-2R T型电阻网络：每一节对应数字量的一位，权值递减（2⁷, 2⁶, ..., 2⁰）
│   ├─ 各节电流叠加得到 Iout = (Vref/2ⁿR) × N，运放将电流转为电压
│   ├─ 内部双缓冲结构：输入寄存器（WR1/CS/ILE 控制）+ DAC寄存器（WR2/XFER 控制）
│   ├─ 三种控制方式：直通（两级锁存直通）、单缓冲（一级锁存）、双缓冲（分两步写再同步）
│   └─ 双缓冲的价值：多片 DAC 可先各自写数，再用同一个 XFER 信号同步输出
│
├─ 【第四层】回答"ADC0809 怎么把电压变成一个数"的问题
│   ├─ A/D 转换四步：采样→保持→量化→编码
│   ├─ ADC0809 采用逐次逼近原理：8位SAR寄存器从高位到低位逐次比较确定
│   ├─ 关键引脚：START（启动正脉冲）、ALE（通道地址锁存上升沿）、EOC（低=转换中，高=结束）、OE（读允许）
│   ├─ 8路模拟输入：ADDA/ADDB/ADDC 三位地址选择 IN0~IN7 中的一路
│   └─ 与 CPU 的接口方式：等待（软件延时100μs）、查询（轮询EOC）、中断（EOC触发）
│
├─ 【第五层】回答"采样定理为什么规定了采样频率的下界"的问题
│   ├─ 采样是将连续时间信号离散化，采样间隔 T 决定能还原的最高频率
│   ├─ 采样定理：fs ≥ 2·fmax，否则高频分量折叠到低频，产生混叠失真
│   ├─ 采样保持器的作用：在ADC转换期间（约100μs）固定住采样值，防止孔径误差
│   └─ 可省略采样保持器的条件：信号变化速度相对于转换速度足够慢，转换期间可视为直流
│
└─ 【第六层】回答"ADC+DAC 如何构成完整的闭环控制系统"的问题
    ├─ 信号链：传感器→多路开关→ADC0809→CPU（含8255A/8253）→DAC0832→执行机构
    ├─ 8253 定时器提供周期性采样触发（如每20ms产生一次中断）
    ├─ CPU 完成控制算法（比较测量值与设定值，计算控制量）
    ├─ DAC0832 输出控制电压（0~5V 单极性，或双极性），驱动执行机构
    └─ 闭环的含义：输出结果通过传感器再次被采集，形成反馈回路
```

各层之间有严格的依赖关系：第一层是存在理由，第二层定量描述性能边界，第三、四层分别是正向（数→模）和反向（模→数）的具体实现，第五层补充采样的时间域约束，第六层把所有器件串联成一个有实际用途的系统。学完第六层之后，综合设计题（温度控制、湿度控制）里出现的任何芯片组合都不会再觉得陌生。

## 第一部分：先问一个问题——CPU 为什么不能直接读温度？

想象你在做一个恒温箱控制系统。箱子里有个温度传感器，输出的是一根随温度变化的模拟电压：20°C 时输出 1V，100°C 时输出 5V。CPU 能直接把这根线接到数据总线吗？

当然不能。数据总线只认 0 和 1，而电压是连续变化的，没有一个"格子"可以精确地把 2.357V 放进去。这就是模拟接口存在的根本原因：**物理世界是连续的，计算机是离散的，两者之间需要翻译器**。

ADC（模数转换器）是从模拟世界到数字世界的翻译器，把连续的电压值量化为一个整数。DAC（数模转换器）是反方向的翻译器，把 CPU 计算出的控制指令还原为能驱动电机、加热器的模拟电压。一个完整的嵌入式控制系统，一定同时需要这两类器件，才能形成"感知→计算→执行"的闭环。

传感器是这一切的起点。它把非电量（温度、压力、湿度、光强）转换为电压或电流，精度直接决定整个系统的上限。多路开关（如 CD4051）让一片 ADC 可以轮流采集多路传感器的信号，节省硬件成本。

## 第二部分：分辨率——精度的数学语言

在讨论任何具体芯片之前，有一个数学关系必须先刻进脑子里：

$$\frac{V}{V_{ref}} = \frac{N}{2^n}$$

这个式子里，$V$ 是模拟电压，$V_{ref}$ 是参考电压，$N$ 是对应的数字量，$n$ 是转换器的位数。它是 D/A 和 A/D 共用的核心公式，D/A 题正着用（已知 N 求 V），A/D 题反着用（已知 V 求 N）。

**分辨率**是这个公式的直接推论：当 N 变化1个最低有效位（LSB）时，对应模拟量的变化就是 $V_{ref}/2^n$。位数越多，能分辨的最小变化越小，精度越高。

```
8位 DAC0832：  分辨率 = 5V/256 ≈ 19.5mV（每步）
12位 DAC1210： 分辨率 = 5V/4096 ≈ 1.2mV（每步）
```

12位的精度约为8位的16倍。当控制精度要求高时（如精密仪器），必须用12位器件。这就是为什么课件同时介绍 DAC0832（8位）和 DAC1210（12位）、ADC0809（8位）和 AD574（12位）——它们两两对应，精度档次不同，实际选型看需求。

## 第三部分：DAC0832——怎么把一个数"画"成电压

D/A 转换的本质是**加权求和**。数字量的每一位都有一个权值（最高位权值最大，最低位权值最小），把各位对应的电流按权值加起来，就得到与数字量成比例的总电流，再经运算放大器转为电压输出。

DAC0832 采用的是 R-2R T型电阻网络，整个电路只用两种电阻：R 和 2R。它的工作原理像一棵二叉分流树——

```
VREF ──R──┬──R──┬──R──┬── ... ──R──┬── IOUT
          2R    2R    2R            2R
          |     |     |              |
         D7    D6    D5             D0
        (开关控制每一位是否接地)
```

D7（最高位）离 VREF 最近，流过它的电流最大（权值 2⁷）；D0（最低位）离得最远，电流最小（权值 2⁰）。当某位为1时，对应开关接 IOUT；为0时接地。所有为1的位的电流叠加就是 IOUT，然后运放把电流转成电压。

**三种控制方式的直觉理解：**

想象一个厨房里有两个传菜窗口（两级锁存），菜（数字量）从厨房（CPU数据总线）出来必须经过两个窗口才能上桌（开始转换）。

- **直通方式**：两个窗口都常开，菜直接穿过，CPU 写什么立刻输出。适合数字量不变的静态场合。
- **单缓冲方式**：只用第一个窗口作为暂存，CPU 写入后立刻触发转换。适合单路输出。
- **双缓冲方式**：先把菜放进第一个窗口（各片 DAC 各自锁存），等所有菜都准备好了，一声令下（XFER信号）同时打开第二个窗口，所有菜同时上桌。这就是**多路模拟量同步输出**的秘密——即使多片DAC分时写入，输出却是同步的。

## 第四部分：ADC0809——怎么把电压"猜"成一个数

A/D 转换比 D/A 转换更有意思，因为它本质上是一个**猜数游戏**（逐次逼近法）。

想象你要猜一个0~255之间的整数，每次猜完我只告诉你"大了"还是"小了"。最优策略是二分搜索：先猜128，大了就在0~127里猜，小了就在129~255里猜，依此类推，8次就能精确确定任意整数。

逐次逼近式 ADC 就是这么工作的：

```
第1次：试探 Vref/2（对应数字量10000000B）
         → 比较器判断 Vin > 还是 < 试探值
         → 该位确定为1或0
第2次：试探 Vref/4 叠加在前一次基础上
         → 再次比较，确定次高位
...
第8次：确定最低位D0
→ 8位数字量全部确定，转换结束，EOC变高
```

**ADC0809 的操作时序**是一个固定的三步动作，必须严格按顺序来：

```
① 写 START（同时 ALE 有效，锁存通道地址）
        ↓
   EOC 信号变低（表示"正在转换，请等待"）
        ↓
② 等待 EOC 变高（转换完成）
   ·等待方式：软件延时≥100μs
   ·查询方式：循环读 EOC，直到为高
   ·中断方式：EOC 上升沿触发中断
        ↓
③ 输出 OE 有效（高电平），读取数据总线上的转换结果
```

这三步的顺序错了就读不到正确结果——尤其是"先写 START 再等 EOC"这一点，写完 START 之后 EOC 不会立刻变高，必须等完整的转换时间。

## 第五部分：采样定理——时间上的精度约束

前面讨论的分辨率是**幅度上**的精度（能分辨多小的电压变化），采样定理规定的是**时间上**的精度（采样多频繁才能还原信号）。

直觉：假设有一个正弦波，你每隔很长时间才采一个点，采到的点连起来可能看起来像一个频率更低的波，根本不像原始信号。这就是**混叠失真**，是无法通过后处理修复的根本性错误。

采样定理告诉我们：只要采样频率不低于信号最高频率的2倍（$f_s \geq 2f_{max}$），就能从采样序列无失真地还原原始信号。低于这个门槛，再好的 ADC 也救不了数据。

```
正弦波（fmax = 1kHz）
  要求采样频率 fs ≥ 2kHz，即至少每0.5ms采一个点

工程实践中通常取 fs = 5~10 × fmax，留出裕量
```

**采样保持器**解决的是另一个问题：ADC 完成转换需要约100μs，在这100μs里如果输入信号还在变化，转换结果就不准确（孔径误差）。采样保持器在采样时刻"拍一张快照"，在转换期间保持这个值不动，给 ADC 一个稳定的输入。

只有信号变化足够慢（比 ADC 转换时间慢得多）的场合，才可以省略采样保持器，直接让模拟信号接入 ADC 输入端。

## 第六部分：闭环系统——把所有器件串起来

现在把前面所有知识整合进一个完整的系统。综合设计题（恒温箱、湿度控制系统）的硬件图看起来复杂，但信号流向是固定的：

```
物理量
  ↓ 传感器（温湿度传感器、压力传感器）
模拟电压
  ↓ 多路开关（可选，多路输入时用 CD4051）
单路模拟电压
  ↓ [采样保持器] → ADC0809（8位，查询方式）
数字量 N（0~255）
  ↓ CPU（8086/8088）
  │  ├── 8255A：扩展I/O，连键盘/显示器
  │  ├── 8253：定时产生采样中断（如每20ms一次）
  │  └── 控制算法：比较N与设定值，计算控制量M
数字量 M（0~255）
  ↓ DAC0832（8位，单缓冲方式）
模拟控制电压（0~5V）
  ↓ 运算放大器（电流→电压转换，可调极性/范围）
执行机构（加热器/电机/加湿器）
  ↓ （效果改变物理量）
物理量（反馈回路开始）
```

**闭环的关键**是"反馈"：执行机构的输出改变了被控物理量，而这个改变又会被传感器重新采集，形成一个不断自我修正的回路。CPU 运行的控制算法（哪怕只是简单的阈值比较：检测值 < 设定值就加热，否则停止）就是让这个回路收敛到目标值的核心逻辑。

真题综合设计题要求画硬件连接图并写初始化程序，本质上就是把这张信号流图"翻译"成芯片引脚连接和初始化控制字代码。信号流向搞清楚了，画图时就不会漏模块或接错方向。

## 本章要带走的五件事

**第一件：核心公式只有一个。**  
$V/V_{ref} = N/2^n$，正向用是 DAC，反向用是 ADC。8位芯片 $2^n = 256$，12位芯片 $2^n = 4096$。所有计算题都是这个公式的变形。

**第二件：DAC0832 的双缓冲是为了多路同步。**  
单片单路用单缓冲够了；多片需要同步输出时，先各自锁存（WR1写输入寄存器），再统一触发（XFER拉低写DAC寄存器），这一刻所有片同时开始转换，输出同步。

**第三件：ADC0809 操作有严格的三步顺序。**  
写 START（启动）→ 等 EOC 变高（不能跳过等待）→ 读数据（OE有效）。查询方式两次判断 EOC：先等它变低（转换开始），再等它变高（转换结束）。

**第四件：采样定理是不可逾越的物理定律。**  
$f_s < 2f_{max}$ 会产生混叠，这是在采集之前就已经犯的错，任何后处理都救不回来。

**第五件：综合设计题的信号流向是固定的。**  
传感器 → ADC0809 → CPU（含8253定时/8255A扩展I/O）→ DAC0832 → 执行机构 → （反馈）。画硬件图时先画这条主干，再填各芯片的控制引脚和译码地址，就不会乱。
