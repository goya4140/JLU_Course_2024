## 1. 知识体系

本章解决的核心问题是：**CPU 如何与用户交互**——用户通过键盘输入信息，CPU 通过显示器输出信息。

键盘部分的逻辑链：硬件结构（线性/矩阵）→ 可靠识别问题（消抖、重键）→ 扫描方式（行扫描法、线反转法）→ 工作方式（程控/定时/中断）→ 系统级接口（BIOS/DOS中断）。

显示器部分的逻辑链：七段显示器物理结构（段 a~h、共阴/共阳）→ 段码编码 → 显示方式（静态/动态）→ 接口设计（配合 8255A）→ 程序设计（查表法 XLAT）。

## 2. 核心知识点精讲

### 2.1 键盘分类：编码键盘 vs 非编码键盘

**核心定义**：
- **编码键盘**：硬件逻辑自动提供 ASCII 码或其他编码，按键后直接输出对应编码。
- **非编码键盘**：仅提供行列矩阵，所有识别工作靠软件完成，灵活性更高。

**考试写法**：非编码键盘必须解决四个问题：① 清除抖动干扰；② 防止重键错误；③ 键盘结构及按键识别；④ 产生按键对应编码。

⚠️ 考点：简答题 | ⭐⭐

---

### 2.2 消抖（Debounce）

**核心定义**：机械按键闭合/断开时，触点在稳定前会反复抖动 **10~20ms**，导致一次按键被读入多次。

**两种解决方法**：

| 方法 | 原理 | 特点 |
|------|------|------|
| 硬件滤波 | 对每个键加 RC 滤波电路或 RS 去抖电路 | 成本高，每键一套电路 |
| 软件延时（常考）| 检测到按键后，延时 **20ms**，再次读取确认 | 成本低，程序复杂度略增 |

**软件消抖流程（必背）**：

```
读端口 → 有键? → 否: 继续轮询
              ↓ 是
         CALL D20ms（延时20ms）
              ↓
         再次读端口 → 还有键? → 否: 是干扰，重新轮询
                           ↓ 是
                      确认为有效按键，继续识别
```

⚠️ 考点：综合题/编程题核心步骤 | ⭐⭐⭐ 🔥 高频
❌ 易错：只判断一次（未做延时后二次确认）就处理按键，会误判抖动为有效按键。

---

### 2.3 重键处理

**核心定义**：两个或以上的键同时按下，或一键未释放另一键又按下。

**三种解决方法**：
1. **忽略**：不理会所有按键，直到只剩一个键按下为止。
2. **缓冲**：所有按键信息存入键盘缓冲区，逐个处理（成本高，简易系统少用）。
3. **n键连锁**：一键按下后，在该键完全释放前，其他键不产生任何代码。

⚠️ 考点：简答题 | ⭐⭐

### 2.4 线性键盘（独立式按键）

**核心定义**：每个按键直接连接一根 I/O 口线，按键按下时该线为低电平（0），弹起为高电平（1）。

**识别方法**：读端口 → AND 屏蔽无关位 → CMP 0FH → 不等于0FH则有键按下 → 逐位比较确定哪个键。

**考试写法（以课件例题为模板）**：
- 8255A A口全部低4位接4个按键 K3~K0
- 控制字 `90H`：A口方式0输入，B/C口输出
- 读入后 `AND AL, 0FH` 屏蔽高4位
- `CMP AL, 0FH`，等于0FH说明无键（所有线为高），不等则有键

```assembly
MOV AL, 90H        ; 控制字：A口方式0输入
OUT 63H, AL        ; 写控制口
A1: IN  AL, 60H    ; 读A口键盘状态
    AND AL, 0FH    ; 取低4位K3~K0
    CMP AL, 0FH
    JZ  A1         ; 全为1，无键，继续查询
    CALL D20ms     ; 有键，延时消抖
    IN  AL, 60H    ; 再次读入
    AND AL, 0FH
    CMP AL, 0FH
    JZ  A1         ; 消抖后为0FH，是干扰，重新查询
    ; ——到此确认有键按下，开始识别具体是哪个键——
    CMP AL, 00001110B  ; 只有K0按下（低4位第0位为0）
    JNZ A2
    MOV AH, 0          ; K0编码=0
    JMP A6
A2: CMP AL, 00001101B  ; 只有K1按下
    JNZ A3
    MOV AH, 1
    JMP A6
; …（K2、K3同理）
A5: MOV AH, 0FFH   ; 多键同时按下，用0FFH标志
A6: NOP
    RET
```

❌ 易错：K0按下时，PA0为0，其余为1，所以读到的是 `xxxx1110B`（低4位为 `1110B`），而不是 `0001B`。按键按下对应该位**变为0**。

### 2.5 矩阵键盘与行扫描法 📝

**核心定义**：行列式键盘，I/O口线分为行线（输出）和列线（输入），按键设置在行列交点。一个 4×4 矩阵键盘仅需 8 根 I/O 线即可支持 16 个键。

**行扫描法原理**（步进扫描）：

```
步骤1：全行输出0（0000B），读列线，判断是否有键按下
步骤2：有键 → CALL D20ms（消抖）→ 再确认
步骤3：逐行扫描：
       第0行输出 11111110B，读列线
       第1行输出 11111101B，读列线
       第2行输出 11111011B，读列线
       第3行输出 11110111B，读列线
       → 某行列线不全为1时，该行有键按下
步骤4：合成位置码 = 行扫描值高4位 OR 列值低4位
步骤5：查TABLE表，得到键号
```

**位置码的构成（重要）**：

```
位置码 = [行号(4位)][列号(4位)]

例：B键（第2行第3列）
  行号 = 1011（第2次ROL后的行扫描值低4位）
  列号 = 0111（列线读入值低4位，第3列为0）
  位置码 = 10110111B
```

**键值表（TABLE）的设计思路**：表中按0~F键的顺序依次存放各键位置码，查表时，找到匹配项的索引（偏移量）即为键号。

**关键代码段**（行扫描核心循环）：

```assembly
MOV AH, 11111110B  ; 行扫描初值，第0行置0
MOV CX, 4          ; 共4行
A2: MOV DX, 200H   ; 行输出端口
    MOV AL, AH
    OUT DX, AL     ; 输出行扫描值
    INC DX         ; 列输入端口201H
    IN  AL, DX     ; 读列线状态
    AND AL, 0FH    ; 取低4位列信号
    CMP AL, 0FH
    JNZ A3         ; 该行有键，转A3处理
    ROL AH, 1      ; 循环左移，扫描下一行
    LOOP A2        ; 4行未完，继续
    MOV AH, 80H    ; 4行全无键（干扰），标志80H
    JMP A6

A3: MOV CL, 4
    SHL AH, CL     ; 行号移到高4位
    OR  AL, AH     ; 行号 OR 列号 = 位置码
    LEA BX, TABLE  ; 指向位置码表
    MOV CL, 0      ; 键号初值0
A4: CMP AL, [BX]   ; 查表
    JZ  A5         ; 找到，CL即为键号
    INC CL
    INC BX
    CMP CL, 10H    ; 查完16项未找到
    JNZ A4
    MOV AH, 0FFH   ; 重键，标志0FFH
    JMP A6
A5: MOV AH, CL     ; 键号存入AH
A6: NOP
    RET
```

📝 画图要点：
- 画出4×4矩阵键盘连接图：行D3~D0连输出锁存器（端口1），列D3~D0连输入缓冲器（端口2）
- 标出+5V上拉电阻（列线通过上拉电阻接+5V，无键时列线为1）
- 行扫描表格：标注每次扫描时行输出值和列读入值

⚠️ 考点：综合题/编程题 | ⭐⭐⭐ 🔥 高频
❌ 易错：ROL 和 SHL 的区别——行扫描用 `ROL AH, 1`（循环移位保持只有一个0），而形成位置码时用 `SHL AH, CL`（逻辑左移4位，将行号移到高4位）。

### 2.6 线反转法

**原理**（两步）：

```
第一步（行输出）：行=输出，行输出全0；列=输入，读入列值（列值中为0的位即为按键所在列）
第二步（线反转）：行←→列方向互换；将第一步读入的列值从列线输出，读入行值（行值中为0的位即为按键所在行）
合并行值+列值 → 键的位置码
```

**与行扫描法的对比**：

| 对比项 | 行扫描法 | 线反转法 |
|--------|---------|---------|
| 接口要求 | 行输出，列输入（固定方向） | 行列方向可变（如用8255A方式0） |
| 硬件复杂度 | 低 | 较高 |
| 软件复杂度 | 较高（需逐行扫描） | 简单 |
| 速度 | 慢 | 快 |

⚠️ 考点：简答题（对比分析） | ⭐⭐

### 2.7 键盘三种工作方式

| 工作方式 | 原理 | 特点 | 适用场合 |
|---------|------|------|---------|
| 程序控制扫描 | CPU 空闲时调用键盘扫描子程序 | 占用CPU时间，实时性低 | 简单系统 |
| 定时扫描 | 定时器中断触发键盘扫描 | 本质是中断，但非实时，是定时响应 | 中等实时性需求 |
| 中断扫描 | 有键按下时产生中断请求 | 实时性最高，CPU效率最高 | 对实时性要求高的系统 |

⚠️ 考点：简答题 | ⭐⭐

### 2.8 BIOS/DOS 键盘中断

| 中断号 | 功能 | 关键参数 |
|--------|------|---------|
| INT 09H | 向键盘缓冲区**写入**数据（从端口60H读扫描码，写入队尾） | 硬件中断，由键盘控制器触发 |
| INT 16H (AH=0) | 从键盘缓冲区**读出**字符（无键则等待） | 出口：AL=ASCII码，AH=扫描码 |
| INT 16H (AH=1) | 检测缓冲区是否有字符（不等待，设置ZF标志） | ZF=0：有键；ZF=1：无键 |
| INT 21H (AH=1) | 从键盘输入一个字符并**回显** | 出口：AL=字符 |
| INT 21H (AH=7) | 从键盘输入一个字符，**不回显** | 出口：AL=字符 |

**键盘缓冲区**：32字节，循环队列，可存16次击键的ASCII码和扫描码。INT 09H写队尾，INT 16H读队首。

⚠️ 考点：简答题 | ⭐⭐

### 2.9 LED 七段显示器结构与段码 📝

**结构**：七段显示器有 a、b、c、d、e、f、g 七段加小数点 h(dp)，共8段，每段对应数据线一位。

```
      a
   ┌──────┐
 f │      │ b
   │  g   │
   ├──────┤
 e │      │ c
   │      │
   └──────┘ ● h(dp)
      d
```

**共阴极 vs 共阳极**：

| 类型 | 公共端 | 段码逻辑 | 点亮条件 |
|------|-------|---------|---------|
| 共阴极 | 接GND | 对应段输出 **1** 点亮 | 送高电平 |
| 共阳极 | 接VCC | 对应段输出 **0** 点亮 | 送低电平 |

**共阴极段码表（必背0~9，A~F）**：

段码位对应：D7=h, D6=g, D5=f, D4=e, D3=d, D2=c, D1=b, D0=a

| 字形 | h g f e d c b a | 段码(H) | 共阳极段码(H) |
|------|----------------|---------|-------------|
| 0 | 0 0 1 1 1 1 1 1 | 3FH | C0H |
| 1 | 0 0 0 0 0 1 1 0 | 06H | F9H |
| 2 | 0 1 0 1 1 0 1 1 | 5BH | A4H |
| 3 | 0 1 0 0 1 1 1 1 | 4FH | B0H |
| 4 | 0 1 1 0 0 1 1 0 | 66H | 99H |
| 5 | 0 1 1 0 1 1 0 1 | 6DH | 92H |
| 6 | 0 1 1 1 1 1 0 1 | 7DH | 82H |
| 7 | 0 0 0 0 0 1 1 1 | 07H | F8H |
| 8 | 0 1 1 1 1 1 1 1 | 7FH | 80H |
| 9 | 0 1 1 0 1 1 1 1 | 6FH | 98H |
| A | 0 1 1 1 0 1 1 1 | 77H | 88H |
| B | 0 1 1 1 1 1 0 0 | 7CH | 83H |

| D | 0 1 0 1 1 1 1 0 | 5EH | A1H |

| F | 0 1 1 1 0 0 0 1 | 71H | 8EH |
| 不显示 | 0 0 0 0 0 0 0 0 | 00H | FFH |

### 2.10 静态显示 vs 动态显示

**静态显示**：每个显示位有独立的输出端口，各位同时持续显示，互不干扰。优点：程序简单，亮度高；缺点：需要端口多。

```
接口图（3位静态显示）：
  8255A PA口 → 第1位显示器段线
  8255A PB口 → 第2位显示器段线
  8255A PC口 → 第3位显示器段线
（每位各自独立，无需位选）
```

**动态显示**：多个显示器共用段线，通过**位码**逐位轮流选通（扫描）。每位每隔一段时间点亮一次，利用视觉暂留使人眼感觉同时显示。

```
接口图（8位动态显示）：
  8255A PB口 → 段码（所有显示器共用，送段码决定显示什么数字）
  8255A PA口 → 位码（PA0~PA7分别控制第0~7位，输出0选中对应位）

程序流程：
  取第i位段码 → OUT到PB口
  取第i位位码 → OUT到PA口（该位对应位置为0）
  延时（让该位亮一会儿）
  → 下一位（i+1）
  → 循环
```

**动态显示关键程序语句**：
```assembly
MOV AL, [BX+SI]   ; 从段码表取段码
OUT 61H, AL       ; 送B口（段码）
MOV AL, AH        ; AH中存位选码
OUT 60H, AL       ; 送A口（位码）
ROR AH, 1         ; 右移，下一个位选码（1位循环移动）
```

❌ 易错：动态显示中，**先送段码，再送位码**（先确定显示什么，再点亮哪一位），顺序不能颠倒，否则可能出现短暂的乱码。

### 2.11 查表译码（XLAT 指令）

**XLAT 指令功能**：`AL ← [BX + AL]`

即：以 BX 为表基址，AL 为偏移量，查表后结果存回 AL。

```assembly
AND AL, 0FH              ; 屏蔽高4位，确保AL是0~F的索引
MOV BX, OFFSET LEDTAB    ; BX指向段码表首地址
XLAT                     ; AL = LEDTAB[AL]，取到对应段码
OUT port, AL             ; 输出段码
```

## 3. 高频计算题型与模板

### 3.1 动态显示位选码计算

**规则**：
- 共阳极显示器：**位码对应位为 0 选中**（低电平有效）
- 共阴极显示器：**位码对应位为 1 选中**（高电平有效）

**解题步骤**：
1. 确定显示器类型（共阳/共阴）
2. 确定每一显示位对应 PA 口的哪一位
3. 写出每位的位选码（16进制）

**示例**（练习题/课件例9.2.2）：
8位共阳极动态显示，PA7~PA0分别控制第7~0位显示器。

- 初始位选码（选中第7位）= `01111111B` = `7FH`
- 每次 `ROR AH, 1`，依次变为 `BFH`、`DFH`、`EFH`…
- 当 AH 回到 `7FH` 时，说明8位全部扫描一遍完毕

---

### 3.2 8255A 控制字计算

**公式（方式0）**：

$$\text{控制字} = 1\underbrace{00}_\text{A组方式}\underbrace{A}_\text{A口方向}\underbrace{C上}_\text{C口高4位}\underbrace{0}_\text{B组方式}\underbrace{B}_\text{B口方向}\underbrace{C下}_\text{C口低4位}$$

- 最高位固定为 1（控制字标志）
- 输入=1，输出=0

**示例**（课件例9.1.2线性键盘）：
A口方式0输入，B口方式0输出，C口方式0输出：

```
D7=1（控制字）
D6D5=00（A组方式0）
D4=1（A口输入）
D3=0（C口高4位输出）
D2=0（B组方式0）
D1=0（B口输出）
D0=0（C口低4位输出）
→ 控制字 = 1001 0000B = 90H
```

❌ 易错：控制字最高位必须为1，否则不是方式控制字而是C口位操作字。

## 4. 练习题精析

### 题目116（练习题第116题）

**题目**：什么是键盘的行扫描法和线反转法？其实现过程有哪些区别？

**解答**：

**行扫描法**：步进扫描方式，行输出列输入。每次只将一行的输出置为0（其余行为1），同时读取列线状态。若列线不全为1，说明该行有键按下，由行扫描值和列值共同确定按键位置。软件需逐行扫描，程序较复杂；但接口简单，行固定输出，列固定输入，方向不变。

**线反转法**：两步完成。第一步：行输出全0，读列值（确定列号）；第二步：行列互换方向，将第一步读到的列值从列线输出，读行线（确定行号）。两步读到的行值和列值合并即为键的位置码。要求 I/O 口方向可切换（如用8255A方式0可通过重写控制字切换），硬件较复杂，但软件简单。

**主要区别**：
| 项目 | 行扫描法 | 线反转法 |
|------|---------|---------|
| I/O 方向 | 固定（行输出、列输入） | 可变（需两次切换） |
| 硬件要求 | 低 | 较高 |
| 软件复杂度 | 较高（需逐行循环） | 较低 |

**考察点**：矩阵键盘扫描原理，两种方法的原理和区别。

### 题目117（练习题第117题）

**题目**：LED显示器的工作原理是什么？何谓共阳极？何谓共阴极？

**解答**：

LED七段显示器由7个发光二极管（段a~g）和1个小数点（h/dp）组成。通过控制各段的亮灭状态，组合显示出0~9、A~F等字符。

- **共阴极**：所有 LED 的阴极连接在一起接地（GND）。对应段的阳极输入**高电平（1）**时，该段点亮。段码中某位为1表示对应段亮。
- **共阳极**：所有 LED 的阳极连接在一起接+5V（VCC）。对应段的阴极输入**低电平（0）**时，该段点亮。段码中某位为0表示对应段亮。

因此，共阳极段码 = 共阴极段码按位取反（D7位一般不使用可置0）。

**考察点**：LED显示器基本工作原理，共阴/共阳极的定义和段码的差异。

### 题目118（练习题第118题）

**题目**：LED显示器接口有哪几种方式？

**解答**：

LED显示器的接口方式主要按显示方式分为两种：

1. **静态显示接口**：每个显示位独占一个输出端口（如8255A的某个口），各位同时持续发光，互不影响。优点：程序简单，亮度高，无闪烁感；缺点：端口占用多，位数多时硬件成本高。

2. **动态显示接口**：所有显示位共用段码线（一个输出端口），另用一个输出端口输出位码，通过轮流点亮各位（扫描）产生视觉暂留效果实现多位显示。优点：节省端口，硬件简单；缺点：亮度较低，程序较复杂，需持续刷新。

**考察点**：两种显示接口方式的原理和优缺点比较。

## 5. 简答题备考模板

### [简答1] 说明键盘消抖的原因和软件消抖的方法

**规范答法**：

机械按键在闭合和断开时，触点会反复抖动约**10~20ms**，在此期间会反复读到高低电平变化，导致一次按键被识别为多次。

软件消抖方法：检测到按键信号后，调用约 **20ms 的延时子程序**，等待触点稳定后再次读取端口状态。若延时后仍检测到按键信号，则确认为有效按键；若读到无按键状态，则判断为抖动干扰，重新等待。

### [简答2] 说明矩阵键盘行扫描法的工作原理 📝

**规范答法**：

行扫描法是一种步进扫描方式。将 I/O 口分为行（输出）和列（输入），按以下步骤工作：

① 向行线输出全0（`00H`），读取列线状态。若列线全为1，说明无键按下；否则说明有键按下，进行消抖确认。

② 确认有键后，依次对每一行输出扫描信号（仅本行为0，其余行为1），同时读取列线状态。

③ 当某行扫描时列线不全为1，说明按键位于该行，由行扫描值（行号）和列线值（列号）组合成键的位置码。

④ 用位置码查键值表，得到按键的键号（0~F）。

### [简答3] 说明LED动态显示的原理和位码/段码的作用

**规范答法**：

动态显示是一位一位地轮流点亮各位显示器（扫描显示）。对某一位显示器来说，每隔一段时间点亮一次，利用人眼视觉暂留效果，使人感觉多位同时显示。

- **段码**：决定该数码管显示什么字形。对共阴极显示器，段码的某位为1时对应段亮；对共阳极，某位为0时对应段亮。
- **位码**：决定选中哪一个数码管。对共阳极显示器，位码对应位为0时选中该数码管；对共阴极，为1时选中。

程序每次先输出段码（确定显示内容），再输出位码（选中显示位），延时后切换到下一位，循环往复。

### [简答4] 说明 INT 09H、INT 16H、INT 21H 的键盘功能区别

**规范答法**：

- **INT 09H**（BIOS键盘中断）：这是键盘硬件中断服务程序，由键盘控制器触发。功能是从键盘接口端口（60H）读取扫描码，判断键类型，将 ASCII 码和扫描码写入键盘缓冲区**队尾**（FIFO队列）。

- **INT 16H**（BIOS键盘功能调用）：从键盘缓冲区**读出**数据（从队首读取）。AH=0时，无键按下则等待，有键则AL=ASCII码，AH=扫描码；AH=1时，不等待，通过ZF标志反映缓冲区是否有键（ZF=0有键，ZF=1无键）。

- **INT 21H**（DOS功能调用）：AH=1，从键盘输入字符并**回显**到屏幕；AH=7，从键盘输入字符但**不回显**。

## 6. 易错点汇总

- ❌ 易错：以为按键按下时对应 I/O 线为高电平（1）  
  ✅ 正确：按键按下时，该 I/O 线通过按键接地，变为**低电平（0）**；未按下时上拉为高电平（1）

- ❌ 易错：共阳极和共阴极段码弄混，写"0"时用错段码  
  ✅ 正确：共阴极"0"的段码为 **3FH**，共阳极为 **C0H**（两者互为按位取反）。拿到题目先看是哪种极性

- ❌ 易错：行扫描法中，`ROL AH, 1` 用了 `ROR` 或 `SHL`  
  ✅ 正确：行扫描初值为 `11111110B`（第0行），每次**循环左移1位**（`ROL AH, 1`），依次变为 `11111101B`（第1行）、`11111011B`（第2行）、`11110111B`（第3行）

- ❌ 易错：软件消抖只判断一次就处理按键  
  ✅ 正确：必须**延时20ms后再次确认**，若延时后读到无键，说明是抖动干扰，需重新查询

- ❌ 易错：8255A控制字最高位写0  
  ✅ 正确：方式控制字最高位**必须为1**（D7=1），否则是C口按位操作命令，不是方式控制字

- ❌ 易错：动态显示程序中先送位码再送段码  
  ✅ 正确：正确顺序是**先送段码，再送位码**，避免位码切换瞬间显示错误字形

- ❌ 易错：XLAT 指令后 AL 保存的是地址而不是数据  
  ✅ 正确：`XLAT` 执行 `AL ← [BX+AL]`，AL 最终存放的是**表中的内容（段码数据）**，不是地址

- ❌ 易错：INT 16H AH=1 功能"无键时等待"  
  ✅ 正确：INT 16H AH=1 是**不等待**，通过 ZF 标志反映状态。等待功能是 AH=0

## 7. 一页纸速记

**核心段码（共阴极，必背）**：  
0→3FH, 1→06H, 2→5BH, 3→4FH, 4→66H, 5→6DH, 6→7DH, 7→07H  
8→7FH, 9→6FH, A→77H, B→7CH, C→39H, D→5EH, E→79H, F→71H  
共阳极 = 共阴极取反（0→C0H, 1→F9H, 2→A4H…）

**8255A 控制字口诀**：最高位1，后跟：A口方式（2位），A口方向，C口高4位方向，B口方式（1位），B口方向，C口低4位方向。输入=1，输出=0。

**行扫描法步骤口诀**：全行输0查有键 → 消抖再确认 → 逐行扫描找位置 → 合成位置码 → 查表得键号

**最易混淆的区别**：
- 共阴极：段线输出**1**点亮；共阳极：段线输出**0**点亮
- INT 16H AH=0：**等待**有键；AH=1：**不等待**，查ZF
- 静态显示：多口多段码；动态显示：一口段码+一口位码轮扫
- 行扫描：`ROL`移动扫描值；形成位置码：`SHL AH, 4` 再 `OR AL, AH`
