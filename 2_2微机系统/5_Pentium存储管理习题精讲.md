### 模块A：为什么需要虚拟存储器（基础动机）

> 这是整章的起点：所有段式、页式机制，本质上都是"虚拟存储器"这个大目标下的具体实现手段。

**知识铺垫**

虚拟存储器由主存储器、辅助存储器、辅助硬件和操作系统管理软件组成，目标是让存储系统的速度接近主存、单位造价接近辅存，从而获得很高的性能价格比。它和Cache存储器都是"分层存储"思想的产物，但解决的问题层次不同：

| 存储体系 | 虚拟存储器 | Cache存储器 |
|---|---|---|
| 存储层次 | 主存-辅存 | Cache-主存 |
| 主要功能 | 主存速度，辅存容量 | CPU速度，主存容量 |
| 信息传送单位 | 信息块（段、页），长度较大 | 信息块（块、行），长度较小且固定 |
| 操作过程 | 部分硬件+操作系统软件实现，对应用程序员透明 | 全部硬件实现，对各类程序员透明 |

> ⚠️考点 🆕（学习要点原文未单列此题，但课件5.1节完整覆盖，属本章开篇必备背景知识）

> **Q11. 说明计算机中采用Cache和虚拟存储器的原因。**

这道题题库未给出现成答案，依据课件5.1节作答：

采用Cache的原因是弥合CPU运算速度与主存访问速度之间的差距——CPU速度远快于主存，靠在CPU与主存之间插入一层小容量高速的Cache，把近期常用的数据/指令块复制进Cache，使CPU大多数时候能以接近自身速度的方式存取数据，从而提高整机性能。

采用虚拟存储器的原因是弥合程序对存储容量的需求与主存实际容量之间的差距——程序需要的逻辑地址空间往往远大于实际安装的主存容量，通过把主存和辅存（磁盘）结合，借助硬件和操作系统的协同管理，让程序员在编程时可以使用远大于主存实际容量的"虚拟"地址空间，而具体的调入调出由系统自动完成，对程序员透明。

简言之：**Cache解决的是"速度"矛盾，虚拟存储器解决的是"容量"矛盾**，两者都是用"对程序员/应用透明"的自动化机制，让有限的硬件资源发挥出更大效益。

**✅参考答案要点：Cache解决CPU与主存的速度差距问题；虚拟存储器解决程序所需地址空间与主存实际容量的差距问题；二者均通过软硬件协同对使用者透明地完成"调入调出/命中替换"。**

### 模块B：Pentium的三种操作模式与各模式下的存储管理方式

> 在深入保护模式细节之前，先明确"什么时候才需要这套段页式机制"——这取决于Pentium处于哪种操作模式。

**知识铺垫**

Pentium有三种操作模式：**实地址模式、保护模式、虚拟8086模式**（与项目大纲第3章条目"⚠️ 三种操作模式：实地址、保护、虚拟8086"对应，本章在此基础上聚焦"实地址模式"与"虚拟8086模式"的细节辨析，保护模式下的具体存储管理是全章主线）。

存储管理方式按映象算法分为三种：**段式、页式、段页式**。Pentium三种都支持。其中分段部件功能是把逻辑地址转换成连续的不分段的线性地址；分页部件功能是把线性地址转换成物理地址。只有进入保护模式，这套"分段+分页"机制才会完整启用；实地址模式下，Pentium的工作方式退化为兼容8086的简单寻址方式。

> ⚠️考点🔥

> **Q13. Pentium实地址模式的特点，8086工作模式、Pentium实地址模式、Pentium虚拟8086模式之间的异同？**

题库原文已给出部分答案，在此基础上扩展讲深：

**Pentium实地址模式特点**：能有效地使用8086所没有的寻址方式、32位寄存器和大部分指令，但段寻址方式与8086完全相同——每个段寄存器仍只能定义一个64KB存储器段的起点（即给出段基址），CS指出当前代码段、SS指出当前堆栈段、DS指出当前数据段、ES指出当前附加段。也就是说，实地址模式在地址转换规则上"形似8086、内核已是32位"。

**实地址方式与8086工作方式的关系**：基本体系结构相同，Pentium与8086在实地址下完全兼容，物理地址同样按"段基址×16+偏移地址"的方式形成。

**虚拟8086方式与实地址方式的本质区别**——这是本题最容易混淆、也是大纲特别标注的辨析点：
- 虚拟8086方式是**一个程序的运行方式**：它是在保护模式下，处理器为运行老的8086程序而提供的一种"模拟环境"，本质上仍处于保护模式的框架内，只是让该任务表现得像运行在8086上一样。
- 实地址方式是**处理器的工作方式**：它是处理器整体所处的一种模式（与保护模式并列的硬件级模式选择），不依赖保护模式存在。

二者层级不同：虚拟8086是保护模式"内部"的一种任务运行环境，而实地址模式是和保护模式平起平坐的整机工作模式。打个比方，实地址模式好比"整台电脑切换成兼容机模式"，虚拟8086方式则像"在多任务操作系统里开一个DOS兼容虚拟机窗口"——后者外部环境（操作系统）依然是现代化的保护模式，只是给这一个任务套了一层"看起来像8086"的壳。

**✅参考答案要点：实地址模式下Pentium段寻址方式与8086相同（段基址×16+偏移），但可用32位寄存器和扩展指令；虚拟8086方式是保护模式下某个任务的运行方式（任务级），实地址方式是处理器整体的工作方式（处理器级），二者所属层级不同。**

### 模块C：虚拟地址的组成（保护模式下的逻辑地址结构）

> 进入保护模式后，"逻辑地址"不再是简单的"段基址+偏移"两元组，而是变成"段选择符+偏移量"。这是后续一切寻址过程的起点。

**知识铺垫**

Pentium在保护模式下的存储器管理单元使用48位的存储器指针，对应地址空间为 $2^{46}$（64T）字节。

虚拟地址（逻辑地址）的构成是：**16位段选择符 + 32位偏移量**。

```
        47 ······ 32  31 ······················ 0
虚拟地址 [  段选择符  ][        偏移量          ]
              16位              32位
```

段选择符内部又可细分为三个字段：

```
15 ··············· 3   2    1  0
[    INDEX索引值    ][ TI ][RPL]
       13位            1位   2位
```

- **INDEX（索引字段，13位）**：指向描述符表中的具体条目；
- **TI（描述符表选择，1位）**：TI=0选GDT，TI=1选LDT；
- **RPL（请求特权级，2位）**：不参与寻址，仅用于特权级检查。

这套结构与项目大纲第5章条目"⚠️ 段选择符的组成及意义"完全对应，本章无独立原题专门考它，但在模块I（GDT寻址）和模块J（LDT寻址）中会反复用到，此处先建立框架印象。

### 模块D：段描述符——保护模式存储管理的核心数据结构

> 有了"段选择符"之后，下一个问题是：选择符具体"选"出来的是什么？答案就是段描述符。段描述符是理解GDT/LDT寻址、特权级检查、TSS等后续所有内容的地基。

#### ⚠️🔥 考点：段描述符的组成及作用（Q12、Q15）

> **Q12. 段描述符用于描述段的信息，它由 个字节组成。**

**知识铺垫+解答**

段描述符用于描述段的基本信息，由**8个字节**组成，保存段的属性、段的大小、段在存储器中的位置以及控制和状态信息，由编译程序、连接程序、装入程序或操作系统产生。

**✅参考答案要点：8个字节。**

> **Q15. 说明段描述符的组成及作用。**

题库未给出现成答案，依据课件5.2.2节完整作答：

**段描述符的作用**：描述一个段的基本信息——包括该段在物理地址空间中的位置（基地址）、大小（界限）、以及该段的访问属性和控制状态信息，是CPU在保护模式下找到并合法访问某个段的"说明书"。

**段描述符的组成（以程序段描述符8个字节为例）**：

| 字节 | 内容 |
|---|---|
| 0 | 段界限 7~0 |
| 1 | 段界限 15~8 |
| 2 | 段基址 7~0 |
| 3 | 段基址 15~8 |
| 4 | 段基址 23~16 |
| 5 | P存在位 / DPL特权级 / S分类 / TYPE类型 |
| 6 | G粒度 / D-B位 / 0（兼容位） / AVL / 段界限19~16 |
| 7 | 段基址 31~24 |

各字段含义：
- **段基址（32位）**：规定该段在4GB物理地址空间中的起始位置；
- **段界限（20位）**：决定段的长度，单位由G位决定——G=0时以字节为单位，G=1时以4KB为单位；
- **P（存在位）**：P=1表示该段当前在内存中，P=0表示不在（需要从辅存调入）；
- **DPL（特权级，2位）**：定义该段被访问所需的特权级，取值0~3（0级最高）；
- **S（分类位）**：S=0是系统段描述符，S=1是非系统段（程序段）描述符；
- **TYPE（类型）**：在不同段描述符中含义不同，详见模块E课件补充节；
- **AVL（系统可用位）**：表示系统软件是否可以使用该段，1可用、0不可用；
- **D/B位**：32/16位大小选择，D/B=1选32位、D/B=0选16位，在代码段中表示操作数和有效地址长度，在堆栈段中表示用ESP还是SP，在数据段中表示操作数长度；
- 第6字节的D5位必须固定为0，用于保证向未来处理器的兼容性。

**✅参考答案要点：段描述符由8字节组成，包含32位段基址、20位段界限、P存在位、DPL特权级、S分类位、TYPE类型字段、G粒度位、D/B位、AVL可用位等，整体作用是为CPU提供该段的位置、大小和访问控制信息。**

### 模块E：课件补充·无对应原题——段描述符按段性质的分类、门描述符

> ⚠️考点 课件补充·无对应原题

这两条官方考点（"段描述符按段性质的分类"和"门描述符含义及其分类"）在本章练习题库中没有对应原题，但课件5.2.2、5.2.2(3)节有完整覆盖，按要求单独成节讲解。

**1. 段描述符按段性质的分类**

```
段描述符
├── 程序段描述符
│   ├── 代码段描述符
│   ├── 堆栈段描述符
│   └── 数据段描述符
└── 系统段描述符
    ├── 任务状态段（TSS）描述符
    ├── 局部描述符表（LDT）描述符
    └── 门描述符
        ├── 调用门描述符
        ├── 中断门描述符
        ├── 任务门描述符
        └── 陷阱门描述符
```

区分程序段还是系统段的关键就是S分类位：S=1是程序段（数据段/堆栈段/代码段），S=0是系统段（TSS/LDT/门）。

在程序段内部，TYPE字段的D3位（即E可执行位）进一步区分：
- 数据段/堆栈段描述符：E=0，此时还包含ED扩展方向位（0向上扩展用于数据段、1向下扩展用于堆栈段）、W可写位、A访问位；
- 代码段描述符：E=1，此时包含C一致性位（C=0非一致性代码段忽略DPL检查，C=1一致性代码段需要特权级检查）、R可读位、A访问位。

**2. 门描述符含义及其分类**

门是一种"关卡"，用来控制从一段程序到另一段程序、或从一个任务到另一个任务的转移。门描述符给出一个逻辑地址及转入这个地址需满足的约束条件，用于控制转入目标代码段的入口点。

门描述符共四类：
- **调用门**：用于改变特权级别（低特权级程序通过调用门访问高特权级的子程序）；
- **任务门**：用于任务切换；
- **中断门、陷阱门**：用于确定中断服务程序入口（区别在于中断门进入后会自动清除IF标志关中断，陷阱门不会）。

门描述符的格式与程序段描述符不同，关键差异在于：门描述符里没有"段基址"字段，取而代之的是**偏移地址（32位，分散在字节0~1和6~7）+ 段选择符（16位，字节2~3）**——因为门本身不直接指向一段连续的物理空间，而是指向"目标代码段的选择符 + 段内偏移量"这样一个逻辑入口点。字节4的"字计数"字段仅用于带特权级切换的调用门，用来标明调用时要从调用者堆栈复制多少个字参数到被调用过程的堆栈。任务门则比较特殊：它的段选择符送入的是TR寄存器而非CS（因为任务门指向的是一整个TSS，而非某段代码的入口）。

> 建议对照课件原文核实门描述符TYPE编码细节（如0100=调用门、0110=中断门等具体二进制编码），考试若涉及具体编码辨析，以课件PPT截图或老师补充材料为准。

### 模块F：三大描述符表寄存器——GDTR、LDTR、IDTR、TR

> 段描述符不是孤立存在的，它们成组地保存在"描述符表"里，而CPU要靠专门的寄存器才能找到这些表。这是从"概念"走向"具体寻址电路"的关键一步。

#### ⚠️🔥 考点：GDTR、LDTR、IDTR、TR含义及保存内容（Q14）

> **Q14. IDTR、GDTR和LDTR分别代表什么寄存器？其内容是什么信息？有什么作用？**

题库未给出现成答案，依据课件5.2.3~5.2.6节完整作答：

| 寄存器 | 全称 | 保存内容 | 作用 |
|---|---|---|---|
| **GDTR** | 全局描述符表寄存器 | 32位GDT基地址 + 16位GDT界限（共48位） | 指定GDT在主存中的起始位置和大小，GDT保存系统使用、各任务共享的段描述符，**全系统只有一个** |
| **LDTR** | 局部描述符表寄存器 | 16位选择符（程序可见部分）+ 32位基址、20位界限、12位属性（不可见Cache部分） | 指出当前任务LDT的基址，LDT保存某任务私有的段描述符，**每个任务一个** |
| **IDTR** | 中断描述符表寄存器 | 32位IDT基地址 + 16位IDT界限（共48位） | 指定IDT位置，IDT保存中断门、陷阱门、任务门（通常不含调用门），**全系统只有一个**，最大可达64KB但Pentium只支持256个中断/异常实占用2KB |

**补充：TR寄存器**（虽然不在本题直接问及范围，但与GDTR/LDTR/IDTR同属一组寄存器，一并理解）：TR是任务寄存器，指向当前正在运行任务的TSS，其内容由LTR、STR指令装入/保存，也可通过远跳转/远调用指令或任务门改变。

三者的关键共性与差异：GDTR和IDTR都是"全局唯一"，直接保存48位（基址+界限），由LGDT/SGDT、LIDT/SIDT指令访问；LDTR则不同——它本身保存的是一个**段选择符**，要通过这个选择符去GDT中查出对应的LDT描述符，再把LDT描述符的内容（基址、界限、属性）装入LDTR的Cache部分，这一点和TR寻址TSS的方式是一致的（即LDT和TSS都"挂在"GDT下面被统一管理，但各自有自己的专用寄存器作为"门面"）。

**✅参考答案要点：GDTR保存32位基址+16位界限指向全局唯一的GDT；LDTR保存16位选择符（指向GDT中的LDT描述符）+ Cache部分（32位基址+20位界限+12位属性），每任务一个；IDTR保存32位基址+16位界限指向全局唯一的IDT（存放中断门/陷阱门/任务门）。**

### 模块G：特权级管理——CPL、DPL、RPL

> 描述符表的存在保证了CPU"能找到"段，但操作系统还需要"控制谁能访问哪个段"，这就是特权级机制要解决的问题。

#### ⚠️🔥 考点：CPL、RPL、DPL的含义（Q16）

> **Q16. 说明CPL、RPL、DPL的含义。**

题库未给出现成答案，依据课件5.3.1节完整作答：

| 名称 | 全称 | 存放位置 | 含义 |
|---|---|---|---|
| **CPL** | 当前特权级 | 当前正在执行的代码段所具有的访问特权级，其值一般等于当前代码段描述符中的DPL | 任务执行时所处的特权级 |
| **DPL** | 描述符特权级 | 保存在该段的段描述符的DPL字段中 | 段被访问所需要的特权级，即"这个段允许多高权限的代码来访问我" |
| **RPL** | 请求特权级 | 存放在段选择符的最低两位 | 新装入段寄存器的段选择符所携带的特权级，代表"这次访问请求声明自己的权限" |

特权级一共4档：00、01、10、11，分别对应0级、1级、2级、3级，**0级特权最高，3级最低**（这与日常直觉"数字越大权限越大"正好相反，是常见易错点）。

特权管理的核心规定：**特权级为P的段中存储的数据，只能由特权级高于或等于P（数值更小或相等）的段中运行的程序使用；特权级为P的代码段/过程，只能由在低于或等于P的特权级下执行的任务调用。**

可以用一个生活化类比理解三者关系：DPL好比"这扇门的门禁等级"（门本身设定的准入要求），CPL好比"你现在所在的楼层权限"（你当前身份的实际权限），RPL好比"你刷卡时声明的权限"（即便你本身权限够高，刷卡时也可以选择以较低权限身份进入，用来防止权限被无意中"越级使用"）。系统在做特权检查时，通常取CPL和RPL中数值较大（权限较低）的一个，与目标段的DPL比较，只有满足"有效请求特权级 ≤ DPL"才允许访问。

**✅参考答案要点：CPL是当前代码段的特权级（≈代码段描述符DPL）；DPL是段描述符自带的访问门槛；RPL是段选择符携带的请求特权级；特权级0~3，数值越小特权越高。**

### 模块H：任务状态段TSS

> 特权级机制是为多任务/多用户操作系统服务的，而真正承载"一个任务的完整运行现场"的数据结构就是TSS。理解TSS是理解任务切换的关键。

#### ⚠️🔥 考点：任务状态段TSS的主要作用（Q17）

> **Q17. 任务状态段TSS的主要作用是什么？**

题库未给出现成答案，依据课件5.2.6节完整作答：

TSS（任务状态段）保存现有任务的机器状态——即处理器的工作环境，比如各通用寄存器、段寄存器、指令指针EIP、标志寄存器EFLAGS的当前值，以及该任务的特权级堆栈指针（ESP0/SS0、ESP1/SS1、ESP2/SS2）、页目录基址CR3、LDT选择符等任务间的关联信息。**TSS本身不包含数据和代码，每个任务拥有一个独立的TSS**。

TSS的主要作用就是支撑"任务切换"：当处理器从任务A切换到任务B时，处理器会把任务A当前所有寄存器的内容保存到任务A的TSS对应单元中，然后把任务B的TSS中保存的内容重新装入处理器的所有寄存器，从而让任务B"接着上次的状态"继续运行。这正是大纲条目"🆕 TSS（任务状态段）的主要作用（保存任务现场信息，支持任务切换），TR 寄存器指向当前任务的 TSS"所对应的内容。

TSS格式中还有一个值得记住的细节：偏移00H处的**返回链BACK_LINK**字段是一个段选择符，保存前一个任务TSS描述符的段选择符（即原TR中16位可见部分），供任务通过IRET指令返回时使用，从而能"回到"前一个TSS。

**✅参考答案要点：TSS保存一个任务的完整处理器现场（寄存器组、各特权级堆栈指针、CR3、LDT选择符），不含数据代码，每任务一个；核心作用是支撑任务切换时的现场保存与恢复，由TR寄存器寻址当前任务的TSS。**

### 模块I：通过GDT访问数据段的完整寻址过程

> 前面几个模块分别讲清楚了"段选择符长什么样""段描述符存什么""描述符表寄存器怎么找到表"，现在把它们串成一条完整的寻址链路——这正是大纲标注的"数据段的访问过程"与"GDT/LDT寻址过程"两条考点本质上是同一套逻辑的不同问法。

#### ✅真题🔥📝 考点：GDT寻址过程/数据段访问过程（Q18）

> **Q18. 简述Pentium通过GDT访问数据段的寻址过程（也可画图说明）。**

题库未给出现成答案，依据课件5.2.3节"GDT寻址可归纳为"三步法及5.3.2节数据段访问图完整作答：

**GDT寻址三步法**：
1. **段选择符 × 8 + GDTR基址 = 段描述符地址**——段选择符中的13位INDEX字段乘以8（每个描述符8字节），加上GDTR中保存的GDT基地址，就定位到目标段描述符在GDT中的具体存储地址；
2. **段描述符内容（包括基址）→ 相应段寄存器的Cache部分**——把从该地址读出的8字节段描述符内容（基址、界限、属性等）装入对应段寄存器（如DS）的不可见Cache部分；
3. **Cache中段基址 + 虚地址偏移量 = 物理地址（线性地址）**——以后再次访问同一个段时，不需要重新查表，直接从段寄存器的Cache部分取出基址，加上偏移量即可。

**画图要点（ASCII示意）**：

```
请求的虚地址
[ 16位段选择符 ] [        32位偏移量        ]
       |                      |
       ① 装入                |
       v                      |
      DS                      |
[13位INDEX|TI|RPL][32位基址|20位界限|12位属性]——DS段寄存器不可见部分
   |                                |
   ② INDEX×8                       |
   v                                |
TI=0选GDTR ----+                    |
   |           |                    |
   v           v                    |
[GDT起始地址+偏移] → ③读出8字节数据段描述符
                          |
                          ④ 装入DS的Cache（基址/界限/属性）
                          v
                    DS段寄存器不可见部分
                          |
                          ⑤ Cache中基址 + 偏移量
                          v
                     32位线性地址（物理地址）
```

五个步骤对应关系：①段选择符装入DS的可见部分；②取出选择符中的INDEX字段乘以8（TI=0确定走GDTR）；③用GDTR基址+这个偏移量去内存中读取8字节数据段描述符；④把描述符中的基址、界限、属性信息装入DS段寄存器的不可见Cache部分；⑤用Cache中取出的32位基址加上请求的32位偏移量，得到最终的32位线性地址（若不分页则该线性地址即物理地址）。

**第一次访问与后续访问的区别**：第一次装入DS段寄存器时，必须老老实实走①~④步骤访问内存（GDT）去取描述符；以后只要DS的内容不变，再次访问该段时可以跳过①~④，直接用Cache中已经缓存好的基址做⑤这一步加法，这正是Cache机制带来的"避免每次访存都要查表"的速度优势。

**✅参考答案要点：段选择符INDEX×8+GDTR基址定位描述符地址→读出描述符装入段寄存器Cache→Cache中基址+偏移量=线性地址；第二次起访问同一段时可跳过查表直接用Cache基址做加法。**

### 模块J：通过LDT访问数据段的完整寻址过程

> LDT寻址和GDT寻址的核心三步法完全一致，唯一区别在于"先用LDTR里的选择符到GDT中查出LDT本身的位置"这一额外前置步骤——也正因如此，第二次用到同一套GDT逻辑时本文按规则简短引用，不重复展开。

#### ✅真题🔥📝 考点：LDT寻址过程（Q19）

> **Q19. 简述Pentium通过LDT访问数据段的寻址过程（也可画图说明）。**

题库未给出现成答案，依据课件5.2.4节及5.3.2节图完整作答：

LDT寻址与GDT寻址的差异点在于：当虚地址中段选择符的**TI=1**时，选择的是局部描述符表LDT而非GDT。而LDT本身的位置又是通过LDTR间接确定的：

1. **LDTR的段选择符** × 8 + **GDTR基址** = LDT描述符在GDT中的地址（**注意：定位LDT本身这一步永远走GDT**，因为LDT描述符就是作为系统段描述符之一登记在GDT里的）；
2. 从该地址读出LDT描述符，取出其中的32位基址、20位界限、12位属性，装入LDTR的Cache（不可见）部分，从而"激活"当前任务的LDT；
3. 此后访问数据段时，用虚地址段选择符中的INDEX×8 + LDTR中已缓存的LDT基址 = 目标段描述符在LDT中的地址；
4. 从该地址读出8字节数据段描述符，装入对应数据段寄存器（如DS）的Cache部分；
5. DS的Cache基址 + 偏移量 = 线性地址。

可以这样理解两层关系：**GDT是"全局总目录"，LDTR/LDT是"每个任务的私人通讯录"，而这本私人通讯录本身的位置信息，又登记在全局总目录里**。所以访问LDT中的段，本质上要先后查两次表：第一次（通过GDT）确定"这本私人通讯录放在哪儿"，第二次（在已确定的LDT里）才真正确定"目标段描述符放在哪儿"。

每次往LDTR装入新的段选择符，就意味着"切换到了一本新的私人通讯录"，即为当前任务激活一个新的LDT。

**✅参考答案要点：先用LDTR中的选择符×8+GDTR基址，从GDT中查出LDT描述符并把LDT基址等信息装入LDTR的Cache；再用虚地址中INDEX×8+LDTR缓存的LDT基址，从LDT中查出目标段描述符并装入数据段寄存器Cache；最后Cache基址+偏移量=线性地址。比GDT寻址多一道"先定位LDT本身"的前置步骤。**

### 模块K：分页机制——课件补充·线性地址到物理地址的映射

> ⚠️考点 课件补充·无对应原题

到模块J为止，"虚拟地址→线性地址"这条分段链路已经讲完。官方考点中还有两条——"Pentium页式管理线性地址与物理地址的地址映象关系"和"页表、页目录表的组成及意义"——在练习题库中没有对应原题，但课件5.5节完整覆盖，按规则单独成节讲解，为模块L的段页式综合大题做最后的知识铺垫。

**核心思想**：分段是把程序按逻辑模块切分成大小可变的段；分页则是把整个线性地址空间切分成大小**固定**的页（每页4KB），各页与程序的逻辑结构没有直接关系，纯粹是物理管理单位。Pentium采用**二级页表**方法：第一级叫页目录，页目录中的每一项指明第二级页表的基址；第二级页表中的每一项才指明真正的物理页基址。

**页目录与页表的组成**：
- 页目录基地址寄存器**CR3**保存页目录的基地址（该基地址必须起始于任意4KB边界）；
- 页目录由**最多1024个**页目录项组成，每项4字节，因此页目录自身恰好占一个4KB页面；
- 页表同样由**最多1024个**页表项组成，每项4字节，页表自身也恰好占一个4KB页面。

页目录项/页表项的关键字段（二者格式高度相似）：高20位是基址（页目录项存的是下一级页表的基址，页表项存的是最终物理页的基址），低12位是各种属性位，其中最重要的是**P存在位**（1=存在、0=不存在）和**R/W读写位**（0=只读、1=读写）。页表项比页目录项多一个D修改位（标记该页是否被写过），二者都有A访问位（标记是否被访问过，供替换算法参考）。

**分页转换三段式（32位线性地址→32位物理地址）**：

```
32位线性地址
[10位页目录索引][10位页表索引][12位页内偏移]
 (A31~A22)       (A21~A12)      (A11~A0)
       |               |              |
       v               v              |
  CR3+索引×4      页表基址+索引×4       |
       |               |              |
       v               v              |
   查出页表基址      查出物理页基址      |
                          |            |
                          +------------+
                                |
                                v
                          32位物理地址
```

具体步骤：①4KB长的页目录存储在CR3寄存器指定的物理地址（根地址）；②用线性地址最高10位（A31~A22）作为页目录索引，找到对应页目录项，取出其中的页表基址；③用线性地址中间10位（A21~A12）作为页表索引，在该页表中找到对应页表项，取出物理页的起始地址；④以该物理页起始地址为基址，加上线性地址最低12位（A11~A0）页内偏移量，就得到最终的32位物理地址。

> 建议对照课件原文核实页目录项/页表项中PCD、PWT等具体属性位的细节定义，若考试涉及具体位含义辨析，以课件PPT截图为准。

### 模块L：段页式存储管理的完整寻址过程（全章压轴 / 真题原题）

#### ✅真题🔥📝 考点：段页式存储管理的完整寻址过程（Q20、Q21，对应真题简答题第5题）

> ✅真题：本题与《以前的考题》试卷简答题第5题为同型题——"画图详细说明在Pentium微处理器保护模式情况下，同时有段式存储器管理和页式存储器管理时，逻辑地址转换为物理地址的过程"，是项目大纲中明确标注的本章最高优先级画图题，必须能脱稿完整画出并讲清楚。

**完整寻址过程（段式+页式串联，依据课件5.6节完整作答）**：

整个过程分两大阶段：**第一阶段走分段部件，把虚拟地址变成32位线性地址；第二阶段走分页部件，把这个线性地址再变成32位物理地址**。Pentium保护模式下存储器管理实际有三种工作组合，本题考的是最完整的"分段又分页"这一种：

**第一阶段——分段（虚拟地址 → 线性地址）**：

这一步就是模块I/J讲过的GDT/LDT寻址过程，在此简短引用：虚拟地址由16位段选择符和32位偏移量组成；段选择符中13位INDEX字段×8加上GDTR（TI=0）或LDTR（TI=1）基址，定位到段描述符表中的目标段描述符；取出描述符中的32位段基址，与32位偏移量相加，得到**32位线性地址**。

**第二阶段——分页（线性地址 → 物理地址）**：

这一步就是模块K讲过的二级页表查找过程，在此简短引用：32位线性地址拆分为10位页目录索引、10位页表索引、12位页内偏移；用页目录索引在CR3指向的页目录中查出页表基址；用页表索引在该页表中查出物理页基址；物理页基址加上12位页内偏移，得到最终的**32位物理地址**。

```
虚地址
[13位INDEX][TI][RPL]      [        32位偏移地址        ]
     |
     | INDEX×8
     v
TI=0选GDTR / TI=1选LDTR  ──→  段描述符表(8字节段描述符)
                                     |
                                取出32位基址
                                     |
                                     v
                          基址 + 偏移地址 = 32位线性地址
                                     |
              +----------------------+----------------------+
              |                      |                       |
        10位页目录索引          10位页表索引              12位页内偏移
              |                      |                       |
              v                      v                       |
     CR3+索引×4 → 页目录项    页表基址+索引×4 → 页表项          |
              |                      |                       |
              +--------→ 取出物理页基址 ←----------------------+
                                     |
                              物理页基址 + 页内偏移
                                     |
                                     v
                               32位物理地址
```

**若不分页（仅分段不分页的情形，对应Q21题干最后一句"若无分页功能，则线性地址就直接是物理地址"）**：此时分页部件不工作，第一阶段算出的32位线性地址直接当作物理地址使用，不再经过页目录、页表查找。这正是课件5.6节"三种模式"中的第一种——"分段不分页"，此时一个任务最大可拥有64T地址空间，好处是不用访问页目录和页表、地址转换速度快，缺点是大容量的段调入调出比较耗时、不够灵活。

与之相对，"段页式"组合（分段又分页）的好处恰恰相反：段提供了符合程序逻辑结构的可变长度划分（便于共享和保护），页提供了固定长度的物理管理单位（便于内存的高效分配与置换），二者结合既保留了段的逻辑意义，又用分页解决了段长度不固定带来的内存碎片和调入调出效率问题——这也是为什么这道画图题被反复作为真题考查的核心原因：它考的不是死记硬背一张图，而是要求考生真正理解"为什么段页式要这样设计"。

**✅参考答案要点：分两阶段——①分段：段选择符INDEX×8+GDTR/LDTR基址→查出段描述符→段基址+偏移量=32位线性地址；②分页：线性地址拆为10位页目录索引+10位页表索引+12位页内偏移→CR3+页目录索引查页表基址→页表基址+页表索引查物理页基址→物理页基址+页内偏移=32位物理地址。若不分页，线性地址直接作为物理地址。**
