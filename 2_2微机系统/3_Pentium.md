## 1. 知识体系

Pentium是8086的32位升级版，本章围绕"Pentium如何比8086更强大"展开。首先从**内部结构**看：Pentium引入超标量双流水线、分立Cache和动态分支预测，大幅提升执行效率。其次从**编程结构**看：寄存器从16位扩展到32位（加E前缀），新增系统级寄存器以支持保护模式。最后从**存储管理**看：Pentium引入保护虚地址模式，通过"分段→线性地址→分页→物理地址"的两级转换机制，支持最大64TB逻辑空间。三种操作模式（实地址、保护、虚拟8086）决定了Pentium如何解析地址和管理权限。

```
Pentium第3章
├── 内部结构 (3.1)
│   ├── 超标量双流水线 (U/V)
│   ├── 分立Cache (指令8K + 数据8K)
│   ├── BTB 分支预测
│   └── 分段/分页部件
├── 编程结构 (3.2)
│   ├── 基本结构寄存器 (16个)
│   ├── 系统级寄存器 (GDTR/LDTR/IDTR/TR + CR0~CR4)
│   └── EFLAGS (在8086基础上新增VM/IOPL等)
├── 外部结构/引脚 (3.3)
│   ├── 地址线 A31~A3 + BE7~BE0
│   ├── 数据线 D63~D0
│   └── Cache控制 #Cache / #KEN
└── 操作模式 (3.5)
    ├── 实地址模式 (与8086兼容)
    ├── 保护虚地址模式 (分段+分页)
    └── 虚拟8086模式 (保护模式下跑8086程序)
```

## 2. 核心知识点精讲

### 2.1 Pentium内部结构特点（6大特性）

**核心定义**：Pentium是超标量流水线处理器，内部结构由11个主要部件组成。

**6大结构特性**（必背）：
1. **超标量流水线，超标量为2**：内部有U、V两条整数流水线，可并行执行两条指令
2. **分立Cache，指令数据各8K**：代码Cache和数据Cache分开，各8KB，避免冲突
3. **重新设计的浮点运算部件，8级流水**：浮点性能大幅提升
4. **动态转移预测（BTB）**：转移目标缓冲器，预测正确则继续流水，预测错误则清空流水线
5. **内部32位，外部DB64位，属于32位机**：数据总线64位但仍是32位机
6. **4种工作方式**：实地址、保护虚地址、虚拟8086、系统管理

**流水线结构**：
- U流水线：执行**全部**整数指令和浮点指令
- V流水线：执行**简单**整数指令和交换类浮点指令
- 整数流水线5级，浮点流水线8级

**考试写法**：
Pentium处理器的主要结构特点：①超标量流水线，超标量为2，有U、V两条流水线并行执行；②分立Cache，指令和数据Cache各8KB；③重新设计的浮点运算部件，采用8级流水线；④动态转移预测（BTB），预测错误则清空流水线；⑤内部32位，外部数据总线64位；⑥支持4种操作模式。

⚠️ **考点**：简答题 | ⭐⭐⭐

### 2.2 Pentium寄存器组

**核心定义**：Pentium寄存器分5类，共71个。

```
寄存器分类（5类）
├── 基本结构寄存器（16个）← 程序员可见，考试重点
│   ├── 通用寄存器（8个）：EAX/EBX/ECX/EDX/ESP/EBP/ESI/EDI
│   ├── 专用寄存器（2个）：EIP（指令指针）、EFLAGS（标志）
│   └── 段寄存器（6个）：CS/DS/SS/ES/FS/GS
├── 系统级寄存器（9个）← 保护模式用，考试重点
│   ├── 系统地址寄存器（4个）：GDTR/IDTR/LDTR/TR
│   └── 控制寄存器（5个）：CR0/CR1/CR2/CR3/CR4
├── 调试寄存器（8个）
├── 模型专用寄存器（20个）
└── 浮点寄存器（18个）
```

**通用寄存器关系**（以EAX为例）：
```
EAX  [31          16][15    8][7     0]
      (高16位不独立)   AH      AL
                      ←  AX  →
      ←           EAX          →
```

**段寄存器特点**：
- 共6个，每个16位
- 每个段寄存器有对应的**段描述符高速缓存器**（64位，对程序员透明），含段基址32位、段限20位、属性12位

### 2.3 GDTR / LDTR / IDTR / TR — 系统地址寄存器

**核心定义**：4个系统地址寄存器，用于保存描述符表的位置信息，支持保护模式存储管理。

| 寄存器 | 全称 | 位数 | 保存内容 |
|--------|------|------|----------|
| **GDTR** | 全局描述符表寄存器 | 48位 | GDT的32位基地址 + 16位界限 |
| **IDTR** | 中断描述符表寄存器 | 48位 | IDT的32位基地址 + 16位界限 |
| **LDTR** | 局部描述符表寄存器 | 16+64位 | 16位选择符 + 64位描述符（含基址/界限/属性） |
| **TR** | 任务状态寄存器 | 16+64位 | 当前任务TSS的段选择符 + 描述符（含基址/界限/属性） |

**三个描述符表区别**：
- **GDT**（全局描述符表）：整个系统只有**一个**，存放OS用的描述符和各任务共用描述符
- **LDT**（局部描述符表）：每个任务各有**一个**，存放该任务专用描述符
- **IDT**（中断描述符表）：整个系统只有**一个**，把中断向量与描述符联系起来

**TR的作用**：任务切换时，自动保存和恢复机器状态（TSS = Task State Segment）

**考试写法**：
GDTR保存全局描述符表GDT的开始地址和段长，共48位；IDTR保存中断描述符表IDT的开始地址和段长，共48位；LDTR保存局部描述符表LDT的开始地址、段长及属性，共16+64位；TR（任务寄存器）保存当前执行任务的任务状态段TSS的信息，用于任务切换时自动保存和恢复机器状态，共16+64位。

⚠️ **考点**：简答 | ⭐⭐⭐ 🔥高频

### 2.4 段描述符

**核心定义**：段描述符用于描述段的信息，由**8字节（64位）**组成，包含段基址、段长和访问属性。

**组成**：
```
[63          32][31         12][11        0]
  段基址(32位)    段限(20位)     属性(12位)
```

**段描述符表**：各段的描述符像数组一样组合在一起构成描述符表。描述符偏移地址 = 段选择符 × 8。

❌ **易错**：段描述符是8字节，不是4字节（8086中断向量才是4字节）。

⚠️ **考点**：简答/选择 | ⭐⭐⭐

### 2.5 控制寄存器CR0

**核心定义**：CR0保存全局性机器状态，关键位含义如下：

| 位 | 名称 | 含义 |
|----|------|------|
| **PE** | 保护允许 | PE=1进入保护模式，PE=0实地址模式 |
| **PG** | 允许分页 | PG=1允许分页，PG=0禁止 |
| WP | 写保护 | WP=1用户级只读页禁止写入 |
| CD | 禁止Cache | CD=1禁止片内Cache填充 |
| NW | 不透明写 | NW=1只写Cache不写主存；NW=0写直达 |

❌ **易错**：进入保护模式只需置 `CR0.PE=1`，但要启用分页还需额外置 `CR0.PG=1`。

---

### 2.6 EFLAGS标志寄存器（新增位）

与8086 FLAGS相同的位：CF/PF/AF/ZF/SF/OF/IF/DF/TF（低16位部分）

**Pentium新增的重要标志位**：

| 位 | 名称 | 含义 |
|----|------|------|
| **VM** | 虚拟8086方式 | VM=1，CPU工作在虚拟8086模式 |
| **IOPL** | I/O特权级 | 0~3级，控制I/O指令执行权限 |
| NT | 嵌套任务 | NT=1当前任务嵌套在另一任务中 |
| RF | 恢复标志 | RF=1忽略断点异常 |
| AC | 对齐检查 | AC=1地址对齐错误产生异常17 |

### 2.7 Pentium引脚功能（考纲要求）

| 引脚 | 功能说明 |
|------|----------|
| **A31~A3** | 地址总线，寻址4GB空间；A2~A0在内部被编码为BE7~BE0 |
| **#BE7~#BE0** | 字节允许，由A2~A0产生，指示8字节数据总线中哪些字节有效 |
| **D63~D0** | 数据总线，64位，读写时按字节偶校验 |
| **D/#C** | 数据/控制，区分当前总线周期传输数据还是控制信息 |
| **M/#IO** | 存储器/IO，区分访问内存还是I/O端口 |
| **W/#R** | 写/读，区分写操作还是读操作 |
| **#NA** | 下一地址有效，用于支持地址流水线操作 |
| **#Cache** | Cache控制，指示目前处于Cache周期 |
| **#KEN** | Cache允许，有效时指示进入突发读周期，将外部数据复制到内部Cache |
| **INTR** | 可屏蔽中断请求 |
| **NMI** | 不可屏蔽中断请求 |

**#Cache 和 #KEN 的区别**（🔥高频题）：
- `#Cache`：输出信号，告诉外部"当前总线周期是可缓存的"
- `#KEN`：输入信号，外部告诉CPU"允许进行Cache填充（突发读）"

### 2.8 三种操作模式 📝

**核心定义**：Pentium有三种操作模式，由CR0.PE位和EFLAGS.VM位共同决定。

```
操作模式
├── 实地址模式 (PE=0)
│   ├── 与8086完全兼容
│   ├── 存储空间1MB，物理地址 = 段地址×16 + 偏移地址
│   └── 不允许分页，线性地址=物理地址
├── 保护虚地址模式 (PE=1, VM=0)
│   ├── 4GB物理空间，64TB逻辑空间
│   └── 地址转换：逻辑地址→(分段)→线性地址→(分页)→物理地址
└── 虚拟8086模式 (PE=1, VM=1)
    ├── 是一个"程序"的运行方式（而非处理器工作方式）
    ├── 相当于程序运行在实地址方式
    └── 通过分页可将1MB空间映射到4GB物理空间任意位置
```

**实地址模式与虚拟8086模式的关键区别**：
- 实地址方式是**处理器**的工作方式
- 虚拟8086方式是**程序**的运行方式（在保护模式下运行8086程序）

⚠️ **考点**：简答/选择 | ⭐⭐⭐
❌ **易错**：虚拟8086不是独立的模式，它是保护模式下的一种子模式（VM标志置1）

### 2.9 保护模式下物理地址形成过程 📝

**核心定义**：保护模式下采用"段页式"两级地址转换：虚拟地址(逻辑地址) → 线性地址 → 物理地址。

**第一阶段：分段机制（逻辑地址 → 线性地址）**

```
虚拟地址(逻辑地址)
┌─────────────┬──────────────────────┐
│ 段选择符(16位)│   偏移量(32位)        │
└──────┬──────┴──────────────────────┘
       │ 索引(13位)×8 = 偏移
       ↓
   GDT或LDT
┌────────────────────────────────────┐
│ 段基址(32位) │ 段限(20位) │ 属性(12位)│
└──────┬───────┴────────────┴─────────┘
       │ 段基址
       ↓
       ⊕ (加法)  ← 偏移量(32位)
       ↓
   线性地址(32位)
```

**第二阶段：分页机制（线性地址 → 物理地址）**

```
线性地址(32位)
┌──────────┬──────────┬──────────────┐
│页目录索引 │ 页表索引  │  页内偏移    │
│ (10位)   │ (10位)   │   (12位)     │
└────┬─────┴─────┬────┴──────────────┘
     │            │
     │ CR3        │
     ↓            ↓
  页目录表     页表
  (找页表地址)  (找物理页基址)
                 │ 物理页基址
                 ↓
                 ⊕ (加法)  ← 页内偏移(12位)
                 ↓
             物理地址(32位)
```

📝 **画图要点**（考试必须标出的元素）：
1. 虚拟地址分成"段选择符（16位）"和"偏移量（32位）"
2. 段选择符通过描述符表（GDT/LDT）找到段基址
3. 段基址 + 偏移量 = 线性地址（32位）
4. 线性地址分三段：页目录索引（10位）+ 页表索引（10位）+ 页内偏移（12位）
5. 通过CR3 → 页目录表 → 页表 → 物理页基址
6. 物理页基址 + 页内偏移 = 物理地址（32位）

**考试写法**：
Pentium保护模式下地址转换分两个阶段：第一阶段，分段机制将逻辑地址（段选择符+偏移量）转换为32位线性地址——由段选择符查GDT/LDT得到段描述符，取出32位段基址，加上32位偏移量得到线性地址；第二阶段，分页机制将线性地址转换为物理地址——将32位线性地址分为10位页目录索引、10位页表索引、12位页内偏移，通过查页目录表和页表找到物理页基址，加上页内偏移得到最终32位物理地址。

⚠️ **考点**：简答+画图 | ⭐⭐⭐ 🔥高频

## 3. 高频计算题型与模板

### 3.1 实地址模式物理地址计算

**公式**：
$$物理地址 = 段地址 \times 16 + 偏移地址$$

**解题步骤**：
1. 将段寄存器值（16位）左移4位（即×16，或末尾添0）
2. 加上偏移地址（16位）
3. 结果为20位物理地址

**示例**（练习题第8题）：

题目：CS=3200H，IP=0FF00H，求指令存放的物理地址？

Step 1：段基址 = 3200H × 16 = 32000H

Step 2：物理地址 = 32000H + 0FF00H = 3FF00H

答：物理地址为 3FF00H

补充：CS和IP的值是否唯一？**不唯一**。不同的CS:IP组合可以对应同一个物理地址。

❌ **易错**：段地址左移4位，不是乘以10（十进制）！16进制段地址后面加一个0即可。

---

### 3.2 逻辑地址 → 物理地址（示例）

**示例**（练习题第7题）：逻辑地址2000:2345H对应的物理地址是多少？

Step 1：段基址 = 2000H × 16 = 20000H

Step 2：物理地址 = 20000H + 2345H = **22345H**

### 题目42：Pentium实地址模式与8086、虚拟8086的异同

**题目**：Pentium实地址模式的特点，与8086工作模式、虚拟8086模式的异同？

**解答**：

**Pentium实地址模式特点**：
- 与8086兼容，基本体系结构相同
- 存储空间1MB，物理地址 = 段地址×16 + 偏移地址
- 不允许分页，线性地址就是物理地址
- 可使用8086没有的32位寄存器、更多寻址方式和大部分32位指令

**与8086工作模式相同点**：地址计算方式相同（段×16+偏移），存储空间同为1MB

**与8086工作模式不同点**：Pentium实地址模式可使用32位寄存器和更多寻址方式

**实地址模式 vs 虚拟8086模式**：
- 实地址方式是**处理器**的工作方式（整个CPU都在实地址方式下运行）
- 虚拟8086方式是**程序**的运行方式（CPU在保护方式下，某个程序以8086方式运行）
- 虚拟8086可通过分页将1MB空间映射到4GB物理空间任意位置；实地址模式不能

**考察点**：操作模式（2.8节）

### 题目43：IDTR、GDTR、LDTR

**题目**：IDTR、GDTR和LDTR分别代表什么寄存器？其内容是什么信息？有什么作用？

**解答**：
- **GDTR**（全局描述符表寄存器，48位）：保存GDT的32位基地址和16位界限。用于在分段机制中定位全局描述符表，整个系统只有一个GDT。
- **IDTR**（中断描述符表寄存器，48位）：保存IDT的32位基地址和16位界限。用于定位中断描述符表，将中断向量与中断服务程序入口地址联系起来，整个系统只有一个IDT。
- **LDTR**（局部描述符表寄存器，16+64位）：包含16位选择符和64位描述符（含基址、界限、属性）。用于定位当前任务的局部描述符表，每个任务各有一个LDT。

**考察点**：系统地址寄存器（2.3节）

### 题目47/50：GDT数据访问寻址过程

**题目**：简述Pentium通过GDT访问数据段的寻址过程。

**解答**：
① 逻辑地址由16位段选择符和32位偏移量组成；
② 从段选择符的高13位（索引字段）取出描述符索引，乘以8得到偏移量；
③ 将GDTR中保存的GDT基地址加上该偏移量，找到对应的段描述符（8字节）；
④ 从段描述符中取出32位段基址；
⑤ 段基址 + 32位偏移量 = 32位线性地址；
⑥ 若分页启用，再通过页目录和页表将线性地址转换为物理地址；若不分页，线性地址即物理地址。

**考察点**：保护模式地址转换（2.9节）

### [简答2] GDTR/LDTR/IDTR/TR的含义及内容 🔥高频

**规范答法**：

### [简答3] Pentium保护模式下物理地址形成过程 📝

**规范答法**：
Pentium保护模式下采用段页式两级地址转换：
第一阶段（分段）：逻辑地址由16位段选择符和32位偏移量组成。用段选择符的高13位作为索引，从GDTR（或LDTR）指向的描述符表中取出段描述符，获得32位段基址，段基址加偏移量得到32位线性地址。
第二阶段（分页）：32位线性地址被分为三部分——高10位为页目录索引，中间10位为页表索引，低12位为页内偏移。由CR3找到页目录表，用页目录索引找到页表地址，再用页表索引找到物理页基址，物理页基址加页内偏移得到最终物理地址。

### [简答4] Pentium三种操作模式的区别

**规范答法**：
（1）实地址模式：与8086兼容，存储空间1MB，物理地址=段地址×16+偏移地址，不支持分页；
（2）保护虚地址模式：4GB物理空间，64TB逻辑空间，采用分段+分页两级地址转换机制，支持内存保护和虚拟内存；
（3）虚拟8086模式：在保护模式下运行8086程序，是程序的运行方式而非处理器工作方式，通过分页可将1MB地址空间映射到4GB物理空间任意位置。

## 6. 易错点汇总

- ❌ 易错：Pentium有3种工作方式
  ✅ 正确：Pentium有**4种**工作方式：实地址、保护虚地址、虚拟8086、**系统管理**

- ❌ 易错：虚拟8086模式是处理器的工作方式
  ✅ 正确：虚拟8086方式是**程序**的运行方式；实地址方式才是处理器的工作方式

- ❌ 易错：段描述符是4字节
  ✅ 正确：段描述符是**8字节（64位）**；8086中断向量表中每个向量才是4字节

- ❌ 易错：GDTR和LDTR结构相同
  ✅ 正确：GDTR/IDTR是48位（基址32+界限16）；LDTR/TR是**16+64位**（含选择符和完整描述符）

- ❌ 易错：Pentium外部数据总线是32位
  ✅ 正确：Pentium外部数据总线是**64位**（D63~D0），但仍是**32位机**（内部处理32位）

- ❌ 易错：#BE7~#BE0 由程序控制
  ✅ 正确：#BE7~#BE0 由CPU内部地址线A2~A0**自动编码产生**，程序员不直接控制

- ❌ 易错：分页后物理地址 = 线性地址
  ✅ 正确：只有**不启用分页**时线性地址=物理地址；启用分页（CR0.PG=1）时需经过两级页表转换

- ❌ 易错：U、V两条流水线都能执行所有指令
  ✅ 正确：U流水线执行全部指令；V流水线只执行**简单**整数指令和交换类浮点指令

## 7. 一页纸速记

**核心公式**：
- 实地址模式：$物理地址 = 段地址 \times 16 + 偏移地址$
- 保护模式：$线性地址 = 段基址 + 偏移量$，再经页表 → 物理地址

**必背结论**：
- 超标量=2（U+V双流水线）；整数5级，浮点8级
- 分立Cache：指令8K + 数据8K
- 段描述符 = **8字节** = 基址32 + 段限20 + 属性12
- GDTR/IDTR = **48位**（基址32+界限16）；LDTR/TR = **16+64位**
- 三种模式：PE=0→实地址；PE=1,VM=0→保护；PE=1,VM=1→虚拟8086
- 线性地址三段：页目录(10) + 页表(10) + 页内偏移(12)

**最易混淆的区别**：
| 对比项 | A | B |
|--------|---|---|
| GDTR vs LDTR | 48位，全局，整个系统一个 | 16+64位，局部，每任务一个 |
| 实地址模式 vs 虚拟8086 | 处理器工作方式 | 程序运行方式 |
| #Cache vs #KEN | CPU输出"这是可缓存周期" | 外部输入"允许Cache填充" |
| U流水线 vs V流水线 | 执行全部指令 | 只执行简单整数指令 |
