## 1. 知识体系

> 本章围绕"可编程接口芯片"展开，核心问题是：CPU如何通过标准芯片与多种外设高效通信。三块芯片各司其职：**8255A** 解决并行I/O（同时传多位数据）；**8253** 解决精确定时/计数（产生周期信号或统计脉冲数）；**8251A** 解决串行通信（一位一位传数据）。这三块芯片在综合题中经常联合出现（例如第120题的恒温箱系统：8255A接键盘和显示，8253产生定时中断和声音，8251A则用于串口通信）。学习方法一致：先看引脚→理解端口地址形成→掌握控制字格式→会写初始化程序→看典型应用例题。

```

├── 7.1  8255A  并行I/O
│   ├── 内部结构（数据端口/控制逻辑/缓冲器/组控制）
```

## 2. 核心知识点精讲

### 2.1 8255A 内部结构与引脚

**核心定义**：8255A 是通用可编程并行I/O接口芯片，有3个8位数据端口（A、B、C），分为A组（A口+C口高4位）和B组（B口+C口低4位）。

**内部四大部件：**

```
       CPU侧                                外设侧
  ┌─────────────┐                    ┌──────────────┐
  │  D7~D0      │◄──► 数据总线缓冲器 ◄──►│              │
  │  RD/WR      │                   │  端口A (PA7~0) │
  │  A1/A0      │──► 读/写控制逻辑 ──►│  端口B (PB7~0) │
  │  CS/RESET   │                   │  端口C (PC7~0) │
  └─────────────┘      A组/B组控制   └──────────────┘
```

| 部件 | 功能 |
|------|------|
| 数据总线缓冲器 | 三态8位双向缓冲器，D7-D0接系统数据总线 |
| 读/写控制逻辑 | 接收CPU地址和控制信号，向A/B两组控制发送命令 |
| A组控制 | 管理端口A和C口高4位 |
| B组控制 | 管理端口B和C口低4位 |

**端口地址选择（A1/A0）：**

| A1 | A0 | 选择端口 |
|----|----|----------|
| 0  | 0  | A口（数据） |
| 0  | 1  | B口（数据） |
| 1  | 0  | C口（数据） |
| 1  | 1  | 控制口 |

⚠️ **考点**：简答题 | ⭐⭐⭐  
❌ **易错**：控制口地址是 A1=1、A0=1，不是某个单独数据口。CS=0 才有效（片选低有效）。

---

### 2.2 8255A 方式选择控制字

**核心定义**：D7=1 时为方式选择控制字，用来设置各端口的工作方式和输入/输出方向。

**控制字格式（必背）：**

```
D7  D6  D5  D4  D3  D2  D1  D0
 1  [A组方式] [C高] [A口] [B组] [C低] [B口]
     SC1 SC0   C高   A口   B组   C低   B口
```

| 位 | 含义 | 编码规则 |
|----|------|----------|
| D7 | 特征位 | 固定为 **1** |
| D6-D5 | A组方式选择 | 00=方式0，01=方式1，1×=方式2 |
| D4 | A口 C高4位 I/O | 0=输出，1=输入 |
| D3 | A口 I/O | 0=输出，1=输入 |
| D2 | B组方式选择 | 0=方式0，1=方式1 |
| D1 | C口低4位 I/O | 0=输出，1=输入 |
| D0 | B口 I/O | 0=输出，1=输入 |

**考试写法**：先确定每位的值，从D7往D0列出，最后转成十六进制。

**示例**：A口方式0输出，B口方式0输入，C口全部输出。  
→ D7=1，D6D5=00（方式0），D4=0（C高输出），D3=0（A输出），D2=0（方式0），D1=0（C低输出），D0=1（B输入）  
→ `1000_0001B = 81H`

⚠️ **考点**：综合题（必考）| 🔥 高频  
❌ **易错**：D3是A口的输入/输出，**0=输出，1=输入**，与直觉相反；D4是C口高4位，不要和D3搞混。

---

### 2.3 8255A C口置位/复位控制字

**核心定义**：D7=0 时为C口置位/复位控制字，可单独对C口某一位进行置1或清0，不影响其他位。

**控制字格式：**

```
D7  D6  D5  D4   D3  D2  D1   D0
 0   ×   ×   ×  [位选择 D3D2D1]  [0=复位 1=置位]
```

| D3 D2 D1 | 选中引脚 |
|----------|----------|
| 000 | PC0 |
| 001 | PC1 |
| ... | ... |
| 111 | PC7 |

**示例**：对PC5置位 → D7=0，D3D2D1=101，D0=1 → `0000_1011B = 0BH`

⚠️ **考点**：综合题 | ⭐⭐  
❌ **易错**：两种控制字都写入控制端口（A1=A0=1），靠D7区分类型，D7=1是方式字，D7=0是C口置位/复位字。

---

### 2.4 8255A 三种工作方式

| 方式 | 名称 | 适用端口 | 特点 |
|------|------|----------|------|
| 方式0 | 基本输入/输出 | A、B、C口 | 无握手，直接读写，最简单 |
| 方式1 | 选通输入/输出 | A口、B口 | 有握手信号（STB/IBF或OBF/ACK），支持中断 |
| 方式2 | 双向选通 | 仅A口 | 同时支持输入和输出，有5个联络信号 |

**方式1联络信号（重要）：**

| 信号 | 方向 | 有效电平 | 含义 |
|------|------|----------|------|
| STB（选通）| 外设→8255A | 低 | 外设将数据送入锁存器 |
| IBF（输入缓冲满）| 8255A→外设 | 高 | 告知外设数据已锁存 |
| OBF（输出缓冲满）| 8255A→外设 | 低 | 告知外设CPU已输出数据 |
| ACK（响应）| 外设→8255A | 低 | 外设已取走数据 |
| INTR（中断请求）| 8255A→CPU | 高 | 请求CPU处理 |
| INTE（中断允许）| 内部控制 | — | A口由PC4/PC6控制，B口由PC2控制 |

⚠️ **考点**：简答题 | ⭐⭐⭐

---

### 2.5 8253 基本功能与引脚

**核心定义**：8253 是可编程计数器/定时器，有3个独立的16位计数器，每个计数器有6种工作方式，支持二进制/BCD码计数。

**关键参数：**
- 3个独立计数器（通道0、1、2）
- 每通道有独立的 CLK（时钟输入）、GATE（门控）、OUT（输出）
- 计数频率：0～2MHz
- 计数极限：二进制65536，十进制10000

**端口选择（A1/A0）：**

| A1 | A0 | 选择端口 |
|----|----|----------|
| 0  | 0  | 计数器0 |
| 0  | 1  | 计数器1 |
| 1  | 0  | 计数器2 |
| 1  | 1  | 控制口 |

**GATE 信号的作用：**
- 方式0、4：GATE=0时停止计数，GATE=1时允许计数
- 方式1、5：GATE上升沿触发启动
- 方式2、3：GATE=0时暂停并复位，上升沿后重新启动

⚠️ **考点**：简答题/综合题 | ⭐⭐⭐

---

### 2.6 8253 控制字格式

**控制字格式（8位，写入控制口）：**

```
D7  D6  D5  D4  D3  D2  D1  D0
SC1 SC0 RL1 RL0  M2  M1  M0  BCD
```

| 字段 | 位 | 含义 |
|------|----|------|
| SC1SC0 | D7D6 | 计数器选择：00=计数器0，01=1，10=2，11=非法 |
| RL1RL0 | D5D4 | 读/写格式：00=锁存命令，01=只读写低8位，10=只读写高8位，11=先低后高 |
| M2M1M0 | D3D2D1 | 工作方式：000=方式0，001=方式1，×10=方式2，×11=方式3，100=方式4，101=方式5 |
| BCD | D0 | 计数制：0=二进制，1=BCD码 |

**示例（练习题91题第1问）**：通道0，方式3，先低后高，二进制  
→ SC=00，RL=11，M=011，BCD=0 → `0011_0110B = 36H`

⚠️ **考点**：综合题/计算题 | 🔥 高频  
❌ **易错**：方式3的M编码是 ×11（即011或111均可），不要写成100；RL=00是"锁存命令"不是"不读写"。

---

### 2.7 8253 六种工作方式

| 方式 | 名称 | 启动方式 | 计数值有效 | OUT波形特点 |
|------|------|----------|------------|-------------|
| 0 | 计数结束中断 | 软件（写入计数值） | 一次 | 计数期间OUT=0，到0后变1 |
| 1 | 可编程单稳 | 硬件（GATE上升沿） | 多次 | 触发后OUT=0持续N个CLK |
| 2 | 脉冲频率发生器 | 软件/硬件 | 重复 | 每N个CLK输出一个负脉冲（低1个CLK） |
| 3 | 方波发生器 | 软件/硬件 | 重复 | 占空比50%方波，高N/2低N/2 |
| 4 | 软件触发选通 | 软件 | 一次 | 计数到0时OUT输出一个负脉冲 |
| 5 | 硬件触发选通 | 硬件（GATE上升沿） | 多次 | 同方式4但由硬件触发 |

**产生方波必选方式3** ← 综合题高频考点

⚠️ **考点**：简答题/综合题 | 🔥 高频  
❌ **易错**：方式2和方式3都重复有效，区别在于：方式2是脉冲（占空比不等），方式3是方波（占空比50%）。

### 2.8 8253 计数初值计算

**公式：**

$$N_{计数} = \frac{f_{CLK}}{f_{OUT}}$$

**定时方式（已知定时时长）：**

$$N_{计数} = \frac{T_{OUT}}{T_{CLK}} = T_{OUT} \times f_{CLK}$$

**解题步骤：**
1. 确定输入时钟频率 $f_{CLK}$（题目给出或从电路图读出）
2. 确定目标输出频率/周期
3. 计算 $N$
4. 将 $N$ 转成十六进制（若BCD计数则保留十进制形式）
5. 分低8位和高8位写入

**示例（练习题91题第1问）**：CLK0=2MHz，输出1kHz方波  
→ $N = 2\text{MHz} / 1\text{kHz} = 2000 = 07D0H$  
→ 低8位：D0H，高8位：07H

⚠️ **考点**：计算题 | 🔥 高频

### 2.9 8253 初始化编程模板

**三步骤（固定顺序）：**

```asm
; 步骤1：写控制字到控制口（A1=A0=1）
MOV  AL, 控制字
OUT  控制口地址, AL

; 步骤2：写计数初值低8位到对应通道
MOV  AL, 初值低8位
OUT  通道地址, AL

; 步骤3：写计数初值高8位（若RL=11）
MOV  AL, 初值高8位
OUT  通道地址, AL
```

**完整示例（练习题第91题第1小问）**：通道0，方式3，CLK=2MHz，OUT=1kHz方波，端口40H-43H

```asm
; N = 2000 = 07D0H
MOV  AL, 36H        ; 控制字：00_11_011_0（通道0，先低后高，方式3，二进制）
OUT  43H, AL        ; 写入控制口（43H = A1A0=11）

MOV  AL, 0D0H       ; 计数初值低8位
OUT  40H, AL        ; 写入计数器0（40H = A1A0=00）

MOV  AL, 07H        ; 计数初值高8位
OUT  40H, AL
```

⚠️ **考点**：综合题/编程题 | 🔥 高频  
❌ **易错**：控制字必须先写，初值后写；控制字写入控制口，初值写入对应通道数据口（两者地址不同）。

### 2.10 8251A 串行通信基础

**三种通信方向：**

```
单工：     A站 ──────────────► B站
半双工：   A站 ──────────────► B站
                B站 ──────── A站（不能同时）
全双工：   A站 ══════════════ B站（双向同时）
```

**异步通信帧格式（必背）：**

```
[空闲位=1] [起始位=0] [D0 D1 D2 D3 D4 D5 D6 D7] [奇偶校验位] [停止位=1或2个] [空闲位=1]
```

- 起始位：固定1位，低电平（0）
- 数据位：5~8位，低位在前
- 奇偶校验位：可选，1位
- 停止位：1位、1.5位或2位，高电平（1）

**每帧总位数** = 1（起始）+ 数据位数 + 校验位（0或1）+ 停止位数

**波特率相关：**

$$f_{时钟} = \text{波特率} \times \text{波特率因子（1/16/64）}$$

$$\text{每秒传送字节数} = \frac{\text{波特率}}{\text{每帧总位数}}$$

⚠️ **考点**：计算题 | ⭐⭐⭐

### 2.11 8251A 方式选择控制字

**格式（D7=高位为停止位/同步字设置）：**

```
D7  D6  D5  D4  D3  D2  D1  D0
[停止位/同步字]  [校] [字符长度] [通信方式]
```

| 位 | D1D0 | 含义 |
|----|------|------|
| 通信方式 | 00 | 同步方式 |
| | 01 | 异步×1（波特率因子1） |
| | 10 | 异步×16（波特率因子16） |
| | 11 | 异步×64（波特率因子64） |

| 位 | D3D2 | 含义 |
|----|------|------|
| 字符长度 | 00 | 5位 |
| | 01 | 6位 |
| | 10 | 7位 |
| | 11 | 8位 |

| 位 | D4 | 含义 |
|----|----|------|
| 奇偶校验允许 | 0 | 不用校验 |
| | 1 | 使用奇偶校验 |

| 位 | D5 | 含义（校验方式）|
|----|----|------|
| 奇偶选择 | 0 | 奇校验 |
| | 1 | 偶校验 |

| 位 | D7D6（异步） | 含义 |
|----|-------------|------|
| 停止位 | 00 | 无效 |
| | 01 | 1位停止位 |
| | 10 | 1.5位停止位 |
| | 11 | 2位停止位 |

**示例（练习题第26题/课件例7.3.1）**：异步×16，7位字符，偶校验，2位停止位  
→ D1D0=10（×16），D3D2=10（7位），D4=1（允许校验），D5=1（偶校验），D7D6=11（2位停止位）  
→ `1111_1010B = 0FAH`

⚠️ **考点**：综合题 | 🔥 高频

### 2.12 8251A 初始化流程

**流程（顺序严格）：**

```
系统 RESET
    │
    ▼
写入方式选择控制字（到控制端口）
    │
    ├─── 异步方式？─── YES ──────────────────────────┐
    │                                               │
    NO                                              │
    │                                               │
    ▼                                               │
写第1个同步字符                                      │
    │                                               │
    ├─── 单同步字符？─── YES ────────────────────────┤
    │                                               │
    NO                                              │
    │                                               │
    ▼                                               │
写第2个同步字符                                      │
    │                                               │
    └──────────────────► 写入操作命令控制字（到控制端口）│
                              ◄──────────────────────┘
                              │
                              ├─── 是否复位？YES → 回到写方式字
                              │
                              NO
                              │
                              ▼
                         传送数据
```

**关键记忆**：异步方式只需写方式字→命令字；同步方式还要在中间写同步字符。

**操作命令控制字关键位（考试常用）：**

| 位 | 名称 | 含义 |
|----|------|------|
| D0 | TXEN | 1=发送允许 |
| D1 | DTR | 1=数据终端准备好 |
| D2 | RXE | 1=接收允许 |
| D4 | ER | 1=错误标志复位（清PE/OE/FE） |

**示例（课件例7.3.1）**：允许发送和接收，清出错标志  
→ D0=1（TXEN），D2=1（RXE），D4=1（ER）  
→ 其余位置0 → `0001_0101B` 但练习题答案为 `37H = 0011_0111B`（含DTR和RTS）

⚠️ **考点**：综合题/简答题 | 🔥 高频  
❌ **易错**：方式选择控制字和操作命令控制字写入的是**同一个控制端口地址**，芯片靠接收顺序区分（先收到的是方式字）。

## 3. 高频计算题型与模板

### 3.1 8253 计数初值计算

**公式：**

$$N = \frac{f_{CLK}}{f_{OUT}} \quad \text{（频率已知）}$$

$$N = T_{OUT} \times f_{CLK} \quad \text{（时长已知）}$$

**解题步骤：**
1. 读出题目给定的 $f_{CLK}$（或从电路读）
2. 读出目标 $f_{OUT}$ 或 $T_{OUT}$
3. 套公式计算 $N$
4. 转换为十六进制（二进制计数）或BCD（十进制计数）
5. 拆分低8位和高8位

**示例（练习题第91题第2问）**：通道1，方式2，CLK1=1MHz，OUT=1kHz脉冲波

Step 1：$f_{CLK} = 1\text{MHz} = 10^6\text{Hz}$

Step 2：$f_{OUT} = 1\text{kHz} = 10^3\text{Hz}$

Step 3：$N = 10^6 / 10^3 = 1000 = 03E8H$

Step 4：低8位 = E8H，高8位 = 03H

答：控制字（通道1，方式2，先低后高，二进制）= `01110100B = 74H`；初值低8位 E8H，高8位 03H

❌ **易错**：16位计数值要分两次写入（先低8位再高8位）；若只用低8位（RL=01），高8位自动清0。

---

### 3.2 8251A 传输时间/字节数计算

**公式：**

$$\text{每帧位数} = 1 + \text{数据位} + \text{校验位} + \text{停止位}$$

$$\text{每秒字节数} = \frac{\text{波特率}}{\text{每帧位数}}$$

$$\text{传输时间} = \frac{\text{字节数} \times \text{每帧位数}}{\text{波特率}}$$

**示例（练习题第102题）**：波特率115200，8位数据，无校验，1位停止位，每秒传多少字节？

Step 1：每帧位数 = 1（起始）+ 8（数据）+ 0（无校验）+ 1（停止）= 10位

Step 2：每秒字节数 = 115200 ÷ 10 = **11520 字节/秒**

❌ **易错**：起始位固定1位，不要忘记计入总位数。

---

### 3.3 8255A 控制字计算（从需求推控制字）

**解题步骤：**
1. 确认各端口工作方式（方式0/1/2）
2. 确认各端口输入/输出方向
3. 按 D7-D0 顺序填表：D7固定为1，D6D5填A组方式，D4填C高I/O，D3填A口I/O，D2填B组方式，D1填C低I/O，D0填B口I/O
4. 转成16进制

**示例（课件例7.1.1）**：A、B、C口均方式0输出

D7=1，D6D5=00（方式0），D4=0（C高输出），D3=0（A输出），D2=0（方式0），D1=0（C低输出），D0=0（B输出）

→ `1000_0000B = 80H`

❌ **易错**：**0=输出，1=输入**（对I/O方向位来说），与直觉相反。

## 4. 练习题精析

### 题目91（来自练习题）

**题目**：某系统中有一片8253，计数器0至控制口地址依次为40H-43H，请按要求编程：  
(1) 通道0：方式3，CLK0=2MHz，OUT0输出1KHz方波  
(2) 通道1：方式2，CLK1=1MHz，OUT1输出1KHz脉冲波  
(3) 通道2：方式4，CLK2=OUT1，计数值为1000，计数到0时输出一个控制脉冲

**解答**：

**(1) 通道0：**
- 控制字：SC=00，RL=11（先低后高），M=011（方式3），BCD=0 → `0011_0110B = 36H`
- 计数初值：N = 2MHz / 1kHz = 2000 = `07D0H`

```asm
MOV  AL, 36H    ; 通道0，方式3，先低后高，二进制
OUT  43H, AL    ; 写控制口
MOV  AL, 0D0H   ; 低8位（D0H）
OUT  40H, AL
MOV  AL, 07H    ; 高8位（07H）
OUT  40H, AL
```

**(2) 通道1：**
- 控制字：SC=01，RL=11，M=010（方式2），BCD=0 → `0111_0100B = 74H`
- 计数初值：N = 1MHz / 1kHz = 1000 = `03E8H`

```asm
MOV  AL, 74H    ; 通道1，方式2，先低后高，二进制
OUT  43H, AL
MOV  AL, 0E8H   ; 低8位
OUT  41H, AL
MOV  AL, 03H    ; 高8位
OUT  41H, AL
```

**(3) 通道2：**
- 控制字：SC=10，RL=11，M=100（方式4），BCD=0 → `1011_1000B = 0B8H`
- 计数初值：1000 = `03E8H`

```asm
MOV  AL, 0B8H   ; 通道2，方式4，先低后高，二进制
OUT  43H, AL
MOV  AL, 0E8H   ; 低8位
OUT  42H, AL
MOV  AL, 03H    ; 高8位
OUT  42H, AL
```

**考察点**：8253控制字计算、计数初值计算、分高低8位写入。

### 题目95（来自练习题）

**题目**：8255A的A口-控制口地址为60H-63H。编一段程序使从PC5输出一个负脉冲。脉冲宽度不够时应如何解决？

**解答**：

PC5先置1再置0再置1，中间产生负脉冲。  
置PC5=1的控制字（C口置位/复位字）：D7=0，D3D2D1=101（PC5），D0=1（置位）→ `0000_1011B = 0BH`  
置PC5=0的控制字：D7=0，D3D2D1=101，D0=0（复位）→ `0000_1010B = 0AH`

```asm
MOV  AL, 0BH    ; PC5置1（置位控制字）
OUT  63H, AL    ; 写控制口（63H）

MOV  AL, 0AH    ; PC5置0（复位控制字）
OUT  63H, AL

MOV  AL, 0BH    ; PC5再置1
OUT  63H, AL
```

**脉冲宽度不够时**：在PC5置0和再置1之间插入软件延时（调用延时子程序），延长低电平持续时间。

**考察点**：C口置位/复位控制字格式，D7=0为标志，D3D2D1选引脚，D0选置位/复位。

### 题目98（来自练习题）

**题目**：简述8251A初始化的一般步骤。

**解答**：

② 若为**同步方式**，则继续写入同步字符（1个或2个，由方式字D7D6决定）。

③ 向控制端口写入**操作命令控制字**，使能发送（TXEN=1）和/或接收（RXE=1），可同时清除错误标志（ER=1）。

④ 之后即可通过数据端口进行收发数据。若需要重新初始化，在命令字中将IR位（D6）置1执行内部复位，再重复上述过程。

**考察点**：初始化顺序，方式字在前、命令字在后，同步方式需额外写同步字符。

### 题目102（来自练习题）

**题目**：8251A异步方式，RxC=TxC=115200Hz，波特率因子为1，格式为115200、8、N、1（波特率115200，8数据位，无奇偶校验，1停止位）。计算每秒可传送多少字节，并画出传送字符'B'的帧格式。

**解答**：

每帧位数 = 1（起始）+ 8（数据）+ 0（无校验）+ 1（停止）= **10位**

每秒字节数 = 115200 ÷ 10 = **11520 字节/秒**

'B' = 42H = 0100_0010B（低位在先传送）

帧格式（时序从左到右）：

```
空闲  起始  D0  D1  D2  D3  D4  D5  D6  D7  停止  空闲
 1     0    0   1   0   0   0   0   1   0    1     1
```

注：低位D0先发，'B'=42H=0100_0010B，所以D0=0,D1=1,D2=0,D3=0,D4=0,D5=0,D6=1,D7=0

**考察点**：帧格式计算，低位先发，每秒字节数公式。

### 题目83（来自练习题）

**题目**：对8253A写入计数值0是最大计数值，为什么？在二进制下相当于多少？CLK=1.193MHz，周期工作方式下最大产生每秒多少次？

**解答**：

写入0时，第一个CLK到来，计数器从0减1变为65535（下溢循环），OUT此时不变化，因此相当于比65535还多1，即**65536**次计数后OUT才变化一次。

二进制最大计数值 = **65536**

最大频率 = 1.193MHz ÷ 65536 ≈ **18.2 Hz**（每秒约18.2次）

**考察点**：计数值0的特殊含义，16位下溢为65536，定时最大频率计算。

## 5. 简答题备考模板

### 8255A 的内部结构与功能

**规范答法**：
8255A 是可编程并行I/O接口芯片，内部由四部分组成：
① **数据端口**：3个8位端口（A、B、C），分A组（A口+C口高4位）和B组（B口+C口低4位）管理；
② **数据总线缓冲器**：三态8位双向缓冲器，连接系统数据总线D7-D0；
③ **读/写控制逻辑**：接收CPU的地址和控制信号（RD、WR、A1、A0、CS），向A/B两组控制发送命令；
④ **A组/B组控制电路**：分别接收控制字，向对应端口发送适当的控制信号。

---

### 8255A 三种工作方式

**规范答法**：
8255A 支持三种工作方式：
① **方式0（基本输入/输出）**：适用于A、B、C口，无握手信号，CPU直接读写端口，适用于外设随时准备好的场合；
② **方式1（选通输入/输出）**：适用于A口和B口，有握手联络信号（输入：STB/IBF/INTR；输出：OBF/ACK/INTR），支持中断方式传送；
③ **方式2（双向选通）**：仅适用于A口，同时支持双向数据传送，使用5个联络信号，INTRA在输入输出时均可产生。

---

### 8253 的六种工作方式

**规范答法**：
8253 有六种工作方式，以启动方式和计数值使用次数区分：
- **方式0（计数结束中断）**：软件启动，一次有效，计数到0时OUT变高；
- **方式1（可编程单稳）**：硬件启动（GATE上升沿），多次有效；
- **方式2（脉冲频率发生器）**：软硬件均可，重复有效，输出连续负脉冲；
- **方式3（方波发生器）**：软硬件均可，重复有效，输出占空比50%方波；
- **方式4（软件触发选通）**：软件启动，一次有效；
- **方式5（硬件触发选通）**：硬件启动，多次有效。

---

### 8251A 初始化步骤

**规范答法**：
8251A 初始化步骤如下：
① 系统复位（RESET）后，向控制端口写入**方式选择控制字**，确定同步/异步方式、字符长度、校验方式、停止位数；
② 若为同步方式，继续写入1个或2个同步字符（由方式字决定）；
③ 向控制端口写入**操作命令控制字**，使能发送（TXEN=1）和接收（RXE=1），并清除出错标志（ER=1）。

❌ 注意：两种控制字写入同一控制端口，芯片靠接收顺序区分，不能颠倒。

---

### 串行通信异步帧格式

**规范答法**：
异步通信以字符为单位进行传输，每帧（每个字符）包含：
- **起始位**：1位，低电平（0），标志一帧的开始；
- **数据位**：5~8位，低位先传；
- **奇偶校验位**：0或1位（可选）；
- **停止位**：1位、1.5位或2位，高电平（1），标志一帧结束。

帧与帧之间可有空闲位（高电平），也可连续发送。

## 6. 易错点汇总

- ❌ 易错：8255A 方式选择控制字中，I/O方向位 **0=输出，1=输入**。  
  ✅ 正确：这与直觉相反，需要死记"0=输出（Output），1=输入（Input）"。

- ❌ 易错：8255A 两种控制字都往控制口写，靠D7区分。以为控制口只能写一种数据。  
  ✅ 正确：D7=1是方式选择字，D7=0是C口置位/复位字，写入同一地址（A1=A0=1的控制口），芯片内部靠D7判断。

- ❌ 易错：8253 控制字写入控制口，计数初值写入对应通道数据口。混淆两者地址。  
  ✅ 正确：控制口是A1=A0=1的地址；计数器0是A1=A0=00，1是01，2是10，要写对端口。

- ❌ 易错：8253 方式3的控制字M字段写成100。  
  ✅ 正确：方式3的M编码是 ×11（即M2M1M0 = 011 或 111 均可），不是100（那是方式4）。

- ❌ 易错：认为8253写入计数值0时计数器不工作。  
  ✅ 正确：写入0相当于最大计数值65536（二进制）或10000（BCD），是最长的计数周期。

- ❌ 易错：8251A初始化时先写操作命令字再写方式选择字。  
  ✅ 正确：必须先写方式选择控制字，再写操作命令控制字，顺序不可颠倒。

- ❌ 易错：异步通信计算每秒字节数时忘记加起始位（只算数据位+停止位）。  
  ✅ 正确：每帧总位数 = 1（起始）+ 数据位 + 校验位 + 停止位，必须全部计入。

- ❌ 易错：8253 产生方波选方式2。  
  ✅ 正确：方式2是脉冲发生器（每N个CLK出一个窄脉冲），方式3才是方波发生器（占空比50%）。

## 7. 一页纸速记

**核心公式：**

- 8253 计数初值：$N = f_{CLK} / f_{OUT}$（或 $N = T_{OUT} \times f_{CLK}$）
- 8251A 每秒字节：字节/s = 波特率 ÷ 每帧位数
- 8251A 每帧位数 = 1 + 数据位 + 校验位 + 停止位

**8255A 控制字D7-D0口诀：**
> **1** [A方式:D6D5] [C高I/O:D4] [A口I/O:D3] [B方式:D2] [C低I/O:D1] [B口I/O:D0]
> I/O位：**0=输出，1=输入**

**8253 控制字D7-D0口诀：**
> [SC:D7D6] [RL:D5D4] [M:D3D2D1] [BCD:D0]
> RL=11 → 先低后高；M=011 → 方式3方波

**8251A 初始化顺序：**
> RESET → 方式选择字 → （同步字符） → 操作命令字

**必背结论：**
- 方式选择控制字：D7=1；C口置位/复位字：D7=0
- 8253 要产生方波 → 必选方式3
- 8253 端口：计数器0/1/2 → A1A0=00/01/10；控制口 → A1A0=11
- 8255A 端口：A口/B口/C口 → A1A0=00/01/10；控制口 → A1A0=11
- 8251A 操作命令字：D0=TXEN（发送），D2=RXE（接收），D4=ER（清错误）

**最易混淆的区别：**
- 8255A 方式1输入中，A口INTE由PC4控制；输出中，A口INTE由PC6控制
- 8253 方式2（脉冲）vs 方式3（方波）：都重复有效，区别在波形—方式3高低各占一半
- 8251A 状态字TXRDY（引脚）需满足3个条件才置位；状态位TXRDY只要输出缓冲区空就置位
