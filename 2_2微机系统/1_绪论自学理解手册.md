## 第一部分：这门课在研究什么？

课程名叫"微机系统"，但这三个字具体指什么？

先从最基础的问题出发：**一台计算机是怎么工作的？**

任何计算机，不管是手机还是服务器，都在做同一件事：**接收输入 → 处理数据 → 产生输出**。为了做到这件事，它需要五个基本部件：

```
运算器（ALU）  ──计算──→  
控制器         ──指挥──→  数据在这五个部件之间流动
存储器         ──存储──→  
输入设备       ──接收──→  
输出设备       ──输出──→  
```

这是冯·诺依曼在1945年提出的体系结构，七十多年过去了，今天的计算机本质上还是这个架构。

**微型计算机**是计算机家族里体积最小、价格最低的一类。它的出现，靠的是**集成电路技术**的进步——把原来要占满一整个房间的"运算器+控制器"，塞进了一块指甲盖大小的芯片里。这块芯片就叫做**微处理器（CPU）**。

这门课研究的就是：**以 Intel 8086 / Pentium 为代表的微处理器，它内部怎么工作，外面怎么连接存储器和外设，怎么用汇编语言控制它。**

## 第二部分：三个层次——微处理器、微型计算机、微型计算机系统

学这门课会反复碰到这三个词，必须分清楚。它们是**同心圆式的包含关系**：

```
┌──────────────────────────────────────────────┐
│              微型计算机系统                    │
│                                              │
│   系统软件（操作系统、编译器……）               │
│   外围设备（显示器、键盘、打印机……）           │
│   电源、机架、面板                            │
│                                              │
│   ┌──────────────────────────────────────┐   │
│   │           微型计算机（裸机）           │   │
│   │                                      │   │
│   │  主存储器（RAM/ROM）                  │   │
│   │  I/O 接口电路                         │   │
│   │  系统总线（AB + DB + CB）              │   │
│   │                                      │   │
│   │   ┌──────────────────────────────┐   │   │
│   │   │    微处理器（CPU）            │   │   │
│   │   │  （一片或少数几片大规模集成    │   │   │
│   │   │   电路组成的中央处理器）       │   │   │
│   │   └──────────────────────────────┘   │   │
│   └──────────────────────────────────────┘   │
└──────────────────────────────────────────────┘
```

用一个生活类比来理解：

- **微处理器** = 汽车的发动机（核心动力，但单独放在那里什么都干不了）
- **微型计算机** = 一辆没有车牌、没有驾驶员的裸车（有车身、有轮子，能动，但还不能上路）
- **微型计算机系统** = 一辆完整可上路的汽车（有驾驶员、有导航、有牌照，能完成运输任务）

三者的**核心区别一句话总结**：

| | 是什么 | 能干什么 |
|---|---|---|
| 微处理器 | CPU芯片本身 | 运算和控制，但不能独立工作 |
| 微型计算机 | CPU + 内存 + I/O接口 + 总线 | 有完整硬件，但没有软件和外设 |
| 微型计算机系统 | 微型计算机 + 外设 + 软件 + 电源等 | 完整可用的系统 |

## 第三部分：微型计算机的四条"血管"——系统总线

微型计算机内部，CPU、内存、I/O接口之间靠**系统总线**连接。总线就是一组公用导线，所有部件都挂在上面，分时复用。

总线分三种：

```
地址总线（AB，Address Bus）
  方向：单向，从 CPU 到 存储器/I/O
  作用：CPU 告诉外部"我要访问哪个地址"
  位数决定：寻址空间大小（N位地址总线 → 最多访问 2^N 个地址单元）
  
数据总线（DB，Data Bus）
  方向：双向，CPU ↔ 存储器/I/O
  作用：传输真正的数据
  位数决定：一次能传多少位数据（8位/16位/32位/64位）
  
控制总线（CB，Control Bus）
  方向：各信号方向不同（有的从CPU出，有的进CPU）
  作用：协调时序和操作类型（读？写？访问内存？访问I/O？中断？）
```

> **为什么要分三种总线？** 因为它们携带的信息性质完全不同：地址是"去哪里"，数据是"传什么"，控制是"怎么传"。分开之后，每条总线可以独立优化宽度和时序。

## 第四部分：CISC vs RISC——指令系统的两条路

CPU的核心是**指令集**：它能理解和执行哪些操作？

历史上出现了两种截然不同的设计哲学：

### CISC——复杂指令系统计算机

**设计思路**：用硬件实现复杂操作，给程序员提供丰富的指令。

想象一个餐厅，菜单上有几百道菜，每道菜都是厨师做好了直接端上来的。程序员写代码很省事（一条指令可以完成很多事），但厨房（硬件）就得很复杂。

特点：
- 指令数量多，指令**长度不等**（有的指令1字节，有的5字节）
- 很多指令可以直接访问内存（`ADD AX, [BX]` 直接把内存里的数加到AX）
- 硬件电路复杂，设计周期长，功耗较大
- **在一条指令执行结束后**才能响应中断（因为指令执行中途状态不稳定）
- 代表：**Intel x86**（也就是这门课研究的主角）、AMD

### RISC——精简指令系统计算机

**设计思路**：指令越少越好，每条指令越简单越好，复杂操作靠多条简单指令组合完成。

想象一个只有五道基础菜的餐厅，但食材质量极好、出菜极快。复杂的菜品要靠搭配组合，但单菜速度很快。

特点：
- 指令数量少，指令**长度固定等长**（便于流水线处理）
- 内存访问只有专用的 Load/Store 指令，其他指令只操作寄存器
- 硬件电路简单，设计周期短，功耗低
- 易于实现并行/流水线，**可以在指令执行的适当地方**响应中断
- 代表：ARM（手机芯片）、MIPS、RISC-V、IBM PowerPC

### 两者对比总结

| | CISC | RISC |
|---|---|---|
| 指令系统 | 复杂，指令多，**不等长** | 简单，指令少，**等长** |
| 内存访问 | 很多指令可访问内存 | 只有专用指令（Load/Store）访问内存 |
| 电路结构 | 复杂，周期长，功耗大 | 简单，周期短，功耗低 |
| 中断响应 | 指令执行**结束后** | 指令执行**适当地方** |
| 典型代表 | Intel x86, AMD | ARM, MIPS, PowerPC |

> **现实中的模糊**：今天的 Intel CPU 在芯片内部已经把 x86 指令"翻译"成类 RISC 的微操作来执行，所以两者的界限越来越模糊。但概念上的区分对理解体系结构仍然重要。

## 第五部分：微处理器发展简史——理解为什么研究8086

知道历史，才能理解为什么选8086作为学习对象。

```
时间       代际     典型芯片              关键突破
─────────────────────────────────────────────────────────
1970s初    4位机    Intel 4004            第一款商用微处理器
1974-77    8位机    Intel 8080/8085       有中断、DMA，能跑BASIC
1978-      16位机   Intel 8086 ←── 本课程  数据线16位，地址线20位（1MB空间）
                   Intel 8088            IBM PC的CPU（准16位）
                   Intel 80286           引入保护模式（24位地址线，16MB）
1980s-     32位机   Intel 80386           32位地址+数据，多任务
                   Intel 80486           集成浮点单元+Cache
                   Pentium ←── 本课程    64位数据线，超标量，首款真流水线
2001-      64位机   Itanium, Core i系列   现代多核CPU
```

**为什么学8086？**

8086诞生于1978年，看起来早已过时，但它是：
1. **最经典的16位CPU**，结构简单清晰，适合学习计算机体系结构
2. **现代x86的祖先**，今天的Intel/AMD CPU仍然兼容8086指令集
3. **分段内存管理的起点**，理解了8086的分段，才能理解Pentium的保护模式

**为什么还学Pentium？**

Pentium代表了32位保护模式，引入了分页、特权级、虚拟内存等现代操作系统的基础机制。这门课的第3、5章专门讲这部分。
