## 1. 知识体系

> 本章核心问题：**Pentium 在保护模式下如何将程序员写的"虚地址"转换为内存中真实的"物理地址"**，同时保证多任务之间互不干扰。

整个地址转换分两大阶段串联完成：

**第一阶段（分段）**：程序产生的逻辑地址（虚地址）= 16位段选择符 + 32位偏移量。段选择符指向段描述符表（GDT 或 LDT）中的某个段描述符，取出段描述符里的32位段基址，与偏移量相加，得到32位**线性地址**。

**第二阶段（分页）**：32位线性地址按 10+10+12 位分成三段，依次查 CR3 指向的**页目录**、**页表**，取出物理页基址，加上12位页内偏移，得到32位**物理地址**。

围绕这两个阶段，需要理解的支撑结构有：段描述符格式、GDT/LDT/IDT 三张表及其对应的寄存器（GDTR/LDTR/IDTR/TR）、特权保护机制（CPL/RPL/DPL）、门描述符、TSS 任务状态段。

```
虚地址 (16位段选择符 + 32位偏移)
    │
    ▼ [分段机制]  查 GDT 或 LDT
    段基址(32位) + 偏移(32位)
    │
    ▼ 32位线性地址
    │
    ▼ [分页机制]  CR3→页目录→页表
    页基址(20位) + 页内偏移(12位)
    │
    ▼ 32位物理地址
```

## 2. 核心知识点精讲

### 2.1 三种地址空间

**核心定义**：
- **虚拟地址（逻辑地址）**：程序员编写程序时使用的地址，保护模式下为48位（16位段选择符 + 32位偏移量），地址空间 $2^{46}$ = 64TB。
- **线性地址**：分段部件输出的32位地址，是连续统一的地址空间（4GB）。
- **物理地址**：实际内存芯片上的地址，由分页部件从线性地址转换而来。

**考试写法**：
> Pentium 保护模式下有三种地址：虚拟地址（逻辑地址）、线性地址和物理地址。虚拟地址由16位段选择符和32位偏移量组成，共48位，地址空间64TB。分段部件将虚拟地址转换为32位线性地址，分页部件再将线性地址转换为32位物理地址。

⚠️ 考点：简答 | ⭐⭐

### 2.2 段描述符（Segment Descriptor）

**核心定义**：用于描述一个段的基本信息，由 **8个字节（64位）** 组成，保存段的基址、界限、属性等信息，由编译程序/操作系统产生。

📝 **画图要点**：程序段描述符各字节分布如下（考试必须能画出此表格并说明各字段含义）

```
字节7  字节6      字节5              字节4  字节3  字节2  字节1  字节0
┌─────┬──────────┬──────────────────┬─────┬─────┬─────┬─────┬─────┐
│段基址│G|D/B|0|AV│P|DPL|S| TYPE字段│段基址│段基址│段基址│段界限│段界限│
│31~24│  L|段界限│                  │23~16│15~8 │ 7~0 │15~8 │ 7~0 │
└─────┴──────────┴──────────────────┴─────┴─────┴─────┴─────┴─────┘
```

**各字段含义（必背）**：

| 字段 | 位数 | 含义 |
|------|------|------|
| 段基址 | 32位 | 段在4GB物理空间中的起始地址（分布在字节2~4和字节7） |
| 段界限 | 20位 | 决定段的长度（分布在字节0~1和字节6低4位） |
| G 粒度位 | 1位 | G=0：界限单位为字节；G=1：界限单位为4KB |
| D/B 位 | 1位 | D/B=1选32位操作；D/B=0选16位操作 |
| P 存在位 | 1位 | P=1该段在内存中；P=0该段不在内存中 |
| DPL 特权级 | 2位 | 00(0级最高)~11(3级最低)，控制访问权限 |
| S 分类位 | 1位 | S=1为程序段描述符；S=0为系统段描述符 |
| AVL 可用位 | 1位 | AVL=1系统软件可用本段；AVL=0不可用 |
| TYPE 类型 | 4位 | 在不同描述符中含义不同（见2.3节） |

⚠️ 考点：简答 | ⭐⭐⭐  
❌ 易错：**段基址被拆散在4个不连续字节中**（字节2、3、4、7），不是连续存放的。段界限也被拆散（字节0、1和字节6的低4位）。

---

### 2.3 段描述符分类

**分类总览**：

```
段描述符
├── 程序段描述符（S=1）
│   ├── 代码段描述符（TYPE: E=1）
│   ├── 数据段描述符（TYPE: E=0, ED=0）
│   └── 堆栈段描述符（TYPE: E=0, ED=1）
│
└── 系统段描述符（S=0）
    ├── 任务状态段(TSS)描述符（TYPE=1001B/1011B）
    ├── LDT描述符（TYPE=0010B）
    └── 门描述符
        ├── 调用门（TYPE=1100B）—— 用于改变特权级别
        ├── 任务门（TYPE=0101B）—— 用于任务切换
        ├── 中断门（TYPE=1110B）—— 用于中断服务程序
        └── 陷阱门（TYPE=1111B）—— 用于异常处理
```

**TYPE字段详解**（程序段，S=1）：

数据段/堆栈段（E=0）：`E=0 | ED | W | A`
- ED=0向上扩展（数据段）；ED=1向下扩展（堆栈段）
- W=0不可写；W=1可写
- A=0未被访问；A=1已被访问

代码段（E=1）：`E=1 | C | R | A`
- C=0非一致性（不检查特权）；C=1一致性（检查特权）
- R=0不可读；R=1可读

⚠️ 考点：简答 | ⭐⭐⭐

### 2.4 全局描述符表 GDT 与 GDTR

**核心定义**：
- **GDT（Global Descriptor Table）**：保存系统使用、各任务共享的段描述符，整个系统**只有一个**。GDT中不存放中断门和陷阱门。
- **GDTR**：全局描述符表寄存器，48位 = 32位基地址 + 16位界限，指定GDT在内存中的位置。

**GDT 寻址过程（三步）**：

```
1. 段选择符的13位INDEX × 8 + GDTR中的基地址 = 段描述符的物理地址
2. 从该地址读出8字节段描述符，将段基址等信息装入段寄存器的不可见Cache部分
3. Cache中的段基址 + 虚地址中的32位偏移量 = 32位线性地址
```

📝 **画图要点**（练习题47题必考）：
```
虚地址（逻辑地址）
┌──────────────────┬────────────────────────────────┐
│  16位段选择符     │         32位偏移量              │
│ [13位INDEX|TI|RPL]│                                │
└─────────┬────────┴─────────────────────┬──────────┘
          │INDEX × 8                     │
          ▼                              │
  GDTR基地址                             │
  ┌──────┐                              │
  │32位  │──→ GDT起始地址               │
  │基地址│       ┌──────────────────┐   │
  └──────┘   (+)→│ 8字节段描述符     │   │
  ┌──────┐       │ 基址|界限|属性   │   │
  │16位  │       └────────┬─────────┘   │
  │界限  │                │32位段基址   │
  └──────┘                ▼             ▼
                         (+)────────────┘
                          │
                          ▼
                    32位线性地址
```

⚠️ 考点：简答/画图 | ⭐⭐⭐ 🔥 高频  
❌ 易错：GDT第0项（索引值0）为空项，不使用。段选择符索引要乘以8（因每个描述符8字节）。

### 2.5 局部描述符表 LDT 与 LDTR

**核心定义**：
- **LDT（Local Descriptor Table）**：保存某个任务专用的段描述符，每个任务**一个**LDT。LDT中无TSS、LDT、中断门、陷阱门。
- **LDTR**：局部描述符表寄存器，由16位段选择符（可见）+ 64位不可见部分（32位基址+20位界限+12位属性）组成。
- LDTR的段选择符指向GDT中的LDT描述符（即LDT本身的描述符存放在GDT中）。

**LDT 寻址过程**：
```
1. LDTR中的16位段选择符 → 在GDT中找到LDT的段描述符
2. 处理器自动将LDT描述符装入LDTR的不可见Cache（含LDT基址）
3. 虚地址段选择符的INDEX × 8 + LDT基址 = 段描述符地址
4. 取出段描述符中的段基址 + 32位偏移量 = 32位线性地址
```

⚠️ 考点：简答/画图 | ⭐⭐⭐  
❌ 易错：**LDT的描述符保存在GDT中**，而不是另一张独立的表。通过LDT访问段时，要经过**两次**查表：先查GDT找到LDT位置，再查LDT找到目标段描述符。

---

### 2.6 中断描述符表 IDT 与 IDTR

**核心定义**：
- **IDT（Interrupt Descriptor Table）**：保存**门描述符**，整个系统一个，包括中断门、陷阱门、任务门（通常无调用门）。
- **IDTR**：中断描述符表寄存器，48位 = 32位基地址 + 16位界限。
- Pentium最多支持256个中断/异常，IDT最多占用256 × 8 = 2KB。
- 每个门与一个**中断类型码**（0~255）对应。

⚠️ 考点：简答 | ⭐⭐

---

### 2.7 任务状态段 TSS 与 TR

**核心定义**：
- **TSS（Task State Segment）**：保存任务的机器状态（所有寄存器状态）及任务间关联信息，每个任务**一个**，无代码和数据。
- **TR（Task Register）**：任务寄存器，结构与LDTR相同（16位可见+64位不可见），指向GDT中当前任务的TSS描述符。

**TSS主要内容**：
- 寄存器保存区：EAX/EBX/ECX/EDX/ESP/EBP/ESI/EDI、ES/CS/SS/DS/FS/GS、EIP、EFLAGS
- CR3（页目录基地址）
- 返回链 BACK_LINK（前一任务的TSS选择符，供IRET返回时用）
- CPL0/1/2的堆栈（ESP0/SS0等）
- LDT选择符
- I/O许可位图

**TR的更新方式**：LTR/STR指令；保护模式下JMP/CALL远转移；任务门。

⚠️ 考点：简答 | ⭐⭐

### 2.8 段选择符组成

**段选择符（16位）= INDEX（13位）+ TI（1位）+ RPL（2位）**

```
 15    ···    3   2   1  0
┌──────────────┬────┬─────┐
│  13位 INDEX  │ TI │ RPL │
└──────────────┴────┴─────┘
```

| 字段 | 含义 |
|------|------|
| INDEX（13位） | 描述符在GDT/LDT中的索引号（乘以8得偏移量） |
| TI（1位） | TI=0选GDT；TI=1选LDT |
| RPL（2位） | 请求特权级，00最高(0级)，11最低(3级)，不参与寻址 |

⚠️ 考点：简答 | ⭐⭐⭐  
❌ 易错：RPL 存放在段选择符最低两位，**不参与描述符表寻址**，只用于特权检查。

---

### 2.9 三种特权级：CPL / DPL / RPL

| 特权级 | 全称 | 位置 | 含义 |
|--------|------|------|------|
| CPL | 当前特权级 | CS寄存器中段选择符的RPL字段 | 当前正在执行的代码段的特权级 |
| DPL | 描述符特权级 | 段描述符的DPL字段 | 该段被访问所要求的特权级 |
| RPL | 请求特权级 | 段选择符的最低两位 | 新装入段寄存器时的请求特权级 |

**特权规则**：特权级为P的段中数据，只能由**特权级高于或等于P**（即数字小于等于P）的代码访问。

⚠️ 考点：简答 | ⭐⭐

### 2.10 分页机制：线性地址→物理地址

**核心概念**：Pentium采用**二级页表**，每页大小4KB，页目录基址由CR3寄存器保存。

**32位线性地址的三段划分**：

```
 31          22 21        12 11          0
┌─────────────┬─────────────┬─────────────┐
│ 10位页目录索引│ 10位页表索引 │ 12位页内偏移 │
└─────────────┴─────────────┴─────────────┘
```

**分页转换过程（四步）**：

```
1. CR3 → 页目录基址（4KB对齐）
2. 线性地址[31:22]（10位）× 4 + 页目录基址 → 页目录项（4字节）
   页目录项[31:12]（20位）= 某页表的基址
3. 线性地址[21:12]（10位）× 4 + 页表基址 → 页表项（4字节）
   页表项[31:12]（20位）= 物理页基址
4. 物理页基址（20位）拼接 线性地址[11:0]（12位）= 32位物理地址
```

📝 **画图要点**（段页式完整图见2.11节）：
```
CR3寄存器（页目录基地址）
    │
    ▼
  页目录（4KB，1024项×4字节）
  [线性地址bit31~22作索引，×4得偏移]
    │
    ▼ 页目录项（含页表基址20位）
  页表（4KB，1024项×4字节）
  [线性地址bit21~12作索引，×4得偏移]
    │
    ▼ 页表项（含物理页基址20位）
  物理页
  [线性地址bit11~0作页内偏移]
    │
    ▼
  物理地址（32位）
```

⚠️ 考点：简答/画图 | ⭐⭐⭐ 🔥 高频  
❌ 易错：页目录和页表各有**1024项**，每项**4字节**，故各自占用**4KB**一个页面。物理页基址只有20位（高20位），低12位由线性地址中的页内偏移提供。

### 2.11 段页式存储管理完整寻址过程

📝 **画图要点**（练习题49/50题，考试极可能要求画此完整图）：

```
虚地址（48位）
┌────────────────┬──────────────────────────────────────┐
│  16位段选择符   │              32位偏移量               │
│[INDEX|TI|RPL]  │                                      │
└───────┬────────┴──────────────────────┬───────────────┘
        │                               │
        ▼ INDEX×8                       │
 GDTR/LDTR基址 → 段描述符表             │
        (+)→ [段描述符：基址|界限|属性] │
                  │32位段基址           │
                  └──────────(+)────────┘
                              │
                        32位线性地址
              ┌───────────────┼───────────────┐
             [31:22]        [21:12]          [11:0]
           10位页目录索引  10位页表索引    12位页内偏移
              │                │                │
    CR3→页目录│×4              │×4              │
          (+)─┘                │                │
        页目录项               │                │
        (页表基址)─────────────┘                │
             (+)→页表项                         │
                 (物理页基址)────────────────────┘
                             (+)
                              │
                        32位物理地址
```

**完整步骤描述（规范答法）**：
1. 从段选择符的TI位判断查GDT还是LDT，用INDEX×8加上GDTR/LDTR中的基地址，找到段描述符。
2. 取段描述符中的32位段基址，与虚地址中的32位偏移量相加，得到32位线性地址。
3. 线性地址高10位（bit31~22）作为页目录索引，乘以4加上CR3中的页目录基地址，找到页目录项，取出页表基地址。
4. 线性地址中间10位（bit21~12）作为页表索引，乘以4加上页表基地址，找到页表项，取出物理页基地址（20位）。
5. 物理页基地址（20位）拼接线性地址低12位（bit11~0）的页内偏移，组成32位物理地址。

## 3. 高频计算题型与模板

### 3.1 段描述符地址计算

**公式**：
$$\text{段描述符地址} = \text{GDTR基地址} + \text{INDEX} \times 8$$

（若TI=1，则将GDTR基地址换为LDTR中的LDT基地址）

**解题步骤**：
1. 读取段选择符，取高13位作INDEX，读TI位判断查GDT/LDT
2. 取GDTR或LDTR中保存的32位基地址
3. 计算 INDEX × 8 + 基地址 = 描述符所在的物理地址

---

### 3.2 虚地址→线性地址计算

**公式**：
$$\text{线性地址} = \text{段基址(32位)} + \text{偏移量(32位)}$$

**示例**（练习题第50题参考）：

题目：设段选择符指向GDT中某描述符，该描述符基地址为 `08000000H`，虚地址偏移量为 `00001234H`，求线性地址。

Step 1：段基址 = `08000000H`  
Step 2：线性地址 = `08000000H + 00001234H = 08001234H`

---

### 3.3 线性地址→物理地址计算（分页）

**解题步骤**：
1. 将32位线性地址写成二进制，划分为 10位|10位|12位
2. 高10位（页目录索引）× 4 + CR3页目录基址 → 找页目录项 → 取出页表基址
3. 中间10位（页表索引）× 4 + 页表基址 → 找页表项 → 取出物理页基址（20位）
4. 物理页基址（20位）左移12位，加上低12位页内偏移 = 物理地址

**示例**（课件图5.5.3，线性地址 `00C02098H`）：

Step 1：`00C02098H = 0000 0000 1100 | 0000 0000 0010 | 0000 1001 1000`  
- 页目录索引 = `0000000011B` = 3，偏移 = 3×4 = 0CH  
- 页表索引 = `0000000010B` = 2，偏移 = 2×4 = 08H  
- 页内偏移 = `000010011000B` = 098H  

Step 2：页目录基址（CR3）+ 00CH → 页目录项 = 页表3基址 `01010000H`  
Step 3：`01010000H + 08H = 01010008H` → 页表项 → 物理页基址 `00160000H`  
Step 4：物理地址 = `00160000H + 098H = 00160098H`

## 4. 练习题精析

### 题目40（练习题）

**题目**：说明计算机中采用 Cache 和虚拟存储器的原因。

**解答**：
- **采用Cache的原因**：CPU运算速度远快于主存访问速度，若CPU每次都直接访问主存，大量时间浪费在等待上。Cache是速度接近CPU、容量小的高速缓存，利用程序局部性原理将常用数据放入Cache，使CPU以接近Cache的速度运行，解决**速度矛盾**。
- **采用虚拟存储器的原因**：程序规模越来越大，常常超过物理主存容量；同时多道程序设计需要同时运行多个程序。虚拟存储器将辅存（磁盘）的大容量与主存的高速度结合，使程序员看到一个远大于物理内存的地址空间，解决**容量矛盾**。

**考察点**：虚拟存储器与Cache的目标差异。

### 题目43（练习题）

**题目**：IDTR、GDTR 和 LDTR 分别代表什么寄存器？其内容是什么信息？有什么作用？

**解答**：
- **GDTR（全局描述符表寄存器）**：48位 = 32位基地址 + 16位界限。保存全局描述符表GDT在内存中的起始地址和大小，CPU通过GDTR定位GDT，进而访问所有段描述符。
- **LDTR（局部描述符表寄存器）**：由16位可见的段选择符和64位不可见的Cache（32位基址+20位界限+12位属性）组成。保存当前任务的LDT位置，用于访问当前任务私有的段。每次任务切换时LDTR随之更新。
- **IDTR（中断描述符表寄存器）**：48位 = 32位基地址 + 16位界限。保存中断描述符表IDT的位置，CPU响应中断时通过IDTR找到IDT，进而找到对应的门描述符和中断服务程序入口。

**考察点**：三个系统级寄存器的用途，是简答题高频考点。

### 题目44（练习题）

**题目**：说明段描述符的组成及作用。

**解答**：
段描述符由8个字节（64位）组成，包含以下主要字段：

① **段基址（32位）**：指定该段在4GB物理地址空间中的起始地址。  
② **段界限（20位）**：决定段的最大长度，受G位控制（G=0时单位为字节，G=1时单位为4KB）。  
③ **P位（存在位）**：P=1表示该段在内存中，P=0表示不在内存中（已换出到辅存）。  
④ **DPL（描述符特权级，2位）**：定义访问该段所需的最低特权级，0级最高，3级最低。  
⑤ **S位（分类位）**：S=1为程序段（代码/数据/堆栈）描述符，S=0为系统段描述符。  
⑥ **TYPE字段（4位）**：在程序段中区分代码段、数据段、堆栈段，并描述其访问属性（可读/可写等）。  
⑦ **G位（粒度位）**：决定段界限的度量单位。  
⑧ **D/B位**：选择32位或16位操作模式。  

作用：段描述符是分段存储管理的核心数据结构，CPU通过它把虚地址中的段选择符转换为物理地址中的段基址，并进行段界限越限检查和特权级访问控制。

**考察点**：段描述符各字段含义。

### 题目45（练习题）

**题目**：说明 CPL、RPL、DPL 的含义。

**解答**：
- **CPL（Current Privilege Level，当前特权级）**：存放在CS段寄存器中段选择符的最低两位，表示当前正在执行的代码段所处的特权级，是程序当前运行时的身份标识。
- **DPL（Descriptor Privilege Level，描述符特权级）**：存放在段描述符的DPL字段，表示该段要求访问者具备的最高特权级（数字上最小的特权级数）。
- **RPL（Requested Privilege Level，请求特权级）**：存放在段选择符的最低两位，是程序员（或系统）在使用该选择符时附加的特权级别声明。

访问规则：要成功访问一个段，需满足 max(CPL, RPL) ≤ DPL（数字上不大于DPL）。

**考察点**：三种特权级的概念及区别，简答题常考。

### 题目46（练习题）

**题目**：任务状态段 TSS 的主要作用是什么？

**解答**：TSS（Task State Segment，任务状态段）的主要作用是**保存任务的机器状态**，包括：
① 任务切换时，将当前任务所有寄存器的内容（通用寄存器、段寄存器、EIP、EFLAGS、CR3等）保存在当前任务的TSS中。
② 将新任务TSS中保存的寄存器内容装入CPU，实现任务上下文切换。
③ 保存不同特权级（0/1/2级）的堆栈段选择符和堆栈指针（SS/ESP），供特权级切换时使用。
④ 通过BACK_LINK字段保存前一个任务的TSS选择符，供IRET指令返回前一任务时使用。
每个任务有且只有一个TSS，TSS中没有代码和数据。

**考察点**：TSS的功能，简答常考。

### 题目47（练习题）

**题目**：简述 Pentium 通过 GDT 访问数据段的寻址过程（也可画图说明）。

**解答**：

通过GDT访问数据段的过程分为以下步骤：

① 将16位段选择符装入DS寄存器（指令如 `MOV DS, AX`）。段选择符中TI=0，表示使用GDT。

② 取段选择符的高13位作为INDEX，INDEX × 8 得到该描述符相对于GDT起始地址的偏移量。

③ 将上述偏移量与GDTR中的32位GDT基址相加，得到段描述符在内存中的物理地址。

④ 从该地址读出8字节段描述符，进行段存在性（P位）、段界限、访问权限检查，通过后将段基址等信息装入DS段寄存器的不可见Cache部分。

⑤ 此后再访问该段时，CPU直接从DS的Cache取出32位段基址，加上指令中的32位偏移量，得到32位线性地址（若不启用分页则即为物理地址）。

**考察点**：GDT寻址五步骤，画图题必考。

### 题目48（练习题）

**题目**：简述 Pentium 通过 LDT 访问数据段的寻址过程（也可画图说明）。

**解答**：

通过LDT访问数据段的过程：

① 段选择符中TI=1，表示使用LDT。

② CPU从LDTR的可见部分取16位选择符，在GDT中查找LDT描述符（INDEX×8+GDTR基址），取出LDT基地址，装入LDTR的不可见Cache。

③ 用虚地址中段选择符的INDEX × 8 + LDTR Cache中的LDT基址，定位到LDT中的段描述符。

④ 读出段描述符，进行P位/界限/权限检查，将段基址装入DS的不可见Cache。

⑤ DS Cache中的32位段基址 + 32位偏移量 = 32位线性地址。

❌ 关键区别：比GDT多一步——先用LDTR在GDT中找LDT的位置，再在LDT中找目标段描述符。

**考察点**：LDT寻址与GDT的区别。

### 题目49（练习题）

**题目**：说明段页式存储管理的寻址过程。

**解答**：段页式管理结合了分段和分页两种机制，寻址分两个阶段：

**第一阶段（分段）**：虚地址（逻辑地址）= 16位段选择符 + 32位偏移量。通过段选择符查GDT或LDT，取出32位段基址，与32位偏移量相加得到**32位线性地址**。

**第二阶段（分页）**：32位线性地址分为三段——高10位作页目录索引、中间10位作页表索引、低12位为页内偏移。
- 高10位 × 4 + CR3指向的页目录基址 → 页目录项 → 取出页表基址
- 中间10位 × 4 + 页表基址 → 页表项 → 取出物理页基址（20位）
- 物理页基址（20位）+ 低12位页内偏移 → **32位物理地址**

**考察点**：完整地址转换链，综合题必考，结合画图最佳。

### 题目50（练习题）

**题目**：Pentium 处理器采用分段存储器管理时的数据访问逻辑地址转换为物理地址的过程。

**解答**：

Pentium分段存储管理将48位虚拟地址映射到32位物理地址，过程如下：

虚地址由**16位段选择符**和**32位偏移量**两部分组成。段选择符的高13位为INDEX（描述符索引），第2位TI决定查GDT（TI=0）还是LDT（TI=1），低2位RPL为请求特权级。

通过INDEX × 8 加上描述符表（GDT/LDT）的基地址，定位到对应的8字节**段描述符**。取出段描述符中的32位**段基地址**，与逻辑地址中的32位**偏移量**相加，得到32位**线性地址**。

若未启用分页（仅分段），则线性地址就是物理地址；若启用分页，则线性地址还需经过二级页表转换才得到最终物理地址。

**解答**：

**Pentium实地址模式特点**：与8086兼容，基本体系结构相同，但能使用Pentium特有的32位寄存器、部分新寻址方式和扩展指令。

### 【简答2】GDT与LDT的区别是什么？

**规范答法**：
> GDT（全局描述符表）整个系统只有一个，由GDTR寄存器指定其位置（32位基地址+16位界限），保存系统级和各任务共享的段描述符（包括TSS描述符、LDT描述符、调用门等，但不含中断门和陷阱门）。
>
> LDT（局部描述符表）每个任务一个，由LDTR寄存器指定其位置，保存该任务私有的代码段、数据段、堆栈段等描述符（不含TSS、LDT、中断门、陷阱门）。LDT描述符本身存放在GDT中。通过LDT访问段时，需先在GDT中查到LDT的位置，再在LDT中查目标描述符。

---

### 【简答3】门描述符的含义及分类

**规范答法**：
> 门描述符是一种特殊的系统段描述符（S=0），用来控制程序从一段代码转移到另一段代码或从一个任务切换到另一个任务，提供了带访问控制的程序入口。门描述符包含目标代码的段选择符和32位偏移量（入口地址），以及DPL特权级控制字段。
>
> 门描述符分四类：
> - **调用门**：用于在不同特权级代码段之间进行带参数传递的远调用（CALL指令使用）。
> - **任务门**：用于任务切换，可存放在GDT、LDT或IDT中。
> - **中断门**：用于硬件中断，存放于IDT，进入时自动关中断（清IF）。
> - **陷阱门**：用于软件异常，存放于IDT，进入时不清IF。

---

### 【简答4】段描述符中P位和G位的含义

**规范答法**：
> **P位（存在位）**：P=1表示该段当前在物理内存中，可以正常访问；P=0表示该段已被换出到辅存，此时CPU产生段不存在异常，由操作系统将该段从辅存调入内存后更新P位。
>
> **G位（粒度位）**：G=0时，20位段界限的度量单位为字节，段最大长度为$2^{20}$=1MB；G=1时，度量单位为4KB，段最大长度为$2^{20}×4\text{KB}=4\text{GB}$。

## 6. 易错点汇总

- ❌ 易错：段描述符是连续8字节，段基址和段界限各自集中存放。  
  ✅ 正确：段基址被拆分在字节2、3、4、7四个不连续位置；段界限拆分在字节0、1和字节6低4位。这是硬件设计历史遗留，必须记住。

- ❌ 易错：LDT 描述符保存在 LDT 自身中。  
  ✅ 正确：LDT 的描述符保存在 **GDT** 中。LDTR通过GDT定位LDT，然后LDT中保存该任务的段描述符。

- ❌ 易错：TI=0选LDT，TI=1选GDT。  
  ✅ 正确：**TI=0选GDT，TI=1选LDT**。

- ❌ 易错：段选择符INDEX直接就是描述符表中的字节偏移。  
  ✅ 正确：INDEX需要**乘以8**才是偏移量，因为每个描述符占8字节。

- ❌ 易错：分页时，线性地址直接分成高20位（页号）+低12位（页内偏移）一级查表。  
  ✅ 正确：Pentium采用**二级**页表，高10位查页目录，中间10位查页表，低12位为页内偏移，共三段。

- ❌ 易错：GDTR、LDTR、IDTR都是相同格式的寄存器。  
  ✅ 正确：GDTR和IDTR均为48位（32位基址+16位界限），直接保存表的位置。LDTR则是16位可见的**段选择符**加上64位不可见的Cache，需要通过GDT间接访问LDT的位置。

- ❌ 易错：进入中断门和陷阱门时，对IF标志的处理一样。  
  ✅ 正确：进入**中断门**时自动清除IF（关中断）；进入**陷阱门**时不改变IF（不关中断）。

- ❌ 易错：虚拟8086模式是一种处理器的工作模式。  
  ✅ 正确：虚拟8086模式是**程序的运行方式**（在保护模式框架下模拟8086环境运行的程序状态），实地址模式才是处理器的工作模式。

## 7. 一页纸速记

**核心公式**：
- 逻辑地址 = 16位段选择符 + 32位偏移量（共48位，$2^{46}=64TB$）
- 线性地址 = 段基址（32位）+ 偏移量（32位）
- 描述符地址 = GDTR/LDTR基址 + INDEX × 8
- 分页：32位线性地址 = [高10位 页目录索引 | 中10位 页表索引 | 低12位 页内偏移]

**必背结论**：
- 段描述符 = **8字节**，段基址32位（分散存放），段界限20位（分散存放）
- GDT全局唯一，GDTR=32位基址+16位界限；LDT每任务一个，LDT描述符在GDT中
- 分页：CR3→页目录（1024项×4B=4KB）→页表（1024项×4B=4KB）→物理页
- TSS每任务一个，保存任务上下文（寄存器状态），无代码无数据
- 段选择符：TI=0→GDT，TI=1→LDT；RPL为请求特权级（不参与寻址）
- 特权级0最高（操作系统）、3最低（用户程序）；数字越小特权越高

**最易混淆的区别**：

| 对比项 | GDT | LDT | IDT |
|--------|-----|-----|-----|
| 数量 | 全系统唯一 | 每任务一个 | 全系统唯一 |
| 寄存器 | GDTR（48位） | LDTR（16位可见+64位Cache） | IDTR（48位） |
| 保存内容 | 系统/共享段描述符 | 任务私有段描述符 | 门描述符（中断/陷阱/任务门） |
| 不含内容 | 中断门、陷阱门 | TSS、LDT、中断门、陷阱门 | 调用门（通常） |

| 对比项 | 中断门 | 陷阱门 |
|--------|--------|--------|
| 存放位置 | IDT | IDT |
| 用途 | 硬件中断 | 软件异常 |
| 进入时IF | **自动清0（关中断）** | 不改变IF |
