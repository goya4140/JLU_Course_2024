### 模块A：汇编语言基本概念与语句格式

这是理解本章一切内容的前提：搞清楚"谁在汇编时起作用、谁在运行时起作用"。

#### A1 核心概念辨析 ✅真题·第9题

> **练习题原文：**  
> 伪指令的作用是什么？

**知识铺垫：** 汇编语言有三类语句：
- **机器指令**（CPU指令）：汇编后生成机器码，CPU运行时执行。
- **伪指令**（pseudo-instruction）：**不产生机器指令码**，只在汇编阶段由汇编程序处理，用来指示汇编程序如何工作（如定义数据、分配存储区、指定段寄存器对应关系等）。
- **宏指令**：汇编时展开为一段指令序列，会产生目标代码。

✅ **参考答案要点：** 伪指令不在运行期间由CPU执行，而是在汇编时由汇编程序处理。主要完成：选择处理器、定义程序模式、定义数据、分配存储区、指示程序结束等功能。

---

#### A2 标号的作用

> **练习题原文：**  
> 标号的作用是什么？

✅ **参考答案要点：** 代码段中名字字段叫做标号，表示一条指令语句的符号地址，可用该符号地址访问（即跳转到）该指令。常作为 JMP/LOOP/CALL 等指令的操作数。

---

#### A3 END / ENDS / ENDP / ENDM 四者区别

> **练习题原文：**  
> 说明伪指令 END、ENDS、ENDP、ENDM 的区别。

| 伪指令 | 含义 |
|---|---|
| END [标号] | 整个源程序汇编结束（必须有，是源文件最后一条语句） |
| ENDS | 段定义结束（与SEGMENT配对使用） |
| ENDP | 过程定义结束（与PROC配对使用） |
| ENDM | 宏定义结束（与MACRO配对使用） |

✅ **参考答案要点：** 四者均以END开头但层级完全不同：END是全局结束，ENDS是段级结束，ENDP是过程级结束，ENDM是宏定义级结束。

### 模块B：寻址方式——有效地址的计算

寻址方式是汇编指令的核心语法，题库中的指令错误辨析、真题的段默认判断，都依赖于对 EA 公式的掌握。

#### B1 有效地址（EA）公式 ⚠️考点

**有效地址公式（8086，16位寻址）：**

$$EA = \begin{bmatrix}BX\\BP\end{bmatrix} + \begin{bmatrix}SI\\DI\end{bmatrix} + \begin{bmatrix}8位位移量\\16位位移量\end{bmatrix}$$

每个部分最多保留一项，至少有一项有值。

**段约定（极重要陷阱）：**
- 使用 **BP**（或SP）作为基址寄存器 → 默认段为 **SS**（堆栈段）
- 使用 **BX、SI、DI** → 默认段为 **DS**
- 取指令 → 默认段为 **CS**
- 字符串目的地址 → 默认段为 **ES**

---

#### B2 寻址方式辨析 ✅真题·第8题

> **真题（以前的考题第8题）原文：**  
> 在实模式下，`ADD AX，[BP + DI]` 的源操作数物理地址为（ ）。  
> A. 16×(SS)+(BP)+(DI)　B. 16×(DS)+(BP)+(DI)  
> C. 16×(ES)+(BP)+(DI)　D. 16×(CS)+(BP)+(DI)

**知识铺垫：** `[BP + DI]` 是**基址变址寻址**，基址寄存器为 BP，根据段约定，BP默认使用 SS 段。

物理地址 = $16 \times (SS) + (BP) + (DI)$

✅ **参考答案要点：** 选 **A**。关键在于 BP 默认段为 SS，不是 DS。

---

> **练习题原文：**  
> 说明下列指令中源操作数的寻址方式：  
> (1) `MOV CX，24[BX][SI]`  
> (2) `MOV BX，[2000H]`  
> (3) `INC BYTE PTR[BX]`  
> (4) `MOV AX，[BP+DI]`

| 指令 | EA成分 | 寻址方式名称 |
|---|---|---|
| `24[BX][SI]` | 基址BX + 变址SI + 位移量24 | 相对基址变址寻址 |
| `[2000H]` | 只有位移量 | 直接寻址 |
| `[BX]` | 只有基址 | 寄存器间接寻址 |
| `[BP+DI]` | 基址BP + 变址DI | 基址变址寻址 |

✅ **参考答案要点：** 辨别关键——看方括号里有几个成分（基址/变址/位移量），三者都有→相对基址变址；只有位移量常数→直接寻址；只有基址→寄存器间接寻址；基址+变址（无位移量）→基址变址。

### 模块C：伪指令详解

伪指令是第4章选择题的高频区，TYPE/OFFSET/LENGTH/SIZE 的返回值规则是陷阱集中点。

#### C1 数据定义伪指令（DB / DW / DD / DUP / ?）

| 伪指令 | 每个操作数字节数 | 变量类型 |
|---|---|---|
| DB | 1字节（8位）| 字节变量 |
| DW | 2字节（16位）| 字变量 |
| DD | 4字节（32位）| 双字变量 |

`?` 表示只分配存储空间，不赋初值。  
`N DUP(x)` 表示将操作数 x 重复 N 次。

---

#### C2 返回值运算符 TYPE / LENGTH / SIZE / OFFSET ✅真题

> **练习题原文：**  
> 对于下面的数据定义，各条MOV指令单独执行后，AX寄存器的内容是什么？  
> ```
> ORG 2000H
> A1    DB    ？
> A2    DW    20 DUP(？)
> A3    DB    'ABCD'
> A4    DW    10 DUP(？)
> (1) MOV AX，TYPE A1
> (2) MOV AX，OFFSET A2
> (3) MOV AX，LENGTH A3
> (4) MOV AX，SIZE A4
> ```

**规则速记表：**

| 运算符 | 作用 | 对变量的返回值 |
|---|---|---|
| TYPE | 返回单个元素字节数 | DB→1, DW→2, DD→4 |
| OFFSET | 返回偏移地址（相对于段首）| 数值 |
| LENGTH | 返回 DUP 括号内**第一个**DUP的重复次数；无DUP则返回1 | 无DUP→1 |
| SIZE | = TYPE × LENGTH | DUP情况下有意义 |

**逐题分析：**

A1在 `ORG 2000H` 处，占1字节（`DB`），所以：
- A2 从 2001H 开始（A1占1字节）
- A3 从 2001H + 20×2 = 2029H 开始
- A4 从 2029H + 4 = 202DH 开始

(1) `TYPE A1`：A1是DB，返回 **1**

(2) `OFFSET A2`：A2在ORG 2000H之后1字节 = **2001H**

(3) `LENGTH A3`：A3定义为 `DB 'ABCD'`，没有DUP，LENGTH返回 **1**（注意：不是4！LENGTH只在有DUP时返回重复次数）

(4) `SIZE A4`：A4为 `DW 10 DUP(?)`，SIZE = TYPE × LENGTH = 2 × 10 = **20**

✅ **参考答案要点：**
- (1) AX = 1
- (2) AX = 2001H
- (3) AX = 1（易错！LENGTH对字符串DB返回1而非字符个数）
- (4) AX = 20

---

#### C3 PTR 临时类型属性运算符

格式：`类型 PTR 变量/标号`，临时改变操作数的类型属性。

常见用法：
```asm
MOV BYTE PTR [BX], 8   ;存入字节单元（否则不知道是字节还是字）
MOV WORD PTR [BX], 8   ;存入字单元
```

当指针寻址时，如果操作数大小不能从上下文推断，**必须**用PTR显式指定，否则汇编报错。

### 模块D：指令错误辨析 🔥高频

指令错误辨析是每届试卷的常见选择题/简答题素材，掌握"错误类型清单"即可应对。

#### D1 常见指令错误类型速查

> **练习题原文：**  
> 指出下列指令的错误：  
> (1) `MOV AH，BX`  
> (2) `ADD 15H，BX`  
> (3) `MOV CS，AX`  
> (4) `MOV AX，[SI][DI]`  
> (5) `MOV BYTE PTR [BX]，1000`

| 指令 | 错误原因 |
|---|---|
| `MOV AH，BX` | 操作数长度不匹配：AH为8位，BX为16位 |
| `ADD 15H，BX` | 立即数不能作目的操作数 |
| `MOV CS，AX` | CS寄存器不能作为MOV目的操作数（CS可出不可入）|
| `MOV AX，[SI][DI]` | 变址寻址中SI和DI不能同时作变址寄存器（必须搭配一个基址寄存器BX/BP）|
| `MOV BYTE PTR [BX]，1000` | 立即数1000超出字节范围（0~255），字节存不下 |

✅ **参考答案要点：** 五类典型错误：①操作数位宽不匹配；②立即数作目的；③CS不可写入；④两个变址不能同时用（SI和DI）；⑤立即数超出操作数范围。

---

> **练习题原文：**  
> 假设VAR12和VAR34为字变量，LAB为标号，试指出下列指令的错误之处：  
> (1) `ADD VAR12，VAR34`  
> (2) `SUB AL，VAR12`  
> (3) `JMP LAB [DI]`  
> (4) `JNZ VAR12`

| 指令 | 错误原因 |
|---|---|
| `ADD VAR12，VAR34` | 两个操作数均为内存操作数，ADD不允许 mem←mem（x86必须有一个寄存器）|
| `SUB AL，VAR12` | 类型不匹配：AL是字节（8位），VAR12是字变量（16位）|
| `JMP LAB [DI]` | JMP后跟标号时不应有 [DI]，这不是合法的JMP语法 |
| `JNZ VAR12` | 条件跳转指令操作数必须是标号，VAR12是变量名不是标号 |

✅ **参考答案要点：** ADD不支持双内存操作数；操作数位宽必须匹配；JMP/JNZ的操作数必须是标号（代码地址），不能是数据变量名。

### 模块E：指令执行结果与标志位 🔥高频

标志位题型固定：给几条指令序列，求某寄存器值和CF/ZF/SF/OF/PF的状态。

#### E1 标志位速查 ⚠️考点

| 标志 | 名称 | 置1条件 |
|---|---|---|
| CF | 进位标志 | 无符号加法最高位产生进位；减法产生借位 |
| ZF | 零标志 | 结果为全0 |
| SF | 符号标志 | 结果最高位为1（负数）|
| OF | 溢出标志 | 有符号运算结果超出范围（同号相加结果变号）|
| PF | 奇偶标志 | 结果低8位中1的个数为偶数时置1 |

**INC/DEC的特殊性：** 这两条指令影响 AF/OF/PF/SF/ZF，但**不影响 CF**。

---

#### E2 ADD标志位计算题 ✅真题

> **练习题原文：**  
> 执行下面的指令序列后，写出标志寄存器CF、PF、ZF、SF、OF位的状态：  
> ```asm
> MOV AX，35E5H
> MOV BX, 7832H
> ADD AX, BX
> ```

**计算过程：**
```
  35E5H  =  0011 0101 1110 0101
+ 7832H  =  0111 1000 0011 0010
= AE17H  =  1010 1110 0001 0111
```

逐步分析：
- $35E5H + 7832H = AE17H$（十六进制直接相加：5+2=7, E+3=11→1进1, 5+8+1=14→4进1, 3+7+1=B→不进）

实际：$35E5 + 7832 = AE17$（结果 = 0xAE17）

- **CF = 0**：最高位无进位（0011+0111 = 1010，未溢出到16位外）
- **ZF = 0**：结果 AE17H ≠ 0
- **SF = 1**：结果最高位为1（AE17H最高位是1）
- **OF = 1**：两个正数相加（SF位均为0），结果变为负数（SF=1），发生有符号溢出
- **PF = 1**：低8位17H = 0001 0111，其中1的个数为3（奇数） → PF = 0

❌易错：题库给出的答案是 PF=1，但按标准计算 17H = 00010111，有三个1（奇数），PF应为0。建议考试时自己按定义算，不要背题库数字。

✅ **参考答案要点（按课件标准）：** CF=0, PF=1, ZF=0, SF=1, OF=1（题库答案）。实际做题时写出计算过程即可，老师看重过程。

#### E3 NOT + ADD + INC 的组合运算

> **练习题原文：**  
> 阅读下列程序段，将执行结果填入空格：  
> ```asm
> MOV AX，BX
> NOT AX
> ADD AX，BX
> INC AX
> AX=____，CF=____
> ```

**知识铺垫（对任意值BX都成立的恒等式）：**

设 BX = X（任意16位值）

```
AX ← X          ; AX = X
AX ← NOT AX     ; AX = ~X（X的按位取反 = -X-1 的补码）
AX ← AX + BX   ; AX = ~X + X
```

数学上：$\sim X + X$ 对16位整数而言等于 $0 \times FFFFH$，即全1，等于 $-1$（有符号）= $FFFFH$（无符号）

```
AX ← AX + 1   ; AX = FFFFH + 1 = 0 （进位，CF=1）
```

因此：$~X + X + 1 = 0$，CF = 1（有进位）

✅ **参考答案要点：** AX = 0，CF = 0

❌**勘误：** 题库答案写的是 CF=0，但按数学推导：$FFFFH + 0001H = 10000H$，最高位进位溢出，CF 应为 **1**。建议考试时写出推导过程。

（注：题库答案CF=0疑为笔误，标准答案应为AX=0, CF=1）

### 模块F：堆栈操作——PUSH指令工作原理

#### F1 PUSH DI 的执行步骤

> **练习题原文：**  
> 解释PUSH DI指令是怎样工作的。

**知识铺垫：** 8086堆栈向**小地址方向**增长，PUSH时先改指针再存数据。

PUSH DI（16位操作）步骤：
1. SP ← SP - 1，调整SP指针（高字节位置）
2. 将 DI 内容的**高8位**压入 SP 所指单元
3. SP ← SP - 1，再次调整SP指针（低字节位置）
4. 将 DI 内容的**低8位**压入 SP 所指单元

```
栈（向低地址增长）：
高地址 ──────────────
          |  DI高8位 |  ← SP-1处
          |  DI低8位 |  ← SP（操作后SP指此处）
低地址 ──────────────
```

✅ **参考答案要点：** ①SP-1，存DI高8位；②SP-1，存DI低8位。POP操作顺序相反（先取低字节，SP+1；再取高字节，SP+1）。

### 模块G：子程序 vs 宏指令

#### G1 两者本质区别

> **练习题原文：**  
> 简述子程序与宏指令的区别。

| 维度 | 子程序（CALL/RET）| 宏指令（MACRO/ENDM）|
|---|---|---|
| 执行时机 | 运行时跳转调用 | 汇编时展开（替换）|
| 代码体积 | 节省（代码只写一份）| 增大（每次调用都展开一份）|
| 执行时间 | 有CALL/RET开销 | 无调用开销，执行更快 |
| 参数传递 | 通过寄存器/堆栈 | 通过宏参数直接替换 |
| 调试 | 需要进入子程序 | 展开在主体中，调试直观 |

✅ **参考答案要点：** 子程序在运行期跳转执行，节省代码空间但有调用开销；宏指令在汇编时展开，节省运行时间但增大代码体积。

### 模块H：RET / IRET / HLT / END 辨析 ✅真题·简答题第3题

> **真题（以前的考题简答题第3题）原文：**  
> 说明在8086中，RET 和 IRET 指令有什么区别？

> **练习题原文：**  
> 说明 RET，END，HLT 的用途和区别。

**RET vs IRET：**

| 指令 | 用途 | 出栈内容 |
|---|---|---|
| RET | 从普通子程序返回 | 弹出 IP（段内）或 IP+CS（段间）|
| IRET | 从中断服务程序返回 | 弹出 IP + CS + **FLAGS**（多弹出标志寄存器）|

关键：IRET 比 RET 多从栈顶弹出一个 **FLAGS 标志寄存器**，这是因为中断响应时会把 FLAGS 压栈保护现场。

**RET vs END vs HLT：**

| 指令 | 类型 | 作用 |
|---|---|---|
| RET | 机器指令 | 子程序返回，弹出IP（和CS）继续主程序 |
| END | 伪指令 | 汇编结束标志，不产生机器码 |
| HLT | 机器指令 | CPU暂停执行，等待中断/复位/DMA信号 |

✅ **参考答案要点：** IRET 比 RET 多弹出 FLAGS（标志寄存器），专用于中断返回；RET 用于普通子程序返回；END 是伪指令表示源程序汇编结束；HLT 使CPU进入暂停状态。

### 模块I：EXE 完整程序结构——编程题模板 ✅真题·编程题1

真题编程题第1题要求"以EXE源程序结构编写"，这是固定格式必须背下来。

#### I1 EXE源程序标准框架

> **真题原文（节选）：**  
> 编写从键盘输入十进制数（0~99范围），输入的十进制数值乘以2后，保存到变量X中，并以十进制方式输出显示X值的程序。要求以EXE源程序结构编写程序，并带有输入和输出的提示字符串。

**EXE结构三段模板：**

```asm
DSEG    SEGMENT 'DATA'
    ; ──── 数据段：定义所有变量和字符串 ────
    PROMPT_IN  DB  'Please input two decimal number: $'
    PROMPT_OUT DB  'The result is: $'
    X          DW  0
DSEG    ENDS

SSEG    SEGMENT STACK 'STACK'
    DW 100H DUP(?)   ; 堆栈段，预留空间
SSEG    ENDS

CSEG    SEGMENT 'CODE'
    ASSUME CS:CSEG, DS:DSEG, SS:SSEG
START   PROC    FAR
    ; ──── EXE程序必须的返回地址设置 ────
    PUSH    DS
    MOV     AX, 0
    PUSH    AX
    ; ──── 初始化DS段寄存器 ────
    MOV     AX, DSEG
    MOV     DS, AX
    
    ; ──── 显示输入提示 ────
    LEA     DX, PROMPT_IN
    MOV     AH, 9
    INT     21H
    
    ; ──── 输入十进制两位数（十位 + 个位）────
    MOV     AH, 1
    INT     21H          ; 输入十位 ASCII→AL
    SUB     AL, '0'      ; ASCII转数值
    MOV     BL, AL
    MOV     AL, 10
    MUL     BL           ; AX = 十位 × 10
    MOV     BH, AL       ; 暂存
    
    MOV     AH, 1
    INT     21H          ; 输入个位 ASCII→AL
    SUB     AL, '0'      ; ASCII转数值
    ADD     AL, BH       ; AL = 十位×10 + 个位
    
    ; ──── 乘以2 ────
    SHL     AL, 1        ; AL × 2（或 MUL 2）
    MOV     X, AX        ; 保存到变量X
    
    ; ──── 显示输出提示 ────
    LEA     DX, PROMPT_OUT
    MOV     AH, 9
    INT     21H
    
    ; ──── 输出十进制数（拆分百位/十位/个位）────
    MOV     AX, X
    MOV     BL, 100
    DIV     BL           ; AL=百位商，AH=余数
    ADD     AL, '0'
    MOV     DL, AL
    MOV     AH, 2
    INT     21H          ; 输出百位
    
    MOV     AL, AH       ; 余数继续
    CBW
    MOV     BL, 10
    DIV     BL           ; AL=十位，AH=个位
    ADD     AL, '0'
    MOV     DL, AL
    MOV     AH, 2
    INT     21H          ; 输出十位
    
    ADD     AH, '0'
    MOV     DL, AH
    MOV     AH, 2
    INT     21H          ; 输出个位
    
    RET
START   ENDP
CSEG    ENDS
    END START
```

**关键结构要点：**
1. `PUSH DS / MOV AX,0 / PUSH AX`：EXE程序开头必须有，用于程序结束后返回DOS的返回地址（FAR过程返回时跳回DOS 0000:0000H入口）
2. `MOV AX, DSEG / MOV DS, AX`：汇编器不自动设置DS，必须手动初始化
3. `MOV AH, 9 / INT 21H`：显示字符串，字符串必须以`$`结尾
4. `MOV AH, 1 / INT 21H`：等待键盘输入一个字符，ASCII码存入AL
5. `MOV AH, 4CH / INT 21H`：终止程序返回DOS（也可用RET替代，因为开头已经PUSH了返回地址）

**常用 INT 21H 功能号备忘：**

| AH值 | 功能 | 入口 | 出口 |
|---|---|---|---|
| 01H | 键盘输入单字符（带回显）| — | AL=ASCII |
| 02H | 显示单字符 | DL=ASCII | — |
| 09H | 显示字符串（到$结束）| DS:DX=字符串首址 | — |
| 4CH | 终止程序返回DOS | — | — |

**ASCII↔数值转换规则（⭐编程题必用）：**
- ASCII→数字：`SUB AL, '0'`（或 `SUB AL, 30H`）
- 数字→ASCII：`ADD AL, '0'`（或 `ADD AL, 30H`）

✅ **参考答案要点：** EXE程序=数据段+堆栈段+代码段（FAR过程），必须手动初始化DS，开头PUSH DS/0保存返回地址，结尾用RET或INT 21H AH=4CH返回DOS；十进制输入输出的核心是ASCII与数值之间的加减30H转换。

### 模块J：编程综合题——数组处理与排序 🔥高频

本模块覆盖题库中最多的编程题类型，核心套路是"循环+CMP+条件跳转"。

#### J1 冒泡排序 ✅真题·编程题2（分析）

> **真题（编程与分析题第2题）原文：**  
> 分析下边对DAC0832操作的程序……（分析题）  
> 
> **练习题相关：**  
> 编写从键盘输入以回车结束的字符串，将其按ASCII码由大到小的顺序输出到显示器的源程序。

**冒泡排序核心逻辑（由大到小，无符号数）：**

```asm
; 假设 NUMBER1 DB N个数据，N为元素个数-1
; 外循环DX控制遍数，内循环CX控制比较次数

    MOV DX, N          ; 外循环：N-1遍
A1: LEA BX, NUMBER1   ; 内循环从头开始
    MOV CX, DX         ; 每遍比较次数 = 当前遍号（优化版，次数递减）
A2: MOV AL, [BX]
    CMP AL, [BX+1]     ; 比较相邻两个
    JNC A3             ; 前≥后，不需交换（无符号：JNC即CF=0，高于等于）
    XCHG AL, [BX+1]   ; 前<后，交换
    MOV [BX], AL
A3: INC BX
    LOOP A2
    DEC DX
    JNZ A1
```

**判断方向选择表：**

| 排序方向 | 无符号数用 | 有符号数用 |
|---|---|---|
| 由大到小（前≥后不换）| JNC（即JAE/JNB）| JGE |
| 由小到大（前≤后不换）| JBE（即JNA）| JLE |

✅ **参考答案要点：** 冒泡排序=外循环控制遍数（N-1遍）+内循环控制比较次数+CMP+条件跳转+XCHG交换。无符号比较用JNC（前大跳过），有符号用JGE。

#### J2 提取正数并排序（完整子程序结构）

> **练习题原文：**  
> 设从BUFFER开始存放若干个以`$`为结束标志的带符号字节数据，试编写将其中的正数按由大到小的顺序存入PLUS开始的缓冲区中的完整源程序。

**核心算法分解（三子程序）：**

```
SUB1 - 提取正数到PLUS：
    遍历BUFFER直到遇'$'（24H）
    用 TEST AL, 80H 判断符号位（80H=1000 0000B）
    JNZ 说明最高位为1即负数，跳过
    否则正数，存入PLUS，计数器DX++

SUB2 - 对PLUS中DX个正数排序（冒泡由大到小）：
    套用上面的冒泡模板

SUB3 - 输出PLUS数组：
    逐字节输出16进制（高4位 + 低4位各转成ASCII）
```

**判断正负数的关键指令：**
```asm
TEST AL, 80H   ; 测试最高位（符号位）
JNZ  is_neg    ; 符号位=1，是负数
               ; 否则是正数（包含0）
```

注意：**0也会被当作正数处理**（0的符号位为0），若题目要求"正数"不含0，还需额外 `CMP AL, 0 / JZ skip`。

✅ **参考答案要点：** 提取正数用 `TEST AL, 80H / JNZ`（跳过负数）；子程序用PROC/ENDP定义，开头PUSH保护现场，结尾POP恢复，最后RET返回；主程序依次CALL三个子程序。

#### J3 最大值/最小值/平均值程序 ⚠️考点

> **练习题原文：**  
> 已知在ARRAY数组中有80个无符号字节数据，将ARRAY数组中的最大值放入MAX单元中。  
> 有100个无符号整数，求最小值及其位置。  
> 有100个无符号整数，求平均值。

**最大值核心套路（以80个字节为例）：**

```asm
    LEA BX, ARRAY
    MOV CX, 80       ; 循环80次
    MOV AL, [BX]     ; 先假设第一个是最大值
    INC BX
    DEC CX
NEXT:
    CMP AL, [BX]
    JAE  NO_SWAP     ; 无符号：当前≥下一个，不更新
    MOV AL, [BX]     ; 发现更大值，更新最大值
NO_SWAP:
    INC BX
    LOOP NEXT
    MOV MAX, AL
```

**最小值只需把 JAE 改为 JBE**（当前≤下一个时不更新）。

**位置记录：** 另用一个寄存器（如DL）记录当前下标，发现更大/更小值时同步更新。

**平均值：** 循环累加到AX/DX:AX，循环结束后除以总个数（DIV CL 或 DIV n）。

✅ **参考答案要点：** 初始化第一个元素为假设最大/最小值 → CX控制循环 → CMP + 条件跳转（无符号JAE/JBE，有符号JGE/JLE）→ 更新时同时记录位置下标。

### 模块K：DB/DW/DD 定义语句题

> **练习题原文：**  
> 设要在地址为DAT1的数据区中顺次存放以下数据：'A'，'B'，0，0，'C'，'D'，0，0  
> 写出分别用命令DB、DW和DD实现的语句。

**知识铺垫：** x86小端（低端）存储，DW和DD定义时，字节按低地址存低字节顺序。

- 'A'=41H, 'B'=42H, 'C'=43H, 'D'=44H

用 **DB**（每字节独立）：
```asm
DAT1 DB 'A', 'B', 0, 0, 'C', 'D', 0, 0
```

用 **DW**（每2字节一个字，注意小端：'B'在高字节，'A'在低字节）：
```asm
DAT1 DW 4241H, 0000H, 4443H, 0000H
```
（'A'=41H是低字节，'B'=42H是高字节，合为字4241H；以此类推）

用 **DD**（每4字节一个双字）：
```asm
DAT1 DD 00004241H, 00004443H
```

✅ **参考答案要点：** DB最直观；DW/DD要注意小端存储——低地址存低字节，所以'A'在低字节写在右边（十六进制低位）。

### 压缩BCD码 ✅真题·选择题第6题

> **真题原文（以前的考题第6题）：**  
> 将十进制99以压缩BCD码格式传送到AL中，正确的指令是（ ）。  
> A. MOV AX, 0909H　B. MOV AX, 0099　C. MOV AX, 0909　D. MOV AX, 0099H

**课件补充·无对应原题（练习题库中无此题型，课件有说明）：**

压缩BCD码：高4位存十位BCD，低4位存个位BCD。  
十进制99 → BCD码 = `1001 1001B` = `99H`

因此传送指令为 `MOV AL, 99H`，即 `0099H`（高字节为0）。

汇编语言规定：**立即数如果以字母开头，必须加前导0**（如 `0ABH`），且十六进制数必须以H结尾。

选项分析：
- `MOV AX, 0909H`：0909H是将09传到AH、09传到AL，不是压缩BCD的99
- `MOV AX, 0099`：无H后缀，汇编认为是十进制99 = 0063H，结果是 AX=0063H，不是BCD
- `MOV AX, 0099H`：正确！AL=99H=压缩BCD的99

✅ **参考答案：** 选 **D**（MOV AX, 0099H，AL = 99H即压缩BCD的99）。
