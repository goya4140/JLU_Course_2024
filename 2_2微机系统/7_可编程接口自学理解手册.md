# 第7章 可编程接口芯片——自学理解手册

> **这份资料的目标**：不是让你背住控制字的每一位，而是让你理解"为什么CPU和外设之间需要这些芯片、它们各自解决了什么问题、三块芯片的设计思路有什么共同逻辑"。读完之后，你应该能在脑子里建立一张地图：从"CPU直接面对外设"的混乱，到三块芯片各司其职的有序协作。

## 知识体系总览——分层知识树

```
§7 可编程接口芯片
│
├─ 第一层：设计动机（为什么CPU不能直连外设，需要"中间人"）
│   ├─ 矛盾一：速度不匹配——CPU GHz级，外设慢几个数量级
│   ├─ 矛盾二：格式不匹配——CPU并行8位，某些外设串行1位
│   ├─ 矛盾三：协议不匹配——外设各有自己的握手时序
│   └─ 解法：可编程接口芯片——通过软件写控制字配置成所需角色
│
├─ 第二层：并行I/O接口——8255A（管"哪个方向传多少位"）
│   ├─ 内部结构：端口A / 端口B / 端口C（高4位+低4位）+ 控制寄存器
│   ├─ 端口选择：A1A0 = 00/01/10/11 → A口/B口/C口/控制口
│   ├─ 两种控制字：D7=1方式选择控制字 / D7=0 C口BSR置位复位控制字
│   ├─ 方式0：基本I/O，直接读写，无握手
│   ├─ 方式1：选通I/O，带握手信号（STB/IBF 输入侧；OBF/ACK 输出侧）
│   │   └─ INTR：STB回高 + IBF=1 + INTE=1 → 向CPU发中断请求
│   ├─ 方式2：双向选通（仅A口），同时持有OBF/ACK/STB/IBF/INTR共5根
│   └─ ⚠️易错：C口在方式1/2下被"征用"为握手线，不再是普通数据口
│
├─ 第三层：计数/定时接口——8253（管"什么时候触发"）
│   ├─ 基本结构：3个独立16位减法计数器（各有CLK/GATE/OUT）
│   ├─ 端口选择：A1A0 = 00/01/10/11 → C0/C1/C2/控制口
│   ├─ 控制字格式：SC1SC0（选通道）+ RL1RL0（读写格式）+ M2M1M0（工作方式）+ BCD
│   ├─ 计数初值公式：N = 目标OUT时间 × CLK频率
│   ├─ 六种工作方式
│   │   ├─ 方式0：软启动，一次有效，计到0 OUT变高（中断）
│   │   ├─ 方式1：硬启动（GATE↑），多次有效，低电平宽度=N个CLK（单稳）
│   │   ├─ 方式2：软/硬启动，重复有效，每轮末尾出1个CLK窄脉冲（脉冲列）
│   │   ├─ 方式3：软/硬启动，重复有效，高低各N/2个CLK（方波）
│   │   ├─ 方式4：软启动，一次有效，计到0出1个CLK窄脉冲（软件触发选通）
│   │   └─ 方式5：硬启动（GATE↑），多次有效，计到0出1个CLK窄脉冲（硬件触发选通）
│   ├─ GATE作用：软启动方式=电平控制暂停/继续；硬启动方式=上升沿触发
│   └─ ⚠️易错：写入计数初值0 = 最大值（二进制65536，BCD 10000）
│
├─ 第四层：串行通信接口——8251A（管"怎么走远路传"）
│   ├─ 并串/串并转换：CPU侧并行8位 ↔ 线路侧逐位串行
│   ├─ 异步帧格式：起始位(低) + 数据位(5~8) + 校验位(可选) + 停止位(高)
│   ├─ 传输速率：每秒字节数 = 波特率 ÷ 每帧总位数
│   ├─ 波特率因子：收发时钟 = 波特率 × 因子（1/16/64）
│   ├─ 两种控制字（顺序严格）：复位后先写方式控制字，再写命令控制字
│   ├─ 三种接收错误
│   │   ├─ PE（奇偶错）：数据位被干扰翻转，校验不符
│   │   ├─ OE（溢出错）：CPU未及时取走，新数据覆盖旧数据
│   │   └─ FE（帧错误）：停止位位置检测到低电平（帧不同步）
│   └─ 接收时钟16倍频原因：在起始位中点采样，抗干扰
│
└─ 第五层：三芯片协同操作范式（综合设计题骨架）
    ├─ 步骤一：读连接图确定各芯片端口基址（+偏移得A/B/C/控制口）
    ├─ 步骤二：按需求推导控制字（D7特征位 / SC位 / 方式位）
    ├─ 步骤三：初始化程序（写控制字 → 写参数/初值）
    └─ 步骤四：功能程序（循环读写数据端口，配合状态查询或中断）
```

**层间逻辑说明：** 第一层是三块芯片共同的存在理由，跳过它就会觉得控制字是"无意义的二进制"。第二到四层是三个方向上的具体解法，三者并列但不互斥——真实系统里往往同时用到。第五层不是新知识，而是把前三层的操作步骤提炼成一套通用范式：理解了这个范式，任何一道综合设计题都可以拆解成对每块芯片各走一遍同样的四步流程。

## 第一部分：先问一个问题——CPU为什么不能直接连外设？

想象你是一位语速极快的演讲者，台下坐着几百个听众：有的在用笔记本电脑打字（速度接近你），有的在手写笔记（慢得多），有的在看你的PPT（根本不接收语音）。你不可能同时对着所有人说话——你需要助手帮你把信息转换成不同格式、控制节奏、管理"谁准备好了"的状态。

CPU和外设的关系完全一样：
- CPU工作在几百MHz乃至GHz，键盘每秒只产生几个字节
- CPU用并行总线传8位数据，串行鼠标一次只传1位
- CPU给出高低电平，某些传感器给出的是模拟电压

**接口芯片就是那个"助手"**，它负责：速度适配、格式转换、状态报告、信号握手。

可编程接口的"可编程"意味着同一块芯片可以通过软件配置成不同角色——这就是为什么你需要先写"控制字"告诉它怎么工作，再去用它。

## 第二部分：8255A——并行I/O的"三口翻译官"

### 2.1 为什么需要三个口？

8255A有三个8位数据端口（A、B、C），不是为了"多多益善"，而是因为一个真实的接口任务通常同时需要：

- **一路输出**（比如向显示器发段码）
- **一路输入**（比如从键盘读按键）
- **一路握手**（用C口的某几根线告诉外设"我好了"/"你好了"）

三个口分工不同，C口尤其特殊：在方式0下它是普通8位I/O；在方式1/2下它的引脚会被"征用"作联络信号（STB、IBF、OBF、ACK、INTR），变成协议信号线而非数据线。

### 2.2 三种工作方式的演进逻辑

可以把三种方式理解为"信任程度"的递进：

```
方式0：基本输出/输入
│ "我不管外设准没准备好，我直接读写"
│ 适用场合：外设始终就绪（如LED灯、拨码开关）
│ 类比：在白板上直接写，不问别人看没看完
│
▼
方式1：选通输入/输出（带握手）
│ "外设告诉我它准备好了，我才传数据"
│ 输入：外设用STB通知8255A，8255A用IBF回应；数据稳定后INTR通知CPU
│ 输出：CPU写完数据后OBF通知外设；外设取走后ACK告知8255A；INTR再通知CPU
│ 类比：用对讲机通话——"听到请回答，over"
│
▼
方式2：双向选通（仅A口）
  "A口既能输入又能输出，同一组线双向复用"
  联络信号从C口挪出5根（STB、IBF、OBF、ACK、INTR）
  适用场合：打印机这类既需要发数据又需要回状态的外设
  类比：一条双向单车道，红绿灯控制方向
```

### 2.3 控制字的两种"身份"——靠D7区分

8255A只有一个控制端口地址，却接受两种完全不同的控制字。CPU怎么知道你发的是哪种？靠**D7位**：

```
D7 = 1  →  方式选择控制字（设置A/B/C口的工作方式和输入输出方向）
D7 = 0  →  C口置位/复位控制字（单独控制C口某一根线的高低电平）
```

这是一个很聪明的设计：用最高位作"类型标识符"，一个端口地址服务两套完全不同的功能，节省了地址空间。

**C口BSR（置位/复位）控制字的实际用途：** 在方式0下，如果你只想改变C口某一根引脚的电平（比如产生一个负脉冲），不必重新写整个C口字节——用BSR控制字只改那一位即可，其他位保持不变。

### 2.4 端口地址的推导——从硬件连接到软件地址

8255A有两根地址线A1、A0：

```
A1=0, A0=0 → A口（偏移0）
A1=0, A0=1 → B口（偏移1）
A1=1, A0=0 → C口（偏移2）
A1=1, A0=1 → 控制口（偏移3）
```

如果你看到题目给出"8255A的A口地址为60H"，那控制口必然是63H。这个规律在综合题里是快速判断端口地址的捷径。

## 第三部分：8253——给系统装一个"精准节拍器"

### 3.1 为什么软件延时不够用？

你可能会想：用一个循环不停执行NOP不就能计时了？问题在于：
- 软件循环占用CPU——CPU什么都做不了，只是在"干等"
- 计时精度随CPU负载浮动——不同速度的CPU跑同样的循环，时间不一样
- 无法产生稳定的外部信号——如果你想让外部电路每20ms收到一个脉冲，软件循环做不到

8253的解法是把计时工作"外包"给硬件：CPU只负责写入初值，之后8253自己在CLK脉冲驱动下倒计时，计到0时通过OUT引脚给出信号，完全不占用CPU。

```
CPU写入计数初值N
        │
        ▼
8253开始倒数：N → N-1 → N-2 → ... → 1 → 0
                                          │
                                          ▼
                                     OUT引脚变化（触发中断/产生脉冲/产生方波）
```

### 3.2 六种方式的本质区别——两个维度

记六种方式最高效的方法是先抓住两个维度，不要硬背：

**维度一：计数值用完之后，会不会自动重装？**
- 一次有效（方式0、4）：计到0就停，下次要用再写新初值
- 多次有效（方式1、5）：可以被再次触发，但不自动重装
- 重复有效（方式2、3）：自动重装，永远循环——**这两种用于产生连续周期信号**

**维度二：谁来启动计数？**
- 软件启动（方式0、4）：CPU写完初值就自动开始
- 硬件启动（方式1、5）：等GATE引脚出现上升沿才开始
- 软硬都行（方式2、3）：两种方式均可启动

交叉对应就得到六种方式的位置：

```
          软件启动      硬件启动      软/硬皆可
一次有效    方式0        方式4（注*）   ——
多次有效    ——          方式1、5       ——
重复有效    ——          ——            方式2、3

*注：方式4是软件启动一次有效，方式5是硬件启动多次有效（可重触发）
```

### 3.3 方式2 vs 方式3——最容易混淆的一对

两者都重复有效、都能产生连续波形，区别只在**输出波形的占空比**：

```
方式2（脉冲频率发生器）：

OUT:  ┌─────────────┐_┌─────────────┐_┌─────────────
      |    N-1个CLK  |1|    N-1个CLK  |1|
      └             └─┘             └─┘
      周期=N个CLK，低电平只有1个CLK宽（窄脉冲）

方式3（方波发生器）：

OUT:  ┌──────┐      ┌──────┐      ┌──────
      | N/2  |  N/2 | N/2  |  N/2 |
      └──────┘      └──────┘      └──────
      周期=N个CLK，高低各占一半（方波）
```

记忆口诀：**方式2出窄脉冲，方式3出方波。** 产生中断信号用方式2（窄脉冲干净利落），产生音频/时钟信号用方式3（方波）。

### 3.4 GATE的作用——三种情况要分开记

GATE不是简单的"开关"，它在不同方式下行为不同：

```
软件启动方式（0/2/3/4）：
  GATE=1 → 允许计数（正常工作）
  GATE=0 → 暂停计数（冻结当前计数值，不清零）
  GATE=1 → 继续计数（从冻结处恢复）

硬件启动方式（1/5）：
  GATE上升沿 → 触发/重新触发计数（计数值重装并开始）
  GATE=0/1的电平本身不影响已运行的计数过程
```

实际系统中，GATE常接+5V（始终允许），或接外部控制信号（做"暂停/继续"开关）。

### 3.5 计数初值为0的陷阱

❌ **常见误解：** 写0 = 什么都不做

✅ **实际含义：** 写0 = 最大计数值（二进制=65536，BCD=10000）

原因：8253是**减1计数**。写入0后，第一个CLK到来时 0-1 = 0FFFFh（溢出到65535），之后从65535一路减到0，共经历65536个CLK周期。所以0实际上是最大初值的编码。

## 第四部分：8251A——让数据"排队单行道"走远路

### 4.1 并行通信 vs 串行通信——为什么需要串行？

并行通信用8根线同时传8位，快但有代价：线多、成本高、长距离时线间干扰（串扰）严重。10米以上的连接，并行通信就开始不可靠了。

串行通信用1根线依次传每一位，慢但线少、可以走很远：

```
并行（短距离，8根线同时传）：
CPU ====D7D6D5D4D3D2D1D0====> 外设
         8根线，快但贵

串行（长距离，1根线依次传）：
CPU ===D0→D1→D2→...→D7====> 远端设备
         1根线，慢但省
```

8251A做的事就是：**把CPU给的并行8位数据，拆成一位一位的序列发出去；同时把收到的串行序列重新拼回8位交给CPU**。这叫做**并/串转换**和**串/并转换**。

### 4.2 帧格式——两端说话的"语法协议"

串行通信双方必须事先约定"一个字符怎么开始、怎么结束、中间有多少位"，否则接收方不知道从哪里切分。这个约定就是**帧格式**：

```
空闲线（高电平）→ 起始位（强制低）→ 数据位(低位先) → [校验位] → 停止位（高）→ 空闲/下一帧起始位

每个时间槽 = 1/波特率 秒
```

起始位是"发令枪"——接收方检测到线从高变低，就知道一帧开始了；停止位是"终点线"——确保两帧之间有足够间隔。

**为什么接收时钟要用16倍频？**

接收方不知道发送方什么时候精确开始，只能靠检测起始位的下降沿来同步。但下降沿检测本身有±1个接收时钟的误差。如果用1倍频（每位只有1个采样时钟），误差就是±100%，完全不可靠。用16倍频后，每一位有16个采样机会，接收方在检测到下降沿后等待8个时钟（到这一位的中点）再采样，误差降到±1/16，极大提升了可靠性。

### 4.3 三种错误——各有成因，各有对应

8251A能检测三种接收错误，每种错误背后都有清晰的物理原因：

```
PE（奇偶错）→ 数据传输过程中某一位被干扰翻转
             检验：接收到的数据+校验位，奇偶性与约定不符
             类比：快递包裹里的验证码对不上

OE（溢出错）→ CPU太慢，上一个字节还没取走，新字节已经到了
             检验：接收缓冲区还满，新数据把旧数据覆盖
             类比：邮箱已满，新信件压在旧信件上导致旧信件丢失

FE（帧错误）→ 停止位位置没检测到高电平（停止位应为高）
             检验：约定停止位时刻，线上是低电平
             类比：电话挂断信号没听到，以为对方还在说话
```

### 4.4 两种控制字的顺序——不能颠倒

8251A复位后，CPU必须按**固定顺序**发送控制字：

```
系统RESET
    │
    ▼
① 发送方式选择控制字（告诉8251A：异步/同步？波特率因子？数据位？校验？停止位？）
    │
    ▼（若异步，直接到③；若同步，还需发同步字符）
③ 发送命令控制字（告诉8251A：现在允许发送？允许接收？清除错误标志？）
    │
    ▼
④ 开始传输数据
```

两种控制字都写**同一个控制端口地址**，8251A靠接收顺序区分（复位后第一个写入的是方式控制字）。这和8255A靠D7区分两种控制字是不同的设计选择。

## 第五部分：三块芯片的共同操作范式

学到这里，你会发现一个规律：不管是8255A、8253还是8251A，操作它们的程序结构几乎相同：

```
第一步：确定端口地址（从硬件连接图读出基址，+偏移得各端口）
    │
    ▼
第二步：推导控制字（按芯片控制字格式，把需求翻译成二进制）
    │
    ▼
第三步：初始化程序（先写控制字到控制端口，再写参数/初值）
    │
    ▼
第四步：功能程序（循环读写数据端口，配合状态判断或中断）
```

这个"四步范式"就是第7章所有综合题的骨架。不同芯片的差异只在第二步（控制字格式不同）和第四步（数据读写时序不同），骨架是一样的。

## 第六部分：综合设计题里的协作关系

真实系统中三块芯片往往同时出现，各自扮演不同角色：

```
┌─────────────────────────────────────────────────┐
│                    完整I/O系统                    │
│                                                 │
│  键盘/开关 ──→ 8255A (A口输入) ──→ CPU处理       │
│                                                 │
│  LED/数码管 ←── 8255A (B口输出) ←── CPU控制      │
│                                                 │
│  8253 (定时) ──OUT0──→ 8259A (中断请求) ──→ CPU  │
│              ──OUT1──→ 扬声器（方波发声）         │
│                                                 │
│  串口设备 ←──→ 8251A ←──→ CPU（远程通信）        │
│                                                 │
│  时钟源 ──CLK──→ 8253（给计数器提供节拍）         │
└─────────────────────────────────────────────────┘
```

三块芯片的分工是正交的——8255A管"哪个方向传"，8253管"什么时候传"，8251A管"怎么走远路传"——正是这种分工让系统设计模块化：换一个外设，只需重新初始化对应的那块芯片。

## 第七部分：常见误解与直觉陷阱

**误解1：C口和A口B口"地位相同"**

不是的。C口是个"万金油"：在方式0下是普通I/O；在方式1下被拆成两半分别服务A组和B组的握手信号；在方式2下几乎全部被征用。C口还能用BSR控制字逐位置位/复位，这是A口B口做不到的。

**误解2：8253写入计数值后立刻开始计数**

不完全对。软件启动方式（0/2/3/4）下，写完初值且GATE=1才开始；硬件启动方式（1/5）下，写完初值后还要等GATE上升沿才启动。

**误解3：8251A的方式控制字和命令控制字可以随时互换顺序**

不行。8251A严格要求：复位后先写方式控制字，再写命令控制字。如果顺序错误，芯片行为不可预测。如果需要重新配置，必须先软件复位（命令控制字中IR位=1）再重新写方式控制字。

**误解4：波特率 = 每秒传送字节数**

不是。波特率是每秒传送的**位**数（包含起始位、停止位、校验位）。真正的字节吞吐量要除以每帧总位数。格式"9600,8,N,1"时，每帧10位，实际字节数=9600÷10=960字节/秒，不是9600字节/秒。

## 本章要带走的五件事

**第一件：接口芯片存在的根本原因是"阻抗不匹配"**——不只是电气阻抗，还有速度阻抗、格式阻抗、协议阻抗。三块芯片分别解决了并行I/O、定时控制、串行通信三类"阻抗"。

**第二件：8255A的C口不是普通口**——它在方式1/2下会变成握手协议线（STB/IBF/OBF/ACK），而且可以用BSR控制字逐位操作。理解C口的双重身份，方式1/2的握手逻辑就自然通了。

**第三件：8253的"重复有效"方式（2和3）是定时中断的标配**——方式2产生等间隔窄脉冲（作中断请求），方式3产生方波（作音频或时钟），两者都自动重装初值，不需CPU介入。

**第四件：波特率不等于字节吞吐量**——帧里有起始位和停止位的"开销"，实际字节数=波特率÷每帧总位数。考试算传输时间必用这个换算。

**第五件：三块芯片遵循同一套"四步范式"**——确定地址→推控制字→初始化→读写数据。综合设计题无论多复杂，本质都是对三块芯片各自走一遍这四步，然后在主程序里串联起来。
